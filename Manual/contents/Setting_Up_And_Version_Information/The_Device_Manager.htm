Der Geräte-Manager

Der Gerätemanager wird durch Klicken auf auf die Bleistiftschaltfläche innerhalb der Zielliste aufgerufen:

Wenn Sie auf diese Schaltfläche klicken, wird der Gerätemanager geöffnet, in dem Sie verschiedene Geräte für die Zielplattform platforms konfigurieren können, auf der Sie kompilieren und testen möchten (beachten Sie, dass die verfügbare platforms von Ihrer Lizenz abhängt und möglicherweise nicht alle Geräte sichtbar sind). Nachfolgend finden Sie Einzelheiten zur Konfiguration von Geräten für jede der Zielplattformen:

Android

Hier können Sie auf die Schaltfläche Neues Gerät hinzufügen klicken, um ein neues Android Gerät manuell zur Liste hinzuzufügen, oder Sie können auf die Schaltfläche Gerät erkennen klicken, damit GameMaker automatisch alle Geräte erkennt, die über eine Kabelverbindung angeschlossen sind. Du kannst auch ein Android Virtual Device einrichten, indem du auf die Schaltfläche AVD ausführen. Dadurch wird der Android Virtual Device Manager geöffnet, in dem du ein neues AVD erstellen kannst (siehe hier für weitere Informationen). Sobald du ein AVD erstellt hast, musst du es vom AVD-Manager aus starten und dann im GameMaker auf die Schaltfläche Gerät erkennen klicken und es sollte automatisch zur Geräteliste hinzugefügt werden:

Sobald Sie ein Gerät (physisch oder virtuell) hinzugefügt haben, können Sie dann die Verbindung mit der Schaltfläche Verbindung testen überprüfen oder das Gerät wieder aus der Liste entfernen, indem Sie auf die Schaltfläche Gerät entfernen klicken.


HTML5

Der HTML5-Gerätemanager dient dazu, verschiedene Browser zur Geräteliste hinzuzufügen, um sie bei der Ausführung anzusprechen. Für den Anfang sollten Sie auf die Schaltfläche Browser erkennen klicken, wodurch Geräte für jeden der installierten Browser auf Ihrem Rechner erstellt werden. Wenn Ihr Browser nicht aufgeführt ist, können Sie auch auf die Schaltfläche Neuen Browser hinzufügen klicken und dann dem Gerät einen Namen geben und GameMaker auf den Pfad für die ausführbare Datei des Browsers verweisen.

Die zusätzlich verfügbaren Felder können eingerichtet werden, um zu steuern, wie der Browser Ihre Projekte ausführt, wobei die folgenden Optionen zur Verfügung stehen:

  • Browser-Befehlszeilen-Argumente: Hier können Sie beliebige Befehlszeilenargumente arguments eingeben, die an den Browserpfad angehängt werden sollen. Eine Liste dieser Argumente finden Sie unter den folgenden Links zu den wichtigsten gängigen Browsern:
    • Chromium (Chrom)
    • Firefox
    • Internet Explorer
  • Web Server Adresse: Die Adresse des Webservers Servers (Standard ist 127.0.0.1). Möglicherweise müssen Sie dies je nach den Spezifikationen verschiedener APIs ändern, z. B. muss dies bei Facebook auf http://localhost gesetzt werden (und dann in der App-Konsole auf die Whitelist gesetzt werden).
  • Web Server Port: Legen Sie die Portnummer für das Web server (Standard ist 51264).
  • Erlaubte Web Server IPs / CIDR IP-Bereiche: Legen Sie die erlaubten Web server, Hostnamen und CIDR-IP-Bereiche nach Bedarf fest.

iOS

Bevor Sie Ihre iOS Geräte einrichten können, müssen Sie müssen Sie Ihr Mac-Gerät einrichten und die Verbindung testen (Einzelheiten siehe unten). Sobald dies geschehen ist, können Sie den Mac, auf den Sie zugreifen möchten, aus dem Dropdown-Menü oben im Fenster auswählen und dann auf die Schaltfläche Gerät erkennen klicken, damit GameMaker automatisch alle angeschlossenen iOS Geräte, einschließlich Simulatoren, findet. Sobald eine Verbindung besteht, werden die Gerätedetails in der Liste in diesem Fenster angezeigt, und du kannst den Gerätenamen bearbeiten, um ihn Sie können den Gerätenamen bearbeiten, um ihn benutzerfreundlicher zu gestalten (diese Änderung wirkt sich nur darauf aus, wie das Gerät im Target Manager dargestellt wird, und auf nichts anderes).

Bevor Sie Ihre tvOS Geräte einrichten können, müssen Sie müssen Sie Ihr Mac-Gerät einrichten und die Verbindung testen (siehe unten für Details). Sobald dies geschehen ist, können Sie den Mac, auf den Sie zugreifen möchten, aus dem Dropdown-Menü oben im Fenster auswählen und dann auf die Schaltfläche Gerät erkennen klicken, damit GameMaker automatisch alle angeschlossenen tvOS Geräte, einschließlich Simulatoren, findet. Sobald eine Verbindung besteht, werden die Gerätedetails in der Liste in diesem Fenster angezeigt, und du kannst den Gerätenamen bearbeiten, um ihn Sie können den Gerätenamen bearbeiten, um ihn benutzerfreundlicher zu gestalten (diese Änderung wirkt sich nur darauf aus, wie das Gerät im Target Manager dargestellt wird, und auf nichts anderes).


macOS

Um ein Mac-Gerät zum Manager hinzuzufügen, müssen Sie zunächst auf auf die Schaltfläche mit der Aufschrift Neues Gerät hinzufügen klicken. Dadurch wird das Gerät hinzugefügt und es werden einige leere Felder erstellt, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zuerst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und geben Sie dann den Hostnamen (oder IP) des Macs angeben. Anschließend müssen Sie den den Benutzernamen des Benutzers, mit dem Sie eine Verknüpfung herstellen wollen, und das Kennwort , das er verwendet hat. Wenn Sie möchten, können Sie auch den Installationsordner auf dem Mac in den Ort ändern, an dem GameMaker die Build-Dateien für Ihre Projekte speichern soll.

Wenn das alles erledigt ist, können Sie auf die Schaltfläche Verbindung testen klicken, um sicherzustellen, dass GameMaker mit dem Mac kommunizieren kann.

HINWEIS: Wenn Sie für iOS entwickeln, müssen Sie dies zuerst tun und eine funktionierende Verbindung zu dem Mac herstellen, den Sie für die Entwicklung verwenden möchten.

HINWEIS: Sie müssen aktiviert haben Remote Login auf dem Ziel-Mac aktiviert haben. Dies können Sie tun, indem Sie zu Einstellungen -> Freigabe und Aktivieren Sie Remote Login für den Benutzer, der sich anmelden kann.

Ubuntu (Linux)

Um ein Gerät Ubuntu (Linux) zum Manager hinzuzufügen, müssen Sie zunächst auf die Schaltfläche Neues Gerät hinzufügen klicken. Dadurch wird das Gerät hinzugefügt und es werden einige leere Felder erstellt, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zunächst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und dann den Hostnamen (oder die IP-Adresse) der Ubuntu Maschine (oder VM) angeben. Dann müssen Sie den Benutzernamen des Benutzers angeben, mit dem den Sie verknüpfen möchten, und das von ihm verwendete Kennwort . Sie müssen auch einen Installationsordner auf dem Ubuntu Rechner angeben, den Sie für die Installation Ihrer Projekte verwenden möchten. Wenn das alles erledigt ist, können Sie auf auf die Schaltfläche Verbindung testen klicken, um sicherzustellen, dass GameMaker mit dem Rechner Ubuntu kommunizieren kann.

Beachten Sie, dass die endgültige ausführbare Datei, die von GameMaker erstellt wird , von der Architektur des Zielgeräts abhängt, die entweder x64 oder ARM sein kann. Wenn Sie zum Beispiel einen Raspberry Pi 3 oder 4 anschließen, wird eine 32-Bit-x64-Datei erstellt ( ARM ), und wenn Sie einen Linux PC anschließen, wird es eine 64-Bit-x64-Datei sein.

Windows UWP

Bevor Sie ein Gerät für das Windows 10 UWP-Ziel hinzufügen können, müssen Sie den Entwicklermodus aktivieren und das Geräteportal auf dem Gerät einrichten. Wenn Sie dies nicht tun, kann GameMaker weder mit dem Gerät kommunizieren mit dem Gerät zu kommunizieren und Ihre UWP-Anwendungen ordnungsgemäß zu erstellen.

Der einfachste Weg, den Entwicklermodus zu aktivieren, ist, einfach " Entwicklermodus" in die Windows Suche einzugeben und dann auf den Eintrag " Einstellungen" zu klicken, der daraufhin angezeigt wird. Daraufhin wird das folgende Fenster geöffnet:

Sie müssen die Option Entwicklermodus auswählen (was möglicherweise erfordert, dass Windows einige zusätzliche Pakete herunterlädt und einige zusätzliche Pakete herunterladen und installieren muss), und wählen Sie dann die darunter liegende Option Geräteportal aktivieren. Über das Geräteportal kann GameMaker mit dem PC Windows 10 kommunizieren, und wenn es aktiviert ist, müssen Sie einen Benutzernamen und ein Passwort eingeben:

Beachten Sie, dass der Benutzername und das Kennwort für das Portal eindeutig sein müssen und nicht mit denen übereinstimmen dürfen, die von Ihrem Microsoft-Konto oder anderen Windows -Anmeldeinformationen verwendet werden. Wenn Sie diese Informationen eingegeben haben, klicken Sie auf die Schaltfläche Übernehmen , schließen Sie den Einstellungsmanager und kehren Sie zu GameMaker zurück.

Weitere Informationen über das Geräteportal finden Sie in diesem Microsoft-Artikel: Geräteportal für Desktop.

Sie können nun den Gerätemanager in GameMaker öffnen und ein Gerät (oder mehrere Geräte) für die Verwendung einrichten:

Um ein Windows 10 UWP-kompatibles Gerät zum Manager hinzuzufügen, müssen Sie zunächst auf die Schaltfläche Neues Gerät hinzufügen klicken. Dadurch wird das Gerät hinzugefügt und es werden einige leere Felder erstellt, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zuerst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und dann den Host angeben - dies ist die IP-Adresse des Windows 10 -Geräts im Netzwerk, das das Spiel hosten soll, und muss mit dem Suffix " :50443" versehen werden, um die Portnummer festzulegen. Danach fügen Sie den Benutzernamen und das Passwort hinzu, die Sie im Geräteportal eingerichtet haben (wie bereits erwähnt). Sie können die PIN leer lassen, da sie für UWP Windows 10 Desktop nicht erforderlich ist.

Für genauere Details zur Einrichtung der verschiedenen UWP-Ausgaben (Desktop, Xbox One) besuchen Sie bitte den YYG Helpdesk: UWP einrichten.

In allen Gerätefenstern gibt es am unteren Rand die Option " Geräte löschen", mit der alle angeschlossenen Geräte aus der Liste entfernt werden können.
