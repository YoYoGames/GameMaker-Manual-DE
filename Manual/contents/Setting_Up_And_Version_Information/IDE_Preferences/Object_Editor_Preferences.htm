Objekteditor-Einstellungen

In den Objekteditor-Voreinstellungen können Sie bestimmte Eigenschaften für das Objekteditor-Fenster festlegen. Der Hauptbereich enthält die folgende Option:

  • Klicken zum Öffnen der Ereignismethode: Diese Option steuert, ob Sie einen Doppelklick oder einen einfachen Klick auf ein Objektereignis ausführen müssen, um es zu öffnen. Standardmäßig ist dies auf Doppelklick eingestellt.
  • Vererbte Ereignisse alpha: Diese Option erlaubt es Ihnen, den Alphawert zu ändern, mit dem geerbte Ereignisse in der Ereignisliste des Objekts gezeichnet werden. Standardmäßig ist dieser Wert auf 0,5 gesetzt (Werte zwischen 0 und 1 sind empfehlenswert).
  • Format der Ereignisbeschriftung: Hier können Sie die verschiedenen möglichen Texte aktivieren oder deaktivieren, die auf die Ereignisbeschriftung in der Ereignisliste des Objekteditors angewendet werden können. Standardmäßig ist nur die Ereignisbeschreibung aktiviert.
  • Standard-Ereignisinhalt hinzufügen: Standardmäßig ist diese Option aktiviert. Sie fügt lediglich einen einfachen Kommentar in den code Editor für alle neuen Veranstaltungen ein, um Sie daran zu erinnern, dass Sie dort Ihre code hinzufügen können.
  • Standard-Ereignisinhalt: Hier können Sie die Kommentare bearbeiten, die standardmäßig im Fenster code für neue Ereignisse hinzugefügt werden.

Die folgende Unterkategorie existiert auch für den Objekteditor:

Dialoge bestätigen

Dieser Abschnitt befasst sich mit den Dialogen, die bei der Verwendung des Objekteditors angezeigt werden können, wobei die folgende Option zur Verfügung steht:

  • Automatische Reaktion auf die Konvertierung von GML Visual in Gamemaker Language: Sie können die rechte Maustaste innerhalb des GML Visual workspace verwenden, um die Option auszuwählen, alle GML Visual für dieses Ereignis in GML code umzuwandeln. Standardmäßig wird eine Warnmeldung angezeigt, in der Sie gefragt werden, ob Sie fortfahren möchten. Sie können dies jedoch auf "Okay" setzen, dann wird der Dialog vor der Konvertierung nicht angezeigt.
  • Automatische Reaktion auf das Überschreiten des Physik-Eckpunkt-Limits: Mit dieser Option können Sie die Popup-Meldung steuern, die angezeigt wird, wenn Sie die zulässige Eckpunktzahl im Physik-Formeditor überschreiten. Standardmäßig wird hier eine Warnmeldung angezeigt, aber Sie können diese Option auf "Okay" setzen, dann wird der Dialog nicht angezeigt.
  • Automatische Auswahl beim Löschen von Ereignissen: Mit dieser Option können Sie die Popup-Meldung steuern, die angezeigt wird, wenn Sie ein Ereignis in der Ereignisliste des Objekteditors löschen. Standardmäßig wird hier eine Warnmeldung angezeigt, aber Sie können diese Option auf "Okay" setzen, dann wird der Dialog nicht angezeigt.
