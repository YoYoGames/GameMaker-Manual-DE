Raum-Editor-Einstellungen

In den Raumeditor-Voreinstellungen können Sie festlegen, wie der Raumeditor aussehen und sich verhalten soll. Die allgemeinen Einstellungen sind wie folgt:

  • Startabstand für die Ausblendung der Kette (px): Legt den Abstand zum übergeordneten Fenster fest, ab dem das Kettenglied ausgeblendet werden soll. Standard ist 10px.
  • Kettenausblendungs-Stoppabstand (px): Legt den Abstand zum übergeordneten Fenster fest, in dem die Ausblendung des Kettenglieds enden soll. Standard ist 100px.
  • Kettenminimum Alpha: Legt den endgültigen Alphawert nach dem Ausblendungspunkt für den Kettenalphawert fest. Standardwert ist 0,3.
  • Standardmäßige Tiefenabstände der Ebenen: Wenn Sie mehrere Ebenen im Raumeditor erstellen, wird ihnen automatisch eine Tiefe zugewiesen, die sich nach ihrer Position in der Ebenenliste richtet. Diese Tiefe wird um den hier angegebenen Wert erhöht. Der Standardwert ist 100, d.h. jede Ebene hat die Tiefe der vorherigen Ebene + 100.
  • Standardraumbreite: Legt die anfängliche Breite fest, die als Standard für einen neuen Raum ausgewählt wird. Standard ist 1024px.
  • Standard-Raumhöhe: Legt die Anfangshöhe fest, die als Standard für einen neuen Raum ausgewählt wird. Standard ist 768px.
  • Standard-Raumebenen hinzufügen: Wenn du eine neue Raumressource erstellst, generiert GameMaker ein paar "Basis"-Ebenen für dich - eine Hintergrundebene und eine Instanzebene. Du kannst switch dieses Verhalten hier deaktivieren, so dass neue Räume keine vorgefertigten Ebenen haben werden. Diese Funktion ist standardmäßig aktiviert.
  • Raumansichten anzeigen: Wenn in einem Raum Ansichten aktiv sind, wird dies durch einen Umriss an der Stelle angezeigt, an der sich die Ansichtsgrenzen innerhalb des Raums befinden würden, und die Sichtbarkeit dieses Bereichs kann über die entsprechende Schaltfläche im Editor selbst umgeschaltet werden. Sie können jedoch festlegen, dass die Ansichtsgrenzen nicht sichtbar sind, wenn Sie einen Raum öffnen, indem Sie diese Option deaktivieren (die Schaltfläche kann immer noch verwendet werden, um sie wieder anzuzeigen). Der Standardwert ist "on", d. h. die Ansichtsgrenzen sind beim Öffnen eines Raums sichtbar.
  • Inspektor-Auswahlfarbe: Die Farbe, mit der das in der Inspektor-Multi-Auswahl-Liste ausgewählte Element hervorgehoben wird.
  • Andere Farboptionen: Diese beziehen sich auf die verschiedenen Farben, die in der gesamten Raumeditor-Oberfläche verwendet werden. Wenn Sie auf eines der Farbfelder doppelklicken, können Sie die Farbe für diese Einstellung bearbeiten.
  • Master-Lautstärke-Einstellung: Die Master-Lautstärkeeinstellung für die Vorschauwiedergabe im Raum-Editor, ein Wert zwischen 0 und 1. Standard ist 1.

Weitere Optionen für die Bearbeitung sind in den unten aufgeführten Unterkategorien verfügbar:

Dialoge bestätigen

Die Voreinstellungen für Bestätigungsdialoge dienen dazu, die Meldungen zu ändern, die im Raumeditor angezeigt werden, wenn Sie bestimmte Aktionen durchführen. Die Meldungen, die Sie ändern können, sind:

  • Automatische Reaktion auf die Aktualisierung von Ebenentiefen: Wenn Sie Ebenentiefen ändern, wird eine Meldung angezeigt, die Sie darauf hinweist, dass dadurch auch alle anderen Ebenentiefen automatisch aktualisiert werden. Mit dieser Option können Sie wählen, ob die Meldung angezeigt werden soll, ob die Ebenen aktualisiert werden sollen, als ob Sie auf "Ok" geklickt hätten, oder ob nichts aktualisiert werden soll, als ob Sie auf "Nein" geklickt hätten.
  • Automatische Reaktion auf Warnung bei gesperrter Ebene: Wenn Sie versuchen, eine Ebene im Raum zu löschen und diese zufällig gesperrt ist, wird standardmäßig eine Meldung angezeigt. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und Sie können die Ebene einfach nicht löschen.
  • Automatische Reaktion auf Warnung bei gesperrtem Objekt: Wenn Sie versuchen, ein gesperrtes Objekt zu löschen, wird Ihnen standardmäßig eine Meldung angezeigt. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und Sie können das Objekt einfach nicht löschen.
  • Automatische Reaktion auf Warnung bei gesperrter Ebene des Elementbesitzers: Wenn Sie versuchen, ein Element auf einer Ebene zu löschen und die Ebene ist gesperrt (auch wenn das Element nicht gesperrt ist), wird standardmäßig eine Meldung angezeigt, die Sie darüber informiert, dass dies nicht möglich ist. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und Sie können das Element einfach nicht löschen.
  • Erstellen einer neuen Elementebene beim Ziehen von Sequenzen/Sprites auf ungeeignete Ebenen: Wenn Sie versuchen, eine sprite (oder eine sequence) zu einer Ebene hinzuzufügen, die keine Vermögensebene ist, wird standardmäßig eine Meldung angezeigt, um Sie darüber zu informieren, dass dies nicht möglich ist, und Sie zu fragen, ob Sie eine neue Vermögensebene für das zu ziehende Element erstellen möchten. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und es wird automatisch eine neue Ebene für sprites (und sequences) erstellt.
  • Automatische Reaktion auf das Löschen von Elementen in übergeordneten Räumen: Hier können Sie wählen, was passiert, wenn Sie ein Element löschen, das Teil eines übergeordneten Raums ist (und somit in allen untergeordneten Räumen vorhanden ist). Standardmäßig wird eine Warnmeldung angezeigt, und Sie können entscheiden, was zu tun ist. Sie können jedoch die Option "Alle löschen" wählen, um die Meldung zu unterdrücken und das Element aus dem übergeordneten Raum und allen untergeordneten Räumen zu löschen, oder Sie können "Alle behalten" wählen, um die Meldung zu unterdrücken und das Löschen abzubrechen.
  • Automatische Reaktion auf das Löschen von Ebenen in übergeordneten Räumen: Hier können Sie wählen, was passiert, wenn Sie eine Ebene löschen, die Teil eines übergeordneten Raums ist (und somit in allen untergeordneten Räumen vorhanden ist). Standardmäßig wird eine Warnmeldung angezeigt, und Sie können entscheiden, was zu tun ist. Sie können jedoch die Option "Alle löschen" wählen, um die Meldung zu unterdrücken und mit dem Löschen der Ebene aus dem übergeordneten Raum und allen untergeordneten Räumen fortzufahren, oder Sie können "Alle behalten" wählen, um die Meldung zu unterdrücken und das Löschen abzubrechen.
  • Automatische Reaktion auf Fehler beim Ziehen eines ungültigen Assets: Wenn Sie versuchen, ein Asset zu einer Ebene hinzuzufügen, die nicht für diesen Asset-Typ vorgesehen ist (z. B. Hinzufügen eines Pfads zu einer Instanzebene), wird standardmäßig eine Meldung angezeigt, die Sie darüber informiert, dass dies nicht möglich ist. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und Sie können das Asset einfach nicht hinzufügen. Beachten Sie, dass diese Option nicht für Instanzen, sprites oder sequences gilt.
  • Erstellen einer neuen Instanzebene beim Ziehen von Objekten auf ungeeignete Ebenen: Wenn Sie versuchen, eine Objektinstanz zu einer Ebene hinzuzufügen, die keine Instanzebene ist, wird standardmäßig eine Meldung angezeigt, die Sie darüber informiert, dass dies nicht möglich ist und Sie fragt, ob Sie eine neue Instanzebene für das gezogene Objekt erstellen möchten. Wenn Sie diese Option jedoch auf "Ok" setzen, wird die Meldung nicht angezeigt und es wird automatisch ein neuer Layer für die Instanz erstellt.
  • Automatische Reaktion auf Fehler beim Ziehen einer ungültigen Ebenenauswahl: Hier können Sie festlegen, was passiert, wenn Sie eine Ressource in den Raum-Editor ziehen, obwohl Sie mehrere Ebenen im Ebenen-Editor ausgewählt haben (oder überhaupt keine Ebene ausgewählt haben). Standardmäßig wird eine Warnmeldung angezeigt, aber Sie können diese Option auf "OK" setzen, um die Meldung zu unterdrücken und fortzufahren, ohne eine Ressource hinzuzufügen.
  • Automatische Reaktion auf Fehler beim Ziehen einer gesperrten oder unsichtbaren Ebene: Hier können Sie festlegen, was passiert, wenn Sie eine Ressource auf eine Ebene ziehen, die gesperrt oder als unsichtbar gekennzeichnet ist. Standardmäßig wird eine Warnmeldung angezeigt, aber Sie können diese Option auf "OK" setzen, um die Meldung zu unterdrücken und fortzufahren, ohne eine Ressource hinzuzufügen.

Raster-Einstellungen

Mit den Rastereinstellungen können Sie die Funktionsweise des Raster-Overlays ändern:

  • Horizontaler Abstand des Gitters (px): Hier wird der horizontale Abstand für die Gürtelzellen des Raumgitters in Pixeln festgelegt. Der Standardwert ist 32px.
  • Vertikaler Abstand des Gitters (px): Hier wird der vertikale Abstand für die Gürtelzellen des Raumgitters in Pixeln festgelegt. Der Standardwert ist 32px.
  • Raster anzeigen: Diese Option schaltet das Raumraster ein oder aus, wenn Sie einen Raum zum ersten Mal öffnen. Sie ist standardmäßig eingeschaltet, aber auch wenn sie ausgeschaltet ist, können Sie sie über die entsprechende Schaltfläche im Raumeditor selbst wieder aktivieren.
  • Am Raster einrasten: Mit dieser Option können Sie festlegen, ob die Assets eines Raums standardmäßig am Raumraster einrasten oder nicht, wenn ein Raum zum ersten Mal geöffnet wird. Sie ist standardmäßig aktiviert, aber auch wenn sie ausgeschaltet ist, können Sie sie über die entsprechende Schaltfläche im Raumeditor selbst wieder aktivieren.
  • Gitterlinienfarbe: Mit dieser Option können Sie die Farbe (einschließlich der Alphakomponente) der Raumgitterlinien festlegen. Der Standardwert ist ( ARGB) #30FFFFFF.


Intelligente Leitfäden

In diesem Abschnitt werden die Smart Guides-Voreinstellungen für den Raum-Editor beschrieben. Die verfügbaren Optionen sind:

  • Smart Guides aktivieren: Hier können Sie die Verwendung von intelligenten Hilfslinien im Raumeditor aktivieren/deaktivieren. Die Voreinstellung ist aktiviert.
  • Lineale einschalten: Mit dieser Option werden die Seitenlineale im Raum-Editor aktiviert/deaktiviert. Standardmäßig ist sie aktiviert.
  • Lineale an Kanten einrasten: Wenn diese Option aktiviert ist, werden Hilfslinien, die von den Linealen in den Raumeditor gezogen werden, an allen im Raum platzierten Elementen ausgerichtet. Standardmäßig ist sie aktiviert.
  • Fangabstand: Diese Einstellung teilt dem Raumeditor mit, wie weit ein Guide von einem Asset entfernt sein muss, damit er an der Seite des Assets "einrastet". Der Standardwert ist 6px.
