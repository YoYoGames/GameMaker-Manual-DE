Texteditor Editor-Einstellungen

In den Texteditor-Einstellungen wird festgelegt, wie die verschiedenen Code-/Skript-Editoren aussehen und sich anfühlen. Die Hauptseite bietet die folgenden Optionen:

  • Verzögerung der Syntaxprüfung (ms): Die Verzögerungszeit (in Millisekunden), bevor der syntax Checker läuft. Der Standardwert ist 2000 ms.
  • Code Completion Delay (ms): Die Verzögerungszeit (in Millisekunden)
  • scripts im Vollbild-Editor öffnen: Wenn diese Option aktiviert ist, werden die Ressourcen von script automatisch in einem neuen workspace geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster innerhalb des aktuellen workspace geöffnet. Diese Option ist standardmäßig nicht aktiviert.
  • Shader im Vollbild-Editor öffnen: Wenn diese Option aktiviert ist, werden die Shader-Ressourcen automatisch in einem neuen workspace geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster innerhalb des aktuellen workspace geöffnet. Diese Option ist standardmäßig nicht aktiviert.
  • Objekt-Events im Vollbild-Editor öffnen: Wenn diese Option aktiviert ist, werden die Objekt-Ereignis-Editoren code automatisch in einem neuen workspace geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster geöffnet, das mit dem Objekt-Ereignis-Editor verbunden ist. Diese Option ist standardmäßig nicht aktiviert.
  • Ereignis scripts als Registerkarten im selben Fenster öffnen: Wenn Sie code oder GML Visual zu einem Ereignis hinzufügen, wird das Fenster code mit dem Ereignisfenster verkettet, und wenn diese Option aktiviert ist, werden alle weiteren Ereignisse, die zur Bearbeitung geöffnet werden, demselben Fenster als Registerkarten am oberen Rand hinzugefügt. Wenn Sie diese Option deaktivieren, wird jedes Ereignis als ein eigenes verkettetes Fenster geöffnet. Diese Option ist standardmäßig aktiviert.
  • Öffnen Sie verknüpfte scripts als Registerkarten im selben Fenster: Wenn Sie mehrere verknüpfte scripts haben und diese öffnen, werden sie standardmäßig im selben Fenster als Registerkarten gruppiert. Wenn Sie diese Option deaktivieren, werden sie jeweils in einem eigenen Fenster/Arbeitsbereich geöffnet. Diese Option ist standardmäßig aktiviert.
  • Texteditor bei Fokus in Ansicht schwenken: Wenn Sie einen Texteditor über ein Objektereignis öffnen, wird bei Auswahl dieser Option der workspace automatisch geschwenkt, um den Editor innerhalb der Grenzen zu platzieren. Diese Option ist standardmäßig deaktiviert.
  • Standardbreite des code Fensters: Legen Sie die Standardbreite (in Pixeln) fest, die das code Fenster haben soll, wenn es geöffnet wird. Der Standardwert ist 900.
  • Standardhöhe des code -Fensters: Legen Sie die Standardhöhe (in Pixeln) fest, die das Fenster code beim Öffnen haben soll. Der Standardwert ist 600.
  • Minimale code Fensterbreite: Legen Sie die minimale Breite (in Pixeln) fest, die das Fenster code haben darf. Der Standardwert ist 300.
  • Minimale code Fensterhöhe: Legen Sie die Mindesthöhe (in Pixeln) fest, die das Fenster code haben darf. Der Standardwert ist 200.

Neben den oben genannten Optionen gibt es zwei Unterkategorien, Code Editor und Visual Editor, die im Folgenden beschrieben werden.

Code-Editor

Mit den Farbeinstellungen können Sie festlegen, wie der Editor code und script verschiedene Texte anzeigen soll. Jeder Eintrag hat eine Reihe von Untereinträgen, mit denen Sie genau festlegen können, wie jeder einzelne Text aussehen soll, wie im Beispielbild oben gezeigt. Im Allgemeinen können Sie die Farbe, die Schriftart, die Größe und den Stil für jede der verschiedenen Optionen einstellen. Weiter unten können Sie auch eine allgemeinere Gestaltung für Dinge wie die Zeilenauswahl von code Klammern usw. festlegen.

Unter den Farbeinstellungen finden Sie die verschiedenen Styling-Optionen. Mit diesen Einstellungen wird das Verhalten des Code-/Skript-Editors festgelegt, wobei folgende Optionen zur Verfügung stehen:

  • Tabulatoren beibehalten: Wenn diese Option aktiviert ist, werden Tabulatortasten im Editor als Tabulatorzeichen - ' \t' - beibehalten, wenn sie nicht aktiviert ist, werden Tabulatoren in Leerzeichen umgewandelt. Die Voreinstellung ist aus.
  • Tabulatorgröße: Die Anzahl der Leerzeichen, um die die code durch Drücken von "Tab" eingerückt werden soll. Standardwert ist 4.
  • Whitespace anzeigen: Zeigt " ." an, um Leerzeichen anzuzeigen (siehe Abbildung unten). Standard ist aus.
  • Zeilennummern anzeigen: Ob die Zeilennummern unten links angezeigt werden sollen oder nicht. Standardmäßig ist diese Option aktiviert.
  • Tabulatorzeilen anzeigen: Zeigen Sie " ...." an, um den Tabulatorbereich anzuzeigen. Standard ist aus.
  • Aktivieren Sie die Farbkodierung: Aktivieren oder deaktivieren Sie die Farbcodierung für den Editor. Wenn Sie diese Option ausschalten, wird code nicht gestaltet, wenn Sie sie einschalten, wird die oben im Abschnitt Farbe beschriebene Formatierung verwendet. Die Voreinstellung ist on.
  • Dokumentzeilenlimit: Legt die maximale Anzahl von Zeilen für ein bestimmtes Editor-Dokument fest. Der Standardwert ist 0 (0 oder weniger ist praktisch unendlich), jeder andere Wert begrenzt die zulässige Zeilenzahl.
  • Einrückung bei neuen Zeilen anpassen: Aktivieren oder deaktivieren Sie die automatische Einrückung für Ihre code. Standardmäßig ist die automatische Einrückung aktiviert, so dass alle Tabulatoren in einer Zeile erhalten bleiben, wenn eine neue Zeile hinzugefügt wird.
  • Automatischer Einzug nach Klammern: Aktivieren oder deaktivieren Sie die automatische Einrückung für Ihre code beim Hinzufügen von Klammern {}. Standardmäßig ist diese Option aktiviert, so dass die nächste Zeile von code eingerückt wird, wenn eine neue Klammer { und eine neue Zeile hinzugefügt werden.
  • Vervollständigung von code aktivieren: Aktivieren oder deaktivieren Sie das code Vervollständigungsfenster. Wenn Sie code schreiben und dies aktiviert ist, erhalten Sie ein Popup-Fenster mit möglichen Funktionen, die Sie auf der Grundlage des aktuellen Eingabetextes verwenden können. Standardmäßig ist diese Option aktiviert.
  • Funktionsklammern automatisch hinzufügen: Aktivieren oder deaktivieren Sie das automatische Hinzufügen von Klammern (), wenn Sie die automatische Vervollständigung verwenden. Wenn Sie code schreiben und diese Funktion aktiviert ist, fügt IDE automatisch die beiden Klammern () hinzu, die für eine Funktion erforderlich sind, wobei der Cursor innerhalb der Klammern platziert wird, wenn die Funktion arguments benötigt, und der Cursor nach den Klammern platziert wird, wenn dies nicht der Fall ist. Diese Funktion funktioniert nur, wenn die Funktion in der Autovervollständigung ausgewählt wurde. Standardmäßig ist sie aktiviert.
  • Undo / Redo Stack Limit: Hier können Sie das Undo/Redo Stack Limit einstellen. Alles unter 0 ist im Grunde unendlich (begrenzt durch den verfügbaren Speicher), während positive Werte den Stapel auf diese Zahl begrenzen und jede Aktion am Ende des Stapels entfernen, wenn das Limit erreicht ist. Die Voreinstellung ist -1.
  • Auswahlübereinstimmungen hervorheben: Damit wird jeder Abschnitt von code hervorgehoben, der an anderer Stelle dupliziert wird. Wenn Sie also z. B. eine Funktion auswählen, werden alle Instanzen dieser Funktion im aktuellen Fenster code ebenfalls hervorgehoben. Standardmäßig ist diese Option aktiviert.
  • Farbkodierung/Codevervollständigung für veraltete Funktionen: Damit wird jeder Abschnitt von code hervorgehoben, der veraltet ist. Standardmäßig ist diese Option aktiviert.

Visueller Editor

Die Kommentareinstellungen steuern die verschiedenen Farb- und Formatierungsoptionen für Kommentare, die zu Ihrem GML Visual scripts hinzugefügt wurden. Diese Optionen sind:

  • Farbe: Die Standardfarbe für den Text, der in das Fenster geschrieben wird (Standard ARGB #FF53E270).
  • Name der Schriftart: Der Name der zu verwendenden Schriftart (die verfügbaren Schriftarten hängen von den auf Ihrem System installierten ab).
  • Stil: Der zu verwendende Schriftstil, z. B. fett oder kursiv (beachten Sie, dass nicht alle Fenster die gleichen Stile zulassen und dass dies auch von der verwendeten Schriftart abhängt).
  • Größe: Die Größe, auf die die Schrift für das angegebene Ausgabefenster eingestellt werden soll.
  • Anti-Aliasing: Hier können Sie den Anti-Aliasing-Modus für das Rendering der Schrift im Ausgabefenster festlegen. Sie können IDE entscheiden lassen, indem Sie Standard verwenden wählen, oder Sie können es ein- oder ausschalten , indem Sie eine der beiden Optionen wählen.
