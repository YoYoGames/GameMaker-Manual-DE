Plugin-Einstellungen

Die Plugin-Voreinstellungen werden verwendet, um alle zusätzlichen Plugins zu steuern, die dem GameMaker IDE hinzugefügt wurden. Standardmäßig ist hier ein Abschnitt dem Source Control Plugin gewidmet, mit dem GameMaker-Projekte in Git integriert werden können.

Quellenkontrolle (Git)

Dieser Abschnitt befasst sich mit den Einstellungen, die erforderlich sind, damit das Git SCM-Plugin mit GameMaker funktioniert.

  • Pfad der ausführbaren Datei von Git: Dies ist der Ort, an dem sich die ausführbare Datei Git befindet, die GameMaker für alle Versionskontrollbefehle verwendet. Wie in der obigen Abbildung zu sehen ist, befindet sich diese Datei im Ordner cmd Ihrer Git -Installation.

Darunter können Sie das Zusammenführungswerkzeug und das Diff-Werkzeug für die Versionskontrolle mit diesen Einstellungen einrichten(optional):

  • Zusammenführen (Speicherort des Tools): Hier geben Sie den vollständigen Dateipfad für das Zusammenführungs-Tool Git an.
  • Zusammenführen (Werkzeugoptionen): In diesem Feld können Sie zusätzliche Befehle hinzufügen, die bei Verwendung des Zusammenführungswerkzeugs ausgeführt werden sollen.
  • Diff (Werkzeugposition): Hier geben Sie den vollständigen Dateipfad für das Tool Git diff an.
  • Diff (Werkzeugoptionen): In diesem Feld können Sie zusätzliche Befehle hinzufügen, die bei Verwendung des Diff-Tools ausgeführt werden sollen.

Finally there are two more options that affect how Git is set up in new projects: 

  • Add skeleton .git defaults to new projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to new projects that you create. It is enabled by default.
  • Add skeleton .git defaults to imported projects: When this setting is enabled, GameMaker will add .gitignore and .gitattributes Files to projects that you import from a .yyz file. It is enabled by default.
