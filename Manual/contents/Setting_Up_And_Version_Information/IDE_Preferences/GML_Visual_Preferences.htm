GML Visuelle Präferenzen

Die GML Visuell Einstellungen werden verwendet, um zu definieren, wie der visuelle Editor aussieht und funktioniert. Die verschiedenen verfügbaren Optionen sind unten aufgelistet, zusammen mit einer Beschreibung, was jede einzelne bewirkt.

  • Standardposition derGML Visual-Toolbox: Standardmäßig befindet sich die GML Visual Toolbox (in der die verschiedenen Aktionsbibliotheken enthalten sind) auf der rechten Seite des Editors, aber Sie können diese Option verwenden, um sie auf der linken Seite zu platzieren.
  • GML Visual Knotenübersicht Sichtbarkeitsmodus: Eine "Knotenübersicht" ist der kurze beschreibende Text, der neben bestimmten Aktionen in der Aktion Workspace erscheint, z. B. diese Aktion "Instanz zerstören":

    Sie können wählen, ob diese immer, nur wenn die Aktion zugeklappt ist (Standardeinstellung) oder ganz deaktiviert werden sollen.
  • Automatische Reaktion auf die Konvertierung von GML Visual in GameMaker Language: Wenn Sie einGML Visuell Ereignis oder script in GML code konvertieren, erhalten Sie standardmäßig eine Meldung, die Sie vor dem Vorgang warnt und Sie fragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen, wird die Änderung einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Automatische Reaktion auf das Löschen von GML Visual-Knoten: Wenn Sie einen GML Visuell Ereignisknoten löschen, erhalten Sie standardmäßig eine Meldung, die Sie darüber informiert, was Sie vorhaben, und Sie fragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen, wird die Löschung einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Maximale Größe für GML Visual Toolbox Aktionsminiaturen: Am unteren Rand der Aktions-Toolbox befindet sich ein Schieberegler, mit dem Sie die Größe derGML-Ansicht Icons. Diese Einstellung wird verwendet, um die maximal zulässige Größe zu definieren (Standard 64px).
  • Mindestgröße für Aktionsminiaturen in der GML Visual Toolbox: Am unteren Rand der Aktions-Toolbox befindet sich ein Schieberegler, mit dem Sie die Größe derGML-Ansicht Icons. Diese Einstellung wird verwendet, um die minimal zulässige Größe zu definieren (Standard 16px).
  • Standardgröße für GML Visual Toolbox Aktionsminiaturen: Diese Einstellung wird verwendet, um die Standardgröße für alle Aktionssymbole in der Toolbox festzulegen (Standard 32px).
  • Maximale Anzahl der zuletzt verwendeten Aktionen, die angezeigt werden sollen: In der Aktions-Toolbox befindet sich oben eine Bibliothek mit den Aktionen, die Sie zuletzt verwendet haben. Mit dieser Option können Sie die maximale Anzahl der zuletzt verwendeten Aktionen ändern, die angezeigt werden (Standardwert 5).
  • Dicke der Kette, die GML Visual-Aktionen miteinander verbindet: Dies definiert die Dicke der Kettengrafik, die die verschiedenen Aktionen in einemGML Visuell code Block verbindet (Standard 3px).
  • Zeit, die die Maus benötigt, um mit dem Ziehen von Knoten zu beginnen (ms): Wenn Sie auf einen Knoten klicken und ihn ziehen, positionieren Sie den Knoten innerhalb der Aktion code neu. Wenn Sie jedoch die Maus einen Moment lang gedrückt halten und dann ziehen, verschieben Sie die Position des Knotens innerhalb der workspace, aber nicht innerhalb der Reihenfolge der code. Diese Einstellung legt die Anzahl der Millisekunden fest, die Sie bei gedrückter Maustaste warten müssen, bevor der Knoten verschoben (nicht neu positioniert) werden kann. Die Voreinstellung ist 350.
  • GML Visual Knotenübersicht ausblenden: Hier können Sie wählen, ob Sie das Übersichtsfenster des Action Code Editors ausblenden möchten. Standardmäßig ist diese Option deaktiviert. Wenn Sie sie aktivieren, wird die Klartextliste der Aktionsketten aus dem Editor entfernt.
