Dateiüberwachungs-Einstellungen

Der File Watcher ist die Bezeichnung für den Teil von IDE, der die Dateien eines Projekts auf Änderungen durch externe Software (z. B. Bearbeitung eines Bildes in einem externen Editor) oder durch interne Änderungen wie die Bearbeitung eines Skripts Vermögenswert. Im Allgemeinen sollten diese Einstellungen nicht bearbeitet werden müssen, aber auf einigen Systemen - z. B. solchen, die eine SSD als Speicherlaufwerk für die Projektdateien verwenden - kann der Datei-Überwacher Sie über Änderungen informieren, wenn Sie Dinge wie das einfache Speichern des Projekt speichern oder wenn Sie ein Projekt-Asset bearbeiten. In diesem Fall müssen Sie diese Einstellungen möglicherweise ändern.

Die verfügbaren Einstellungen sind:

  • Datei-Überwachung aktivieren: Hier können Sie das Datei-Überwachungssystem aktivieren oder deaktivieren. Wenn diese Funktion deaktiviert ist, erkennt IDE keine von externen Quellen geänderten Dateien mehr, daher sollte diese Funktion nur mit Vorsicht deaktiviert werden. diese Funktion. Der Datei-Überwacher ist standardmäßig aktiviert.
  • Stellen Sie die Toleranz des Dateiüberwachers in MS nach dem Speichern in der IDE ein: Hier wird die Zeit zwischen dem Speichern einer Datei und der Prüfung auf Änderungen durch den Dateiwächter eingestellt. Sie würden diesen Wert anpassen, wenn Sie Ihr Projekt speichern und der Datei-Überwacher Sie würden diesen Wert anpassen, wenn Sie Ihr Projekt speichern und der Dateiwächter fälschlicherweise anzeigt, dass sich etwas geändert hat. In diesem Fall würden Sie die Verzögerung erhöhen, bis die Meldung nicht mehr erscheint. Der Standardwert ist 2000ms, kann aber von 1ms bis zu 10.000ms reichen.
