Kompilieren von Präferenzen

Die Kompilierungseinstellungen betreffen bestimmte Optionen, die sich auf die Ausgabe beziehen, wenn Sie Ihr Spiel zum Testen kompilieren oder ein Paket erstellen, sowie bestimmte Bereiche des Kompilierungsprozesses. Die Optionen sind:

  • Automatisches Öffnen des syntax Fehlerfensters bei Erkennung: Aktivieren oder deaktivieren Sie die syntax Fehlermeldung beim Kompilieren. Diese Option ist standardmäßig deaktiviert.
  • DasKompilierfenster während der Kompilierung anzeigen: Standardmäßig wird das Kompilierfenster angezeigt, wenn Sie Ihre Projekte kompilieren, auch wenn das Kompilierdock geschlossen ist (es wird für die Kompilierung geöffnet). Wenn Sie dieses Kontrollkästchen deaktivieren, wird switch dieses Verhalten ausgeschaltet und das Kompilierfenster ist nur sichtbar, wenn es geöffnet ist.
  • Zeige compiler Befehle in der Kompilierausgabe: Diese Option ist standardmäßig aktiviert und zwingt compiler, alle relevanten Befehle im Kompilierfenster anzuzeigen. Wenn Sie diese Option deaktivieren, wird die Ausgabe kleiner und enthält weniger Informationen.
  • Ausführliche compiler Ausgabe anzeigen: Wenn Sie diese Option aktivieren, zeigt compiler zusätzliche Befehle im Zusammenhang mit dem Kompilierungsprozess an. Im Allgemeinen ist dies nur bei debugging erforderlich und kann vom Support angefordert werden, wenn Sie ein Problem haben. Standardmäßig ist sie ausgeschaltet (und dies ist auch die empfohlene Einstellung).
  • Subst für Laufwerkspfade verwenden: Wenn diese Option aktiviert ist, erstellt GameMaker virtuelle Laufwerke, die als Ersatz für vollständige Dateipfade dienen, und entfernt sie wieder, wenn sie nicht mehr benötigt werden. Diese virtuellen Laufwerke umgehen mögliche Probleme mit Dateipfaden, die für das Betriebssystem zu lang sind (insbesondere beim Kompilieren für Android oder bei der Verwendung von Erweiterungen), und es wird empfohlen, diese Option zu aktivieren. Wenn Sie jedoch Probleme damit haben, dass GameMaker virtuelle Laufwerke auf diese Weise erstellt, können Sie die Option deaktivieren. Standardmäßig ist diese Option aktiviert.
  • Paket nach Abschluss der Erstellung im Explorer anzeigen: Wenn diese Option aktiviert ist, öffnet GameMaker das Fenster des Betriebssystem-Explorers an dem Ort, an dem das endgültige ausführbare Paket erstellt wurde. Diese Option ist standardmäßig aktiviert.
  • Ausgabefenster beim Start der Kompilierung löschen: Standardmäßig löscht das compiler -Fenster alle vorherigen Meldungen aus dem compiler -Ausgabefenster zu Beginn jeder Kompilierung. Wenn Sie dies jedoch deaktivieren, bleiben die vorherigen compiler -Protokolle erhalten. Dies ist standardmäßig aktiviert und die empfohlene Konfiguration, da große compiler Protokolle die Kompilierzeiten verlangsamen können.
  • Automatisches Beenden der Ausführung vor dem Verpacken: Wenn Ihr Spiel bereits läuft und Sie versuchen, eine ausführbare Datei zu erstellen, wird standardmäßig eine Fehlermeldung angezeigt und Sie müssen das Spiel manuell beenden, bevor Sie eine ausführbare Datei erstellen ( ). Sie können diese Option aktivieren, um diese Fehlermeldung zu überspringen und das Spiel automatisch zu beenden, wenn es läuft, wenn "Ausführbare Datei erstellen" ausgewählt wird. Diese Option ist standardmäßig deaktiviert.
  • Maximale parallele Kompilierungsaufträge: Dieser Wert legt die maximale Anzahl der CPU-Kerne fest, die für die Kompilierung eines Projekts verwendet werden sollen. Der Standardwert ist 8, und im Allgemeinen sollten Sie diesen Wert nicht ändern müssen, aber wenn Sie mit einem Laptop oder einem weniger leistungsfähigen Rechner arbeiten, sollten Sie diesen Wert möglicherweise verringern. Die Standardeinstellung ist 8.
