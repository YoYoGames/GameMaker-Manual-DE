Pfadpräferenzen

Die Pfadvorgaben steuern die verschiedenen Dateipfade, die für Elemente in GameMaker IDE sowie für externe Editoren benötigt werden, die Sie benötigen. Die folgenden Optionen sind für die IDE verfügbar:

  • Temp-Verzeichnis: Der Ort, an dem alle temporären Verzeichnisse gespeichert werden. Standardmäßig ist dies auf Windows

    \Users\<Username>\AppData\Local\Temp\GameMaker

    Und auf macOS ist es:

    /var/folders/<Path_Hash>/GameMaker

  • IDE cache Verzeichnis: Der Speicherort für die IDE cache. Standardmäßig ist dies auf Windows

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    Und auf macOS ist es:

    /Users/<Username>/.config/GameMaker/Cache

  • Asset-Verzeichnis cache: Der Ort, an dem das Asset cache für jedes Projekt gespeichert wird. Standardmäßig ist dies auf Windows

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    Und auf macOS ist es:

    /Users/<Username>/.config/GameMaker/Cache

  • Speicherort "Meine Projekte": Der Ort, an dem GameMaker anfänglich neue Projekte erstellt. Standardmäßig ist dies auf Windows

    \Users\<Username>\Documents\GameMaker

    Und auf macOS ist es

    /Users/<Username>/GameMaker

  • Temp-Verzeichnis beim Schließen automatisch löschen: Wenn Sie diese Option aktivieren, löscht GameMaker automatisch den Temp-Ordner, den er für jedes Projekt zum Kompilieren usw. anlegt. Diese Einstellung ist standardmäßig aktiviert, und wenn Sie das Häkchen entfernen, wird switch deaktiviert (was bedeutet, dass Sie später alle Temp-Dateien manuell entfernen müssen).
  • Asset cache beim Beenden automatisch löschen: Wenn diese Option aktiviert ist, wird der Ordner compiler cache entfernt, wenn Sie GameMaker beenden. Diese Option ist standardmäßig ausgeschaltet, und wenn Sie sie aktivieren, bedeutet dies, dass bei jedem Laden und Starten eines Projekts der Ordner cache neu erstellt werden muss (was je nach Größe des Spiels einige Zeit dauern kann).
  • IDE cache beim Beenden automatisch löschen: Wenn diese Option aktiviert ist, wird der Ordner IDE compiler cache entfernt, wenn Sie GameMaker beenden. Diese Option ist standardmäßig deaktiviert, und wenn Sie sie aktivieren, bedeutet dies, dass bei jedem Start von GameMaker der Ordner cache neu erstellt werden muss.
  • Temp-Ordner löschen: Wenn Sie auf diese Schaltfläche klicken, wird der temporäre Ordner für das Projekt gelöscht.
  • Asset-Cache löschen: Wenn Sie auf diese Schaltfläche klicken, wird das Asset compiler cache für das Projekt gelöscht.
  • IDE Cache löschen: Wenn Sie auf diese Schaltfläche klicken, wird der Cache von IDE gelöscht.

Es gibt folgende Möglichkeiten, Pfade zu externen Editoren zu setzen:

  • Pfad zum externen Editor/Viewer für SWF-Dateien: Wenn Sie mit dem SWF-Format sprites arbeiten, können Sie hier den Pfad zu Ihrem bevorzugten Viewer/Editor angeben. Wenn Sie dann im Sprite Editor auf die Schaltfläche Bild bearbeiten klicken, wird das angegebene Programm geöffnet. Der Standardwert ist hier kein Pfad.
  • Pfad zum externen Editor/Betrachter für Spine Dateien: Wenn Sie mit dem Format Spine sprites arbeiten, können Sie hier den Pfad zu Ihrem bevorzugten Viewer/Editor angeben. Wenn Sie dann im Editor Sprite auf die Schaltfläche Bild bearbeiten klicken, wird das angegebene Programm geöffnet. Der Standardwert hier ist, dass kein Pfad angegeben wird.
