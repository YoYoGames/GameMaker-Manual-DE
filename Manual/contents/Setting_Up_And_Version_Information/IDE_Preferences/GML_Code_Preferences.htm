GML-Code-Einstellungen

Die GML-Code-Voreinstellungen werden verwendet, um zu definieren, wie einige der Sprachfunktionen für die Codierung in IDE von GameMaker angezeigt werden. Die Optionen für GML-Code sind wie folgt:

  • argument Typen in der Statusleiste anzeigen: Diese Option bezieht sich auf die Verwendung Ihrer eigenen benutzerdefinierten scripts und die Verwendung der JSDoc-Notation zur Identifizierung der verschiedenen arguments usw... (siehe hier für weitere Informationen). Wenn diese Option aktiviert ist und Sie die argument Typen in script angegeben haben, werden sie in der Statusleiste am unteren Rand des Editors angezeigt, eingeschlossen in {}. Diese Option ist standardmäßig ausgeschaltet.
  • argument Beschreibungen in der Statusleiste anzeigen: Wenn diese Option aktiviert ist, werden alle Beschreibungen, die Sie Ihren benutzerdefinierten scripts unter Verwendung der JSDoc-Notation gegeben haben, in der Statusleiste am unteren Rand des Editorfensters angezeigt. Diese Option ist standardmäßig ausgeschaltet.
  • US-Schreibweise für Systemfunktionen anzeigen: Standardmäßig verwendet GameMaker die englische Schreibweise für Funktionen, z. B. draw_text_colour(). Da dies jedoch möglicherweise nicht genau dem entspricht, was Sie gewohnt sind, können Sie mit dieser Option switch auf die US-amerikanische Schreibweise umstellen, so dass die Funktionen z. B. draw_text_color() werden. Diese Option ist standardmäßig ausgeschaltet.
  • Maximale Anzahl von syntax Fehlern, die angezeigt werden: Mit diesem Wert wird die maximale Anzahl von syntax Fehlern festgelegt, die gleichzeitig angezeigt werden. Wenn Sie feststellen, dass IDE zu langsam ist, können Sie versuchen, diesen Wert zu verringern. Dieser Wert ist standardmäßig auf 50 eingestellt.
  • Variablenreferenz-Warnungen einschalten: Wenn diese Option aktiviert ist, werden mögliche Probleme mit Variablen angezeigt, z. B. wenn eine Variable irgendwo definiert wurde, aber nicht in code verwendet wird. Wenn sie deaktiviert ist, werden diese Warnungen nicht mehr angezeigt. Standardmäßig ist diese Option aktiviert.
  • Aktivieren Sie das Falten von Anweisungen als Regionen: Wenn diese Option aktiviert ist, können die Klammern in code wie Regionen gefaltet werden. Dies ist standardmäßig aktiviert.
