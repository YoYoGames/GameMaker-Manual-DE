Drag And Drop™-Einstellungen

Mit den Drag and Drop™-Voreinstellungen können Sie festlegen, wie der Drag and Drop™-Editor script aussieht und funktioniert. Die verschiedenen verfügbaren Optionen werden im Folgenden zusammen mit einer Beschreibung ihrer jeweiligen Funktion aufgeführt.

  • DnD™-Werkzeugkasten Standardposition: Standardmäßig befindet sich die DnD™-Toolbox (in der die verschiedenen Aktionsbibliotheken enthalten sind) auf der rechten Seite des Editors, aber Sie können diese Option verwenden, um sie auf der linken Seite zu platzieren.
  • DnD™ Knotenübersicht Sichtbarkeitsmodus: Eine Knoten-"Übersicht" ist der kurze beschreibende Text, der neben bestimmten Aktionen in der Aktion Workspace angezeigt wird, z. B. diese Aktion "Instanz zerstören":

    Sie können wählen, ob diese immer aktiviert werden sollen, ob sie nur aktiviert werden sollen, wenn die Aktion eingeklappt ist (die Standardeinstellung), oder sie ganz deaktivieren.
  • Automatische Reaktion auf die Konvertierung von DnD™ in GameMaker-Sprache: Wenn Sie ein Ereignis von Drag and Drop oder script in GML code konvertieren, wird standardmäßig eine Meldung angezeigt, die Sie über den Vorgang informiert und Sie fragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen, wird die Änderung einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Automatische Reaktion auf das Löschen von DnD™-Knoten: Wenn Sie einen Drag and Drop Ereignisknoten löschen, erhalten Sie standardmäßig eine Warnung über das, was Sie gerade tun, und werden gefragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen setzen, wird der Löschvorgang einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Maximale Größe für DnD™-Toolbox-Aktionsminiaturen: Am unteren Rand der Aktions-Toolbox befindet sich ein Schieberegler, mit dem Sie die Größe der Symbole von Drag and Drop ändern können. Diese Einstellung wird verwendet, um die maximal zulässige Größe zu definieren (Standard 64px).
  • Mindestgröße für DnD™-Toolbox-Aktionsminiaturen: Am unteren Rand des Aktions-Werkzeugkastens befindet sich ein Schieberegler, mit dem Sie die Größe der Symbole von Drag and Drop ändern können. Diese Einstellung wird verwendet, um die minimal zulässige Größe zu definieren (Standard 16px).
  • Standardgröße für DnD™-Werkzeugkasten-Aktionsminiaturen: Diese Einstellung wird verwendet, um die Standardgröße für alle Aktionssymbole in der Toolbox zu definieren (Standard 32px).
  • Maximale Anzahl der zuletzt verwendeten Aktionen, die angezeigt werden sollen: In der Aktions-Toolbox befindet sich oben eine Bibliothek mit den Aktionen, die Sie zuletzt verwendet haben. Mit dieser Option können Sie die maximale Anzahl der zuletzt verwendeten Aktionen ändern, die die angezeigt werden (Standard: 5).
  • Dicke der Kette, die DnD™-Aktionen miteinander verbindet: Dies definiert die Dicke der Kettengrafik, die die verschiedenen Aktionen in einem DnD code Block verbindet (Standard 3px).
  • Zeit, die die Maus benötigt, um mit dem Ziehen von Knoten zu beginnen (ms): Wenn Sie auf einen Knoten klicken und ihn ziehen, positionieren Sie den Knoten innerhalb der Aktion code neu. Wenn Sie jedoch die Maus einen Moment lang gedrückt halten und dann ziehen, verschieben Sie die Position des Knotens innerhalb von workspace, aber nicht innerhalb der Reihenfolge von code. Diese Einstellung legt die Anzahl der Millisekunden fest, die bei gedrückter Maustaste gewartet werden muss, bevor der Knoten verschoben (nicht neu positioniert) werden kann. Die Voreinstellung ist 350.
  • DnD™-Knotenübersicht ausblenden: Hier können Sie wählen, ob Sie das Übersichtsfenster des Action Code Editors ausblenden möchten. Standardmäßig ist diese Option deaktiviert. Wenn Sie sie aktivieren, wird die Klartextliste der Aktionsketten aus dem Editor entfernt.

Es gibt auch ein Untermenü für DnD™-Kommentare:

Die Kommentareinstellungen steuern die verschiedenen Farb- und Formatierungsoptionen für alle Kommentare, die die zu Ihrer Drag and Drop™-Aktion scripts hinzugefügt wurden. Diese Optionen sind:

  • Farbe: Die Standardfarbe für den Text, der in das Fenster geschrieben wird (Standard ARGB #FF53E270).
  • Name der Schriftart: Der Name der zu verwendenden Schriftart (die verfügbaren Schriftarten hängen von den auf Ihrem System installierten ab).
  • Stil: Der zu verwendende Schriftstil, z. B. fett oder kursiv (beachten Sie, dass nicht alle Fenster die gleichen Stile zulassen und dass dies auch von der verwendeten Schriftart abhängt).
  • Größe: Die Größe, auf die die Schrift für das angegebene Ausgabefenster eingestellt werden soll.
  • Anti-Aliasing: Hier können Sie den Anti-Aliasing-Modus für die Darstellung der Schrift im Ausgabefenster festlegen. Sie können IDE entscheiden lassen, indem Sie Standard verwenden wählen, oder Sie können es ein- oder ausschalten , indem Sie eine der beiden Optionen wählen.
