Android-Einstellungen

Die Android-Einstellungen sind in verschiedene Abschnitte unterteilt, die im Folgenden aufgeführt sind:

Oben in den Einstellungen können Sie die maximale Größe des JDK-Speicher-Heaps festlegen (Standard ist 2 GB). Als Nächstes müssen Sie die verschiedenen Pfade einrichten, damit GameMaker weiß, wo er nach den verschiedenen Tools suchen muss, um das endgültige ausführbare Paket zu erstellen und Ihr Spiel zu testen. Dies ist in drei Abschnitte für das Android SDK, Android NDK und Java JDK unterteilt. Diese Pfade sollten automatisch für Sie ausgefüllt worden sein. Sollten Sie jedoch Probleme haben, können Sie auf die Schaltfläche auf der rechten Seite klicken, um den Datei-Explorer zu öffnen und die Speicherorte für jedes Element manuell auszuwählen. Einzelheiten zu den erforderlichen Versionen für die einzelnen SDK-Elemente finden Sie hier. Die Pfade sollten beim Anwenden validiert werden, und eine Meldung zeigt "Gefunden" an oder weist Sie auf Fehler hin (in diesem Fall sollten Sie alle Informationen überarbeiten, um sicherzustellen, dass sie korrekt sind).

Darunter finden Sie die verschiedenen Verpackungsoptionen, die da wären

  • APK immer vollständig installieren: Wenn Sie dies markieren, entfernt GameMaker alle früheren runners Ihres Spiels von Ihrem Gerät sowie alle zugehörigen Dateien (z. B. *.ini ) von Ihrem Testgerät, bevor eine neue Version des Spiels installiert wird.
  • Auf Paket installieren: Wenn diese Option aktiviert ist, wird beim Kompilieren eines endgültigen ausführbaren *.apk Pakets dieses auch automatisch auf dem aktuell ausgewählten Zielgerät installiert.

Schließlich kommen wir zum KeyStore. Dies ist eine Datei, die verwendet wird, um alle Ihre Android Apps zu "signieren". Füllen Sie also bitte alle Details korrekt aus und denken Sie daran, dass diese Datei benötigt wird, um alle Ihre Android-Apps in Zukunft zu erstellen und zu aktualisieren, was bedeutet, dass sie nach der Einrichtung von Es wird empfohlen, einen Screenshot der verwendeten Einstellungen zu machen und eine Sicherungskopie der endgültigen Datei zu erstellen.. Wenn Sie diese Datei verlieren, können Sie bestehende Spiele, die in den Store hochgeladen wurden, nicht mehr aktualisieren.

Um eine neue Keystore-Datei zu erstellen, sind die folgenden Angaben erforderlich:

  • Dateiname: Dies ist der Name der Keystore-Datei.

HINWEIS: Dies ist nicht Ihr Name oder der Firmenname, sondern der Name der zu erstellenden Datei! Wenn Sie bereits eine Keystore-Datei aus früheren Projekten haben, können Sie hier auf die Schaltfläche Dateibrowser klicken und GameMaker auf die zuvor erstellte Keystore-Datei verweisen (oder auf die Schaltfläche Importieren klicken). In diesem Fall musst du immer noch den Rest der erforderlichen Informationen ausfüllen (genauso wie bei der ersten Erstellung der Datei), aber du darfst nicht auf die Schaltfläche Keystore generieren ! klicken. Dadurch wird eine neue Keystore-Datei erzeugt, die die vorherige überschreibt.

  • Allgemeiner Name: Dies ist normalerweise Ihr Name.
  • Passwort: Ihr Sicherheitspasswort für die Keystore-Datei, das mindestens sechs Zeichen lang sein muss.
  • Alias: Dies ist ein weiterer Name, der für die Keystore-Datei verwendet werden kann und sich von dem oben eingegebenen Namen unterscheiden sollte.
  • Alias-Passwort: Ihr Alias-Sicherheitspasswort für die Keystore-Datei, das mindestens sechs Zeichen lang sein muss.

    HINWEIS Wenn Ihre Android Studio-Version 4.2 oder höher ist, müssen Sie für die Felder "Passwort" und "Alias-Passwort" das gleiche Passwort verwenden. Die Verwendung eines anderen Passworts und Alias-Passworts führt dazu, dass Ihre Builds fehlschlagen.
  • Organisatorische Einheit: Die Abteilung innerhalb des Unternehmens, der Sie angehören.
  • Organisation: Der Name Ihres Unternehmens.
  • Ort: Der Name der Stadt, in der Sie ansässig sind.
  • Land: Der zweistellige Standardcode für das Land, in dem Sie ansässig sind.

Wenn Sie mit den Angaben zufrieden sind, klicken Sie auf die Schaltfläche " Keystore für GameMaker generieren ", um die erforderliche Keystore-Datei zu erstellen. Dies kann einen Moment dauern, aber wenn es fertig ist, muss es normalerweise nicht mehr geändert werden. Beachten Sie, dass Sie auf die Schaltfläche Importieren klicken können, um eine zuvor erstellte Keystore-Datei zu importieren (siehe Abschnitt Dateiname oben), und dass Sie den SHA1-Hash des Keystore abrufen können, indem Sie auf die Schaltfläche Schlüsselhash generieren klicken. Wenn Sie auf diese Schaltfläche klicken, werden zwei spezielle Sicherheits-Hash-Schlüssel generiert, die direkt mit Ihrer Keystore-Datei verknüpft sind und von einigen Websites für die korrekte Funktion benötigt werden (z. B. Facebook oder Google Play für den Sha1-Hash). Sollten Sie dies benötigen, drücken Sie einfach auf die Schaltfläche, kopieren Sie die generierten Hash-Schlüssel und fügen Sie sie ein.
