Fernarbeitskräfte

Wenn Sie Ihr Spiel kompilieren, verwenden Sie normalerweise den "lokalen" Worker, d.h. den Computer, auf dem GameMaker installiert wurde. Unter bestimmten Umständen kann es jedoch notwendig sein, einen anderen Computer für die Erstellung Ihrer Projekte zu verwenden. Zum Beispiel, Wenn du auf einem Mac arbeitest und ein Windows Projekt erstellen möchtest, dann solltest du einen Remote Worker konfigurieren und einen Windows PC mit der Erstellung beauftragen. Wenn Sie einen Remote Worker verwenden, sendet GameMaker im Wesentlichen alle alle erforderlichen Dateien von dem (lokalen) Rechner, auf dem er läuft, an den Remote Worker-Rechner und verwendet dann diesen Rechner, um die endgültige ausführbare Datei für das Projekt zu erstellen (und sie entweder auf dem Remote-Rechner zu speichern oder sie anschließend an den lokalen Rechner zurückzugeben). Bevor Sie diese Funktion nutzen können, müssen Sie jedoch bestimmte Tools auf dem Build-Rechner installieren und einige Einstellungen vornehmen, die wir hier erläutern werden.

WICHTIG! Sie können einen Remote Worker zwischen einem lokalen Windows PC und einem entfernten Windows PC oder zwischen einem lokalen Mac PC und einem entfernten Windows PC einrichten, aber derzeit können Sie keinen lokalen Windows (oder Mac) PC und einen entfernten Mac PC einrichten, d.h.: Sie können keinen einen entfernten Mac-Arbeiter haben, sondern nur einen Windows.

Erstmalige Einrichtung

Um einen Remote Worker einzurichten, müssen Sie zunächst das Remote Worker Build Tool auf dem PC installieren, der verwendet werden soll. Sie finden dieses Tool, indem Sie auf dem lokalen Rechner das Build-Menü in GameMaker aufrufen und die Option "Show Remote Worker Installer" auswählen:

Alternativ können Sie auch auf das "Auge"-Symbol im Ziel-Manager klicken und den Remote Worker-Installationsprogramm auf diese Weise abrufen:

Das Installationsprogramm heißt in etwa "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe" und muss auf den PC kopiert werden, den Sie als Remote Worker verwenden wollen (auf dem Remote Worker muss GameMaker nicht installiert sein, nur dieses Programm).

Sobald Sie es auf den Build-PC kopiert haben, installieren Sie es, indem Sie den Anweisungen auf dem Bildschirm folgen. Wenn das Installationsprogramm fertig ist, werden Sie möglicherweise vom Betriebssystem aufgefordert, über die Firewall auf Ihr Netzwerk zuzugreifen. Sobald Sie den Zugang gewährt haben, öffnet das Installationsprogramm Sie müssen sich die IP-Adresse und die Portnummer notieren, die auf dieser Seite angezeigt werden, da Sie sie für den nächsten Schritt benötigen.

Sie müssen nun zu dem Rechner zurückkehren, auf dem Sie GameMaker 2 installiert haben, und klicken Sie auf die Schaltfläche Target Manager , um das Fenster Target Manager zu öffnen, und dort in den Bereich Worker gehen und auf die Schaltfläche Add Worker klicken. Daraufhin wird das folgende Fenster geöffnet folgende Fenster:

Hier müssen Sie Ihrem Remote Worker einen Namen geben (dieser wird in der (dieser Name wird in der Worker-Liste im Target Manager angezeigt) und fügen Sie dann im Abschnitt Hostname die Adresse und den Port hinzu, die Sie zuvor vom Worker-Rechner angezeigt und notiert haben.

Wenn alles richtig gelaufen ist, sollten Sie nun eine Meldung erhalten, dass Sie sich erfolgreich mit dem Remote Worker verbinden können. Sobald Sie Ihren Remote Worker installiert und mit Ihrem Hauptcomputer IDE verbunden haben, müssen Sie die lokalen Build-Einstellungen für jedes platform einrichten (siehe den Abschnitt unten für weitere Details), bevor Sie Ihr Projekt kompilieren können.

Kompilieren

Sobald Sie Ihren Remote Worker eingerichtet und die lokalen Build-Einstellungen vorgenommen haben (siehe Abschnitt oben), können Sie ihn für die Kompilierung Ihrer Spiele verwenden. Dazu müssen Sie lediglich den Remote Worker aus der Target Manager den Remote Worker aus der Target Manager-Liste auszuwählen, die platform zum Testen/Kompilieren sowie andere Optionen (wie Kompilierungstyp oder Konfiguration) zu wählen und dann auf Play oder Create Executable zu klicken.

HINWEIS: Die verfügbaren platforms im Ziel-Manager ändern sich je nach den Fähigkeiten des Fernarbeitercomputers, und nicht alle Ziele sind verfügbar. Beachten Sie auch, dass es für Remote Worker kein "Test"-Ziel gibt.

Wenn Sie das Projekt einfach nur abspielen, kompiliert der Remote Worker das Projekt und führt es aus. Wenn Sie jedoch eine ausführbare Datei erstellen, kompiliert Remote Worker das Projekt und gibt dann eine *.zip Datei an den lokalen Rechner zum Speichern zurück.

Es ist zu beachten, dass weder die Schaltfläche Debuggen noch die Schaltfläche Bereinigen verfügbar sind, wenn Sie einen Remote Worker verwenden. Beachten Sie auch, dass Sie derzeit nur einen Remote-Build zur gleichen Zeit durchführen können.


Sobald der Remote Worker auf dem entfernten Rechner installiert ist und läuft, erscheint er in der Taskleiste als Symbol, auf das Sie mit der rechten Maustaste klicken können, um die folgenden Optionen aufzurufen:

  • Öffnen: Öffnen Sie die Anwendung Remote Worker in Ihrem Standardbrowser.
  • Protokoll anzeigen: Öffnen Sie die Protokolldatei für den Remote Worker in Ihrem Browser.
  • Netzwerk-Einstellungen: Hier können Sie filtern, welche Rechner in der Lage sein sollen, Builds an den Remote Worker-Rechner zu senden. Jeder Filter kann ein CIDR-Filter oder ein Hostname eines Rechners sein, der zugelassen werden soll. Wenn Sie möchten, dass mehrere Rechner eine Verbindung zu einem Remote Worker herstellen, oder wenn Sie verschiedene Verbindungstypen verwenden (z. B. kabelgebundene und WLAN-Verbindungen), müssen Sie hier möglicherweise den zulässigen IP-Bereich bearbeiten.
  • Einstellungen bearbeiten: Öffnen Sie das Menü für die Fernarbeitereinstellungen (siehe Abschnitt "Einstellungen" oben).
  • Beenden: Beenden Sie die Remote Worker App. Dies bedeutet, dass Sie sich nicht mehr mit der App von einem lokalen Rechner aus verbinden (oder kompilieren) können, bis Sie die Remote Worker App neu starten.
