Dateien rückgängig machen

In den vorangegangenen Abschnitten haben Sie gesehen, wie man ein Repository erstellt und Dinge dorthin überträgt, aber was passiert, wenn Sie einen Fehler machen und zu einer früheren Übertragung zurückkehren wollen? Lassen Sie uns einen absichtlichen Fehler machen und sehen, wie wir dann damit umgehen können...

Erstellen Sie zunächst eine neue Ressource - in diesem Beispiel eine Ressource Script - und Sie werden zunächst kein Statussymbol sehen, da die Ressource für die Versionskontrolle noch nicht existiert. In der Ressource script fügen Sie eine einfache Zeile code ein, etwa so:

show_debug_message("Hello World");

Wenn Sie nun den Editor code schließen, wird die Datei script gespeichert und das geänderte Dateisymbol erscheint neben ihr im Asset-Browser. Wenn Sie nun das Fenster " Commit" öffnen, sehen Sie, dass die Änderungen in vier Stufen erfolgen, da neue Ressourcen automatisch hinzugefügt werden:

Die Änderungen werden wie folgt aussehen:

  • die Projektdatei .yyp
  • eine Ansichtsdatei
  • die GML script selbst
  • die Datei .yy, die darauf verweist

Sie sollten eine Commit-Nachricht eingeben und dann auf die Schaltfläche Commit klicken, so wie wir es zuvor erklärt haben. Jetzt gehen wir zurück zu unserem script und ändern die Zeile von code in (zum Beispiel):

show_debug_message("Hello World, how are you?");

Wenn wir den code -Editor schließen, wird wieder ein rotes Symbol neben script angezeigt, aber wir haben festgestellt, dass wir mit unserem code einen Fehler gemacht haben, und wir wollen zu einer früheren Version zurückgehen, um ihn zu beheben (natürlich könnten Sie einfach script öffnen und den Fehler beheben, da das Beispiel so einfach ist, aber in großen Projekten ist das normalerweise kein gangbarer Weg). Was wir tun müssen, ist unsere Änderungen rückgängig zu machen .

Um eine Änderung rückgängig zu machen, müssen wir zunächst das Übergabefenster erneut öffnen, also gehen wir noch einmal zu Versionsverwaltung > Änderungen übergeben. Unsere geänderte Datei wird wieder zum Übertragen bereitgestellt, aber das ist nicht das, was wir wollen, also klicken wir auf die Schaltfläche Unstage All, um sie aus dem Bereitstellungsbereich zu entfernen.

Wenn wir nun mit der rechten Maustaste auf die geänderte Datei klicken, wird ein Kontextmenü mit der Option " Pfad zurücksetzen" angezeigt:

Wenn Sie diese Option wählen, werden die Dateien in den vorherigen Zustand zurückversetzt, in dem sie sich befanden.

WICHTIG! Wenn Sie eine Rückgängigmachung durchführen, erhalten Sie eine Meldung, in der Sie aufgefordert werden, das aktuelle Projekt neu zu laden oder zu speichern, da die Dateiprüfung Änderungen auf der Festplatte festgestellt hat:
. Sie müssen auf "Neu laden" und nicht auf "Speichern" klicken, da beim Speichern der Speicherinhalt gespeichert wird, anstatt die rückgängig gemachten Dateien neu zu laden.

Das ist großartig, wenn Sie etwas bearbeiten und zurückgehen wollen, ohne bereits einen Commit durchgeführt zu haben, aber was ist, wenn Sie eine oder mehrere Änderungen vorgenommen haben und zu einem früheren Commit zurückgehen wollen? Nun, lassen Sie uns unser script noch einmal bearbeiten, um zu sagen:

show_debug_message("This is a bad idea");

Jetzt übertragen wir es wie zuvor, um es unter die Versionskontrolle zu bekommen. Das war also eine schlechte Idee und wir wollen es rückgängig machen, was bedeutet, dass wir zunächst das Fenster Versionsverwaltung > Verlauf anzeigen öffnen müssen. In diesem Fenster sehen wir nun eine Liste aller bisherigen Commits, beginnend mit dem ersten Commit, dem Commit für die script und dann dem Commit für die script, die wir als "schlechte Idee" bearbeitet haben:

Wir haben nun zwei Möglichkeiten. Wir können eine ganze Revision rückgängig machen - dann werden alle Dateien, die geändert wurden, auf die des gewählten Commits zurückgesetzt - oder wir können einen Pfad zurücksetzen - dann wird nur eine einzelne Datei zurückgesetzt. Beginnen wir mit der ersten Option:

  • Klicken Sie mit der rechten Maustaste unseren zweiten Commit im Commit-Verlaufsfenster (den "Hello World"-Commit).
  • Wählen Sie " Zu dieser Revision zurückkehren" und dann " Neu laden" in der Warnmeldung.

Sie sollten sehen, dass script jetzt ein rotes Statussymbol hat, und wenn wir es öffnen, sehen wir Folgendes:

Wir können dann einen Commit für das Projekt durchführen und sind wieder da, wo wir waren. Wir können aber auch diese Aktion für die Datei rückgängig machen und unsere "schlechte Idee" zurückholen, indem wir folgendes tun:

  • Öffnen Sie das Fenster Änderungen festschreiben.
  • Deaktivieren Sie die script mit dem "Hello World" code darin.
  • Klicken Sie mit der rechten Maustaste und Pfad umkehren.

Jetzt haben wir wieder die fehlerhafte Datei im Projekt! Wir können diese Gelegenheit nutzen, um nur den Dateipfad wiederherzustellen, anstatt alles auf eine frühere Revision zurückzusetzen. Dazu müssen wir folgendes tun:

  • Öffnen Sie das Fenster Verlauf anzeigen.
  • Klicken Sie auf die zweite Übertragung (die "Hello World"-Übertragung).
  • Klicken Sie mit der rechten Maustaste auf unsere script GML im Dateifenster und wählen Sie Pfad wiederherstellen.

Wir sollten wieder unser "Hello World" code haben und wir können dann eine neue Übergabe durchführen, um die Versionskontrolle mit der korrigierten Datei zu aktualisieren.
