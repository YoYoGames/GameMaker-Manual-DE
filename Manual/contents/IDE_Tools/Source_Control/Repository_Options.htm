Repository-Optionen

Nachdem Sie die ausführbare Datei Git eingerichtet und die Versionskontrolle für Ihr Projekt aktiviert haben, müssen Sie nun den GameMaker IDE mit einem Repository verknüpfen, was Sie auf zwei Arten tun können:

  • Ein Projekt-Repository erstellen
  • Klonen eines vorhandenen Repositorys

Im ersten Fall, dem Anlegen eines Projekt-Repositorys, wird ein Git Repository direkt dort angelegt, wo sich Ihr Projekt befindet. Im Sprachgebrauch von Git ist dies ein "Nicht-Bare-Repository", so dass Push/Pull in diesem Fall keinen Sinn macht, aber Sie können trotzdem Änderungen am Projekt festschreiben und nachverfolgen, sowie Änderungen und Revisionen zurücknehmen, wenn es nötig ist. Um dies einzurichten, müssen Sie lediglich im Hauptmenü auf Versionsverwaltung > Projekt-Repository erstellen klicken.

Als praktische Funktion können Sie auch ein externes Repository über IDEklonen . Auch hier benötigen Sie den Benutzernamen und das Passwort für das Repository, mit dem Sie sich verbinden wollen, in Ihren Identitätseinstellungen, und dann müssen Sie nur noch auf Versionskontrolle > Repository klonen klicken. Geben Sie die URL ein (verwenden Sie die HTTPS-Variante anstelle von SSH) und sagen Sie, wohin die Verbindung gehen soll:

Sobald der Klonvorgang abgeschlossen ist, öffnet sich automatisch ein Dateibrowser, in dem Sie das Projekt öffnen können, wenn Sie dies wünschen.
