Konflikte

In diesem Abschnitt werden wir uns mit dem Auflösen von Konflikten zwischen einem geklonten lokalen Repository und einem Haupt-Repository beschäftigen. Wir gehen davon aus, dass Sie jetzt ein lokales Repository verwenden. Wir müssen eine Datei bearbeiten und dann eine Art von Konflikt verursachen, also öffnen Sie den Test script, den wir verwendet haben, und ändern Sie die "Hello World"-Nachricht in etwas wie dieses:

show_debug_message("Hello World, How Are You?");

Sie sollten nun das Projekt speichern und es in das lokale Repository übertragen. Da wir nun mit einem entfernten Master-Repository verbunden sind (auch wenn wir es auf dieselbe Festplatte geklont haben), müssen wir einen Push durchführen, um die Änderungen in das Master-Repository zu übertragen. Gehen Sie dazu auf Versorgungskontrolle > Änderungen pushen, was uns eine Statusaktualisierung in der Registerkarte Versorgungskontrollausgabe gibt:

Wir müssen nun unser Originalprojekt öffnen (nicht das geklonte), also suchen Sie nach dem Ort, an dem Sie es auf der Festplatte gespeichert haben - oder schauen Sie einfach unter " Letzte Projekte" im Menü "Datei" nach, da es direkt unter dem aktuellen Projekt stehen sollte - und laden Sie es in GameMaker. Ein einfacher Weg, um zu überprüfen, ob Sie das richtige Projekt haben, ist, einfach die script zu überprüfen, und wenn die Nachricht "Hello World" lautet, dann haben Sie das richtige Projekt, oder Sie können das Fenster " View History" öffnen und auf diese Weise überprüfen.

Sie müssen nun die script erneut bearbeiten, so dass sie etwa so lautet:

show_debug_message("This will cause a conflict.");

Dies kann auf der Festplatte gespeichert werden, und die Änderungen werden festgeschrieben, aber wenn Sie versuchen, diese Änderungen zu übertragen, erhalten Sie die folgende Meldung:

Jemand hat bereits Änderungen veröffentlicht, die Sie noch nicht haben! Das bedeutet, dass Sie einen Pull auf das Repository durchführen müssen, um herauszufinden, was genau das Problem ist, also gehen Sie zu Versionsverwaltung > Änderungen ziehen. Daraufhin werden zwei Fenster angezeigt: Das erste ist eine Warnung über geänderte Dateien (wie bei den vorherigen Aktionen sollten Sie hier Neu laden und nicht Speichern wählen), das zweite ist das Fenster Konflikte :

In diesem Fenster werden die konfliktbehafteten Dateien in einer Liste auf der linken Seite angezeigt, und Sie können die Dateien aus dieser Liste mit der linken Maustaste (oder mehrere Dateien mit / oder zusammen mit ) auswählen. Über die Schaltflächen auf der rechten Seite haben Sie vier Möglichkeiten, mit den konfliktbehafteten Dateien umzugehen:

  • Use Theirs - Damit werden die von Ihnen vorgenommenen Änderungen mit denen aus dem entfernten Repository überschrieben.
  • Meine verwenden - Dies überschreibt die Änderungen aus dem entfernten Repository mit denen, die Sie gerade vorgenommen haben.
  • Zusammenführen - Damit wird versucht, ein externes Zusammenführungswerkzeug zu öffnen, um den Konflikt zu bearbeiten.
  • Wählen Sie Merge Tool - Damit können Sie das Merge Tool konfigurieren (weitere Informationen finden Sie im Abschnitt über externe Merge/Diff Tools ).

Sie können nun die Schaltflächen verwenden, um den Konflikt zu lösen, aber bevor Sie das tun, sollten Sie die konfliktbehaftete Datei öffnen, um genau zu sehen, was das Problem ist. Wenn Sie die konfliktbehaftete Datei script öffnen, sieht sie jetzt so aus:

Dies zeigt, dass sich "This Will Cause A Conflict" im HEAD befindet (das ist das, was wir in das lokale Repository übertragen haben) und dass "Hello World, How Are You?" vom Master, dem entfernten Repository, gezogen wird. Sie können nun zum Konfliktfenster zurückgehen und auf Use Theirs klicken, um die geänderte Datei zu ziehen und die aktuelle HEAD-Datei zu überschreiben. Die script wird nun als geändert gekennzeichnet und Sie sollten einen neuen Commit und Push durchführen, bevor Sie fortfahren.

Beachten Sie, dass wir dies im GameMaker IDE beheben können. Wenn wir die Zeilen 1, 2, 3 und 5 entfernen, so dass nur noch show_debug_message("Hello World, How Are You?") übrig bleibt, und es speichern, wird der Konflikt behoben, da Git davon ausgeht, dass Sie wissen, was Sie mit Ihrer eigenen Datei tun. Da dies jedoch fehleranfällig ist, empfiehlt es sich, ein Zusammenführungswerkzeug zu installieren, zu konfigurieren und zu verwenden, anstatt manuell zu versuchen, die Dateien zusammenzuführen.
