Quellenkontrolle

Source Control Management (SCM) ist die Bezeichnung für die Methode, mit Unterversionen oder Backups Ihrer Projekte über ein Repository und lokale Quellen zu arbeiten. Im Grunde genommen ist eine SCM-Lösung ein unabhängiges Softwarepaket, das alle Aspekte der Pflege, der Änderung und des Vergleichs von Versionen Ihres Projekts während der Arbeit daran kontrolliert. Dies ist besonders nützlich für diejenigen, die in einem Team arbeiten und in der Lage sein müssen, zu kontrollieren, wer was tut, und sich keine Sorgen über den Verlust von Daten oder über Änderungen machen müssen, die zu einem späteren Zeitpunkt wieder rückgängig gemacht werden müssen, aber auch Einzelpersonen können von diesem leistungsstarken und flexiblen System profitieren.

GameMaker verwendet Git für sein SCM, das separat installiert und eingerichtet werden muss, bevor SCM innerhalb von GameMaker verwendet werden kann. Siehe die Git-Dokumentation für weitere Informationen zu Git.

Nachfolgend finden Sie Links zu einer kleinen Anleitung, wie Sie dieses Plugin einrichten und die SCM-Tools mit einem Projekt verwenden können.

GameMaker zeigt in der Menüleiste ein Dropdown-Menü für die Versionskontrolle an, das mit den folgenden Optionen gefüllt wird, sobald die Versionskontrolle für Ihr Projekt aktiviert wurde (siehe den Abschnitt Einrichten des Git Plugins weiter oben für Details):

Im Folgenden werden die verfügbaren Optionen kurz erläutert (die meisten davon werden in den obigen Abschnitten des Tutorials näher erläutert):

  • Projekt-Repository erstellen: Mit dieser Option können Sie ein lokales Repository in demselben Verzeichnis wie Ihre Projektdateien erstellen.
  • Repository klonen: Mit dieser Option können Sie ein Repository von einer Quelle zu einem neuen Ziel klonen.
  • Änderungen festschreiben: Mit dieser Option können Sie geänderte Dateien in Ihrem Projekt bereitstellen und dann an das Repository übertragen.
  • Änderungen pushen: Nach einem Commit oder Merge werden mit dieser Option die Änderungen in das Master-Repository verschoben.
  • Änderungen ziehen: Mit dieser Option können Sie das lokale Repository aktualisieren, indem Sie die geänderten Dateien aus dem Haupt-Repository ziehen.
  • Verlauf anzeigen: Dadurch wird das Verlaufsfenster geöffnet, in dem Sie den gesamten Versionsverlauf des Projekts einsehen und bestimmte Dateipfade oder ganze Revisionen zurücksetzen können.
  • Konflikte anzeigen: Dies öffnet das Konfliktfenster und listet alle im Projekt vorhandenen konfliktbehafteten Dateien auf, so dass Sie sie entweder über GameMaker IDE oder mit einem speziellen Zusammenführungswerkzeug bearbeiten können.
