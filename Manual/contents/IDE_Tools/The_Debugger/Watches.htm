Uhren

Die ganze Idee des Debuggers ist es, Ihnen zu ermöglichen, jedes Detail Ihres Spiels zu "beobachten". Dies geschieht mit Hilfe von "Beobachtungsfenstern", die standardmäßig im Ausgabefenster am unteren Rand des Fensters angedockt sind. am unteren Rand von IDE geöffnet sind. Sie können auf die verschiedenen Registerkarten klicken, um die darin enthaltenen Beobachtungen zu sehen, von denen einige in Echtzeit aktualisiert werden können (wenn die Echtzeitfunktion debugging in der Debugger-Symbolleiste aktiviert ist), während andere erfordern, dass der Debugger das Spiel pausiert hat.

Wenn Sie Variablen in einem der Fenster beobachten, kann es sich um einen von mehreren Datentypen handeln. Aufgrund der Art und Weise, wie GameMaker bestimmte Dinge speichert, weiß der Debugger jedoch nicht immer genau, was die Variable enthält. Das liegt daran, dass die internen ID-Werte für Datenstrukturen oder Instanzen Integer-Werte sind und der Debugger daher nicht immer weiß, ob der Integer-Wert, der in einer Variablen gespeichert wird, für eine Datenstruktur oder eine Instanz steht oder einfach ein Integer-Wert ist. Aus diesem Grund können Sie mit der rechten Maustaste auf einen beliebigen Variablenwert klicken und erhalten ein Popup-Fenster, das die verfügbaren Datentypen auflistet für die diese Variable stehen könnte:

Strings und Arrays erlauben diese Option nicht, und es ist erwähnenswert, dass selbst wenn eine Datenstruktur in einer Instanzvariablen gespeichert ist, ihre ID-Nummer global ist. Wenn Sie also vier Instanzen haben und jede von ihnen eine ds_list erzeugt (zum Beispiel), hat jede Liste eine ID von 0 bis 3, und sie werden nicht alle 0 sein.

Sobald Sie Ihren Datentyp ausgewählt haben, zeigt die Variable die Einträge in der Struktur an, indem Sie einfach auf das kleine neben ihrem Namen klicken. Dadurch wird der Inhalt der Struktur und Sie können auch einige der Werte für die angezeigten Variablen ändern, indem Sie einfach auf den Wert doppelklicken und ihm einen neuen Wert geben.

HINWEIS: Sie können Arrays oder Datenstrukturen auf diese Weise nicht neu zuordnen, aber Sie können einige ihrer Inhalte ändern (Sie können weder die "Schlüssel"-Werte in Maps noch den Prioritätswert für Prioritätswarteschlangen ändern).

Nachfolgend finden Sie eine Liste der verschiedenen Überwachungsfenster mit einer Erläuterung ihrer Funktionen und ihrer Verwendung:

Einheimische

Das Fenster " Locals" zeigt die aktuellen Schritteigenschaften an, darunter den Namen des aktuellen Ereignisses (oder script), das ausgeführt wird, den Offset in der Virtuellen Maschine (VM) code für den aktuellen script (das Akronym "PC" steht für "Programmzähler"), die "Selbst"-Instanz, die den -Block ausführt den code Block ausführt, und die "andere" Instanz in dem Ereignis. Der letzte Teil ist nur dann gültig, wenn das Ereignis ein Kollisionsereignis ist oder ein "with" im code verwendet wird. Wenn die Instanz in Kollision ist, Wenn die Instanz kollidiert, zeigt "other" die andere Instanz (und ihre Variablen) in der Kollision an, aber wenn es sich um eine with handelt, dann wird die Instanz "Self" diejenige, die die code ausführt, während "Other" dann die Instanz ist, die die enthält. die tatsächlich die with enthält. Zu allen anderen Zeiten zeigt "other" einfach die gleichen Daten wie "Self".

Beachten Sie, dass die Instanzen "Self" und "Other" durch Anklicken von erweitert werden können, um eine eine Liste aller lokalen und Instanzvariablen zu sehen, die derzeit für den code Block deklariert sind, der gerade durchlaufen wird. Wenn Sie mit der rechten Maustaste auf auf einen Wert klicken, können Sie seinen Typ festlegen, um Details zu Datenstrukturen anzuzeigen.


Globales

Das Fenster Globals zeigt eine Liste mit aller derzeit deklarierten globalen Variablen und ihrer Werte. Wenn Sie mit der rechten Maustaste auf einen Wert klicken, können Sie seinen Typ festlegen, um Details zu den Datenstrukturen anzuzeigen.


Uhren

In diesem Fenster können Sie eine bestimmte Variable hinzufügen, die die Sie im Auge behalten wollen. Dies kann eine globale oder Instanzvariable , eine eingebaute Variable oder sogar ein Array oder eine Datenstruktur sein. Wenn die zu durchlaufende Instanz code durchlaufen wird, wird die überwachte Variable hier angezeigt, d. h. Sie können einen Wert leicht verfolgen, ohne ihn in den Fenstern Locals oder Globals suchen zu müssen. Wenn Sie mit der rechten Maustaste auf einen Wert klicken, können Sie seinen Typ einstellen, ihn löschen oder alle Überwachungen aus dem Fenster entfernen.


Instanz

Wie der Name schon sagt, werden in diesem Fenster alle Instanzvariablen angezeigt, lokale Variablen für das Instanzereignis, eingebaute Variablen und Strukturen, die mit der Instanz verbunden sind, die gerade im Source-Fenster durchlaufen wird (wenn auf eine Instanz mit dem Schlüsselwort das Schlüsselwort "other" oder die Anweisung with referenziert wird, zeigt dieses Fenster stattdessen die Eigenschaften dieser Instanz an). Wenn Sie mit der rechten Maustaste auf einen Wert klicken, können Sie seinen Typ einstellen, ihn löschen oder alle Beobachtungen aus dem Fenster löschen.


Alle Instanzen

Wenn Sie alle Instanzen sehen möchten, die die sich derzeit im Raum befinden, können Sie dieses Fenster auswählen und sie werden aufgelistet. Sie können auch den Status aller eingebauten Variablen, Instanzvariablen oder Strukturen sehen, die sie haben, und Sie können mit der rechten Maustaste auf einen Wert klicken, um seinen Typ zu ändern.


Ausgewählte Instanz

Dieses Fenster dient dazu, wenn Sie eine Instanz aus Ihrem Spiel auswählen. Dazu pausieren Sie einfach das Spiel und klicken dann auf eine beliebige sichtbare Instanz im Spielfenster. Die Informationen über die Instanz, die sich unter dem Mauszeiger befindet, werden dann im Fenster Selected Instance-Fenster angezeigt, wo du die eingebauten Variablen, Instanzvariablen und Strukturen der Instanz sehen kannst, genau wie im Instanzen-Fenster.


Zustände rendern

Diese Uhr zeigt Ihnen alle verfügbaren Informationen für den Zustand des aktuellen Frames, der gerade gerendert wird, wie z.B. die Zeichen-Alpha- oder Füllfarbe. Sie kann nicht in Echtzeit aktualisiert werden, so dass Sie das Spiel anhalten müssen, um die Frame Details.


Oberflächen / Texturen

Mit dieser Überwachung können Sie sehen Sie können genau sehen, was auf die Oberflächen in Ihrem Spiel gezeichnet wird (einschließlich der Anwendungsoberfläche), und dieTexturseiten im Speicher sehen. Sie können Texturen oder Oberflächen aus dem Menü am unteren Rand der Uhr auswählen und dann mit der Maus über die Textur- oder Oberflächen-ID fahren, um sie in einem Pop-up-Fenster anzuzeigen:

Beachten Sie, dass der Debugger angehalten werden muss, bevor diese aktualisiert werden können.


Profil

In diesem Fenster sehen Sie das aktuelle Systemprofil des getesteten Spiels. Hier können Sie alles in Ihrem Spiel überprüfen, von ganzen Ereignissen bis hin zu einzelnen Funktionsaufrufen, um zu sehen, wie sie funktionieren und und eventuelle Problembereiche oder Engpässe zu erkennen. Puffer

Wenn Sie eine buffers in Ihrem Spiel initialisiert haben, zeigt dieses Fenster die Pufferdaten an. Sie können jede buffer auswählen, indem Sie den Wert für die Puffer-ID einstellen, der bei 0 für den ersten buffer beginnt in Ihrem Spiel, und erhöht sich für jeden weiteren Puffer um 1:

Wenn Sie mit der rechten Maustaste auf das Datenfenster klicken oder auf die Schaltfläche Datenanzeige klicken, können Sie auswählen, wie die Daten angezeigt werden sollen, indem Sie die Ausrichtung und den Typ festlegen und außerdem festlegen, wie die Daten im Feld Spaltenauswahl Sie können auch festlegen, wie die Daten im Feld Spaltenauswahl angeordnet werden. Hier gibt es ein Dropdown-Fenster, in dem Sie aus einer Standardliste von Layouts auswählen können.
