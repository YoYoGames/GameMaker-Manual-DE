Der Profiler

Mit dem Debugger können Sie Ihre code durchgehen und alle Probleme mit der code Basis finden, aber manchmal müssen Sie auf subtilere Details achten, um Ihre Spiele richtig zu optimieren. Die Option " Profil" hilft Ihnen dabei, indem sie Ihnen einen Überblick über die Leistung gibt, indem sie die Zeit anzeigt, die für den Aufruf von Funktionen benötigt wird, scripts, und sogar, wie lange es dauert, bis ein bestimmtes Ereignis abgeschlossen ist.

Neben dem Hauptfenster, in dem die verschiedenen Ereignisse, scripts und aufgerufenen Funktionen angezeigt werden, haben Sie die folgenden Möglichkeiten, den Profiler zu verwenden:

Aktivieren / Deaktivieren

Mit dieser Schaltfläche können Sie den Profiler aktivieren oder deaktivieren. Wenn er aktiviert ist, wird der Profiler gestartet und Sie sehen im Hauptfenster eine Liste von Ereignissen, scripts und Funktionen mit verschiedenen Datenwerten für jedes dieser Elemente. Wie diese Daten angezeigt werden hängt von der Zeitanzeige und dem Ansichtsmodus ab (beide werden weiter unten näher erläutert).


Zeitanzeige

Mit diesem Kontrollkästchen können Sie auf switch zwischen der durchschnittlichen Zeit und der absoluten Zeit für den Profiler wählen. Wenn das Kontrollkästchen nicht aktiviert ist, werden im Hauptfenster die Gesamtzahl der Aufrufe der Funktion, script oder Ereignis und die Gesamtzeit , die im Verlauf des Profils benötigt wurde (die Zeiten sind in Mikrosekunden angegeben). Wenn Sie dies jedoch aktivieren, zeigen diese Spalten die durchschnittliche Zeit pro Schritt für jede Funktion script oder Ereignis, und die durchschnittliche Anzahl der Aufrufe der Funktion.


Ansichtsmodus

In diesem Menü können Sie den Ansichtsmodus auswählen, der entweder von oben nach unten oder von unten nach oben sein kann.

Wenn Sie den Ansichtsmodus "Von oben nach unten" wählen, wird das Profil von oben nach unten angezeigt (entspricht mehr oder weniger der Hierarchie des Aufrufstapels), d. h. es wird das Ereignis, dann script und dann alle verwendeten Funktionen angezeigt. Beachten Sie, dass Sie auf auf einen der Einträge doppelklicken können, um ihn im Source-Debug-Fenster zu öffnen, falls Sie ein solches haben. ein solches haben. Wenn Sie ein -Symbol neben einem Namen sehen, gibt es weitere script oder Funktionsaufrufe innerhalb dieses Abschnitts. Wenn Sie auf klicken, wird der Baum erweitert, um sie anzuzeigen.

Wenn Sie den Ansichtsmodus "Von unten nach oben" wählen, werden die Dinge von unten nach oben dargestellt, so dass Sie alle Funktionen und script Aufrufe einzeln sehen können. Wenn Sie in diesem Ansichtsmodus auf klicken, wird die Funktion oder script erweitert, um zu zeigen, was sie tatsächlich aufgerufen hat.


Ansicht Ziel

Mit dem Ansichtsziel können Sie wählen, ob Sie das Projekt GML (Ereignis- und code -Daten), die Prozesse der Hintergrundmaschine oder beides anzeigen möchten. Wenn Sie "GML" wählen, erhalten Sie die Ereignisse, Funktionen und scripts, die in jedem Schritt aufgerufen werden. während die "Engine"-Ansicht nur die für das Projekt erforderlichen Engine-Aufrufe anzeigt, so dass Sie eventuelle Engpässe in der Art und Weise, wie Ihr Projekt Dinge handhabt, erkennen können. Wenn Sie "Beide" wählen, werden beide Datensätze angezeigt. werden beide Datensätze angezeigt. Beachten Sie jedoch, dass in Kombination mit der "Bottom-Up"-Ansicht die verschiedenen script und Funktionsaufrufe innerhalb der Engine-Aufrufe verschachtelt sind und Sie auf klicken müssen, um sie zu sehen.



Die Profildaten werden in vier Spalten angezeigt, und wenn Sie auf eine davon klicken, werden die Daten nach den Kriterien der Spalte sortiert.

  • Name: Dies ist der Name des Ereignisses, script oder der Funktion, die aufgerufen wird, und kann ein daneben haben, um weitere Aufrufe zu oder von ihm zu kennzeichnen (je nach dem von Ihnen gewählten Ansichtsmodus ). Durch einen Doppelklick auf auf einen der Namen wird der code Block oder script im Quelltextfenster geöffnet. Wenn Sie auf den oberen Teil dieser Spalte klicken, werden die Namen in alphabetischer Reihenfolge sortiert.
  • Zeit: Hier wird entweder die Gesamtzeit angezeigt, die eine Funktion (oder script oder ein Ereignis) während der Profilerstellung benötigt hat, oder die durchschnittliche Zeit, die jeder einzelne Schritt für den Aufruf benötigt hat (je nach gewählter Option Zeitanzeige ), und zwar in Mikrosekunden.
  • Aufrufen: Hier sehen Sie entweder die Gesamtzahl der Aufrufe des gegebenen Ereignisses, der Funktion oder script oder die durchschnittliche Anzahl der Aufrufe in einem einzelnen Schritt (wiederum abhängig von der gewählten Zeitanzeigeoption ).
  • Schritt%: Der hier angezeigte Wert ist der Gesamtprozentsatz eines einzelnen Schritts, den der Anruf gemacht hat.
