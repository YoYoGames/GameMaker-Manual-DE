Der Debugger

Der Debugger ist ein leistungsfähiges Werkzeug, um Ihr Spiel zu überprüfen. Er ist besonders nützlich, um Fehler und Bugs in Ihrem code aufzuspüren und um zu überprüfen, ob scripts zum richtigen Zeitpunkt ausgeführt wird und ob Variablen und Arrays die erwarteten Werte enthalten. Werte enthalten, die Sie erwarten usw... Sie können den Debugger starten, indem Sie Ihr Spiel in der GameMaker-IDE über die Debug-Schaltfläche oben auf IDE starten oder indem Sie im Menü "Build" die Option "Debug" wählen.

HINWEIS: Möglicherweise werden Sie von Windows Firewall (oder einem anderen Firewall-Programm, das Sie ausführen) aufgefordert, eine Sicherheitsausnahme für dieses module zu erstellen. Sie müssen dies tun, da es sonst nicht richtig funktioniert. Dies liegt an der Funktionsweise Funktionsweise des Debuggers und ist für debugging mobile platforms unerlässlich.

Graphische Ansicht

Die Diagrammansicht zeigt Ihnen eine grafische Darstellung des Speicherverbrauchs und der Framerate Ihres Projekts im Zeitverlauf:

Das Diagramm zeigt die aktuelle und durchschnittliche Speichernutzung sowie die aktuelle und durchschnittliche FPS über die Zeit an. Sie können mit / und dem Mausrad hinein- oder herauszoomen. Wenn Sie den Mauszeiger über eines der Elemente des Diagramms bewegen, wird der Wert für dieses Element an dem ausgewählten Punkt angezeigt.

In der Diagrammansicht werden auch bestimmte Debug-Ereignisse als Punkte entlang des Diagramms angezeigt. Diese Ereignisse werden von GameMaker an bestimmten Punkten im Spielverlauf automatisch generiert, um bestimmte wichtige Ereignisse anzuzeigen, aber Sie können auch eigene Diagrammelemente einfügen, indem Sie die Funktionen show_debug_message() und debug_event(). Systemereignisse werden in einer anderen Farbe gekennzeichnet als Ereignisse, die durch Code ausgelöst werden:

Wenn Sie das Spiel im Debug-Modus laufen lassen und mit der Maus über diese Ereignisse fahren, wird eine Liste mit allen Ereignissen angezeigt, die an diesem Punkt im Spiel ausgelöst wurden, und Sie haben die Möglichkeit, die Liste in ein separates Fenster zu ziehen, um sie zu überarbeiten, indem Sie auf den Text "In einem neuen Fenster öffnen" klicken:

Ressource Ansicht

Die Ressourcenansicht zeigt Ihnen eine Liste aller im Projekt verwendeten Ressourcen, die code oder GML Visual-Aktionen enthalten können. Dieser Bereich funktioniert ähnlich wie der Asset-Browser und Sie können die Elemente erweitern, um weitere Informationen anzuzeigen, z. B. die verschiedenen Ereignisse für ein bestimmtes Objekt. Mit einem Doppelklick auf auf eine der Ressourcen öffnen Sie Sie können dann auf eine beliebige Ressource doppelklicken, um die code in der Codeansicht zu öffnen, wo Sie Werte überprüfen und Haltepunkte hinzufügen oder entfernen können.

Stapel aufrufen

In diesem Fenster wird der aktuelle Aufrufstapel von Ereignissen sowie die Zeilennummer der code angezeigt, die gerade durchlaufen wird. Der Aufrufstapel ist im Wesentlichen eine "last in - first out"-Datenstruktur, die die Ausführungsreihenfolge des gesamten Spiels anzeigt code sowohl aus dem GameMaker runner - da debugging mit der Virtual Machine (VM) ausgeführt wird - als auch aus dem, was Sie für das Spiel geschrieben haben. Der Debugger muss pausiert sein, bevor hier Informationen angezeigt werden.

Symbolleiste

Die Symbolleiste enthält die Debugger-Bedienelemente sowie einige grundlegende Informationen wie die Speichernutzung, die aktuellen (realen) FPS für das Spiel und eine Anzeige, die angibt, ob der Debugger derzeit verbunden ist oder nicht (die Anzeige ist grün, wenn er mit einer laufenden Instanz Ihres Projekts mit einer laufenden Instanz Ihres Projekts verbunden ist und andernfalls rot). Die verfügbaren Steuerelemente sind:

Damit starten Sie das Spiel erneut, nachdem es angehalten wurde (Tastaturkürzel "f5").
Dadurch wird das Spiel angehalten (unterbrochen).
Startet das Spiel neu.
Stoppt das Spiel.
Treten Sie in einen code Block oder script (Tastaturkürzel "f11").
Gehen Sie über einen code Block oder script (Tastaturkürzel "f10").
Verlassen Sie einen code Block oder script (Tastaturkürzel + "f11").
Aktivieren oder deaktivieren Sie die Echtzeitaktualisierung der Debug-Informationen (standardmäßig ist sie deaktiviert).

Die ersten drei Schaltflächen dienen zur Steuerung Ihres Spiels, mit denen Sie es jederzeit starten, unterbrechen, neu starten oder beenden können. Wenn das Spiel pausiert ist (entweder durch Klicken auf die Schaltfläche Pause oder von einem Haltepunkt aus), können Sie mit den drei Schrittschaltflächen durch Ihr code schreiten.

Wenn wir von einem "Schritt" durch code sprechen, meinen wir einfach, dass Sie Ihr Spiel anweisen können, die aktuelle Zeile von code ablaufen zu lassen und dann mit der nächsten Zeile fortzufahren. So können Sie Ihr gesamtes Spiel mit einer Zeile von code nach der anderen durchspielen. wenn Sie es wünschen. Normalerweise würden Sie dazu einfach die Schaltfläche " Step In " verwenden, aber es gibt auch eine Schaltfläche für das Überspringen von Code und eine für das Verlassen von code. Das Überspringen ist nützlich, wenn Sie scripts durchlaufen müssen, denn wenn Sie darauf klicken, wird der gesamte Block script oder code in einem einzigen Schritt ausgeführt und danach in der nächsten Zeile angehalten, während Sie mit der Schaltfläche "Aussteigen" aus einem script aussteigen (sie werden ausgeführt, aber alle auf einmal) und beim nächsten Schritt nächsten Schritt, wenn sich der Bereich ändert.

Das letzte Symbol für Echtzeit-Updates ermöglicht es dem Debug-Fenster, seine Daten in Echtzeit zu aktualisieren, so dass Sie sehen können, wie sich Variablen ändern oder Instanzen erstellt und zerstört werden, während dies geschieht. Beachten Sie, dass dies die Leistung des Spiels beeinträchtigen kann und dass nicht alle Informationen in Echtzeit verfügbar sind. Zum Beispiel kann die Render-Status-Überwachung (siehe den Abschnitt über Überwachungen) nicht in Echtzeit angezeigt werden.

Code-Ansicht

In der Ansicht code können wir verschiedene scripts oder code Ereignisse aus unserem laufenden Projekt sehen und überprüfen. Standardmäßig ist diese Ansicht leer, aber Sie können auf auf ein beliebiges Asset in der Ressourcenansicht doppelklicken, um es hier zu öffnen. Sie können auch mehrere verschiedene Ereignisse oder scripts öffnen, und sie werden der Code Ansicht workspace als Registerkarten am oberen Rand hinzugefügt.

Während ein beliebiges code Fenster in der Code Ansicht geöffnet ist, können Sie mit der linken Maustaste auf im Zwischenraum (wo sich die Zeilennummern befinden) klicken, um Haltepunkte hinzuzufügen oder zu entfernen (siehe weiter unten auf dieser Seite für Informationen über Haltepunkte).

Sie können auch die rechte Maustaste verwenden, um das Kontextmenü zu öffnen, mit dem Sie auch Haltepunkte hinzufügen oder entfernen und alle ausgewählten code in die Zwischenablage zu kopieren.

In diesem Abschnitt geht es darum, verschiedene Aspekte Ihres Spiels zu "beobachten", um eine tiefere und detailliertere debugging zu ermöglichen. Dies wird auf der folgenden Seite eingehend behandelt:

Es ist wichtig zu beachten, dass während des Spiels die Debug-Fenster standardmäßig nicht mit Variablenwerten und anderen Details aktualisiert werden. Daher müssen Sie zunächst entweder die Schaltfläche Pause im Debugger verwenden, um das Spiel anzuhalten, bevor Locals, Globals und andere Daten verfügbar sind, oder die Echtzeitaktualisierung über die Symbole oben aktivieren. Die einzige Ausnahme von dieser Regel ist die Profilansicht , die in Echtzeit aktualisiert wird, wenn sie unabhängig von der Echtzeitansicht switch aktiviert wird, da sie über eine eigene Schaltfläche zum Ein- und Ausschalten verfügt. Ein weiterer wichtiger Bestandteil von debugging ist die Verwendung von Haltepunkten. Ein Haltepunkt ist eine Stelle in Ihrem Projekt code, an der Sie anhalten möchten, damit Sie sehen können, was vor sich geht. Wenn der Debugger läuft, können Sie jederzeit und an jeder beliebigen Stelle Haltepunkte jederzeit und in jedem code oder script Block hinzufügen oder entfernen, indem Sie mit der linken Maustaste auf im Zwischenraum (wo sich die Zeilennummern befinden) klicken oder mit der rechten Maustaste zum das Kontextmenü öffnen, über das Sie ebenfalls Haltepunkte hinzufügen oder entfernen können:

Haltepunkte können auch über den GameMaker IDE gesetzt werden. Beim Schreiben Ihres Projekts können Sie sie jederzeit hinzufügen oder entfernen, indem Sie f9 (oder mit der rechten Maustaste und der Auswahl von "Haltepunkt umschalten"), entweder auf einer Aktion im Objektfenster (für GML Visual-Benutzer), oder auf einer Zeile in einem code Editor-Fenster. Wenn Sie nun Ihr Spiel im Debug-Modus laufen lassen, wird es an dem Punkt (oder den Punkten) angehalten, den Sie definiert haben.

Unterbrechungspunkte, die Sie auf IDE setzen, werden zusammen mit Ihrem Projekt gespeichert und bleiben über mehrere Durchläufe erhalten, bis sie wieder entfernt werden. Wenn Sie jedoch im Debugger einen Unterbrechungspunkt zu code hinzufügen, geht dieser verloren, wenn module geschlossen oder ein neuer Testlauf gestartet wird. Beachten Sie, dass alle Haltepunkte im Ausgabefenster erscheinen, wo sie aktiviert, deaktiviert oder sogar entfernt werden können.
