Fehlerberichterstattung

Die erste Form der Fehlermeldung ist der Syntaxfehler. Dabei handelt es sich um einen Fehler in Ihrem code, den GameMaker erkannt hat, bevor Sie versucht haben, ihn zu kompilieren. Diese werden im IDE angezeigt, entweder im code Editor oder im Ausgabefenster für Syntaxfehler:

Die nächste Ebene der Fehlermeldung ist derCompiler-Fehler. Diese Arten von Fehlern sind allgemeiner als runner Fehler und können auf einen subtilen Fehler im Spiel code zurückzuführen sein, der zum Absturz führt, während das Spiel kompiliert wird, bevor es überhaupt eine Chance bekommt, zu laufen. Auch diese Fehler werden im Compiler Fehlerausgabefenster angezeigt:

Wenn das Spiel kompiliert wird und ein Fehler auftritt, erhalten Sie einen Runner-Fehler. Bei dieser Art von Fehler wird eine Meldung auf dem Bildschirm angezeigt, die der unten abgebildeten ähnelt:

Die meisten Fehlermeldungen von runner sind nach dem gleichen Schema aufgebaut: eine Meldung, die die Art des Fehlers angibt, die Instanz und das Ereignis, bei dem der Fehler auftrat, dann eine Reihe von runner Details, die den Fehler genauer erklären, und schließlich eine Liste aller deklarierten globalen, Instanz- und lokalen Variablen in Ihrem Spiel zu diesem Zeitpunkt. Ganz unten befindet sich die Schaltfläche Abbrechen , mit der das Spiel abgebrochen wird, sowie die Schaltfläche Kopieren (mit der der Fehler in die Zwischenablage kopiert wird) und die Schaltfläche Löschen , mit der die Fehlermeldungen aus dem Fenster gelöscht werden. Die Informationen, die die Fehlermeldung hier liefert, sind sehr wichtig, da sie die großen, mit syntax zusammenhängenden Fehler aufspürt und meldet, die Ihr fertiges Spiel zum Absturz bringen werden, aber beachten Sie, dass sie keine subtileren Fehler meldet und auch kein Feedback zur Leistung gibt... dafür müssen Sie den Debugger verwenden.
