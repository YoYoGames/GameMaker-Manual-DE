Kompilier-Fehler

Die zweite Art von Fehler, die für Ihr Spiel gemeldet werden kann, ist der Compiler-Fehler. Ein compiler -Fehler tritt auf, wenn Ihr Spiel auf eine Art von Fehler stößt, den der syntax -Checker möglicherweise nicht erkennen konnte - es ist sehr einfach, zu vergessen, eine Variable zu deklarieren oder eine Zeichenkette fälschlicherweise als Real einzugeben, zum Beispiel - oder wenn ein Fehler damit zusammenhängt, wie Sie die Kompilierwerkzeuge in den Plattformeinstellungen für das Ziel platform eingerichtet haben. Diese Fehler werden von GameMaker beim Kompilieren des Spiels erkannt und auch in der Compiler-Fehlerausgabe angezeigt, wie unten dargestellt:

HINWEIS: Bei diesen Fehlern wird in der Regel kein Fenster angezeigt, sondern lediglich die Meldung "Build Failed" in der Ausgabe von compiler. Wenn ein Fenster erscheint, handelt es sich normalerweise um einen Runner Fehler, der hier erklärt wird.

Die Fehlermeldungen auf compiler haben alle das gleiche Format:

[object] - [event] - [Line Number]: [error string]

Wenn der Fehler in einer script gefunden wird, wird er einfach sein:

[script] - [Line Number]: [error string]

Mit einem Doppelklick auf auf einen der compiler Fehlereinträge können Sie das betreffende Asset an der Stelle öffnen, an der der Fehler aufgetreten ist. Anhand der in der Meldung enthaltenen Informationen können Sie dann genau feststellen, an welcher Stelle des Objekts oder script der Fehler aufgetreten ist. Da diese Fehler jedoch manchmal etwas kryptisch erscheinen können, finden Sie unten eine vollständige Liste aller Fehler und eine kurze Erklärung ihrer Bedeutung:
