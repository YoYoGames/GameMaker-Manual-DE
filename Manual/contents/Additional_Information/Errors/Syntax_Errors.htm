Syntaxfehler (Feather-Meldungen)

Die erste Ebene der Fehlermeldung bei der Programmierung Ihrer Spiele in GameMaker befindet sich im code oder GML visuellen Editor, und zwar in der Syntaxprüfung. Während Sie Ihre code schreiben - oder Ihre Aktionen hinzufügen - prüft GameMaker, ob es den etablierten syntax für die Sprache folgt, und markiert alle Probleme, die es findet, damit Sie sie beheben können, bevor das Spiel ausgeführt oder eine ausführbare Datei kompiliert wird.

Die Syntaxfehlerprüfung erfolgt unterschiedlich, je nachdem, ob Sie Feather in den Feather-Einstellungen aktiviert haben.

Syntax-Fehler (Feather deaktiviert)

Diese Fehler lassen sich in zwei Hauptkategorien einteilen:

  • Warnungen: Diese werden für Dinge angezeigt, die ein Fehler sein könnten, es aber nicht sind, und die das Spiel wahrscheinlich nicht am Kompilieren hindern, aber möglicherweise zu runtime Fehlern führen. Die häufigsten Ursachen für diese Warnungen sind, wenn eine Variable in einem Ausdruck verwendet wird und noch nirgendwo anders definiert wurde, oder wenn eine Variable zwar definiert, aber nicht verwendet wurde.
  • Fehler: Dies sind Probleme, die es nicht erlauben, das Spiel zu kompilieren, geschweige denn zu starten. Diese sollten behoben werden, bevor Sie mit code weiterarbeiten. Die folgende Tabelle zeigt die häufigsten Ursachen für diese Fehler.

Syntaxfehler werden aktualisiert, während Sie Ihre code schreiben (es gibt eine kurze Pause zwischen der Eingabe und dem Erscheinen von Fehlern in diesem Fenster, um zu verhindern, dass Fehler für unvollendete code gemeldet werden), und folgen dem Format:

[object] - [event] - [Line Number] - [Position In Line]: [error string]

Oder wenn der Fehler in einer script zu finden ist:

[script] - [Line Number] - [Position in line]: [error string]

Durch einen Doppelklick auf auf einen der in der Ausgabekonsole angezeigten Fehler gelangen Sie zu der Zeile im Objektereignis oder script, in der der Fehler auftritt, so dass Sie diese bearbeiten können.

Es ist zu beachten, dass einige dieser Fehler an bestimmten Stellen in Ihrem code angezeigt werden, der eigentliche Fehler jedoch an anderer Stelle liegen kann oder ein einziges Problem mehrere Fehlermeldungen hervorrufen kann:

Federnachrichten (Federn aktiviert)

Wenn Feather aktiviert ist, wird das Fenster Syntax Errors durch das Fenster Feather Messages ersetzt:

Das Fenster Feather-Meldungen listet die Fehler und Warnungen auf, die Feather in Ihrem code findet, sowie die Vorschläge, die es macht, wenn es bestimmte code Muster findet. Welche Arten von Fehlern hier angezeigt werden, hängt von dem Profil ab, das Sie in den Feather-Einstellungen unter Message Severity definieren.

In diesem Fenster kann die Liste der Meldungen auf- oder absteigend nach der Meldung code, nach dem Meldungstext (alphabetisch) oder nach der Ressource (alphabetisch nach dem Ressourcennamen) sortiert werden. Sie können auch über das Suchfeld in der oberen rechten Ecke des Fensters suchen.

Federnachrichten lassen sich in drei Kategorien einteilen:

  • Warnungen: Diese werden bei möglichen Fehlern angezeigt, die in den meisten Fällen die Kompilierung des Spiels nicht verhindern, aber zu runtime Fehlern führen können. Die häufigsten Ursachen für diese Warnungen sind, wenn eine Variable in einem Ausdruck verwendet wird und sie noch nirgendwo anders definiert wurde, oder wenn eine Variable definiert, aber nicht verwendet wurde.
  • Fehler: Dies sind Probleme, die es nicht erlauben, das Spiel zu kompilieren, geschweige denn zu starten. Diese sollten behoben werden, bevor Sie mit code weitermachen.
  • Vorschläge: Dies sind Vorschläge für mögliche Verbesserungen der code, die sich auf eine Vielzahl von Dingen beziehen.
