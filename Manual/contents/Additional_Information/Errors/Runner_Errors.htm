Läufer-Fehler

Selbst nach der Überprüfung von syntax im Editor code und der anschließenden Überprüfung von compiler auf Compiler-Fehler durch code kann es immer noch vorkommen, dass etwas schief geht. In den meisten Fällen führt dies zu einem Virtual Machine (VM)- Runner-Fehler (auch Laufzeitausnahme genannt), der wie folgt aussieht:

Runner-Fehler sind in der Regel schwerwiegender als Kompilierfehler, da sie bedeuten, dass mit Ihrer code etwas ernsthaft schief läuft, was weder der code -Editor noch die compiler erkennen konnten, und deshalb sollten Sie auf solche Fehler achten. Wenn ein solcher Fehler auftritt, können Sie die Schaltfläche Kopieren auf dem Pop-up-Fenster verwenden, um den Fehler in die Zwischenablage zu kopieren, die Sie dann in eine Textdatei (oder wo auch immer) einfügen können, um später darauf Bezug zu nehmen.

Die Struktur dieses Fehlers ist wie folgt:


  • Hier wird das Ereignis angezeigt, in dem der Fehler aufgetreten ist, sowie das Objekt (falls zutreffend).

  • Hier wird der tatsächliche Fehler angezeigt. Er zeigt also den Umfang des Fehlers und die Art des Fehlers an. Der erste Wert in den Klammern ist die Instanz-ID der Instanz, auf der code läuft - oder einer der speziellen Werte, die in der Tabelle unten aufgeführt sind - und der zweite ist ein interner Wert, den GameMaker zur Identifizierung von Fehlern verwendet und der ignoriert werden kann. Im Fall des obigen Bildes heißt es, dass die Variable _image nicht auf das Objekt obj_EA_DEMO_Control gesetzt wurde (d.h. nicht existiert, bevor darauf zugegriffen wird).

  • Dies zeigt die Zeilennummer an, in der der Fehler auftritt, sowie einen Ausschnitt von code, der die Position anzeigt. Beachten Sie, dass dies aufgrund der Art und Weise, wie das Spiel kompiliert wird, nicht immer zu 100 % genau ist, aber es sollte ausreichen, um genau herauszufinden, wo das Problem liegt.

Wie bereits erwähnt, werden bestimmte Fehlermeldungen nicht durch einen Instanz-ID-Wert, sondern durch einen negativen Wert identifiziert. Diese Werte können verwendet werden, um die genaue Art des Fehlers zu bestimmen und worauf er sich bezieht, wobei die folgenden Werte möglich sind:

Vorsilbe Umfang
-1 Selbst
-2 Andere
-3 Alle
-4 Niemand
-5 Global
-6 Nicht spezifiziert
-7 Lokales

Die möglichen Fehler der VM runner sind wie folgt:

Fehler Nachricht Operation Beschreibung
DoSet Ungültiger Vergleichstyp Datenarten Dies bedeutet, dass runner versucht hat, zwei inkompatible Datentypen zu vergleichen, z. B. eine reelle Zahl mit einer Zeichenkette.
DoConv Ausführungsfehler Datenarten Dies bedeutet einen Fehler bei der Umwandlung eines Datentyps in einen anderen.
DoAdd Ausführungsfehler Mathe Bei der Verwendung des Additionsausdrucks (+) ist etwas schief gelaufen.
DoMul Ausführungsfehler Mathe Bei der Verwendung des Multiplikationsausdrucks (*) ist etwas schief gelaufen.
DoSub Ausführungsfehler Mathe Bei der Verwendung des Subtraktionsausdrucks (-) ist etwas schief gelaufen.
DoSub Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Mathe Sie versuchen, den falschen Typ von Variablen zu subtrahieren (z. B. eine Zeichenkette von einer reellen Zahl zu subtrahieren).
DoDiv Ausführungsfehler Mathe Bei der Verwendung des Divisionsausdrucks (/ oder div) ist etwas schief gelaufen.
DoDiv Ausführungssystem - Kann nicht mit dem Typ String arbeiten Mathe Sie versuchen, den falschen Variablentyp zu dividieren (z. B. eine Zeichenkette durch eine reelle Zahl zu dividieren).
DoDiv Dividieren durch Null Mathe Sie versuchen, durch 0 zu dividieren (beachten Sie, dass dies nur bei der Ganzzahldivision möglich ist; die Division einer reellen Zahl (ungleich Null) durch 0 ergibt unendlich als Antwort, die Division von Null durch Null ergibt NaN als Antwort). Sie können diese Werte mit (is_infinity) und (is_nan) überprüfen.
DoMod Ausführungsfehler Mathe Bei der Verwendung des Modulo-Ausdrucks (mod) ist etwas schief gelaufen.
DoMod Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Mathe Sie versuchen, modulo (mod) auf den falschen Typ von Variablen anzuwenden (z. B. mod a string by a real).
DoAnd Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "und"-Ausdrucks (&) ist etwas schief gelaufen.
DoAnd Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Bitweise Sie versuchen, bitweise "und" (&) auf den falschen Typ von Variablen anzuwenden (z. B. versuchen Sie, eine Zeichenkette mit einer reellen Zahl "und" zu verbinden).
DoOr Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "oder"-Ausdrucks (|) ist etwas schief gelaufen.
DoOr Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Bitweise Sie versuchen, "oder" (|) für den falschen Typ von Variablen zu verwenden (z. B. versuchen Sie, eine Zeichenkette mit einer reellen Zahl zu "oder").
DoXor Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "xor"-Ausdrucks (^) ist etwas schief gelaufen.
DoXor Ausführungssystem - Kann nicht mit dem Typ String arbeiten Bitweise Sie versuchen, "xor" (^) auf den falschen Variablentyp anzuwenden (z. B. versuchen Sie, eine Zeichenkette mit einer reellen Zahl zu "xor").
DoShl Ausführungsfehler Bitweise Beim Bitshifting nach links (<<) eines Wertes ist etwas schief gelaufen.
DoShl Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Bitweise Sie versuchen, den falschen Variablentyp nach links zu schieben (<<) (z. B. versuchen Sie, eine Zeichenkette nach links zu schieben).
DoShr Ausführungsfehler Bitweise Beim Bitshifting nach rechts (>>) eines Wertes ist etwas schief gelaufen.
DoShr Ausführungsmodul - Kann nicht mit dem Typ String arbeiten Bitweise Sie versuchen, den falschen Variablentyp nach rechts zu schieben (>>) (z. B. versuchen Sie, eine Zeichenkette nach rechts zu schieben).
DoNeg Ausführungsfehler Negieren Sie versuchen, einen Variablentyp in ein Negativ zu verwandeln, obwohl dieser Typ eine solche Operation nicht zulässt.
DoNot Ausführungsfehler Negieren Sie versuchen, einen Variablentyp "nicht" zu verwenden, obwohl dieser Typ eine solche Operation nicht zulässt.
Push Ausführungsfehler - Variablenindex außerhalb des Bereichs (var) Stapel Die Variable, auf die zugegriffen wird, liegt außerhalb des festgelegten Bereichs für die runner.
Push Variable holen (var) Stapel Die angegebene Variable wurde nicht definiert oder ist unbekannt.
Pop Variable Index außerhalb des Bereichs (var) Stapel Die Variable, auf die zugegriffen wird, liegt außerhalb des festgelegten Bereichs für die runner.
Pop Variable holen (var) Stapel Die angegebene Variable wurde nicht definiert oder ist unbekannt.
With Keine Verwendung von global in with-Anweisung möglich Mit Sie haben versucht, "global" als Variable innerhalb einer "with" Anweisung zu verwenden, d.h.:
with (global)
   {
   //do something;
   }
With Keine Verwendung von local in with-Anweisung möglich Mit Sie haben versucht, "local" als Variable innerhalb einer "with" Anweisung zu verwenden, d.h.:
with (local)
   {
   //do something;
   }
DoCall Execution Engine Typ Fehler Motor Dies ist ein undefinierter Fehler innerhalb von Virtual Machine. Sollte dieser Fall eintreten, sollten Sie einen Fehlerbericht einreichen (siehe: Das Hilfemenü für Details, wie Sie dies tun können.
Stack Overflow - Motor Ein Stapelüberlauf tritt auf, wenn zu viel Speicher auf dem Aufrufstapel verwendet wird und wenn Ihr Spiel versucht, mehr Speicherplatz zu verwenden, als auf dem Aufrufstapel verfügbar ist (d. h. wenn es versucht, über die Grenzen des Aufrufstapels hinaus auf Speicher zuzugreifen, was im Wesentlichen ein buffer Überlauf ist), wird gesagt, dass der Stapel überläuft, was zu einem Programmabsturz führt. Starten Sie Ihren Computer und GameMaker neu, und wenn der Fehler weiterhin besteht, wenden Sie sich bitte an den Support und/oder melden Sie einen Fehler (wie oben beschrieben).
