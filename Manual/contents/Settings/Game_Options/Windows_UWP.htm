Windows UWP-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre UWP (Universal Windows Platform)-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Im allgemeinen Abschnitt der UWP-Optionen können Sie die Produktinformationen festlegen:

  • Anzeigename: Gibt den freundlichen Namen für die App an, der den Benutzern angezeigt wird (diese Zeichenfolge ist lokalisierbar).
  • Paketname: Dies ist der Name des Pakets, das Sie erstellt haben, und sollte mit dem Namen des Entwicklerschlüssels übereinstimmen, den Sie für das Spiel erstellt haben (siehe den Abschnitt über Pakete weiter unten).
  • Paket-Anzeigename: Dies ist der "Kurzname" des Pakets und sollte im Idealfall mit dem Anzeigenamen identisch sein.
  • Herausgeber-Anzeigename: Dies ist der Name Ihres Unternehmens oder der Person, die Sie als Herausgeber der Spiele angeben möchten.
  • Version: Die Versionsnummer Ihres Spiels.

Nach der Einrichtung können Sie die verschiedenen Ausrichtungen auswählen, in denen Ihr Spiel ausgeführt werden kann, um ein optimales Spielerlebnis zu gewährleisten. Die verfügbaren Optionen sind:

  • Landschaft
  • Porträt
  • Landschaftlich gewendet
  • Hochformatig gespiegelt

Standardmäßig sind alle diese Felder ausgewählt, und Sie sollten die Felder, die Sie nicht benötigen, abwählen.

Nachdem Sie die Ausrichtungsoptionen festgelegt haben, können Sie die verschiedenen Berechtigungen , die Ihre UWP-Anwendung benötigt, aus den folgenden Optionen auswählen:

  • Internetzugang aktivieren: Flags das Spiel hat Zugang zum Internet. Diese Option ist standardmäßig deaktiviert.
  • Mikrofonberechtigung aktivieren: Flags das Spiel hat Zugriff auf das Mikrofon des Geräts. Diese Option ist standardmäßig deaktiviert.
  • IAP-Sandbox aktivieren: Flags das Spiel verwendet In-App-Käufe in einer Sandbox. Diese Option ist standardmäßig deaktiviert.

Geräte-Familien

Hier können Sie auswählen, für welche Gerätefamilien der Windows UWP Export bei der Erstellung Ihrer ausführbaren Datei gelten soll. Standardmäßig ist die Option "Universelle Gerätefamilie unterstützen" aktiviert, was bedeutet, dass die App auf alle Geräte und Familien ausgerichtet ist. Um die App auf die Unterstützung bestimmter Gerätefamilien zu beschränken, sollte die Option "Universal" deaktiviert und die entsprechenden Gerätefamilien markiert werden. Es ist auch möglich, die Universal-Gerätefamilie zu unterstützen, aber andere Gerätefamilien auf bestimmte platform Versionen zu beschränken. In diesem Fall lassen Sie die Universal-Gerätefamilie markiert, markieren aber auch die andere Gerätefamilie, für die Sie eine andere platform Version unterstützen möchten, und geben die Versionsdetails für diese Familie in die entsprechenden Textfelder ein. Wie diese Logik funktioniert, können Sie auf der folgenden Microsoft-Supportseite nachlesen:

  • Zielgerätefamilie: Beispiele

Im Allgemeinen möchten Sie die Ziel- und Mindestversionen von platform nicht ändern und sollten sie auf den Standardwerten belassen. Wenn Sie jedoch Probleme mit der Erstellung haben - insbesondere, wenn Sie das standardmäßig eingestellte SDK Windows nicht installieren können -, müssen Sie diese Werte auf die von Ihnen installierten Werte ändern. Wenn Sie sich nicht sicher sind, welche Werte Sie einstellen müssen, erstellen Sie ein neues UWP-Projekt in Visual Studio und sehen Sie sich an, welche Werte eingestellt sind.

Weitere Informationen zu Gerätefamilien finden Sie in dem folgenden Microsoft-Dokument:

  • Übersicht über die Gerätefamilien

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres UWP-Zielgeräts nutzen soll. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen.
  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der Standard-Betriebssystem-Cursor in Ihrem Spiel angezeigt. Wenn diese Option deaktiviert ist, wird kein Cursor angezeigt, während sich die Maus über dem Spielfenster befindet, es sei denn, Sie haben einen Cursor als Teil des Spielprojekts erstellt (dies ist standardmäßig aktiviert).
  • Vollbildmodus starten: Wenn diese Option aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Diese Option ist standardmäßig deaktiviert.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer switch mit den Standard-OS-Tastenkombinationen von Vollbild zu Fenster und zurück wechseln. Ubuntu Diese Option ist standardmäßig deaktiviert.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob das Bild vollständig skaliert werden soll (so dass es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Sie die Anwendungsoberfläche ausschalten, werden alle in den Spieloptionen von Windows eingestellten Skalierungsoptionen deaktiviert, bis Sie sie wieder einschalten. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite festzulegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel auf PCs mit niedrigeren Spezifikationen.

Bilder

Im ersten Teil des Abschnitts "Bilder" haben Sie die Möglichkeit, die verschiedenen Kacheln festzulegen, die Ihr Spiel auf den verschiedenen Geräten verwenden wird, auf denen UWP laufen kann. Sie können die folgenden Einstellungen vornehmen (und sie sollten alle Dateien im Format *.png sein):

  • Das kleine Kachelbild ist das Logobild, das neben dem App-Namen in der Suchleiste und anderen Bereichen der Benutzeroberfläche erscheint. Es sollte 44 x 44 Pixel groß sein.
  • Das mittlere Kachelbild sollte 71x71 Pixel groß sein.
  • Die nächste Kachel ist die Kachel mit dem Ladenlogo und sollte 50 x 50 Pixel groß sein.
  • Die breiteBildkachel ist diejenige, die angezeigt wird, wenn die Kachel in ihrem breiten Format vorliegt, und sollte 310 x 150 Pixel groß sein. Wird dieses Bild nicht bereitgestellt, kann die Kachel nur im quadratischen Format angezeigt werden und kann keine Benachrichtigungen auf der Grundlage von breiten Vorlagentypen annehmen. Da der Benutzer entscheiden kann, welches Format die Kachel verwendet, empfiehlt es sich, eine breite Bildkachel einzubinden. Beachten Sie auch, dass die Kachel bei Angabe einer breiten Bildkachel anfangs möglicherweise im breiten Format angezeigt wird.
  • Die große Bildkachel ist das große Live-Kachelformat und sollte 310 x 310 Pixel groß sein. Beachten Sie, dass dies nur auf dem Desktop-Betriebssystem verwendet wird.

Im nächsten Abschnitt wird das Spiellogo definiert. Dies ist im Wesentlichen das Symbol, das Ihr Spiel innerhalb des UWP-Ökosystems verwenden wird. Es ist erwähnenswert, dass diese Bilder transparente .png Dateien sein können, in diesem Fall werden sie über die in diesem Abschnitt angegebene Hintergrundfarbe gezeichnet. Diese Hintergrundfarbe wird auch auf die Schaltflächenfarbe in allen App-eigenen Dialogfeldern und auf die App-Beschreibungsseite im Store auf Geräten angewendet.

Nachdem Sie das Logo eingerichtet haben, können Sie den Startbildschirm für das Spiel festlegen. Dies ist der Bildschirm, der angezeigt wird, wenn Sie Ihr Spiel starten. Er darf nicht größer als 620 x 300 Pixel sein und kann wie das Logo eine transparente .png Datei sein; in diesem Fall wird die von Ihnen gewählte Hintergrundfarbe verwendet.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird: platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Verpackung

GameMaker liefert einen Standard-Entwicklerschlüssel für die Entwicklung von Windows UWP, aber es wird empfohlen, dass Sie Ihren eigenen generieren und hier darauf verweisen. Um dies zu tun, erstellen Sie ein leeres Projekt in Visual Studio und speichern Sie irgendwo eine Kopie der automatisch generierten *.pfx Datei, die in das leere Projekt eingefügt wird (dies ist der Entwicklerschlüssel). Sie sollten dann diese *.pfx Datei hier verlinken.

Nachdem Sie die Datei verlinkt haben, sollten Sie auf die zugehörige Schaltfläche Installieren klicken und dann den Anweisungen in der sich öffnenden Eingabeaufforderung folgen. Wenn Sie dies nicht tun, schlägt die Verwendung der Schaltflächen Ausführen/Debuggen (oder Drücken von F5) fehl und Ihr Spiel kann nicht auf der UWP platform getestet werden (wenn Sie jedoch Ausführbare Datei erstellen verwenden, wird der Zertifizierungsinstallationsprozess automatisch gestartet, wenn die *.pfx noch nicht registriert wurde).

HINWEIS: Wenn Sie Ihre App in den Store hochladen wollen, müssen Sie die richtige *.pfx für Ihren Titel erhalten. Dies geschieht über die Verknüpfung Ihrer App mit dem Windows Store in Visual Studio, was Visual Studio veranlasst, eine neue *.pfx speziell für die App zu erstellen. Es ist auch erwähnenswert, dass Sie jede App mit dem Windows Store in Visual Studio verknüpfen können, d.h. Sie können ein leeres Projekt mit der Windows Store App verknüpfen und die generierte *.pfx einfach an einen sicheren Ort kopieren, der dann in den UWP Game Options verwendet werden kann. Eine Anleitung dazu finden Sie in der YoYo Games Knowledge Base.

In diesem Abschnitt müssen Sie auch den Namen des Herausgebers angeben, der für die Erstellung des Signierzertifikats verwendet wurde. Der Eingabe sollte immer ein "CN=" vorangestellt werden, z. B.: CN=YoYoGames.

Xbox Live

Im Abschnitt Xbox Live der UWP-Spieleoptionen können Sie Ihr UWP-Projekt für die Verwendung auf der Xbox One-Konsole aktivieren. Wenn Sie das Projekt hier für die Xbox aktivieren, können Sie Folgendes tun, wenn das Projekt auf einer Xbox One ausgeführt wird:

  • Anmeldung bei Xbox Live aus der App heraus
  • Verwenden Sie die Xbox Live-Benutzeridentität in irgendeiner Weise im Projekt (z. B. durch Anzeige des Gamertags oder des Spielerbildes)

Bevor Sie diese Dinge tun können, müssen Sie jedoch sicherstellen, dass Sie das Kontrollkästchen Xbox Live aktivieren angeklickt haben, und wenn Sie Teil des Microsoft Creators Program sind, sollten Sie auch die Option Creators Program App aktivieren. Wenn Sie am Creators Program teilnehmen, müssen Sie anschließend die Titel-ID und die Dienstkonfigurations-ID eingeben (weitere Informationen dazu finden Sie bei Microsoft).

Beachten Sie, dass Sie derzeit, um die Xbox Live-Funktionalität in Ihrem Projekt zu testen, Ausführbare Datei erstellen statt Ausführen verwenden müssen, da die UWP-Xbox-Ausgabe das Windows Store-Signaturzertifikat verwenden muss, das Sie als Teil der Paketierungsinformationen (siehe oben) angeben, und nicht das temporäre, das mit GameMaker geliefert wird, was verwendet wird, wenn Sie auf Ausführen klicken.

Sobald Sie XBox Live aktiviert haben, können Sie die XBox Live-Funktionen in Ihrem Projekt nutzen. Beachten Sie jedoch, dass Sie für die grundlegenden Funktionen Teil des Creator-Programms sein müssen und für die erweiterten Funktionen ein ID@XBox-Konto benötigen.
