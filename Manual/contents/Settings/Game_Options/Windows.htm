Windows-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre Windows Spieleprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Auf der Seite Allgemein für Windows Spiele sollten Sie die notwendigen Informationen angeben, damit Windows anzeigen kann, dass Ihr Spiel installiert wurde und läuft. Diese Informationen gehören in den Abschnitt " Produktinformation " und sollten vollständig ausgefüllt werden.

Im Abschnitt " Optionen" dieses Fensters können Sie auswählen, wohin das Spielpaket installiert werden soll, entweder in den Ordner "%localappdata%" oder "%appdata%", und ob Sie die Option " Copy exe to output folder" aktivieren möchten. Im Allgemeinen sollten Sie diese Option nie benötigen, aber sie wird für Benutzer bereitgestellt, die verschiedene DLLs haben, die miteinander und mit Ihrem Spiel kommunizieren. Wenn sie aktiviert ist, wird die Spiele-EXE beim Testen im Ausgabeordner beibehalten und nicht im Temp-Ordner abgelegt.

Als Nächstes können Sie den Ruhebereich von Windows einstellen. Diese Option dient dazu, das Stottern zu reduzieren, wenn Ihr Spiel auf bestimmten Systemen läuft. Wenn das Spiel schneller läuft als die Raumgeschwindigkeit, "schläft" GameMaker für die verbleibende Zeit, aber dieser Schlafmodus kann ziemlich ungenau sein und man schläft oft länger als nötig, was dazu führt, dass das Bild länger braucht, was wiederum Stottern verursacht. Um dieses Problem zu umgehen, können wir den Ruhezustand kürzer halten und dann für den Rest der Zeit auf loop bleiben, um die Genauigkeit zu erhöhen. Das Problem mit loop ist jedoch, dass die CPU-Auslastung dadurch steigt, was wiederum die CPU-Temperaturen erhöht und den Lüfter schneller laufen lässt. Standardmäßig ist dieser Wert auf 10 eingestellt, und in 99,99% der Fälle werden Sie damit gut zurechtkommen, aber für Low-End-Maschinen oder für Maschinen mit vielen Hintergrundprozessen ist dies möglicherweise keine ideale Lösung (oder auch für sehr High-End-Maschinen), und ein Wert von 1 oder 15 oder so könnte erforderlich sein. Beachten Sie jedoch, dass es sich hierbei um eine sehr systemspezifische Konfiguration handelt, und dass das, was auf Ihrem Rechner funktioniert, für einen anderen Benutzer möglicherweise nicht geeignet ist, weshalb Sie im Zweifelsfall den Wert 10 beibehalten sollten.

HINWEIS: Zusätzlich zur Sleep Margin können Sie die Leistung und den Stromverbrauch auf Windows weiter kontrollieren, indem Sie die Auflösung des Thread Schedulers unter runtime anpassen.

Die nächste Option ist, Steam für Ihr Spiel zu aktivieren. Wenn Sie diese Option aktivieren, stellen Sie sicher, dass Sie das Steamworks SDK installiert haben (siehe hier) und dass Sie die Steam App ID in den Allgemeinen Optionen für das Spiel hinzugefügt haben. Beachten Sie, dass Sie, wenn Sie GameMaker auf Windows und von der Version Steam von IDE aus ausführen und Spiele für Steam entwickeln, möglicherweise auch die Option " Alternativen Launcher für Steam-Anwendung verwenden " aktivieren müssen. Mit dieser Option wird Ihr Spiel beim Testen mit einem alternativen Startprogramm gestartet, um zu verhindern, dass es von Steam mit der Anwendungs-ID von GameMaker anstelle der eigenen Anwendungs-ID Ihres Spiels gekennzeichnet wird. Du solltest diese Option nur aktivieren, wenn du für Steam entwickelst und Probleme beim Starten deines Spiels hast.

Als vorletzte Option haben Sie die Möglichkeit, die Dateisystem-Sandbox zu deaktivieren. Wenn Sie diese Option aktivieren, wird die GameMaker-Sandbox für das Speichern und Laden von Dateien deaktiviert, sodass Ihre Spiele von überall auf dem System, auf dem das Spiel läuft, auf Dateien zugreifen können. Diese Option ist standardmäßig nicht aktiviert, da wir empfehlen, die GameMaker-Sandbox sandbox für Dateien auf allen Systemen beizubehalten, und wenn sie aktiviert ist, sind einige Speicher-/Ladebereiche je nach den Berechtigungen auf Betriebssystemebene möglicherweise immer noch nicht verfügbar. Weitere Informationen zu sandbox finden Sie auf den Seiten über das GameMaker-Dateisystem.

Schließlich haben Sie die Möglichkeit, GameMaker eine x64-Datei für Ihre Windows -Projekte erstellen zu lassen. Wenn diese Option aktiviert ist, wird beim Kompilieren eine ausführbare 64-Bit-Datei für Ihr Spiel erstellt, und wir empfehlen dringend, dass alle neuen Projekte diese Option verwenden. Dies wird langfristig das Standardformat (und das einzige) für die ausführbaren Dateien von Windows sein, da die 32-Bit-Version von runner auslaufen wird. Beachten Sie, dass, wenn diese Option aktiviert ist, alle Erweiterungs-DLLs, die vom Projekt verwendet werden, ebenfalls 64bit sein müssen (siehe hier für weitere Informationen).

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um zu bestimmen, wie Ihr Spiel die Grafikkarte Ihres Ziel-PCs Windows nutzen wird. Die folgenden Optionen können Sie ändern:

  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der normale Windows-Cursor angezeigt. Wenn Sie die Option deaktivieren, wird kein Cursor angezeigt, es sei denn, Sie haben in Ihrem Spiel einen erstellt code. Diese Option ist standardmäßig aktiviert.
  • Vollbildmodus starten: Wenn diese Option aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Diese Option ist standardmäßig deaktiviert.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer switch mit den Standardtastenkombinationen von Windows von Vollbild auf Fenster umschalten und wieder zurück. Diese Option ist standardmäßig deaktiviert.
  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen.
  • Synchronisierung verwenden, um Tearing zu vermeiden: Dies schaltet v-sync ein oder aus. Beachten Sie, dass bei einem Spiel mit einer Raumgeschwindigkeit von 120 und einem Monitor mit einer Bildwiederholrate von 60 die Spielgeschwindigkeit ebenfalls auf 60 festgelegt wird, wenn Sie diese Option aktivieren. Diese Option ist standardmäßig ausgeschaltet.
  • Größenänderung des Fensters zulassen: Wenn diese Option aktiviert ist, kann der Benutzer die Größe des Spielfensters ändern (die Option "Randloses Fenster" muss deaktiviert sein, damit dies funktioniert). Diese Option ist standardmäßig deaktiviert.
  • Randloses Fenster: Wenn Sie diese Option aktivieren, wird Ihr Spiel mit einem randlosen Fenster ausgeführt, wobei die normalen Schaltflächen zum Minimieren, Maximieren und Schließen sowie der Spieltitel entfernt werden. Diese Option ist standardmäßig deaktiviert.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob das Bild vollständig skaliert werden soll (so dass es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Sie die Anwendungsoberfläche ausschalten, werden alle in den Spieloptionen von Windows eingestellten Skalierungsoptionen deaktiviert, bis Sie sie wieder einschalten. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite festzulegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel auf PCs mit niedrigeren Spezifikationen.

Bilder

Im Abschnitt "Bilder" geben Sie die Bilder an, die für Ihr Spiel benötigt werden. Für Windows müssen Sie eine Symboldatei (im Format .ico ) und einen Splash-Screen (im Format .png, .bmp, .jpg oder .gif ) bereitstellen. Der Begrüßungsbildschirm wird beim Laden des Spiels angezeigt und kann durch Aktivieren der Option " Begrüßungsbildschirm verwenden " (die standardmäßig deaktiviert ist) aktiviert oder deaktiviert werden.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird: platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Installateur

Windows-Spiele können mit einem Installer erstellt werden. Hier können Sie die verschiedenen Einstellungen und Grafiken festlegen, die GameMaker bei der Erstellung verwenden wird. Zuallererst haben wir die grafischen Optionen, die das Aussehen des Installers selbst bestimmen:

  • Beendet: Die Grafik, die auf der Seite "Fertig" des Installationsprogramms erscheint. Sie muss im Format *.bmp vorliegen und 164x314px groß sein.
  • Kopfzeile: Die Grafik, die angezeigt wird, während das Installationsprogramm ausgeführt wird. Sie muss das Format *.bmp haben und 150 x 57 Pixel groß sein.

Sie können auch auf eine bestimmte Lizenzvertragsdatei (ein Beispiel wird mit GameMaker mitgeliefert) und eine angepasste NSI script Datei (Details hier) verweisen, falls Sie eine benötigen, oder auf die entsprechenden Schaltflächen klicken, um zu den mit GameMaker gelieferten Standarddateien zurückzukehren.
