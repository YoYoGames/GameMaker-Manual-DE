iOS-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre iOS Spieleprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Der erste Teil des Abschnitts "Allgemein" umfasst die Build-Einstellungen für Ihr Spiel. Zu Beginn legen Sie die Ausrichtungen fest, in denen Ihr Spiel ausgeführt werden kann, um ein optimales Benutzererlebnis zu gewährleisten. Die verfügbaren Optionen sind:

  • Porträt
  • Umgekehrtes Porträt
  • Landschaft
  • Umgekehrte Landschaft

Sie können dann zum Abschnitt Geräteunterstützung gehen, wo die folgenden Optionen zur Verfügung stehen, um festzulegen, welcher Gerätetyp angesprochen werden soll:

  • iPhone oder iPod Touch: iPhone 6 und höher oder iPod Touch-Geräte
  • iPad: iPad-Geräte
  • Beide: Sowohl iPhone/iPod als auch iPad

Als Zweites müssen Sie die Produktinformationen zu Ihrem Spiel ausfüllen, einschließlich des Anzeigenamens, der Bündel-ID und der Versionsinformationen. Beachten Sie, dass die Bundle-ID im korrekten umgekehrten URL-Format angegeben werden muss, damit der endgültige Build des Spiels korrekt funktioniert, z. B.: com.[Company].[GameName]. Hier können Sie auch die Team-Kennung angeben, die Sie zum Signieren der endgültigen Anwendung, die GameMaker für Sie erstellt, verwenden möchten. Wenn Sie sie hier festlegen, wird die Team-ID überschrieben, die Sie in den iOS-Voreinstellungen angegeben haben.

Als Nächstes können Sie die Option Home-Indikator-Ausgabe verschieben aktivieren oder deaktivieren. Wenn diese Option aktiviert ist, wird die Home-Leiste immer bei niedrigem Alpha angezeigt, und der erste Wisch weckt die Leiste auf und der zweite Wisch bringt Sie nach Hause. Wenn die Option nicht aktiviert ist, wird die Leiste im Spiel nicht angezeigt, aber der erste Wisch über die Kante bringt Sie nach Hause. Standardmäßig ist die Option nicht aktiviert.

Schließlich haben Sie die Möglichkeit, eine Poddatei und eine Podfile.lock zu erzeugen. Diese Dateien werden benötigt, wenn Sie CocoaPods mit dem Extension Editor zu einer iOS-Erweiterung hinzufügen. Die Dateien enthalten spezifische Makros, die von GameMaker benötigt werden, um alle aufgelisteten CocaoPods beim Erstellen Ihres Spiels einzurichten. Wenn Sie auf die Schaltflächen klicken, werden die Dateien mit den vorgenerierten Makros erstellt, und es wird ein Dateiexplorer für den Speicherort der Dateien geöffnet, falls Sie sie manuell bearbeiten möchten. Es gibt auch eine Schaltfläche, die Sie zu dem folgenden Leitfaden führt:

  • iOS und tvOS: Verwendung von Cocoa Pods

Grafiken

Hier können Sie die folgenden Details in Bezug auf die Anzeige Ihres Spiels ändern, wobei folgende Optionen zur Verfügung stehen:

  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alphamischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen. Standardmäßig ist diese Option ausgeschaltet.
  • Halbe iPad 1-Texturen: Das iPad 1 hat viel weniger Texturspeicher als neuere Geräte. Daher können Sie diese Option aktivieren, um die Größe der Texturen um die Hälfte zu reduzieren, wenn Sie dieses Gerät ansprechen oder maximale Kompatibilität erreichen möchten.
  • Skalieren: Hier können Sie auswählen, ob die Zeichenfläche unter Beibehaltung des Seitenverhältnisses auf dem Gerätebildschirm skaliert werden soll - wobei die Ränder an den Bildschirm angepasst werden - oder ob die Zeichenfläche an den Gerätebildschirm angepasst werden soll.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite festzulegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau , die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel mit verschiedenen Browsern und Geräten.

Icons

In diesem Abschnitt können Sie die verschiedenen Icons hinzufügen, die für die verschiedenen iPhone- und iPad-Geräte und die verschiedenen Bereiche des iOS App Store erforderlich sind. Diese Symbole sollten als 24-Bit-Bilder *.png in der für jedes Symbol angegebenen Größe erstellt werden.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird: platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überprüfen, um sicherzustellen, dass sie Ihren Anforderungen entsprechen. Das Tool generiert sowohl Icons als auch Bilder (siehe den folgenden Abschnitt).

HINWEIS Wenn Sie ein älteres Projekt importieren, kann die folgende Warnung angezeigt werden:Diese Meldung erscheint, weil die aktuelle Zielversion iOS das Format der erforderlichen Symbole geändert hat, seit das Projekt zuvor geladen wurde. Sie müssen alle Symbole neu erstellen, da sie durch die generischen GameMaker-Symbole ersetzt wurden (Sie können das oben erwähnte Projektbild-Generator-Tool verwenden, um dies schnell zu erledigen).

Bilder

In diesem Abschnitt können Sie separate Grafiken als Startbildschirme (das Bild, das kurz angezeigt wird, während Ihre App geladen wird) für jede der Geräteausrichtungen sowie die Randfarbe hinzufügen, die als Hintergrund für Startbildschirme verwendet wird, die nicht den gesamten Bildschirm ausfüllen sollen. Diese Bilder müssen als einzelne 24-Bit-Bilder *.png ohne Transparenzen und in einer angemessenen Größe erstellt werden. Beachten Sie, dass es sich hierbei nicht um einen Splash-Screen handelt und Apple einige strenge Richtlinien für die Einrichtung und Verwendung dieses Bildes aufgestellt hat. Weitere Informationen finden Sie auf der Entwicklerseite zu Human Interface Guidelines.

Sie können auch festlegen, wie der Startbildschirm angezeigt werden soll, indem Sie die folgenden Optionen wählen:

  • Seitenverhältnis beibehalten: Das Seitenverhältnis des Bildes wird im Vergleich zum Bildschirm des Geräts beibehalten, wobei die Ränder bei Bedarf "aufgefüllt" werden.
  • Auf Größe zuschneiden: Damit wird das Bild 1:1 mit dem Bildschirm des Geräts übereinstimmen und die Ränder werden nach Bedarf zugeschnitten (oder aufgefüllt).
  • Strecken: Damit wird das Bild so gestreckt, dass es den Bildschirm des Geräts ausfüllt (was zu einer Verzerrung des angezeigten Bildes führen kann).
