tvOS-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre tvOS Spieleprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Hier können Sie die Produktinformationen zu Ihrem Spiel eingeben, einschließlich des Anzeigenamens, der Bundle-ID und der Versionsinformationen . Beachten Sie, dass die Bundle-ID im korrekten umgekehrten URL-Format vorliegen muss, damit der endgültige Build des Spiels korrekt funktioniert, z. B.: com.[Company].[GameName]. Sie können auch den Speicherort für die App-Ausgabe auf dem Build-Mac festlegen und die Team-Kennung angeben, die Sie zum Signieren der endgültigen Anwendung, die GameMaker für Sie erstellt, verwenden möchten. Die Einstellung hier überschreibt die Team-ID, die Sie in den tvOS-Einstellungen angegeben haben.

Hier haben Sie auch die Möglichkeit, ein Podfile und ein Podfile.lock zu erzeugen. Diese Dateien werden benötigt, wenn Sie CocoaPods mit dem Extension Editor zu einer tvOS-Erweiterung hinzufügen. Die Dateien enthalten spezifische Makros, die von GameMaker benötigt werden, um alle aufgelisteten CocaoPods beim Erstellen Ihres Spiels einzurichten. Wenn Sie auf die Schaltflächen klicken, werden die Dateien mit den vorgenerierten Makros erstellt, und es öffnet sich ein Dateiexplorer mit dem Speicherort der Dateien, falls Sie sie manuell bearbeiten möchten. Es gibt auch eine Schaltfläche, die Sie zu dem folgenden Leitfaden führt:

  • iOS und tvOS: Verwendung von Cocoa Pods

Grafiken

Hier können Sie die folgenden Details in Bezug auf die Anzeige Ihres Spiels ändern, wobei folgende Optionen zur Verfügung stehen:

  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alphamischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen. Standardmäßig ist diese Option ausgeschaltet.
  • Skalieren: Hier können Sie auswählen, ob die Zeichenfläche unter Beibehaltung des Seitenverhältnisses auf dem Gerätebildschirm skaliert werden soll - wobei die Ränder an den Bildschirm angepasst werden - oder ob die Zeichenfläche an den Gerätebildschirm angepasst werden soll.

Als nächstes können Sie die Größe der Texturseite festlegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau , die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel mit verschiedenen Browsern und Geräten.

Schließlich können Sie wählen, ob ein Cursor angezeigt werden soll oder nicht. Wenn diese Option aktiviert ist, zeigt das Betriebssystem einen Cursor an, wenn die Maus bewegt wird (an der Mausposition), und der Cursor wird ausgeblendet und verschwindet, wenn die Maus eine gewisse Zeit lang nicht bewegt wird. Ist das Häkchen nicht gesetzt, wird kein Cursor angezeigt.

Icons

In diesem Abschnitt können Sie die verschiedenen Icons hinzufügen, die für die verschiedenen Geräte und die verschiedenen Bereiche des App Stores erforderlich sind. Diese Icons sollten als 24-Bit-Bilder *.png in der jeweils angegebenen Größe erstellt werden.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen, insbesondere für dieses platform, da die benötigten Symbole rechteckig und nicht quadratisch sind. Dieses Tool erzeugt neben den Icons auch Bilder (siehe unten).

Bilder

In diesem Abschnitt können Sie separate Grafiken als "Top Shelf"-Bilder und Splash-Screens für jede der verschiedenen Zielgrößen von tvOS hinzufügen. Diese Bilder müssen als einzelne 24-Bit-Bilder *.png ohne Transparenzen und in der richtigen Größe erstellt werden. Sie können auch eine Mindestdauer für die Anzeige der Startbildschirme im Dropdown-Menü am unteren Rand festlegen (von 0 bis 10 Sekunden).

Wie im Abschnitt über Symbole erwähnt, können Sie den Projektbildgenerator verwenden, um diese Bilder zu erstellen.
