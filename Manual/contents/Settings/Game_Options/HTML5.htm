HTML5-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre HTML5-Spieleprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

In diesem Abschnitt können Sie die folgenden Informationen über Ihr Spiel eingeben:

  • Browsertitel: Hier können Sie einen Text eingeben, der für die Titelleiste der Registerkarte für Ihr Spiel im Browser verwendet werden soll.
  • Version: Die Versionsnummer für Ihr Spiel.
  • Name des Ordners: Der Name des Ordners, in dem Ihre HTML5-Spieldateien gespeichert werden (Standard ist "html5game").
  • Name der Ausgabe: Dies ist der Name der HTML-Ausgabedatei und kann in einen anderen Namen als den Standardnamen "index.html" geändert werden.

Unterhalb der Spielinformationsfelder haben Sie außerdem die folgenden Optionen:

  • Debug-Ausgabe an die Konsole: Damit werden alle Debug-Ausgaben an die JS-Konsole Ihres Browsers gesendet. Dies ist standardmäßig deaktiviert.
  • Cursor anzeigen: Damit wird der Mauszeiger deaktiviert, wenn er sich über der Spielfläche befindet. Diese Option ist standardmäßig aktiviert.
  • Warnung "Außerhalb des Servers ausgeführt" anzeigen: Wenn Sie ein Spiel lokal testen (d. h., Sie klicken auf die Indexdatei auf Ihrem Computer und nicht auf server - nicht empfohlen), erhalten Sie normalerweise eine Warnung, die Sie darüber informiert, dass die Dinge aufgrund der Browsersicherheit für lokale Dateien nicht so funktionieren, wie Sie es vielleicht erwarten. Sie können diese Warnung unterdrücken, indem Sie das Häkchen bei dieser Option entfernen. Sie ist standardmäßig aktiviert.

Unterhalb der allgemeinen Einstellungen finden Sie die erweiterten Optionen, die wie folgt lauten

  • Eingeschlossene Datei als index.html: Hier können Sie eine benutzerdefinierte "index.html"-Datei hinzufügen, die anstelle der vordefinierten Datei verwendet wird, die GameMaker standardmäßig generiert. Dies kann sehr nützlich sein, wenn Sie ein Website-Design implementiert haben oder bestimmte Funktionen in alle Ihre HTML5-Builds aufnehmen möchten. Die zu verwendende Datei muss als eingeschlossene Datei in das Spielprojekt aufgenommen werden.
  • Ladebalken-Erweiterung: Damit wird der Standard-Ladebalken durch einen Ladebalken ersetzt, der als Erweiterung zu Ihrem GameMaker-Projekt hinzugefügt wurde.
  • Prepend output .js: Hier können Sie der Ausgabe von *.js ein zusätzliches JavaScript voranstellen, das für Ihr Spiel erforderlich ist. Für eine Chrome Store-App kann es beispielsweise erforderlich sein, dass Sie window.localStorage = undefined; zur JS-Ausgabe hinzufügen, wofür Sie diese Option verwenden würden.

Analytik

In diesem Abschnitt können Sie Ihrem Spiel Analysen hinzufügen und bestimmte Produkte von Drittanbietern verwenden, um Ihr Spiel und seine Verkäufe oder Spiele zu verfolgen. Sie können angeben, ob Sie Flurry Analytics oder Google Analytics verwenden möchten, um Ihr Spiel zu verfolgen. Sobald Sie Flurry Analytics oder Google Analytics aktivieren markiert haben, sollten Sie auch die eindeutige ID, die Ihnen zugewiesen wurde, in den entsprechenden Bereich eingeben.

Informationen zu den Funktionen, die Sie zum Senden von Analysedaten verwenden können, finden Sie hier.

Grafiken

Hier können Sie die folgenden Angaben zur Darstellung Ihres Spiels ändern (beachten Sie, dass Ihr Spiel nicht in allen Browsern gleich angezeigt wird. Testen Sie diese Optionen daher in möglichst vielen Browsern, um sicherzustellen, dass sie die gewünschte Wirkung haben):

  • Umschalten auf Vollbildmodus zulassen: Wenn diese Option aktiviert ist, kann der Benutzer das Spiel mit / + "F" auf Mac und F10 auf Windows vom Fenster- in den Vollbildmodus umschalten. Diese Option ist standardmäßig aktiviert.
  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alphamischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen. Diese Option ist standardmäßig aktiviert.
  • Das Spiel im Browser zentrieren: Wenn Sie diese Option aktivieren, wird der Spielbildschirm automatisch im Browser zentriert, in dem das Spiel ausgeführt wird. Diese Option ist standardmäßig deaktiviert.
  • Eingebaute Partikel verwenden: Diese Option exportiert die sprites, die für das eingebaute Partikelsystem benötigt wird, zusammen mit Ihrem Spiel. Diese sprites sind in einzelnen Texturseiten gespeichert, was bedeutet, dass sie nicht die effizienteste Methode zur Erzeugung von Partikeln für dieses platform sind. Daher wird empfohlen, dass Sie (sofern Sie nicht WebGL verwenden) diese Option deaktivieren und Ihre eigenen Partikel sprites verwenden, die auf den Haupttexturseiten für Ihr Spiel gespeichert werden. Diese Option ist standardmäßig aktiviert.
  • Eingebaute Schriftart verwenden: Mit dieser Option wird die in GameMaker eingebaute Schriftart zusammen mit den restlichen Spieldateien exportiert. Dadurch wird die Größe der Texturseiten leicht erhöht und es wird etwas mehr Speicherplatz benötigt. Wenn Ihr Spiel also nicht die eingebaute Schriftart Arial verwendet, sollte diese Option deaktiviert werden. Sie ist standardmäßig aktiviert.
  • WebGL: Wählen Sie, ob switch die WebGL-Unterstützung deaktivieren, optional machen (automatische Erkennung) oder zur Voraussetzung für Ihr Spiel machen soll. Um die Leistung unter allen Umständen zu testen, sollten Sie diese Option deaktivieren und für die Veröffentlichung normalerweise auf automatische Erkennung setzen. Wenn Sie dies zur Bedingung machen, kann dies die Kompatibilität Ihres Spiels mit verschiedenen Browsertypen erheblich beeinträchtigen. Standardmäßig ist dies auf automatische Erkennung eingestellt.
  • Skalierung: Ihr Spiel kann so konfiguriert werden, dass die Zeichenfläche automatisch skaliert wird, um das Seitenverhältnis im Browser beizubehalten , oder Sie können wählen, dass es in voller Größe ausgeführt wird. Bei der Option "Voller Maßstab" wird das Spiel im Browser nicht im Vollbildmodus angezeigt, sondern das Gezeichnete wird so gestreckt, dass es in die Leinwandgröße passt, die durch den ersten Raum des Spiels definiert ist. Das Seitenverhältnis wird standardmäßig beibehalten.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite festzulegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel mit verschiedenen Browsern und Geräten.

Bilder

In diesem Abschnitt können Sie das Favicon und den Splash Screen definieren. Das Icon sollte als *.ico Datei erstellt werden und kann 16x16, 32x32, 48x48 oder 64x64 Pixel groß sein und eine Farbtiefe von 8-bit, 24-bit oder 32-bit haben. Der Begrüßungsbildschirm sollte die Größe der Leinwand haben, auf der das Spiel angezeigt wird, und überschreibt alle Ladebalken, wenn Sie die Option "Begrüßungsbildschirm verwenden" aktivieren (d. h. Sie können einen Ladebalken ODER einen Begrüßungsbildschirm haben, aber nicht beides). Kleinere oder größere Bilder sind erlaubt, werden aber skaliert, damit sie passen. Der Startbildschirm sollte ein *.png Bild sein.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird: platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Soziales

Die Registerkarte "Soziales" dient zum Einrichten der Facebook-Kommunikation in Ihrem Spiel. Wenn Sie die Facebook-Funktionen benötigen, müssen Sie das Kontrollkästchen "Facebook verwenden" anklicken und dann die Facebook-App-ID und den Facebook-App-Anzeigenamen angeben. Beachten Sie, dass der Zugriff auf alle Facebook-Funktionen über eine Erweiterung erfolgt. Wenn Sie das Kästchen " Facebook verwenden" zum ersten Mal anklicken, werden Sie aufgefordert, die Facebook-Erweiterung vom Marktplatz herunterzuladen und zu installieren (möglicherweise müssen Sie sich in Ihr YoYo-Konto einloggen, bevor sie heruntergeladen werden kann).
