macOS-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre macOS Spieleprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Im Abschnitt " Produktinformationen" müssen Sie zunächst den Anzeigenamen Ihres Spiels und dann die Versionsnummer, die App-ID, die App-Ausgabedatei und die Copyright-Angaben eingeben. Die App-Ausgabedatei ist der Speicherort, an dem die Installationsdateien Ihres Spiels auf Ihrem Mac-Entwicklungscomputer abgelegt werden. Wenn Sie Ihre .app.zip erstellen, kann diese Datei an einem beliebigen Ort gespeichert werden (der Speicherort, den Sie beim Speichern angeben, ist nicht die einzige Kopie, die von dieser .app.zip erstellt wird), aber die .pkg und andere Dateien werden in diesem Ordner belassen, so dass Sie lokale Installationen über Terminal testen können, um den Kauf im Mac Store zu simulieren usw..

Hier müssen Sie auch die Team-Kennung angeben, die Sie zum Signieren der endgültigen Anwendung, die GameMaker für Sie erstellt, verwenden möchten. Die Einstellung hier überschreibt die Team-ID, die Sie in den macOS-Einstellungen angegeben haben. Nach der Team-Kennung haben Sie auch die Möglichkeit, die Signier-Kennung anzugeben, die von Apple für alle Nicht-Appstore-Anwendungen (wie z. B. Steam Spiele) benötigt wird.

Die nächste Option betrifft die Aktivierung von Steam in Ihrem Spiel. Wenn Sie diese Option aktivieren, stellen Sie sicher, dass Sie das Steamworks SDK installiert haben (siehe hier) und dass Sie die Steam-App-ID in den allgemeinen Optionen für das Spiel hinzugefügt haben.

Schließlich haben Sie die Möglichkeit, die Dateisystem-Sandbox zu deaktivieren. Wenn Sie diese Option aktivieren, wird die GameMaker-Sandbox für das Speichern und Laden von Dateien deaktiviert, sodass Ihre Spiele von überall auf dem System, auf dem das Spiel läuft, auf Dateien zugreifen können. Diese Option ist standardmäßig nicht aktiviert, da wir empfehlen, die GameMaker-Sandbox sandbox für Dateien auf allen Systemen beizubehalten, und wenn sie aktiviert ist, sind einige Speicher-/Ladebereiche je nach den Berechtigungen auf Betriebssystemebene möglicherweise immer noch nicht verfügbar. Weitere Informationen zu sandbox finden Sie auf den Seiten über das GameMaker-Dateisystem.

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres Ziel-Macs nutzen wird. Die folgenden Optionen können Sie ändern:

  • Menü und Dock im Vollbildmodus zulassen: Wenn diese Option aktiviert ist, werden das Betriebssystemmenü und das Dock angezeigt, wenn sich das Spiel im Vollbildmodus befindet. Diese Option ist standardmäßig deaktiviert.
  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der reguläre Cursor macOS angezeigt. Wenn Sie die Option deaktivieren, wird kein Cursor angezeigt, es sei denn, Sie haben einen in Ihrem Spiel code erstellt. Diese Option ist standardmäßig deaktiviert.
  • Vollbildmodus starten: Wenn diese Option aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Diese Option ist standardmäßig deaktiviert.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer switch mit den Standardtastenkombinationen von macOS vom Vollbild zum Fenster und wieder zurück wechseln. Diese Option ist standardmäßig deaktiviert.
  • Interpoliert die Farben zwischen den Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alphamischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen. Standardmäßig ist diese Option ausgeschaltet.
  • Synchronisierung verwenden, um Tearing zu vermeiden: Dies schaltet v-sync ein oder aus. Beachten Sie, dass bei einem Spiel mit einer Raumgeschwindigkeit von 120 und einem Monitor mit einer Bildwiederholrate von 60 die Spielgeschwindigkeit ebenfalls auf 60 festgelegt wird, wenn Sie diese Option aktivieren. Diese Option ist standardmäßig ausgeschaltet.
  • Größenänderung des Fensters zulassen: Wenn diese Option aktiviert ist, kann der Benutzer die Größe des Spielfensters ändern (die Option "Randloses Fenster" muss deaktiviert sein, damit dies funktioniert). Diese Option ist standardmäßig deaktiviert.
  • Retina aktivieren: Wenn diese Option aktiviert ist, wird die Rückseite von buffer auf die höhere (tatsächliche) Auflösung eingestellt, wenn das Spiel auf einem Retina-fähigen Monitor ausgeführt wird, während die Rückseite von buffer auf die scheinbare (niedrigere) Auflösung eingestellt wird, wenn diese Option nicht aktiviert ist. Das bedeutet, dass das anfängliche Spielfenster auf die doppelte Breite und Höhe des Raums/Ansichtsport eingestellt wird, auf den der erste Raum eingestellt ist. Wenn Sie also möchten, dass Ihr Spiel die Retina-Anzeige optimal nutzt, sollten Sie die Anwendungsoberfläche auf die gleiche Größe wie das Fenster einstellen (beachten Sie, dass dadurch die Pixel in der Breite und Höhe verdoppelt werden, aktivieren Sie diese Option also nur und skalieren Sie die Anwendungsoberfläche, wenn Ihr Spiel von der höheren Auflösung profitieren soll).

    Diese Option ist standardmäßig ausgeschaltet, und wenn Sie sie aktivieren, wird es keinen Unterschied machen, wenn Sie das Spiel mit VM ausführen oder debuggen, da die runner in diesem Fall bereits vordefiniert ist. Sie funktioniert nur, wenn Sie eine ausführbare Datei über VM erstellen, oder YYC zum Testen oder Erstellen des Spiels verwenden, da dies die runner mit Ihren angepassten Spieloptionen neu erstellt.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob das Bild vollständig skaliert werden soll (so dass es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Sie die Anwendungsoberfläche ausschalten, werden alle in den Spieloptionen von Windows eingestellten Skalierungsoptionen deaktiviert, bis Sie sie wieder einschalten. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite festzulegen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese platform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen über Texturseiten finden Sie hier.

HINWEIS: Je größer die Texturseite ist, desto weniger kompatibel ist Ihr Spiel auf PCs mit niedrigeren Spezifikationen.

Bilder

Im Abschnitt "Bilder" geben Sie die Bilder an, die für Ihr Spiel erforderlich sind. Für Mac müssen Sie eine Icon-Datei (im Format .png und 1024x1024px) und einen Splash-Screen (im Format .png, .bmp, .jpg oder .gif ) angeben. Der Begrüßungsbildschirm wird angezeigt, während das Spiel geladen wird.

Für DMG-Pakete können Sie ein Installer-Hintergrundbild angeben, das als Hintergrund des DMG-Installationsprogramms erscheint.

Es ist erwähnenswert, dass GameMaker ein Projektbild-Generator-Tool hat, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Ziele benötigt werden, für die Ihr Spiel kompiliert wird: platforms. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Verpackung

Wenn Sie möchten, dass das fertige Spiel App-Store-fähig ist, müssen Sie die Option Build for Mac App Store aktivieren. Beachten Sie jedoch, dass dies nur funktioniert, wenn Sie ein registrierter Entwickler sind und die erforderlichen Zertifikate besitzen. Darunter befinden sich die App-Berechtigungen , die Sie nur ankreuzen sollten, wenn sie zutreffend sind, da Apple Ihre App ablehnen kann, wenn sie angekreuzt sind, obwohl sie nicht benötigt werden, oder umgekehrt. Diese Optionen erlauben Ihrem Spiel lediglich, die Funktionen http_ und url_ zu nutzen. Wenn Sie GamePads in Ihrem Spiel unterstützen möchten, sollten Sie die Option zum Erstellen eines App Store-fertigen Pakets deaktivieren.

Schließlich müssen Sie die App-Kategorie auswählen (weitere Informationen finden Sie hier).

Soziales

Diese Option ist für die Verwendung der Apple Sign In Extension auf dem Mac. Sie können diese Erweiterung über den GameMaker Marketplace beziehen, und die Erweiterung selbst enthält eine vollständige Anleitung zur Verwendung. Wenn Sie diese Erweiterung nicht verwenden, sollten Sie diese Option nicht ankreuzen.
