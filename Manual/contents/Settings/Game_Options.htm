Spiel-Optionen

Die Spieloptionen sind in zwei Bereiche unterteilt, die Hauptoptionen und die Platform Einstellungen. Die Hauptoptionen beginnen mit den allgemeinen Optionen, in denen Sie bestimmte Einstellungen vornehmen können, die sich auf das Spiel auswirken, unabhängig von der platform, auf der es läuft.

Das Fenster besteht aus den folgenden Bereichen:

  • Standard-Skriptsprache: Wählen Sie, ob Objects events und Scripts im aktuellen Projekt GML Code oder GML Visual verwenden soll, oder ob es immer beim Erstellen eines neuen script nachfragen soll.
  • Spiel-Frames pro Sekunde: Dieser Wert legt die Grenze für die Anzahl der Frames pro Sekunde fest, die GameMaker versuchen soll, Ihre Spiele während der Ausführung beizubehalten. Dieser Wert gibt an, wie oft das Spiel loop in einer Sekunde abläuft (ein einzelnes Spiel loop wird als einzelner "Frame" betrachtet) und ist auf einen Standardwert von 60 eingestellt, obwohl auch oft 30 verwendet wird. Es gibt viele arguments über die ideale Raumgeschwindigkeit, aber die tatsächliche Antwort hängt davon ab, welche Art von Spiel Sie machen, die Größe des Raums, die Anzahl der Instanzen und wie intensiv die code ist. Wenn du zum Beispiel einen großen Raum und eine Raumgeschwindigkeit von 30 hast, kann das Spiel "abgehackt" erscheinen, denn je größer der Bereich ist, den du betrachtest, desto einfacher ist es für das Auge, niedrige Raumgeschwindigkeiten wahrzunehmen. Wenn das Spiel jedoch sehr intensiv ist, kann die Einstellung der Raumgeschwindigkeit auf 120 dazu führen, dass das Spiel "hinkt", da die Bildwiederholrate aufgrund des Verarbeitungsaufwands sinkt. Seien Sie also vorsichtig mit diesem Wert und versuchen Sie, ihn immer so einzustellen, wie es Ihr Spiel benötigt.
  • Standard-Zeichenfarbe: Die Standardfarbe zum Zeichnen von Text oder Foren (und anderen Dingen), wenn keine Farbe mit code eingestellt wurde.
  • Standardfarbe zum Zeichnen außerhalb des Raums: Die Standardfarbe, die verwendet wird, um den Hintergrund eines Bereichs außerhalb des Spielraums zu löschen.
  • Projektbilder generieren: Wenn Sie darauf klicken, wird das Werkzeug Projektbildgenerator geöffnet.
  • Datum des Projektbeginns: Das Datum, an dem das Projekt begonnen wurde.
  • Projektnutzungsdauer: Die Zeitspanne, die vergangen ist, seit Sie das Projekt initiiert haben.
  • Projekt-GUID: Dies ist die eindeutige Kennung, die für jedes Projekt in GameMaker erstellt wird. Wenn Sie sie ändern möchten, können Sie auf die Schaltfläche Neue GUID erstellen klicken, um eine neue zu erzeugen.
  • Mipmaps für separate Texturseiten generieren: Mit dieser Option erstellt GameMaker Mipmaps für diejenigen sprites, die in den Textureinstellungen des Sprite Editors die Option "Separate Texturseite" markiert haben. Das Aktivieren von Mipmapping bedeutet, dass die ursprüngliche Texturseite für die sprite skaliert und in mehrere Auflösungen innerhalb der Texturdatei gefiltert wird. Normalerweise ist dies ausgeschaltet, aber wenn Sie Ansichten verwenden, um große Bereiche des Spielraums zu vergrößern oder zu verkleinern, oder wenn Sie mit einem 3D-Spiel arbeiten, dann sollten Sie dies aktivieren und dann die Mipmapping-Funktionen verwenden, um einzustellen, wie das Spiel mit den verschiedenen Mipmap-Ebenen umgeht. Beachten Sie, dass Sie, wenn Sie sprites nicht als individuelle Texturseiten kennzeichnen, die Mipmapping-Funktion für jede einzelne Texturgruppe über den Texturgruppen-Manager aktivieren können. Diese Option ist standardmäßig ausgeschaltet.
  • Versionskontrolle aktivieren: Wenn Sie möchten, dass das Projekt unter SCM (Source Control Management) steht, müssen Sie diese Option aktivieren. Standardmäßig ist diese Option deaktiviert (weitere Informationen zur Versionskontrolle finden Sie hier).
  • Autor: Der Name des Spieleautors (oder Herausgebers oder Studios).
  • Letzte Änderung des Projekts: In diesem Abschnitt wird lediglich der Datums- und Zeitstempel für die letzte Änderung des Projekts angezeigt (nicht bearbeitbar).
  • Spine Lizenz akzeptieren: Hier können Sie das Kästchen ankreuzen, um die Spine Runtime Lizenz anzuzeigen und sie zu akzeptieren. Dies erlaubt Ihnen, Spine sprites in Ihrem Spiel zu verwenden (diese Lizenz wird auch angezeigt, wenn Sie zum ersten Mal versuchen, Spine sprite zu Ihrem Projekt hinzuzufügen und die Lizenz hier nicht vorher angekreuzt haben).
  • Kollisionskompatibilitätsmodus: Hiermit wird GameMaker angewiesen, das alte Kollisionssystem zu verwenden, das vor der Version 2022 verwendet wurde (das Ganzzahlwerte anstelle von Fließkommazahlen verwendete). Lesen Sie Kollisionskompatibilitätsmodus für weitere Informationen. Diese Option ist standardmäßig deaktiviert und es wird empfohlen, sie für neue Projekte deaktiviert zu lassen.
  • Verhalten "Kopieren beim Schreiben" für Arrays aktivieren: Dies aktiviert das veraltete Verhalten "Kopieren beim Schreiben " für das aktuell geöffnete Projekt. Lesen Sie Copy on Write für weitere Informationen. Diese Option ist standardmäßig deaktiviert und es wird empfohlen, sie für neue Projekte deaktiviert zu lassen.

Der Abschnitt Hauptoptionen enthält auch ein Menü für Vorlageninformationen.

Im Asset-Browser stehen Ihnen je nach Ziel platforms weitere Spieloptionen zur Verfügung, die für die von Ihnen erworbene Lizenz verfügbar sind.

HINWEIS Die verschiedenen Optionen für Konsolenspiele sind hier nicht aufgelistet, da die verschiedenen Unternehmen rechtliche Beschränkungen aufrechterhalten. Wenn Sie eine Konsolenlizenz haben, können Sie diese Informationen vom YoYo Games Helpdesk erhalten, aber Sie müssen zuerst YoYo Games mit Ihrer Konsolen-ID-E-Mail kontaktieren und um die Erlaubnis bitten, die verschiedenen Bereiche einzusehen.

Einige GameMaker-Projekte zeigen möglicherweise auch Spieloptionen für das Amazon Fire-Ziel an. Dabei handelt es sich um veraltete, schreibgeschützte Optionen, da GameMaker das Fire platform jetzt direkt über die Android-Spieloptionen unterstützt.
