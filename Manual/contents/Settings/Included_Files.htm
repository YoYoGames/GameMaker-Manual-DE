Enthaltene Dateien

GameMaker ist nicht nur auf die Assets beschränkt, die Sie im Asset-Browser hinzufügen, sondern Sie können dem Projekt auch Dateien als "Included Files" hinzufügen, auf die Sie dann während des Spiels zugreifen können. Dies sind einfach Dateien, die in Ihrem Spielpaket enthalten sind, so dass Sie mit ihnen machen können, was Sie wollen.

Welche Arten von Dateien Sie hinzufügen, hängt davon ab, was Sie tun möchten, aber Dinge wie Standarddateien *.ini, sprites und Hintergrundbilder oder Tabellenkalkulationen werden häufig als eingeschlossene Dateien verwendet.

Einbezogene Dateien hinzufügen

Am einfachsten fügen Sie eine eingebundene Datei hinzu, indem Sie sie aus dem Explorer/Finder auf IDE ziehen. Sie werden gefragt, ob die gezogene Datei zu den eingebundenen Dateien hinzugefügt werden soll. Nachdem Sie auf Ja gedrückt haben, öffnet sich das Fenster Enthaltene Dateien, das die soeben hinzugefügte Datei enthält.

Fenster "Enthaltene Dateien

Öffnen Sie das Menü Extras des Asset-Browsers (Abb. 1), um das Fenster Enthaltene Dateien manuell zu öffnen (Abb. 2):

Dieses Fenster zeigt Ihnen nur die in Ihrem Projekt enthaltenen Dateien an. Um Ihre Dateien neu anzuordnen oder umzubenennen, müssen Sie das eigentliche Verzeichnis bearbeiten, in dem diese Dateien gespeichert sind.

Wenn Sie auf Im Explorer/Finder öffnen klicken, wird der Ordner datafiles in Ihrem Projektverzeichnis geöffnet, der alle eingeschlossenen Dateien enthält. Sie können in diesem Verzeichnis nach Belieben Dateien hinzufügen oder entfernen, und die Oberfläche von GameMaker wird automatisch aktualisiert, um die Struktur in Ihrem datafiles-Ordner widerzuspiegeln.

Sobald Sie Dateien zum Editor hinzugefügt haben, können Sie auf doppelklicken, um die Dateieigenschaften zu öffnen:

In diesem Fenster werden der Name der Datei, ihre Größe auf der Festplatte und die platforms angezeigt, in die sie exportiert werden soll, wenn Sie Ihr Spiel kompilieren/testen. Es gibt Schaltflächen, mit denen Sie "Alle" oder "Keine" auswählen können, oder Sie können in der Liste nach unten gehen und die Ziele, in die die Datei exportiert werden soll, ankreuzen oder abwählen.

Weitere Informationen darüber, wie eingebundene Dateien in Ihrem Spiel gespeichert werden und wie auf sie zugegriffen wird, finden Sie unter Dateisystem.

WICHTIG Wenn Sie Sounddateien einbinden, sollten Sie darauf achten, dass die Dateien keine Namen tragen, die mit Streaming-Sound-Assets kollidieren. Ein Sound-Asset mit dem Namen "main_music" führt zum Beispiel dazu, dass eine eingebundene Datei mit dem Namen "main_music.ogg" bei der Ausführung des Projekts undefiniert wird. Dies liegt daran, dass gestreamte Dateien auch außerhalb des Hauptprojekt-Bundles gespeichert werden, so dass in diesen Situationen Vorsicht geboten ist.

Sie können auch DLL (auf Windows) oder DyLib (auf macOS) Dateien als eingeschlossene Dateien einbinden und dann Ihre eigenen benutzerdefinierten Funktionen erstellen, um mit ihnen umzugehen, indem Sie external_define(). Diese können dann verwendet werden, indem man die external_call() Funktion aufrufen.

HINWEIS Die Verwendung von eingeschlossenen Dateien, um auf diese Weise zusätzliche Funktionalität zu schaffen, ist mit dem Erweiterungssystem nicht kompatibel, da Sie nicht die extern_*() Funktionen nicht mit Dateien verwenden können, die in einer Erweiterung hinzugefügt wurden. Erweiterungen verfügen bereits über eine Methode zum Hinzufügen von Konstanten und Funktionen.
