Konfigurationen

Aufgrund der Anforderungen der Cross platform Entwicklung und der Notwendigkeit, ein Produkt für verschiedene Clients oder Geräte oder platforms verfügbar zu machen und anzupassen, enthält GameMaker Konfigurationen, um die Dinge zu vereinfachen. Mit diesem System können Sie schnell switch zwischen verschiedenen Startbildschirmen, Symbolen und enthaltenen Dateien wechseln und jederzeit nur diejenigen testen oder exportieren, die für das Ziel platform oder den Client relevant sind, und das alles von derselben Basis code aus.

Um den unterschiedlichen Anforderungen von Kunden und platforms gerecht zu werden, können Sie den Konfigurationseditor verwenden, der über das Menü Extras im Asset-Browser zugänglich ist:

Alle mit GameMaker erstellten Spiele beginnen mit einer "Standardkonfiguration", die beim ersten Öffnen des Editors aufgelistet wird:

Um Ihre Konfigurationen einzurichten, empfiehlt es sich, zunächst diese "Standardkonfiguration" vorzubereiten, indem Sie alle konfigurierbaren Assets für das Projekt in sie aufnehmen. Gehen Sie dazu einfach die folgende Liste von Optionen durch und setzen Sie sie auf die standardmäßig erforderlichen Werte:

  • Erweiterungsoptionen: Setzen Sie die verschiedenen "Kopieren nach..."-Felder auf die für die Konfiguration erforderlichen platforms. Alle hier erstellten "Proxy"-Dateien werden mit ihrem "Ziel" platform ebenfalls mit der Konfiguration verknüpft.
  • Sprite-Editor: Legen Sie die Texturgruppe fest, zu der die sprites hinzugefügt werden soll.
  • Tileset-Editor: Legen Sie die Texturgruppe fest, zu der die Tilesets hinzugefügt werden sollen.
  • Schriftarten-Editor: Legen Sie die Texturgruppe fest, zu der die Schriftarten hinzugefügt werden sollen.
  • Texturgruppen-Manager: Hier können Sie alle Eigenschaften pro Konfiguration einstellen, einschließlich der übergeordneten Gruppe, zu der eine Texturgruppe gehört.
  • Audiogruppen-Manager: Die Klänge, die den verschiedenen Audiogruppen zugewiesen werden, können hier konfiguriert werden...
  • Enthaltene Dateien: Setzen Sie das Feld "Kopieren nach..." auf die für die Konfiguration erforderliche platforms.

Sobald Sie die Standardkonfiguration eingerichtet haben, können Sie dem Konfigurationseditor eine neue Konfiguration hinzufügen, indem Sie mit der rechten Maustaste auf auf den Ordner "configurations" klicken und " Add Config" auswählen oder auf die Schaltfläche "Create" am unteren Rand des Fensters klicken. Die neue Konfiguration wird als "Kind" der Standardkonfiguration angezeigt - alle benutzerdefinierten Konfigurationen sind Kinder der Standardkonfiguration - und Sie können ihr einen passenden Namen geben.

Um die Konfiguration zu bearbeiten, müssen Sie entweder mit einem Doppelklick auf oder mit der rechten Maustaste im Menü "Set Config" die Konfiguration im Editor auswählen. Dazu gehen Sie einfach die oben aufgeführten Optionen durch und ändern sie auf die gewünschten Einstellungen für die neu erstellte Konfiguration. GameMaker speichert diese Werte automatisch für die Konfiguration, und Sie können jederzeit zwischen den Konfigurationen wechseln, indem Sie einfach die Menüoption RMB verwenden oder auf doppelklicken.

Weitere Konfigurationen können entweder als Unterkonfigurationen der Standardkonfiguration oder als Unterkonfigurationen der von Ihnen erstellten benutzerdefinierten Konfigurationen hinzugefügt werden, je nachdem, welche Konfiguration beim Erstellen im Editor hervorgehoben wurde. Im Beispielbild unten sehen Sie, dass wir drei benutzerdefinierte Konfigurationen definiert haben, von denen zwei Kinder der Standardkonfiguration sind und eine weitere das Kind einer benutzerdefinierten Konfiguration ist:

Es ist erwähnenswert, dass Sie beim Kompilieren oder Testen Ihrer Projekte jede beliebige Konfiguration mit jedem beliebigen Target platform verwenden können. Dazu wählen Sie die gewünschte Konfiguration im Abschnitt Config des Target Managers aus:

Sie können den Konfigurations-Editor auch jederzeit vom Target Manager aus öffnen, indem Sie auf die Menü-Schaltfläche oben rechts klicken (wie in der Abbildung oben hervorgehoben).

Sobald Sie eine Konfiguration auf diese Weise festgelegt haben, wird jedes Mal, wenn Sie zu einem Ziel kompilieren, sei es zum Testen oder zum Erstellen eines endgültigen Asset-Pakets, die gewählte Konfiguration verwendet. Weitere Informationen über das Fenster "Ziele" finden Sie im Abschnitt " Kompilieren".
