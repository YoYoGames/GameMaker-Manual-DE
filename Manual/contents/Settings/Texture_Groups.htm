Textur-Gruppen

Der Texturgruppen-Manager ist über das Menü "Werkzeuge" auf IDE verfügbar. Hier können Sie Texturgruppen hinzufügen, löschen und umbenennen.

GameMaker erlaubt es Ihnen, jedes der Grafik-Assets (sprites, Tilesets und Fonts) verschiedenen Texturgruppen zuzuordnen, um die Anzahl der Texturwechsel zu optimieren, die die Hardware zu einem bestimmten Zeitpunkt in Ihrem Spiel durchführt. Damit das funktioniert, müssen Sie die erforderlichen Texturgruppen in diesem Fenster definieren und dann jeden der entsprechenden Asset-Typen durchgehen und die Gruppe, zu der er gehören soll, aus dem Dropdown-Menü auswählen.

Wenn Sie mehrere für eine Texturgruppe aufgelistete Assets auswählen und mit der rechten Maustaste klicken, können Sie diese Assets in eine andere Gruppe verschieben:

Alternativ können Sie einer Texturgruppe auch Assets über das Menü der rechten Maustaste im Asset-Browser hinzufügen. Sie können ein oder mehrere Sprite-/Kachelset-/Schriften-Assets auswählen und dann das RMB-Menü verwenden, um die Assets zu einer beliebigen Gruppe hinzuzufügen, wie unten für Kachelsets gezeigt:

ist es wichtig zu verstehen, dass eine einzelne Texturgruppe mehrere Texturseiten haben kann und diese nicht gleichwertig sind. Wenn Ihr Spiel viele einzigartige Grafiken für jede Ebene hat, dann können Sie eine einzelne Texturgruppe für jede Ebene definieren, aber die Grafiken können zwei oder mehr Texturseiten pro Gruppe erfordern. Dies hängt davon ab, ob die gewählte Texturseitengröße (in den Spieloptionen für Ihr Ziel) für alle Grafiken innerhalb einer Texturgruppe ausreicht.

Standardmäßig werden alle Bilder in Ihrem Spiel der Texturgruppe "Standard" hinzugefügt, die immer vorhanden ist. Sie können jedoch auf die Schaltfläche " Neu hinzufügen" klicken, um eine neue Texturgruppe zu erstellen, und wenn Sie den Namen der Gruppe auswählen, können Sie ihn entsprechend ändern. Zu Beginn zeigt die Gruppe "Diese Gruppe ist leer" für jeden Abschnitt der Gruppe an (sprites, Kachelsätze und Schriftarten), so dass Sie Ihre Grafikressourcen durchgehen und sie einer Gruppe hinzufügen müssen. Sie können eine Ressource über die Schaltfläche " Ressource hinzufügen " am unteren Rand hinzufügen oder indem Sie die Texturgruppe im Ressourceneditor für das Bild, das Kachelset oder die Schriftart auswählen. Andere Texturgruppen als die "Standard"-Gruppe können auch jederzeit gelöscht werden, wodurch ihr Inhalt in die "Standard"-Gruppe verschoben wird (beachten Sie, dass Sie, wenn Sie eine Gruppe versehentlich löschen, / + "Z" verwenden können, um die Aktion rückgängig zu machen).

Sobald Sie Texturgruppen definiert - und ihnen Grafik-Assets zugewiesen - haben, werden sie links im Hauptfenster des Texturgruppen-Editors aufgelistet, und wenn Sie auf eine der sprites in dieser Liste klicken, werden sie im Bildvorschaufenster rechts angezeigt. Dies ist besonders hilfreich, wenn Sie an großen Projekten arbeiten und sehen müssen, welche Bilder welcher Gruppe zugewiesen wurden. Wenn die der Gruppe zugewiesenen Bilder mehrere Unterbilder haben (d. h., sie sind animiert), wird nur das erste Unterbild im Vorschaufenster angezeigt. Sie können auch mit der rechten Maustaste auf eine Ressource klicken, um ein kleines Menü zu öffnen, das Ihnen die Möglichkeit gibt, sie in eine andere Texturgruppe zu verschieben oder den Editor für dieses Element zu öffnen.

Obwohl die Assets in der Liste als getrennt angezeigt werden, werden alle sprites, Tilesets und Schriftarten der gleichen Texturseite (oder den gleichen Seiten) hinzugefügt und erhalten nicht für jeden Ressourcentyp eine eigene Seite. Die einzige Ausnahme von dieser Regel ist, wenn ein sprite -Asset als "Separate Texturseite" gekennzeichnet wurde. In diesem Fall erhält es eine eigene Texturseite (siehe den Abschnitt "Textureinstellungen" im Sprite-Editor für weitere Details).

Gruppeneinstellungen

Die Gruppeneinstellungen im Texturgruppen-Editor werden verwendet, um genau festzulegen, wie die Texturseiten für eine bestimmte Gruppe erstellt werden sollen, wobei die folgenden Optionen zur Verfügung stehen:

  • Skalierung zulassen - Wenn Sie diese Option deaktivieren, wird GameMaker gezwungen, die Texturgruppe unter keinen Umständen zu skalieren. Dies kann vor allem bei Universal-Apps nützlich sein, da es Ihnen erlaubt, bestimmte Ressourcen wie Schriften in eine eigene Gruppe zu legen, die verwendet wird, wenn die App auf einem bestimmten Gerät ausgeführt wird. Zum Beispiel können Schriftarten auf einem Retina-Display großartig aussehen, aber wenn sie verkleinert werden, um auf einem normalen Display zu zeichnen, können sie schrecklich aussehen, so dass Sie sie einer speziellen Gruppe zuweisen können, die nicht skaliert - wenn Sie für das iPhone entwickeln, möchten Sie dies vielleicht aktiviert haben, da Apps abgelehnt wurden, weil die Bilder durch die Skalierung der Textur verschwommen waren. Diese Option ist standardmäßig aktiviert .
  • Automatisch beschneiden - Wenn Sie Bilder in GameMaker einfügen, werden diese automatisch beschnitten, wenn sie einen "Raum" um sich herum haben (d.h. 100% Transparenz) und "Automatisch beschneiden" ist markiert. Wenn Sie das Bild jedoch z. B. als sich nicht wiederholende Textur für Primitive oder Ähnliches verwenden, bedeutet dies, dass die UV-Koordinaten für das Textur-Mapping nicht korrekt dargestellt werden. In diesen Fällen können Sie diese Option deaktivieren, und sprite wird der Texturseite "so wie sie ist" hinzugefügt, wobei alle Transparenzen an den Rändern erhalten bleiben. Diese Option ist standardmäßig aktiviert .
  • Mipmaps generieren - Wenn Sie diese Option aktivieren (standardmäßig ist sie ausgeschaltet), teilt GameMaker mit, dass die ursprüngliche Texturseite skaliert und in mehrere Auflösungen innerhalb der Texturdatei gefiltert werden soll. Normalerweise ist diese Option ausgeschaltet, aber wenn du Kameras verwendest, um große Bereiche des Spielraums zu vergrößern oder zu verkleinern, oder wenn du mit einem 3D-Spiel arbeitest, dann solltest du diese Option aktivieren und dann die Mipmapping-Funktionen verwenden, um einzustellen, wie das Spiel mit den verschiedenen Mipmap-Ebenen umgeht. Wenn du dieses Kontrollkästchen aktivierst, wird der Wert für "Randgröße" auf 8 gesetzt (wenn er ursprünglich auf weniger als 8 gesetzt war). Der Grund dafür ist, dass man bei der Verwendung von Mipmaps größere Ränder zwischen den Bildern auf den Texturseiten braucht, da sie sonst bei höheren Mip-Levels zusammenlaufen. Der Standardwert von 8 erlaubt es, das Bild auf etwa ein Achtel seiner ursprünglichen Größe zu verkleinern, bevor die Teilbilder zusammenlaufen, aber Sie können diesen Wert auf jeden beliebigen Wert ändern, der Ihren Bedürfnissen entspricht. Beachten Sie, dass Sie, wenn Sie die Option "Separate Texture Page" für sprites verwenden, nicht durch jede Textur gehen und sie für die Erzeugung von Mipmaps markieren müssen, sondern Sie können in den allgemeinen Spieloptionen die Option "Mipmaps für separate Texturseiten erzeugen" aktivieren.
  • Randgröße - Die Randgröße ändert die Überlappung (oder "Pixelverdopplung") an den Rändern aller Assets auf den für die Texturgruppe generierten Seiten, so dass beim Skalieren der Bilder keine Lücken (oder Nähte) zwischen ihnen sichtbar sind. Der Standardwert ist 2 Pixel, aber in extremen Fällen, in denen Sie die Ansicht stark vergrößern oder verkleinern, müssen Sie diesen Wert möglicherweise erhöhen. Beachten Sie, dass der Kachelsatz-Editor eine Option zum Festlegen des Ausgaberahmens für Kacheln hat (und auch eine Erklärung liefert, warum dies notwendig ist), und dies ist unabhängig von der Rahmeneinstellung hier - im Grunde werden Kacheln mit einem Ausgaberahmen für jede Kachel auf einer temporären Textur erzeugt, und diese wird dann der Texturseite mit dem hier angegebenen Rahmen um die Außenkante hinzugefügt. Beachten Sie auch, dass die Aktivierung von Mipmaps automatisch die Standard-Rahmengröße für die Texturseite auf 8 setzt, um ein "Ausbluten" zwischen Texturen bei höheren Mip-Levels zu verhindern, aber das kann geändert werden.
  • Übergeordnete Texturgruppe - Hier können Sie eine der vorhandenen Gruppen als "übergeordnete" Gruppe für die aktuell ausgewählte Gruppe auswählen. "Untergeordnete" Gruppen erben alle Änderungen, die an der übergeordneten Texturgruppe vorgenommen wurden, und dienen in erster Linie der Ressourcenverwaltung bei der Arbeit mit platform Spielen. Eine Texturgruppe mit einer übergeordneten Gruppe ist immer noch eine eigenständige Texturgruppe und erzeugt Texturseiten wie jede andere Gruppe, aber die Zuweisung einer übergeordneten Gruppe ermöglicht es Ihnen, große Teile für verschiedene platforms auszuschließen. Sie können zum Beispiel eine ganze Reihe von Android Spieler sprites zu einer "Spieler"-Gruppe und Android Feind sprites zu einer "Feind"-Gruppe hinzufügen und dann dafür sorgen, dass sie beide eine "Android"-Elterngruppe verwenden. Wenn Sie dann die "android"-Elterngruppe aus dem Windows oder einem anderen platform Build ausschließen, werden auch die Kindgruppen ausgeschlossen.
  • Texturgruppenformat - Hier können Sie das Format für Ihre Texturseite wählen, zwischen BZ2 + QOI, QOI und PNG. Es wird empfohlen, bei der Standardoption BZ2 + QOI zu bleiben, da sie die beste Kompression bei gleichbleibender Qualität bietet.

    Sie können QOI wählen, wenn die BZ2-Option die Geschwindigkeit auf einer Zielseite platform (z. B. Android) beeinträchtigt, oder PNG, wenn Sie die exportierten PNGs manuell bearbeiten oder die Option PNGCrush verwenden möchten.

Abgesehen von diesen Einstellungen können Sie auch wählen, ob eine bestimmte Texturgruppe für eine bestimmte platform exportiert werden soll oder nicht, indem Sie einfach die entsprechenden Kästchen im Abschnitt Use Group On... des Fensters aktivieren oder deaktivieren. Beachten Sie jedoch, dass die Texturgruppe <Default> immer auf alle Plattformen exportiert wird und Sie keine der Texturgruppen an- oder abwählen können (nur benutzerdefinierte Texturgruppen erlauben dies).

Texturgruppen sind auch mit den Einstellungen für Konfigurationen verknüpft. Das bedeutet, dass Sie Texturgruppen für jede einzelne Konfiguration Ihres Spiels definieren können und dann die sprite, Kachelsatz- oder Schriftart-Ressourcenzuweisungen für diese verschiedenen Gruppen festlegen können, die dann beim Wechsel zu einer anderen Konfiguration "gespeichert" werden. Das Gleiche gilt für alle Exportoptionen, die Sie im Texturgruppen-Fenster ausgewählt haben, so dass Sie die Texturgruppen so einstellen können, dass sie für jede Konfiguration auf bestimmte platforms exportiert werden.

Sie könnten zum Beispiel eine iPad-Konfiguration erstellen und dafür Texturseiten mit einer Größe von 2048x2048 Pixeln anlegen (dies wird über die Spieloptionen eingerichtet). Sie würden dann alle Grafik-Assets der entsprechenden Texturgruppe zuweisen. Wenn Sie fertig sind, würden Sie dann zu einer anderen Konfiguration (z. B. für HTML5) wechseln und im Fenster "Texturgruppe" neue Texturgruppen mit einer Größe von 512x512px erstellen und die Assets diesen neuen Gruppen zuweisen. Es gibt auch eine Schaltfläche am unteren Rand der Seite, mit der Sie die Texturgruppeneinstellungen von einer Konfiguration in eine andere kopieren können.

Schließlich sind Texturgruppen nur für Bitmap-Bilder relevant, und Nicht-Bitmap-Bilder sprites werden keinen Texturgruppen zugewiesen. Einige von ihnen haben bereits ihre eigene Texturseite (Skelettanimation sprites), während andere einfach keine Texturen benötigen, da sie aus Vektorenbestehen.
