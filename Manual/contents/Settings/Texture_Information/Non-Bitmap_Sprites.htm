Nicht-Bitmap-Sprites

GameMaker unterstützt Nicht-Bitmap-Dateien sprites in Form von Vektor-Dateien SWF oder Spine Skeletal Animation-Dateien. Beide Formate haben ihre Vor- und Nachteile, und die beiden folgenden Abschnitte erklären, wie man sie in den Sprite-Editor importiert und wie sie gezeichnet werden.

Skelett-Animation Sprites (Spine)

Bei einer sprite, die mit einer Skelettanimation erstellt wurde, wurde ein Basis-"Skelett" erstellt und mit Hilfe von Keyframes animiert, um die Komponenten des Skeletts im Laufe der Zeit zu bewegen. Dieses Skelett wird dann von einem Texturatlas gehäutet und die resultierende Animation in einem der vielen Dateitypen exportiert. GameMaker erlaubt es Ihnen, diese Art von sprite zu importieren, solange sie als *.json exportiert wurde und sich die korrekt zugeordnete Texturatlasdatei (*.atlas) und die Bilddatei (als *.png) im selben Verzeichnis befinden.

Derzeit kann GameMaker nur vector Bilder aus Dateien im JSON-Format importieren, die vom Programm Spine erstellt wurden, aber die Art und Weise, sie in das Programm zu importieren, ist fast identisch mit der, ein normales Bitmap-Bild hinzuzufügen. Um eine Skelettanimation hinzuzufügen, müssen Sie eine neue sprite erstellen, die den Standarddialog Sprite laden aufruft, nur müssen Sie hier sicherstellen, dass Sie *.json aus dem Dateifilter am unteren Rand ausgewählt haben:

Wenn Sie auf die Schaltfläche "Laden" klicken, wird die Skelettanimation dem Asset-Browser als sprite hinzugefügt, wobei während der Verarbeitung der Datei ein Fortschrittsbalken angezeigt wird. Beachten Sie, dass dies je nach Komplexität der Datei einen oder zwei Augenblicke dauern kann. Wenn die Verarbeitung abgeschlossen ist, kehren Sie zum Eigenschaftsdialog sprite zurück, der nun wie folgt aussieht:

Sobald Sie die Animation importiert haben, können Sie die Kollisionseigenschaften einstellen, aber beachten Sie, dass Sie hier nur präzise Kollisionen oder Bounding-Box-Kollisionen verwenden können und dass die Kollisionsdaten für eine Skelettanimation explizit aus den bereitgestellten Daten übernommen werden. GameMaker generiert keine Kollisionsmaske, wenn die Daten in der importierten Datei fehlen, d. h. Sie erhalten einfach keine funktionierenden Kollisionen, wenn die Masken in dem Programm, mit dem das importierte Bild erstellt wurde, nicht korrekt eingestellt sind.

HINWEIS Aufgrund der Komplexität von Skelettanimationen ist das Vorschaubild, das im sprite Editor angezeigt wird, nicht dazu gedacht, Ihre Animation genau darzustellen, sondern es soll Ihnen ein einfaches Bild geben, das die Animation zur Visualisierung im Raumeditor darstellt.

Im Gegensatz zur Bitmap spriteskann die importierte Skelettanimation sprite im Editor nicht verändert werden, aber es gibt eine Reihe von Funktionen innerhalb der GameMaker Language (GML), die zum Ändern von Skins, zum Einstellen von Eigenschaften und zur Steuerung anderer Aspekte der Animation verwendet werden können. Sie können auch keine der Texturoptionen für sprite einstellen, da GameMaker den mitgelieferten Texturatlas verwendet, der von dem Programm erstellt wurde, das die sprite generiert hat.

Es gibt eine Reihe von Dingen, die Sie beachten sollten, wenn Sie die Skelettanimation sprites importieren. Das wichtigste ist, dass einige der sprite Zeichen- und sprite Verwaltungsfunktionen aus technischen Gründen nicht unterstützt werden, insbesondere jene Zeichenfunktionen, die nur einen Teil einer sprite zeichnen oder die Position in irgendeiner Weise "verdrehen" (siehe die einzelnen draw_sprite_*() Funktionen für genaue Details). Die grundlegenden sprite Zeichenfunktionen werden jedoch voll unterstützt, ebenso wie die eingebauten sprite Variablen. Abgesehen von diesen Funktionen und Variablen können Sie die Animationseigenschaften auch mit speziellen skeleton_* Funktionen einstellen und ändern, die im Referenzabschnitt über Skelettanimationen aufgeführt und erläutert werden.

Vektor Sprites (SWF)

Im Allgemeinen verwenden alle sprites in GameMaker Bitmaps, die zwar inhaltlich flexibel sind, aber sowohl die Größe als auch die Anzahl der Frames, die in einem sprite möglich sind, einschränken, bevor der Speicherverbrauch untragbar wird. Vector sprites umgeht diese Einschränkungen, indem es seinen Inhalt anders speichert und zeichnet: Anstelle eines Pixelnetzes, das bei der Skalierung blockig oder unscharf werden kann, werden sie als Dreiecke unter Verwendung der Mathematik von vector gezeichnet, und diese können nach oben (oder unten) skaliert werden, ohne an Schärfe zu verlieren:

Zur Veranschaulichung zeigt das folgende Bild dasselbe vector Feuerwehrmann sprite, nur dass jetzt die Polygone zu sehen sind, aus denen das Bild in GameMaker besteht:

Bei der Programmierung von Spielen gibt es jedoch nichts umsonst, und der Nachteil von vector sprites ist, dass das Zeichnen von Animationen mehr CPU-Leistung erfordert als das Zeichnen von Bitmaps sprites. Dieser Geschwindigkeitsunterschied nimmt mit zunehmender Komplexität der importierten Animation zu. Außerdem wird ihre Speichernutzung durch ihre visuelle Komplexität beeinflusst, im Gegensatz zu Bitmap sprites. Abgesehen davon ermöglichen sie das Hinzufügen großer animierter Grafiken auf eine Weise, die mit herkömmlichen Bitmaps sprites unmöglich wäre.

Die Kehrseite der Leistungsgleichung ist, dass vector sprites nur die unbedingt erforderlichen Pixel zeichnet (im Gegensatz zu Bitmap sprites, die auch den leeren Raum um das Bild herum "zeichnet"), was aus Sicht der GPU günstiger ist. Außerdem sind vector sprites im Allgemeinen viel kleiner als ihre Bitmap-Äquivalente - für das obige Feuerwehrmann-Beispiel sind die Größen der verschiedenen sprite -Typen wie folgt:

Bitmap at 70x68 = 4x70x68 bytes x 12 frames = 223k
Vector = 54k

Wenn Sie die Auflösung des Sprites vervierfachen:

Bitmap at 280x272 = 4x280x272 bytes x 12 frames = 3570k
Vector = 54k

Wie Sie also sehen, kann durch die Verwendung von vector sprites eine Menge Speicherplatz eingespart werden.

Derzeit unterstützt GameMaker nur begrenzt vector sprites. und kann nur vector Bilder aus SWF Formatdateien importieren. Die Art und Weise, wie man sie in das Programm importiert, ist fast identisch mit der, wie man ein normales Bitmap-Bild hinzufügt. Um ein vector sprite hinzuzufügen, erstellen Sie ein neues sprite, woraufhin der Standarddialog Sprite laden erscheint. Stellen Sie jedoch sicher, dass Sie *.swf aus dem Dateifilter am unteren Rand ausgewählt haben.

Wenn Sie auf die Schaltfläche "Laden" klicken, wird das Bild oder die Animation von vector dem Asset-Browser hinzugefügt, wobei ein Fortschrittsbalken angezeigt wird, während die Datei verarbeitet wird. Beachten Sie, dass dies je nach Komplexität der Datei eine Weile dauern kann (bis zu einer Minute bei komplexeren Animationen). Wenn die Verarbeitung abgeschlossen ist, kehren Sie zum Eigenschaftsdialog von sprite zurück, der nun wie folgt aussieht:

Im Editor sprite hat sich nicht viel geändert, aber die Schaltfläche Bild bearbeiten ist jetzt ausgegraut und kann nicht angeklickt werden, da es bei dieser Art von sprite keine Bitmaps zu bearbeiten gibt. Zweitens sind auch alle Texturoptionen ausgegraut. SWF sprites ist mathematikbasiert und hat daher in der Regel keine Texturen, so dass keine der verfügbaren Optionen für diese Art von sprite anwendbar ist.

HINWEIS Die Darstellung des Bildes in dem von Ihnen gewählten Vorschau-Editor entspricht nicht der Darstellung in Ihrem Spiel und auch nicht der Darstellung im Vorschaufenster des Editors sprite. Das Bild von vector wird in der Größe importiert, in der es erstellt wurde, was nicht unbedingt der Größe entspricht, die in der Vorschau angezeigt wird (Browser skalieren das Bild normalerweise so, dass es passt, z. B.).

Eine Neuerung des Sprite Editors ist, dass Sie die Qualität der vector sprite beim Zeichnen mit dem Schieberegler SWF-Präzision am oberen Rand einstellen können. Dadurch wird einfach die Anzahl der Dreiecke, die für die Erstellung von sprite verwendet werden, verringert oder erhöht - wobei ein Wert von etwa 50 % normalerweise für die meisten Spiele ausreicht. Sie sollten jedoch mit diesem Wert experimentieren, wenn Sie extreme Skalierung verwenden oder Leistungsprobleme haben.

Es gibt eine Reihe von Dingen, die Sie beim Importieren von vector sprites beachten sollten. Das wichtigste ist, dass einige der sprite Zeichen- und sprite Verwaltungsfunktionen aus technischen Gründen nicht unterstützt werden, insbesondere jene Zeichenfunktionen, die nur einen Teil eines sprite zeichnen oder die Position in irgendeiner Weise "verdrehen" (siehe die einzelnen draw_sprite_*() Funktionen für genaue Details). Die grundlegenden sprite Zeichenfunktionen werden jedoch voll unterstützt, ebenso wie die eingebauten sprite Variablen. Sie können auch verschiedene Anti-Aliasing-Werte (AA) für die zu zeichnende SWF sprites mit den Funktionen draw_enable_swf_aa(), draw_get_swf_aa_level() oder draw_set_swf_aa_level() einstellen.

Sie sollten auch Folgendes beachten:

  • Beim Importieren einer *.SWF Datei wird Actionscript nicht unterstützt. Wenn Ihr SWF also darauf angewiesen ist, um korrekt zu funktionieren, wird es wahrscheinlich nicht richtig funktionieren. Ebenso wird bei eingebetteten Filmclips, die eine eigene Zeitleiste haben, nur das erste Bild während der gesamten Animation angezeigt - alle Animationen müssen sich auf der Hauptzeitleiste befinden.
  • Wenn Ihre vector Datei besonders feine Details aufweist, kann es gelegentlich vorkommen, dass Löcher oder seltsame Dreiecke in der resultierenden sprites erscheinen. Das liegt daran, dass die Geometrie in kleinen Maßstäben manchmal zusammenfallen kann, so dass Formen entstehen, die sich nicht gut triangulieren lassen. In diesem Fall haben Sie zwei Möglichkeiten: Sie können den Inhalt der Datei vector vergrößern, um dem Importer mehr Spielraum zu geben, oder Sie können versuchen, den Detailgrad des Objekts, das zerbricht, zu verringern.
  • GameMaker verwendet die Bühnengröße der SWF als Begrenzung der resultierenden sprite. Wenn Sie jedoch etwas auf der Bühne außerhalb dieser Grenzen haben, wird es trotzdem gezeichnet. Das bedeutet, dass Sie entweder versuchen sollten, alles außerhalb der Bühnengrenzen zu platzieren, wenn Sie nicht möchten, dass es angezeigt wird, oder eine Maskenebene verwenden, um das Zeichnen auf diesen Bereich zu beschränken.
  • Die Bühnengröße wirkt sich auch auf die Größe der generierten Kollisionsmasken aus. Wenn Sie also eine große Bühne und viele Frames in Ihrer Animation haben, können Sie möglicherweise viel Speicherplatz verbrauchen. Verwenden Sie also nur dann präzise Kollisionsmasken, wenn Sie sie wirklich brauchen.
  • Da eine SWF-Datei aus mehreren Ebenen erstellt wird, von denen sich einige möglicherweise überschneiden, funktioniert Alpha nicht ganz so wie bei einer Bitmap sprites - überlappende Bereiche sehen nicht so transparent aus wie andere Teile der sprite, da die Pixel dort mehrfach überzeichnet werden.
  • Bitmap-Füllungen werden unterstützt. Wenn Sie jedoch gekachelte Bitmap-Füllungen als Teil Ihrer SWF-Datei verwenden, müssen Sie sicherstellen, dass die Bitmap eine Zweierpotenz groß ist, da sie sonst nicht richtig gekachelt werden kann. Das Rendern von Text wird ebenfalls unterstützt, aber Sie müssen die Schriftart in Ihre SWF-Datei einbetten, damit sie in der resultierenden sprite angezeigt wird. Außerdem wird derzeit nur linksbündiger einzeiliger Text unterstützt.
