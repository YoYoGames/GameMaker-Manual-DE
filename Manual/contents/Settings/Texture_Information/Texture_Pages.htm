Textur-Seiten

Wenn Sie ein Spiel mit GameMaker erstellen, haben Sie sicherlich auch Grafiken erstellt - sprites, Tilesets und Schriftarten - die dazu gehören. Diese Grafiken werden auf Texture Pages gespeichert, die GameMaker für Sie aus allen Bildressourcen erstellt, die Ihr Spiel enthält. Im Folgenden sehen Sie ein Beispiel für eine fertige Textur-Seite:

Wie Sie sehen können, sind die Spielgrafiken so angeordnet, dass sie alle auf eine Seite in der Größe einer Potenz von 2 passen, z. B. 512x512, 1024x512 usw... bis zu einer maximalen Größe von 4096x4096 Pixel Seite. Beachten Sie, dass die maximale Größe einer Texturseite von dem gewählten Export platform abhängt, da einige Ziele keine Seiten mit einer Größe von mehr als 2048x2048px zulassen, und selbst dann ist die maximal verfügbare Größe möglicherweise nicht für alle Geräte, die das Zielsystem platform verwenden, optimal ist.

Wenn Ihre Bilder viel leeren Raum (d. h. transparente Pixel) haben, werden sie standardmäßig beschnitten, um diese "unsichtbaren" Pixel zu entfernen und so viele Bilder wie möglich auf eine einzige Texturseite zu packen. Wenn Sie dies nicht wünschen, müssen Sie dies im Fenster Texturgruppen deaktivieren.

Es ist wichtig zu verstehen, wie sich die verschiedenen verfügbaren Textureinstellungen auf ein Bild auswirken, also schauen wir uns den Abschnitt Textureinstellungen im Sprite-Editor an, der unten abgebildet ist:

Hier können Sie festlegen, wie das Bild auf der Texturseite gespeichert werden soll und ob es ein Bild speziell für die Verwendung mit 3D ist und separat gespeichert werden muss. separat gespeichert werden soll und welcher Texturseite (falls vorhanden) es zugewiesen werden soll, sowie einige andere Optionen (ausführlich erklärt auf der Seite Sprite-Editor ). Die Optionen, an denen wir interessiert sind, sind die ersten beiden: Horizontal kacheln und Vertikal kacheln. Diese beziehen sich darauf, wie die Kanten des Bildes erstellt werden, wenn es auf der Texturseite platziert wird.

Bei einem Bild, das gekachelt werden soll, werden die Ränder (horizontal oder vertikal) durch die Ränder der gegenüberliegenden Seite ergänzt, während, wenn keine Kachelung ausgewählt ist, die sprite geklammert wird und die Randpixel wiederholt werden (beachten Sie, dass Kachelsätze auch eine Einstellung für die eine Einstellung für die Breite des Ausgaberandes haben, die in den Eigenschaften des Kachelsatzes zu finden ist und die das Gleiche bewirkt, nur für jede Kachel im Satz). Unterhalb dieser Einstellungen befindet sich die Einstellung Separate Texture Page . Mit dieser Option wird GameMaker gezwungen, dieses Bild auf einer eigenen Texturseite zu platzieren, und es gibt keine Optionen, die damit verbunden sind, da die Kachelung oder Einspannung direkt über code gesteuert wird. Dies wird am häufigsten für die Texturierung von 3D-Modellen verwendet (in diesem Fall sollte die Textur eine Potenz von 2 sein, z. B.: 128x128, 512x512, 256x1024, etc...), aber Sie können damit auch jedes Bild auf eine eigene Seite zeichnen lassen.

HINWEIS: Seien Sie vorsichtig, wenn Sie diese Option verwenden, da Sie schnell den gesamten verfügbaren Texturspeicher verbrauchen können, insbesondere auf mobilen Geräten, wenn Sie zu viele Texturseiten haben.

Wenn Ihr Spiel umfangreich ist und viele künstlerische Ressourcen enthält, kann es sein, dass die Art und Weise, wie GameMaker diese Seiten automatisch erstellt, nicht optimal ist, um das Zielgerät optimal zu nutzen, denn wenn ein Raum in Ihrem Spiel Wenn ein Raum in Ihrem Spiel Ressourcen von 4 verschiedenen Texturseiten benötigt, muss das Gerät diese Seiten ständig in den (und aus dem) Speicher auslagern, was zu Leistungseinbußen führen kann. Zu diesem Zweck haben Sie die Möglichkeit, dem GameMaker mitzuteilen, welche Bilder welche Bilder auf einer bestimmten Texturseite platziert werden sollen, indem Sie sie bestimmten Texturgruppen zuweisen. Das Ziel ist es, die Anzahl der Tauschvorgänge in einem Spiel zu minimieren (Texturseiten können im Fenster Texturgruppen hinzugefügt werden).

Es sei darauf hingewiesen, dass es auf den einzelnen Registerkarten platform in den Spieloptionen eine Schaltfläche gibt, mit der Sie die von Ihnen definierten Texturseiten anzeigen können, so wie sie für jedes Ziel erstellt werden. Dies ist sehr praktisch, um herauszufinden, wie viele Seiten erstellt werden und was sich auf ihnen befindet.

Sollten Sie eine Textur in Ihrem Spiel haben, die größer ist als die maximal gewählte Texturseitengröße, dann wird diese um die Hälfte (und ggf. nochmals um die Hälfte) verkleinert , bis sie auf die maximale Seitengröße passt. Sie wird nicht in Abschnitte aufgeteilt Abschnitte über verschiedene Texturseiten aufgeteilt. Das bedeutet, wenn Sie Ihr Spiel auf Windows, Ubuntu (Linux) oder MacOS testen, kann das Spiel gut aussehen, da Sie größere Texturseiten verwenden, aber auf iOS (oder einem der anderen Ziele, die eine kleinere Texturseite (oder anderen Zielen, die für eine optimale Leistung eine kleinere Texturseite erfordern) kann die Textur verschwommen aussehen oder Artefakte aufweisen. Es ist wichtig, dass Sie sicherstellen, dass Ihre Bilder alle auf die kleinste der Texturseitengrößen passen, die Ihr Spiel verwenden wird, und wenn nicht, sollten Sie sie Wenn nicht, sollten Sie sie in kleinere Bilder aufteilen und diese dann in Ihrem Spiel zusammenfügen, um die Illusion eines großen, vollständigen Bildes zu erzeugen.

HINWEIS: Bevor Sie damit beginnen, Ihre Grafiken auf verschiedene Texturseiten aufzuteilen, sollten Sie sich darüber im Klaren sein, dass dies nicht immer notwendig ist und auch nicht immer die effizienteste Arbeitsweise darstellt. Einige Spiele sind mehr an die CPU als an die GPU gebunden (siehe Weitere Informationen finden Sie im Abschnitt " Debugging" ), und diese Spiele werden von dieser Optimierung nicht sehr profitieren. Texturseiten zu haben und zwischen ihnen zu wechseln, anstatt sie auf mehrere Seiten zu verteilen, was nur Speicherverschwendung bedeutet.
