Compiler-Batch-Dateien/Skripte

Mit dem GameMaker compiler können Sie Batch-Dateien (Windows) oder Shell-Dateien scripts (auf macOS und Linux) in Ihr Projektverzeichnis und in das Verzeichnis für jede Erweiterung in Ihrem Projekt legen.

Platzierung

Eine script -Datei kann im Stammverzeichnis Ihres Projekts neben der .yyp-Datei des Projekts abgelegt werden.

Eine Datei script kann auch in das Verzeichnis einer Erweiterung gelegt werden, das wie folgt aussehen kann: <Projektverzeichnis>/extensions/<extension_folder>/.

Die Erweiterungsversionen einer script Datei werden zuerst ausgeführt, die Wurzelversion wird zuletzt ausgeführt. Die Reihenfolge, in der die Erweiterungen ausgeführt werden, kann nicht kontrolliert werden, so dass ein Entwickler von Erweiterungen sicherstellen muss, dass die scripts einer Erweiterung nicht auf die scripts einer anderen Erweiterung angewiesen ist.

Ausführung

Skriptdateien mit bestimmten Namen, die sich in einem der oben beschriebenen Verzeichnisse befinden, werden an verschiedenen Punkten während des Kompilierungsprozesses ausgeführt:

Windows Batch File Name macOS/Linux-Shell Script Name Beschreibung
pre_run_step.bat pre_run_step.sh Dies wird ausgeführt, bevor der Einsatz des Spiels beginnen soll
post_run_step.bat post_run_step.sh Dieses Skript wird ausgeführt, wenn das Spiel vorbereitet und bereit ist; nach der Ausführung des Skripts wird das Spiel gestartet.
pre_build_step.bat pre_build_step.sh Dies wird ausgeführt, bevor das Asset compiler für die Erstellung des Spiels gestartet wird.
post_build_step.bat post_build_step.sh Dies wird ausgeführt, nachdem das Asset compiler für die Erstellung des Spiels gestartet wurde.
remote_build_step.bat remote_build_step.sh This is executed on the remote machine, when you are on a Windows machine and compiling remotely for macOS/iOS/tvOS or Ubuntu.
pre_package_step.bat pre_package_step.sh Dies wird vor dem letzten Verpackungsschritt ausgeführt, wenn alle Dateien fertig sind, aber die endgültige ZIP-Datei oder das Speicherpaket noch nicht erstellt wurde.
post_package_step.bat post_package_step.sh Dies wird ausgeführt, nachdem der letzte Verpackungsschritt abgeschlossen und die endgültige ZIP-Datei oder das Speicherpaket vorbereitet wurde.
post_textures.bat post_textures.sh Dieser Schritt wird ausgeführt, nachdem die Texturen für Ihr Spiel generiert worden sind. Sie können diesen Schritt verwenden, um Ihre Spieltexturen manuell zu optimieren.

Diese script erhält eine Umgebungsvariable namens TexturesDir, in der der Pfad zu den generierten Texturen gespeichert ist.
Platform-Specific Steps
pre_gradle_step.bat pre_gradle_step.sh Android This is executed when the files necessary for the Android tools have been created, but before Gradle is called. You can use this step to access and modify the Android files yourself, before the Android tools compile it into a final executable.

Diese script Dateien werden bei der Erstellung auf allen Zielsystemen platforms unterstützt.

Deaktivieren von Scripts für eine Nebenstelle

Wenn Sie mehrere Erweiterungen verwenden, die unter scripts laufen, z. B. Steamworks und GDK, dann sollten Sie Steamworks beim Exportieren nach GDK deaktivieren und umgekehrt.

Siehe dazu: Wie deaktiviere ich Erweiterungen?
