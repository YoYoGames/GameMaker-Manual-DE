GameMaker-Sprache

Sie können GameMaker Language verwenden, um Ihre Spiele auf zwei verschiedene Arten zu programmieren: indem Sie Blöcke platzieren, um code visuell zu "schreiben", oder indem Sie code manuell schreiben.

Die erste Option, GML Visual, eignet sich am besten für Anfänger und Bastler, die schnell mit der Entwicklung ihrer eigenen Spiele beginnen wollen.

Die zweite Option, GML Code, ermöglicht es Ihnen, die volle Leistung von GameMaker Language zu nutzen, indem Sie Ihre code manuell schreiben.

Während GML Visual den Einstieg erleichtert, ist GML Code eine einfach zu bedienende Programmiersprache mit leistungsstarken Funktionen(Funktionen, Methoden, Strukturen & Konstruktoren, etc.).

Erstellen von Visual/Code-Skripten

Sie werden aufgefordert, zwischen GML-Code und GML-Visual zu wählen, wenn Sie ein neues Objekt-Ereignis, ein neues Script -Asset oder irgendetwas anderes erstellen, das zur Erstellung eines Skripts führt:

Neben der Auswahl zwischen GML Code und GML Visual enthält dieses Fenster zwei Kontrollkästchen:

  • Für dieses Projekt nicht mehr nachfragen: Wenn diese Option aktiviert ist, wird die Spieloption "Standard-Skriptsprache" auf Ihre neue Auswahl gesetzt, so dass Sie für das aktuelle Projekt nicht erneut zur Auswahl aufgefordert werden.
  • Nie wiederfragen: Wenn diese Option aktiviert ist, wird die Voreinstellung "Standard-Skriptsprache" auf Ihre neue Auswahl gesetzt, so dass Sie nie wieder gefragt werden, ob Sie diese Sprache für ein Projekt auswählen möchten.
