Ändern von GML Visual To Code

Ein weiteres Merkmal von GML Visual ist, dass Sie es in GML Code umwandeln können und auf diese Weise sehen können, was genau hinter den Kulissen passiert, wenn Sie Aktionen verwenden. Bevor Sie die Aktionen jedoch in code umwandeln, können Sie zunächst eine Vorschau anzeigen, indem Sie die rechte Maustaste Menüoption Live-Vorschau verwenden:

Dadurch wird ein neues Fenster geöffnet:

Wenn Sie GML Visuelle Aktionen hinzufügen, ändern oder entfernen, wird die GML Live-Vorschau aktualisiert, um Ihnen die tatsächliche code zu zeigen, die "hinter den Kulissen" erstellt wird. Die code in der Live-Vorschau kann nicht direkt bearbeitet werden, aber Sie können Abschnitte auswählen und kopieren, um sie z. B. in GML scripts oder Code Aktionen einzufügen.

Wenn Sie sich entscheiden, dass Sie GML Visual in code umwandeln möchten, nachdem Sie die Vorschau gesehen haben, dann können Sie dies erneut tun, indem Sie in einem beliebigen Ereignis workspace mit Aktionen auf die rechte Maustaste klicken und In GML Code umwandeln wählen.

Wenn Sie dies zum ersten Mal tun, erhalten Sie eine Warnmeldung, die besagt, dass es sich um eine einseitige Umwandlung handelt, da Sie Aktionen in code umwandeln können, diese aber später nicht mehr in einzelne Aktionen zurückverwandeln können. Wenn Sie hier auf "OK" klicken, wird die Umwandlung für Sie durchgeführt.

Das resultierende code verwendet {}, um einzelne Aktionen abzugrenzen, und Sie können klar erkennen, welche Aktionen sich auf welche Funktionen oder Variablendeklarationen innerhalb des code beziehen. Wenn das GML Visual komplexer ist, wird das code auch sein, aber es gelten dieselben allgemeinen Regeln und code wird sequentiell genau so aufgebaut sein, wie Sie das GML Visual geschrieben haben. Hier erstellt code zunächst eine lokale (temporäre) Variable und setzt sie auf false. Dann wird geprüft, ob die Instanz existiert und die lokale Variable auf den Rückgabewert des Funktionsaufrufs gesetzt. Die lokale Variable wird dann überprüft, um festzustellen, ob sie true oder false ist, und wenn sie true ist, wird der Rest von code ausgeführt.

Wenn Sie lernen, mit GML Visual zu programmieren, kann dies ein wichtiges Werkzeug sein, um später mit GML weiterzumachen, aber es ist keineswegs obligatorisch und Sie können immer noch tolle Spiele mit GML Visual machen! Es ist auch erwähnenswert, dass der Konvertierungsprozess zwar einseitig ist, Sie aber nach der Konvertierung von Aktionen in code wieder zu GML Visual zurückkehren können, indem Sie im code -Editor mit der rechten Maustaste auf klicken und "Convert To GML Visual" auswählen.
