Konstruktion von Aktionsblock-Code

Um Objekten Verhaltensweisen hinzuzufügen, können Sie Ihre code mit Aktionen aus den verschiedenen Bibliotheken erstellen, die Ihnen in der Action Toolbox zur Verfügung stehen. Zunächst müssen Sie ein neues GML Visual-Projekt erstellen und dann ein neues Objekt erstellen (Sie können dem Objekt bei Bedarf auch ein Sprite zuweisen). In Ihrem neuen Objekt können Sie beginnen, Folgendes hinzuzufügen eventsund in den Ereignissen fügen Sie Ihre GML Visual code Aktionen hinzu.

Beachten Sie, dass beim Hinzufügen eines neuen Ereignisses ein "Code"-Fenster mit einer Registerkarte für das betreffende Ereignis geöffnet wird (siehe Abbildung oben), und Sie können nun eine beliebige Aktion aus der Toolbox auf der rechten Seite in den Bereich "Aktionsblock" des Fensters code ziehen. Es stimmt zwar, dass Sie jede beliebige Aktion hinzufügen können, aber das bedeutet nicht, dass sie alle funktionieren oder das Projekt damit kompiliert werden kann. Einige Aktionen erfordern mindestens eine Variable, um zu funktionieren, während andere - wie die Zeichnungsaktionen - nur funktionieren, wenn sie in einem bestimmten Ereignis verwendet werden. Woher wissen Sie, welche Aktionen Sie verwenden müssen? Nun, im Allgemeinen ist es einfach eine Frage der Logik... wenn eine Aktion eine Variable erfordert und wir noch keine definiert haben, dann sollten wir sie nicht verwenden, bis wir eine Aktion hinzugefügt haben, um die Variable zu erzeugen.

Wenn Sie eine Aktion aus der Toolbox in den Hauptaktionsblock workspace ziehen, wird sie erweitert und zeigt die verfügbarenArgumente (Parameter) an, die Sie eingeben und ändern können, um das Verhalten festzulegen. Sie sehen, dass die neue Aktion auch auf der linken Seite des code Fensters in Kurzform angezeigt wird. Diese Liste von Aktionen, die so genannte Aktionsübersicht, kann angeklickt werden, um schnell zu dieser Aktion zu navigieren und sie zu bearbeiten. Bei Bedarf können Sie dem Ereignis weitere Aktionen hinzufügen, wobei jede neue Aktion mit der vorhergehenden "verkettet" wird, um den Ablauf des GML Visual code zu zeigen, das Sie erstellen. Beachten Sie, dass der Bereich, in dem Sie weitere Aktionen ablegen können, unterhalb der ersten Aktion hervorgehoben ist. Je nach verwendeter Aktion werden verschiedene Bereiche hervorgehoben, um zu zeigen, an welcher Stelle der Kette Sie sie hinzufügen können:

Wenn Sie dem workspace Aktionen hinzufügen, werden diese mit den darüber liegenden Aktionen "verkettet", so dass Sie sehen können, wie GML Visual code abläuft, wobei eine Aktion zur nächsten führt, und das Übersichtsfenster die Aktionen in Kurzform und in der Reihenfolge ihrer Ausführung anzeigt:

Einige Aktionen platzieren code in einer separaten Kette abseits des Hauptflusses - Dinge wie die If-Variable erstellen eine Unterkette von Aktionen, die ausgeführt werden sollen, wenn die richtigen Bedingungen erfüllt sind, bevor mit der Hauptkette fortgefahren wird:

Beachten Sie, dass bei der Verwendung von Aktionen, die einen Seitenkettenblock wie diesen haben können, die Aktion zwei hervorgehobene Bereiche für das Ablegen weiterer Aktionen hat:

  • Eine unterhalb der Aktion als normal
  • Eine rechts von der Aktion, um zu zeigen, dass Sie einen separaten Block von Aktionen erstellen möchten, der ausgeführt wird, wenn die Bedingung der ersten Aktion wahr ist

Sie können die Position von Aktionen innerhalb der Kette ändern, indem Sie einfach auf klicken und sie an die gewünschte neue Position ziehen. Wenn Sie auf klicken, die Maustaste eine Sekunde lang gedrückt halten und dann die Maus bewegen, können Sie die Aktion innerhalb von workspace verschieben, während ihre Position innerhalb der Aktionsblockkette erhalten bleibt.

Es ist auch möglich, Kommentare zu GML Visual-Aktionen hinzuzufügen, so dass Sie Notizen neben den verschiedenen Blöcken hinterlassen können, um zu erklären, was sie tun (was besonders nützlich ist, wenn Sie im Team arbeiten):

Zielvariablen

Es ist wichtig zu beachten, dass viele Aktionen eine "Ziel"-Variable bieten, die z.B. als "Temporär" gekennzeichnet werden kann:

Das bedeutet, dass Sie eine Variable angeben können, die als Ziel den von der Aktion zurückgegebenen Wert speichert. In der obigen Beispielaktion gibt die Aktion die Lautstärke der von Ihnen ausgewählten Audioressource zurück, so dass Sie eine Zielvariable angeben, die diesen Wert enthält, damit Sie später darauf verweisen können.

Diese Zielvariable muss zuvor entweder mit Variable zuweisen (wodurch eine Instanzvariable erstellt wird) oder mit Temporäre Variable deklarieren (wodurch eine lokale temporäre Variable erstellt wird) deklariert worden sein. Wenn Sie die Option "Temporär" aktivieren, können Sie einfach einen Variablennamen hinzufügen, und die Aktion erstellt diese Variable und legt sie so fest, dass sie den Rückgabewert für Sie enthält (wodurch eine temporäre lokale Variable erstellt wird). Alle Aktionen, die danach ausgeführt werden, können nun auf den Wert der temporären Variable zugreifen, allerdings nur innerhalb desselben Ereignisses oder Skripts. Temporäre Variablen sind nur innerhalb des Bereichs verfügbar, in dem sie erstellt wurden. Ausführlichere Informationen über Variablen und Variablenbereiche finden Sie hier.

Auto komplett

Wenn Sie mit GML Visual arbeiten, müssen Sie Variablen und Ausdrücke in die verschiedenen Eingabefelder der Aktionen eingeben. Dabei wird jedoch häufig das Fenster " Auto-Vervollständigen " angezeigt, um Ihnen zu helfen:

In diesem Pop-up-Fenster werden alle eingebauten GML (GameMaker Language) Variablen, Konstanten und Funktionen sowie alle Ressourcen aufgelistet, die die Anfangsbuchstaben des eingegebenen Wortes enthalten. Damit können Sie schnell die Ressource oder Variable finden, die Sie für eine Aktion benötigen, ohne alles selbst eingeben zu müssen. Wenn z. B. alle Ihre Räume den Anfangsbuchstaben "rm_" tragen, wird nach der Eingabe und einem kurzen Moment Wartezeit das Fenster für die automatische Vervollständigung mit allen Ressourcen angezeigt, die mit "rm_" beginnen. Beachten Sie, dass alle eingebauten Variablen, die im Autovervollständigungsfenster angezeigt werden, überall dort verwendet werden können, wo eine Variable oder ein Ausdruck in einer Aktion erwartet wird, ebenso wie die meisten GML Funktionen.

Aktion Erweiterung

Wenn Sie eine Aktion verwenden, sehen Sie manchmal ein kleines Plus-Symbol an der Seite. Das bedeutet, dass Sie die Aktion erweitern können, um zusätzliche Aufgaben auszuführen oder weitere arguments zu übernehmen. Wenn Sie sich zum Beispiel die Aktion "Temporäre Variablen deklarieren" ansehen, sehen Sie, dass sie dieses Symbol hat:

Wenn Sie auf das Symbol klicken, wird die Aktion erweitert und ermöglicht es Ihnen, weitere Variablen zu deklarieren, was die gleichzeitige Definition mehrerer Variablen erleichtert und beschleunigt.
