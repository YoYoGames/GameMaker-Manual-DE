Menüoptionen der rechten Maustaste

Wenn Sie auf Ihrem GML Visual code arbeiten, können Sie mit der rechten Maustaste auf die Aktion, auf das workspace oder auf den Text in der linken Auswahlliste klicken um das Menü der rechten Maustaste zu öffnen:

Dieses Menü bietet die folgenden Optionen:

  • Ausschneiden / Kopieren / Einfügen: Damit können Sie jede Aktion, die in workspace ausgewählt ist, ausschneiden, kopieren oder einfügen. Beachten Sie, dass Sie + außerhalb der Aktionen verwenden und ziehen können, oder / + für einzelne Aktionen verwenden können, um mehr als eine Aktion auszuwählen und den Befehl auf alle anzuwenden.
  • Kommentar / Kommentar entfernen: Mit dieser Option können Sie Kommentare zu Ihren GML Visual code Blöcken hinzufügen oder entfernen. Wenn Sie Kommentar wählen, wird rechts neben dem ausgewählten GML Visual-Block ein Texteditor geöffnet, in dem Sie einen beliebigen Text hinzufügen, sei es eine Erinnerung, etwas zu ändern, ein hilfreicher Kommentar für ein Tutorial oder sogar Anweisungen für Ihre Teamkollegen:
    Sobald Sie den Kommentar hinzugefügt haben, können Sie ihn anklicken und ziehen, um ihn vertikal und horizontal zu positionieren - allerdings Wenn Sie den Knoten GML Visual einklappen, wird auch der Kommentar eingeklappt und eine Zusammenfassung seines Inhalts angezeigt (in diesem Fall kann er nicht neu positioniert werden). Wenn der Kommentar nicht mehr benötigt wird, können Sie entweder den gesamten den gesamten Text im Kommentar löschen oder die Option Kommentar entfernen aus dem Menü verwenden. Beachten Sie, dass beim Kopieren eines Knotens der Kommentar mitkopiert wird, und wenn Sie den Knoten von GML Visual in GML konvertieren, bleibt der Kommentar erhalten beibehalten und zu dem erstellten GML hinzugefügt, wie in der folgenden Abbildung dargestellt:
    Wie oben erwähnt, kann ein Kommentar nur auf einen einzelnen Knoten angewendet werden, aber Sie können ihn auf mehrere Sie können ihn jedoch auf mehrere Knoten ausdehnen, indem Sie einfach Zeilenumbrüche einfügen, um ihn zu verlängern - allerdings müssen Sie dann möglicherweise den Kommentar neu positionieren, um Überschneidungen mit nachfolgenden Knoten zu vermeiden:
    Sobald ein Kommentar hinzugefügt wurde, können Sie die Option "Kommentar" aus dem RMB-Menü auswählen oder auf doppelklicken, um die Bearbeitung zu aktivieren.
  • Deaktivieren / Aktivieren: Mit diesen Befehlen können Sie die ausgewählte Aktion (oder Aktionen) deaktivieren oder aktivieren. Wenn eine Aktion deaktiviert ist, ist sie immer noch auf workspace sichtbar, aber sie wird nicht ausgeführt, wenn Sie das Spiel kompilieren. Auf diese können Sie auf diese Weise Aktionen zum Testen und debugging hinzufügen/entfernen. Die folgende Abbildung zeigt einige GML Visual code mit deaktivierten Aktionen:
    Sie können sehen, dass die Aktionen "If Score" und "Go To Next Room" deaktiviert wurden - Sie wurden visuell ausgegraut - und werden daher übersprungen, wenn das Spiel getestet oder kompiliert wird.
  • Haltepunkt umschalten: Dieser Befehl schaltet einen Haltepunkt für die ausgewählte Aktion ein oder aus. Ein Haltepunkt ist einfach ein Punkt in GML Visual code, an dem der Debugger anhalten soll, wenn das Spiel im Debug-Modus ausgeführt wird. Eine Aktion, der ein Haltepunkt zugewiesen wurde, wird hervorgehoben, wie in der Abbildung unten gezeigt (die Aktion "Wenn Gesundheit" wurde als Haltepunkt festgelegt):
  • Alle Haltepunkte löschen: Mit diesem Befehl können Sie alle Haltepunkte aus dem GML Visual code entfernen.
  • Alles einklappen / Alles ausklappen: GML Visuelle Aktionen können entweder eingeklappt oder ausgeklappt werden, wobei bei einer eingeklappten Aktion lediglich der Name der Aktion, nicht aber die internen Werte angezeigt werden, so dass der verfügbare Platz für die Seite workspace. Beispiel:

Beschreibung auf der rechten Seite. Sie können die Option "Alle erweitern" wählen, um dies für jeden Knoten in workspace rückgängig zu machen.
  • Layout zurücksetzen: Damit wird GML Visual workspace zurückgesetzt, die Zoomstufe auf 100 % gesetzt, der Fokus auf die aktuell ausgewählte Aktion gesetzt und alle eingeklappten Aktionen aufgeklappt (und alle anderen).
  • Live-Vorschau: Wenn Sie diese Option wählen, aktivieren oder deaktivieren Sie das Vorschaufenster GML Visual to GML (GameMaker Language) code. Dies wird auf der Seite Ändern von GML Visuell in Code näher erläutert.
  • In GML konvertieren: Damit werden alle Ihre Aktionen in der aktuellen Veranstaltung in GML (GameMaker Language) code konvertiert. Seien Sie dabei sehr vorsichtig, denn Sie können GML nicht zurück in die GML Visual Actions konvertieren, aus der es erstellt wurde. Sie können aus GML in ein code Visual-Ereignis zurückverwandeln, aber jede code wird einfach in eine Execute Code-Aktion verpackt. Dies wird auf der Seite Ändern von GML Visual in Code näher erläutert.
  • Fenster: Dieser Abschnitt öffnet das folgende Untermenü:
    Die hier verfügbaren Menüpunkte sind:
    • Zum Objekt gehen: Damit wird die Seite workspace zu dem Objekt verschoben, das das Ereignis aufruft.
    • Geerbtes Ereignis öffnen: Diese Option ist nur verfügbar, wenn das zu bearbeitende Objekt ein untergeordnetes Objekt eines übergeordneten Objekts ist, und öffnet das zu bearbeitende Ereignis des übergeordneten Objekts und verschiebt die workspace dorthin.
    • Einzelner Bereich / 2 Spalten / 3 Spalten: Diese Optionen können verwendet werden, um das aktuelle workspace in 1, 2 oder 3 verschiedene "Fenster" aufzuteilen. Jeder Bereich ist ein separates workspace, auf das Skripte oder Ereignisse auf sie gezogen werden können, so dass Sie 2 oder mehr GML Visual code Blöcke gleichzeitig bearbeiten und vergleichen können. Die folgenden beiden Optionen sind nur verfügbar, wenn das RMB-Menü auf einem GML Visual-Knoten geöffnet ist:

  • Schwenken: Wenn Sie diese Option für einen beliebigen Knoten auswählen, schwenkt die workspace so, dass dieser Knoten in der Mitte liegt.
  • Hilfe anzeigen... : Wenn Sie dies auswählen, wird das Handbuch in einem neuen workspace auf der Seite geöffnet, die für den ausgewählten Knoten relevant ist. Sie können das gleiche Ergebnis auch mit auf dem Knoten erreichen.
