Anwenden von Aktionen auf andere Instanzen

Die meisten Aktionen in den GML Visual-Bibliotheken haben eine Option, um die Aktion auf verschiedene Arten anzuwenden. Dies wird als Festlegen des Aktionsbereichs bezeichnet und kann eine von mehreren Möglichkeiten sein:

Im Wesentlichen sagst du GameMaker, welche Instanz die Aktion ausführen soll. Der Standard-Aktionsbereich ist self, was bedeutet, dass, sobald das Objekt als Instanz im Raum angelegt ist, diese Instanz die Aktion code ausführen wird. Dies ist jedoch nicht immer das, was Sie wollen, und Sie möchten vielleicht, dass einige Aktionen andere oder sogar alle Instanzen des Raums auswirken. An dieser Stelle kommt die Änderung des Aktionsbereichs ins Spiel.

Die verschiedenen Geltungsbereiche für die Durchführung von Aktionen sind unten aufgelistet. Es ist jedoch zu beachten, dass die Änderung des Geltungsbereichs einer Aktion auf diese Weise nur für diese Aktion gilt und nicht für nachfolgende Aktionen in der Kette. Wenn Sie eine Änderung des Aktionsbereichs auf mehrere verkettete Aktionen anwenden möchten, verwenden Sie zuerst die Aktion Anwenden auf....

Selbst

Dies ist der Standardbereich für eine Aktion und besagt einfach, dass die Aktion nur von der Instanz aufgerufen werden soll, die die code ausführt.

Andere

Der Bereich other hat zwei Hauptfunktionen, und der Wert, den er zurückgibt, hängt davon ab, wo und wie Sie ihn verwenden. Im Kollisionsereignis gibt other den eindeutigen Instanz-ID-Wert zurück (ein eindeutiger Wert, der verwendet wird, um einzelne Instanzen jedes Objekts zu unterscheiden), so dass Sie zum Beispiel eine Instanz "bullet" erstellen und ein Kollisionsereignis mit einer Instanz "player" haben können und dabei den Bereich other verwenden, um dem Objekt "player" Trefferpunkte zu entziehen und dann zum Bereich self zurückzukehren, um die Instanz "bullet" zu zerstören.

Außerhalb des Kollisionsereignisses verhält sich die Einstellung other so, als wäre sie auf noone gesetzt, es sei denn, sie wird innerhalb eines begrenzten Aktionsblocks aufgerufen. Das bedeutet, dass wenn Sie den Bereich einer Gruppe von Aktionen auf ein bestimmtes Objekt ändern, dann wird, während diese Aktionen aufgerufen werden, der other Bereich die Instanz-ID der Instanz zurückgeben, die ursprünglich die Aktionsgruppe aufgerufen hat. Zum Beispiel könnten Sie eine Aktion " Anwenden auf..." ausführen und dann im nächsten code Block den Bereich auf other setzen, um eine Aktion auf der Instanz auszuführen, die den gesamten Ereignisblock ausführt, und nicht auf der Instanz, die in den Blöcken "Anwenden auf code " betroffen ist. Alle

Wenn du eine Aktion für alle auswählst, sagst du GameMaker, dass er diesen Block für jede einzelne aktive Instanz im aktuellen Raum ausführen soll. Wenn du zum Beispiel eine Aktion "Objektinstanz zerstören und setzt ihren Geltungsbereich auf all, so wird jede Instanz im Raum verschwinden, egal aus welchem Objekt sie erzeugt wurde.

Objekt

Einer Aktion kann auch ein Objekt als Geltungsbereich zugewiesen werden. Das bedeutet, dass alle Instanzen des gegebenen Objekts diese Aktion zur gleichen Zeit ausführen, wenn sie aufgerufen wird. Wenn du also z.B. 100 Feindinstanzen im Raum hast, und du sie alle auf einen bestimmten Punkt ausrichten willst. Du rufst die Aktion " Set Point Direction " auf und legst den Aktionsbereich auf das Objekt "obj_Enemy" fest und wenn sie aufgerufen wird, ändern alle Instanzen dieses Objekts ihre Richtung.

Ausdruck

In das Eingabefeld "Ausdruck" können Sie die ID einer bestimmten Instanz eingeben, auf die die Aktion angewendet werden soll. Es kann der eindeutige ID-Wert sein, der einer Instanz aus dem Raum-Editor zugewiesen wurde, oder es kann die ID einer Instanz sein, die du in einer Variablen gespeichert hast in einer Variablen gespeichert haben (wobei die Variable der Eingabewert wäre), oder es kann sogar ein Ausdruck sein, der code verwendet, wie in der Abbildung unten gezeigt:
