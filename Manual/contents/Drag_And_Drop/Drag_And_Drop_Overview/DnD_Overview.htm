GML Visuelle Übersicht

GML Visual ist ein visuelles Skripting-Tool, mit dem Sie Ihre Spiele erstellen können, ohne tatsächlich etwas code eingeben zu müssen. Das soll nicht heißen, dass Sie nicht programmieren code, wenn Sie GML Visual verwenden, denn das tun Sie, nur dass Ihre code auf visuelle Art und Weise erstellt und präsentiert wird, indem "Blöcke" von Aktionen anstelle von Text verwendet werden. Aktionen sind einfache AusdrückeoderAnweisungen, die miteinander "verkettet" werden können, um eine Instanz eines Objekts etwas tun zu lassen, wenn sie in einem Raum platziert wird. Sie könnten zum Beispiel eine Aktion in einem Erstellungs-Ereignis haben, um eine Instanz in eine zufällige Richtung zu bewegen, und eine andere Aktion in einem Kollisions-Ereignis, um sie von den Dingen abprallen zu lassen, mit denen sie kollidiert (weitere Informationen über die verschiedenen verfügbaren Ereignisse und wie sie zusammenarbeiten, finden Sie im Abschnitt über Objekt-Ereignisse).

Das obige Bild zeigt ein typisches Objekt, das auf einer Arbeitsfläche geöffnet ist und bearbeitet werden soll. Es hat ein "Ereignis"-Fenster und ein "Code"-Fenster, die mit ihm verbunden sind, und im code Fenster können Sie sehen, wo wir unser GML Visual aufgebaut haben. Registerkarten für Ereignisse

Wenn Sie Ihrem Objekt ein Ereignis hinzufügen, wird im Fenster code eine neue Registerkarte geöffnet, die den Inhalt dieses Ereignisses anzeigt. Sie können mehrere Ereignisse hinzufügen und mehrere Registerkarten gleichzeitig geöffnet haben, zwischen denen Sie wechseln und sie nach Belieben bearbeiten können. Sie können die Registerkarten auch jederzeit schließen, um die Übersichtlichkeit zu wahren (durch erneutes Auswählen des Ereignisses im Objekt werden sie wieder geöffnet), und Sie können auf klicken und die Registerkarte aus dem Fenster in ein eigenes Fenster code ziehen, wobei das Schließen des Objekts oder des Fensters code nicht zum Schließen des neuen Fensters führt. Registerkarten können auch nach links oder rechts gezogen werden, um sie neu zu ordnen.

Aktionsübersicht

Wenn Sie Ihrem Ereignis Aktionsblöcke hinzufügen, werden diese in "Kurzform" als einfacher englischer Überblick über die Vorgänge angezeigt. Die Reihenfolge der Übersicht ist die Reihenfolge, in der die einzelnen Aktionsblöcke für das Ereignis ausgeführt werden, und Sie können auf ein beliebiges Element in der Liste klicken, um die Ansicht der Aktionsblöcke zur Bearbeitung an diese Position zu zoomen.

Aktionsblock Arbeitsbereich

In diesen Bereich workspace können Sie Aktionen aus der Toolbox ziehen, um Ihr GML Visual code zu erstellen. Klicken Sie auf auf eine Aktion und ziehen Sie sie in diesen Bereich, um sie der Liste der Aktionen hinzuzufügen und sie zu bearbeiten. Die meisten Aktionen haben verschiedeneArgumente (Parameter), die Sie festlegen können, und die meisten nehmen auch Schlüsselwörter, Instanzvariablen oder sogar GML-Funktionen an - zum Beispiel können Sie die Aktion Richtung festlegen aufrufen und die Funktion GML random(val) um eine zufällige Richtung festzulegen.

Wenn Sie Ihrem code Aktionsblöcke hinzufügen, werden diese visuell "verkettet", um die Reihenfolge anzugeben, in der die verschiedenen Aktionen ausgeführt werden, wobei die Art und Weise, wie diese Kette erstellt wird, vom verwendeten Ausgangsblock abhängt. Ein If-Variable-Block hat beispielsweise zwei Ketten - eine zu den Blöcken, die aufgerufen werden sollen, wenn die überprüfte Bedingung true lautet, und dann der nächste Block danach. Weitere Informationen zum Aufbau von Aktionsblöcken in code finden Sie in den unten verlinkten Abschnitten.

Wenn Sie sich im Aktionsblock workspace befinden, können Sie die mittlere Maustaste drücken und gedrückt halten, um die Ansicht zu verschieben, und das Mausrad verwenden, um die Aktionsliste nach oben oder unten zu verschieben. Wenn Sie gedrückt halten und das Mausrad bewegen, wird die Ansicht nach links/rechts gescrollt, und wenn Sie / gedrückt halten und das Mausrad bewegen, wird die Ansicht vergrößert und verkleinert, so dass Sie mehr von workspace mit den Aktionen sehen können. Sie können auf auf eine beliebige Aktion aus der Aktionsliste auf der linken Seite klicken, um den Fokus auf diese Aktion zu setzen und den Zoom und die Position zurückzusetzen. Beachten Sie auch, dass Sie die Größe der Aktionsblöcke ändern können, indem Sie die Maus an den Rand des Blocks bewegen und dann - wenn sich der Cursor ändert - in die gewünschte Richtung ziehen.

Sie können auch mit der rechten Maustaste auf an beliebiger Stelle in GML Visual workspace klicken, um das RMB-Menü aufzurufen, das hier im Detail erklärt wird.

Werkzeugkasten

In der Toolbox werden alle Aktionen von GML Visual gespeichert, wobei verschiedene Abschnitte Sammlungen (Bibliotheken) von Aktionen enthalten, die ähnlich oder verwandt sind. Jede Sammlung von Aktionen bezieht sich auf ein bestimmtes Thema oder einen gemeinsamen Zweck. Sie klicken einfach auf und ziehen die gewünschte Aktion aus der Toolbox in den Aktionsblock workspace, um sie dem aktuellen Ereignis hinzuzufügen. Zur Vereinfachung - da es sehr viele Aktionen gibt - können Sie die Suchleiste oben verwenden, um nach einer bestimmten Aktion zu suchen oder die sichtbaren Aktionen zu filtern (geben Sie z. B. "Zeichnen" ein, um alle Zeichenaktionen zu erhalten), und Sie können auch auf klicken und Symbole aus einer beliebigen Bibliothek in Ihren Favoritenordner ziehen, um häufig verwendete Aktionen zusammenzufassen. Dies ist besonders nützlich, wenn Sie bestimmte Dinge immer wieder tun werden. Der Favoriten-Ordner kann durch Klicken auf das kleine "Mülleimer"-Symbol geleert werden.

Eine vollständige Liste der verfügbaren Aktionen sowie die Informationen, die sie benötigen und wie sie verkettet werden können, finden Sie unter arguments im Abschnitt GML Visual Reference.

Sie können mit der rechten Maustaste auf eine beliebige Aktion in der Toolbox klicken, um das folgende Menü zu öffnen:

Hier haben Sie die folgenden Möglichkeiten:

  • Verwenden Sie<Aktion>: Dies fügt die Aktion, die sich unter der Maus befand, als die rechte Maustaste geklickt wurde, zum aktuellen Ereignis workspace hinzu.
  • Favorit erstellen/entfernen: Damit wird die Aktion zur Favoritenbibliothek oben in der Toolbox hinzugefügt oder entfernt. In dieser Bibliothek können Sie eine benutzerdefinierte Gruppe von Aktionen erstellen, die Sie regelmäßig verwenden, so dass Sie weniger Zeit für die Suche in den eigentlichen Bibliotheken aufwenden müssen.
  • Favoriten löschen: Wenn Sie diese Option wählen, werden alle Aktionen aus der Favoritenbibliothek entfernt.
  • Zuletzt verwendetes löschen: Wenn Sie diese Option wählen, werden alle Aktionen aus der Bibliothek "Zuletzt verwendet" entfernt.
  • Alle einklappen: Mit dieser Option werden alle GML Visual-Bibliotheken eingeklappt, so dass nur die Bibliothekstitel sichtbar sind, nicht aber die Aktionsknoten. Jede Bibliothek kann wieder aufgeklappt werden, indem Sie auf sie klicken.
