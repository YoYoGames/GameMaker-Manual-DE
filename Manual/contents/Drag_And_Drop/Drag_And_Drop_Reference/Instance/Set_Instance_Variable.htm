Instanzvariable setzen

Mit dieser Aktion können Sie eine der eingebauten Variablen setzen. Alle Instanzen aller Objekte haben bereits eine Reihe von Variablen in sich, die wichtige Informationen wie die aktuelle x/y-Position, die Geschwindigkeit, die Richtung, die Sichtbarkeit und viele andere Dinge. Die Instanzvariablen, die Sie verwenden können, sind:

  • X-Koordinate - Die Position entlang der X-Achse im Raum.
  • Y-Koordinate - Die Position entlang der Y-Achse im Raum.
  • Richtung - Die Bewegungsrichtung der Instanz (Standard ist 0, wobei rechts 0°, oben 90°, links 180° und unten 270° ist).
  • Geschwindigkeit - Die Geschwindigkeit (in Pixel pro Spielframe), mit der sich die Instanz bewegt.
  • Horizontale Geschwindigkeit - Die horizontale Geschwindigkeit (in Pixel pro Spielframe), mit der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich nach links, eine positive Geschwindigkeit nach rechts und der Standardwert ist 0.
  • Vertikale Geschwindigkeit - Die vertikale Geschwindigkeit (in Pixel pro Spielframe), mit der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich nach oben bewegt, eine positive Geschwindigkeit bedeutet nach unten und der Standardwert ist 0.
  • Sichtbar - Die Variable zum Umschalten der Sichtbarkeit (Standard ist true, sofern im Objekteditor nicht anders eingestellt, auf false setzen, um unsichtbar zu machen). Beachten Sie, dass eine unsichtbare Instanz immer noch alle Ereignisse mit Ausnahme des Zeichnungsereignisses ausführt.
  • Persistent - Die Toggle-Variable für die Persistenz (Standard ist false, sofern im Objekteditor nicht anders eingestellt, setze auf true um persistent zu machen). Dies lässt eine Instanz von Raum zu Raum "bestehen", d.h. die Instanz wird am Ende des Raumes nicht aufgeräumt, sondern wird in den nächsten Raum übertragen. Beachte, dass wenn eine Instanz persistent gemacht wird weder das Ereignis Create noch Destroy aufruft, sondern die Ereignisse Clean Up, Room End und Room Start.
  • Ebene - Die Ebene, der die Instanz zugewiesen wurde.
  • Tiefe - Der der Ebene zugewiesene Tiefenwert.
  • Sprite - Die sprite Ressource, die der Instanz zugewiesen ist.
  • Bildindex - Der Bildindex des zugewiesenen sprite. Dieser Wert legt das Bild des sprite fest, das der Instanz zugewiesen ist, und wenn Wenn der Wert größer ist als die Anzahl der Frames in der Animation, wird eine Schleife erstellt, so dass ein sprite mit 5 Frames, das auf Frame 7 eingestellt ist, tatsächlich Frame 2 zeigt.
  • Bildwinkel - Der Winkel, auf den die Instanz eingestellt werden soll (Standard ist 0, wobei rechts 0°, oben 90°, links 180° und unten 270° ist).
  • Image Alpha- Der Alphawert für die Instanz (Standardwert ist 1) von 0 bis 1, wobei 0 für vollständig transparent und 1 für vollständig undurchsichtig steht.
  • Image Blend - Die Farbe, die mit der Instanz sprite überblendet werden soll (Standard ist weiß).
  • Bildgeschwindigkeit - Die Animationsgeschwindigkeit für die Instanz.
  • Horizontaler Maßstab - Der horizontale Maßstab, mit dem die Instanz sprite gezeichnet werden soll (dies wirkt sich auch auf den Begrenzungsrahmen der Instanz und die Kollisionen aus). Box und Kollisionen). Ein Maßstab von 1 bedeutet keine Skalierung (1:1), kleinere Werte skalieren nach unten (0.5, zum Beispiel, halbiert die Breite der sprite), größere Werte skalieren nach oben und negative Werte spiegeln die sprite und skalieren sie, außer der verwendete Wert ist genau -1 (in diesem Fall wird sprite einfach an der y-Achse gespiegelt und nicht skaliert).
  • Vertikaler Maßstab - Der vertikale Maßstab, mit dem die Instanz sprite gezeichnet werden soll (dies wirkt sich auch auf den Begrenzungsrahmen der Instanz und die Kollisionen aus). Box und Kollisionen). Ein Maßstab von 1 bedeutet keine Skalierung (1:1), kleinere Werte skalieren nach unten (0.5, zum Beispiel, halbiert die Höhe der sprite), größere Werte skalieren nach oben und negative Werte spiegeln die sprite und skalieren sie, außer der verwendete Wert ist genau -1 (in diesem Fall wird sprite einfach entlang der y-Achse gespiegelt und nicht skaliert).
  • Maske - Der Maskenindex für die Instanz. Der Standardwert ist -1 (was bedeutet, dass die Instanz die Kollisionsmaske für den aktuell zugewiesenen Index sprite ), aber wenn er auf eine andere sprite Ressource gesetzt wird, wird die Kollisionsmaske für diese Instanz anstelle der sprite verwendet, die dem Index sprite zugewiesen ist.
  • Reibung - Die Reibung, die bei jedem Spielframe des Spiels verwendet wird, um die Instanz zu verlangsamen. Dieser Wert wird einfach von dem aktuellen Geschwindigkeitswert abgezogen (oder hinzugefügt, wenn der Geschwindigkeitswert negativ ist).
  • Schwerkraft - Schwerkraft übt eine konstante Kraft in Richtung der Schwerkraft (siehe unten) auf die Instanz aus und beeinflusst ihre Geschwindigkeit (Standard ist 0). Beachten Sie, dass die Schwerkraft eine kumulative Kraft ist und das Objekt beschleunigt, wenn Sie die Endgeschwindigkeit nicht begrenzen wollen.
  • Gravity Direction - Hier kann die Richtung der Bewegung festgelegt werden, wenn die Schwerkraft der Instanz größer als 0 ist (Standard ist 270). Beachten Sie, dass die Richtungen in GameMaker so berechnet werden, dass 0° rechts, 90° oben, 180° links und 270° unten ist.
  • Score - Setzen Sie die Variable score.
  • Lives - Setzen Sie die Variable lives.
  • Gesundheit - Setzen Sie die Variable health.

Sobald Sie die zu verwendende integrierte Variable ausgewählt haben, legen Sie den Wert für diese Variable fest. Der von Ihnen verwendete Wert hängt von der ausgewählten Variable ab (Standardwerte und Bereiche siehe Liste oben), und Sie können den Wert relativ zum aktuellen Wert für die Variable. Sie können auch weitere Variablen hinzufügen, die in derselben Aktion gesetzt werden sollen, indem Sie auf das Plus-Symbol neben neben der Aktion klicken, eine weitere Instanzvariable auswählen und ihr einen Wert zuweisen.
