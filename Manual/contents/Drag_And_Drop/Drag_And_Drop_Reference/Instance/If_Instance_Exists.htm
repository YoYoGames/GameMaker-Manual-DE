Wenn die Instanz existiert

Diese Aktion kann verwendet werden, um zu prüfen, ob eine Instanz eines bestimmten Objekts im Spielraum existiert oder ob eine bestimmte Instanz selbst existiert. Du gibst das Objekt aus dem Asset Explorer an, und wenn eine Instanz im Raum existiert, gibt die Aktion true zurück, andernfalls false. Beachten Sie, dass Sie hier eine Variable mit der eindeutigen ID für eine bestimmte Instanz verwenden können (z. B. wie sie von der Aktion Objektinstanz erstellen zurückgegeben wird) oder sogar die Instanz-ID Wert, der vom Raumeditor zurückgegeben wurde, als du die Instanz hinzugefügt hast.

Wenn Sie das Feld "Not" argument markieren, prüft die Aktion, ob keine Instanz existiert, und wenn keine gefunden wird, wird true zurückgegeben, und wenn sogar eine existiert, wird false zurückgegeben.

Der obige Aktionsblock code prüft auf eine Kollision und gibt den eindeutigen ID-Wert der Instanz zurück, mit der kollidiert wird. Dieser Wert wird in einer temporären Variable gespeichert und dann überprüft, ob der Wert einer Instanz entspricht, die sich gerade im Raum befindet (wenn keine Kollision gefunden wird, enthält die Variable das Schlüsselwort "noone"), Wenn dies der Fall ist, wird eine Variable in der Instanz gesetzt, mit der kollidiert wird.
