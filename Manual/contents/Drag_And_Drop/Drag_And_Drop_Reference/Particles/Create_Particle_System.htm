Partikelsystem erstellen

Diese Aktion initialisiert das Partikelsystem, so dass Sie Partikeltypen und Emitter damit erstellen und verwenden können. Du gibst den Layernamen als String an - zum Beispiel "Particle Layer" (der im Raumeditor existieren muss) - und kennzeichnest dann das System als persistent oder nicht. Wenn das System als persistent gekennzeichnet ist, bleibt es in allen Räumen aktiv, nachdem es erstellt wurde, und ihm wird eine verwaltete Ebene zugewiesen, wenn die ursprüngliche Ebene in nachfolgenden Räumen nicht vorhanden ist. Wenn die Persistenz nicht markiert ist, wird das System am Ende des Raums, in dem es erstellt wurde, automatisch zerstört, ebenso wie alle mit dem System verbundenen Emitter. Wenn Sie das System manuell aus dem Speicher entfernen möchten, müssen Sie die Aktion die Aktion Partikelsystem zerstören.

Sie müssen auch eine Zielvariable angeben, die den eindeutigen Partikelsystem-ID-Wert enthält. Dieser ID-Wert wird in anderen Funktionen verwendet, um Eigenschaften des Systems festzulegen oder um GameMaker mitzuteilen, welches System bei der Erstellung von Partikeln verwendet werden soll. Du kannst diese Variable als temporäre lokale Variable kennzeichnen, aber das bedeutet, dass sie am Ende der Aktion script oder des Ereignisses verworfen wird, was bedeutet, dass das System immer noch existiert, du aber keine Möglichkeit hast, auf es zuzugreifen. (im Allgemeinen müssen Systeme nie in einer temporären lokalen Variable gespeichert werden, aber für bestimmte Dinge, wie das Zeichnen von Partikeln auf einer Oberfläche, kann dies erforderlich sein).

HINWEIS: Ein einmal erstelltes persistentes Partikelsystem verbleibt für die Dauer des Spiels im Systemspeicher. Wenn Sie es also zu irgendeinem Zeitpunkt neu erstellen müssen (z. B. wenn Sie die Aktion " Spiel neu starten" aufrufen), müssen Sie das aktuelle System zuerst zerstören, da sonst ein Speicherleck entsteht. Ein Speicherleck entsteht, wenn eine Ressource dem Speicher hinzugefügt und dann "de-referenziert" wird, was bedeutet, dass GameMaker keine Möglichkeit mehr hat, mit dieser Ressource zu interagieren. dieser Ressource zu interagieren. Dies führt dazu, dass der Speicher durch unzugängliche Ressourcen verstopft wird, was wiederum zu Problemen im Spiel führen kann. Um ein persistentes Partikelsystem zu zerstören, wenn es nicht mehr benötigt wird, verwenden Sie die Aktion Partikelsystem zerstören.

Aktion Syntax:

Argumente:

Argument Beschreibung
Layer Die Ebene, auf der die Partikel in diesem System gezeichnet werden sollen
Persistent Ob das Partikelsystem aktiv bleiben soll, nachdem der aktuelle Raum beendet wurde
Target Die Zielvariable, die den eindeutigen ID-Wert des erstellten Systems enthält

Der obige Aktionsblock erzeugt eine globale Variable, erstellt dann ein neues Partikelsystem und weist dessen eindeutigen ID-Wert der globalen Variablen zu. Das erstellte Partikelsystem ist als persistent gekennzeichnet, so dass es auch nach dem Ende des aktuellen Raums aktiv bleibt.
