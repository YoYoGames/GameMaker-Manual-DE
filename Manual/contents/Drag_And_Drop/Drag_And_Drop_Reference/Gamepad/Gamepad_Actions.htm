Gamepad-Aktionsbibliothek

GameMaker verfügt über eine Reihe spezieller Aktionen, die verwendet werden können, um sowohl analoge als auch digitale Steuerungen von mehreren angeschlossenen Gamepads zu erkennen. Für diese Aktionen müssen Sie einen Gamepad-Indexwert angeben, d. h. eine Zahl (ab 0), die das ausgewählte Gamepad darstellt. Beachten Sie, dass ein asynchrones Systemereignis ausgelöst wird, wenn ein Gamepad an Ihr Gerät angeschlossen wird (oder wenn es entfernt wird) (um dies zu bewältigen, müssen Sie möglicherweise code verwenden).

Die Gamepad-"Steckplätze" sind von 0 an indiziert, wobei die Steckplätze 0 bis einschließlich 3 nur für Xinput-Gamepads, d. h. Xbox360-Controller und kompatible Geräte, gelten. Sie können jedoch auch die Steckplätze 4 bis einschließlich 11 für DirectInput-Gamepads überprüfen, was bedeutet, dass Sie viele andere Controller-Modelle erkennen können, wenn sie über diese Steckplätze angeschlossen sind.

Eingangskonstanten

Bei der Arbeit mit den Gamepad-Aktionen kann die Eingabe über Achsen, Tasten oder Hats erfolgen, die GameMaker den folgenden eingebauten Konstanten zuordnet (beachten Sie, dass "Hats" im Allgemeinen nur bei Nicht-Standard-Controllern erkannt werden):

Konstante Beschreibung
gp_face1 Obere Taste 1 (diese entspricht dem "A" auf einem Xbox 360-Controller und dem Kreuz auf einem PS-Controller)
gp_face2 Obere Taste 2 (diese entspricht dem "B" auf einem Xbox 360-Controller und dem Kreis auf einem PS-Controller)
gp_face3 Obere Taste 3 (diese entspricht dem "X" auf einem Xbox 360-Controller und dem Quadrat auf einem PS-Controller)
gp_face4 Obere Taste 4 (diese entspricht dem "Y" auf einem Xbox 360-Controller und dem Dreieck auf einem PS-Controller)
gp_shoulderl Linke Schultertaste
gp_shoulderlb Linker Schulterauslöser
gp_shoulderr Rechte Schultertaste
gp_shoulderrb Rechter Schulterauslöser
gp_select Die Auswahltaste (bei einem DS4-Controller wird diese ausgelöst, wenn Sie das Touchpad nach unten drücken)
gp_start Die Starttaste (dies ist die Optionstaste auf einem PS4-Controller)
gp_stickl Der linke Stick wird gedrückt (als Taste)
gp_stickr Der rechte Stick wird gedrückt (als Taste)
gp_padu D-Pad nach oben
gp_padd D-Pad nach unten
gp_padl D-Pad links
gp_padr D-Pad rechts
gp_axislh Linker Knüppel horizontale Achse (analog)
gp_axislv Linker Knüppel vertikale Achse (analog)
gp_axisrh Rechter Knüppel horizontale Achse (analog)
gp_axisrv Rechter Knüppel vertikale Achse (analog)

Die folgende Liste zeigt die derzeitige Kompatibilität von platforms (bitte beachten Sie, dass sich diese Liste mit zukünftigen Updates ändern wird):

  • Windows wird vollständig unterstützt, wobei bis zu 11 angeschlossene Geräte gleichzeitig zulässig sind. Beachten Sie, dass unter Windows die ersten 4 Gamepad-Steckplätze (0 - 3) über die XInput dll verwaltet werden, was bedeutet, dass nur XBox-Controller zu 100 % kompatibel sind und dass Sie für andere Controller-Typen die restlichen Gamepad-Steckplätze (4 - 11) überprüfen sollten.
  • macOS wird mit maximal 4 angeschlossenen Geräten auf einmal unterstützt, und diese Geräte können NUR vom Typ Playstation3 oder Xbox 360 sein. Bitte beachten Sie, dass die Option "Für Mac AppStore bauen" in den Mac-Spieloptionen ausgeschaltet sein muss, damit die Pad-Unterstützung funktioniert.
  • Ubuntu unterstützt auch die GamePad-Eingabe, aber Sie müssen möglicherweise zusätzliche Bibliotheken aus dem Ubuntu Repository installieren. Sie können dies ganz einfach tun, indem Sie ein Kommandozeilenterminal öffnen und Folgendes eingeben, wodurch die GUI-Unterstützung für den Joystick sowie der Joystick selbst installiert wird:

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5-Spiele unterstützen Gamepads in den meisten gängigen Browsern, mit Ausnahme von Safari.
  • Die Gamepad-Unterstützung erstreckt sich auch auf iOS mit dem iCade-Gehäuse. Die linke Achse entspricht dem Stick-Controller (obwohl die Eingabe digital und nicht analog ist), die vier "Gesichtstasten" entsprechen den Tasten an der Vorderseite des Gehäuses und die vier Schultertasten entsprechen denen auf der Rückseite des Gehäuses.
  • Android export unterstützt NYKO-Controller und generische Bluetooth-Controller, aber nur, wenn sie aktiviert sind, d. h. Sie müssen die Option iCade/Bluetooth im Abschnitt "Allgemein" der Android-Spieloptionen aktivieren. Sie benötigen API-Level 12, um vollständig zu funktionieren, und es ist zu beachten, dass GameMaker alle Bluetooth-Geräte, mit denen Ihr Gerät gekoppelt ist, als verbunden registriert, unabhängig davon, ob es tatsächlich verbunden ist oder nicht. Daher sollte dies bei der Zuweisung und Überprüfung von "Slots" berücksichtigt werden.
  • Auf PS4 müssen Sie, wenn Sie das Touchpad-Tracking verwenden möchten, die Funktionen für die device_mouse_* Tasten verwenden (dafür gibt es keine GML Visual-Aktionen).
