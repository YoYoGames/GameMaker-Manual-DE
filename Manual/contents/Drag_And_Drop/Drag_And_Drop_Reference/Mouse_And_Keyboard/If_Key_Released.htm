Wenn der Schlüssel freigegeben ist

Diese Aktion prüft, ob eine Taste der Tastatur losgelassen wurde. Sie gibt true bei dem einen Spielschritt (Frame) zurück, bei dem die Taste losgelassen wurde, und false zu allen anderen Zeiten. Wenn Sie den Modifikator not aktivieren, wird diese Aktion umgekehrt und prüft, ob die Taste nicht losgelassen wurde, und gibt false bei dem Spielschritt (Frame) zurück, bei dem sie losgelassen wurde, und true zu allen anderen Zeiten.

Beachten Sie, dass Aktionen, die Sie in den "if"-Block einfügen möchten, an der Seite der Aktion abgelegt werden müssen, wie in der folgenden Abbildung gezeigt:

Diese Aktionen werden nun ausgeführt, wenn das "if" den Wert true ergibt, während alle Aktionen, die an anderer Stelle ausgelassen werden, nach dem "if"-Block ausgeführt werden.

Aktion Syntax:

Argumente:

Argument Beschreibung
Key Der Name des zu prüfenden Schlüssels.
Not Negieren der Prüfung (aus wahr wird falsch und umgekehrt)

Beispiel:

Der obige Aktionsblock code fragt bei jedem Schritt den Tastaturstatus ab und wenn eine Taste gedrückt gehalten wird, prüft er, ob die Taste zum ersten Mal gedrückt wird. Lautet die Prüfung true auf den ersten Tastendruck, wird ein Ton abgespielt, und während die Taste gedrückt gehalten wird, wechselt die Überblendung sprite auf Rot. Wenn die Taste nicht gedrückt gehalten wird wird beim Loslassen der Taste eine Prüfung durchgeführt, um die Mischfarbe auf Weiß zurückzusetzen.
