Bei gedrückter Taste

Diese Aktion prüft, ob eine Taste zum ersten Mal gedrückt wird. Sie gibt true bei dem einen Spielschritt (Frame) zurück, bei dem die Taste zum ersten Mal gedrückt wird, und false zu allen anderen Zeiten. Wenn Sie den Modifikator not aktivieren, wird diese Aktion umgekehrt und prüft, ob die Taste nicht gedrückt wurde, wobei false bei dem Spielschritt (Frame) zurückgegeben wird, bei dem die Taste gedrückt wurde, und true zu allen anderen Zeiten. Wenn Sie prüfen müssen, ob eine Taste gedrückt gehalten wird und nicht nur das anfängliche Druckereignis, dann verwenden Sie die Aktion If Key Down.

Beachten Sie, dass Aktionen, die Sie in den "if"-Block einfügen möchten, an der Seite der Aktion abgelegt werden müssen, wie in der folgenden Abbildung gezeigt:

Alle Aktionen, die an anderer Stelle entfallen, werden nach dem "if"-Block ausgeführt.

Aktion Syntax:

Argumente:

Argument Beschreibung
Key Der Name des zu prüfenden Schlüssels.
Not Negieren der Prüfung (aus wahr wird falsch und umgekehrt)

Beispiel:

Der obige Aktionsblock code fragt bei jedem Schritt den Tastaturstatus ab und prüft, ob eine Taste gedrückt gehalten wird, prüft er, ob die Taste zum ersten Mal gedrückt wurde. Lautet die Prüfung true auf den ersten Tastendruck, wird ein Ton abgespielt und dann, während die Taste gedrückt gehalten wird, die Überblendung sprite auf rot geändert. Wenn die Taste nicht gedrückt gehalten wird, wird eine Prüfung beim Loslassen der Taste durchgeführt, um die Mischfarbe auf Weiß zurückzusetzen.
