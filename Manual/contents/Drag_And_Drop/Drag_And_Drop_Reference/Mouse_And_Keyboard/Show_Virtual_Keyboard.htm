Virtuelle Tastatur anzeigen

Diese Aktion kann verwendet werden, um die virtuelle Tastatur auf dem Gerät anzuzeigen, auf dem das Spiel läuft. Wenn Sie diese Aktion aufrufen, werden Sie aufgefordert, für jedes der ersten drei Argumente eine der folgenden Konstanten anzugeben:

  • Tastaturtyp: Mit dieser Konstante wird festgelegt, welcher Tastensatz auf der virtuellen Tastatur verfügbar sein soll. Die verfügbaren Tastaturtypen sind:
    Tastatur Typ Beschreibung
    kbv_type_default Der Standard-Tastaturtyp für das aktuelle System.
    kbv_type_ascii Eine reine ASCII-Tastatur.
    kbv_type_url Eine normale Tastatur, die für die Eingabe von URLs optimiert ist. Enthält in der Regel eine ".com"- oder eine andere Domänen-Taste sowie "/"- und "."-Tasten.
    kbv_type_email Eine normale Tastatur, die für die Eingabe von E-Mails optimiert ist. Enthält in der Regel die Zeichen "@" und ".".
    kbv_type_numbers Eine reine Zahlentastatur, die in der Regel als Ziffernblock angezeigt wird.
    kbv_type_phone Eine Telefontastatur. Normalerweise nur Zahlen mit den Tasten "*" und "#".
    kbv_type_phone_name Eine Tastatur, die für die Eingabe einer Telefonnummer und eines Namens optimiert ist. In der Regel ähnlich wie eine ASCII-Tastatur, jedoch mit einer eingeschränkten Auswahl an Sonderzeichen.
  • Rückgabetyp: Diese Konstante wird verwendet, um festzulegen, was auf der Rückgabe-/Aktionstaste der virtuellen Tastatur angezeigt wird. Die verfügbaren Rückgabearten sind:
    Rückgabe Typ Beschreibung
    kbv_returnkey_default Der Standardtitel der Return-Taste für das aktuelle System.
    kbv_returnkey_go Setzt den Titel der Rückgabetaste auf "Go".
    kbv_returnkey_google Setzt den Titel der Rückgabetaste auf "Google" oder in manchen Fällen auf ein allgemeines Suchsymbol.
    kbv_returnkey_join Setzt den Titel der Rückgabetaste auf "Los".
    kbv_returnkey_next Setzt den Titel der Rückgabetaste auf "Weiter".
    kbv_returnkey_route Legt den Titel der Rückgabetaste auf "Route" fest.
    kbv_returnkey_search Setzt den Titel der Rückgabetaste auf "Suchen" oder in einigen Fällen auf ein allgemeines Suchsymbol.
    kbv_returnkey_send Setzt den Titel der Rückgabetaste auf "Senden".
    kbv_returnkey_yahoo Setzt den Titel der Rückgabetaste auf "Yahoo" oder in einigen Fällen auf ein allgemeines Suchsymbol.
    kbv_returnkey_done Setzt den Titel der Rückgabetaste auf "Fertig".
    kbv_returnkey_continue Setzt den Titel der Rückgabetaste auf "Weiter".
    kbv_returnkey_emergency Setzt den Titel der Rückgabetaste auf "Notruf".
  • Typ der automatischen Großschreibung: Mit dieser Konstante wird festgelegt, wie die über die virtuelle Tastatur eingegebenen Wörter automatisch großgeschrieben werden sollen. Die verfügbaren Typen der automatischen Großschreibung sind:
    Autokapitalisierung Typ Beschreibung
    kbv_autocapitalize_none Die automatische Großschreibung ist deaktiviert.
    kbv_autocapitalize_words Wörter werden automatisch großgeschrieben.
    kbv_autocapitalize_sentences Sätze werden automatisch großgeschrieben.
    kbv_autocapitalize_characters Alle Zeichen werden großgeschrieben.

Die letzte argument ist die Aktivierung/Deaktivierung der Textvorhersage, die auf true gesetzt wird, um sie zuzulassen, und andernfalls auf false. Beachten Sie jedoch, dass die Zulassung der Textvorhersage nicht bedeutet, dass sie auch verwendet wird, da dies von den Präferenzen des Benutzers auf dem Gerät abhängt. Im Modus "Textvorhersage" erzeugt die virtuelle Tastatur keine normalen GameMaker-Tastendruck-Ereignisse. Stattdessen werden nur das zuletzt gedrückte Zeichen und die Tastaturstring-Variablen aktualisiert. Das liegt daran, dass die Unfähigkeit zu erkennen, ob eine Änderung im internen Textfeld, das zur Erkennung von Tastendrücken verwendet wird, von einer tatsächlichen Taste der virtuellen Tastatur oder von einem Textvorschlag stammt. In diesen Fällen sollten Sie die Eingabe von keyboard_string lesen und nicht als jede Art von roher Tasteneingabe zu lesen.

Es ist auch wichtig zu beachten, dass der Benutzer je nach platform OS unterschiedliche Tastaturen mit unterschiedlichen Fähigkeiten erhält, mit den folgenden Vorbehalten für die Verwendung pro Ziel:

  • UWP: Die UWP-Implementierung unterstützt keine return_key_type, autocapitalization_type oder predictive_text_enabled. Die Übergabe dieser Parameter an eine Funktion führt nicht zu Fehlern, sie haben nur keine Auswirkungen auf die Tastatur.
  • Android/Amazon Fire: Auf Android wird der Tastaturtyp kbv_type_phone_name nicht unterstützt und es wird stattdessen eine ASCII-Tastatur angezeigt, und der Rückgabetastentyp kann nur kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next sein, kbv_returnkey_send oder kbv_returnkey_done sein. Wenn andere Return-Tastentypen auf platform verwendet werden, wird das System standardmäßig die unten aufgeführten Ersatz-Return-Tasten verwenden:
    Nicht verfügbare Rückgabetaste Ersatztaste
    kbv_returnkey_google
    kbv_returnkey_yahoo    		 kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		 kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV/FireTV: Benutzerdefinierte Return-Tasten werden auf ASCII-Tastaturen nicht unterstützt - die Standard-Return-Taste wird immer angezeigt.
  • tvOS: Prädiktiver Text und automatische Großschreibung werden auf tvOS nicht unterstützt, und die Höhe der Tastatur, die von Systemereignissen und der Funktion keyboard_virtual_height() zurückgegebene Höhe der Tastatur entspricht standardmäßig der Bildschirmhöhe, da sie sich über den gesamten Bildschirm erstreckt und die Funktionen zur Rektaldimensionierung der Tastatur auf tvOS deaktiviert sind. Beachten Sie auch, dass physische (Bluetooth-)Tastaturereignisse nur dann übertragen werden, wenn die virtuelle Tastatur geöffnet wurde.

Der Aufruf dieser Funktion erzeugt ein asynchrones Systemereignis, bei dem die async_load  DS-Map mit den folgenden Schlüssel/Wertpaaren gefüllt wird:

  • "event_type" - dies ist der "Status der virtuellen Tastatur", wenn er durch Aktionen der virtuellen Tastatur ausgelöst wird.
  • "screen_height" - die Höhe der virtuellen Tastatur (in Pixel). Dieser Wert ist 0, wenn die Tastatur unsichtbar ist.
  • "keyboard_status" - der aktuelle Status der Tastatur, zurückgegeben als eine der folgenden Zeichenketten:
    • "versteckt"
    • "versteckt"
    • "zeigend"
    • "sichtbar"

Aktion Syntax:

Argumente:

Argument Beschreibung
keyboard_type Bestimmt den auf der virtuellen Tastatur verfügbaren Tastensatz.
return_key_type Legt fest, was auf der Return-/Aktionstaste der virtuellen Tastatur angezeigt wird.
autocapitalization_type Legt fest, ob und wie die über die virtuelle Tastatur eingegebenen Wörter automatisch großgeschrieben werden.
predictive_text_enabled Stellen Sie true/false ein, um die Texterkennung zu aktivieren/deaktivieren.
