Wenn undefiniert

Diese Aktion wird verwendet, um zu prüfen, ob eine Variable oder ein Ausdruck den Wert "undefiniert" zurückgibt. Wenn dies der Fall ist, gibt diese Aktion true zurück, wenn nicht, false. "Undefiniert" kann aus verschiedenen Gründen zurückgegeben werden Ein undefinierter Wert kann einen Absturz oder einen schwerwiegenden Fehler in Ihrem Projekt verursachen, so dass Sie diese Aktion verwenden können, um dies zu überprüfen und abzufangen, bevor es passiert. Wenn Sie das Kennzeichen "Not" argument setzen, dann prüfen Sie, ob ein Wert nicht "undefiniert" ist.

Beachten Sie, dass Aktionen, die Sie in den "if"-Block einfügen möchten, an der Seite der Aktion abgelegt werden müssen, wie in der folgenden Abbildung gezeigt:

Diese Aktionen werden nun ausgeführt, wenn der "if"-Block den Wert true ergibt, während alle Aktionen, die an anderer Stelle ausgelassen werden, nach dem "if"-Block ausgeführt werden. Beachten Sie, dass Sie undefinierte Prüfungen in derselben Aktion "stapeln" können, indem Sie auf das Plus-Symbol neben neben der Aktion klicken und eine weitere zu prüfende Variable angeben. Dadurch wird geprüft, ob alle angegebenen Variablen in einer Prüfung undefiniert sind, und nur wenn dies der Fall ist, wird die hinzugefügte Aktion code ausgeführt.

Argument Beschreibung
