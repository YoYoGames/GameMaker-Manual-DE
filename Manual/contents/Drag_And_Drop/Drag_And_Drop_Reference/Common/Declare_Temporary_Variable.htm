Temporäre Variable deklarieren

Diese Aktion wird verwendet, um eine temporäre Variable (auch lokale Variable genannt) zu deklarieren und ihr einen Anfangswert zu geben. Eine solche Variable existiert nur für die Dauer des Ereignisses oder script, in dem sie deklariert wurde. Temporäre Variablen sind sehr nützlich, um Werte zu speichern, die nur für einen einzigen Vorgang benötigt werden, und verhindern, dass Ihre Objekte mit Variablen "überladen" werden, die nur einmal verwendet werden können. Sie sind auch sehr nützlich, um komplexe Operationen aufzuschlüsseln, oder um Werte in anderen Instanzen zu setzen, wenn Sie den Aktionsbereich ändern. Das liegt daran, dass als temporär deklarierte Variablennichtauf eine bestimmte Instanzbeschränkt sind, sondern auf das Ereignis oder script selbst. Wenn wir von "Werten" sprechen, meinen wir nicht nur numerische Werte, denn eine Variable kann eine Zeichenkette, ein Zeiger, eine Ressourcen-ID oder etwas anderes sein, das eine Funktion zurückgeben oder verwenden kann, sogar ein Ausdruck. Weitere Informationen finden Sie im Abschnitt über Datentypen.

Beachten Sie, dass viele Aktionen ein Kontrollkästchen "Temp" haben, mit dem Sie eine neue temporäre lokale Variable erstellen können, die den Rückgabewert der Aktion enthält. Auf diese Variable kann in der Aktion script oder im Ereignis wie auf jede andere Variable zugegriffen werden Danach. Beachten Sie auch, dass temporäre lokale Variablen auf den script oder das Ereignis, in dem sie erstellt wurden, beschränkt sind und daher von anderen Instanzen im selben Block verwendet werden können, wenn Sie Apply To verwenden.

Sie können auch weitere Variablen in derselben Aktion hinzufügen, indem Sie auf das Plus-Symbol neben der Aktion klicken und eine weitere temporäre lokale Variable Name und Wert geben. Weitere Informationen über Variablen finden Sie im Abschnitt über Variablen und Variablenumfang.
