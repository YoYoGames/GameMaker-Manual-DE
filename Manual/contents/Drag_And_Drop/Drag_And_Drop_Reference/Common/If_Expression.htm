Wenn Ausdruck

Mit dieser Aktion wird geprüft, ob ein Ausdruck entweder true oder false ist. Sie können hier jeden logischen Ausdruck prüfen, solange er den einen oder den anderen Wert ergibt. Zum Beispiel kann jeder der folgenden Ausdrücke verwendet werden:

x < (room_width / 2)
a == b
++val >= 100

Wenn Sie das Kennzeichen "Not" argument setzen, werden die obigen Ausdrücke negiert, z. B. wird "==" zu "not=". Beachten Sie auch, dass der Ausdruck nicht auf numerische Werte beschränkt ist und Sie Strings, Zeiger, Ressource-IDs oder alles andere prüfen können, was eine Funktion zurückgeben oder verwenden kann (weitere Informationen finden Sie im Abschnitt über Datentypen ), aber Sie können nicht zwei verschiedene Datentypen vergleichen, ohne einen Fehler zu erhalten - wenn Sie zum Beispiel versuchen, einen String und eine ganze Zahl im Ausdruck zu verwenden, erhalten Sie einen Fehler. Weitere Informationen zu Ausdrücken finden Sie hier.

WICHTIGER HINWEIS: Aufgrund von Problemen mit der Fließkommagenauigkeit kann die Prüfung, ob zwei Werte genau gleich sind, false zurückgeben, da einer genau 1 sein kann, während der andere 1,00000000000001 sein kann. Dies kann vermieden werden, indem vor der Prüfung die Aktion Dezimal in Ganzzahl verwendet wird oder die Ausdrücke "größer als (>)" oder "kleiner als (<)" benutzt werden.

Diese Aktionen werden nun ausgeführt, wenn der "if"-Block den Wert true ergibt, während Aktionen, die an anderer Stelle ausgelassen werden, werden nach dem "if"-Block ausgeführt. Beachten Sie, dass Sie Ausdrücke in derselben Aktion "stapeln" können, indem Sie auf das Plus-Symbol neben neben der Aktion klicken und einen weiteren zu prüfenden Ausdruck angeben. Auf diese Weise wird geprüft, ob alle angegebenen Ausdrücke in einer Prüfung den Wert true ergeben, und nur wenn dies der Fall ist, wird die hinzugefügte Aktion code ausgeführt (siehe das unten stehende Beispiel).
