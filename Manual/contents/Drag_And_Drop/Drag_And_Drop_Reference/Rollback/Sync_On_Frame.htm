Synchronisierung bei Frame (Rollback)

Diese Aktion prüft, ob alle Spieler im aktuellen Frame synchronisiert sind, d. h. ihre Zustände beruhen nicht auf Vorhersagen, sondern auf den tatsächlichen Daten, die sie empfangen haben. Ist dies der Fall, gibt sie true zurück.

Wenn die Spieler im aktuellen Frame nicht synchronisiert sind, wird false zurückgegeben, und es wird versucht, alle Spieler zu synchronisieren.

Der Versuch, Spieler mit dieser Aktion zu synchronisieren, kann zu einem Einfrieren führen. Sie müssen sicherstellen, dass diese Aktion nicht ständig in einem Step-Ereignis aufgerufen wird.

Verwenden Sie sie stattdessen als Umschaltfunktion. Siehe das Beispiel unten auf der Seite.

Die Variable rollback_confirmed_frame wird automatisch aktualisiert, wenn alle Spieler synchronisiert sind.

Aktion Syntax:

Argumente:

Argument Beschreibung
Target Die Variable, in der der zurückgegebene Wert(wahr oder falsch) gespeichert wird
