Puffer Aktionsbibliothek

Ein Puffer ist ein Bereich des physischen Speichers, in dem Daten vorübergehend gespeichert werden, während sie von einem Ort zum anderen verschoben oder auf irgendeine Weise manipuliert werden, z. B. können Sie einen buffer verwenden, um Spielstatusdaten zu speichern. GameMaker verfügt über eine Reihe spezifischer Aktionen, die sich auf das Erstellen, Ändern und Laden/Speichern von buffers beziehen, aber bevor Sie diese verwenden, beachten Sie bitte die folgenden Punkte:

  • Wenn eine buffer erstellt wird, wird sie automatisch geleert und auf 0 gefüllt.
  • Die "Größe" argument der buffer Funktionen bezieht sich immer auf die Größe in Bytes der buffer.
  • GML Visual buffers ist immer 1 Byte ausgerichtet. Die "Ausrichtung" bezieht sich auf die Art und Weise, wie Daten innerhalb von buffer gespeichert werden und wird bei der Verwendung der GML buffer Funktionen verwendet, ist aber für GML Visual nicht relevant.
  • Die Daten in buffers werden sequentiell gelesen und geschrieben (d.h.: Sie rufen die Aktion " Puffer schreiben " dreimal auf, dann werden die ersten Daten geschrieben, dann geht es weiter, die zweiten Daten werden richtig geschrieben, dann geht es weiter und dann werden die letzten Daten geschrieben), wobei der so genannte "seek"-Wert angibt, wo in buffer das nächste Lesen/Schreiben stattfinden wird.
  • Alle Daten werden in die buffer geschrieben und von der buffer als String zurückgegeben. Wenn Sie also echte Zahlen gespeichert haben, müssen Sie eine der Datentyp-Aktionen verwenden, um sie von einem String in einen Wert umzuwandeln.

Der Speicher, der für die Erstellung von buffers verwendet wird, ist der Systemspeicher, so dass alle Daten, die in buffer gespeichert sind, auch dann sicher sind, wenn das Spiel nicht fokussiert ist; wenn die App jedoch geschlossen oder neu gestartet wird, gehen sie verloren. Es ist auch erwähnenswert, dass buffers auch als Dateien gespeichert und dann über die Datei-Aktionen wieder ins Spiel geladen werden können.
