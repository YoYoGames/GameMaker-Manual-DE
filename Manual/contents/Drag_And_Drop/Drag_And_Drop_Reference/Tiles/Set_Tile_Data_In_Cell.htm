Kacheldaten in Zelle setzen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle auf einer Kachelkartenebene festlegen. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Werten für Spiegeln, Drehen und Spiegeln wiedergibt, sowie alle benutzerdefinierten Kachelmasken, die Sie verwendet haben. Mit dieser Aktion geben Sie zunächst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette), wie im Raumeditor definiert, und dann die Spalte und Zeile der Kachelkartenebene angeben, für die die Kacheldaten zu setzen. Die "Zelle" ist der Bereich auf dem Kachelraster, der die Kachel enthält, für die Sie die Daten festlegen möchten. Wenn Ihre Kacheln beispielsweise 16x16 groß sind und der Raum 1024x768 groß ist, hat die Kachelkarte 64 Spalten und 48 Zeilen.

HINWEIS: Die ausgewählte Ebene muss im Raumeditor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Die letzte argument für die Aktion sind die Kacheldaten selbst. Normalerweise würden Sie die Kacheldaten für die Zelle der Kachelkarte mit der Aktion Kacheldaten in Zelle abrufen und dann mit der Aktion Kacheldaten-Transformation festlegen manipulieren, bevor Sie sie mit dieser Aktion wieder festlegen. Weitere Informationen zu Kacheldaten finden Sie unter GML im Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Layer Die Ebene mit der anzuvisierenden Kachelkarte
Columns Die Position der Zellspalte auf der horizontalen Achse, für die die Kacheldaten festgelegt werden sollen
Row Die Position der Zellreihe entlang der vertikalen Achse, um die Kacheldaten festzulegen
Data Die für die Einstellung der Zelle zu verwendenden Kacheldaten

Beispiel:

Der obige Aktionsblock code loops durchläuft jede Kachelzelle im Raum, ruft die die Kacheldaten für die Zelle, spiegelt sie und setzt die Zelle dann wieder.
