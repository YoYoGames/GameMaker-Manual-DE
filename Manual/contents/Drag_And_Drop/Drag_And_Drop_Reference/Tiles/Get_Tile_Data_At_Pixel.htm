Kacheldaten bei Pixel abrufen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle auf einer Kachelkartenebene von einer Position abrufen. Bei den Kacheldaten handelt es sich einfach um einen Wert, der den Index der Kachel zusammen mit den Werten für Spiegeln, Drehen und Spiegeln sowie etwaigen benutzerdefinierten Kachelmasken wiedergibt, die Sie verwendet haben. Bei dieser Aktion geben Sie zuerst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette), wie im Raumeditor definiert, und dann die x- und y-Achse im Raum angeben, für die Sie die Kacheldaten erhalten möchten. Der zurückgegebene Wert der Kacheldaten stimmt überein mit mit der Zelle der Kachelkarte überein, die sich an diesem Punkt befindet. Zum Beispiel, wenn Ihre Kacheln 16x16 sind, dann werden die Kacheln in Zellen von 16x16 Pixeln gespeichert. Wenn Sie also diese Aktion verwenden, um die Raumposition (8, 8) anzuvisieren, erhalten Sie die Kacheldaten für die Zelle (0, 0) der Kachelkarte.

HINWEIS: Die ausgewählte Ebene muss im Raumeditor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Der zurückgegebene Wert wird dann in der Zielvariablen gespeichert, die zuvor erstellt worden sein kann oder eine neue temporäre Variable sein kann (wenn Sie das Kontrollkästchen "Temp" aktivieren). Weitere Informationen zu Kacheldaten finden Sie unter GML im Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Layer Die Ebene mit der anzuvisierenden Kachelkarte
X Die Position auf der horizontalen Achse (X), für die die Kacheldaten festgelegt werden sollen
Y Die Position auf der vertikalen (Y-)Achse, an der die Kacheldaten gesetzt werden sollen

Beispiel:

Der obige Aktionsblock code testet auf einen Mausklick und wenn ein solcher empfangen wird ruft er die Kacheldaten für die Zelle ab, die mit der Mausposition übereinstimmt. Diese Daten werden dann geändert und an die Kachelkartenebene zurückgegeben, wobei die Kachel in dieser Zelle gedreht wird.
