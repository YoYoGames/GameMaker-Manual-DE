Kacheldaten am Pixel einstellen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle festlegen, die sich an einer bestimmten Position auf einer Kachelkartenebene befindet. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Spiegeln, Drehen und Spiegeln sowie alle benutzerdefinierten Kachelmasken, die Sie verwendet haben. Bei dieser Aktion geben Sie zunächst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette), wie im Raumeditor definiert, und dann die x- und y-Achse im Raum für die die Kacheldaten festgelegt werden sollen. Beachten Sie, dass die Kacheldaten für die "Zelle" der Kachelkarte gesetzt werden, in der die angegebene Position liegt. Wenn Ihre Kacheln zum Beispiel 16x16 groß sind, dann enthält die Kachelkartenebene Kacheln in Zellen von 16x16 Pixeln, also Wenn Sie diese Aktion für die Raumposition (8, 8) verwenden, ändern Sie die Kachel in der Zelle (0, 0) der Kachelübersicht.

HINWEIS: Die ausgewählte Ebene muss im Raumeditor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Die endgültige argument für die Aktion sind die Kacheldaten selbst. Normalerweise würden Sie die Kacheldaten für die Kachelkartenzelle mit der Aktion Kacheldaten bei Pixel abrufen und dann mit der Aktion Kacheldaten transformieren bearbeiten, bevor Sie sie mit dieser Aktion wieder einstellen. Weitere Informationen zu Kacheldaten finden Sie unter GML im Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Layer Die Ebene mit der anvisierten Kachelkarte
X Die Position auf der horizontalen Achse (X), für die die Kacheldaten festgelegt werden sollen
Y Die Position auf der vertikalen (Y-)Achse, an der die Kacheldaten gesetzt werden sollen
Data Die Kacheldaten, die zum Setzen der Zelle an der Position verwendet werden sollen

Beispiel:

Der obige Aktionsblock code testet auf einen Mausklick und wenn ein solcher empfangen wird ruft er die Kacheldaten für die Zelle ab, die mit der Mausposition übereinstimmt. Diese Daten werden dann geändert und an die Kachelkartenebene zurückgegeben, wobei die Kachel in dieser Zelle gedreht wird.
