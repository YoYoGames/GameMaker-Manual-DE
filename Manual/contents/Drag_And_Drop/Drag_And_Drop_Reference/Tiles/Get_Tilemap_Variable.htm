Tilemap-Variable abrufen

Mit dieser Aktion können Sie eine beliebige Variable aus einer bestimmten Anzahl von Variablen für das Kachelkartenelement auf einer Ebene abrufen. Wenn Sie im Raumeditor eine Kachel-Ebene erstellen, enthält diese Ebene ein Kachel-Map-Element, das dann mit Kacheln aus einem Kachel-Set gefüllt wird. Diesem Tile-Map-Element sind bestimmte Werte zugeordnet, z. B. eine Offset-Position, das verwendete Tile-Set, die Breite, die Höhe usw.. Die vollständige Liste der Werte, die Sie erhalten können, lautet:

  • X-Koordinate: Die Position entlang der X-Achse auf der Ebene, auf der das Kachelkartenelement platziert wurde.
  • Y-Koordinate: Die Position entlang der Y-Achse auf der Ebene, auf der das Kachelkartenelement platziert wurde.
  • Spalten: Die Breite (definiert als die Anzahl der Spalten von Kachelzellen) des Kachelkartenelements.
  • Zeilen: Die Höhe (definiert als die Anzahl der Reihen von Kachelzellen) des Kachelkartenelements.
  • Kachelbreite: Die Breite einer einzelnen Kachelzelle.
  • Kachelhöhe: Die Höhe einer einzelnen Kachelzelle.
  • Kachelsatz-Ressource: Die Tile-Set-Ressource, die dem Tile-Map-Element zum Zeichnen zugewiesen wurde.
  • Aktueller Rahmen: Das aktuelle Bild, das für animierte Kacheln gezeichnet wird.
  • Maske: Die Maskendaten für das Kachelkartenelement.
  • Globale Maske: Die globalen Maskendaten für alle Kachelkartenelemente.

Wenn Sie diese Aktion verwenden, geben Sie den Namen der Ebene (eine Zeichenkette, wie im Raum-Editor definiert) an, von der Sie die Daten des Kachelkartenelements abrufen möchten, und dann den Typ der Daten, die Sie abrufen möchten (wie in der Liste oben gezeigt). Der zurückgegebene Wert wird dann gespeichert in der Zielvariable gespeichert, die zuvor erstellt worden sein kann oder eine neue temporäre Variable sein kann (wenn Sie das Kontrollkästchen "Temp" aktivieren). Beachten Sie, dass Sie weitere Werte abrufen können, indem Sie auf das Plus-Symbol neben neben der Aktion klicken, eine andere Variable auswählen und eine weitere Variable zum Speichern des zurückgegebenen Wertes angeben.

Aktion Syntax:

Argumente:

Argument Beschreibung
Layer Die Ebene, von der Daten abgerufen werden
Variable Die Variable, deren Wert abzurufen ist (wie oben aufgeführt)
