Dateien Aktionsbibliothek

Die Datei-Aktionen befassen sich mit zwei verschiedenen Dateitypen - Buffer Dateien und Ini-Dateien - und verfügen auch über bestimmte allgemeine Datei-Aktionen zum Umbenennen oder Kopieren vorhandener Dateien.

Pufferdateien werden erstellt, indem die Daten aus einer zuvor erstellten buffer abgespeichert werden. Diese Daten können dann jederzeit wieder in eine buffer geladen werden. Weitere Informationen über buffers finden Sie hier: Zwischenspeicher-Aktionen.

Ini-Dateien sind kleine, leichte Dateien, die mit den meisten platforms kompatibel sind. Sie sind ideal, um kleine Informationen zu speichern, wie z.B. Interface-Einstellungen, lokale Highscores, Level-Daten usw... und sind sehr einfach zu benutzen. Ini-Dateien müssen nicht Ini-Dateien müssen nicht vorher erstellt worden sein, um diese Aktionen zu verwenden, und Sie können aus einer nicht existierenden Ini-Datei lesen und erhalten einfach einen default Rückgabewert (den Sie angeben), allerdings empfehlen wir, dass Sie zumindest eine "Basis"-Ini-Datei Ini-Datei zu erstellen, die Sie öffnen und ändern können, bevor Sie die Aktionen verwenden. Diese Basis-Ini-Datei kann erstellt werden, indem man einfach die Aktion Open Ini File aufruft, gefolgt von der Aktion Close Ini File, da das Schließen Die Datei wird beim Schließen auf die Festplatte geschrieben, oder Sie können eine Datei in die Liste der eingeschlossenen Dateien des Asset-Browsers aufnehmen. Wenn Sie eine im Asset-Browser enthaltene Datei als Basis-Ini verwenden, sollten Sie auch den Abschnitt den Abschnitt des Handbuchs über die Funktionsweise des Dateisystems lesen.

HINWEIS: Für Spiele mit Lokalisierungen ist es sehr wichtig, dass akzentuierte Buchstaben aus externen Dateien gelesen werden können. Das bedeutet, dass Sie die ini-Datei zuerst im UTF8-Format erstellen und sie dann in GameMaker als eingebundene Datei hinzufügen sollten einfügen, damit sie beim Ausführen des Spiels exportiert und anstelle der standardmäßigen Ini-Datei im ANSI-Format verwendet wird, die von den Ini-Funktionen erstellt wird, wenn vorher keine angegeben wurde. Auf diese Weise können Sie die Datei korrekt lesen und schreiben, wobei alle Akzente und nicht-römischen Buchstaben beibehalten werden.
