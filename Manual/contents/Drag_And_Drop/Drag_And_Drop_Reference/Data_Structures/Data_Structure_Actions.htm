Datenstruktur Aktionsbibliothek

In Spielen müssen Sie oft Informationen präzise und geordnet speichern. Zum Beispiel müssen Sie Listen von Gegenständen speichern, die eine Person bei sich trägt, oder Sie möchten ein Raster von Orten speichern, die noch besucht werden müssen. Sie können dafür mehrere Variablen verwenden verwenden, aber wenn Sie kompliziertere Operationen durchführen wollen, wie z. B. das Sortieren von Daten oder die Suche nach einem bestimmten Gegenstand, müssen Sie große Teile von Aktionsblöcken code schreiben, die nur langsam ausgeführt werden können, schwierig zu debuggen sind und eine echte Qual beim Schreiben darstellen!

Um hier Abhilfe zu schaffen, verfügt GameMaker über eine Reihe von eingebauten Datenstrukturen, auf die über spezielle Funktionen zugegriffen werden kann. Es gibt vier verschiedene Arten von Datenstrukturen, die über GML Visual zur Verfügung stehen (es sind sechs, wenn Sie GML verwenden), Jede hat ihre eigenen Vor- und Nachteile, je nachdem, welche Art von Informationen Sie speichern wollen und wie Sie sie später bearbeiten möchten - die vier Typen sind

Verzeichnisse Eine Liste ist eine Datenstruktur, die Informationen nacheinander speichert, während sie hinzugefügt werden. Die Listenindizes beginnen bei 0 und steigen mit jedem Element, das am Ende der Liste hinzugefügt wird, obwohl Sie auch Daten an einer anderen als der letzten Position einfügen können. Anders als bei der Datenstruktur Stack (siehe unten) können Sie jederzeit von jedem Index der Liste aus auf die Daten zugreifen.
Karten Die Map-Datenstruktur ist außerordentlich nützlich, denn mit ihr können Sie Folgendes speichern Schlüssel und Wert Paare. Zum Beispiel kann ein Charakter in Ihrem Spiel eine Anzahl von verschiedenen Gegenständen (Schlüssel) haben und für jeden einzelnen Gegenstand kann er eine Anzahl von ihnen (Werte) haben, wie in einem RPG, wo Sie 10 Gesundheitstränke, 5 Manatränke und 100 Gold haben können. Karten halten solche Paare an einem Ort zusammen, und Sie können Paare zur Karte hinzufügen und mit einigen einfachen Funktionen auch nach dem Wert suchen, der bestimmten Schlüsseln entspricht. Die Karten sind in keiner (erkennbaren) Weise sortiert, was bedeutet, dass man, um einen bestimmten Schlüssel zu finden, unter Umständen die ganze Karte durchgehen muss (was sehr langsam ist), und es gibt auch keine Möglichkeit, zwei gleiche Schlüssel zu speichern, noch kann man einem Schlüssel zwei Werte zuweisen.
Raster Ein Raster ist eine zweidimensionale Anordnung von Werten, deren Breite und Höhe Sie festlegen. Damit wird die Gesamtzahl der "Zellen" innerhalb des Gitters festgelegt (Breite und Höhe multipliziert), und diese Zellen werden dann zur Aufnahme verschiedener Datenwerte verwendet, bei denen es sich um jeden der zulässigen Datentypen handeln kann. Mit diesem Strukturtyp können Sie den Wert von Zellen im Gitter setzen und abrufen, indem Sie die x- (Spalte) und y-Position (Zeile) angeben (sowohl Zeilen als auch Spalten beginnen bei 0 und gehen bis zur Gitterbreite - 1 und der Gitterhöhe -1). Beim Zugriff auf Rasterdatenstrukturen sollten Sie immer versuchen, ganzzahlige Werte für die Zellenposition zu verwenden, und alle nicht ganzzahligen Indizes werden von GameMaker Studio 2 unterdrückt, wenn Sie dies nicht tun. Wenn das nicht der Fall ist, müssen Sie die Rundung selbst vornehmen, bevor Sie den zu prüfenden Index übergeben.
Stapel Eine Stack-Datenstruktur ist eine so genannte Last-in-First-out (LIFO)-Struktur. Man kann Werte auf einen Stapel schieben und wieder entfernen, indem man sie vom Stapel löscht. Der Wert, der zuletzt auf den Stapel geschoben wurde, ist der erste, der wieder vom Stapel entfernt wird (man denke nur an einen Münzstapel, bei dem jede Münze, die man hinzufügt, erst wieder entfernt werden muss, bevor man an die restlichen Münzen darunter gelangt). Stapel werden häufig verwendet, wenn es um Unterbrechungen geht oder um rekursive Funktionen oder sogar um eine rudimentäre KI für Ihre Spiele.

Im Grunde funktionieren alle Datenstrukturen auf die gleiche Weise: Sie erstellen eine Datenstruktur und speichern ihren Indexwert in einer Variablen. Anschließend verwenden Sie diesen Index, um die Datenstruktur in allen weiteren Funktionsaufrufen zu referenzieren, die Operationen auf ihr durchführen können. Zum Schluss, wenn Sie fertig sind, zerstören Sie die Datenstruktur wieder, um sie aus dem Speicher zu entfernen. Sie können so viele Strukturen gleichzeitig verwenden, wie Sie benötigen, und alle Strukturen können sowohl Zeichenketten als auch reale Werte speichern.

HINWEIS: Wie alle dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die das Spiel verlangsamen und schließlich zum Absturz bringen.
