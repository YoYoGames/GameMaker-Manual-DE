Stapel erstellen

Diese Aktion erstellt eine neue Stack-Datenstruktur und gibt den Indexwert zurück, so dass Sie später über die anderen Datenstruktur-Aktionen auf den Stack zugreifen können. Der Stack-Index wird an die von Ihnen angegebene Zielvariable zurückgegeben, die entweder Sie können auch das Kontrollkästchen " Temp" markieren, um eine temporäre lokale Variable zu benennen und zu erstellen, die den Wert bis zum Ende der Variable zu benennen und zu erstellen, die den Wert bis zum Ende des script oder des Ereignisses speichert. Eine neu erstellte Stack-Datenstruktur wird als "leer" betrachtet, d.h. sie enthält keine Einträge. Beachten Sie, dass Sie zusätzliche DS-Stapel erstellen können, indem Sie auf das Plus-Symbol neben neben der Aktion klicken und eine weitere Variable auswählen, die die Stack-ID enthält.

WICHTIG! Das Erstellen von Datenstrukturen verbraucht Speicher auf dem Zielsystem platform, und daher sollten alle Datenstrukturen mit der Aktion Datenstruktur freigeben freigegeben werden, wenn sie nicht mehr benötigt werden, da sonst ein Speicherleck, das die Leistung Ihres Spiels beeinträchtigen oder es sogar zum Absturz bringen kann. Dies ist besonders wichtig, wenn temporäre lokale Variablen zum Speichern von Datenstrukturindizes verwendet werden, da diese Variablen am Ende des code oder des Ereignisses entfernt werden, aber aber das bedeutet nicht, dass die Datenstruktur ebenfalls entfernt wird! Die Datenstruktur existiert weiterhin, Sie haben nur keine Möglichkeit, sie zu referenzieren. Verwenden Sie also entweder eine Instanzvariable und geben Sie die Struktur zu einem späteren Zeitpunkt frei, oder geben Sie die Struktur vor dem Ende von des Ereignisses oder der Funktion, wenn ihr Index in einer temporären Variablen gespeichert ist.
