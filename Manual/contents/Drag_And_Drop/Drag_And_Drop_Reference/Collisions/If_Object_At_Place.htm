Wenn Objekt am Ort

Diese Aktion wird verwendet, um anhand der Kollisionsmaske der aufrufenden Instanz zu prüfen, ob es eine Kollision mit einer oder mehreren Instanzen eines bestimmten Objekts gibt, wobei die aktuelle Instanz an einem Punkt im Raum platziert ist. Die Aktion wird auf true ausgewertet, wenn eine Kollision erkannt wird, oder false andernfalls. Du gibst das zu prüfende Objekt und die Position an, an der die Instanz "platziert" werden soll, um die Prüfung durchzuführen - was eine absolute Position im Raum oder eine Position relativ zur Instanz sein kann - und du kannst auch das Not-Flag ankreuzen, um zu prüfen, ob es an der angegebenen Position keine Kollision gibt; in diesem Fall wird die Aktion nur als true ausgewertet, wenn keine Kollisionen gefunden werden, andernfalls wird sie als false ausgewertet.

Nachdem Sie das Objekt und die Position angegeben haben, können Sie die Option " Liste zurückgeben" aktivieren/deaktivieren. Wenn Sie dies ankreuzen, wird die Aktion eine Datenstruktur mit allen Instanzen, die mit der Instanz kollidieren, erstellen und auffüllen. mit allen Instanzen, die mit der Instanz kollidieren, und setzt die Variable Target auf die DS-Listen-ID. Wenn die Option nicht aktiviert ist, wird die Variable Target nur dann auf einen Wert kleiner als 0 gesetzt, wenn keine Kollision erkannt wurde, oder auf den eindeutigen ID-Wert einer der Instanzen in der Kollision (beachten Sie, dass Sie bei einer Kollision mehrerer Instanzen nicht wissen können, welche Instanz-ID zurückgegeben wird, wenn Sie keine Liste verwenden).

Wenn Sie sich für die Rückgabe einer Liste entschieden haben - und die Zielvariable nicht auf Temp gesetzt ist -, können Sie auch die Option Free Target wählen. Wenn diese auf true gesetzt ist, wird die in der Zielvariablen enthaltene Liste zerstört und eine neue Liste erstellt (und an die Zielvariable zurückgegeben), wenn Sie die Aktion aufrufen. Auf diese Weise können Sie dieselbe Variable wiederverwenden, ohne sich Gedanken über Speicherlecks zu machen, denn wenn Sie die Option Free Target nicht aktivieren, sind Sie dafür verantwortlich für das Löschen der Liste verantwortlich, bevor Sie die Variable erneut in der Aktion verwenden. Beachten Sie, dass unabhängig davon, ob diese Option aktiviert ist oder nicht, die Zielvariable immer eine Listen-ID enthält, wenn Sie die Aktion auf Liste zurückgeben eingestellt haben, und als solche muss diese freigegeben werden wenn die Instanz zerstört wird oder der Raum endet (du kannst das Aufräumen Ereignis zusammen mit der Aktion Datenstruktur freigeben ). Es ist erwähnenswert, dass du die Option "Free Target" so einstellen kannst, dass eine vordefinierte Variable oder ein Ausdruck verwendet wird und du so steuern kannst, wann die Listendatenstruktur freigegeben wird, indem du die Variable an anderer Stelle setzt (oder verschiedene Ergebnisse für den Ausdruck hast) solange der Ausdruck den Wert true/false hat.

Schließlich geben Sie die Zielvariable an, die den zurückgegebenen Wert oder die Listen-ID enthalten soll. Wenn Sie die Zielvariable, die den zurückgegebenen Wert enthalten soll, als Temp-Variable (lokale Variable) kennzeichnen, erstellt die Aktion diese Variable, um den In diesem Fall hat die Option "Free Target" keine Auswirkung und Sie sind für die Zerstörung der DS-Liste verantwortlich, wenn diese Option aktiviert wurde. Wenn keine Zielvariable angegeben wird und die Option Rückgabeliste aktiviert ist, wird keine Liste erstellt.

WICHTIG! Kollisionen werden nur für die Instanzen registriert, die eine gültige Kollisionsmaske haben, d.h. sie haben eine sprite, die der sprite_index zugeordnet ist, oder eine sprite, die der mask_index zugeordnet ist. Wenn eine der Instanzen in der Kollision keine Kollisionsmaske hat, wird die Kollision nicht erkannt, unabhängig davon, was die Instanz zu diesem Zeitpunkt zeichnet.
