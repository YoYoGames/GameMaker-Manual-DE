Der Script Editor

In diesem Abschnitt geht es um den Skript-Editor (auch Text-Editor genannt), in den Sie alle code eingeben, die Ihr Spiel zum Ausführen benötigt. Dieser Editor wird für die Erstellung von benutzerdefinierten script Assets sowie für die Codierung von Objektereignissen, für das Hinzufügen von Raumerstellungscode und für viele andere Dinge verwendet.

Dieser Abschnitt beschreibt lediglich die Funktionsweise des Editors script, nicht aber die Erstellung von script oder die Verwendung von GML Code oder GML Visual zur Erstellung von scripts. Dazu lesen Sie bitte die folgenden Abschnitte:

  • GML Code Scripts
  • GML Visuelle Skripte

Beim Erstellen eines script Assets werden Sie möglicherweise aufgefordert, zwischen GML Visual und GML Code zu wählen. Weitere Informationen finden Sie unter GameMaker-Sprache.

Im Editor script schreiben Sie Ihre GameMaker Language code, die eingebaute Programmiersprache, die GameMaker verwendet (mehr dazu im Abschnitt GameMaker-Sprache ). Sobald Sie sich mit GameMaker vertraut gemacht haben und ihn in vollem Umfang nutzen wollen, ist es ratsam, diese Sprache zu erlernen, da sie Ihre Möglichkeiten beim Erstellen von Spielen erheblich erweitert.

Ein script -Asset kann umbenannt werden, indem Sie im Asset-Browser mit der rechten Maustaste auf klicken und Umbenennen wählen. Beachten Sie jedoch, dass der script -Name den Skripting-Regeln für Assets entsprechen muss, d. h. er darf nur alphanumerisch sein, muss mit einem Buchstaben beginnen und darf als einziges Symbol das "_"-Unterstrich-Symbol enthalten. Der script -Editor an anderen Stellen kann nicht umbenannt werden und hat einen Namen, der sich auf das zu bearbeitende Objekt bezieht, wie z. B. ein Objekt-Ereignis.

Registerkarte Skript / Ereignis

Der Editor script wird in einem Fenster mit Registerkarten am oberen Rand geöffnet, damit Sie mehrere scripts oder Ereignisse in einem Fenster anzeigen können (dieses Verhalten kann jedoch in den Einstellungen geändert werden, so dass für jedes script oder Ereignis ein neues Fenster angezeigt wird). Sie können auf auf eine Registerkarte klicken und sie dann ziehen, um die Reihenfolge der Registerkarten zu ändern, oder - wenn Sie es vorziehen - können Sie die Registerkarte aus dem aktuellen Fenster herausziehen und auf workspace platzieren, um ein neues Fenster für dieses script zu erstellen (oder es zu einem anderen Fenster hinzuzufügen), und Sie können auch den script Editor maximieren, um ein neues workspace zu erstellen. Wenn Sie eine script Registerkarte aus dem IDE-Fenster ziehen, wird ein neues IDE erzeugt, das diese script Ressource aufnimmt, und es kann wie das Hauptfenster verwendet werden.

Wenn Sie code von einem Objekt-Ereignis im script -Editor aus bearbeiten und den script -Editor maximiert haben oder ihn in einem separaten Fenster oder auf workspace haben, dann bietet das Rechtsklickmenü des -Editors einige zusätzliche Optionen:

  • Im Arbeitsbereich wiederherstellen: Damit wird der Editor script aus dem maximierten Zustand/Arbeitsbereich herausgenommen und wieder an das Objekt in workspace gekoppelt.
  • Gehen Sie zu Objekt: Dadurch gelangen Sie zu dem workspace, auf dem sich das Objekt mit der code befindet, und fokussieren auf das Objekt.
  • Ereignis hinzufügen/öffnen: Damit können Sie dem Objekt, zu dem das aktuelle script gehört, ein neues Ereignis hinzufügen und eine neue Registerkarte code im Script Editor für das hinzugefügte Ereignis öffnen. Wenn das ausgewählte Ereignis bereits code enthält, wird es in einem neuen in einer neuen Registerkarte geöffnet.

Informationen zu den übrigen Optionen des Rechtsklick-Menüs finden Sie im Abschnitt " Bearbeiten" weiter unten.

Wenn Sie diese Option aktivieren, werden nur die Zeichenfolgen hervorgehoben, die mit der gesamten eingegebenen Zeichenfolge übereinstimmen. Haupt Code Editor

Der Haupteditor code ist der Ort, an dem Sie alle Ihre code schreiben, um das Asset script zu erstellen oder ein Objektereignis zu füllen. Die eigentliche Bearbeitung Ihrer code wird im Abschnitt " Bearbeiten" weiter unten auf dieser Seite behandelt.

Informationen

Die Informationsleiste am unteren Rand von IDE zeigt die aktuelle Zeilennummer und die Position in der Zeile an, in der Sie sich befinden. Hier sehen Sie auch die Hilfestellung code, eine Textzeile, die die Funktion, die Sie gerade bearbeiten, zusammen mit der mit der erforderlichen arguments. Während Sie die Funktion im Editor ausfüllen, wird die arguments hervorgehoben, um Ihnen zu zeigen, welche Funktion Sie gerade bearbeiten.

Beachten Sie, dass alle optionalen arguments in Ihren script Funktionen mit [] Klammern um sie herum angezeigt werden. Wenn Sie die JSDoc Script Kommentare innerhalb eines script Assets verwendet haben, werden die Informationen, die Sie bereitgestellt haben, auch hier angezeigt.
