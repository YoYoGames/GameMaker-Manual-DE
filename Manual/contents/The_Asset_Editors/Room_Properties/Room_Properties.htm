Raum-Eigenschaften

Im Abschnitt "Raumeigenschaften" können Sie die Vererbung der Einstellungen aktivieren, Optionen wie die Dauerhaftigkeit und die Größe des Raums ändern und Ansichtsfenster verwalten.

Persistent

Wenn Sie einen Raum verlassen und später in denselben Raum zurückkehren, wird dieser normalerweise auf seine ursprünglichen Einstellungen zurückgesetzt. Das ist für die meisten Spiele in Ordnung, aber es ist vielleicht nicht das, was Sie wollen, z. B. in einem RPG oder einem nicht-linearen Spiel, wo Sie zwischen Räumen hin- und hergehen und sie so vorfinden wollen, wie Sie sie das letzte Mal verlassen haben. Wenn Sie das Kästchen " Persistent" ankreuzen, wird genau das erreicht. Der Zustand des Raums wird gespeichert und wenn Sie später dorthin zurückkehren, ist er genau so, wie Sie ihn verlassen haben, und wird nur dann auf den Anfangszustand zurückgesetzt, wenn das Spiel neu gestartet wird. Beachten Sie, dass es eine Ausnahme gibt: Wenn Sie bestimmte Objekte als persistent markiert haben, bleiben die Instanzen dieses Objekts nicht im Raum, sondern wandern in den nächsten Raum.

Anzeigepuffer löschen

Danach haben Sie die Möglichkeit, den Anzeigepuffer zu löschen. Wenn diese Option aktiviert ist, wird der Anzeigepuffer mit einer Farbe gefüllt, bevor etwas anderes für einen bestimmten Rahmen gezeichnet wird. Wenn Sie wissen, dass Ihre Ansichten immer den gesamten Bildschirm ausfüllen werden oder ein bildschirmfüllender, undurchsichtiger Hintergrund gezeichnet wird, können Sie diese Option deaktivieren, was ein erneutes Zeichnen erspart und so zur Optimierung Ihres Spiels beiträgt.

Wenn Sie jedoch mehrere Ansichten haben, die auf dem Bildschirm offene Stellen lassen, oder wenn Ihr Hintergrund transparente Teile hat, können Sie diese Option aktivieren, damit alle leeren Bereiche mit der Zeichenfarbe Ihrer Wahl gefüllt werden. Derzeit kann diese Farbe nur über code eingestellt werden. window_set_colour().

HINWEIS Wenn Sie die automatische Seitenverhältniskorrektur verwenden (wie in den Spieloptionen eingestellt), sollten Sie diese Option immer aktivieren, da es sonst zu merkwürdigen Effekten bei der "Letterbox" kommen kann, in der Ihr Spiel gezeichnet wird. Wenn Sie diese Option nicht verwenden, können Sie sie deaktivieren, um die Leistung Ihres Spiels ein wenig zu steigern (besonders auf Android und anderen mobilen Geräten platforms).

Breite, Höhe und andere Optionen

Ein Raum muss eine Größe haben, und diese wird durch die Werte definiert, die Sie für seine Breite und Höhe in Pixeln eingeben.

Sie können dann den Erstellungscode hinzufügen, wenn Sie ihn benötigen. Der Erstellungscode code wird über die Schaltfläche am unteren Rand hinzugefügt (zusammen mit dem Schalter "inherit", der angibt, ob der Raum den Erstellungscode code erben soll oder nicht). Wenn Sie darauf klicken, öffnet sich ein code oder visueller Editor. Dieser Editor erlaubt es dir, Funktionen/Aktionen und code einzugeben, die beim Start des Raumes ausgeführt werden, nach dem Erstellungs-Ereignis aller Instanzen, aber vor ihrem Raum-Start-Ereignis (für weitere Informationen über die Reihenfolge der Ereignisse, siehe hier). Dieses code wird jedes Mal ausgeführt, wenn du den Raum betrittst, es sei denn, der Raum ist als persistent gekennzeichnet. In diesem Fall wird es nur einmal ausgeführt, wenn der Raum zum ersten Mal betreten wird, aber nicht bei späteren Besuchen des Raums.

Reihenfolge der Instanzerstellung

Mit der letzten Schaltfläche in den Raumeigenschaften können Sie das Fenster Instanzerstellungsauftrag öffnen:

In diesem Fenster werden alle Instanzen des Raums in der Reihenfolge aufgelistet, in der sie erstellt werden (von oben nach unten). Wenn Sie möchten, dass eine bestimmte Instanz vor einer anderen erstellt wird, können Sie einfach auf klicken und sie an die gewünschte Position ziehen. Beachten Sie, dass die Instanzen in der Reihenfolge von oben nach unten in der Liste erstellt werden.

Kameras und Sichtfenster

Die nächste Gruppe von Eigenschaften, die Sie für den Raum definieren können, betrifft die Raumkamera und die Ansichtsfenster. Mit Hilfe von Kameraansichten können Sie verschiedene Teile Ihres Raums an verschiedenen Stellen des Bildschirms darstellen oder nur einen Teil Ihres Raums so zeichnen, dass er den gesamten Bildschirm abdeckt. In den meisten Spielen von platform folgt die Kameraansicht zum Beispiel der Hauptfigur, denn wenn man den gesamten Level auf dem Bildschirm sehen könnte, wäre die Figur zu klein, um sie zu sehen, und es gäbe keine Überraschungen für den Spieler. Kameraansichten können auch in Multiplayer- oder Koop-Spielen verwendet werden, da sie es ermöglichen, einen geteilten Bildschirm zu erstellen, in dem man in einem Teil des Bildschirms einen Spieler und in einem anderen Teil den anderen Spieler sieht. All dies lässt sich im GameMaker mit Hilfe von Kameraansichten leicht realisieren.

Oben in den Eigenschaften der Ansicht können Sie die Vererbung der Ansicht ein- oder ausschalten, dann gibt es ein Kästchen mit der Aufschrift Enable Viewports. Dies muss markiert werden, bevor eine der Kameraansichten in Ihrem Spiel verwendet werden kann.

HINWEIS Sie können die Vererbung für die drei Hauptansichtsfenstereinstellungen ein- oder ausschalten und dann die Vererbung für jede einzelne Kameraansicht ein- oder ausschalten.

Das nächste Kontrollkästchen heißt " Viewport-Hintergrund löschen" und ähnelt der oben genannten Option "Anzeigepuffer löschen", allerdings speziell für Viewports. Wenn sie aktiviert ist, wird die Anwendungsoberfläche vor dem Zeichnen jedes Rahmens mit der Fensterfarbe geleert. Dadurch wird sichergestellt, dass Sie keine unbeabsichtigten Grafiken durch transparente Bereiche in Ihrem Hintergrund sehen, wie z. B. den Inhalt früherer Frames; daher ist es wichtig, diese Option (zusammen mit "Ansichtsfenster aktivieren") zu aktivieren, wenn Ihr Hintergrund eine Transparenz enthält.

Eine Kameraansicht wird durch zwei verschiedene Gruppen von Werten definiert: die Kameraansicht selbst und der Anschluss auf dem Bildschirm, an dem diese Ansicht gezeichnet werden soll. Dies kann manchmal zu Verwirrung führen, also lassen Sie uns dies ein wenig erklären, bevor wir erläutern, wie wir die beiden Werte definieren:

  • Die Kamera: Ein Punkt innerhalb des Raums, der verwendet wird, um festzulegen, wie der Raum auf dem Bildschirm angezeigt wird (dies ist ein abstrakter Punkt im Raumeditor und seine Position wird automatisch basierend auf den Einstellungen für die Ansicht und den Ansichtsanschluss festgelegt)
  • Die Ansicht: Was die Kamera sieht, basierend auf der Position, Projektion und Drehung der Kamera
  • Der View Port: Der Bereich des physischen Bildschirms, in dem die Kameraansicht angezeigt wird.

Das bedeutet zum Beispiel, dass Sie eine Kameraansicht von 640x480 in Ihrem Raum haben können und dann den Anschluss auf 320x240 einstellen, wodurch die Ansicht auf dem Bildschirm auf diese Größe des Anschlusses skaliert wird. Sie können auch dasselbe tun und die Ansicht auf einen kleineren Wert und den Anschluss auf einen größeren Wert einstellen, wodurch das Bild auf die Größe des Anschlusses skaliert und auf dem Bildschirm größer als es ist angezeigt wird. Auf diese Weise können Sie die Größe des Bildschirms (des Anschlusses) beibehalten, während Sie die Kameraansicht ändern, um mehr oder weniger vom Raum im gleichen Bereich des Bildschirms anzuzeigen.

Die Kameraansicht wird immer als rechteckiger Bereich im Raum definiert, wobei Sie die Position der linken oberen Ecke, die Breite und die Höhe dieses Bereichs angeben. Dann müssen Sie festlegen, wo dieser Bereich im Fenster auf dem Bildschirm angezeigt wird, indem Sie den Ansichtsanschluss definieren, wobei Sie wiederum die Position der linken oberen Ecke und die Größe angeben (beachten Sie, dass alles andere als 0,0 für die linke obere Ecke zu seltsamen Ergebnissen führen kann). Sie können mehr als einen Ausschnitt haben und sie können sich überlappen, in diesem Fall werden sie in der angegebenen Reihenfolge übereinander gezeichnet.

Bitte beachten Sie, dass der gesamte Bildschirmbereich immer als rechteckiger Bereich definiert ist, so dass Ihre Ports, auch wenn sie versetzt sind, ein Rechteck bilden, wobei alle leeren Bereiche durch die Fensterfarbe des Anzeigepuffers ausgefüllt werden, was bedeutet, dass Sie unter diesen Umständen immer die Option Anzeigepuffer löschen aktiviert haben sollten, da Sie sonst seltsame Artefakte in den Bereichen zwischen den Ports erhalten. Die Kameras verfügen auch über die Option Objektverfolgung . Damit können Sie die Kamera einem bestimmten Objekt "folgen" lassen (d. h. den Blick auf ein bestimmtes Objekt gerichtet lassen). Klicken Sie dazu auf das Menüsymbol und wählen Sie ein Objekt aus der sich öffnenden Liste aus (wenn es mehrere Instanzen dieses Objekts im Raum gibt, wird nur eine von ihnen von der Kamera verfolgt).

Normalerweise bewegt sich eine Kamera nur, wenn die verfolgte Instanz zu nahe an eine "Pufferzone" herankommt, die eine unsichtbare Grenze um den Rand der Ansicht bildet. Diese Zone kann von Ihnen mit den Werten Horizontaler Rand und Vertikaler Rand definiert werden. Wenn Sie diese Werte beispielsweise auf 64 setzen, bedeutet dies, dass die Ansicht erst dann beginnt, sich zu bewegen und der Figur zu folgen, wenn sie 64 Pixel vom Rand der Ansicht entfernt ist.

Schließlich können Sie die horizontale und vertikale Geschwindigkeit angeben, mit der sich die Kamera bewegt, wenn der Charakter den Bereich buffer erreicht hat, und dies hat einen Standardwert von -1. Dieser Standardwert ist im Grunde "sofort" und bedeutet, dass in dem Moment, in dem sich die Verfolgungsinstanz außerhalb des horizontalen oder vertikalen Randes buffer befindet, die Ansicht an ihre aktuelle Position springt. Dies ist nicht immer das, was Sie wollen, und deshalb können Sie die vertikale und horizontale Scrollgeschwindigkeit für die Kamera einstellen, indem Sie die Werte auf etwas anderes als -1 setzen. Beachten Sie, dass ein Wert von 0 dazu führt, dass sich die Ansicht überhaupt nicht bewegt, und dass jeder andere positive Wert angibt, um wie viele Pixel sie sich in jedem Bild bewegt. Wenn Sie also die horizontale Geschwindigkeit auf 5 setzen, folgt die Ansicht dem Objekt mit 5 Pixeln pro Bild in horizontaler Richtung.

Raum Physik

Bevor Sie die eingebauten Physikfunktionen in Ihren Spielen verwenden können, müssen Sie GameMaker mitteilen, dass der Raum ein Physikraum ist. Dazu müssen Sie die Option " Physik aktivieren " oben im Abschnitt " Raumphysik" ankreuzen (Sie können die Vererbung dieses Abschnitts auch unabhängig von den übrigen Raumeinstellungen umschalten). Wenn Sie dies getan haben, können Sie damit fortfahren, die physikalischen Welteigenschaften einzurichten. Dies sind einige grundlegende Eigenschaften, die Sie vordefiniert haben müssen, bevor Ihr Raum physikalische Instanzen als solche funktionieren lässt. Für eine genauere Kontrolle über die Welt kannst du code verwenden (siehe die Physikfunktionen für weitere Informationen).

Als Nächstes müssen Sie die Schwerkraft der Welt einrichten. Die Stärke und Richtung der Schwerkraft wird als Vektor der x/y-Position berechnet, die Sie um einen (0, 0) Punkt herum festlegen. Also, ein x von 0 und ein y von 1 wird die Schwerkraft Richtung als nach unten mit einer Kraft von 1 Meter pro Sekunde (für eine detailliertere Erklärung siehe - The Physics World).

Schließlich müssen Sie das Verhältnis von Pixeln zu Metern einstellen, das GameMaker als Basis für alle Physikberechnungen verwenden soll. Das liegt daran, dass die Physikfunktionen mit realen Maßen arbeiten, weshalb wir diesen Wert festlegen müssen. Sie sollten diese Einstellung so lange anpassen, bis die durchschnittliche Pixelgröße der von Ihnen verwendeten Objekte ungefähr in simulierte Physikobjekte mit einer angemessenen Größe übersetzt wird.

Es sollte beachtet werden, dass die Aktivierung von Physik in einem Raum bedeutet, dass alle Instanzen in dem Raum die Physikfunktionen und -variablen verwenden müssen, um sich zu bewegen. Grundsätzlich gibt es die "traditionelle" Bewegung, bei der du die X/Y-Position einer Instanz oder ihre Geschwindigkeit und Richtung festlegen kannst, und dann gibt es die "Physik"-Bewegung, die physikalische Kräfte und Impulse erfordert, um sich zu bewegen. Diese Systeme schließen sich gegenseitig aus, und man kann eine physikalische Instanz nicht mit Nicht-Physik-Funktionen und eine nicht-physikalische Instanz nicht mit den Physik-Funktionen bewegen.
