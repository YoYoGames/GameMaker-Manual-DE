Bearbeitung des Codes

Wenn Sie den Editor script verwenden, geben Sie einfach die code ein, die Sie für das Asset script oder für das Ereignis haben möchten, und folgen dabei der korrekten GML-Syntax. Während Sie dies tun, erhalten Sie Hilfe durch das Autovervollständigungsfenster:

Dieses Fenster erscheint, während Sie Ihre code schreiben (normalerweise nach einer kurzen Pause, die aber in den Einstellungen festgelegt werden kann) und zeigt Ihnen eine Auswahl der "am besten passenden" Antworten in alphabetischer Reihenfolge an.

Wenn Sie den Editor code verwenden, können Sie auf klicken und die Maus ziehen, um eine Zeile von code auszuwählen, oder gedrückt halten und dann auf klicken und ziehen, um Spalten von code auszuwählen. Alles, was auf diese Weise ausgewählt wird, kann dann mit den Standard-Tastaturkürzeln / + " C", / + "X" und / + "V" kopiert, ausgeschnitten und eingefügt werden .

Es ist erwähnenswert, dass Sie Ihre code auf verschiedene Weise kommentieren können. Das Kommentieren Ihrer code ist sehr wichtig für die Aufrechterhaltung der Produktivität, da es die Dinge klar hält und die Abschnitte von code so trennt, dass es offensichtlich ist, was jeder Teil tut und warum er dort ist. Das bedeutet auch, dass Sie, wenn Sie in einem Team arbeiten, Notizen für andere Mitglieder hinterlassen können und dass diese genau wissen, was Sie getan haben oder wofür ein bestimmter Abschnitt von code gedacht ist. Auch als Einzelentwickler ist es wichtig, Kommentare auf code zu hinterlassen, vor allem, wenn Sie das Projekt verlassen und zu einem späteren Zeitpunkt darauf zurückkommen müssen.

Das Auskommentieren von code ist einfach, denn Sie müssen nur "//" vor jeder Textzeile verwenden oder Sie können ganze Absätze auskommentieren, indem Sie "/* text here */" verwenden. Es gibt sogar zwei Tastenkombinationen, um ausgewählte Codezeilen zu kommentieren und wieder zu entfernen: / + "K" zum Kommentieren und / + + "K" zum Dekommentieren. Das folgende Bild zeigt code, das mit diesen beiden Methoden geschrieben wurde, sowie einige JS Doc-Kommentare (die im Folgenden näher erläutert werden):

Skripte können auch mit bestimmten JSDoc-Kommentaren versehen werden, um die Eigenschaften der darin enthaltenen Funktionen zu definieren, so dass sie bei der Verwendung der Funktionen in Ihrem code in der Autovervollständigung zusammen mit ihrer arguments und anderen Details angezeigt werden. Weitere Informationen hierzu finden Sie im Abschnitt JSDoc Script Kommentare.

Standardmäßig haben alle code geschriebenen Texte Symbole zum Falten von Code neben jedem Satz passender geschweifter Klammern {}. Diese haben die Form kleiner Plus- + oder Minus-Symbole -, und wenn Sie darauf klicken, wird der code innerhalb der passenden Klammern "gefaltet", so dass er weniger sichtbaren Platz einnimmt.

Sie können auch manuell code Faltbereiche festlegen, indem Sie die speziellen #region Tags verwenden, um einen Abschnitt von code zu markieren. Das funktioniert so, dass Sie einen #region Tag (zusammen mit jeglichem Kommentartext) am oberen Ende eines Abschnitts von code hinzufügen und dann einen entsprechenden #endregion Tag am unteren Ende des Abschnitts und dann können Sie den code"falten", d.h. ausblenden, so dass Sie die Anzahl der Zeilen von code, die im Editor gleichzeitig sichtbar sind, reduzieren können:

Die Website code sieht in etwa so aus:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Wie Sie vielleicht schon bemerkt haben, sind Teile des script Textes farbig. Der script Editor "weiß" über vorhandene Objekte, eingebaute Variablen und Funktionen usw. Bescheid und färbt codes entsprechend ein. Die Farbkodierung hilft sehr dabei, Fehler zu vermeiden, da Sie sofort sehen können, ob Sie einen Namen falsch geschrieben oder ein Schlüsselwort als Variable verwendet haben, oder ob ein syntax Fehler vorliegt. Wenn Ihnen die Farbkodierung jedoch nicht gefällt, können Sie sie in den Voreinstellungen switch ein- und ausschalten (Sie können auch F10 verwenden, um sie im Editor ein- oder auszuschalten) und die verwendeten Farben anpassen.

Sie können auch im Editor mit der rechten Maustaste auf klicken, um das folgende Menü zu öffnen:

Neben den üblichen Funktionen zum Ausschneiden, Kopieren und Einfügen von Zeilen von code haben Sie auch die Möglichkeit, den markierten Text zu kommentieren oder unkommentiert zu lassen, Einzüge hinzuzufügen oder zu entfernen sowie das Fenster code in 1, 2 oder 3 getrennte Bereiche aufzuteilen (beachten Sie, dass die Optionen in Bezug auf #regions nicht verfügbar sind, wenn Sie in dem zu bearbeitenden code keine definiert haben).

Mit der letzten Option für die Aufteilung in Fenster wird das eigentliche Fenster in verschiedene Bereiche aufgeteilt, in die Sie dann ein script oder ein Ereignis ziehen können, das neben den anderen zum Vergleichen und Bearbeiten angezeigt wird.

Dies ist besonders nützlich, um zwei scripts zu vergleichen oder um miteinander verbundene scripts gleichzeitig zu betrachten, ohne ein weiteres Fenster zu benötigen oder ständig zwischen den Registerkarten switch wechseln zu müssen.

Die anderen Optionen in diesem Menü sind das Umschalten eines Haltepunkts für die Verwendung mit dem Debugger und die Konvertierung von GML code in GML Visual oder umgekehrt. Beachten Sie, dass bei der Konvertierung von code in GML Visual keine einzelnen Aktionen für jede Zeile von code erstellt werden, sondern eine einzige Aktion code verwendet wird, die alles enthält, was zuvor in dem Ereignis oder script stand.
