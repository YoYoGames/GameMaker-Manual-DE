Merkmale der Federn

Feather bietet eine intelligente code Vervollständigung und eine verbesserte syntax Überprüfung in Ihrem GML Code scripts , zusammen mit intelligenten Refactoring-Optionen.

Diese Seite listet die Funktionen auf, die Feather im GML Code Editor für Skripte und Objekt-Ereignisse bietet. Diese Funktionen sind nur verfügbar, wenn Feather enabled.

Informationen zum Schweben

Wenn Sie den Mauszeiger über eine Funktion, sprite, eine Farbe oder einen anderen speziellen Wert bewegen, werden die entsprechenden Informationen angezeigt. Nachstehend finden Sie einige Beispiele.

Asset-Refactoring

Wenn Sie ein Asset in Ihrem Asset-Browser umbenennen, aktualisiert Feather automatisch alle Erwähnungen dieses Assets in Ihrem Projekt code, damit diese Referenzen nicht verloren gehen. Dies kann in den Feather-Einstellungen aktiviert oder deaktiviert werden.

Sie können ein Asset auch über den Code Editor umbenennen. Setzen Sie den Textcursor auf die Asset-Referenz und drücken Sie CTRL/CMD + SHIFT + R.

Geben Sie einen neuen Namen ein und klicken Sie auf "Übernehmen". Das Asset selbst und alle Verweise darauf in Ihrem Projekt werden umbenannt.

Quick Fixes

Wenn Sie in Ihrem code einen Fehler oder eine Warnung erhalten, setzen Sie den Textcursor auf die gemeldete Zeile und drücken Sie STRG/CMD + Q, um das Menü "Quick Fixes" aufzurufen. Alternativ können Sie auch auf das Symbol in der Zwischenablage klicken.

Dieses Menü zeigt Ihnen eine schnelle Lösung an, wenn dies möglich ist, und ermöglicht es Ihnen, die Regel für diese Zeile zu deaktivieren oder ihren Schweregrad zu ändern.

Das obige Beispiel zeigt eine schnelle Lösung für ein Ereignis, bei dem das Oberflächenziel nach einer Änderung nicht zurückgesetzt wurde.

Wenn Sie eine Funktion erstellen, können Sie über das Menü Quick Fix automatisch JSDoc-Kommentare für die Funktion erzeugen.

Alle Referenzen finden

Mit Feather können Sie alle Verweise auf eine Variable in Ihrem Projekt finden.

Setzen Sie den Textcursor auf eine Variable, und drücken Sie F3. Am oberen Rand Ihres Arbeitsbereichs wird eine kleine Suchleiste angezeigt:

Navigieren Sie zum vorherigen oder nächsten Verweis, indem Sie auf den entsprechenden Pfeil drücken, oder schließen Sie ihn.

Diese Suche ist intelligent, d. h. sie sucht nicht einfach nach einem Textstring, sondern nach Verweisen auf die ausgewählte Variable in ihrem deklarierten Bereich. Das bedeutet, dass zwei Variablen, die denselben Namen haben, aber separat in zwei verschiedenen Objekten existieren, nicht übereinstimmen werden.

Alle Referenzen auflisten

Setzen Sie den Textcursor auf eine Variable und drücken Sie SHIFT + F3. Dadurch wird das Fenster "Suchergebnisse" geöffnet und alle Verweise auf die ausgewählte Variable werden in einer Liste angezeigt.
