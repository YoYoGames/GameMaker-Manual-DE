Feder-Datentypen

Feather verwendet Datentypen, um beim Schreiben von code eine intelligente Syntaxprüfung durchzuführen und sicherzustellen, dass Sie nicht den falschen Datentyp für eine Variable oder einen Funktionsparameter verwenden.

Außerdem können Sie mit JSDoc-Kommentaren die Datentypen für die Parameter und Rückgabewerte Ihrer eigenen Skriptfunktionen angeben.

Mit den JSDoc-Tags @param und @return können Sie einen der folgenden Typen angeben:

HINWEIS JSDoc-Typen unterscheiden nicht zwischen Groß- und Kleinschreibung, d. h. Id.DsList und id.dslist sind identisch.

HINWEIS * In der Spalte Beispiele für Spezifizierer sind einige Beispielspezifizierer aufgeführt, aber nicht alle. So kann es beispielsweise unter den Gruppen Id, Asset und Constant weitere mögliche Typen geben, die hier nicht aufgeführt sind.

Bei der Verwendung von Feather kann in IDE der Typ Any* erscheinen, was bedeutet, dass der Typ des Bezeichners (der eine Variable, ein Parameter, ein Rückgabewert usw. sein kann) von Feather im Moment nicht erkannt werden kann. Wenn mehr code für diesen Bezeichner hinzugefügt wird, kann er möglicherweise einen bestimmten Typ annehmen.

Anforderer

Nach dem Basisdatentyp wird ein Spezifizierer mit einem Punkt hinzugefügt . hinzugefügt, um den genauen Typ der Daten in dieser Gruppe anzugeben.

Typen wie Id.DsList, Asset.GMObject und Constant.Color verwenden Spezifizierer.

Arten von Sammlungen

Typen wie Array und Id.DsList, bei denen es sich um Datenstrukturen handelt, die mehrere Werte enthalten, können einen einzigen Datentyp für alle ihre Inhalte angeben.

Dies geschieht mit Collection Types, die mit spitzen Klammern <> an den Typ angehängt werden .

Ein Array mit Strings wäre zum Beispiel Array<String>, und eine DS-Liste, die ein Array mit Reals enthält, wäre Id.DsList<Array<Real>>.
