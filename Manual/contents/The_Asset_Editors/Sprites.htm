Der Sprite Editor

Im Sprite-Editor können Sie die allgemeinen Eigenschaften der von Ihnen erstellten Sprites festlegen. Sie können dieses Fenster öffnen, indem Sie ein neues sprite erstellen (klicken Sie mit der rechten Maustaste auf im Asset-Browser und wählen Sie "Sprite erstellen"), indem Sie auf auf ein bestehendes sprite doppelklicken oder indem Sie mit der rechten Maustaste auf auf ein bestehendes sprite klicken und "Eigenschaften" wählen.

Der Sprite Editor besteht aus den folgenden Teilen:

Sprite Name

Hier können Sie die sprite benennen. Der Name sprite wird in Ihrem gesamten Spielprojekt verwendet, um die sprite als Ressource zu identifizieren, und kann nur alphanumerisch sein und darf nur das Unterstrichsymbol "_" verwenden. Während Sie Zahlen im Namen verwenden können sprites Namen verwenden, aber der Name darf nicht mit einer Zahl beginnen, da dies bei der Verwendung von GML oder GML Visual einen ungültigen Ressourcennamen darstellen würde.

Bild-Eigenschaften

In diesem Bereich können Sie die Bildgröße einstellen, indem Sie auf die Schaltfläche Sprite Size klicken. Daraufhin wird das folgende Fenster geöffnet:

Hier sehen Sie eine Vorschau von sprite vor und nach der Anwendung der gewählten Transformationen, und der Abschnitt darunter ist für die Einstellung der neuen sprite Eigenschaften und ist in zwei Teile aufgeteilt:

  • Bild skalieren: Mit dieser Option können Sie die Größe der Zeichenfläche sprite ändern. Alle auf sprite enthaltenen Bilder werden gestreckt und skaliert, um der neuen Größe zu entsprechen. Sie können wählen, ob Sie skalieren und das Seitenverhältnis des Basisbildes beibehalten wollen, und Sie können auch wählen, ob Sie in Pixeln oder in Prozent skalieren möchten. Je nach Art des Bildes, mit dem Sie arbeiten, können Sie auch die lineare Interpolation aktivieren oder nicht (im Allgemeinen "glättet" die lineare Interpolation die Pixel in einem Bereich und ist daher am besten deaktiviert). einen Bereich und ist daher für Pixelgrafiken am besten nicht zu aktivieren).
  • Größe der Leinwand ändern: Mit dieser Option wird die Größe der Zeichenfläche an die neue Größe angepasst, ohne den Inhalt zu skalieren. Sie können den "Ankerpunkt" für die Größenänderung mit den Richtungspfeilen festlegen und dann den Größenwert einstellen. Es gibt eine Option zur Beibehaltung des Seitenverhältnisses bei der Größenänderung der Leinwand und unten können Sie wählen, ob Sie die Werte für die Größenänderung in Pixeln oder als Prozentsatz angeben möchten.

Wenn Sie mit der neuen Größe zufrieden sind, können Sie auf Anwenden klicken, um die Änderungen zu übernehmen und das Eigenschaftenfenster Größe ändern zu schließen. Beachten Sie, dass Sie bei der Anwendung einer Skalierungs- oder Größenänderung auf sprite die Möglichkeit haben, die Änderung auch auf den Ursprung des sprite anwenden, indem Sie das Kontrollkästchen Auf Ursprung anwenden neben der Schaltfläche Anwenden aktivieren. Wenn das Kontrollkästchen aktiviert ist - wenn der Ursprung nicht zu den Standardeinstellungen gehört und nicht in sprite gesperrt ist -, wird seine Position skaliert, um sprite Wenn das Kontrollkästchen nicht aktiviert ist, wird keine Skalierung auf den Ursprung angewendet. Wenn der Ursprung kein benutzerdefinierter Wert ist oder gesperrt ist, wird er ausgegraut und ist nicht verfügbar. Sie können diese Option für alle sprites in den Sprite-Editor-Voreinstellungen.

HINWEIS Diese Option ist nur für Bitmaps verfügbar sprites und wird bei SWF- oder Skelettanimationen ausgegraut sprites.

In diesem Bereich können Sie auch entweder den Bildeditor öffnen, um Ihre sprite zu zeichnen, oder Sie können auf die Schaltfläche "Importieren" klicken, um eine zuvor erstellte sprite zu importieren. Sprites kann aus den meisten gängigen Bitmap-Formaten ( PNG, JPG, GIF, etc...) importiert werden, aber GameMaker unterstützt auch vector sprites (SWF) und Skelettanimationen sprites ( Spine). Weitere Informationen zu diesen Formaten finden Sie im Abschnitt über Nicht-Bitmap-Sprites. Beachten Sie, dass Sie beim Importieren von Bitmap sprites mehrere Bilddateien im Dateiexplorer auswählen können, die dann jeweils als separate Teilbilder (Frames) zu demselben sprite hinzugefügt werden.

Textur-Einstellungen

In diesem Abschnitt des Eigenschaftsfensters sprite geht es darum, wie GameMaker die Bilder, aus denen Ihre sprite besteht, aufTexturseiten für die Verwendung mit Geräten und Browsern. Für den Desktop platforms ist dies vielleicht nicht so wichtig, aber wenn Sie anfangen, für mobile Geräte oder das Web zu entwickeln, wird die korrekte Verwaltung Ihrer Bild-Assets (Texturen) sehr wichtig, da schlecht verwaltete Texturen nachteilige Auswirkungen auf Ihr Spiel haben und zu Leistungsproblemen führen können.

Die Kontrollkästchen " Horizontal kacheln " und "Vertikal kacheln" sind normalerweise nicht aktiviert, da Sie in den meisten Fällen nicht kacheln wollen sprites. Unter bestimmten Umständen kann es jedoch erwünscht sein, dass sie gekachelt werden, was bedeutet, dass Das bedeutet, dass Sie diese Optionen aktivieren sollten, insbesondere wenn Sie die Ansicht oder den Raum skalieren, da die Skalierung zu Artefakten in der Grafik eines Spiels führen kann, wenn die Texturseite nicht richtig generiert wird.

Wenn Ihre sprite als Texturkarte in einem 3D-Spiel verwendet werden soll, sollten Sie das Kontrollkästchen Separate Texture Page (Separate Texture Page) aktivieren, damit die sprite eine eigene Texturseite erhält (beachten Sie, dass jedes Unterbild ebenfalls eine eigene Texturseite erhält. Seite erhält, d.h. wenn Ihre sprite 10 Bilder hat, werden 10 Texturseiten erzeugt). Dies erhöht den Texturspeicherbedarf deines Spiels und daher muss man sehr vorsichtig sein, wenn man diese Option benutzt. Beachten Sie auch, dass im Allgemeinen, wenn Ihr sprite diese Option verwendet für 3D verwendet, dann sollten die Texturen eine Potenz von 2 sein (d.h. 128x128, 256x256, 512x512 etc...).

Die nächste Option ist die Aktivierung von " Vormultipliziertes Alpha" für die Bilder von sprite. Wenn diese Option aktiviert ist, wird der Alphawert aller Unterbilder von sprite vormultipliziert. Dies ist normalerweise nur beim Zeichnen von sprites auf Oberflächen oder für einige spezielle Spezialeffekte. Bei normaler Verwendung von sprite sollten Sie keinen merklichen Unterschied zwischen dem normalen sprite und dem vormultiplizierten Bild feststellen.

Wir haben auch eine Option zur Kantenfilterung. Diese Option ist speziell für den Fall gedacht, dass Sie lineare Interpolation für Ihr Spiel aktiviert haben und Sie "Halos" sehen der Farbe um Ihre sprites herum sehen. Dies wird dadurch verursacht, dass die Hardware aus mehreren Quelltexelnauf der Texturseite auf einmal interpoliert. besonders deutlich, wenn Sie die Auflösung von Spielen erhöhen. Wenn Sie diese Funktion aktivieren, werden die Randpixel gefiltert und nehmen die Farbe des nächstgelegenen Vollalphapixels sprite an, so dass die interpolierten Pixel ohne den vorher vorhandenen Lichthof überblendet werden.

Schließlich können Sie die Texturgruppe auswählen, zu der die Ressource sprite gehören soll. Grundsätzlich ist eine Texturgruppe (die zuvor im Texturgruppen-Manager definiert wurde) etwas, das Sie einrichten können, damit alle Bildressourcen, die Sie für bestimmte Räume oder Ebenen Ihres Spiels benötigen, zusammen gespeichert werden können. So kannst du zum Beispiel alle Bilder der Ebene 1 in einer Texturgruppe haben, alle Bilder der Ebene 2 in einer anderen usw... und GameMaker wird versuchen, alle diese gruppierten Ressourcen auf der gleichen Texturseite zu platzieren, um den Wechsel der Texturseite zu reduzieren, während das Spiel auf dem gewählten Zielsystem platform läuft.

HINWEIS: Dies ist nicht immer notwendig, und die Leistungssteigerung durch diese Methode hängt davon ab, ob das Zielgerät CPU- oder GPU-gebunden ist.

Weitere Informationen über Texturseiten finden Sie hier.

Nicht-Bitmap-Bilder sprites werden keinen Texturgruppen zugewiesen und einige der oben genannten Optionen sind für sie nicht verfügbar für sie verfügbar. Einige Nicht-Bitmap-Bilder sprites haben bereits ihre eigene Texturseite (Skelettanimation sprites), während andere einfach keine Texturen benötigen, da sie aus Vektoren bestehen (SWF sprites). Kollisionsmaske

Alle sprites haben eine Kollisionsmaske, die dem Bereich der sprite entspricht, der zur Erkennung von Kollisionen verwendet wird, wenn die sprite einer Instanz zugewiesen wird. Sie können zunächst den Modus für die Kollisionsmaske festlegen, der einer der folgenden sein kann:

  • Automatisch: GameMaker berechnet die Kollisionsmaske automatisch auf der Grundlage der sprite Alphawerte (dies hängt von der unten erläuterten Toleranzeinstellung ab).
  • Vollständiges Bild: Die Maske wird so berechnet, dass ihre Grenzen mit den linken, rechten, oberen und unteren Grenzen des sprite übereinstimmen.
  • Manuell: Wenn Sie diese Option wählen, können Sie die Grenzen der Kollisionsmaske anhand der Eingabewerte für links, rechts, oben und unten festlegen.

Neben dem Kollisionsmaskenmodus gibt es auch verschiedene Arten von Kollisionsmasken, die Sie aus dem Dropdown-Menü auswählen können:

  • Rechteck: Eine einfache rechteckige Kollisionsmaske. Diese ist am schnellsten aufzulösen, wenn sie in Ihrem Projekt verwendet wird.
  • Rotated Rectangle: Dies ist eine einfache rechteckige Kollisionsmaske, die sich mit dem image_angle Wert der Instanz dreht, der sprite zugewiesen ist. Sie ist langsamer aufzulösen als die normale rechteckige Kollisionsmaske, aber schneller als eine präzise Kollisionsmaske.
  • Ellipse: Erzeugt eine elliptische Kollisionsmaske. Diese ist langsamer aufzulösen als die rechteckige Maske.
  • Diamant: Erzeugt eine "rautenförmige" Kollisionsmaske. Diese ist langsamer aufzulösen und erzeugt mehr CPU-Overhead als die rechteckigen und elliptischen Masken.
  • Präzise: Erstellt eine präzise Kollisionsmaske auf der Grundlage des Umrisses von sprite. Wenn sprite mehrere Unterbilder hat, wird die Maske aus den Kanten aller übereinander liegenden Unterbilder zusammengesetzt . Beachten Sie, dass die präzise Maske auf das Begrenzungsrechteck beschnitten wird, wenn der Modus auf Manuell eingestellt ist und Sie die Standardwerte ändern. Dies ist sehr langsam und erzeugt eine Menge CPU-Overhead.
  • Präzise (pro Bild): Hiermit wird eine präzise Kollisionsmaske auf Frame-Basis erstellt (d. h. die Maske passt sich der Kontur jedes einzelnen Frames an). Beachten Sie, dass die präzise Maske am Begrenzungsrahmen abgeschnitten wird, wenn der Modus auf Manuell eingestellt ist und Sie die Standardwerte ändern. Diese Art von Kollisionsmaske ist bei weitem die langsamste und sollte nur in sehr speziellen und seltenen Fällen verwendet werden. Fällen.

Warum ist das so? Unterhalb der Einstellung für die Kollisionsmaske befindet sich der Schieberegler für die Alphatoleranz , der die Toleranz für die Erstellung der Maske in Bezug auf die Transparenz der Pixel von sprites angibt - bei einer höheren Toleranz werden Pixel, die teilweise transparent sind, außerhalb der Maske gelassen, während bei einer niedrigeren Bei einer höheren Toleranz werden teilweise transparente Pixel nicht in die Maske aufgenommen, während bei einer niedrigeren Toleranz diese Pixel in der Maske berücksichtigt werden. Dies beeinflusst sowohl den Begrenzungsrahmen (wenn er nicht auf manuell eingestellt ist) als auch die Maske für eine präzise Kollisionsprüfung.

Schließlich können Sie die Bounding Box-Grenzen für die Maske festlegen. Der Begrenzungsrahmen ist der Name für den rechteckigen Bereich, in den die Maske sprite passt. Standardmäßig wird dies automatisch berechnet, wobei die Alpha-Toleranz berücksichtigt wird. Wenn Sie die Option "Präzise (pro Bild)" aktiviert haben, wird dies auch für jedes Teilbild separat erfolgen. Wenn Sie dies selbst einstellen, können Sie Sie können die Werte in die Felder für links, rechts oben und unten eingeben, oder Sie klicken auf im Vorschaufenster sprite und ziehen es über sprite ziehen. Beachten Sie, dass beim "Ziehen" des Begrenzungsrahmens die Maske automatisch so erstellt wird, dass sie in die Endwerte passt, und dass der Begrenzungsrahmen die Maske "abschneiden" kann.

Im Vorschaufenster können Sie die Kollisionsmaske mit Hilfe der "Griffe" in jeder Ecke anpassen. Durch Klicken und Ziehen dieser Griffe wird die Kollisionsmaske genauso angepasst wie durch die Eingabe von Werten für den Begrenzungsrahmen. Box.

Neun Scheibe

Neunfach-Slicing (auch bekannt als 9-Slicing) ist eine Technik zur Skalierung rechteckiger Bilder, bei der Details erhalten bleiben, so dass sie nach der Skalierung ihre ursprüngliche Form beibehalten. Weitere Informationen finden Sie auf der Seite über Nine Slice.

Im Bereich Neunfache des Sprite Editors können Sie das Neunfache auf einer sprite aktivieren und im Neunfache-Vorschaufenster auf der rechten Seite in Aktion sehen.

Das Neun-Scheiben-Vorschaufenster ermöglicht es Ihnen, die Größe des Bildes zu ändern und zeigt, wie das endgültige Bild aussehen wird, nachdem es mit der Neun-Scheiben-Technik skaliert wurde. Diese Vorschau sprite kann auch gedreht werden.

Der Abschnitt Neun Scheiben bietet folgende Optionen:

  • Nine Slice aktivieren: Diese Option steuert, ob die Skalierung von Nine Slice verwendet wird, wenn sprite skaliert wird. Wenn diese Option aktiviert ist, wird die Neun-Scheiben-Skalierung für sprite (und alle Objektinstanzen, die diese sprite verwenden) verwendet, die in einer Sequence oder einer Raumebene platziert sind, und für skalierte sprites, die mit draw_sprite_ext() und anderen ähnlichen Funktionen gezeichnet werden.

    HINWEIS: Nine Slicing funktioniert nicht mit Funktionen, die nur einen Teil der sprite zeichnen oder verzerren, wie draw_sprite_part() oder draw_sprite_pos().
  • Führungen: Dieser Bereich zeigt den Versatz jeder Hilfslinie auf ihrer Achse an und kann zum Ändern dieser Versätze verwendet werden. Diese Hilfslinien können auch innerhalb des Sprite-Vorschaufensters verschoben werden, indem Sie sie einfach mit der Maus ziehen.

    HINWEIS: Hilfslinien, die sich auf derselben Achse befinden, können sich nicht kreuzen, und wenn Sie eine Hilfslinie über eine andere ziehen, wird die zweite Hilfslinie verschoben.


    Der Versatz jeder Hilfslinie ist relativ zu ihrer entsprechenden Kante. Beispielsweise ist die linke Hilfslinie gegenüber der linken Kante versetzt, die rechte Kante gegenüber der rechten Kante, und so weiter.

    Die Farbe und der Alphawert jeder Hilfslinie können durch Klicken auf die farbige Schaltfläche neben der Position der Hilfslinie geändert werden.
  • Hervorheben: Diese Option wirkt sich darauf aus, wie ein ausgewählter Abschnitt im Vorschaufenster Sprite angezeigt wird. "Invertiert" hebt die sprite mit einer Überlagerung hervor, mit Ausnahme des ausgewählten Abschnitts, und "Überlagerung" hebt nur den ausgewählten Abschnitt mit einer Überlagerung hervor.

    Die Farbe und der Alphawert der Überlagerung können durch Klicken auf die farbige Schaltfläche neben dem Dropdown-Menü geändert werden.
  • Ausgewähltes Slice: Mit dieser Option kann ein Slice ausgewählt werden, wobei eines der Rand-Slices (links, oben, rechts, unten) und das mittlere Slice ausgewählt wird. Ein Slice kann auch ausgewählt werden, indem man im Sprite-Vorschaufenster mit der linken Maustaste anklickt und durch erneutes Anklicken wieder abwählt.
  • Kachelmodus: Diese Option wird in Verbindung mit der Option Ausgewähltes Slice verwendet, um zu steuern, wie das Slice angezeigt wird, wenn die sprite skaliert wird. Sie können einen der folgenden Kachelmodi auf jedes Slice anwenden:
    • Strecken: Die Scheibe wird gestreckt, um die Lücke zwischen den verbundenen Ecken zu füllen.
    • Wiederholen: Der Ausschnitt wird wiederholt oder gekachelt, um Details zu erhalten.
    • Spiegeln: Das Slice wird durch Spiegelung wiederholt.
    • Leere Wiederholung: Der Ausschnitt wird nur einmal angezeigt, so dass nach ihm ein leerer Bereich entsteht.
    • Ausblenden: Das Slice wird überhaupt nicht angezeigt.

Broadcast-Nachrichten

Diese Schaltfläche fügt der Zeitachse sprite eine Broadcast-Nachricht hinzu und öffnet einen Dialog, in dem Sie die zu sendende Nachricht hinzufügen können. Die Nachricht wird an der Stelle eingefügt, an der sich der Abspielkopf in der Frame-Ansicht befindet, und besteht aus einer einfachen Zeichenfolge, die im Ereignis "Broadcast Message " aufgegriffen werden kann, wenn es ausgelöst wird, und auf die dann reagiert wird.

Sobald die Nachricht hinzugefügt wurde, wird sie in der Rahmenansicht angezeigt, und Sie können mit der rechten Maustaste darauf klicken, um ein Menü zu öffnen, in dem Sie die Nachricht löschen oder bearbeiten können. Weitere Informationen zu Broadcast-Nachrichten finden Sie hier.

Rahmengeschwindigkeit

In diesem Bereich können Sie die Geschwindigkeit einstellen, mit der die Animationsbilder aus dem Eingabefeld auf der linken Seite wiedergegeben werden sollen. Die Bildgeschwindigkeit kann entweder auf "Bilder pro Sekunde" oder auf "Bilder pro Spielbild" eingestellt werden, wobei eine Einstellung von 30 "Frames pro Sekunde" 30 Frames der Animation in einer Sekunde zeigt, während eine Einstellung von 30 "Frames pro Spielframe" versucht, alle 30 Frames in einem einzigen Spielschritt zu zeigen, so dass "Frames pro Spielframe" Werte in der Regel auf 1 oder weniger gesetzt werden (z. B. wird bei einer Geschwindigkeit von 0,5 Bildern pro Spielbild alle zwei Spielschritte ein neues Bild angezeigt).

Es ist wichtig zu wissen, dass die Einstellungen für die Bildgeschwindigkeit nicht nur die Wiedergabe der Animation im Vorschaufenster beeinflussen, sondern auch die Art und Weise, wie die sprite in Ihren Spielen animiert wird. Die Geschwindigkeit, die Sie hier einstellen, zeigt die sprite Vorschau so an, wie sie im Spiel mit einem Bildgeschwindigkeitsmultiplikator von 1 abläuft, aber dies kann in code mit der image_speed Variable Das bedeutet, dass sprite schneller oder langsamer laufen kann, je nachdem, was programmiert ist. Wenn du zum Beispiel sprite auf 10 "Bilder pro Sekunde" einstellst und dann image_speed auf 0,5, wird das Spiel das Sprite zeichnen mit halber Geschwindigkeit - 5 Bilder pro Sekunde. Setzt man dagegen den image_speed für dieselbe sprite auf 2, wird das Sprite im Spiel doppelt so schnell gezeichnet - 20 Bilder pro Sekunde. Beachten Sie, dass die Bildgeschwindigkeit auch im Bildeditor eingestellt werden kann.

Sprite Herkunft

Die obere Leiste des sprite Editors enthält die Eigenschaften des Sprite-Ursprungs. Dies ist der Punkt auf sprite, der seiner Position im Raum entspricht, d.h. wenn du eine Instanz an einer bestimmten x/y-Position erstellst, wird der Ursprung von sprite dort platziert.

Standardmäßig ist der Ursprung die linke obere Ecke von sprite, aber es ist oft bequemer, die Mitte zu verwenden, oder Sie möchten vielleicht eine andere Position verwenden (z. B. für eine platform Spielfigur könnten Sie die untere Mitte wählen), Klicken Sie daher auf auf das Dropdown-Menü für den Ursprung und wählen Sie einen der Standardwerte. Sie können den Ursprung auch manuell festlegen, indem Sie auf im Vorschaubild sprite klicken, wodurch sich das Kreuz der Ursprungsmarkierung an den von Ihnen angeklickten Punkt bewegt, oder indem Sie verschiedene Werte für x und y in die entsprechenden Felder eingeben. Beachten Sie, dass Sie auch einen Ursprung außerhalb des Bereichs von sprite setzen können, indem Sie negative Zahlen (für links und oben) oder positive Zahlen, die größer sind als die Breite und Höhe von sprite (für rechts und unten), festlegen können, was sehr nützlich sein kann, wenn Sie mit Objekten arbeiten, die zusammengesetzte sprites zeichnen müssen.
