Die asynchronen Ereignisse

Ein asynchrones Ereignis wird ausgelöst, wenn GameMaker einen Rückruf von einer externen Quelle empfängt, z. B. aus dem Web oder von dem Gerät, auf dem das Spiel läuft. Im Grunde genommen, sagst du dem GameMaker, dass er etwas tun soll - z. B. ein Bild laden - und er wird damit beginnen, aber in der Zwischenzeit alles andere tun, was er zu tun hat. Wenn die Anforderung abgeschlossen ist, wird ein Rückruf an GameMaker Studio 2 gesendet und alle asynchronen Ereignisse, die für diese Art von Callback definiert sind, werden ausgelöst.

Bitte beachten Sie, dass die asynchronen Ereignisse für alle Instanzen, die sie haben, ausgelöst werden, ähnlich wie die Tastaturereignisse, so dass Sie z.B. einen http_get() -Aufruf in einer Instanz durchführen können, aber das asynchrone HTTP-Ereignis in einer anderen haben, um den Rückruf zu verarbeiten.

HINWEIS: Sie sollten sich bewusst sein, dass aufgrund des XSS-Schutzes in Browsern Anfragen und Versuche, Ressourcen aus anderen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse liefern, wenn Sie eines der folgenden Ereignisse verwenden.
