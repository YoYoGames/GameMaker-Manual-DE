Objekt-Ereignisse

Was sind also Objektereignisse? Im Grunde sind das diskrete Momente im Spiel loop, in denen Dinge geschehen, die Sie für sie programmiert haben. GameMaker arbeitet mit Zyklen dieser Ereignisse - von dem Moment, in dem ein Raum gestartet wird, bis zu dem Moment, in dem er beendet wird, läuft ein Spiel loop, in dem bei jedem Schritt eine Reihe von Ereignissen ausgeführt oder überprüft wird, und Sie können wählen, ob Sie code oder GML visuelle Aktionen in Ihren Objekten platzieren, die auf diese Ereignisse reagieren (ein Schritt ist ein Moment in der Spielzeit, der durch die Raumgeschwindigkeitseinstellung bestimmt wird und auch als Frame bezeichnet werden kann).

Wenn Sie jedoch ein Objekt erstellen, ist diese Liste zunächst leer und Sie müssen entscheiden, welche Ereignisse Sie benötigen und was die Instanzen des Objekts tun sollen, wenn diese Ereignisse ausgelöst werden. Um dem Objekt Ereignisse hinzuzufügen, klicken Sie auf die Schaltfläche " Ereignis hinzufügen " am unteren Rand der Ereignisliste, woraufhin das folgende Fenster erscheint:

Dies ist die Liste aller grundlegenden Ereignisse und Ereigniskategorien, auf die ein Objekt reagieren kann, und innerhalb jeder Kategorie gibt es Unterereignisse zur weiteren Verfeinerung des Verhaltens. Wenn Sie z. B. auf die Ereigniskategorie "Tastendruck" klicken, öffnet sich ein weiteres Fenster mit den Ereignissen, mit denen Sie auswählen können, auf welche Taste das Objekt reagieren soll.

Sobald Sie Ihr Ereignis ausgewählt haben, wird das Editorfenster des Ereignisses mit diesem verknüpft und öffnet sich auf der rechten Seite.

Sie können nun die GML code (oder Blöcke) bearbeiten, um Ihrem Objekt ein bestimmtes Verhalten oder eine bestimmte Reaktion auf dieses Ereignis zu geben.

Sie können mit der rechten Maustaste auf ein beliebiges Ereignis klicken, das einem Objekt hinzugefügt wurde, um die folgenden Menüoptionen zu erhalten:

Diese Optionen sind:

  • Ereignis hinzufügen - Fügen Sie ein neues Ereignis aus der Ereignisliste hinzu.
  • Ausschneiden / Kopieren / Einfügen von Ereignissen - Ausschneiden, Kopieren oder Einfügen des ausgewählten Ereignisses. Sie können auch die Standard-Tastaturkürzel verwenden: / + X, / + C, und / + V.
  • Ereignis duplizieren - Dupliziert das ausgewählte Ereignis. Dabei wird der Inhalt des gewählten Ereignisses in ein anderes, von Ihnen festgelegtes Ereignis dupliziert.
  • Ereignis ändern - Ändern Sie das Ereignis. Sie werden aufgefordert, eine andere Ereigniskategorie auszuwählen. Sobald Sie diese ausgewählt haben, wird der Inhalt des aktuellen Ereignisses auf das neue Ereignis übertragen (das ursprüngliche Ereignis wird dabei entfernt).
  • In GML Visual / Code konvertieren - Objekt-Ereignisse werden mit der Methode erstellt, die Sie beim Erstellen des Projekts gewählt haben, entweder mit code oder mit GML Visual-Aktionen. Mit dieser Option können Sie unabhängig vom Projekttyp von der einen zur anderen Methode wechseln.
  • Ereignis löschen - Löschen Sie das Ereignis (oder die Ereignisse).

Beim Entfernen von Ereignissen können Sie + verwenden, um mehrere Ereignisse auszuwählen und sie dann alle zusammen zu löschen. Diese Ereignisse fallen in zwei Kategorien:

  • diejenigen, die jeden einzelnen Spielschritt ausführen
  • diejenigen, die durch ein Spielereignis "ausgelöst" werden, z. B. wenn die Instanz den Rand des Raumes erreicht oder eine Tastatur- oder Maustaste drückt.

erstellen.

Dieses Ereignis tritt ein, wenn eine Instanz des Objekts zum ersten Mal erstellt wird, und ist das allererste, was innerhalb einer Instanz geschieht, die über den Raumeditor in den Raum gesetzt wird, wenn ein Raum betreten wird. Das bedeutet, dass dieses Ereignis der ideale Ort ist, umVariablen zu initialisieren, Zeitlinien zu starten, Pfade zu setzen etc... und alles andere zu tun, was im Allgemeinen nur einmal getan werden muss oder nur, wenn eine Instanz zum ersten Mal im Raum erstellt wird. Wenn Ihr Objekt im Objekteditor oder im Raumeditor Objekt- oder Instanzvariablen hinzugefügt hat, werden diese Variablen zuerst initialisiert und dann wird das Erstellungsereignis ausgeführt.

Denken Sie daran, dass Sie alles, was Sie im Erstellungsereignis festgelegt haben, über den Instanzerstellungscode im Raumeditor ändern können, da dieser direkt nach dem Erstellungsereignis für die Instanz ausgeführt wird und dazu verwendet werden kann, Instanzvariablen zu erstellen oder Variablen, die als Objektvariablen oder im eigentlichen Erstellungsereignis hinzugefügt wurden, außer Kraft zu setzen.

Zerstören Sie

Dieses Ereignis ist das Ereignis, das ausgeführt wird, wenn eine Instanz zerstört wird. Es wird oft übersehen, wenn man Objekten Verhaltensweisen hinzufügt, aber es kann sehr nützlich sein, z. B. durch das Erzeugen von Explosions- oder Partikeleffekten, wenn ein Feind getötet wird, oder um eine neue Instanz des Objekts in einem anderen Teil des Raums wieder auftauchen zu lassen, oder sogar um Punkte zu einem Spielstand hinzuzufügen.

Aufräumen

Dieses Ereignis wird nach jedem Ereignis aufgerufen, das eine Instanz des Objekts aus dem Raum entfernt. Es wird also ausgelöst, wenn:

  • die Instanz wird zerstört
  • der Raum endet
  • das Spiel endet

Es wurde entwickelt, um alle dynamischen Ressourcen, die Sie in Ihrem Spiel haben (wie Oberflächen, Datenstrukturen, etc...) zu "bereinigen" oder um jede Aufgabe auszuführen, die Sie einmalig benötigen, wenn die Instanz aus dem Spiel entfernt wird.

Beachten Sie, dass dieses Ereignis sofort nach dem Ereignis, das es ausgelöst hat, aufgerufen wird, aber die Instanz wird erst nach dem Ende des aktuellen Ereignisses aus dem Spiel entfernt. Wenn du zum Beispiel instance_destroy() im Step-Event aufrufst, wird das Destroy-Event aufgerufen, dann das Clean Up-Event, und dann wird der Rest des Step-Events beendet. Das bedeutet, dass alle code, die Sie nach dem Aufruf von instance_destroy() haben, immer noch ausgeführt werden und eine potenzielle Ursache für Fehler sein können, wenn Sie eine Datenstruktur oder eine andere Ressource aufgeräumt haben, die code benötigt, daher ist bei der Verwendung dieses Ereignisses Vorsicht geboten.

Alarm

Die Alarmkategorie ist in 12 Ereignisse unterteilt, eines für jeden der möglichen Alarme, die in einer Instanz eingestellt werden können. Wenn Sie also auf die Kategorie " Alarm hinzufügen" klicken, wird dieses Fenster angezeigt:

Hier wählen Sie den Alarm aus, den Sie erstellen möchten. Sobald dies geschehen ist, sehen Sie, dass er dem Ereignisfenster hinzugefügt wurde, so dass Sie ihm wie gewohnt code hinzufügen können. Aber was ist ein Alarm? Nun, es ist ein spezielles Ereignis, das nichts tut, es sei denn, der Alarm wurde zuvor eingestellt. Dann wird gewartet, bis der Alarm auf 0 heruntergezählt wurde, bevor die Aktionen oder code ausgeführt werden, die Sie hinzugefügt haben.

Sobald der Alarm den Wert 0 erreicht hat und die code ausgeführt wird, zählt er auf -1 zurück, wo er verbleibt, bis er erneut gesetzt wird (d.h. Sie können den Wert eines Alarms überprüfen, um zu sehen, ob er größer als -1 ist, was Ihnen sagt, ob er läuft oder nicht). Wenn Sie also alarm[0] im Erstellungsereignis des Objekts auf 30 setzen, bedeutet dies, dass GameMaker 30 Spielschritte herunterzählt, bevor er die Aktionen oder code ausführt, die im Ereignis alarm[0] platziert sind. Wenn Sie einen Alarm auf 0 setzen, wird der Alarm codenicht ausgeführt, da das Ereignis ausgelöst wird, aber der Alarm wird sofort auf -1 gesetzt, so dass code übersprungen wird. Wenn Sie einen Alarm benötigen, um den nächsten Schritt auszuführen, sollten Sie ihn auf 1 setzen.

Dies kann sehr nützlich sein, da man damit Dinge zu bestimmten Zeitpunkten in Bewegung setzen kann, und man kann sie sogar wiederholen lassen, da nichts dagegen spricht, einen Alarm in seinem eigenen Ereignis zu setzen. Stell dir vor, du hast ein Monster und möchtest, dass es sich alle drei Sekunden nach rechts dreht... dann würdest du in seinem Erstellungsereignis einen Alarm auf die Raumgeschwindigkeit * 3 setzen (wenn die Raumgeschwindigkeit 30 beträgt, sind das 30 Schritte pro Sekunde, also multipliziere das mit 3 und du erhältst 3 Sekunden!) und dann hättest du im Alarmereignis die code oder Aktion, um seine Richtung festzulegen, sowie die Aktion (oder code), um seinen Alarm wieder auf Raumgeschwindigkeit * 3 zu setzen. Auf diese Weise können Sie ein einfaches Spiel loops einrichten, bei dem Dinge nur in bestimmten Zeitabständen geschehen.

Es ist zu beachten, dass ein Alarm, der keine Aktionen oder code enthält, nicht herunterzählt. Aber auch wenn nur ein Kommentar und keine code oder Aktionen enthalten sind, wird der Alarm weiter heruntergezählt und kann wie gewohnt eingestellt und überprüft werden.

Schritt

GameMaker unterteilt die Zeit in Schritte , wobei die Raumgeschwindigkeit festlegt, wie viele dieser Schritte es pro Sekunde geben soll (ein Schritt kann auch als Frame bezeichnet werden). Ein einzelner Schritt ist im Grunde die loop, die ständig mit allen Ereignissen läuft, die überprüft und bei Bedarf ausgelöst werden, während das Spiel läuft, so dass Sie sich vorstellen können, dass das Schritt-Ereignis ein Ereignis ist, das bei jedem einzelnen Schritt des Spiels überprüft wird, während die Instanz existiert.

Das Schritt-Ereignis besteht eigentlich aus drei Unterereignissen, die im Folgenden beschrieben werden:

Für die meisten Dinge reicht das Standard-Schritt-Ereignis aus, aber manchmal möchte man ein bisschen mehr Kontrolle darüber haben, was code zu welchem Zeitpunkt ausführt, und dafür gibt es die Schritt-Ereignisse Begin und End . Alle drei werden bei jedem Schritt überprüft, aber ihre Reihenfolge ändert sich auch dann nicht, wenn zukünftige Updates der GameMaker-Engine andere Ereignisse ändern, was bedeutet, dass dies die einzige zuverlässige Methode ist, um sicherzustellen, dass etwas immer vor etwas anderem passiert.

Wofür kann das Schritt-Ereignis verwendet werden? Nun, es kann für Aktionen oder code verwendet werden, die kontinuierlich ausgeführt werden müssen. Wenn z. B. ein Objekt einem anderen folgen soll, können Sie hier die Bewegungsrichtung in Richtung des Objekts, dem wir folgen, anpassen, damit es sich gleichmäßig hinter uns bewegt. Seien Sie jedoch vorsichtig mit diesem Ereignis und fügen Sie nicht viele komplizierte Aktionen in das Schritt-Ereignis von Objekten ein, besonders wenn Sie planen, viele Instanzen des Objekts in Ihrem Spielraum zu haben, da dies das Spiel verlangsamen könnte. Viele Dinge können in Alarmen untergebracht oder so eingestellt werden, dass sie durch einige der anderen Ereignisse ausgelöst werden, anstatt ständig zu passieren.

Kollision

Bei der Entwicklung eines Spiels ist es natürlich sehr wichtig, dass Sie wissen, wann zwei (oder mehr) Instanzen eines Objekts kollidiert sind, und dafür gibt es das Kollisionsereignis. Dies ist ein Ereignis, das Sie in einem Objekt platzieren und dann angeben, gegen welches andere Objekt Sie auf Kollisionen prüfen sollten.

Wenn Sie die Physik nicht aktiviert haben, werden diese Kollisionen auf der Grundlage der Maske der beiden Objekte berechnet (die Maske wird in den sprite Eigenschaften definiert oder kann unabhängig in den Objekteigenschaften zugewiesen werden) und ob sie sich überschneiden oder nicht. Beachten Sie, dass wenn eine der beiden Instanzen in der Kollision keine Maske zugewiesen hat (oder die sprite Maske auf nichts gesetzt ist), selbst wenn sie etwas zeichnet, keine Kollisionen erkannt werden.

Wenn Sie Physik aktiviert haben, basiert die Kollision auf der Art der Kollisionsform(Fixture), die Sie für das Objekt in seinen Physikeigenschaften definiert haben, ebenso wie seine Reaktion auf die Kollision. Das bedeutet, dass Sie möglicherweise keine code benötigen, um die Kollision zu verarbeiten, aber dieses Ereignis muss trotzdem mindestens einen Kommentar enthalten , damit die Kollisionen erkannt werden.

Schließlich ist zu beachten, dass alle Kollisionen einmal pro Spielschritt berechnet werden, bevor das Kollisionsereignis ausgelöst wird, so dass zum Zeitpunkt der Ausführung des Kollisionsereignisses alle Kollisionen bereits berechnet und zugewiesen worden sind. Das bedeutet, dass, wenn Sie eine Instanz in diesem Ereignis erstellen und dann versuchen, eine Kollision mit ihr zu überprüfen, die Kollision bis zur nächsten Iteration des Spiels nicht entdeckt oder aufgelöst wird loop.

Tastatur, Tastatur drücken, Tastaturfreigabe

Es ist sehr wichtig, dem Spieler die Möglichkeit zu geben, die verschiedenen Aspekte Ihres Spiels zu steuern, und zu diesem Zweck bietet GameMaker Ihnen eine sehr umfassende Liste von Tastaturereignissen, die in jeder der drei Hauptkategorien der Tastatur verwendet werden können. Für die allgemeine Tastatur-Kategorie wird es kontinuierlich bei jedem Schritt ausgelöst, solange die ausgewählte Taste gedrückt ist, während die Ereignisse der Kategorie Drücken und Loslassen nur einmal ausgelöst werden, wenn die Taste anfangs gedrückt oder losgelassen wird.

Es sollte beachtet werden, dass Tastaturereignisse in allen aktiven Instanzen in einem Raum ausgelöst werden, wenn eine Taste benutzt wird, aber nur diejenigen, die ein Ereignis für diese bestimmte Taste definiert haben, werden reagieren und du kannst mehrere Tastaturereignisse in jedem Objekt erstellen und die Instanzen dieses Objekts werden auf alle von ihnen reagieren, während das Spiel läuft.

Wenn Sie einem Objekt ein Tastaturereignis hinzufügen, wird das Untermenü für Tastaturereignisse angezeigt, in dem Sie die Taste angeben können, auf die Sie prüfen wollen:

Die meisten von ihnen sind ziemlich offensichtlich, aber lassen Sie uns kurz durch die Abschnitte gehen - an der Spitze haben wir die Pfeiltasten, gefolgt von den am häufigsten verwendeten Modifikatortasten, dann den Rest der Tastatur (aufgeteilt in weitere Unterabschnitte, so dass Sie die genaue benötigte Taste wie oder erhalten können) und schließlich zwei sehr spezielle Unterereignisse, No Key und Any Key. Wie die Namen schon andeuten, handelt es sich dabei um Unterereignisse, die prüfen, ob keine Taste gedrückt wird oder ob eine beliebige Taste gedrückt wird. Bitte beachten Sie, dass die Tasten des Ziffernblocks nur dann die entsprechenden Ereignisse erzeugen, wenn die Zahlensperre aktiviert ist.

Die Ereignisse " Drücken" und " Loslassen" für die Tastatur sind fast identisch mit den regulären Tastaturereignissen, mit der Ausnahme, dass sie nicht kontinuierlich, sondern nur einmal ausgelöst werden. Wenn die Tastatur zum ersten Mal registriert, dass eine Taste gedrückt wurde, wird ein Ereignis " Tastatur gedrückt" (sowie ein reguläres Tastatur-Ereignis) erzeugt, und das erste Mal danach, wenn eine Taste nicht mehr als gedrückt erkannt wird, wird ein einzelnes Ereignis "Tastatur loslassen" ausgelöst.

Maus

Die Kategorie "Maus" ist in eine Reihe von Ereignissen unterteilt, die ausgewählt werden können, um Ihnen eine genauere Kontrolle über das Spielgeschehen zu ermöglichen. Hier können Sie genau sehen, was diese Ereignisse sind:

Die Ereignisse der linken , rechten und mittleren Taste (ob normal, gedrückt oder losgelassen) wirken alle auf die Maske der Instanz, die das Ereignis hat. Das bedeutet, dass GameMaker die Position der Maus im Raum überprüft, wenn diese Tasten betätigt werden, und zwar anhand der Kollisionsmasken der Instanzen, die ein Mausereignis haben. Wenn es eine "Kollision" mit dem Begrenzungsrahmen der Instanz gibt, wird das Ereignis ausgelöst. Stelle also sicher, dass jede Instanz mit diesen Ereignissen eine sprite mit einer gültigen Kollisionsmaske hat oder dass das Objekt eine Maske sprite in den Objekteigenschaften ausgewählt hat. Wie ihre Namen andeuten, werden diese Ereignisse entweder einmal ausgelöst, wenn die gewählte Maustaste zum ersten Mal gedrückt oder losgelassen wird, oder kontinuierlich bei jedem Schritt, solange die Taste gedrückt wird.

Die Mauseintritts- und Mausaustrittsereignisse ähneln den Schaltflächenereignissen insofern, als sie ebenfalls von der Maske der Instanz abhängen, aber dieses Mal werden sie ausgelöst, wenn die Maus zum ersten Mal die Instanz "betritt" (berührt) oder wenn die Maus die Instanz "verlässt" (nicht mehr berührt). Diese Ereignisse sind jedoch nicht kontinuierlich, sondern werden nur einmal ausgelöst, wenn die Maus das Objekt betritt oder verlässt. Sie sind daher eine ideale Methode, um z. B. Schaltflächen zu erstellen, die sich ändern müssen, wenn die Maus über sie fährt, bevor sie wieder normal werden, wenn die Maus entfernt wird.

Schließlich gibt es noch einen weiteren Abschnitt zu den Mausereignissen, der sich Global Mouse nennt. In diesem Untermenü finden Sie eine Auswahl von Ereignissen, die zur Aufzeichnung von Mausereignissen in Instanzen dienen, auch wenn sich die Maus nicht über ihnen oder in ihrer Nähe befindet. Dies sind Ereignisse, die für alle Instanzen generiert werden, und wenn für das angegebene Ereignis Aktionen oder code definiert sind, wird es ausgeführt, unabhängig von der Position der Maus im Spielraum.

Bitte beachten Sie, dass auf mobilen Geräten oder Geräten mit Touchscreen die linke Maustaste auch zur Prüfung auf einen Fingertab auf einem Touchscreen verwendet werden kann und die rechte Maustaste durch einen Doppeltipp auf den Bildschirm ausgelöst wird (dieses Verhalten kann mit code geändert werden).

Gesten

Dieses Ereignis wird ausgelöst, wenn der Benutzer den Bildschirm berührt (auf mobilen Geräten) oder klickt und die Maus bewegt (auf allen anderen platforms). Diese Ereignisse ähneln den Mausereignissen, wobei es reguläre Versionen und globale Versionen gibt. Die regulären Versionen dieser Ereignisse werden nur ausgelöst, wenn die Berührungen auf einer Instanz stattfinden, die eine sprite (oder eine Maske) hat und die Berührungen innerhalb ihres Begrenzungsrahmens stattfinden. Die globalen Versionen dieser Ereignisse werden jedoch ausgelöst, wenn der Benutzer eine beliebige Stelle auf dem Bildschirm berührt.

Die Gestenereignisse erkennen Folgendes:

  • Taps - wenn ein Nutzer klickt/berührt und schnell loslässt
  • Ziehen - wenn ein Benutzer berührt/klickt und es beibehält und dann seinen Finger/Cursor bewegt
  • Schnipsen - wenn der Benutzer eine Berührung/Klick in einer Bewegung auslöst
  • Kneifen - wenn der Benutzer zwei Finger auf dem Bildschirm hat und sie dann zusammen/ auseinander bewegt
  • Drehen - wenn der Benutzer zwei Finger auf dem Bildschirm hat und sie dann um einen Punkt dreht

Die verschiedenen Ereignisse enthalten immer eine DS-Map mit der Bezeichnung "event_data", die eine Reihe von Schlüssel/Wert-Paaren mit Daten über die Berührungs-/Klickposition und Bewegung enthält. Zeichnen Sie

Diese Ereigniskategorie ist diejenige, die bestimmt, was Sie auf dem Bildschirm sehen, wenn Sie Ihr Spiel ausführen, und ist in verschiedene diskrete Ereignisse unterteilt:

Wie Sie sehen können, gibt es in der Kategorie der Zeichnungsereignisse mehrere verschiedene Ereignistypen. Draw Begin, Draw und Draw End sind die "Standard"-Zeichnungsereignisse, die Sie wahrscheinlich am häufigsten verwenden werden. Standardmäßig wird das Hauptereignis "Zeichnen" immer für jede Instanz aufgerufen, unabhängig davon, ob sie eine sprite hat oder nicht, obwohl das Ereignis nicht ausgelöst wird, wenn du die Instanz als unsichtbar kennzeichnest (beachte dies also, wenn du Spiellogik im Draw-Ereignis eines unsichtbaren Objekts hast, da sie nicht ausgeführt wird). Das Hauptzeichen-Ereignis ist auch der Ort, an dem GameMaker die Instanz sprite standardmäßig zeichnet, wenn es weder code noch Aktionen in dem Ereignis gibt (d.h.: du hast es nicht in die Ereignisliste für das Objekt aufgenommen). Das Standardzeichnen verwendet die sprite, die mit der Instanz verknüpft ist, und zeichnet diese mit allen Transformationen, die in code gesetzt sind, oder mit den angewendeten Aktionen.

Die Standard-Zeichenereignisse werden vor den Draw-GUI-Ereignissen und zwischen den Pre-Draw- und Post-Draw-Ereignissen gezeichnet, was bedeutet, dass alles, was in diesem Ereignis gezeichnet wird, unter dem Draw-GUI-Ereignis gezeichnet wird, unabhängig von der Ebene (d.h. alles, was im Draw-GUI-Ereignis gezeichnet wird, wird immer über alles gezeichnet, was im normalen Draw-Ereignis gezeichnet wird, unabhängig von der Ebenenreihenfolge).
