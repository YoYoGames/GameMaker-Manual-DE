Das Cloud-Ereignis ist ein Ereignis, das durch den Rückruf von einer der cloud_*() Funktionen, wie cloud_synchronise(). Es erzeugt eine DS-Map, die exklusiv für dieses Ereignis ist und in der der speziellen Variable async_load gespeichert wird (siehe die einzelnen Funktionen für code Beispiele, die die Verwendung dieses Ereignis näher erläutern). Diese DS-Map hat die folgende Struktur:

  • "status": Hier steht der Status code, wobei ein negativer Wert einen Fehler bedeutet, dessen Beschreibung im "errorString" enthalten ist. Ein Wert von 0 (oder ein positiver Wert) bedeutet einen Erfolg (siehe unten für genaue Werte), und der "resultString" enthält die zurückgegebenen Daten oder eine Erfolgsmeldung.
  • "id": Die ID, die von der aufgerufenen Funktion zurückgegeben wurde. Wenn Sie eine Reihe von cloud_-Anfragen absetzen, müssen Sie wissen, auf welche Sie eine Antwort erhalten, und daher würden Sie diesen Wert mit dem Wert vergleichen, den Sie beim Senden der Anfrage gespeichert haben, um die richtige Anfrage zu finden. den Wert, den Sie beim Senden der Anfrage gespeichert haben, um die richtige Anfrage zu finden.
  • "description": Die Beschreibung der zuletzt hochgeladenen Datei.
  • "resultString": Dies enthält eine Zeichenkette, die den von der Cloud zurückgegebenen Datenblob darstellt.
  • "errorString": gibt eine Fehlerzeichenfolge für jeden Fehler zurück.

Die genaue Bedeutung des zurückgegebenen "Status"-Karteneintrags wird in der folgenden Tabelle erläutert:

Status Wert errorString / resultString Beschreibung
-1 errorString = "Nicht angemeldet bei <SERVICE>" You have not successfully logged in to the given Cloud Service
0 resultString = wiederhergestellte Daten New game data downloaded from the cloud (following a cloud_synchronise() aufrufen)
1 resultString = "BereitsSynchronisiert" No new data since you last called cloud_synchronise
2 resultString = "ConflictDeferral" A conflict was encountered, but the gamer chose to ignore it
3 resultString = "GameUploadSuccess" data from cloud_string_save() oder cloud_file_save() wurde erfolgreich in die Cloud hochgeladen
-n errorString = Beschreibung des Fehlers Any other negative number means a synchronisation failure
