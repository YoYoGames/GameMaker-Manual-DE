Speichern/Laden

Dieses Ereignis wird durch den Callback von bestimmten Funktionen ausgelöst, die mit dem Laden und Speichern von buffers in Dateien, sowie beim Laden oder Entladen von Audio aus dem Speicher. Das Ereignis selbst enthält die eingebaute async_load DS-Map, die mit den für die jeweilige Funktion erforderlichen Tasten gefüllt wird. Diese sind in den folgenden Abschnitten aufgeführt.

Puffer

Wenn Sie die Funktionen buffer_save_async() oder buffer_load_async() ein wird ein asynchrones Ereignis ausgelöst, wenn die Datenübertragung abgeschlossen ist. Dieses Ereignis füllt die Karte async_load mit den folgenden Schlüssel/Wertpaaren

  • "id": die ID der asynchronen Funktion, wie sie von der verwendeten Funktion zurückgegeben wird.
  • "status": gibt true zurück, wenn die Daten korrekt gespeichert/geladen wurden, andernfalls false.

Damit können Sie den Fortschritt des Speicherns/Ladens abfragen und eine Meldung anzeigen oder den Raum wechseln usw., wenn der Vorgang abgeschlossen ist.

Audio-Gruppen

Wenn Sie mit Audiogruppen arbeiten, können Sie diese mit den folgenden Funktionen laden und aus dem Speicher entfernen audio_group_load() und audio_group_unload(). Wenn Sie die Ladefunktion verwenden, wird dieses Ereignis ausgelöst, wenn alle Audiodateien Dateien für die Gruppe in den Speicher geladen wurden, und füllt die Map mit den folgenden Schlüssel/Wert-Paaren:

  • "type": Dies gibt die Art des Ereignisses an, das aufgerufen wird, und wird "audiogroup_load" für das Laden von Audio sein.
  • "group_id": gibt die ID der geladenen Audiogruppe zurück (wie im Audiogruppen-Editor definiert).

Wenn alle Audiodaten für eine Gruppe geladen wurden, wird dieses Ereignis ausgelöst und kann dann zum Wechseln der Räume oder zum Abspielen eines Musiktitels usw. verwendet werden.
