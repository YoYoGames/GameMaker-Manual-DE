Dialog

Das asynchrone Ereignis Dialog wird nur ausgelöst, wenn es einen Rückruf von einer der speziellen asynchronen Dialogfunktionen erhält, wie get_login_async() (bitte siehe diese Funktion für ein erweitertes code Beispiel, wie dieses Ereignis verwendet wird).

Bei diesen Ereignissen handelt es sich um solche, die eine Benutzereingabe erfordern, z. B. einen Namen, Anmeldedaten, eine Zahl oder eine Farbe usw.. Da die meisten Geräte es nicht mögen, auf loop auf eine Antwort zu warten, müssen sie asynchron sein und GameMaker wird läuft im Hintergrund weiter, während diese Funktionen einen Dialog geöffnet haben und bis sie die erforderliche Benutzereingabe erhalten, die dieses Ereignis auslöst. Das bedeutet, dass Sie, wenn der Dialog geöffnet ist, während ein Spiel läuft, Maßnahmen ergreifen sollten, um das Spiel zu pausieren Spiel zu unterbrechen, bis der Dialog-Rückruf empfangen wurde.

Auch hier wird eine DS-Map zurückgegeben, deren ID in der speziellen Variable async_load. Welche Werte in dieser Map enthalten sind, hängt von der verwendeten Funktion ab. Weitere Einzelheiten finden Sie in den einzelnen Einträgen zu jeder Funktion in diesem Handbuch.

HINWEIS Die asynchronen Dialogfunktionen sind nur zum Debuggen und Testengedacht und sollten nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.
