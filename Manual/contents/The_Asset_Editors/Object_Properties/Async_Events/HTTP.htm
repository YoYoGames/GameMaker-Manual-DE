HTTP

Das HTTP-Ereignis wird durch den Callback einer der Funktionen vonhttp_*() ausgelöst, wie z.B. http_post_string(). Es erzeugt eine DS-Map, die ausschließlich für dieses Ereignis gilt und in der speziellen Variablen async_load gespeichert wird (siehe die einzelnen Funktionen für code Beispiele, die die Verwendung dieses Ereignisses näher erläutern). Diese DS-Map hat die folgende allgemeine Struktur:

  • "id": Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von http_ Anfragen absetzen, müssen Sie wissen, auf welche Sie eine Antwort erhalten, und daher würden Sie diesen Wert mit dem Wert vergleichen, den Sie beim ursprünglichen Senden der Anfrage gespeichert haben, um die richtige zu finden.
  • "status": Gibt einen Wert kleiner 0 für einen Fehler, 0 für Erfolg und 1, wenn der Inhalt heruntergeladen wird, zurück.
  • "result": Die empfangenen Daten (nur String) oder der Pfad zu der heruntergeladenen Datei, wenn Sie die Option http_get_file(). Sie können diesen Schlüssel nur erhalten, wenn der Status 0 ist, aber das ist plattformabhängig.
  • "url": Die vollständige URL, die Sie angefordert haben.
  • "http_status": Der rohe http-Status code (falls verfügbar). Dies gibt den Standard-Webstatus code für die meisten Browser zurück, z. B. 304 für "Not Modified" oder 204 für "No Content" usw...

Die obige Abbildung zeigt, was Sie erhalten, wenn Sie die Funktion http_post_string() Funktion, aber jede der http_ Funktionen kann eine leicht abweichende Karte zurückgeben, daher lesen Sie bitte den Handbucheintrag für jede Funktion, um die genauen Daten zu erfahren, die für sie zurückgegeben werden.

HINWEIS: Da async_load eine DS-Map erzeugt, sind diese Funktionen besonders nützlich in Verbindung mit den json_encode() und json_decode() Funktionen.

Es können auch zusätzliche Daten von dieser Karte geliefert werden, wenn Sie Dateien zum Herunterladen angefordert haben. In diesem Fall hat der "Status" den Wert 1 und die DS-Karte enthält diese zusätzlichen Schlüssel:

  • "contentLength": Dies ist die Dateigröße, die Sie laut Web server erwarten sollten (kann -1 sein, wenn server diese Daten nicht zurückgibt).
  • "sizeDownloaded": Die Größe der bereits heruntergeladenen Daten.

Beachten Sie, dass das Ereignis nicht für jedes einzelne empfangene Datenpaket ausgelöst wird, sondern zu jedem Zeitpunkt während des Downloads innerhalb des Hauptspiels loop aktualisiert wird. Beachten Sie auch, dass diese Funktion derzeit nur für das reguläre Ziel Windows platforms verfügbar ist.
