System

Dieses Ereignis kann nur durch einen Rückruf von einem Ereignis auf Systemebene ausgelöst werden (z. B. die Erkennung eines Gamepads oder die automatische Anmeldung bei XBox Live) und gibt eine DS-Map zurück, die in der Variablen async_loadgespeicherte DS-Map zurück, die je nach dem Systemereignis, das den Rückruf ausgelöst hat, unterschiedliche Schlüssel/Wertpaare enthält. Diese Ereignisse sind:

Spiel-Pads

Wenn dieses Ereignis für ein Gamepad ausgelöst wird, das angeschlossen oder getrennt wird, wird eines der folgenden Schlüssel/Wert-Paare in der Karte async_load zurückgegeben:

  • "event_type" - der Typ des empfangenen Systemereignisses, der eine der folgenden Zeichenketten sein wird:
    • "gamepad discovered" - passiert, wenn das System meldet, dass ein neues Gamepad angeschlossen wurde
    • "gamepad lost" - tritt ein, wenn das System die Verbindung zu einem Gamepad verloren hat
  • "pad_index" - der Index des hinzugefügten oder entfernten Pads

Mit diesem Ereignis können Sie nun die gesamte Gamepad-Prüflogik aus dem Schritt- oder Alarmereignis in das Systemereignis verschieben und nur dann ausführen, wenn sie tatsächlich erforderlich ist.

WICHTIG! Dieses Ereignis wird NICHT ausgelöst, es sei denn, Sie haben mindestens eine gamepad_* Funktion oder eine GML Visual Aktion in Ihrem Spiel haben code. runner initialisiert das Gamepad-Subsystem nur, wenn die Funktionen im Projekt verwendet werden. Wenn sie also nicht vorhanden sind, wird das Hinzufügen/Entfernen von Gamepads das Systemereignis nicht auslösen.

Virtuelle Tastaturen

Wenn dieses Ereignis für eine virtuelle Tastatur ausgelöst wird, die geöffnet oder geschlossen wird, werden die folgenden Schlüssel/Wertpaare in der Map async_load zurückgegeben:

  • "event_type" - die Art des empfangenen Systemereignisses, d.h. "virtual keyboard status" für virtuelle Tastaturen.
  • "screen_height" - die Höhe der virtuellen Tastatur (in Pixel). Dieser Wert ist 0, wenn die Tastatur unsichtbar ist.
  • "keyboard_status" - der aktuelle Status der Tastatur, der als eine der folgenden Zeichenketten zurückgegeben wird:
    • "versteckt"
    • "versteckt"
    • "zeigend"
    • "sichtbar"

Für weitere Informationen über die virtuelle Tastatur siehe hier.

Xbox Live

Das asynchrone Systemereignis kann ausgelöst werden, wenn man die Xbox One mit dem UWP-Export ansteuert und die Option Xbox Live aktivieren in den UWP-Spieloptionen aktiviert. Wenn Sie ein GameMaker UWP-Projekt starten, in dem Xbox Live aktiviert ist, versucht das Projekt automatisch, sich stillschweigend bei Xbox Live anzumelden. Die Ergebnisse dieses Anmeldeversuchs werden als eines der folgenden Schlüssel/Wertpaare in der async_load Map zurückgegeben:

  • "event_type" - die Art des empfangenen Systemereignisses, das eine der folgenden Zeichenfolgen ist:
  • "user signed in" - die stille Benutzeranmeldung wurde erfolgreich abgeschlossen
  • "user sign in failed" - die stille Benutzeranmeldung ist fehlgeschlagen (in diesem Fall können Sie die Funktion xboxlive_show_account_picker() verwenden, um den Benutzer dazu zu bringen, ein Konto zu wählen, mit dem er sich anmeldet)
  • "user signed out" - der Benutzer hat sich abgemeldet

Andere Funktionen für Xbox Live können dieses Ereignis ebenfalls auslösen, aber die verschiedenen Rückrufe für diese Funktionen werden auf den entsprechenden Funktionsseiten detailliert beschrieben. Weitere Informationen zu den spezifischen Funktionen, die für Xbox Live verfügbar sind, finden Sie hier.

Status des Audiosystems

Dieser Ereignistyp ist speziell für die Arbeit mit dem HTML5-Ziel gedacht, da er jedes Mal ausgelöst wird, wenn sich der Status des Web Audio-Kontexts ändert. Das bedeutet, dass Sie z. B. ein Stück Hintergrundmusik in einer Schleife haben, das Sie anhalten oder stoppen und neu starten können, wenn dieses Ereignis ausgelöst wird. Dies kann durch die Suche nach dem folgenden Schlüssel/Wert-Paar in der async_load -Map überprüft werden:

  • "event_type" - die Art des empfangenen Systemereignisses, d. h. die Zeichenketteaudio_system_status", wenn das Audiosystem initialisiert wurde oder sich der Kontext geändert hat.

Wenn dieser Ereignistyp ausgelöst wird, gibt es einen zusätzlichen Schlüssel in der Karte async_load:

  • "status" - Der Status des Audiosystems, der eine der beiden folgenden Zeichenfolgen ist
  • "available" - Das Audiosystem ist initialisiert und kann Töne abspielen
  • "unavailable" - Das Audiosystem ist nicht initialisiert oder der Kontext läuft gerade nicht und kann daher keine Sounds abspielen (alle Funktionen zum Abspielen von Sounds geben -1 zurück)

Weitere Informationen zu den spezifischen Funktionen, die für Audio zur Verfügung stehen, finden Sie hier.

HINWEIS: Obwohl dieses Ereignis für die Verwendung mit dem HTML5-Ziel vorgesehen ist, wird es auch bei allen anderen platforms ausgelöst. Bei allen (außer HTML5) wird es jedoch nur einmal beim Spielstart ausgelöst, wenn die Audio-Engine zum ersten Mal initialisiert wird.
