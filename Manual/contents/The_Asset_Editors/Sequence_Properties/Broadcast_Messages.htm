Broadcast-Nachrichten

Sowohl Sequenzen als auch Sprites können an jedem Punkt ihrer Länge so genannte Broadcast-Nachrichten erzeugen. Diese Nachrichten sind einfache Strings, die zu bestimmten Frames entlang der Animationszeitleiste hinzugefügt werden. Wenn dieser Punkt in der Zeitleiste erreicht wird, wird der String an alle Instanzen gesendet, die darauf hören. Jede Instanz, die ein Broadcast Message Event (zu finden in der Event-Kategorie"Other") hat, empfängt den Message-String, wenn dieser Punkt in der Timeline erreicht wird. An diesem Punkt liegt es an dir, eine code zu haben, die ihn analysiert und entsprechend reagiert. Beachten Sie auch, dass sequences auch auf Broadcast-Nachrichten hören kann, aber dazu müssen Sie zuerst die Listener-Methode mit code einrichten (siehe den Abschnitt GML über Sequence Events And Moments für weitere Informationen).

HINWEIS Wenn Sie den image_index einer Instanz direkt auf ein Bild setzen, werden keine Broadcast-Nachrichten ausgelöst, die möglicherweise auf diesem Bild des Objekts sprite vorhanden sind.

Um eine Broadcast-Nachricht zur Zeitleiste sequence oder sprite hinzuzufügen, klicken Sie einfach auf die Schaltfläche . Daraufhin wird ein Dialogfeld geöffnet, in dem Sie die zu sendende Nachricht hinzufügen können:

Sobald die Nachricht hinzugefügt wurde, wird sie auf Dope Sheet (oder in der Rahmenansicht sprite ) angezeigt. Um sie zu bearbeiten, müssen Sie mit der rechten Maustaste auf das Nachrichtensymbol klicken, um eine Liste aller Nachrichten zu öffnen, die sich mit dem Rahmen überschneiden (es kann nur eine Nachricht auf einem Rahmen sein, aber wenn Sie mehrere Nachrichten auf aufeinanderfolgenden Rahmen platzieren, überschneiden sie sich visuell, was es schwieriger macht, die gewünschte auszuwählen, und Sie können auch eine Nachricht und einen Moment auf demselben Rahmen haben).

Wenn ein Ereignis "Broadcast Message" ausgelöst wird, wird eine spezielle DS Map erstellt und in der eingebauten Variablen event_data gespeichert. Diese Variable ID ist eine eingebaute Variable mit globalem Geltungsbereich, enthält aber nur DS-Map-Daten in dem Ereignis, das sie ausgelöst hat - in diesem Fall das Ereignis "Broadcast Message" - und gibt in allen anderen Fällen -1 zurück. Die Schlüssel für das Ereignis sind wie folgt:

Nachricht Ereignis Struktur
Variabel Typ Beschreibung
event_type Zeichenfolge Dies ist die Zeichenfolge "sequence event" für eine Nachricht, die von einem Sequence gesendet wird, und "sprite event" für eine Nachricht, die von einem Sprite gesendet wird.
message Zeichenfolge Sie enthält die empfangene Nachrichtenzeichenfolge.
element_id Ebene Element-ID Diese enthält die ID des Ebenenelements (Sequenz/Sprite/Instanz), das die Meldung ausgelöst hat. Mit Hilfe dieser ID können Sie herausfinden, welches Element die Meldung ausgelöst hat, indem Sie die Funktion layer_get_element_type() herauszufinden, welches Element die Nachricht ausgelöst hat, und die anderen relevanten Funktionen zu verwenden, um es anzusprechen oder eine Aktion wie das Abspielen eines Tons auszuführen.

Beachten Sie, dass das Broadcast-Ereignis einer Instanz mehrere Zeichenfolgen für jedes Mal, wenn eine Broadcast-Nachricht gesendet wird, empfängt. Sie müssen nicht auf alle reagieren und können verschiedene Instanzen haben, die auf verschiedene Broadcast-Strings hören. Es kann jedoch vorkommen, dass eine Instanz auf Nachrichten wartet, die zur gleichen Zeit empfangen werden. In diesem Fall wird das Ereignis mehrfach ausgelöst - einmal für jede Nachrichtenfolge.

Nachfolgend finden Sie ein kurzes Beispiel, wie dieses Ereignis überprüft werden kann:

if event_data[? "event_type"] == "sequence event" // or you can check "sprite event"
{
    switch (event_data[? "message"])
    {
        case "hit":
            audio_play_sound(snd_hit, 0, false);
        break;

        case "destroy":
            sequence_destroy(event_data[? "element_id"]);
        break;
    }
}
