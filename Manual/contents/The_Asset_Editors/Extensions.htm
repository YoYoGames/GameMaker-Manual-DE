Der Erweiterungseditor

Ein Erweiterungspaket kann verwendet werden, um eine Reihe zusätzlicher Funktionen und Konstanten zu GameMaker Language (GML) hinzuzufügen und sie leicht zwischen Projekten zu übertragen. Erweiterungsfunktionen können auch mit externen SDKs oder Quellcode verknüpft werden, der in anderen Sprachen geschrieben wurde (z. B. Java für Android, Objective-C für iOS, usw.), die zur Erweiterung der Funktionalität Ihres Spiels verwendet werden können (in der Regel für die Implementierung von APIs von Drittanbietern, z. B. für Werbung, In-App-Käufe und mehr).

So müssen Sie nicht in jedes Projekt dieselbe scripts importieren, sondern können einfach ein Erweiterungs-Asset hinzufügen, das eine Sammlung Ihrer bevorzugten Funktionen und Konstanten enthält. Solche Funktionen können auch spezifisch für ein bestimmtes platform sein, und Sie können einfach auswählen, für welche platforms eine Erweiterung verwendet werden kann.

Ein schrittweises Beispiel für die Erstellung einer Erweiterung finden Sie unter: Eine Erweiterung erstellen

Um Erweiterungen auf der Grundlage des Ziels zu deaktivieren platform, siehe: Wie kann man Erweiterungen deaktivieren?

Erweiterungen importieren

Normalerweise stammen Erweiterungen von Drittanbietern aus dem GameMaker Marketplace. Diejenigen, die Sie abonniert haben, werden dann im Fenster Marketplace Library aufgelistet und können von dort aus installiert werden. Wenn Sie Ihr Erweiterungspaket nicht von Marketplace erhalten haben, können Sie es trotzdem zu einem Projekt hinzufügen, indem Sie im Asset-Browser mit der rechten Maustaste auf klicken und " Vorhandenes hinzufügen" wählen und dann zum Speicherort der Erweiterung navigieren, um sie auszuwählen.

HINWEIS Wenn Sie ein .yymps/.yymp/.gmez Paket haben, können Sie es auf IDE ziehen, um es zu importieren, oder wählen Sie im Menü "Tools" die Option "Import Local Package".

Zielplattformen

Sobald Sie eine Erweiterung zum Asset-Browser hinzugefügt haben, sollten Sie ihr einen Namen und eine Versionsnummer geben und außerdem festlegen, in welche platforms Sie sie exportieren möchten. Indem Sie das Häkchen bei ausgewählten platforms entfernen, können Sie GameMaker anweisen, die Erweiterung nicht als Teil der ausführbaren Datei für diese platforms zu exportieren. Sie sollten auch sicherstellen, dass die Eigenschaften der Erweiterung mit der/den ausgewählten Zielplattform(n) übereinstimmen, und kompatible Exportziele für die Erweiterungsdateien basierend auf den folgenden Punkten auswählen:

  • Alle Ziele: GML Erweiterungen. Es handelt sich um Erweiterungen, die mit dem GameMaker Language (GML) erstellt wurden und mit allen Zielen platforms kompatibel sind. (Beachten Sie, dass dies der einzige Erweiterungstyp ist, den das Opera GX-Ziel derzeit unterstützt.)
  • Windows (Standard, UWP, Xbox): Sie können Windows DLL Dateien (Dynamic Link Libraries) verwenden. Bitte beachten Sie, dass alle DLLs für das Projekt 64-Bit-kompatibel sein müssen, wenn Sie die Option "x64 Windows Runtime verwenden" in den Windows-Spieleoptionen aktiviert haben. 32-Bit-DLLs werden nicht mit einer 64-Bit-Laufzeitumgebung funktionieren.
  • macOS: Das Ziel macOS kann die Standarddateien DyLib für Erweiterungen verwenden.
  • Ubuntu (Linux): Für das Ziel Ubuntu müssen Sie sicherstellen, dass Ihre Erweiterung .so Dateien verwendet.
  • HTML5: Diese platforms können JavaScript Erweiterungen verwenden.
  • Android: Diese platform erfordert eine Erweiterung, die in Java oder Android Framework SDK geschrieben wird.
  • iOS / tvOS: Erweiterungen für dieses platform sind in Objective C geschrieben und erfordern eine *.mm und eine *.h Datei, oder, wenn ein Framework verwendet wird, eine vorkompilierte native *.a Datei.
  • PlayStation: Für PlayStation sollten Ihre Erweiterungen PRX Dateien sein, wobei jedes Ziel eine andere PRX benötigt, um den Spezifikationen des Ziels (PS3, PS4 oder PS5) zu entsprechen. Bei der Arbeit an einer PSVita sollte die Erweiterung stattdessen SUPRX lauten.

HINWEIS Einige Erweiterungen müssen bestimmten Namenskonventionen für GameMaker folgen, um die platform und die CPU-Architektur, für die sie entwickelt wurden, korrekt zu identifizieren. Weitere Informationen finden Sie unter Benennungskonventionen.

Da eine Erweiterung mehrere zugehörige Dateien haben kann, ist es möglich, eine einzige Erweiterung für mehrere platforms zu erstellen. In diesem Fall müssen Sie alle kompatiblen platforms im Haupteditor für Erweiterungen angeben (d. h. das Kontrollkästchen daneben aktivieren) und dann in den individuellen Dateieigenschaften für jeden Teil der Erweiterung die platform festlegen, in die diese Dateien exportiert werden sollen.

Um das Ziel platform für eine bestimmte Erweiterungsdatei oder einen Platzhalter zu ändern, müssen Sie zunächst die Eigenschaften der Erweiterung öffnen (doppelklicken Sie auf im Asset-Browser) und dann in den Eigenschaften der Erweiterung die Ressourcendatei rechts auswählen, die Sie bearbeiten möchten. Dekomprimieren beim Erstellen

Die Option "Uncompress on build" kann für komprimierte Archivdateien verwendet werden, die für ein bestimmtes Ziel platform bestimmt sind. Wenn diese Option aktiviert ist, wird der Inhalt des Archivs in das Verzeichnis "Included Files" des erstellten Pakets extrahiert, aber erst nachdem er auf das Ziel kopiert wurde platform.

Wenn Sie dies beispielsweise für eine Datei .zip in Ihrer Erweiterung aktivieren und dann einen macOS-Build von Windows aus starten, wird das Archiv .zip unverändert von Windows nach macOS kopiert und dann während der Kompilierung auf dem Ziel platform (in diesem Zusammenhang macOS) dekomprimiert.

Aufrufen von Erweiterungsfunktionen

Sobald Sie Ihre Erweiterung eingerichtet haben, können Sie die Funktionen und Konstanten dafür genauso verwenden wie die eingebauten Funktionen und Konstanten, die in GameMaker Language verwendet werden. Wenn Ihr Spiel plattformübergreifend ist (z. B. für Windows und Android ), sollten Sie die Betriebssystemkonstanten und -funktionen verwenden, um die aktuelle platform zu prüfen, bevor Sie erweiterungsspezifische Funktionen aufrufen, um z. B. Probleme zu vermeiden:

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

Beachten Sie, dass Ihre Erweiterungen für verschiedene platforms die gleichen Funktionsnamen haben können. Wenn Sie also eine Erweiterung mit verschiedenen Dateien haben, um (zum Beispiel) eine Nachricht auf Windows und für Android anzuzeigen, können beide Erweiterungsdateien die gleichen Funktionsnamen haben, solange sie beide so eingestellt sind, dass sie nur in ihre jeweiligen platforms exportiert werden.

Offizielle Erweiterungen von Drittanbietern

Ältere Versionen von GameMaker hatten viele Funktionen von Drittanbietern integriert, aber das war ineffizient und schwierig zu warten, da bei einer Änderung einer bestimmten Funktion durch einen Drittanbieter die gesamte IDE und Runtime von GameMaker aktualisiert werden musste. Daher werden in den aktuellen Versionen Erweiterungspakete verwendet, um die Unterstützung von Drittanbietern zu Ihren Spielen hinzuzufügen. Dies bedeutet, dass Sie nur die zusätzlichen Funktionen hinzufügen können, die Sie benötigen, und dass Sie diese sogar an Ihre speziellen Bedürfnisse anpassen und bearbeiten können.

Die meisten dieser Erweiterungen werden für Sie heruntergeladen und installiert, wenn Sie eine bestimmte Spieloption auswählen (z. B. Facebook oder Google Mobile Ads). Weitere Informationen dazu finden Sie im entsprechenden Abschnitt für jede platform in der GameMaker Knowledge Base:

  • GameMaker Platforms und Einrichtung

Die Erweiterungen selbst finden Sie auf der folgenden Seite Marketplace:

  • Offizielle Marketplace Vermögenswerte

Beachten Sie, dass diese Erweiterungen mit einer Demo und einer Hilfedatei (im Format *.pdf ) geliefert werden, in denen ihre Verwendung erklärt wird, und dass ihre Funktionen in diesem Handbuch nicht behandelt werden.
