Android-Erweiterungen

Um eine Erweiterung für Android zu erstellen, müssen Sie in zwei Teilen vorgehen. Der erste Teil ist das Hinzufügen der Erweiterung selbst, zusammen mit den erforderlichen Dateien, und der zweite Teil ist die Erstellung der Funktionen und Makros/Konstanten, die die Erweiterung benötigt.

Die Funktionen und Konstanten werden mithilfe von Platzhalterdateien hinzugefügt, um sie zu gruppieren. Sie fügen also einen Platzhalter hinzu und definieren dann die Funktionen und Makros wie im vorherigen Abschnitt beschrieben. Um den Rest der Dateien hinzuzufügen, müssen Sie jedoch zunächst das Kontrollkästchen Android im Abschnitt Extra Platforms des Editors aktivieren, wodurch sich die Eigenschaften der Erweiterung Android öffnen:

Hier können Sie die folgenden Angaben machen:

  • Klassenname: Ihre Erweiterung kann mehrere Klassen haben, wobei jede Klasse ihre eigenen Funktionen und Konstanten hat, daher sollten Sie ihr einen Namen geben, der ihren Zweck widerspiegelt.
  • Android-Berechtigungen: Hier können Sie alle zusätzlichen Berechtigungen hinzufügen, die Ihre Erweiterung benötigt. Welche Berechtigungen dies sind, hängt ganz von der Verwendung der Erweiterung ab. Daher sollten Sie die von Google bereitgestellte Dokumentation für die Android platform oder, wenn Sie ein SDK eines Drittanbieters verwenden, die mit dem SDK gelieferte Dokumentation prüfen. Um eine neue Berechtigung hinzuzufügen, müssen Sie auf die Schaltfläche klicken, um eine Platzhalterberechtigung hinzuzufügen, und dann einen Doppelklick auf ausführen, um sie auf die gewünschte Weise zu bearbeiten. Sie können Berechtigungen über die Schaltfläche entfernen.
  • Code-Injektion: Hier können Sie alle code hinzufügen, die in Ihre Android Anwendung injiziert (hinzugefügt) werden sollen, wenn Ihr Spiel zum Testen oder zur endgültigen Veröffentlichung erstellt wird. Stellen Sie sicher, dass Sie dies (und Ihre Berechtigungen) sorgfältig überarbeiten, bevor Sie Spiele in den Store einreichen, da falsche Einstellungen dazu führen, dass Ihr Spiel nicht eingereicht werden kann. Weitere Informationen finden Sie im folgenden Abschnitt.

Code-Injektion

Alle code, die in das Fenster "Code Injection" eingegeben werden, müssen in verschiedene XML-Gruppen aufgeteilt werden, wie im obigen Screenshot gezeigt. Das XML-Tag, das Sie für eine Gruppe verwenden, bestimmt, wo die code in Ihre Anwendung injiziert wird.

Sie können die folgenden syntax verwenden, um code Injektionsgruppen zu erstellen:

<YYTagName>

// Code to inject

</YYTagName>

Die folgenden Tags sind für die Plattform Android verfügbar:

YYAndroidTopLevelGradle
YYAndroidTopLevelGradleBuildscript
YYAndroidTopLevelGradleBuildscriptRepositories
YYAndroidTopLevelGradleBuildscriptDependencies
YYAndroidTopLevelGradleAllprojects
YYAndroidTopLevelGradleAllprojectsRepositories
YYAndroidTopLevelGradleEnd

YYAndroidGradle
YYAndroidGradleEnd
YYAndroidGradleAndroid
YYAndroidGradleDependencies 
YYAndroidGradleProperties
YYAndroidBuildTypesRelease
YYAndroidManifestAttributes
YYAndroidManifestApplicationAttributes
YYAndroidManifestActivityAttributes
YYAndroidManifestManifestInject
YYAndroidManifestActivityInject
YYAndroidManifestApplicationInject
YYAndroidStringValuesInjection
YYAndroidLayout
YYAndroidProguard

Die runtime Stelle, an der die code injiziert wird, hängt von der Art des Tags ab:

Tag Typ Injektionspfad
Oberste Ebene Gradle {build.gradle
Modulebene Gradle {Projektdateien bauen.gradle
Android-Manifest {Androiden-Projektdateien Androiden-Manifest.xml
Streicher {Projektdateien\src\main\werte\strings.xml
Layout {Projektdateien\src\main\layout\main.xml
ProGuard {RUNTIME}\android\runner\ProjectFiles\proguard-rules.pro

HINWEIS Diese Pfade gelten nur für VM; für YYC wird code in das Verzeichnis {RUNTIME}\yyc\ injiziert, wo die Pfade mit denen für VM übereinstimmen können, aber nicht müssen.

Quelldateien

Sobald Sie dies korrekt eingerichtet haben, müssen Sie die Dateien hinzufügen, die für die Funktion Ihres Erweiterungspakets erforderlich sind. Klicken Sie dazu auf die Schaltflächen am unteren Rand, entweder Add SDK oder Add Source, und suchen Sie dann die Dateien, die Sie hinzufügen möchten. Die hinzugefügten Dateien werden zusammen mit Ihrer Erweiterung im Verzeichnis AndroidSource gespeichert. Sie können dieses Verzeichnis jederzeit öffnen, indem Sie mit der rechten Maustaste auf auf die Erweiterung klicken und Im Explorer öffnen wählen.

Informationen über die Erstellung nativer Erweiterungen für Android finden Sie unter Quellcode-Beispiele.
