gc_target_frame_time

Mit dieser Funktion können Sie dem Garbage Collector mitteilen, dass er nicht mehr als die angegebene Zeit für die Ausführung jedes Frames benötigen soll. Die Funktion nimmt einen Zeitwert an , der in Mikrosekunden angegeben wird (wobei 1.000.000 Mikrosekunden einer Sekunde entsprechen), und die standardmäßige Ziel-Frame-Zeit ist 100 Mikrosekunden.

Bitte beachten Sie, dass es sich hierbei lediglich um einen Zielwert ist, da einige Vorgänge noch vollständig in einem Frame ausgeführt werden müssen und manchmal länger als die Zielzeit dauern können. Beachten Sie auch, dass eineErhöhung der Zielzeit den Garbage Collector schneller auf schnelle Änderungen in der Speichernutzung reagieren lässt und dazu führt, dass der Speicher schneller freigegeben wird, obwohl dies in der Praxis in den meisten Fällen nicht erforderlich sein dürfte. Beachten Sie auch, dass das Setzen der Ziel-Frame-Zeit auf 0 nicht dazu führt, dass der Collector vollständig gestoppt wird - er wird weiterhin eine minimale Menge an Arbeit pro Frame verrichten. Um den Collector vollständig zu deaktivieren, verwenden Sie die Funktion gc_enable() .

Syntax:

gc_target_frame_time(time);

ArgumentTyp Beschreibung
time The target time - in microseconds -  that the garbage collector should work each frame

Rückgabe:

N/A

Beispiel:

if (gc_get_target_frame_time() != 50)
{
    gc_target_frame_time(50);
}

Die obige code prüft die aktuelle Rahmenzeitvorgabe für den Garbage Collector, und wenn sie nicht 50 Mikrosekunden beträgt, wird sie auf diesen Wert gesetzt.
