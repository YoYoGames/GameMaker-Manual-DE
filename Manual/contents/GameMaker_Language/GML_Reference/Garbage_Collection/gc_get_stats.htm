gc_get_stats

Mit dieser Funktion können Sie Informationen über den aktuellen Zustand des Garbage Collectors abrufen. Die Funktion gibt eine Struktur zurück, die die folgenden Mitgliedsvariablen enthält (beachten Sie, dass sich "Objekte" hier auf alles bezieht, was vom Garbage Collector erfasst werden kann, und nicht auf allgemeine Objektinstanzen, wie sie im Asset Browser definiert sind):

GC-Statistiken Struktur
Variabel Typ Beschreibung
objects_touched Real Dies ist die Anzahl der aktiven Objekte, die der Garbage Collector im vorherigen Frame gefunden hat. Dies hängt davon ab, welche Generation gesammelt wurde.
objects_collected Real Die Anzahl der Objekte, bei denen der Garbage Collector festgestellt hat, dass sie im vorherigen Frame nicht aktiv waren und daher gelöscht werden konnten.
traversal_time Real Dies ist die Zeit in Mikrosekunden (auf dem Hauptthread), die der Garbage Collector brauchte, um herauszufinden, welche Objekte aktiv waren.
collection_time Real Dies ist die Zeit in Mikrosekunden (auf einem separaten Thread), die der Garbage Collector benötigt hat, um die als inaktiv eingestuften Objekte aufzuräumen.
gc_frame Real Dies ist ein Zähler, der jedes Mal erhöht wird, wenn ein Garbage Collection-Durchgang stattfindet. Wenn die Garbage Collection deaktiviert ist, wird er nicht erhöht.
generation_collected Real Dies ist der Index der Generation, die zuletzt erfasst wurde. 0 ist die jüngste Generation und 3 ist die derzeit älteste.
num_generations Real Dies ist die Gesamtzahl der Garbage Collection-Generationen.
num_objects_in_generation Array von Reals Dies ist ein Array (der Größe num_generations), das die Anzahl der Objekte in jeder Generation enthält.

HINWEIS: Auf dem HTML5-Ziel platform wird die Garbage Collection von der JavaScript-Engine durchgeführt, daher geben die Member-Variablen in der obigen Struktur alle 0 zurück, wenn diese Funktion auf platform verwendet wird.

Wenn Sie diese Funktion verwenden, beachten Sie bitte, dass die für die Objekte angezeigten Informationen erst dann aktualisiert werden, wenn die Verarbeitung einer vollständigen Generation abgeschlossen ist, was je nach Einstellung der Frame-Zeit mehrere Frames dauern kann (siehe hier für weitere Informationen zur Frame-Zeit).

Syntax:

gc_get_stats();

Rückgabe:

GC Stats Struct

Beispiel:

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

Die obige code prüft eine globale Variable, und wenn sie true ist, holt sie sich Informationen vom Garabge-Kollektor und gibt sie als Debug-Meldungen auf der Konsole aus.
