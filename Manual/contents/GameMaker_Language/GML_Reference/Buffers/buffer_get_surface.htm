buffer_get_surface

Mit dieser Funktion können Sie Informationen aus einer Oberfläche in eine bestimmte buffer schreiben. Das buffer muss zuvor erstellt worden sein und sollte ein 1-Byte-ausgerichtetes buffer sein, das groß genug ist, um Daten für die zu schreibende Oberfläche zu speichern (wenn Sie unsicher sind, verwenden Sie einen Wachstumspuffer oder sehen Sie sich das Beispiel am Ende dieser Seite an).

Die Informationen für die Oberfläche werden immer an den Anfang der buffer geschrieben (unabhängig von der Suchposition), aber Sie können einen Offset-Wert (in Bytes) wählen, um die Schreibposition vom Anfang der buffer zu verschieben.

Beachten Sie, dass die Funktion jedes Pixel der Oberfläche in die buffer schreibt, wobei eine RGBA-Formatierung auf dem Windows Ziel verwendet wird, aber auf anderen Zielen kann dies je nach Betriebssystem oder sogar dem Gerät anders sein.

HINWEIS: Diese Funktion hat sich ab dem GameMaker-Update 2.3.1 geändert. Frühere Versionen verwendeten die Konstante buffer_surface_copy, die nun veraltet ist.

Syntax:

buffer_get_surface(buffer, surface, offset);

Argument Typ Beschreibung
buffer Puffer-ID The index of the buffer to use.
surface Oberfläche ID The index of the surface to use.
offset Real The data offset value (in bytes).

Rückgabe:

N/A

Beispiel:

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

Unter code wird eine buffer mit fester Größe erstellt, die 4 Byte pro Pixel (für die R-, G-, B- und A-Komponenten) auf der Grundlage der exakten Größe der Oberfläche speichern kann, und dann werden die Oberflächendaten dorthin kopiert.
