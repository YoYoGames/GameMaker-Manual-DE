Puffer

Ein buffer ist ein Bereich des physischen Speichers, der verwendet wird, um Daten vorübergehend zu speichern, während sie von einem Ort zu einem anderen verschoben oder auf irgendeine Weise manipuliert werden. Sie können zum Beispiel ein buffer verwenden, um Paketdaten zu speichern, die auf die Übertragung über ein Netzwerk warten, oder um ein Paket zu speichern, das über ein Netzwerk empfangen wurde. GameMaker verfügt über eine Reihe spezifischer Funktionen zum Erstellen, Ändern und Laden/Speichern von buffers. Bevor Sie diese nutzen, beachten Sie bitte die folgenden Punkte:

  • Wenn eine buffer erstellt wird, wird sie automatisch geleert und auf 0 aufgefüllt.
  • Die "Größe" argument der buffer Funktionen bezieht sich immer auf die Größe in Bytes der buffer.
  • Die "Ausrichtung" argument bezieht sich auf die Art und Weise, wie Daten innerhalb einer buffer gespeichert werden. Wenn Ihre Ausrichtung z.B. auf 4 eingestellt ist und Sie ein einzelnes Datenstück mit einer Größe von 1 Byte schreiben und dann eine buffer_tell ausführen, erhalten Sie einen Versatz von 1. Wenn Sie jedoch ein weiteres Datenstück mit einer Größe von ebenfalls 1 Byte schreiben und dann eine buffer_tell ausführen, erhalten Sie einen Versatz von 5, da die Ausrichtung die Daten auf diese Größe "aufgefüllt" hat. Die Ausrichtung wirkt sich nur darauf aus , wohin die Daten geschrieben werden. Wenn Sie also buffer_tell ausführen, nachdem Sie etwas geschrieben haben, wird die aktuelle Schreibposition zurückgegeben, die unmittelbar auf die geschriebenen Daten folgt. Beachten Sie jedoch, dass buffer die Schreibposition intern auf das nächste Vielfache der Ausrichtungsgröße verschiebt, bevor die Daten tatsächlich geschrieben werden, wenn Sie ein weiteres Stück Daten schreiben.
  • Beim "Offset" handelt es sich um den Wert in Bytes, um den die Daten innerhalb der angegebenen buffer versetzt werden sollen. Wenn Ihre buffer also 2 Byte ausgerichtet ist und Sie die ersten 4 ausgerichteten Stellen überspringen wollen, würden Sie einen Offset von 2 * 4 bytes verwenden.
  • Einige der Funktionen von buffer erstellen auch ein neues buffer (wie z. B. buffer_load ). Denken Sie daran, dass auch diese buffers aus dem Speicher entfernt werden müssen, wenn sie nicht mit der Funktion buffer_delete verwendet werden.
  • Der Speicher, der für die Erstellung von buffers verwendet wird, ist der Systemspeicher. Daher sollten alle Daten, die in buffer gespeichert sind, auch dann sicher sein, wenn das Spiel nicht fokussiert ist; wird die App jedoch geschlossen oder neu gestartet, gehen sie verloren.

WICHTIG! Wenn Sie ein buffer erstellen, ist der Indexwert, mit dem es identifiziert wird, ein ganzzahliger Wert, der bei 0 beginnt. Diese Indizes werden von GameMaker wiederverwendet, so dass ein zerstörter buffer Indexwert anschließend von einem neu erstellten verwendet werden kann, und wir empfehlen, dass Sie jede Variable, die einen Pufferindex enthält, auf -1 setzen, nachdem Sie das buffer zerstört haben.
