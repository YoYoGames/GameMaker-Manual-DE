Puffer_erstellen

Sie verwenden diese Funktion, um einen Teil des Speichers als buffer in Ihrem Spiel zuzuweisen, wobei die Funktion die eindeutige Puffer-ID zurückgibt, die in einer Variablen gespeichert und für alle weiteren Funktionsaufrufe an den buffer verwendet werden sollte. Der buffer kann dann verwendet werden, um verschiedene Arten von Daten zu speichern (die beim Schreiben in den buffer mit der buffer_write() Funktion, wobei die folgenden Konstanten zur Definition des buffer Typs verwendet werden:

Puffertyp Konstante
Konstante Beschreibung
buffer_fixed Eine buffer mit fester Größe.
buffer_grow Eine buffer, die dynamisch "wächst", wenn Daten hinzugefügt werden
buffer_wrap Eine buffer, in der die Daten "umbrochen" werden. Wenn die hinzugefügten Daten die Grenze der Größe von buffer erreichen, wird die überschriebene Datei wieder an den Anfang von buffer gesetzt, und das weitere Schreiben wird von diesem Punkt an fortgesetzt.
buffer_fast Spezielle "gestrippte" buffer, die extrem schnell zu lesen/schreiben ist. Kann nur mit buffer_u8 Datentypen verwendet werden und muss an 1 Byte ausgerichtet sein.
buffer_vbuffer Dieser Typ von buffer ist nur als Scheitelpunkt buffer zu verwenden.

Neben dem Typ buffer müssen Sie auch die Byte-Ausrichtung für buffer festlegen. Dieser Wert hängt von den Daten ab, die Sie in buffer speichern wollen, und in den meisten Fällen ist ein Wert von 1 völlig ausreichend. Beachten Sie jedoch, dass für einige Operationen eine bestimmte Ausrichtung unerlässlich ist und eine falsche Ausrichtung zu Fehlern führen kann (weitere Einzelheiten zur Ausrichtung siehe Puffer). Im Folgenden finden Sie einen allgemeinen Leitfaden, der zeigt, welche Werte für die einzelnen Datentypen am besten geeignet sind (im Zweifelsfall sollten Sie eine Ausrichtung von 1 verwenden):

  • Strings sollten auf 1 Byte ausgerichtet sein.
  • Vorzeichenbehaftete oder vorzeichenlose 8-Bit-Ganzzahlen können auf einen beliebigen Wert ausgerichtet werden, aber beachten Sie, dass für eine schnelle buffer (siehe buffer_write()) an 1 ausgerichtet sein muss.
  • Vorzeichenbehaftete oder vorzeichenlose 16-Bit-Ganzzahlen sollten auf 2 Bytes ausgerichtet werden.
  • Vorzeichenbehaftete oder vorzeichenlose 32-Bit-Ganzzahlen sollten auf 4 Bytes ausgerichtet werden.
  • Fließkommazahlen von bis zu 16 Bit sollten auf 2 Byte ausgerichtet werden. (Derzeit nicht unterstützt!)
  • Floats mit bis zu 32 Bit sollten auf 4 Bytes ausgerichtet werden.
  • Floats mit bis zu 64 Bit sollten auf 8 Byte ausgerichtet werden.

HINWEIS: Die Byte-Ausrichtung kann sehr wichtig sein, da eine falsche Wahl die Leistung beeinträchtigen kann.

HINWEIS: Es ist wichtig, dass Sie alle dynamisch erstellten Ressourcen wie diese aus dem Speicher entfernen, wenn Sie sie nicht mehr benötigen, um Speicherlecks zu vermeiden. Wenn Sie also mit der von Ihnen erstellten buffer fertig sind, sollten Sie sie wieder freigeben mit buffer_delete().

Syntax:

buffer_create(size, type, alignment)

Argument Typ Beschreibung
size Real The size (in bytes) of the buffer.
type Puffertyp Konstante The type of buffer to create (see the constants list above).
alignment Real The byte alignment for the buffer

Rückgabe:

Buffer ID

Beispiel:

player_buffer = buffer_create(16384, buffer_fixed, 2);

Die obige code weist einem buffer 16384 Byte Speicher zu und gibt den Index dieses buffer zurück, der in der Variablen "player_buffer" zur späteren Verwendung gespeichert wird. Die buffer ist an einer Zwei-Byte-Grenze ausgerichtet.
