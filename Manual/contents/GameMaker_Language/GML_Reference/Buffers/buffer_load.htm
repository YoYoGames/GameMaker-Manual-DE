Puffer_laden

Mit dieser Funktion können Sie eine buffer laden, die zuvor mit den buffer_save() Funktionen gespeichert wurde, sowie alle eingeschlossenen Dateien oder Dateien, die extern geladen wurden. Sie gibt die ID eines neuen Puffers zurück, der von der Funktion als "grow" buffer (siehe hier) mit einer Byteausrichtung von 1 erstellt wird. Diese ID sollte in einer Variablen gespeichert und bei allen weiteren Funktionsaufrufen zu diesem buffer verwendet werden. Wenn das Laden aus irgendeinem Grund fehlschlägt, gibt die Funktion -1 zurück und im Ausgabefenster compiler wird eine Meldung angezeigt, dass das Laden fehlgeschlagen ist.

HINWEIS Es ist wichtig, dass Sie alle dynamisch erstellten Ressourcen wie diese aus dem Speicher entfernen, wenn Sie sie nicht mehr benötigen, um Speicherlecks zu vermeiden. Wenn Sie also mit der von Ihnen erstellten buffer fertig sind, sollten Sie sie wieder freigeben mit buffer_delete().

Plattformspezifische Hinweise

  • Bei HTML5 werden alle buffers, die aus dem lokalen Speicher geladen werden, mit base64 dekodiert (da sie als base64-Strings gespeichert werden). Bei der Verwendung dieser Funktion für eingeschlossene Dateien oder externe Dateien wird jedoch keine base64-Dekodierung verwendet, da sie nicht im base64-Format vorliegen sollten.
  • Bei HTML5 werden alle eingeschlossenen Dateien als UTF-8 kodierter Text gespeichert, was dazu führen kann, dass die Größe der geladenen buffer im Vergleich zur Originaldatei zunimmt, da die UTF-8 kodierte Version der Datei eine andere Größe haben kann als Ihre Originaldatei (aufgrund der Änderung des Formats).

Syntax:

buffer_load(filename);

ArgumentTyp Beschreibung
filename The name of the file to load from.

Rückgabe:

Buffer ID

Beispiel:

player_buffer = buffer_load("Player_Save.sav");

Die obige code lädt ein zuvor gespeichertes buffer in den Speicher und erstellt ein neues buffer. Der Index dieses neuen buffer wird in der Variablen "player_buffer" gespeichert.
