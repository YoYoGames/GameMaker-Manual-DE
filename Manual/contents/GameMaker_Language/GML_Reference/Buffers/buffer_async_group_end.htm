buffer_async_group_end

Mit dieser Funktion wird die Definition einer buffer async-Gruppe abgeschlossen. Sie müssen zuvor die Funktion buffer_async_group_begin() aufrufen, um die Gruppe zu starten, und dann die Funktion buffer_save_async() für jede Datei auf, die Sie auslagern möchten (oder buffer_load_async(), um buffers zu laden). Schließlich rufen Sie diese Funktion auf, die das Speichern der Dateien startet. Die Funktion gibt einen eindeutigen ID-Wert für die Speicherung zurück, der dann im Ereignis Asynchrones Speichern/Laden verwendet werden kann, um die Ergebnisse aus der async_load DS-Map.

Syntax:

buffer_async_group_end();

Rückgabe:

Async Request ID

Erweitertes Beispiel:

Die Funktion buffer_async_group_end() kann von jedem Ereignis aus aufgerufen werden, und da sie asynchron ist, kann der Rückruf fast sofort erfolgen oder mehrere Sekunden dauern. Der Aufruf der Funktion ist einfach und würde etwa so aussehen:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

Die obige code startet eine buffer Gruppe und setzt sie so, dass 4 Dateien asynchron gespeichert werden. Die Gruppendefinition wird dann beendet (zu diesem Zeitpunkt beginnt die Speicherung), wobei die ID des Funktionsaufrufs in der Variablen "save_id" gespeichert wird. Wenn der Speichervorgang abgeschlossen ist, wird das asynchrone Save/Load-Ereignis ausgelöst, und Sie können die Map async_load nach der korrekten ID der Funktion durchsuchen, etwa so:

if ds_map_find_value(async_load, "id") == saveid
{
    if ds_map_find_value(async_load, "status") == false
    {
        show_debug_message("Save failed!");
    }
}

Die obige code prüft zunächst die ID der erstellten DS-Map, dann den Status des Rückrufs und gibt eine Debug-Meldung aus, wenn Probleme aufgetreten sind.
