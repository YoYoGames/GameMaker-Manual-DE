buffer_async_group_begin

Diese Funktion wird aufgerufen, wenn Sie das Speichern mehrerer buffers in mehrere Dateien (oder das Laden mehrerer buffers) beginnen wollen. Der Gruppenname ist eine Zeichenkette und wird als Verzeichnisname für den Ort verwendet, an dem die Dateien gespeichert werden, und sollte als Teil des Dateipfads verwendet werden, wenn die Dateien später wieder in IDE geladen werden (mit einer der buffer_load() Funktionen). Diese Funktion ist nur für die Verwendung mit den Funktionen buffer_save_async() und buffer_load_async() und Sie müssen auch die asynchrone Gruppe durch den Aufruf der buffer_async_group_end() beenden, da die Dateien sonst nicht gespeichert/geladen werden können.

Beachten Sie, dass für die Konsole platforms (z. B. PS4) der Gruppenname als Beschreibung des Speicherplatzes verwendet wird. Mit dieser Funktion können Sie vermeiden, dass die Benutzeroberfläche für jede Datei angezeigt wird, die gespeichert wird. Beachten Sie auch, dass Sie bei der Verwendung von UWP Ihre Spielstände zwischen nativen Xbox- und Microsoft-Store-Fenstern hin- und herbewegen können, indem Sie eine Gruppe erstellen, die z. B. mit einer ERA Xbox-Gruppe identisch ist:

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

Syntax:

buffer_async_group_begin(groupname);

Argument Typ Beschreibung
groupname Zeichenfolge The name of the group (as a string).

Rückgabe:

N/A

Beispiel:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Mit der obigen code wird eine Gruppe buffer gestartet und dann so eingestellt, dass 4 Dateien asynchron gespeichert werden. Die Gruppendefinition wird dann beendet (und das Speichern beginnt).
