puffer_sha1

In der Kryptografie ist SHA-1 eine kryptografische Hash-Funktion, die von der Nationalen Sicherheitsbehörde der Vereinigten Staaten entwickelt wurde und in mehreren weit verbreiteten Anwendungen und Protokollen wie dem beliebten Git verwendet, wo es zur Überprüfung von Dateiänderungen eingesetzt wird. Diese Funktion nimmt Eingabedaten von buffer entgegen und gibt einen 160-Bit-Message-Digest im ASCII-Format zurück. Auf diese Weise können Sie einen sicheren Schlüssel generieren, der gespeichert und verwendet werden kann, um die Integrität der an eine externe server gesendeten (oder von ihr empfangenen) Informationen zu überprüfen. Wenn Sie diese Funktion auf buffers anwenden, müssen Sie die buffer id des zu verwendenden buffer angeben, dann einen Offset-Wert (in Bytes) für den Anfang und dann eine Größe (wiederum in Bytes) für den zu hashenden Bereich.

HINWEIS: SHA-1 ist nicht völlig sicher und kann geknackt werden. Siehe diese Seite für weitere Informationen.

Syntax:

buffer_sha1(buffer, offset, size);

ArgumentTyp Beschreibung
buffer The index of the buffer to use.
offset The data offset value.
size The size of the buffer.

Rückgabe:

Beispiel:

check_string = buffer_sha1(buff, 0, buffer_get_size(buff));

Die obige code erstellt einen sha1-Hash für die vollständigen Daten, die in der buffer gespeichert sind, indiziert durch die Variable "buff", und speichert den zurückgegebenen Hash in der Variable "check_string".
