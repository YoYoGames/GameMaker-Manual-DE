xboxlive_sprite_add_from_gamerpicture

Mit dieser Funktion können Sie das Spielerbild für einen bestimmten Benutzer-ID-Zeiger abrufen. Die Funktion arbeitet asynchron und löst ein asynchrones Ereignis Image Loaded aus, um Sie zu informieren, dass die Funktion beendet ist (wie bei HTML5/URL-Aufrufen an sprite_add()). Das Asynchronous Image Loaded-Ereignis hat eine DS-Map, die in der eingebauten Variable async_load. Die Map enthält die folgenden Informationen:

  • "Dateiname": Der vollständige Pfad zu der von Ihnen angeforderten Datei.
  • "id": Die ID der Ressource, die Sie geladen haben. Dies ist dieselbe wie die Variable, der Sie die Ressource zugewiesen haben.
  • "Status": Gibt bei einem Fehler einen Wert kleiner als 0 zurück.

HINWEIS: Wenn Sie ein sprite dynamisch in Ihr Spiel laden ( runtime), müssen Sie daran denken, es wieder zu entfernen, indem Sie sprite_delete() wieder zu entfernen, wenn es nicht mehr benötigt wird. Andernfalls besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

ArgumentTyp Beschreibung
user_id The user ID (a pointer) to get the gamer picture for
imagesize Size in pixels of the sprite to be returned
xorig Indicate the x position of the origin in the sprite
yorig Indicate the y position of the origin in the sprite

Rückgabe:

Async Request ID

Beispiel:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
    var _uid = xboxlive_get_user(i);
    if _uid != pointer_null
    {
        global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
        global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
        global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
        global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
        ++a;
    }
}

Die obige code loops durchläuft die angemeldeten Benutzer und speichert ihre Spieldaten in verschiedenen globalen Arrays.
