uwp_check_privilege

Mit dieser Funktion können Sie prüfen, ob ein bestimmter Benutzer über ein bestimmtes Privileg verfügt. Die Funktion gibt im asynchronen Systemereignis true oder false zurück, je nachdem, ob das Privileg aktiviert ist oder nicht, und wenn Sie attempt_resolution argument auf true setzen und das Privileg nicht aktiviert ist, öffnet sie auch einen Systemdialog (der das Spiel aussetzt), um den Benutzer aufzufordern, das Konto zu aktualisieren oder was auch immer erforderlich ist, um das Privileg zu erhalten. Wenn der Benutzer dann die erforderliche Option aktiviert, gibt die Funktion true zurück.

Die Funktion erfordert, dass Sie eine der folgenden Konstanten verwenden UWP Privilege Constants für die Privilegienabfrage:

UWP Privileg Konstante
Konstante Beschreibung
uwp_privilege_internet_browsing Prüfen Sie, ob das Surfen im Internet erlaubt ist
uwp_privilege_social_network_sharing Prüfen Sie, ob die Weitergabe an soziale Netzwerke erlaubt ist
uwp_privilege_share_kinect_content Prüfen Sie, ob die gemeinsame Nutzung des Kinect-Controllers erlaubt ist
uwp_privilege_video_communications Prüfen Sie, ob Videokommunikation erlaubt ist
uwp_privilege_communications Prüfen Sie, ob die Internetkommunikation erlaubt ist
uwp_privilege_user_created_content Prüfen, ob der Zugriff auf vom Benutzer erstellte Inhalte erlaubt ist
uwp_privilege_multiplayer_sessions Prüfen Sie, ob Online-Multiplayer-Sitzungen erlaubt sind
uwp_privilege_sessions Prüfen Sie, ob Online-Sitzungen erlaubt sind
uwp_privilege_fitness_upload Prüfen Sie, ob das Hochladen von Fitnessdaten erlaubt ist

Sobald die Funktion aufgerufen wurde, löst sie auf XBox One ein asynchrones Systemereignis aus, wenn das Ergebnis des Rückrufs empfangen wurde, das die spezielle DS-Map hat async_load. Diese Map sollte dann nach dem folgenden Schlüssel geparst werden:

  • "event_type" - sollte den Wert "check_privilege_result" enthalten, wenn das Ereignis durch diese Funktion ausgelöst wurde

Wenn sich der Ereignistyp auf diese Funktion bezieht, gibt es auch die folgenden zusätzlichen Tasten, wenn die Funktion auf der XBox aufgerufen wird:

  • "Ergebnis" - Eine oder mehrere der folgenden Angaben UWP-Privileg Ergebniskonstantes (kombinierte Werte):
    UWP Privileg Ergebnis Konstante
    Konstante Beschreibung Wert
    uwp_privilege_result_no_issue Es gibt keine Probleme mit den Berechtigungen des Benutzers 0
    uwp_privilege_result_purchase_required Der Benutzer muss etwas Zusätzliches, normalerweise ein Abonnement, für den Zugriff erwerben 1
    uwp_privilege_result_aborted Die Prüfung wurde abgebrochen 2
    uwp_privilege_result_banned Der Benutzer ist gesperrt 4
    uwp_privilege_result_restricted Der Benutzer ist vom Zugriff ausgeschlossen, in der Regel durch die elterliche Kontrolle 8

  • "Privileg" - ist eine der oben aufgeführten Konstanten für die Verwendung in der Funktion, so dass Sie überprüfen können, welches Privileg Sie im Falle von Mehrfachprüfungen beantragt haben.

Wenn das Projekt auf einer UWP platform ausgeführt wird, die nicht XBox One ist, wird das Async-Ereignis nicht ausgelöst, und stattdessen gibt die Funktion sofort entweder -1, 0 oder 1 zurück:

  • -1 bedeutet -1, dass der Benutzer das Recht nicht hat
  • 0 bedeutet, dass die Funktion den Benutzer nicht abfragen konnte
  • 1 bedeutet, dass der Benutzer die Berechtigung hat.

Syntax:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Argument Typ Beschreibung
user_id Xbox-Benutzer-ID The User ID pointer to check.
privilege UWP Privileg Konstante The privilege constant to check (see description, above).
attempt_resolution Boolesche Enable attempting a resolution if set to trueoder ignorieren, wenn sie auf false eingestellt ist.

Rückgabe:

Real

Beispiel:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

Die obige Funktion prüft, ob der Benutzer angemeldet ist, und wenn ja, prüft sie die Berechtigung für Multiplayer-Sitzungen.
