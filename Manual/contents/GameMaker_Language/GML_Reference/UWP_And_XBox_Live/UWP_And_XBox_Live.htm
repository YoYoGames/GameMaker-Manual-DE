UWP und Xbox Live

In diesem Abschnitt werden die Funktionen beschrieben, die speziell für das UWP-Ziel platform verwendet werden. Die ersten Funktionen dienen der Steuerung des allgemeinen Spielverhaltens beim Erstellen einer UWP-Anwendung:

  • uwp_suspend
  • uwp_is_suspending
  • uwp_is_constrained
  • uwp_was_terminated
  • uwp_was_closed_by_user
  • uwp_show_help
  • uwp_license_trial_version
  • uwp_license_trial_user
  • uwp_license_trial_time_remaining
  • uwp_check_privilege

Mit UWP können Sie auch Spiele für XBox One platform erstellen, indem Sie die XBox Live API mit den folgenden Funktionen verwenden. Diese Funktionen setzen voraus, dass Sie die Option XBox Live aktivieren in den UWP-Spieloptionen aktiviert haben. Für einige dieser Funktionen muss der Benutzer bei XBox Live angemeldet sein, was GameMaker automatisch versucht, wenn Sie das Projekt ausführen. Diese "stille" Anmeldung erzeugt ein asynchrones Systemereignis, das vor dem Fortfahren überprüft werden kann, um festzustellen, ob sie erfolgreich war oder nicht. Beachten Sie, dass im Gegensatz zum Xbox One-Konsolenexport beim UWP-Export für XBox Live nur ein Benutzer gleichzeitig angemeldet werden kann.

WICHTIG! Das UWP-Ziel verfügt nur über ein Minimum an Unterstützung für XBox Live, wenn es als Teil des Creator-Programms eingerichtet wurde, das auf einige grundlegende Benutzer- und Kontofunktionen beschränkt ist (weitere Informationen zur Einrichtung finden Sie hier ). Um alle hier aufgeführten Funktionen nutzen zu können, benötigen Sie ein ID@XBox-Konto.
