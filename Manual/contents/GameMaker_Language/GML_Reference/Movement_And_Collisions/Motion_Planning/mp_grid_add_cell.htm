mp_grid_add_cell

Mit dieser Funktion können wir einzelne Zellen innerhalb des gegebenen MP-Gitters als "verboten" markieren, was bedeutet, dass die Pfadfindungsfunktionen sie niemals durchqueren werden. Normalerweise arbeitet man in x/y-Raumkoordinaten und nicht in MP-Gitterzellenkoordinaten. Betrachten wir also das folgende Bild, das zeigt, wie die beiden miteinander korrelieren können:

Wie du sehen kannst, ist das mp_grid als eine Reihe von Zellen nummeriert (beginnend bei 0,0 in der oberen linken Ecke), und wenn du dieses Gitter über den Raum legst, gibt es eine direkte Korrelation zwischen den Raumkoordinaten (die ebenfalls bei 0,0 oben links beginnen) und den Zellen innerhalb des Gitters. Im Bild sieht man, dass eine Instanz an der Position 260,130 auf die Gitterzelle 8,4 fällt. Sie können die genaue Zelle, auf die eine Position fällt, mathematisch leicht berechnen, indem Sie entweder die Zahl durch die Gitterauflösung dividieren und dann abrunden oder, wenn Sie eine Potenz von 2 Gitterauflösungen (wie 8, 16, 32 usw.) verwenden, die bitweisen Operatoren benutzen. Es sei darauf hingewiesen, dass die bitweise Operation schneller ist, und das folgende Beispiel zeigt beide Methoden.

Syntax:

mp_grid_add_cell(id, h, v);

Argument Typ Beschreibung
id MP Raster-ID Index of the mp_grid that is to be used
h Real Horizontal position of the cell in the grid
v Real Vertical position of the cell in the grid

Rückgabe:

N/A

Beispiel:

with (obj_Wall)
{
    mp_grid_add_cell(grid, floor(x / 32), floor(y / 32));
}
with (obj_Door)
{
    mp_grid_add_cell(grid, x>>5, y>>5);
}

Die obige code bewirkt, dass alle "obj_Wall" und alle obj_Door ihre entsprechenden Zellenpositionen in das mp_grid einfügen, das in der Variablen "grid" indiziert ist. Im ersten Teil des Beispiels geschieht dies, indem die x/y-Koordinaten des Objekts genommen und durch die Auflösung des Gitters geteilt werden (unter Verwendung von floor, um die Werte als ganze Zahlen zu erhalten), und im zweiten Teil (da die Auflösung des Gitters eine Potenz von 2 ist) geschieht dies durch Bit-Shifting der x,y-Koordinaten um 5 Stellen nach rechts.
