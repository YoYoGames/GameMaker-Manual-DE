mp_grid_add_instances

Diese Funktion verwendet die Kollisionsmaske der Instanz (bestimmt durch die sprite Eigenschaften oder den Maskenindex der aufrufenden Instanz), um Zellen in einem MP-Gitter als verboten oder nicht verboten zu markieren, wobei verbotene Zellen von keiner der Pfadfindungsfunktionen durchquert werden können. Sie können in der Funktion angeben, ob präzise Kollisionen berücksichtigt werden sollen oder nicht, und die markierten verbotenen Zellen ändern sich in Abhängigkeit von dieser Einstellung. Dieses Bild veranschaulicht dieses Verhalten:

Die beiden obigen Instanzen wurden mit der Funktion mp_grid_add_instances() in das MP-Gitter eingefügt, wobei "precise" auf "true" gesetzt wurde. Wie Sie sehen können, hat die grüne Instanz nur die Gitterquadrate, die sie "berührt", als verboten markiert, da ihre sprite Maske ebenfalls auf präzise eingestellt ist. Die zweite Instanz hat jedoch auch andere Zellen markiert, die sich nicht zu berühren scheinen. Das liegt daran, dass die Maske sprite nicht auf präzise eingestellt ist, d. h. selbst wenn Sie die präzise Einstellung in der Funktion verwenden, wird nur der Begrenzungsrahmen der sprite berücksichtigt. Dies würde auch passieren, wenn Sie die präzise argument der Funktion auf false setzen - selbst Instanzen mit einer präzisen sprite Maske werden auf der Grundlage ihrer Begrenzungsrahmen in das Raster eingefügt.

Syntax:

mp_grid_add_instances(id, obj, prec);

Argument Typ Beschreibung
id MP Raster-ID Index of the mp_grid that is to be used
obj Objekt Asset- oder Instanz-ID Object index, or instance id, of the instances to be added into the mp_grid
prec Boolesche Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).

Rückgabe:

N/A

Beispiel:

mp_grid_add_instances(grid, obj_Wall, true);

Die obige code fügt alle Instanzen von "obj_Wall" in das mp_grid ein, das in der Variablen "grid" indiziert ist, wobei die genaue Kollisionsmaske und nicht die Bounding Box verwendet wird.
