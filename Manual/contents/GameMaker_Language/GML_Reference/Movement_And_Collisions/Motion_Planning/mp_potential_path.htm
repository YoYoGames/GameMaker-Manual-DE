mp_potential_pfad

Diese Funktion berechnet einen Pfad für die Instanz von ihrer aktuellen Position und Orientierung zu der durch xgoal,ygoal arguments angegebenen Position. Sie verwendet die angegebene Schrittweite und versucht dabei, Kollisionen entweder mit allen Instanzen oder nur mit den als solide markierten Instanzen zu vermeiden. Wie bei der Funktion mp_potential_step() verwendet sie dazu Potentialfeldschritte, und die Art und Weise, wie sie dies tut, kann durch die Funktion mp_potential_settings(). Der angegebene Pfad muss bereits existieren und wird durch den neuen Pfad überschrieben. Die Funktion gibt zurück, ob ein Pfad gefunden wurde (true) oder nicht (false).

Um zu vermeiden, dass die Funktion ewig weiterrechnet, müssen Sie einen Faktor größer als 1 angeben - die Funktion hält an und meldet einen Fehler, wenn sie keinen Weg findet, der kürzer als dieser Faktor mal der Entfernung zwischen Start und Ziel ist. Ein Faktor von 4 ist normalerweise gut genug, aber wenn Sie lange Umwege erwarten, können Sie ihn auch größer wählen. Wenn die Funktion fehlschlägt, wird immer noch ein Pfad erstellt, der in die ungefähre Richtung des Ziels verläuft, dieses aber nicht erreichen wird.

HINWEIS: Mit dieser Funktion wird die Instanz nicht bewegt. Sie legt nur einen Pfad fest, und Sie müssen die Pfadfunktionen für die Bewegung verwenden.

Syntax:

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

ArgumentTyp Beschreibung
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

Rückgabe:

Beispiel:

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

Die obige code erstellt eine neue Pfadressource und speichert ihren Index in der Variablen "path". Dann wird mit mp_potential_path ein Pfad zwischen dem Objekt mit dieser code und dem Objekt "obj_Player" erzeugt, wobei auf Kollisionen mit allen Instanzen im Raum. Schließlich wird das Objekt entlang dieses Pfades gestartet, auch wenn die Funktion keine vollständige Route zum "obj_Player" zurückgegeben hat.
