mp_potential_settings

Die Funktionen von mp_potential_ arbeiten mit einer Reihe von Parametern, die mit dieser Funktion geändert werden können. Insgesamt funktioniert die Methode wie folgt: Sie versucht zunächst, sich geradeaus auf das Ziel zuzubewegen, wobei sie eine bestimmte Anzahl von Schritten vorausschaut, die mit dem Parameter "ahead" eingestellt werden kann (Standardwert ist 3). Eine Verringerung dieses Wertes bedeutet, dass die Instanz später mit der Richtungsänderung beginnt, da sie nicht so weit nach vorne "sieht", und eine Erhöhung bedeutet, dass sie früher mit der Richtungsänderung beginnt. Wenn diese Prüfung zu einer Kollision führt, beginnt sie, die Richtungen links und rechts von der idealen, geradlinigen Richtung zu betrachten. Dies geschieht in Schritten der Größe "rotstep" (Standardwert ist 10), und eine Verringerung dieser Größe gibt der Instanz mehr Bewegungsmöglichkeiten, wird aber langsamer sein, da sie mehr Rechenleistung benötigt.

Der Parameter "maxrot" ist ein bisschen schwieriger zu erklären! Die Instanz hat eine aktuelle Bewegungsrichtung, und maxrot (Standardwert ist 30) gibt an, wie stark sie ihre Richtung in einem Schritt zu beiden Seiten dieser aktuellen Richtung ändern darf. Selbst wenn sie sich also z.B. geradeaus zum Ziel bewegen kann, wird sie das nur tun, wenn sie diese maximale Richtungsänderung nicht überschreitet. Wenn du maxrot zu einem großen Wert machst, dann kann die Instanz in jedem Schritt die Richtung sehr oft ändern, und das wird es einfacher machen, einen kurzen Weg zu finden, aber der Weg wird hässlicher und weniger natürlich sein. Wenn du den Wert kleiner wählst, wird der Weg glatter, aber er kann längere Umwege machen (und manchmal sogar das Ziel verfehlen). Schließlich, wenn kein Schritt gemacht werden kann, hängt das Verhalten vom Wert des Parameters "onspot" ab. Wenn onspot true ist (der Standardwert), dann dreht sich die Instanz auf ihrem Punkt um den mit "maxrot" angegebenen Betrag und wenn er false ist, dann bewegt sie sich überhaupt nicht. Die Einstellung false ist z.B. für Autos nützlich, verringert aber die Chance, dass die Instanz einen Weg findet.

Syntax:

mp_potential_settings(maxrot, rotstep, ahead, onspot)

Argument Typ Beschreibung
maxrot Real The number of degrees either side of the current direction that the instance can rotate in a step
rotstep Real The number of degrees either side of the current direction that the instance can check for a collision
ahead Real The number of steps ahead that the instance can check for a collision. Larger values are slower than lower ones.
onspot Boolesche Use this to allow the instance to rotate on the spot when no path is found (true) or not (false)

Rückgabe:

N/A

Beispiel:

mp_potential_settings(45, 5, 5, 0);

Unter code werden die Funktionen von mp_potential_ so eingestellt, dass sie folgende Einstellungen verwenden: Die Instanz kann fünf Schritte vorausschauen, bei jedem Schritt die Richtung um 45 Grad ändern, bei einem Hindernis um 5 Grad nach beiden Seiten der aktuellen Richtung schauen und sich bei einer Kollision nicht auf der Stelle drehen.
