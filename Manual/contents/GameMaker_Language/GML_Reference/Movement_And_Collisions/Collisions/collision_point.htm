collision_point

Collision_point prüft den Punkt, der durch arguments x1,y1 angegeben ist, auf eine Kollision mit einer beliebigen Instanz des Objekts, das durch argument "obj" angegeben ist. Diese Prüfung kann entweder präzise oder nicht präzise sein, aber damit präzise Kollisionen aktiviert werden können, müssen für das Objekt oder die Instanz, auf die Sie prüfen, auch präzise Kollisionen für deren sprite aktiviert sein. Ist dies nicht der Fall, basiert die Standardprüfung auf Bounding Boxes. Das folgende Bild veranschaulicht, wie dies funktioniert:

Damit präzise Kollisionen berücksichtigt werden, müssen sowohl das Objekt sprite als auch die Kollisionsfunktion als präzise markiert sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, die als kollidierend betrachtet werden. Wenn sich also drei Instanzen an diesem Punkt überschneiden, könnte jede ihrer IDs der Rückgabewert der Funktion sein.

Beachten Sie, dass Sie anstelle eines Objektindex eine Instanz angeben können id angeben können, um nach einer bestimmten Instanz zu suchen, oder die Instanz-Schlüsselwörter all oder other (je nach Ereignis und aktuellem Bereich code ).

Syntax:

collision_point(x, y, obj, prec, notme);

Argument Typ Beschreibung
x Real The x coordinate of the point to check.
y Real The y coordinate of the point to check.
obj Objekt Asset- oder Instanz-ID The object to check for instance collisions.
prec Boolesche Whether the check is based on precise collisions (true(was langsamer ist) oder seine Bounding Box im Allgemeinen (false, schneller).
notme Boolesche Whether the calling instance, if relevant, should be excluded (true) oder nicht (false).

Rückgabe:

Instance ID or noone

Beispiel:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

Hier prüfen wir den Punkt an der Position des Objekts, das die code für das Objekt "obj_Cursor" hat. Wenn es einen gibt, dann addieren wir 10 auf die Punktevariable.
