Rechteck_im_Kreis

Diese Funktion prüft einen von Ihnen definierten rechteckigen Bereich daraufhin, ob er entweder nicht kollidiert, vollständig innerhalb der Grenzen des Ziels liegt oder ob er lediglich einen definierten kreisförmigen Bereich berührt. Wenn sie sich nicht berühren, gibt die Funktion den Wert 0 zurück, wenn die Quelle vollständig innerhalb des Ziels liegt, gibt sie den Wert 1 zurück, und wenn sie sich lediglich überschneiden, gibt sie den Wert 2 zurück. Die folgende Abbildung veranschaulicht dies:

Syntax:

rectangle_in_circle(sx1, sy1, sx2, sy2, x, y, rad)

ArgumentTyp Beschreibung
sx1 The x coordinate of the left side of the source rectangle.
sy1 The y coordinate of the top side of the source rectangle.
sx2 The x coordinate of the right side of the source rectangle.
sy2 The y coordinate of the bottom side of the source rectangle.
x The x coordinate of the centre of the circle
y The y coordinate of the centre of the circle.
rad The radius around the center point in which to check for a collision.

Rückgabe:

Beispiel:

inst = instance_nearest(x, y, obj_Bullet);
if (instance_exists(inst))
{
    if (rectangle_in_circle(_inst.x - 5, _inst.y - 5, _inst.x + 5, _inst.y + 5, x, y - 25, 20) > 0)
    {
        hit = true;
    }
}

Die obige code verwendet die Funktion rectangle_in_circle, um auf eine Kollision innerhalb eines kreisförmigen Bereichs und des Rechtecks um eine gefundene Instanz zu prüfen. Liegt eine Kollision vor (entweder eine Kantenüberschneidung oder eine Umschließung), wird eine Variable auf true gesetzt.
