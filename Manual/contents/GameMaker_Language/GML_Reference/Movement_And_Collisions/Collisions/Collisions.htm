Kollisionen

Bei der Planung von Bewegungen oder der Entscheidung für bestimmte Aktionen ist es oft wichtig zu sehen, ob es an bestimmten Stellen in der Spielwelt Kollisionen mit anderen Objekten gibt, und oft ist die Wahl der richtigen Kollision für die Aufgabe die wichtigste Aufgabe überhaupt. GameMaker hat eine Reihe von Funktionen eingebaut, die Ihnen helfen, mit Kollisionen korrekt und den Bedürfnissen Ihres Projekts entsprechend umzugehen.

HINWEIS In früheren Versionen von GameMaker haben Kollisionsfunktionen Ganzzahlwerte verwendet. Jetzt verwenden sie Fließkommawerte, was bedeutet, dass Kollisionsfunktionen nicht mehr auf nicht-ganzzahlige Werte runden arguments. Für weitere Informationen lesen Sie bitte Kollisionskompatibilitätsmodus.

Die folgenden Funktionen befassen sich mit den verschiedenen Möglichkeiten, wie Sie eine Instanz auf Kollisionen mit einer anderen Instanz prüfen lassen können:

  • place_empty
  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Erweiterte Kollisionsprüfung

Die folgenden Routinen können dafür verwendet werden (sowie andere Funktionen, die in den Abschnitten über das Herumbewegen und die Instanzen beschrieben werden). Diese Kollisionsfunktionen arbeiten alle etwas anders, aber sie haben drei gemeinsame arguments, die wir hier erläutern werden:

  • obj: Hier wird das Objekt angegeben, mit dem die Funktion auf eine Kollision prüfen soll. Es kann eine Instanz-ID, ein Objekttyp oder das spezielle GM-Schlüsselwort all sein. Es ist auch erwähnenswert, dass, wenn der gewählte Objekttyp der designierte Elternteil von anderen Objekten ist, alle Kinder dieses Objekts ebenfalls in diese Prüfung einbezogen werden.
  • prec: Dies kann entweder true oder false sein und bezieht sich auf die Genauigkeit der Kollisionsprüfung. Bei true werden präzise Kollisionen berechnet (aber nur, wenn das zu prüfende Objekt in seinen Maskeneigenschaften "präzise" für seine Kollisionserkennung markiert hat), was eine etwas komplexere Aufgabe für die CPU darstellt und daher mit Vorsicht verwendet werden sollte. Bei false werden die Kollisionen auf der Grundlage des Begrenzungsrahmens des zu überprüfenden Objekts berechnet.
  • notme: Es kann sein, dass Sie nach Kollisionen mit einem Objekt suchen müssen, das den gleichen Index hat wie das Objekt, das die code ausführt. Um zu verhindern, dass code eine Kollision mit sich selbst findet, können Sie dies auf true setzen. Wenn Sie speziell nach der Instanz des Objekts suchen, das die code ausführt, oder es nicht wichtig ist, können Sie es auf false lassen.

Die Basisfunktionen geben entweder die ID einer kollidierenden Instanz oder das spezielle Schlüsselwort noone zurück, wenn keine Kollision vorliegt, während die Listenfunktionen die Anzahl der kollidierenden Instanzen zurückgeben und eine vorgefertigte DS-Liste mit deren IDs füllen. Beachten Sie, dass, wenn es mehrere Kollisionen mit den von diesen Funktionen definierten Bereichen und Instanzen des gegebenen Objekts gibt, nur eine Instanz-ID zurückgegeben wird, und diese kann eine beliebige der Instanzen in der Kollision sein.

Es gibt die folgenden Funktionen, die sich mit fortgeschrittenen Kollisionen befassen.

  • collision_circle
  • collision_circle_list
  • collision_ellipse
  • collision_ellipse_list
  • collision_line
  • collision_line_list
  • collision_point
  • collision_point_list
  • collision_rectangle
  • collision_rectangle_list

Kollisionsprüfung ohne Maske

Alle oben genannten Funktionen beziehen sich auf die Kollisionsprüfung von Instanzen und basieren auf der Kollisionsmaske, die für die Instanz definiert ist. Es gibt jedoch viele Momente, in denen Sie auf "Kollisionen" mit einem Punkt oder einem Bereich prüfen müssen, insbesondere wenn Ihrer Instanz keine sprite zugewiesen ist, oder wenn Sie mit der Maus arbeiten usw... Deshalb bietet GameMaker auch die folgenden Funktionen, um dir in diesen Situationen zu helfen:

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

Physik Kollisionen

Wenn Sie die eingebaute Physik verwenden, funktionieren die oben genannten Kollisionsfunktionen nicht garantiert für physikaktivierte Instanzen. Dies liegt vor allem daran, dass diese Instanzen nicht mehr die meisten der regulären eingebauten Variablen verwenden (stattdessen haben physikaktivierte Instanzen ihren eigenen Satz von Variablen) und auch nicht die Kollisionsmaske oder die Bounding Box verwenden, da sie stattdessen Fixtures verwenden. Es gibt jedoch eine spezielle Funktion zum Testen von Kollisionen mit aktivierter Physik, die auf überlappende Fixtures prüft:

  • physics_test_overlap
