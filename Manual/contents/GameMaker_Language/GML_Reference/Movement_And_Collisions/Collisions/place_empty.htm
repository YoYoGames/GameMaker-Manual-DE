place_empty

Mit dieser Funktion kannst du prüfen, ob die aufrufende Instanz mit einer anderen Instanz eines Objekts oder mit allen Instanzen in deinem Spiel kollidieren würde. Damit die Kollision korrekt aufgelöst werden kann, muss die Instanz, die die code ausführt, eine gültige Kollisionsmaske haben (entweder für die sprite selbst, oder durch die mask_index) und sie wird nur Kollisionen mit den Instanzen registrieren, die ebenfalls eine gültige Maske haben.

Die Funktion prüft, ob es keine Kollisionen gibt, wenn die aufrufende Instanz an einer bestimmten Position platziert wird. Sie können eine optionale argument angeben, um die Prüfung zu verfeinern und nur zu prüfen, ob eine Position frei von Kollisionen mit Instanzen des angegebenen Typs ist. Beachte, dass wenn keine optionale Objekt-ID angegeben wird, die Prüfung für alle Instanzen innerhalb des Raumes durchgeführt wird. Die Kollisionsprüfung ist entweder präzise oder basiert auf der Bounding Box der Instanz, abhängig davon, welche Art von Kollisionsmaske ausgewählt wurde, aber damit präzise Kollisionen korrekt funktionieren, sollten alle Instanzen in der Prüfung präzise Kollisionsmasken haben.

Beachten Sie, dass die angegebenen x/y-Koordinaten vor der Prüfung auf die nächstliegende ganze Zahl heruntergerechnet werden.

Syntax:

place_empty(x, y, [object_id]);

Argument Typ Beschreibung
x Real The x position to check.
y Real The y position to check.
[object_id] Objekt Anlage OPTIONAL Das Objekt, gegen das geprüft werden soll.

Rückgabe:

Boolean

Beispiel:

if place_empty(mouse_x, mouse_y, obj_Enemy)
{
    x = mouse_x;
    y = mouse_y;
}

Die obige code prüft, ob eine Kollision mit einer anderen Instanz des Objekts "obj_Enemy" vorliegt, als ob die aufrufende Instanz an der gleichen Position wie die Maus platziert wäre. Wenn keine Kollision festgestellt wird, werden die x/y-Koordinaten der Instanz auf die der Maus gesetzt.
