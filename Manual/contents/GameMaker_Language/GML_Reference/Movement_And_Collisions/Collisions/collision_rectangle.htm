kollision_rechtwinklig

Collision_rectangle verwendet die ersten vier arguments (x1,y1,x2,y2), um einen Bereich innerhalb des aktuellen Raums zu definieren und prüft dann, ob irgendein Objekt, das durch "obj" argument definiert ist, mit diesem Bereich kollidiert. Diese Kollision kann als präzise oder nicht präzise geprüft werden, und du kannst auch wählen, ob du die Instanz, auf der code läuft, selbst prüfen willst oder nicht. Betrachten Sie dieses Bild:

Hier verwendet die Instanz in der Mitte ein Kollisionsrechteck, um nach Kugelobjekten zu suchen. Die blauen Bälle haben keinen genauen Begrenzungsrahmen, und wie Sie sehen können, wird die Kollision auch dann stattfinden, wenn sprite das Rechteck nicht berührt (selbst wenn Sie die Option precise in der Funktion auf true setzen), da der Begrenzungsrahmen von sprite das collision_rectangle überlappt. Andererseits werden die grünen Kugeln nur dann als kollidierend betrachtet, wenn die aktuelle sprite das Rechteck überlappt. Damit präzise Kollisionen berücksichtigt werden können, müssen sowohl das Objekt sprite als auch die Kollisionsfunktion als präzise markiert sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, die als in Kollision befindlich betrachtet werden.

Beachten Sie, dass Sie anstelle eines Objektindex eine Instanz angeben können id angeben können, um nach einer bestimmten Instanz zu suchen, oder die Instanz-Schlüsselwörter all oder other (je nach Ereignis und aktuellem Bereich code ).

Syntax:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

Argument Typ Beschreibung
x1 Real The x coordinate of the left side of the rectangle to check.
y1 Real The y coordinate of the top side of the rectangle to check.
x2 Real The x coordinate of the right side of the rectangle to check.
y2 Real The y coordinate of the bottom side of the rectangle to check.
obj Objekt Asset- oder Instanz-ID The object to check for instance collisions.
prec Boolesche Whether the check is based on precise collisions (true(was langsamer ist) oder seine Bounding Box im Allgemeinen (false, schneller).
notme Boolesche Whether the calling instance, if relevant, should be excluded (true) oder nicht (false).

Rückgabe:

Instance ID or noone

Beispiel:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
{
    with (inst) instance_destroy();
}

Diese kurze code benutzt collision_rectangle, um einen Bereich im Raum von 50x, 50y (oben links im Rechteck) bis 200x, 200y (unten rechts im Rechteck) auf eine Instanz eines Objekts namens "obj_ball" zu überprüfen. Der Rückgabewert wird in einer temporären Variable gespeichert, die dann daraufhin überprüft wird, ob es sich bei diesem Wert um eine Instanz-ID oder um das Schlüsselwort niemand. Wenn dies nicht der Fall ist, wird die gespeicherte Instanz-ID verwendet, um das Objekt zu zerstören.
