kollision_ellipse

Collision_ellipse verwendet die ersten vier arguments (x1,y1,x2,y2), um die Breite und Höhe einer Ellipse innerhalb des aktuellen Raums zu definieren und prüft dann, ob irgendein Objekt, das durch "obj" argument definiert ist, mit diesem Bereich kollidiert. Diese Kollision kann als präzise oder nicht präzise geprüft werden, und du kannst auch wählen, ob die Instanz, auf der die code läuft, geprüft werden soll oder nicht. Betrachten Sie dieses Bild:

Hier verwendet die Instanz in der Mitte eine Kollisionsellipse, um nach Ballobjekten zu suchen. Die blauen Bälle haben keine präzise Bounding Box, und wie du sehen kannst, kann die Kollision auch dann stattfinden, wenn sprite die Ellipse nicht berührt (selbst wenn du die präzise Option in der Funktion auf true setzt), da die Bounding Box von sprite den elliptischen Bereich überlappt, der durch collision_circle definiert ist. Andererseits werden die grünen Kugeln nur dann als kollidierend betrachtet, wenn die tatsächliche sprite die definierte Ellipse überschneidet. Damit präzise Kollisionen berücksichtigt werden, müssen sowohl das Objekt sprite als auch die Kollisionsfunktion als präzise markiert sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, die als in Kollision befindlich betrachtet werden.

Beachten Sie, dass Sie anstelle eines Objektindex eine Instanz angeben können id angeben können, um nach einer bestimmten Instanz zu suchen, oder die Instanz-Schlüsselwörter all oder other (je nach Ereignis und aktuellem Bereich code ).

Syntax:

collision_ellipse(x1, y1, x2, y2, obj, prec, notme);

Argument Typ Beschreibung
x1 Real The x coordinate of the left side of the ellipse to check.
y1 Real The y coordinate of the top side of the ellipse to check.
x2 Real The x coordinate of the right side of the ellipse to check.
y2 Real The y coordinate of the bottom side of the ellipse to check.
obj Objekt Asset- oder Instanz-ID The object to check for instance collisions.
prec Boolesche Whether the check is based on precise collisions (true(was langsamer ist) oder seine Bounding Box im Allgemeinen (false, schneller).
notme Boolesche Whether the calling instance, if relevant, should be excluded (true) oder nicht (false).

Rückgabe:

Instance ID or noone

Beispiel:

if collision_ellipse(50, 50, 200, 100, obj_Player, false, true)
{
    instance_create_layer(obj_Player.x, obj_Player.y, "Effects", obj_Splash);
}

Dies wird eine elliptische Zone innerhalb der Grenzen von 50x, 50y und 200x, 100y für das Objekt "obj_Player" überprüfen. Wenn es eine Kollision mit diesem Objekt gibt, dann wird eine Instanz von "obj_Splash" an den x/y-Koordinaten von obj_Player erstellt.
