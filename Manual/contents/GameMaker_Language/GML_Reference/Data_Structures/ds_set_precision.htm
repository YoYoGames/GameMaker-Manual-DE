ds_set_precision

Beim Vergleich von Werten, z. B. beim Suchen in einer Karte oder beim Sortieren einer Liste, muss GameMaker entscheiden, wann zwei Werte gleich sind. Bei Strings und Integer-Werten ist das klar, aber bei reellen Zahlen können aufgrund von Fließkomma-Rundungsfehlern aus scheinbar gleichen Zahlen leicht ungleiche werden. Zum Beispiel ist es möglich, dass (5 / 3) * 3 nicht gleich 5 ist! Um dies zu vermeiden, wird bei allen Funktionen für reelle Zahlen ein Präzisionswert verwendet, und wenn die Differenz zwischen zwei Zahlen kleiner als diese Präzision ist, werden sie als gleich angesehen. Die Standardpräzision von 0,0000001 wird für alle Datenstrukturfunktionen verwendet, sofern sie nicht durch diese Funktion geändert wird.

HINWEIS: Diese Genauigkeit wird in allen Datenstrukturen verwendet, nicht aber bei anderen Vergleichen in GML!

Syntax:

ds_set_precision(prec);

Argument Typ Beschreibung
prec Real The precision value (default 0.0000001)

Rückgabe:

N/A

Beispiel:

ds_set_precision(0.0001);

Die obige code ändert die Standardpräzisionseinstellung für alle Datenstrukturfunktionen.
