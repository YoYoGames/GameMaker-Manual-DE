ds_list_destroy

Diese Funktion entfernt die gegebene Listendatenstruktur aus dem Speicher, gibt die Ressourcen frei, die sie verwendet hat und entfernt alle Werte, die sie enthielt. Diese Funktion sollte immer verwendet werden, wenn Sie die Verwendung von ds_list beendet haben, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und zum Absturz bringen können. Beachten Sie, dass das Zerstören einer Liste alle in ihr gespeicherten Datenstrukturen de-referenziert, was zu einem Speicherleck führt. Das einzige Mal, dass dies nicht erforderlich ist, ist, wenn Sie Elemente in der Liste als eine andere DS-Liste oder als eine DS-Karte gekennzeichnet haben, in diesem Fall werden diese Elemente zerstört und ihr Speicher wird automatisch bereinigt.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein ganzzahliger Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, so dass ein zerstörter Indexwert einer Datenstruktur anschließend von einer neu erstellten verwendet werden kann. Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach der Zerstörung immer auf -1 zu setzen.

Syntax:

ds_list_destroy(id);

ArgumentTyp Beschreibung
id The id of the data structure to remove.

Rückgabe:

Beispiel:

if (lives == 0)
{
    ds_list_destroy(AI_list);
    AI_list = -1;
    room_goto(rm_Menu);
}

Die obige code prüft den Wert der eingebauten globalen Variable "lives", und wenn er 0 ist, wird die DS-Liste, die in der Variablen "AI_list" indiziert ist, zerstört und dann die Räume gewechselt.
