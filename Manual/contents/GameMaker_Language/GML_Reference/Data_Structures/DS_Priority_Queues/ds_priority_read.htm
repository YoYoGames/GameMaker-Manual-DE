ds_prioritaet_lesen

Mit dieser Funktion können Sie eine gespeicherte DS-Priorität wiederherstellen (eine, die zuvor als Zeichenkette mit ds_priority_write()). Sie müssen zuerst eine neue DS-Priorität erstellen, um die Zeichenkette einzulesen. Wenn die DS-Priorität bereits existiert und Informationen darin gespeichert sind, werden diese vor dem Einlesen gelöscht. Diese Funktion ist von entscheidender Bedeutung bei der Erstellung von Speicher-/Lademechanismen für Ihr Spiel.

Beachten Sie, dass Sie, wenn die angegebene DS-Zeichenkette von GameMaker geschrieben wurde: Studio 1.2.x runtime (oder älter) geschrieben wurde, sollten Sie das optionale argument "legacy" angeben und es auf true setzen, da sich das String-Format danach geändert hat.

Syntax:

ds_priority_read(id, str, [legacy]);

Argument Typ Beschreibung
id DS-Prioritäts-ID The id of the data structure to write.
str Zeichenfolge The string to read from.
legacy Boolesche OPTIONAL Kann entweder true oder false sein oder ganz weggelassen werden.

Rückgabe:

N/A

Beispiel:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
    ds_priority_read(p_queue, str);
}
ini_close();

Die obige code erstellt eine Prioritätswarteschlange und speichert den Index in der Variablen "p_queue". Dann wird eine Datei ini geöffnet und eine Zeichenkette daraus gelesen, wobei zunächst geprüft wird, ob die Zeichenkette nicht als leer zurückgegeben wird. Diese Zeichenkette wird dann in die neu erstellte DS-Priorität eingelesen.
