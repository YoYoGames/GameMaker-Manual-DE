ds_priority_destroy

Diese Funktion entfernt die gegebene Datenstruktur der Prioritätswarteschlange aus dem Speicher, gibt die Ressourcen frei, die sie verwendet hat, und entfernt alle Werte, die sie enthielt. Diese Funktion sollte immer verwendet werden, wenn Sie die DS-Prioritätswarteschlange nicht mehr verwenden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und zum Absturz bringen können.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein ganzzahliger Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, so dass ein zerstörter Indexwert einer Datenstruktur anschließend von einer neu erstellten verwendet werden kann. Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach der Zerstörung immer auf -1 zu setzen.

Syntax:

ds_priority_destroy(id);

ArgumentTyp Beschreibung
id The id of the data structure to remove.

Rückgabe:

Beispiel:

if (lives == 0)
{
    ds_priority_destroy(AI_queue);
    AI_queue = -1;
    room_goto(rm_Menu);
}

Die obige code prüft den Wert der eingebauten globalen Variable "lives", und wenn er 0 ist, wird die DS-Prioritätswarteschlange, die in der Variablen "AI_queue" indiziert ist, zerstört und dann die Räume gewechselt.
