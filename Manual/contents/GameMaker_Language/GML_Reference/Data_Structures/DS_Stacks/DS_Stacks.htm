DS-Stapel

Ein DS-Stapel ist eine so genannte Last-in-First-out (LIFO)-Datenstruktur, ähnlich einer DS-Warteschlange. Man kann Werte auf einen Stapel schieben und sie dann wieder entfernen, indem man sie vom Stapel entnimmt, wobei der Wert, der zuletzt auf den Stapel geschoben wurde, als erster wieder vom Stapel entnommen wird (man denke nur an einen Münzstapel, bei dem jede Münze, die man hinzufügt, erst wieder entfernt werden muss, bevor man an die restlichen Münzen darunter gelangt). Stapel werden häufig verwendet, wenn es um Unterbrechungen geht oder um rekursive Funktionen oder sogar um eine rudimentäre KI für Ihre Spiele.

HINWEIS: Wie alle dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die das Spiel verlangsamen und schließlich zum Absturz bringen.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur existiert, können Sie die ds_exists() Funktion verwenden.

Es gibt die folgenden Funktionen, die sich mit DS-Stapeln befassen:

  • ds_stack_create
  • ds_stack_destroy
  • ds_stack_clear
  • ds_stack_empty
  • ds_stack_size
  • ds_stack_copy
  • ds_stack_top
  • ds_stack_pop
  • ds_stack_push
  • ds_stack_read
  • ds_stack_write
