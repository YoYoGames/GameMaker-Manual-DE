ds_stack_read

Mit dieser Funktion können Sie einen gespeicherten DS-Stapel wiederherstellen (einen, der zuvor als String mit ds_stack_write()). Sie müssen zuerst einen neuen DS-Stapel erstellen, um die Zeichenkette einzulesen. Wenn der DS-Stapel bereits existiert und Informationen darin gespeichert sind, werden diese vor dem Lesen gelöscht. Diese Funktion ist von entscheidender Bedeutung bei der Erstellung von Speicher-/Lademechanismen für Ihr Spiel.

Beachten Sie, dass Sie, wenn die angegebene DS-Zeichenkette von GameMaker geschrieben wurde: Studio 1.2.x runtime (oder älter) geschrieben wurde, sollten Sie das optionale argument "legacy" angeben und es auf true setzen, da sich das String-Format danach geändert hat.

Syntax:

ds_stack_read(id, str [, legacy]);

Argument Typ Beschreibung
id DS-Stapel-ID The id of the data structure to read into.
str Zeichenfolge The string to read from.
legacy Boolesche OPTIONAL Kann entweder true oder false sein oder ganz weggelassen werden.

Rückgabe:

N/A

Beispiel:

stack = ds_stack_create();
ini_open("save.ini");
var str = ini_read_string("Stacks", "0", "");
if (str != "")
{
    ds_stack_read(stack, str);
}
ini_close();

Die obige code erstellt einen Stapel und speichert den Index in der Variablen "stack". Dann wird eine Datei ini geöffnet und eine Zeichenkette daraus gelesen, wobei zunächst geprüft wird, ob die Zeichenkette nicht als leer zurückgegeben wird. Diese Zeichenkette wird dann in den neu erstellten DS-Stapel eingelesen.
