ds_map_destroy

DS-Karten benötigen Speicherplatz, der ihnen bei ihrer Erstellung zugewiesen wird. Das bedeutet, dass Sie diesen Speicher auch wieder freigeben müssen, wenn die DS-Map nicht benötigt wird, um Fehler, Speicherlecks und Leistungseinbußen bei der Ausführung Ihres Spiels zu vermeiden. Diese Funktion erledigt genau das. Beachten Sie, dass das Zerstören einer Karte alle Datenstrukturen, die in der Karte gespeichert sind, de-referenziert, was zu einem Speicherleck führt. Sie müssen also die Karte durchgehen und alle Datenstrukturelemente manuell zerstören, bevor Sie sie zerstören, um dies zu verhindern. Die Dies ist nur dann nicht erforderlich, wenn Sie Elemente in der Map als DS-Liste oder als eine andere DS-Map gekennzeichnet haben; in diesem Fall werden diese Elemente zerstört und ihr Speicher auch automatisch gelöscht.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein ganzzahliger Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können. Im Zweifelsfall sollten Sie daher den ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, so dass ein zerstörter Indexwert einer Datenstruktur anschließend von einer neu erstellten verwendet werden kann. Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach der Zerstörung immer auf -1 zu setzen.

Syntax:

ds_map_destroy(id);

ArgumentTyp Beschreibung
id The id of the map to destroy.

Rückgabe:

Beispiel:

ds_map_destroy(inventory);
inventory = -1;

Die obige code wird die DS-Map mit der in der Variablen "inventory" indizierten ID zerstören.
