ds_map_secure_save

Diese Funktion speichert den Inhalt der angegebenen DS-Map in einer Datei, die mit dem Gerät verknüpft ist, auf dem sie erstellt wurde (d.h. sie kann nicht gelesen werden, wenn sie auf ein anderes Gerät übertragen wird). Die Datei selbst kann fast jede beliebige Erweiterung haben (z.B. *.dat, *.json, *.bin, usw...) und wird verschleiert und im lokalen Speicher des Zielgeräts platform gespeichert. Sie können dann die ds_map mit der folgenden Funktion neu laden ds_map_secure_load(). Beachten Sie, wenn die zu speichernde DS-Map ein Array enthält, wird dieses Array beim Speichern stattdessen in eine DS-Liste umgewandelt.

WICHTIG! Eine der Eigenschaften einer sicher gespeicherten Datei ist, dass sie an das Gerät gebunden ist, auf dem sie erstellt wurde. Sie können also eine auf einem Gerät gespeicherte Datei nicht in ein Projekt laden, das auf einem anderen Gerät läuft.

Syntax:

ds_map_secure_save(map, filename);

ArgumentTyp Beschreibung
map The id of the data structure to use
filename The name of the file to save the map to

Rückgabe:

Beispiel:

ds_map_secure_save(purchase_map, "p_data.dat");

Die obige code speichert die in der Variablen "p_data" indizierte DS-Map in der angegebenen Datei zum späteren Abruf.
