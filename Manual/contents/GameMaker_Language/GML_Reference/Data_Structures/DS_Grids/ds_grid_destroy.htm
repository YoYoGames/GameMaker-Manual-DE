ds_grid_destroy

Diese Funktion entfernt die angegebene Datenstruktur des Gitters aus dem Speicher, gibt die verwendeten Ressourcen frei und entfernt alle Werte, die sie enthielt. Diese Funktion sollte immer verwendet werden, wenn Sie die Verwendung des DS-Gitters beendet haben, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und zum Absturz bringen können.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein ganzzahliger Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, so dass ein zerstörter Indexwert einer Datenstruktur anschließend von einer neu erstellten verwendet werden kann. Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach der Zerstörung immer auf -1 zu setzen.

Syntax:

ds_grid_destroy(index);

ArgumentTyp Beschreibung
index This index of the grid to destroy.

Rückgabe:

Beispiel:

if (lives == 0)
{
    ds_grid_destroy(Wall_Grid);
    Wall_Grid = -1;
    room_goto(rm_Menu);
}

Die obige code prüft den Wert der eingebauten globalen Variable "lives" und wenn er 0 ist, wird das DS-Gitter, das in der Variable "Wall_Grid" indiziert ist, zerstört und dann die Räume gewechselt.
