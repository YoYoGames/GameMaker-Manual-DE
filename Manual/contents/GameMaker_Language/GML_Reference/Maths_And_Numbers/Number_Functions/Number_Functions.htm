Zahlenfunktionen

Reelle Zahlen gelten in GameMaker als doppelgenaue Fließkommazahlen - das heißt, sie haben einen Dezimalpunkt und keine feste Anzahl von Vor- oder Nachkommastellen - oder als Ganzzahlen - das heißt, sie sind ganze Zahlen ohne Dezimalpunkt. 2 zum Beispiel ist eine ganze Zahl, aber 2,01 ist eine Fließkommazahl.

HINWEIS: Auf dem HTML5-Ziel sind alle reellen Zahlen verdoppelt.

Diese Unterscheidung zwischen Ganzzahlen und Fließkommazahlen ist bei der Entwicklung von platform sehr wichtig, da die Genauigkeit von Berechnungen, die auf einem PC ( Windows ) durchgeführt werden, nicht dieselbe ist wie die Genauigkeit derselben Berechnungen, wenn sie auf einem mobilen Gerät durchgeführt werden. Das bedeutet, dass Sie zum Beispiel bei Vergleichen besonders aufmerksam sein sollten:

if (image_index == 1)
{
    do something...
}

Wenn wir im obigen Beispiel den Wert image_speed beispielsweise auf 0,25 gesetzt haben, können Sie nach vier Schritten davon ausgehen, dass der Wert image_index 1 sein würde, aber aufgrund der Funktionsweise der Fließkommamathematik kann es sich um einen Wert wie 1,0000002 handeln, so dass die Auswertung nicht true sein wird. Diese Art von Fehlern kann ziemlich schwer zu beheben sein, und daher ist es immer gut, sich dessen bewusst zu sein und im Voraus zu planen und andere Mittel zur Überprüfung von Werten zu verwenden oder die entsprechenden Boden- oder Rundungsfunktionen (unten aufgeführt) zu verwenden, um die zu prüfende Zahl in eine ganze Zahl umzuwandeln (weitere Informationen zur Fließkommamathematik und warum dies ein Problem ist, finden Sie hier). Zum Beispiel könnte die obige code geschrieben werden als:

if (floor(image_index) == 1)
{
    do something...
}

Es ist auch erwähnenswert, dass bei der Verwendung der YoYo Compiler Targets alle Ausdrücke und Funktionen von links nach rechts ausgewertet werden, während sie bei allen anderen Targets platforms von rechts nach links ausgewertet werden, was bedeutet, dass dies - zum Beispiel - je nach Plattform zu unterschiedlichen Ergebnissen führen wird:

val = max(num, ++num, num++);

HINWEIS: Weitere Informationen finden Sie im Abschnitt über den Bewertungsauftrag.

Sie können auch eine spezielle Funktion im GameMaker verwenden, um den Epsilon-Wert für die Fließkomma-Mathematik einzustellen. Wenn eine reelle Zahl auf die nächste Fließkommazahl gerundet wird, bildet der Epsilon-Wert (auch "Maschinen-Epsilon" genannt) eine Obergrenze für den relativen Fehler, und Sie können den Epsilon-Wert mit den folgenden Funktionen ermitteln und einstellen:

  • math_set_epsilon
  • math_get_epsilon

HINWEIS Wenn Sie die Zufallsfunktionen verwenden, behält GameMaker jedes Mal, wenn Sie das Spiel starten, denselben Zufallswert bei. Das macht debugging sehr viel einfacher, da Sie sicher sein können, dass die Zufallsfunktionen anfangs immer denselben Wert zurückgeben. Wenn Sie dies jedoch nicht wünschen, müssen Sie zu Beginn des Spiels einen neuen Zufallswert setzen, entweder mit randomise() oder random_set_seed().

  • choose
  • random
  • random_range
  • irandom
  • irandom_range
  • random_set_seed
  • random_get_seed
  • randomise
