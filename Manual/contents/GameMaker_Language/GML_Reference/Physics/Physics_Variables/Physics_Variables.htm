Physik-Variablen

Wie das "traditionelle" Kollisions- und Bewegungssystem verfügt auch das Physiksystem über eine Reihe von Instanzvariablen, die jeder Instanz eines Objekts bei dessen Erstellung zugewiesen werden. Einige dieser Eigenschaften können vom Benutzer in GML definiert werden, aber viele sind schreibgeschützt und noch mehr sind nur im Kollisionsereignis anwendbar.

Die folgenden Variablen sind in allen physikfähigen Objekten vorhanden und können mit den Funktionen code from instances und script gelesen und geschrieben (d.h. geändert) werden:

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

Bestimmte Variablen sind schreibgeschützte Variablen, d.h. die Informationen, die sie enthalten, können nicht über die Instanz code oder die Funktion script geändert werden (sie können aber intern von GameMaker geändert werden, weshalb es sich um Variablen und nicht um Konstanten handelt). Die folgende Liste zeigt die schreibgeschützten Variablen, die in allen physikfähigen Objekten existieren:

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

Schließlich gibt es eine Reihe von temporären Variablen, die in physikfähigen Objekten existieren und nur beim Kollisionsereignis erstellt werden. Diese Variablen sind wie die oben aufgelisteten und können nur gelesen werden:

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
