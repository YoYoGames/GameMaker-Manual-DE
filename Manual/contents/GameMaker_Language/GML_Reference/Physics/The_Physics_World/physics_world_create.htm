physik_welt_erschaffen

Selbst wenn Sie die Physik für Ihre Objekte definiert haben, werden sie ohne eine Welt, in der sie platziert werden, standardmäßig das traditionelle GameMaker-Kollisionssystem verwenden. Daher benötigen Sie diese Funktion, um das Physiksystem für den angegebenen Raum zu erstellen, entweder in einem Objektereignis oder bei der Erstellung code des Raums selbst (es kann auch ohne code im Raumeditor definiert werden). Diese Funktion assoziiert eine Physik-"Welt" mit dem Raum und alle Instanzen, die in dem Raum erstellt werden, verhalten sich mit der Physik, die für sie und die Welt selbst definiert wurde.

Sie werden feststellen, dass wir eine argument haben, die ein Verhältnis zwischen Pixelgröße und Metern definiert. Das liegt daran, dass die Physikfunktionen mit realen Maßen arbeiten und wir daher Pixelkoordinaten und Pixelgrößen mit Hilfe eines "Pixel-Meter-Maßstabs" in diese realen Maße übersetzen müssen. Sie sollten die Einstellung "Pixel-Meter-Skala" so lange anpassen, bis die durchschnittliche Pixelgröße der von Ihnen verwendeten Objekte ungefähr in simulierte Physikobjekte einer angemessenen Größe übersetzt wird. Wie Sie oben sehen können, haben wir 32 Pixel eingestellt, was einem Meter entspricht. Es sollte beachtet werden dass das Physiksystem sowohl mit größeren als auch mit kleineren Verhältnissen funktioniert, dass aber die beste Leistung erzielt wird, wenn die realen Abmessungen Ihrer Objekte nicht weniger als 0,1 Meter und nicht mehr als 50 Meter betragen.

Syntax:

physics_world_create(pixeltometrescale);

ArgumentTyp Beschreibung
pixeltometrescale defines the ratio of pixels on screen to metres in the real world. A ratio of 32:1 will be specified as 1/32 (or 0.03125).

Rückgabe:

Beispiel:

physics_world_create(1/32);

Die obige code erstellt eine physikalische Welt im aktuellen Raum mit einem Maßstabsverhältnis von 32 Pixeln, was 1 Meter entspricht.
