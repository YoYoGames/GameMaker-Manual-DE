physics_world_update_speed

GameMaker aktualisiert die Dinge auf der Grundlage der Raumgeschwindigkeit, d. h. wenn Sie diese auf z. B. 30 einstellen, führt GameMaker 30 Schritte im Laufe einer Sekunde aus. Für die Physikfunktionen kann das jedoch nicht ausreichen, und man möchte vielleicht, dass die Dinge mit einer etwas schnelleren Geschwindigkeit aktualisiert werden, um die Stabilität oder Präzision zu erhöhen. Zu diesem Zweck verwenden wir die Funktion physics_world_update_speed(), die die Aktualisierungsgeschwindigkeit für das Physiksystem unabhängig von der Raumgeschwindigkeit einstellt. Das bedeutet, dass Sie eine Raumgeschwindigkeit von 30 haben können, aber die Physik auf 60 einstellen können, wodurch sich die Geschwindigkeit, mit der das Physiksystem aktualisiert und seine Berechnungen durchführt, im Vergleich zu der Geschwindigkeit, mit der die Schritte aktualisiert werden, effektiv verdoppelt.

HINWEIS: Sie können derzeit keine geringere Geschwindigkeit als die Raumgeschwindigkeit einstellen, aber zukünftige Updates könnten dies ändern.

Syntax:

physics_world_update_speed(speed)

Argument Typ Beschreibung
speed Real The number of times per second that the physics system updates

Rückgabe:

N/A

Beispiel:

physics_world_update_speed(room_speed * 2);

Die obige code stellt die Aktualisierungsgeschwindigkeit des Physiksystems auf das Doppelte der Raumgeschwindigkeit ein.
