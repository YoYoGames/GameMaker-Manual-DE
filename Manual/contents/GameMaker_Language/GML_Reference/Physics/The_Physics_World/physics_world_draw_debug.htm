physics_world_draw_debug

Diese Funktion zeichnet eine Darstellung des physikalischen Weltsystems des Raums, um bei debugging zu helfen, wenn Sie Probleme haben, die Physikszene richtig zum Laufen zu bringen. Manchmal kann das Problem im Rendering-Teil des Spiels liegen, z.B. kann ein sprite an der falschen Position oder mit der falschen Drehung gezeichnet werden, was den Anschein einer falschen Physik erweckt, oder es kann sein, dass Ihre Berechnungen nicht korrekt sind. Wie auch immer, diese Funktion wird Ihnen helfen, das Problem zu finden und zu lösen, indem sie aufzeigt, was in Ihrem Physiksystem vor sich geht. Dazu müssen Sie ihr mitteilen, welcher Teil des Systems gezeichnet werden soll. Dies geschieht durch die Übergabe eines Wertes, der aus den in verschiedenen "Flag"-Konstanten gespeicherten Bits erstellt wird:

Physik-Debug-Flag
Flagge Beschreibung
phy_debug_render_aabb Dies zeigt den absoluten Begrenzungsrahmen jedes Geräts in Bezug auf die Raumachse
phy_debug_render_collision_pairs Dies zeigt alle Vorrichtungen an, die derzeit in Kollision sind
phy_debug_render_coms Dies markiert den Schwerpunkt der einzelnen Leuchten im Raum.
phy_debug_render_core_shapes Zeigt die Grundformen, aus denen die Einrichtungsgegenstände im Raum bestehen
phy_debug_render_joints Dadurch werden alle Verbindungen aller Einrichtungsgegenstände im Raum gezeichnet
phy_debug_render_obb Dies zeigt den relativen Begrenzungsrahmen für die Einrichtungsgegenstände im Raum
phy_debug_render_shapes Dies zeigt die tatsächlichen Formen, aus denen die Einrichtungsgegenstände im Raum bestehen

Um also zwei dieser Optionen zu zeichnen, müssen wir ein bitweises "oder" (in GameMaker wird dies durch | dargestellt) mit den beiden Optionen durchführen und die resultierende Variable für die Funktion speichern. Hier ist ein Beispiel dafür, wie man eine Variable für die Funktion setzt, die nur die Massenschwerpunkte und die Gelenke aller Einrichtungsgegenstände in einem Raum zeichnet:

flag = phy_debug_render_coms | phy_debug_render_shapes;

Das würde in etwa Folgendes ergeben:

Wie Sie in der obigen Abbildung code sehen können, werden nur die Formen und Massenschwerpunkte auf dem Bildschirm gezeichnet (sowie die Farben, die ihre Zustände darstellen - weitere Informationen finden Sie in der Online-Dokumentation zu Box 2D). Auf die gleiche Weise können Sie mehr oder sogar alle Konstanten anzeigen, die Sie untersuchen möchten... Verknüpfen Sie sie einfach mit "oder"! Eine weitere Sache sollte beachtet werden - diese Funktion sollte nur im Draw-Ereignis einer Instanz aufgerufen werden, die eine geringere Tiefe hat als die, die Sie debuggen wollen, sonst können Sie die Debug-Informationen nicht sehen.

Syntax:

physics_world_draw_debug(flag)

Argument Typ Beschreibung
flag Physik-Debug-Flag(s) The flag value for showing different debug features

Rückgabe:

N/A

Beispiel:

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

Die obige code zeichnet alle absoluten Bounding Boxen, die relativen Bounding Boxen sowie die Fixtures in Kollision innerhalb des aktuellen Physiksystems.
