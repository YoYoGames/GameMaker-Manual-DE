physik_welt_gravitation

Wie bei allen Physikfunktionen funktioniert physics_world_gravity() nur dann auf einer Instanz, wenn für den Raum eine Physikwelt im Raumeditor definiert wurde oder mit physics_world_create(). Es wird verwendet dann den 0,0 definierenden Punkt der Physikwelt, um die Richtung und Stärke der Schwerkraft auf der Grundlage von vector zu berechnen, die aus den Eingaben xg und yg berechnet wird. Hier sind einige Bilder zur Veranschaulichung...

Wie Sie sehen können, sind die xg,yg-Koordinaten relativ zur Position 0,0, wie sie für definiert, und die Schwerkraft selbst wird als vector dieser beiden Komponenten berechnet. Die Standardwerte von 0,10 ergeben also eine Gravitationsrichtung von unten (270 Grad) mit einer Beschleunigung von 10 Metern pro Sekunde zum Quadrat. Beachten Sie die Meter pro Sekunde zum Quadrat! Wie alle Physikfunktionen wird auch die Schwerkraft auf der Basis von Metern berechnet, daher ist der Pixel-zu-Meter-Maßstab, den Sie in der Funktion physics_world_create einstellen, hier wichtig. Die Schwerkraft wirkt sich auch kumulativ auf das Objekt aus (aber die genaue Menge hängt davon ab, was Sie für die Funktion eingestellt haben physics_fixture_set_linear_damping()), was bedeutet, dass in einem Raum mit einer room_speed von 60, ohne Dämpfung, die Instanz jede Sekunde um 10 Meter pro Sekunde beschleunigen wird - was bei einer Skala von 0,1 Pixel zu Meter dasselbe ist, wie wenn man sagt, dass sie in 60 Schritten auf eine Geschwindigkeit von 100 Pixel pro Sekunde beschleunigt.

Syntax:

physics_world_gravity(xg, yg)

ArgumentTyp Beschreibung
xg The x component of the gravity vector
yg The y component of the gravity vector

Rückgabe:

Beispiel:

physics_world_gravity(-5, 0);

Mit der obigen code wird die Schwerkraft (unter der Annahme, dass die Raumphysik 0,0 auf oben links eingestellt ist) nach rechts - 0 Grad - mit einer Beschleunigung von 5 Metern pro Sekunde im Quadrat eingestellt.
