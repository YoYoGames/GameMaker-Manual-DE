physics_joint_wheel_create

Ein Radgelenk kombiniert einfach einen Kolben und ein Drehgelenk, wie ein Rad, das auf dem Stoßdämpfer eines Autos montiert ist. Sie geben den Ankerpunkt für das Gelenk sowie die beiden physikfähigen Instanzen des Gelenks an. Die erste Instanz fungiert als "Körper" für das Gelenk, während die zweite Instanz das "Rad" ist und sich frei um die Gelenkachse bewegen kann. Sie können auch eine Achse vector festlegen, die die imaginäre Linie ist, entlang der das Gelenk als "Feder" fungiert und dem Rad erlaubt, entlang dieser Linie auf- und abzuspringen, wenn es auf Hindernisse trifft (ähnlich wie das Kolbengelenk). Wenn Sie einen Motor aktivieren, dreht sich die zweite Instanz um die Ankerposition, und Sie können das maximale Motordrehmoment einstellen, das verwendet wird, um die gewünschte Motorgeschwindigkeit (N/m) zu erreichen, sowie die Geschwindigkeit, mit der sich der Motor drehen soll. Da das Radgelenk auch eine Achse hat, entlang der es sich bewegen kann, können Sie die Schwingungsfrequenz (in Hz) sowie das Dämpfungsverhältnis für das Gelenk einstellen - möglicherweise müssen Sie mit diesen Werten spielen, um eine Feinabstimmung vorzunehmen, und es wird empfohlen, dass Sie mit kleineren Werten beginnen und diese erhöhen, bis Sie den gewünschten Effekt erzielen.

Wie bei allen Gelenken, wenn Sie die "col" Wert auf true dann die beiden Instanzen können interagieren und miteinander kollidieren, aber nur, wenn sie Kollisions-Ereignisse haben, aber wenn es auf false eingestellt ist, werden sie nicht kollidieren, egal was.

Syntax:

physics_joint_wheel_create(inst1, inst2, anchor_x, anchor_y, axis_x, axis_y, enableMotor, max_motor_torque, motor_speed, freq_hz, damping_ratio, col)

Argument Typ Beschreibung
inst1 Instanz-ID The first instance to connect with the joint
inst2 Instanz-ID The second instance to connect with the joint
anchor_x Real The x coordinate where the joint is anchored, within the game world
anchor_y Real The y coordinate where the joint is anchored, within the game world
axis_x Real The x component of the wheel axis vector
axis_y Real The y component of the wheel axis vector
enableMotor Boolesche Whether the motor should be active (true) or not (false)
max_motor_torque Real Sets the maximum motor torque used to achieve the desired motor speed (in Newtons per meter)
motor_speed Real This is the speed at which the motor should rotate
freq_hz Real This is the oscillation frequency for the joint, in hertz
damping_ratio Real This damping ratio for the joint
col Farbe Whether the two instances can collide (true) or not (false)

Rückgabe:

Physics Joint ID

Beispiel:

var mainFixture = physics_fixture_create();
physics_fixture_set_box_shape(mainFixture, 128, 32);
var wheelFixture = physics_fixture_create();
physics_fixture_set_circle_shape(wheelFixture, 32);
var o_id = instance_create_layer(x+25, y + 100, "Instances", obj_Wheel);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(wheelFixture, o_id);
physics_joint_wheel_create(id, o_id, o_id.x, o_id.y, 0, 1, true, 10, 1, 15, 6, false);
physics_fixture_delete(mainFixture);
physics_fixture_delete(wheelFixture);

Die obige code erstellt und definiert zwei neue Einbauteile. Diese werden dann an die aufrufende Instanz und eine neue Instanz gebunden, die erstellt wird, bevor ein Radgelenk angewendet wird, um sie zu verbinden, wonach die Einbauten aus dem Speicher gelöscht werden.
