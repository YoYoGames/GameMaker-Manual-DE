physics_fixture_set_kinematic

In einem physikbasierten Spiel gibt es bestimmte Situationen, in denen Sie sich bewegen möchten, aber nicht möchten, dass Kräfte wie die Schwerkraft oder Kräfte, die durch Kollisionen mit dynamischen Objekten entstehen, auf Sie einwirken (denken Sie z. B. an die Bewegung von platforms in einem platform Spiel). zum Beispiel). Wenn Sie bei solchen Objekten einfach die Dichte auf 0 setzen, geht die Physik davon aus, dass das Objekt statisch sein soll. Sollten Sie jedoch den Wert phy_speed_x, phy_speed_y oder phy_angular_velocity Variablen für eine Instanz setzen, an die ein solches Fixture gebunden wurde, wird es in ein in ein kinematisches Objekt umgewandelt und beginnt sich zu bewegen. Die Änderung der Instanz von statisch zu kinematisch mitten in der Simulation kann sehr aufwendig sein (abhängig vom Zustand der Instanzen in der Physikwelt), daher wird empfohlen, die kinematisch zu machen, bevor es an eine Instanz gebunden wird, so dass es von Kollisionen und Kräften (wie der Schwerkraft) unbeeinflusst bleibt, sich aber mit konstanter linearer und/oder Winkelgeschwindigkeit bewegen kann.

Syntax:

physics_fixture_set_kinematic(fixture)

ArgumentTyp Beschreibung
fixture the index of the fixture

Rückgabe:

Beispiel:

physics_fixture_set_kinematic(fix_Cloud);

Die obige code zeigt an, dass die in "fix_Cloud" indizierte Halterung als kinematisch markiert werden sollte.
