physik_teilchen_erzeugen

Mit dieser Funktion können Sie ein einzelnes Partikel an beliebiger Stelle in Ihrem Spielraum erstellen und dabei bestimmte flags und Eigenschaften einstellen. Die "Flags" sind der Rückgabewert einer Kombination aus den folgenden Konstanten:

Physik Teilchen Flagge Konstante
Konstante Beschreibung
phy_particle_flag_water Die Standardeigenschaften für einen Weichkörperpartikel.
phy_particle_flag_zombie Ein Zombiepartikel ist ein Partikel, der nach einem einzigen Schritt zerstört wird, während alle anderen auf diese Weise gekennzeichnet sind.
phy_particle_flag_wall Damit wird das Partikel als statisch definiert, was es zu einem unbeweglichen Objekt in der Physiksimulation macht, da es in einer festen Position bleibt, egal was mit ihm kollidiert. Sie sollten dieses Flag verwenden, anstatt die Dichte auf 0 zu setzen.
phy_particle_flag_spring Federpartikel erzeugen den Effekt, dass sie wie durch eine Feder miteinander verbunden sind. Teilchen, die mit dieser Flagge erzeugt werden, sind paarweise "verbunden", wobei jedes Teilchen mit demjenigen verbunden ist, das ihm zum Zeitpunkt der Erzeugung am nächsten war. Einmal gepaart, wechseln die Teilchen ihre "Partner" nicht mehr, und je weiter eine äußere Kraft sie voneinander wegzieht, desto größer ist die Kraft, mit der sie kollidieren, wenn diese äußere Kraft wegfällt. Beachten Sie, dass die Verbindung zwischen gepaarten Teilchen nicht abreißt, egal wie weit sie sich voneinander entfernen.
phy_particle_flag_elastic Elastische Partikel verformen sich und können auch abprallen, wenn sie mit anderen starren Körpern in der Physiksimulation zusammenstoßen.
phy_particle_flag_viscous Ein zähflüssiges Teilchen ist ein Teilchen, das "klebrig" ist, wie Öl. Viskose Partikel verklumpen und kleben stärker zusammen.
phy_particle_flag_powder Puderpartikel erzeugen einen Streueffekt, wie man ihn bei Sand oder Staub beobachten kann.
phy_particle_flag_tensile Zugpartikel werden verwendet, um den Effekt der Oberflächenspannung oder die gespannte Krümmung der Oberfläche einer Flüssigkeit zu erzeugen. Sie können zum Beispiel verwendet werden, um die Oberflächenspannung eines Wassertropfens zu erzeugen. Sobald die Spannung gebrochen ist, prallen die Teilchen ab, als ob sie elastisch wären, ziehen sich aber auch weiterhin gegenseitig an. Infolgedessen neigen die Teilchen dazu, beim Aufprall Cluster zu bilden.
phy_particle_flag_colourmixing Farbmischende Partikel nehmen einen Teil der Farbe anderer Partikel an, mit denen sie kollidieren. Wenn nur eines der beiden kollidierenden Teilchen ein farbmischendes Teilchen ist, behält das andere Teilchen seine Farbe vor dem Zusammenstoß.

Diese flags verwenden Bit-Maskierung, um einen endgültigen Ausgabewert zu erzeugen, der dann geprüft wird, um die verschiedenen grundlegenden Eigenschaften des Partikels einzustellen (wobei die Basiseigenschaft immer die von phy_particle_flag_water ist). Wenn Sie z. B. eine viskose Flüssigkeit mit Oberflächenspannung simulieren möchten, würden Sie die entsprechenden Bits bitweise oder mit "|" markieren, wie im folgenden Beispiel gezeigt. Auf diese Weise können Sie für jedes einzelne erzeugte Partikel unterschiedliche Eigenschaften (abgesehen von den globalen Eigenschaften) festlegen.

Neben flags können Sie auch die Position im Raum, an der das Partikel erstellt werden soll, die anfängliche horizontale und vertikale Geschwindigkeit, die anfängliche Farbe und Alpha sowie die Benutzerkategorie festlegen. Die Werte für Farbe und Alpha werden nur beim Zeichnen des Partikels mit der Funktion physics_particle_draw()verwendet, aber der Wert für die Kategorie wird in mehreren anderen Funktionen verwendet. Dieser Wert ist ein willkürlicher ganzzahliger Wert, den du jedem Partikel gibst, der ähnliche Eigenschaften haben soll, und er kann verwendet werden, um später flags für diese spezifische Kategorie sowie eine Reihe anderer Dinge einzustellen. Wenn Sie beispielsweise ein "Wasser"- und ein "Öl"-Partikel definiert haben, geben Sie allen Partikeln, die als "Wasser" gekennzeichnet sind, die Benutzerkategorie 1 und allen Partikeln, die als "Öl" erstellt wurden, die Benutzerkategorie 2. So können Sie später eine einzelne Kategorie anvisieren, um ihre flags zu ändern oder bestimmte Daten zu erhalten.

HINWEIS: Sie können jeden ganzzahligen Wert für die Kategorie verwenden , mit Ausnahme von 0, der von der Simulation reserviert ist und für die Auswahl aller Kategorien in anderen Funktionen (z. B. für das Zeichnen) verwendet wird.

Die Funktion gibt einen eindeutigen Indexwert (oder ID) für den Partikel zurück, der in einer Variablen gespeichert werden kann, um in anderen Funktionen, die direkt auf den Partikel verweisen, verwendet zu werden.

Syntax:

physics_particle_create(flags, x, y, xv, yv, col, alpha, category)

Argument Typ Beschreibung
flags Physik Teilchen Flagge Konstante(s) The flags to set on the particle.
x Real The x position to create the particle at.
y Real The y position to create the particle at.
xv Real The initial horizontal velocity.
yv Real The initial vertical velocity.
col Farbe The base colour to use for the particle.
alpha Real The base alpha to use for the particle.
category Real The user defined category to which the particle belongs.

Rückgabe:

Physics Particle ID

Beispiel:

var flags = phy_particle_flag_water | phy_particle_flag_viscous | phy_particle_flag_tensile;
var x_vel = lengthdir_x(5, image_angle);
var y_vel = lengthdir_y(5, image_angle);
var p = physics_particle_create(flags, x, y, x_vel, y_vel, c_white, 1, 1);

Die obige code erstellt Variablen, um den Partikeltyp und seine Anfangsgeschwindigkeit festzulegen, und verwendet diese dann, um ein Partikel an der x/y-Position der Instanz zu erstellen.
