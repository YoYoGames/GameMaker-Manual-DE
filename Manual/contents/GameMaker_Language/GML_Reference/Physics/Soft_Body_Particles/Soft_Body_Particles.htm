Weichkörperpartikel

GameMaker verwendet die Box2D Starrkörper-Physikbibliothek und erlaubt daher keine Weichkörperphysik. Allerdings fügt die Flüssiger Spaß Erweiterung fügt dem Physiksystem jedoch Partikel hinzu, die die Erstellung von simulierten Weichkörpern unter Verwendung der Starrkörperphysik ermöglichen. Grundsätzlich können Sie eine Form von Partikeln erstellen, die den Zusammenhalt aufrechterhalten und so einen vollen Körper simulieren, der aufgrund der Interaktion der Partikel den Eindruck eines weichen Körpers vermittelt. Sie können die Physikpartikel auch zur Simulation von Flüssigkeiten und anderen partikelbasierten Physiksystemen verwenden.

Bevor Sie mit der Erstellung Ihrer Physikpartikel beginnen, müssen Sie die grundlegenden Physikeigenschaften für diese Partikel festlegen, so wie Sie es auch mit den regulären Physikfunktionen tun würden. Beachten Sie jedoch, dass sich die Einstellung dieser Eigenschaften auf alle zuvor erstellten Partikel sowie auf alle neu erstellten Partikel auswirkt, so dass Sie diese Funktionen normalerweise nur zu Beginn des Spiels oder der Ebene verwenden. Es gibt jedoch spezielle Funktionen, um die Partikel-Flags für einzelne Partikel (und Partikelgruppen) zu setzen, die ihre Eigenschaften wie "Abprallverhalten" oder "Viskosität" ändern, so dass Sie verschiedene Partikeltypen innerhalb der Bereiche dieser globalen Einstellungen erstellen können.

Die folgenden Funktionen stehen zum Einstellen und Abrufen von globalen Partikelsystemdaten zur Verfügung:

  • physics_particle_count
  • physics_particle_group_count
  • physics_particle_set_max_count
  • physics_particle_set_radius
  • physics_particle_set_density
  • physics_particle_set_damping
  • physics_particle_set_gravity_scale
  • physics_particle_get_max_count
  • physics_particle_get_radius
  • physics_particle_get_density
  • physics_particle_get_damping
  • physics_particle_get_gravity_scale

Außerdem stehen Ihnen die folgenden Funktionen zur Verfügung, um einzelne Partikel (oder Gruppen von Partikeln) zu kennzeichnen und um Informationen wie Position oder Farbe von einem einzelnen Partikel oder einer einzelnen Gruppe zu erhalten:

  • physics_particle_set_flags
  • physics_particle_set_category_flags
  • physics_particle_set_group_flags
  • physics_particle_get_group_flags
  • physics_particle_group_get_inertia
  • physics_particle_group_get_mass
  • physics_particle_group_get_centre_x
  • physics_particle_group_get_centre_y
  • physics_particle_group_get_vel_x
  • physics_particle_group_get_vel_y
  • physics_particle_group_get_ang_vel
  • physics_particle_group_get_x
  • physics_particle_group_get_y
  • physics_particle_group_get_angle
  • physics_particle_get_data
  • physics_particle_get_data_particle
  • physics_particle_group_get_data

Sie können Partikel auf zwei Arten erstellen - einzeln oder in Gruppen. Einzelne Partikel verhalten sich genau so, wie es durch die globalen Einstellungen und die flags definiert ist, die bei ihrer Erstellung festgelegt wurden, aber Gruppen von Partikeln können je nach ihren Gruppenflags unterschiedliche Verhaltensweisen aufweisen. Diese Gruppenflags können auf unterschiedliche Werte gesetzt werden, um der Gruppe unterschiedliche Eigenschaften zu verleihen, wie z. B. Penetration und Steifigkeit.

Die folgenden Funktionen stehen für die Erstellung von Partikeln zur Verfügung:

  • physics_particle_create
  • physics_particle_group_begin
  • physics_particle_group_circle
  • physics_particle_group_box
  • physics_particle_group_polygon
  • physics_particle_group_add_point
  • physics_particle_group_end
  • physics_particle_group_join
  • physics_particle_group_delete

Sie können nicht nur Partikel erstellen, sondern sie auch löschen, entweder einzeln oder in einer Region. Zum Löschen von Partikeln aus dem System stehen die folgenden Funktionen zur Verfügung:

  • physics_particle_delete
  • physics_particle_delete_region_circle
  • physics_particle_delete_region_box
  • physics_particle_delete_region_poly

Solange Sie nicht die Debug-Zeichnung aktiviert haben, indem Sie physics_draw_debug()aktiviert haben, werden Sie keine Partikel auf dem Bildschirm sehen, wenn Sie sie erstellen. Um die Partikel tatsächlich zu sehen, musst du sie im Draw Event einer Instanz mit den folgenden Funktionen auf den Bildschirm zeichnen (beachte, dass du sie auch mit den regulären draw_sprite Funktionen zeichnen kannst, wenn du die Daten für jeden Partikel mit der Funktion physics_particle_get_data()):

  • physics_particle_draw
  • physics_particle_draw_ext
