physics_apply_force

Wenn Sie mit Physik arbeiten, müssen Sie nicht nur die Schwerkraft auf Objekte anwenden, sondern auch Kräfte. Eine Kraft ist jeder Einfluss, der eine Geschwindigkeits-, Richtungs- oder Formänderung eines Objekts bewirkt, und wird in der GameMaker-Physikwelt genauso definiert wie die Schwerkraft, indem man die vector aus den beiden Komponenten xforce/yforce erhält. Es ist zu beachten, dass Kräfte nicht auf den Massenschwerpunkt des Objekts wirken, sondern auf einen Punkt im Raum, und dass sie die Richtung und die Geschwindigkeit sowie das Drehmoment (Spin) in Abhängigkeit von anderen Kräften, die auf das Objekt wirken (wie die Schwerkraft), ändern. Hier ist ein Beispielbild zur Veranschaulichung:

Im obigen Beispiel hat das Federobjekt am unteren Ende eine Kraft auf das Kugelobjekt ausgeübt, die es dazu veranlasst, sich in eine andere Richtung zu bewegen. Die genaue Kraft wird durch die vector definiert, die wir aus den Komponenten xforce/yforce in Bezug auf die xpos/ypos-Koordinaten erhalten - was einfach bedeutet, dass die Kraft als der Abstand von xpos/ypos zu xforce/yforce in Newton berechnet wird (ein Newton ist definiert als die Nettokraft, die erforderlich ist, um eine Masse von einem Kilogramm mit einer Geschwindigkeit von einem Meter pro Sekunde im Quadrat zu beschleunigen) und die Richtung ist der Winkel, den wir von xpos/ypos zu xforce/yforce erhalten. Die code für eine Kollision wie die oben beschriebene würde in das Kollisionsereignis des Federobjekts eingehen und etwa so aussehen wie im folgenden Beispiel.

HINWEIS: Die Physikwelt verwendet Meter für alle Berechnungen, und das Verhältnis von Pixeln pro Meter wird im Raumeditor oder mit physics_world_create().

Syntax:

physics_apply_force(xpos, ypos, xforce, yforce)

Argument Typ Beschreibung
xpos Real The x coordinate in the room where the force will be applied
ypos Real The y coordinate in the room where the force will be applied
xforce Real The x component of the force vector
yforce Real The y component of the force vector

Rückgabe:

N/A

Beispiel:

with (other)
{
    physics_apply_force(x, y, 0, -30);
}
