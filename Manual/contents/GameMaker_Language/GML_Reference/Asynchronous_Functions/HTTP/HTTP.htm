HTTP

In diesem Abschnitt werden die verschiedenen asynchronen HTTP-Funktionen aufgeführt, die in GameMaker verfügbar sind. Diese Funktionen erzeugen ein asynchrones HTTP-Ereignis in allen Instanzen, die es haben:

  • http_request
  • http_get
  • http_get_file
  • http_post_string

Bitte beachten Sie, dass die oben genannten Funktionen von http_* aufgrund von domänenübergreifenden Sicherheitsproblemen möglicherweise nicht wie erwartet funktionieren. Dies bedeutet, dass Anfragen an Ihre server oder Versuche, Ressourcen von anderen Domains zu laden, blockiert werden und leere Ergebnisse oder 404-Fehler zurückgeben können. Eine Möglichkeit, dieses Problem zu umgehen, besteht darin, server in PHP einzubinden, das bestimmten Domains den Zugriff auf server erlaubt (dies ist auch eine Möglichkeit, Ihre Ressourcen zu schützen und servers zu blockieren, die nicht in der PHP-Zulassungsliste enthalten sind).

Im Folgenden finden Sie ein Beispiel für PHP code, das Sie dafür verwenden können:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

Für Bildladeanfragen, bei denen es wichtig ist, den herkunftsübergreifenden Typ zu bestimmen oder einzustellen, gibt es die folgenden Funktionen:

  • http_get_request_crossorigin
  • http_set_request_crossorigin

In den meisten Fällen sollten diese Funktionen nicht verwendet werden müssen, aber wenn das Spiel auf einer gesicherten Website ( server ) gespeichert ist - wo der Zugriff auf bestimmte Inhalte eine grundlegende Authentifizierung erfordert und beim Laden Sicherheitsfehler erzeugt - kann es notwendig sein, den Cross-Origin-Typ für Bildanforderungen auf "use-credentials" zu setzen (im Gegensatz zur Standardeinstellung "anonymous").
