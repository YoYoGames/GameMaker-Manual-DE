http_set_request_crossorigin

Mit dieser Funktion können Sie den Cross-Origin-Typ festlegen, der beim Laden von Bildern aus einer Datei verwendet werden soll. Die Funktion ist ausschließlich für HTML5 platform und ist nur nützlich, wenn sprites aus einer Datei mit der Funktion sprite_add() geladen wird. Beachten Sie, dass Sie bei der Einstellung "use-credentials" keinen absoluten Pfad zu dem geladenen sprite mehr angeben müssen, sondern stattdessen einen relativen Pfad (wie im Beispiel unten gezeigt). Standardmäßig ist der Cross-Origin-Typ auf "anonymous" eingestellt.

Syntax:

http_set_request_crossorigin(origin_type);

ArgumentTyp Beschreibung
origin_type The cross origin type to use (a string)

Rückgabe:

Beispiel:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Die obige code prüft zunächst den aktuell eingestellten Cross-Origin-Typ, und wenn dieser nicht "use-credentials" ist, wird er auf "use-credentials" gesetzt, und dann wird eine sprite aus einer Datei hinzugefügt.
