push_get_next_local_notification

Diese Funktion füllt eine vorgefertigte DS-Map mit einer Reihe von Schlüssel/Wertpaaren für die nächste lokale Push-Benachrichtigung in der Warteschlange, die angezeigt werden soll. Sie gibt -1 zurück, wenn keine weiteren Benachrichtigungen in der Warteschlange stehen, oder andernfalls einen realen Wert, der die ID für die Benachrichtigung darstellt. Diese ID kann dann verwendet werden, um die Benachrichtigung mit der Funktion push_cancel_local_notification(). Sie sollten die Funktion push_get_first_local_notification() aufrufen, um die erste Benachrichtigung in der Warteschlange zu erhalten, und dann diese Funktion verwenden, um sie zu durchlaufen.

HINWEIS: Diese Funktion ist auf das iOS- und Android-Ziel modules beschränkt.

Die DS-Map enthält die folgenden Schlüssel/Wertpaare:

  • "title" - Der Titel der Benachrichtigung (dieser ist nur auf Android platform verfügbar, da iOS die Titelinformationen nicht speichert)
  • "Nachricht" - Der Text der Nachricht
  • "data" - Die Zeichenkette des Datenpakets

Syntax:

push_get_next_local_notification(map);

ArgumentTyp Beschreibung
map The DS Map ID zu verwenden.

Rückgabe:

Push Notification ID or -1

Beispiel:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Die obige code erstellt eine DS Map und fordert dann Daten für die erste verfügbare Meldung in der Warteschlange an. Wenn eine Meldung vorhanden ist, wird die Karte ausgefüllt, und code überprüft dann den Schlüssel "data" auf eine bestimmte Zeichenfolge. Wird die Zeichenfolge gefunden, wird die Meldung abgebrochen und die Prüfung der nächsten Meldung fortgesetzt, bis alle Meldungen in der Warteschlange geprüft worden sind.
