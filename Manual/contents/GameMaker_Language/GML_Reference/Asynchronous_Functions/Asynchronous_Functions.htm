Asynchrone Funktionen

Im Kontext von GameMaker ist eine asynchrone Funktion eine Funktion, die an ein Web server sendet, eine Benutzereingabe abfragt oder sogar Daten streamt, während GameMaker weiterläuft, ohne das laufende Spiel zu blockieren. Wenn der Informations-Callback empfangen wird, wird ein spezielles Async-Ereignis ausgelöst, das wiederum jeder Instanz, der ein Async-Ereignis zugewiesen wurde, die Ausführung weiterer Funktionen ermöglicht (obwohl dies nicht der Fall sein muss). Im Grunde ist es also eine Möglichkeit, mit einer externen Ressource (server, Benutzer oder Gerät) zu kommunizieren, ohne dass das Spiel oder Projekt angehalten wird, während es auf eine Antwort wartet.

Warum ist das wichtig? Nun, weil es GameMaker erlaubt, weiterhin zu funktionieren, während es Daten sendet oder empfängt, und das bedeutet, dass Sie viele Dinge tun können, ohne das Spiel zu blockieren loop wie:

  • Daten vom Gerät in das Spiel streamen, ohne dass der Spieler warten muss, bis alles geladen ist
  • über Rückrufereignisse verfügen, um nur dann etwas zu tun, wenn die richtigen Informationen eingegangen sind
  • mit einem Webserver kommunizieren und Daten austauschen

Dadurch funktioniert alles viel flüssiger und unauffälliger. Es ist zu beachten, dass die asynchronen Ereignisse für alle Instanzen, die sie haben, ausgelöst werden. Sie können also eine asynchrone Funktion in einer Instanz verwenden und das asynchrone Ereignis, das sich mit der Antwort befasst, in einer anderen oder sogar in verschiedenen anderen Instanzen haben. Weitere Informationen über die Funktionsweise des asynchronen Ereignisses finden Sie hier - Asynchrone Ereignisse.
