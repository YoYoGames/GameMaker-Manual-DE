leistung_laden_leaderboard

Diese Funktion sendet eine Anfrage an server, um Informationen über das angegebene Leaderboard zu erhalten. Die Bestenliste muss zuvor auf dem Entwickler-Dashboard für Ihr Spiel erstellt worden sein, und die angegebene ID ist diejenige, die Sie als erstes Argument der Funktion angeben. Es wird ein Callback Social Asynchronous Event ausgelöst, das die async_load Map mit den relevanten Schlüssel/Wert-Paaren gefüllt wird. Der ID-Schlüssel dieser DS-Map wird verwendet, um den richtigen Rückruf zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_leaderboard_info sowie einer Reihe anderer Schlüssel/Wertpaare für jeden Spieler gepaart. Der genaue Inhalt der Karte ist unten dargestellt:

HINWEIS: Diese Funktion ist nur für iOS verfügbar.

  • "id" - Für diese Funktion sollte es achievement_leaderboard_infosein
  • "leaderboardid" - Die eindeutige ID für das Leaderboard, wie sie auf dem Dashboard des Anbieters definiert ist.
  • "numentries" - Die Anzahl der Einträge in der Rangliste, die Sie erhalten haben.
  • "PlayerN" - Der Name des Spielers, wobei "N" ein ganzzahliger Wert ist, der seiner Position in der Liste der Ranglisteneinträge entspricht.
  • "PlayeridN" - Die eindeutige Benutzerkennung des Spielers, "N".
  • "RangN" - Der Rang des Spielers "N" innerhalb der Rangliste.
  • "ScoreN" - Der Punktestand des Spielers "N".

Sie können den genauen Teil der Rangliste angeben, der angezeigt werden soll, indem Sie einen minimalen und einen maximalen Indexwert angeben. Außerdem können Sie verlangen, dass die Ergebnisse gefiltert werden, damit nur die Ergebnisse Ihrer Freunde oder die aller Spieler angezeigt werden, indem Sie die folgenden Filterkonstanten:

Konstante Beschreibung
achievement_filter_friends_only  Erhalten Sie nur die Spielstände der Freunde des Spielers.
achievement_filter_all_players  Alle Spielstände für alle Spieler abrufen.

Syntax:

achievement_load_leaderboard(ident, minindex, maxindex, filter)

Argument Beschreibung
ident Die eindeutige ID des Leaderboards, wie sie auf dem Entwickler-Dashboard angezeigt wird.
minindex Der Startindexwert, von dem die Daten der Bestenliste abgeleitet werden.
maxindex Der maximale Indexwert, von dem die Daten der Bestenliste abgeleitet werden.
filter Ob die Ergebnisse auf die Spieler in Ihrer Freundesliste oder auf alle Spieler gefiltert werden sollen (siehe Beschreibung unten)

Rückgabe:

N/A

Die folgende code würde wahrscheinlich aufgerufen werden, nachdem sich der Spieler in sein Spielkonto eingeloggt hat, indem er achievement_login() aufgerufen, um eine Liste aller Ranglistenpositionen der Freunde für das Spiel zu erhalten, entweder auf Android oder iOS:

if os_type == os_ios

    achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
    }

Dies sendet eine Anfrage für die Informationen über das angegebene Leaderboard und erzeugt einen asynchronen Callback mit der speziellen async_load DS-Map, die die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_leaderboard_info
    {
    var lbid = ds_map_find_value(async_load,"leaderboardid");
    if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
        {
        global.numentries = ds_map_find_value(async_load,"numentries");
        for(var i = 0; i < numentries; i++;)
            {
            global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
            global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
            global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
            global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
            achievement_get_pic(global.playerid[i]);
            }
        }
    }

Die obige code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis, und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, prüft sie dann, ob die gesendete Bestenliste mit dem ursprünglichen Funktionsaufruf übereinstimmt, bevor bevor sie die Karte in einer Schleife durchläuft und alle verschiedenen Werte in einer Reihe von Arrays speichert. Außerdem wird für jeden Eintrag in der Karte ein Bild angefordert (siehe achievement_get_pic()).
