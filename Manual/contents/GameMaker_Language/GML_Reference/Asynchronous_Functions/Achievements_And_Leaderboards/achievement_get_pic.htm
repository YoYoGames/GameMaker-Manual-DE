leistungs_get_pic

Diese Funktion sendet eine Anfrage an server für das Bild eines Spielers oder eines Freundes und löst einen Callback Social Asynchronous Event aus, der die async_load Map mit den relevanten Schlüssel/Wert-Paaren enthält. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Rückruf zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_pic_loaded sowie einer Reihe anderer Schlüssel/Wertpaare gepaart. Der genaue Inhalt der Map ist im Folgenden dargestellt:

HINWEIS: Diese Funktion ist nur für iOS verfügbar.

  • "id" - Für diese Funktion sollte es achievement_pic_loaded sein
  • "playerid" - Die ID des Spielers oder Freundes, von dem wir das Bild erhalten.
  • "spriteid" - Die sprite id für das Bild, die dann verwendet werden kann, um es auf dem Bildschirm mit einer der Funktionen von sprite zuzeichnen.

WARNUNG! Diese Funktion erstellt sprites und als solche sollten ihre IDs in Variablen gespeichert und dann aus dem Speicher entfernt werden, indem man die sprite_delete() Funktion aus dem Speicher entfernt werden, andernfalls führt die wiederholte Verwendung zu einem Speicherleck, das schließlich das Spiel verlangsamt oder zum Absturz bringt.

Syntax:

achievement_get_pic(char)

Argument Beschreibung
char Die ID der Person, von der das Bild abgerufen werden soll (entweder aus einer Freundesliste oder einem Leaderboard)

Rückgabe:

N/A

Erweitertes Beispiel:

Die folgende code würde aufgerufen werden, nachdem man eine Freundesliste oder eine Bestenliste erhalten hat, indem man Leistung_Freunde_laden() oder leistung_load_leaderboard(). Diese Funktionen geben eindeutige "id"-Werte für jede Person zurück, die dann in der Funktion verwendet werden, um das Profilbild für sie abzurufen:

achievement_get_pic(global.player_id[0]);

Dadurch wird eine Anforderung für das Bild gesendet, das mit der im globalen Array gespeicherten Player-ID verknüpft ist, und ein asynchroner Rückruf mit der speziellen async_load DS-Map erzeugt, die die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
    {
    var picid = ds_map_find_value(async_load, "playerid");
    var spriteid = ds_map_find_value(async_load, "spriteid");
    if global.playerid[0] == picid)
        {
        global.playerpic[0] = spriteid;
        }
    }

Die obige code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis, und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, prüft sie, ob die mit dem Bild verknüpfte id mit der angeforderten identisch ist, und wenn Wenn ja, wird ein weiteres globales Array für die eindeutige ID des Bildes festgelegt (das zum Zeichnen des Bildes als sprite verwendet werden kann).
