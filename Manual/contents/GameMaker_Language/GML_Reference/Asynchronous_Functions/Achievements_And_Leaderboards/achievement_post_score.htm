leistung_post_score

Mit dieser Funktion können Sie Ihren Punktestand an den gewählten Leaderboard- und Achievement-Dienst senden. Auf Android senden Sie die ID der Punktetabelle als String (dies ist die eindeutige Leaderboard-ID, die beim Einrichten des Leaderboards zugewiesen wurde), während Sie auf allen anderen platforms geben Sie den definierten Namen der Bestenliste als String und dann den aktuellen Punktestand an. Die Funktion löst ein soziales asynchrones Ereignis aus, wobei die zurückgegebene async_load DS-Map die folgenden Schlüssel/Wert-Paare enthält:

  • "type" - Dies ist der Typ des Ereignisses, das ausgelöst wurde. Für diese Funktion ist dies die Zeichenkette "achievement_post_score_result".
  • "id" - Dies ist die ID des Ereignisses, und für diese Funktion wird eine GML Konstante GooglePlayServices_EVENT_ID_POST_SCORE zurückgegeben.
  • "leaderboard_id" - Dies ist die eindeutige ID des Leaderboards, das aktualisiert wurde.
  • "Status" - Dieser Wert ist 1, wenn die Werte erfolgreich aktualisiert wurden, oder 0, wenn die Anfrage fehlgeschlagen ist.
  • "daily_new_best" - Dieser Wert ist 1, wenn die eingereichte Punktzahl die beste des letzten Tages ist, andernfalls 0.
  • "daily_best" - Hier wird die höchste Punktzahl des Benutzers vom letzten Spieltag gespeichert.
  • "weekly_new_best" - Dieser Wert ist 1, wenn die eingereichte Punktzahl die beste des Nutzers in der letzten Woche war, andernfalls 0.
  • "weekly_best" - Hier wird die höchste Punktzahl des Nutzers in der letzten Woche gespeichert.
  • "all_time_new_best" - Dieser Wert ist 1, wenn die eingereichte Punktzahl die beste aller Zeiten des Benutzers war, andernfalls 0.
  • "all_time_best" - Hier wird die beste Punktzahl des Benutzers aller Zeiten gespeichert.

Syntax:

achievement_post_score(leaderboard, score)

Argument Beschreibung
leaderboard Der Name/die ID der Bestenliste.
score Die zu sendende Partitur.

Rückgabe:

N/A

Erweitertes Beispiel:

if achievement_available()
    {
    achievement_post_score(global.LeaderboardID, 1000);
    }

Die obige code prüft, ob der gewählte Leaderboard- und Achievement-Dienst verfügbar ist, und wenn ja, wird eine Punktzahl in die Punktetabelle "Leaderboard" eingetragen. Dies löst ein soziales Async-Ereignis aus, das wie folgt geparst werden kann:

if async_load[? "id"] == GooglePlayServices_EVENT_ID_POST_SCORE
    {
    if async_load[? "type"] == "achievement_post_score_result"
        {
        if async_load[? "status"] == 1
            {
            global.Best_Day = async_load[? "daily_best"];
            global.Best_Week = async_load[? "weekly_best"];
            global.Best = async_load[? "all_time_best"];
            }
        }
    }

Die obige code prüft die async_load DS-Map auf die Schlüssel "id" und "type", die dieser Funktion entsprechen, und prüft dann den Status des Ereignisses. Wenn der Status 1 ist, werden einige Werte aus der DS-Map in globalen Variablen gespeichert.
