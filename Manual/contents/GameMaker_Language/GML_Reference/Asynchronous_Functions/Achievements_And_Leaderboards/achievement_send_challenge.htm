Leistung_senden_Herausforderung

Diese Funktion sendet eine Herausforderung über das Netzwerk an den ausgewählten Spieler. Man kann die Spielernummer mit der Funktion Leistung_Freunde_laden() oder die achievement_load_leaderboard() Funktionen ermittelt werden. Außerdem muss die challengeid angegeben werden, der eindeutige Wert, der der Herausforderung bei der Erstellung im iTunes Connect oder Google Play Dashboard zugewiesen wurde. Außerdem müssen Sie einen Punktestand und eine kurze Textnachricht angeben sowie den Aufgabentyp festlegen. Dies kann eine der folgenden Konstanten sein:

Konstante Beschreibung
achievement_type_score_challenge  Eine Herausforderung auf der Grundlage des Punktwerts.
achievement_type_achievement_challenge  Eine Herausforderung, die auf einer Leistung beruht.

Diese Funktion löst einen Rückruf Social Asynchronous Event für den Spieler aus, der die Herausforderung erhalten soll, und in diesem Ereignis wird eine DS-Karte referenziert in der Variablen async_load. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_challenge_received sowie einer Reihe anderer Schlüssel/Wertpaare für jeden Spieler gepaart. Der genaue Inhalt der Karte ist unten dargestellt:

HINWEIS: der Benutzer kann auch Toast-Benachrichtigungen für erhaltene Herausforderungen erhalten, aber dies hängt von den Werten ab, die man mit der Funktion achievement_show_challenge_notifications().

  • "id" - Für diese Funktion sollte es achievement_leaderboard_infosein
  • "playerid" - Die Spieler-ID für die Herausforderung.
  • "issuerid" - Die ID des Ausstellers der Herausforderung.
  • "state" - Der Status der Challenge, der einen Wert von 0 - 3 (als String) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat.
  • "Nachricht" - Die Textnachricht für die Herausforderung.
  • "completeddate" - Das Datum des Abschlusses der Challenge.
  • "issueddate" - Das Ausstellungsdatum für den Widerspruch.
  • "type" - Die Art der gestellten Aufgabe. Kann eine der beiden oben aufgeführten Konstanten sein
  • "Bezeichner" - Die Identifizierungszeichenfolge für die Herausforderung, wie auf dem Dashboard des Anbieters festgelegt
  • "Punktzahl" - Die mit der Herausforderung verbundene Punktzahl (falls zutreffend).

Syntax:

achievement_send_challenge(playerid, challengeid, score, type, message)

Argument Beschreibung
playerid Die eindeutige ID des herauszufordernden Spielers.
challengeid Die eindeutige Challenge-ID (wie auf dem Dashboard des Anbieters zu finden).
score Die zu übertreffende Punktzahl.
type die Art der auszustellenden Aufforderung - eine der beiden unten angegebenen Konstanten.
message Die Nachricht, die an die Herausforderung angehängt werden soll.

Rückgabe:

N/A

Erweitertes Beispiel:

Um eine Aufforderung über das Netz zu senden, müssten Sie diesen Code eingeben:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Diese Anfrage löst dann das soziale Ereignis in Ihrem Spiel für den Spieler aus, an den die Herausforderung gerichtet war, und dies kann wie folgt gehandhabt werden:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
    {
    player_id = ds_map_find_value(async_load, "playerid");
    issuer_id = ds_map_find_value(async_load, "issuerid");
    state = ds_map_find_value(async_load, "state");
    message = ds_map_find_value(async_load, "message");
    date_completed = ds_map_find_value(async_load, "completeddate");
    date_issued = ds_map_find_value(async_load, "issueddate");
    ach_type = ds_map_find_value(async_load, "type");
    ach_ident = ds_map_find_value(async_load, "identifier");
    ach_score = ds_map_find_value(async_load, "score");
    }

Die obige code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis, und wenn ihre "id" mit der geprüften Konstante übereinstimmt, extrahiert sie die relevanten Werte für jeden der Schlüssel in der Map und speichert sie in Variablen für die zukünftige Verwendung.
