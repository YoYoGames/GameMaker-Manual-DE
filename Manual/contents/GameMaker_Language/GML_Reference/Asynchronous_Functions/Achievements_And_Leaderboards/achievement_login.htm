leistungs_login

Diese Funktion meldet den Benutzer bei einem entsprechenden Ranglisten- und Leistungsdienst an. Wenn der Dienst nicht verfügbar ist, wird der Benutzer bei einem "vorgetäuschten" Game Center angemeldet und alle Erfolge und Punkte werden auf dem Gerät gespeichert, so dass, wenn der tatsächliche Dienst verfügbar ist, können diese Details hochgeladen werden. Die Funktion löst einen Callback Social Asynchronous Event aus, der die async_load Map mit den relevanten Schlüssel/Wert-Paaren enthält. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_our_info sowie einer Reihe anderer Schlüssel/Wertpaare für jede Herausforderung gepaart. Der genaue Inhalt der Map ist unten dargestellt:

  • "id" - Für diese Funktion sollte es achievement_our_info sein
  • "Name" - Der Name des Benutzers, der gerade angemeldet ist.
  • "playerid" - Die eindeutige Spieler-ID für den derzeit angemeldeten Benutzer.

Syntax:

achievement_login()

Rückgabe:

N/A

Das folgende code wird wahrscheinlich gleich zu Beginn des Spiels oder von einem speziellen Schaltflächenobjekt aufgerufen, das Sie in einem Raum platziert haben, und fordert den Spieler auf, sich bei seinem Google Game Services- oder Apple Game Center-Konto anzumelden:

achievement_login();

Dies sendet eine Anfrage nach den aktuellen Spielerinformationen und erzeugt einen asynchronen Rückruf mit der speziellen async_load DS-Map, die die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_our_info
    {
    var name = ds_map_find_value(async_load, "name" );
    var playerid = ds_map_find_value(async_load, "playerid" );
    global.OurName = name;
    global.OurId = playerid;
    if os_type == os_android
        {
        achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
        }
    else
        {
        achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
        }
    achievement_get_challenges();
    achievement_load_friends();
    achievement_load_progress();
    }

Die obige code prüft die zurückgegebene DS Map im Social Asynchronous Event und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, dann loops durch die Map und speichert alle verschiedenen Werte in Variablen, bevor sie den Rest der Funktionen zur Vorbereitung der Bestenlisten, Herausforderungen und Freundeslisten aufruft.
