cloud_file_save

Mit dieser Funktion wird eine Datei an den gewählten Cloud-Dienst zur Speicherung übergeben. Die Funktion gibt einen eindeutigen ID-Wert zurück, der dann in dem entsprechenden asynchronen Ereignis verwendet werden sollte, um die DS-Map zu identifizieren, die als "Rückruf" vom Cloud-Dienst zurückgegeben wird. Die Datei sollte alle Informationen enthalten, die Sie für Ihr Spiel speichern müssen, da Sie nur einen einzigen "Datenblob" in der Cloud speichern können und die erneute Ausführung dieser Funktion alle zuvor gespeicherten Werte überschreibt (ebenso wie die Verwendung der cloud_string_save() Funktion). Die Beschreibung sollte eine kurze Zeichenfolge sein, die die Speicherung beschreibt, z. B.: "Level2, Stage2".

Weitere Informationen zu den zurückgegebenen asynchronen Daten finden Sie in der Funktion cloud_synchronise().

Syntax:

cloud_file_save(file, description);

ArgumentTyp Beschreibung
string The file to be uploaded (as a string).
description A brief description of the data being stored.

Rückgabe:

Async Request ID

Beispiel:

var t_str = "";
for (var i = 0; i < 10; i++)
{
    t_str += string(global.Highscore[i]) + "|"
}
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Current Highscores");

Die obige code erstellt eine Zeichenkette aus den im globalen Array "Highscores" gespeicherten Werten und schreibt diese Zeichenkette dann in eine Datei zur lokalen Speicherung. Die Datei wird dann zur Speicherung an den Cloud-Dienst gesendet.
