show_question_async

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die Frage an, die Sie in der Funktion definieren. Da es sich um eine asynchrone Funktion handelt, blockiert der GameMaker das Gerät, auf dem sie ausgeführt wird, nicht, während er auf eine Antwort wartet, sondern lässt die Ereignisse wie gewohnt weiterlaufen. Die Funktion hat zwei Schaltflächen, die "Ja" und "Nein" anzeigen, und sobald der Benutzer eine davon gedrückt hat, wird ein asynchrones Dialog-Ereignis ausgelöst, das nur für die Dauer des Ereignisses eine DS-Map in der Variablen async_load.

Diese Karte enthält die beiden Schlüssel "id" und "status". "id" ist der Wert, der beim Aufruf der Funktion zurückgegeben wurde, während "status" entweder true oder false für "Ja" bzw. "Nein" ist.

HINWEIS Diese Funktion ist nur zum Debuggen und Testengedacht und sollte nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Syntax:

show_question_async(string);

Argument Typ Beschreibung
string Zeichenfolge The question to ask to the user.

Rückgabe:

Async Request ID

Erweitertes Beispiel:

Das linke Mausklick-Ereignis (zum Beispiel) des Objekts, das die Nachricht anzeigt, würde den folgenden Code enthalten:

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

Im obigen Beispiel wird eine Frage mit der angegebenen Zeichenfolge angezeigt und der Benutzer aufgefordert, entweder "Ja" oder "Nein" zu drücken. Die Funktions-ID wird in der Variablen "msg" gespeichert und im Ereignis "Asynchroner Dialog" wie unten gezeigt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if (i_d == msg)
{
    if (ds_map_find_value(async_load, "status"))
    {
        coins -= armour[0,5];
        global.protection += armour[0,0];
    }
}

Die obige code prüft den Schlüssel "id" der zurückgegebenen DS Map mit dem in der Variablen "msg" gespeicherten Wert. Wenn sie übereinstimmen, wird geprüft, ob eine der beiden Tasten gedrückt wurde, und wenn true zurückgegeben wird, wird ein Wert von einer Variablen abgezogen und ein Wert zu einer globalen Variablen hinzugefügt.
