Dialog

Die meisten modernen Computer und Mobilgeräte mögen es nicht, wenn ein Spiel angehalten wird, während ein Dialogfenster angezeigt wird. Sie neigen dazu, dies so zu interpretieren, dass das Spiel runner angehalten wurde, und könnten dies als Fehler betrachten und die App vorzeitig schließen. Um dies zu verhindern, verfügt GameMaker über asynchrone Dialogfunktionen, die es erlauben, bestimmte Formen von Benutzerfeedback oder -eingaben anzufordern, ohne dass das Spiel runner zu irgendeinem Zeitpunkt blockiert wird, d. h. das Spiel läuft - wenn es aufgerufen wird - ganz normal im Hintergrund, während der Benutzer auf das angezeigte Dialogfenster antwortet. Sobald der Benutzer geantwortet hat, wird ein asynchrones Dialogereignis aufgerufen, damit Sie die Ergebnisse verarbeiten können.

HINWEIS: Mit Ausnahme von shop_leave_rating()sind diese Funktionen nur zum Debuggen und Testengedacht und sollten nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Im Folgenden finden Sie eine Liste aller verfügbaren asynchronen Dialogfunktionen:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
