show_message_async

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die Nachricht an, die Sie in der Funktion definieren. Da es sich um eine asynchrone Funktion handelt, blockiert der GameMaker das Gerät, auf dem sie ausgeführt wird, nicht, während er auf eine Antwort wartet, sondern lässt die Ereignisse wie gewohnt weiterlaufen. Sobald der Benutzer die Schaltfläche "Okay" gedrückt hat, wird ein asynchrones Dialog Ereignis ausgelöst, das nur für die Dauer dieses Ereignisses eine ds_map in der Variablen async_load.

Diese Map enthält die beiden Schlüssel "id" und "status", wobei "id" der Wert ist, der von der Funktion beim Aufruf zurückgegeben wurde, und "status" entweder true für die gedrückte Schaltfläche "Okay" oder false, wenn die Nachricht abgebrochen wurde (sofern verfügbar, da nicht alle Ziele platforms den Abbruch von Dialogen erlauben).

HINWEIS Diese Funktion ist nur zum Debuggen und Testengedacht und sollte nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Syntax:

show_message_async(string);

Argument Typ Beschreibung
string Zeichenfolge The message to show to the user.

Rückgabe:

Async Request ID

Erweitertes Beispiel:

Das linke Mausklick-Ereignis (zum Beispiel) des Objekts, das die Nachricht anzeigt, würde den folgenden Code enthalten:

msg = show_message_async("You will now be taken to the store");

Das obige Beispiel zeigt eine Nachricht mit der angegebenen Zeichenkette an. Die Nachrichten-ID wird in der Variablen "msg" gespeichert und im asynchronen Ereignis "Dialoge" wie unten gezeigt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        url_open("https://play.google.com/store");
    }
}

Die obige code überprüft den "id"-Schlüssel der zurückgegebenen DS Map mit dem in der Variablen "msg" gespeicherten Wert. Wenn sie übereinstimmen, wird geprüft, ob die Schaltfläche "Okay" gedrückt wurde (und nicht das Fenster geschlossen/abgebrochen wurde), und wenn true zurückgegeben wird, wird eine URL geöffnet.
