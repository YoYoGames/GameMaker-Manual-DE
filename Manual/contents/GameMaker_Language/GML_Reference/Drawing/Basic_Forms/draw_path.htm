zeichnen_pfad

Mit dieser Funktion kannst du GameMaker dazu bringen, einen Pfad auf den Bildschirm zu zeichnen. Der Pfad wird als einfache Linie gezeichnet und kann entweder relativ zur aufrufenden Instanz oder an der absoluten Position sein, an der er im Pfad-Editor oder über code erstellt wurde. Diese Funktion ist äußerst nützlich, wenn debugging dynamische Pfade (zum Beispiel solche, die für Instanzen mit der mp_grid_path() Funktion).

HINWEIS: Wenn Sie eine Form mit einem Shader zeichnen wollen, sollten Sie wissen, dass die meisten Shader die folgenden Eingaben erwarten: Scheitelpunkt, Textur, Farbe. Wenn Sie diese Funktion verwenden, werden jedoch nur Scheitelpunkt- und Farbdaten übergeben, so dass der Shader möglicherweise nichts zeichnet (oder etwas zeichnet, aber nicht korrekt). Wenn Sie Formen auf diese Weise zeichnen müssen, sollte der Shader entsprechend angepasst werden.

Syntax:

draw_path(path, x, y, absolute);

ArgumentTyp Beschreibung
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true) oder die relative Position (false)

Rückgabe:

Beispiel:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

Die obige code verwendet die Funktion mp_grid_path, um einen Pfad zu erzeugen und ihn in der Variablen "path" zu speichern. Wenn der Pfad erfolgreich erstellt wurde, wird er auf dem Bildschirm an einer Position relativ zu der Instanz gezeichnet, auf der die code läuft.
