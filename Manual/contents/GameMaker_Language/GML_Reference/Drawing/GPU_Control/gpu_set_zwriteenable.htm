gpu_set_zwriteenable

Mit dieser Funktion kann die Tiefenordnung über den z-Buffer ein- (true) und ausgeschaltet (false) werden. Wenn eine Instanz in GameMaker gerendert wird, wird die Tiefe eines erzeugten Pixels (seine z-Koordinate) im z-Puffer gespeichert. Wenn dann eine andere Instanz der Szene an demselben Pixel gerendert werden muss, werden die beiden Tiefen verglichen (z-Testing - siehe gpu_set_ztestenable()) und diejenige, die dem Betrachter "näher" ist, wird dann im z-Puffer gespeichert und ersetzt die alte. Dies ist jedoch nicht immer das, was Sie wünschen, und es gibt Momente, in denen Sie möchten, dass etwas über alles andere gezeichnet wird, unabhängig davon, welchen z-Wert es hat. In solchen Fällen können Sie diese Funktion switch verwenden, um das Schreiben in den z-Puffer abzuschalten und so zu erzwingen, dass das, was gezeichnet wird, über alles andere gezeichnet wird. Beachten Sie, dass die hier gegebene Beschreibung für den standardmäßigen z-Test-Vergleichsmodus gilt, der aber geändert werden kann

Der Standardwert ist, dass die z-Pufferung aktiviert ist (true).

Syntax:

gpu_set_zwriteenable(enable);

ArgumentTyp Beschreibung
enable The comparison mode to use (see list above)

Rückgabe:

Beispiel:

gpu_set_zwriteenable(false);
with (obj_menu_containers)
{
    draw_self();
}
gpu_set_zwriteenable(true);

Die obige code schaltet das Schreiben des Z-Puffers aus, zeichnet eine Reihe von Objekten und schaltet ihn dann wieder ein, um mit dem Zeichnen fortzufahren.
