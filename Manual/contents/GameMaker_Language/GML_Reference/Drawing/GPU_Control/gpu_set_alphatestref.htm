gpu_set_alphatestref

Diese Funktion setzt den Referenzwert für den Alphatest (wenn dieser mit der Funktion gpu_set_alphatestenable()). Dies ist der "Grenzwert", bei dem Pixel mit Alpha nicht gezeichnet werden.

Wenn Sie diesen Wert auf 0 gesetzt haben und die lineare Interpolation aktiviert ist (entweder in den Spieloptionen oder mit gpu_set_texfilter()), kann es immer noch zu visuellen Fehlern an den Rändern von sich überlappenden Objekten kommen. Das liegt daran, dass es fast vollständig transparente Bereiche geben kann, die die dahinter gezeichneten Objekte blockieren. Wenn Sie diesen Referenzwert jedoch z. B. auf 254 (d. h. einen Wert unter dem Maximalwert) einstellen, wird sichergestellt, dass nur 100% undurchsichtige Pixel gezeichnet werden. Sie können die Texturinterpolation auch ausschalten, so dass Sie normalerweise die Standardwerte ohne Probleme verwenden können und diese Funktion überhaupt nicht brauchen, allerdings hängt viel von der Ausgangsgrafik ab.

HINWEIS: Dies ist eine fortgeschrittene Zeichenfunktion. Wenn Sie sich also nicht sicher sind, ob Sie sie brauchen oder nicht, oder nicht genau wissen, was sie bewirkt, sollten Sie sie besser nicht verwenden, da sie die Zeichnung Ihres Spiels erheblich beeinträchtigen kann.

Syntax:

gpu_set_alphatestref(val);

ArgumentTyp Beschreibung
val Sets the reference value for alpha testing from 0 to 255 (default value is 0)

Rückgabe:

Beispiel:

if !gpu_get_alphatestenable()
{
    gpu_set_alphatestenable(true);
    gpu_set_alphatestref(128);
}

Die obige code prüft, ob der Alphatest aktiviert ist, und wenn nicht, wird switch auf Alphatest und den Testschwellenwert auf 128 gesetzt (nur Pixel mit einem Alphawert über 0,5 werden gezeichnet).
