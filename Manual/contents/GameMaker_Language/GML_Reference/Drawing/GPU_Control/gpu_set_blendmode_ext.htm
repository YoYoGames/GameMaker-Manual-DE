gpu_set_blendmode_ext

Wenn GameMaker ein Pixel zeichnet, gibt es eine Quellfarbe (die Farbe des Pixels, das wir zeichnen wollen) und eine Zielfarbe (die Farbe, die bereits in dem Pixel enthalten ist, zu dem wir zeichnen), so dass bei der Bestimmung der neuen Farbe des Pixels die Quell- und Zielfarben entsprechend dem gewählten Mischmodus berechnet werden. Jede Komponente der Farben wird als Fließkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird berechnet, indem jede Komponente der Ausgangsfarbe mit einem Faktor und jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert wird und die Ergebnisse dann Komponente für Komponente addiert werden.

Mit dieser Funktion können Sie die verschiedenen Komponenten festlegen, die zusammengefügt werden sollen, um einen benutzerdefinierten Mischmodus zu erstellen. Quelle und Ziel haben jeweils eine Rot-, Grün-, Blau- und Alphakomponente, und in der folgenden Tabelle werden die RGBA der Quelle als (Rs, Gs, Bs, As) und die des Ziels als (Rd, Gd, Bd, Ad) betrachtet. Die elf Konstanten, die für die Verwendung zur Verfügung stehen, können entweder Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstante Überblendfaktor (Rot, Grün, Blau, Alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) mit f = min(As, 1-Ad)

WICHTIG! HTML5 ohne aktiviertes WebGL kann die folgenden Modi nicht korrekt anzeigen:

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

Beachten Sie, dass Sie dieser Funktion entweder zwei einzelne arguments oder ein Array von arguments übergeben können (wie es von der Funktion gpu_get_blendmode_ext() zum Beispiel). Wenn Sie ein Array übergeben, sollte es die folgenden zwei Elemente enthalten:

  • [0] = Quell-Mischmodus (Standard ist bm_src_alpha)
  • [1] = Ziel-Mischmodus (Standard ist bm_inv_src_alpha)

Damit Sie die Mischmodi optimal nutzen können und um zu verstehen, wie sie funktionieren und wie sie sich auf das endgültige Bild auf dem Bildschirm auswirken, empfehlen wir Ihnen, den folgenden Leitfaden zu lesen:

  • Anleitung zur Verwendung von Blendmodes

Syntax:

gpu_set_blendmode_ext(src, dest);

Argument Typ Beschreibung
src Blend Mode Faktor Konstante Source blend mode factor (see constants above).
dest Blend Mode Faktor Konstante Destination blend mode factor (see constants above)

Rückgabe:

N/A

Beispiel:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Dadurch wird das Schwarz transparent und es entsteht ein "Glüheffekt", da das Weiß außen stark ist und von der Kreismitte aus allmählich schwächer wird. Der Mischmodus wird nach dem Zeichnen des Kreises zurückgesetzt, um sicherzustellen, dass die additive Überblendung nicht auch auf alles andere nach dem Kreis angewendet wird.
