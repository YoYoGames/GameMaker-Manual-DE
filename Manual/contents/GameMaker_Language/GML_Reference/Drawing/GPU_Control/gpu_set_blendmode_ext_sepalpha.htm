gpu_set_blendmode_ext_sepalpha

Wenn GameMaker ein Pixel zeichnet, gibt es eine Quellfarbe (die Farbe des Pixels, das wir zeichnen wollen) und eine Zielfarbe (die Farbe, die bereits in dem Pixel enthalten ist, zu dem wir zeichnen), so dass bei der Bestimmung der neuen Farbe des Pixels die Quell- und Zielfarben entsprechend dem gewählten Mischmodus berechnet werden. Jede Komponente der Farben wird als Fließkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird berechnet, indem jede Komponente der Ausgangsfarbe mit einem Faktor und jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert wird und die Ergebnisse dann Komponente für Komponente addiert werden.

Mit dieser Funktion können Sie die verschiedenen Komponenten festlegen, die zusammengefügt werden sollen, um einen benutzerdefinierten Überblendungsmodus zu erstellen, indem Sie verschiedene Teile verschiedener Überblendungsmodusfaktoren für die RGB-Komponenten und die Alphakomponente auswählen. So verwenden "src" und "dest" argument die ersten 3 Werte des Überblendungsmodus (RGB) und "alphasrc" und "alphadest" arguments nehmen den letzten Teil des Überblendungsmodusfaktors. Wenn Sie zum Beispiel die RGB-Quelle auf bm_zero setzen, wird (0, 0, 0) verwendet. Wenn Sie dann die Alpha-Quelle als bm_one einstellen, wird (1) verwendet, was einen endgültigen Quellüberblendungsmodus von (0, 0, 0, 1) ergibt.

Das folgende Diagramm zeigt die RGBA-Werte für jeden Mischmodus, wobei (Rs, Gs, Bs, As) als Quellwerte und (Rd, Gd, Bd, Ad) als Zielwerte gelten. Die elf Konstanten, die für die Verwendung zur Verfügung stehen, können entweder Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstante Überblendfaktor (Rot, Grün, Blau, Alpha)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) mit f = min(As, 1-Ad)

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

Beachten Sie, dass Sie dieser Funktion entweder vier einzelne arguments oder ein Array von arguments übergeben können (wie es von der Funktion gpu_get_blendmode_ext_sepalpha() zum Beispiel). Wenn Sie ein Array übergeben, sollte es die folgenden vier Elemente enthalten:

  • [0] = Quell-Mischmodus abzüglich der Alphakomponente (Standard ist bm_src_alpha (As, As, As))
  • [1] = Ziel-Mischmodus abzüglich der Alphakomponente (Standard ist bm_inv_src_alpha (1-As, 1-As, 1-As))
  • [2] = Quell-Alpha-Komponente (Standard ist bm_src_alpha (As))
  • [3] = Ziel-Alpha-Komponente (Standard ist bm_inv_src_alpha (1-As))

Damit Sie die Mischmodi optimal nutzen können und um zu verstehen, wie sie funktionieren und wie sie sich auf das endgültige Bild auf dem Bildschirm auswirken, empfehlen wir Ihnen, den folgenden Leitfaden zu lesen:

  • Anleitung zur Verwendung von Blendmodes

Syntax:

gpu_set_blendmode(src, dest, alphasrc, alphadest);

Argument Typ Beschreibung
src Blend Mode Faktor Konstante Source blend mode factor for RGB (see constants above).
dest Blend Mode Faktor Konstante Destination blend mode factor for RGB (see constants above)
alphasrc Blend Mode Faktor Konstante Source blend mode factor Alpha (see constants above).
alphadest Blend Mode Faktor Konstante Destination blend mode factor Alpha (see constants above)

Rückgabe:

N/A

Beispiel:

var bm = gpu_get_blendmode_ext_sepalpha();
bm[0] = bm_src_alpha;
bm[1] = bm_inv_src_alpha;
gpu_set_blendmode_ext_sepalpha(bm);

Die obige code erstellt ein lokales Array und ruft den aktuellen erweiterten Mischmodus mit separaten Alphawerten ab. Dieses Array wird dann bearbeitet und zurückgegeben, um den Mischmodus festzulegen.
