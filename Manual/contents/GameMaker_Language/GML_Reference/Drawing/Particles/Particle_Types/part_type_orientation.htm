teil_art_ausrichtung

Diese Funktion wird verwendet, um die Ausrichtung des Partikels sprite zu bestimmen, wenn er erstellt wird, und kann auch verwendet werden, um die Ausrichtung des Partikels während seiner Lebensdauer zu erhöhen oder zu verringern. Die Werte für die minimale und maximale Ausrichtung sind standardmäßig auf 0 eingestellt, können aber geändert werden, um die Ausrichtung nach dem Zufallsprinzip zu bestimmen, wobei die GameMaker-Standardrichtungen 0 Grad nach rechts, 90 Grad nach oben, 180 Grad nach links und 270 Grad nach unten gelten. Wenn Sie sie auf den gleichen Wert setzen, werden die Partikel alle mit der gleichen Ausrichtung erstellt.

Sie können auch einen Inkrementwert festlegen, der über die gesamte Lebensdauer der Ausrichtung eine bestimmte Gradzahl addiert (bei einer positiven Zahl) oder subtrahiert (bei einer negativen Zahl). Dieser Wert kann ein Minimum von (+/-) 0,01 sein.

Wie bei den anderen Partikelfunktionen können Sie auch den "Wackelfaktor" einstellen. Dies ist ein Wert, der bei jedem Schritt der Partikellebensdauer zufällig zur Ausrichtung addiert oder subtrahiert wird. Größere Werte sind natürlich stärker ausgeprägt als kleinere, und dieser Wert kann sogar negativ sein, wobei der maximale Bereich zwischen -20 und 20 liegt.

Schließlich können Sie wählen, ob die Ausrichtung relativ sein soll oder nicht, was bedeutet, dass der Partikel sprite um diese vector herum ausgerichtet wird, solange er eine Richtung (und Geschwindigkeit) hat. Beachten Sie, dass, wenn Sie die Geschwindigkeit des Partikels reduzieren und er 0 erreicht, die fehlende Geschwindigkeit die Richtung auf den Standardwert von 0° setzt, so dass eine relative Ausrichtung dazu führt, dass der Partikel sprite zu einem anderen Winkel "springt".

Syntax:

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );

Argument Typ Beschreibung
ind Partikel Typ ID The index of the particle type to change.
ang_min Real The minimum starting angle of the particles.
ang_max Real The maximum starting angle of the particles.
ang_incr Real The increase (or decrease if set negative) of the angle in degrees per step.
ang_wiggle Real The amount the angle wiggles clockwise/counterclockwise per step.
ang_relative Real Whether to set its angle relative to the direction of the particle's motion (true) or not (false).

Rückgabe:

N/A

Beispiel:

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

Die obige code legt verschiedene Partikelwerte fest, einschließlich der Ausrichtung, die ein Zufallswert zwischen 0º und 359º sein wird. Außerdem wird bei jedem Schritt ein Zufallswert zwischen 0 und 20 hinzugefügt, und die Ausrichtung ist relativ zur Bewegungsrichtung.
