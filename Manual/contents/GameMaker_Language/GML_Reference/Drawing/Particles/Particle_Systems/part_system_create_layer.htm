part_system_create_layer

Diese Funktion erstellt ein neues Partikelsystem auf einer bestimmten Ebene. Du gibst die eindeutige Layer-ID an, wie sie von der Funktion layer_create() zurückgegebene eindeutige Layer-ID oder den Namen des zu verwendenden Layers als String an - zum Beispiel "instance_layer" - und kennzeichnen dann das System als persistent oder nicht. Wenn das System nicht als persistent gekennzeichnet ist, wird es automatisch am Ende des Raumes, in dem es erstellt wurde, zerstört (dies ist dasselbe, als wenn du die Funktion part_system_destroy() aufgerufen hätten, und es werden auch alle mit dem System verbundenen Emitter zerstört). Wenn das System jedoch als persistent gekennzeichnet ist, wird es in den nächsten Raum übertragen, wenn der Raum gewechselt wird, und wenn der folgende Raum keine Ebene mit dem gleichen Namen oder der gleichen Tiefe wie die zugewiesene hat, wird eine neue Ebene für das System erstellt, die über die Räume hinweg persistent ist, und sie wird den gleichen Namen wie die ursprüngliche Ebene erhalten. Wenn bei einem Raumwechsel in den folgenden Räumen eine andere Ebene mit demselben Namen vorhanden ist, wird die persistierte Instanz der Ebene mit demselben Namen zugewiesen , unabhängig von der Tiefe der Ebene. Bewegt sich ein persistiertes System schließlich in einen Raum mit einer Ebene in derselben Tiefe wie die Ebene, auf der das System erstellt wurde, wird es nicht dieser Ebene zugewiesen, sondern es wird eine neue Ebene in derselben Tiefe (mit demselben Namen wie die ursprüngliche Ebene) erstellt.

Die Funktion gibt einen eindeutigen ID-Wert für das Partikelsystem zurück, der bei allen weiteren Funktionsaufrufen verwendet werden sollte, bei denen Sie eine System-ID angeben müssen.

WICHTIG! Wenn Sie das Partikelsystem als persistent kennzeichnen, müssen Sie es (und alle ihm zugewiesenen Emitter) manuell mit den entsprechenden Zerstörungsfunktionen bereinigen, wenn es nicht in Gebrauch ist, sonst riskieren Sie ein Speicherleck, das sich negativ auf Ihr endgültiges Spiel auswirkt.

Syntax:

part_system_create_layer(layer_id, persistent);

Argument Typ Beschreibung
layer String oder Ebenen-ID The layer ID (or name) to assign the particle system to (can be any layer type)
persistent Boolesche Flag the particle system as persistent (set to true) oder nicht (eingestellt auf false)

Rückgabe:

Particle System ID

Beispiel:

global.p_sys = part_system_create_layer("effects_layer", true);

Die obige code erstellt ein neues Partikelsystem auf der angegebenen Ebene und kennzeichnet es als dauerhaft für nachfolgende Räume. Die ID für das Partikelsystem wird in einer globalen Bereichsvariablen für zukünftige Referenzen gespeichert.
