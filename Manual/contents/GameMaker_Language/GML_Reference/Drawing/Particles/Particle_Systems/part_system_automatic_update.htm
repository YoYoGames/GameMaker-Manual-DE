teil_system_automatisch_aktualisieren

Diese Funktion steuert, ob GameMaker das Partikelsystem automatisch aktualisieren soll oder nicht. Normalerweise brauchen Sie diese Funktion nicht zu verwenden, da der Standardwert true (automatische Aktualisierung ist eingeschaltet) das ist, was Sie wünschen. Für spezielle Effekte oder um das Spiel zu pausieren, kannst du diese Funktion jedoch auf false setzen, und sie wird verhindern, dass Aktualisierungen für das gegebene Partikelsystem durchgeführt werden, es sei denn, sie werden explizit durch die Verwendung der Funktion part_system_update() oder du verwendest diese Funktion erneut, um die automatische Aktualisierung einzuschalten. Keine Aktualisierung bedeutet, dass Partikel, die in diesem System erstellt wurden, nicht mehr ihre Position, Farbe oder andere Parameter ändern und dass Emitter ebenfalls aufhören zu funktionieren. Das System wird genau an dem Punkt "eingefroren", an dem die automatische Aktualisierung auf false eingestellt wurde.

Zu beachten ist, dass Sie bei der Verwendung der einfachen Effekte, die durch die Funktionen effect_create_above() oder effect_create_below() erstellten einfachen Effekte verwenden, können Sie die Werte 0 (für die unteren Effekte) oder 1 (für die oberen Effekte) als Partikelsystemindex verwenden und so die automatische Aktualisierung auch für diese einschalten (dies funktioniert auch für die GML Visual-Partikel-Effekte).

Syntax:

part_system_automatic_update(ind, automatic);

Argument Typ Beschreibung
ind Partikel System ID The index of the particle system to change.
automatic Boolesche Whether automatic updating is on (true) oder nicht (falsch).

Rückgabe:

N/A

Beispiel:

if global.Pause
{
    part_system_automatic_update(global.Sname, false);
}

code Die obige switch schaltet die Partikel-Updates aus, wenn die globale Variable "Paused" true ist.
