texture_global_scale

Mit dieser Funktion können Sie die Skalierung der Texturseiten beim Laden aus der WAD-Datei steuern, die Teil der endgültigen ausführbaren Datei Ihres Spiels ist. Der Eingabewert muss eine Zweierpotenz sein und wird so funktionieren, dass:

  • Um die Texturseiten auf 1:1 zu setzen, verwenden Sie texture_global_scale(1)
  • Für die Hälfte der Texturen die ganze Zeit verwenden texture_global_scale(2)
  • Um sie zu vierteln, verwenden Sie texture_global_scale(4)...

Auf diese Weise können Sie die Speichernutzung von Texturseiten auf platforms besser kontrollieren, wenn Sie Probleme mit geringem Speicher haben. Beachten Sie, dass Textur-Seiten bei Bedarf aus dem WAD erstellt werden, und Sie können dies entweder zu Beginn des Spiels aufrufen, um alle Texturen skaliert zu laden, oder Sie können zu bestimmten Zeiten aufrufen, um bestimmte Texturen zu laden, in diesem Fall sollten Sie wahrscheinlich draw_texture_flush() vor dem Aufruf dieser Funktion aufrufen und dann die Pre-Fetch-Funktionen verwenden, um die richtigen Textur-Seiten in den Speicher zu bekommen.

Syntax:

texture_global_scale(pow2integer);

ArgumentTyp Beschreibung
pow2integer The scale factor to use (1, no scale, 2, half scale, 4, quarter scale, etc...)

Rückgabe:

Beispiel:

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

Die obige code wird alle Texturen aus dem Speicher flushen, dann die Textur-Skalierung auf 2 setzen (so dass die Textur-Seiten halb so groß sind) und schließlich die Textur-Seite, in der die sprite "spr_Trees" enthalten ist, vorholen.
