gpu_set_tex_mip_filter_ext

Mit dieser Funktion können Sie den Mip-Filtermodus für einen bestimmten Shader-Sampler einstellen. Sie geben den Indexwert für den Shader-Sampler an (wie er von der Funktion shader_get_sampler_index()zurückgegeben wird, und dann eine der unten aufgeführten Moduswertkonstanten.

Konstante Beschreibung
tf_point Das bedeutet, dass das Überblenden zwischen den Mipmap-Ebenen deaktiviert ist, was zu sichtbaren Texturübergängen führen kann, aber die beste Leistung bringt.
tf_linear Das bedeutet, dass das Überblenden zwischen den Mipmap-Ebenen aktiviert ist (auch bekannt als trilineare Filterung), was die Texturübergänge glättet, aber die Leistung ein wenig beeinträchtigt.
tf_anisotropic Dies bedeutet, dass die anisotrope Filterung aktiviert ist, was die Qualität der Texturübergänge erheblich verbessert und die bei anderen Filterungsmodi sichtbare Unschärfe reduzieren kann, aber die Leistung am stärksten beeinträchtigt.

Syntax:

gpu_set_tex_mip_filter_ext(sampler_index, filter);

Argument Typ Beschreibung
sampler_index Shader Sampler Handle The index of the shader sampler to get.
filter Mipmapping Filter Konstante The mip filter mode to use (a constant, default: tf_point).

Rückgabe:

N/A

Beispiel:

var _sampleIndex = shader_get_sampler_index(shd_Glass, "s_Background");
var _spriteTex = sprite_get_texture(sprite_index, 0);
shader_set(shd_Glass);
if (gpu_get_tex_mip_filter_ext(_sampleIndex) != tf_point)
{
    gpu_set_tex_mip_filter_ext(_sampleIndex, tf_point);
}
texture_set_stage(_sampleIndex, _spriteTex);
shader_reset();

Die obige code setzt den Mip-Filter-Modus für den angegebenen Shader-Textur-Sampler auf tf_point (Deaktivierung von Mipmapping), falls er nicht bereits gesetzt wurde.
