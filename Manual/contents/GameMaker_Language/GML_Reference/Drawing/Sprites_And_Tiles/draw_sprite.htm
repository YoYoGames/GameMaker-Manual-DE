zeichnen_sprite

Diese Funktion zeichnet das angegebene sprite und das Unterbild an einer Position innerhalb des Spielraums. Für die sprite kannst du die Instanzvariable sprite_index verwenden, um die aktuelle sprite zu erhalten, die der Instanz zugewiesen ist, auf der code läuft, oder du kannst jedes andere sprite Asset verwenden. Das Gleiche gilt für das Sub-Image, das ebenfalls auf die Instanzvariable gesetzt werden kann image_index gesetzt werden, wodurch das Sub-Image auf das für die aktuelle Instanz sprite ausgewählte gesetzt wird (beachten Sie, dass Sie ein anderes sprite zeichnen und trotzdem den Sub-Image-Wert für die aktuelle Instanz verwenden können), oder Sie können einen beliebigen anderen Wert dafür verwenden, um ein bestimmtes Sub-Image des gewählten sprite zu zeichnen. Wenn der Wert größer ist als die Anzahl der Unterbilder, wählt GameMaker automatisch loop, um das entsprechende Bild auszuwählen (wenn z. B. das sprite, das gezeichnet werden soll, 5 Unterbilder mit den Nummern 0 bis 4 hat und wir den Indexwert auf 7 setzen, dann zeichnet die Funktion das Unterbild 3 mit der Nummer 0). Schließlich ist die x- und y-Position die Position innerhalb des Raums, an der sprite gezeichnet wird, und sie ist zentriert auf den sprite x-Offset und y-Offset.

HINWEIS Diese Funktion funktioniert möglicherweise nicht wie erwartet, wenn Sie die Skelettanimation sprites verwenden, und Sie werden feststellen, dass die Funktion nur das erste Bild der Standardpose zeichnet. Sie sollten stattdessen die draw_skeleton_* Funktionen verwenden.

Syntax:

draw_sprite(sprite, subimg, x, y);

Argument Typ Beschreibung
sprite Sprite-Asset The index of the sprite to draw.
subimg Real The sub-image (frame) of the sprite to draw (image_index or -1 correlate to the current frame of animation in the object).
x Real The x coordinate of where to draw the sprite.
y Real The y coordinate of where to draw the sprite.

Rückgabe:

N/A

Beispiel:

draw_sprite(sprite_index, image_index, x, y);
draw_sprite(spr_Halo, 0, x, y-32);

Dies zeichnet die Instanz, der sprite (sprite_index) zugewiesen ist, mit dem aktuellen Teilbild an der x- und y-Position der Instanz innerhalb des Raumes. Dann wird das erste Bild des sprite, das durch "spr_Halo" indiziert ist, an der gleichen x- und y-Position gezeichnet, aber 32 Pixel "darüber".
