draw_enable_drawevent

Mit dieser Funktion können Sie wählen, ob Sie das Draw-Ereignis für alle Instanzen im Spiel aktivieren (true) oder deaktivieren (false) wollen. Damit haben Sie die Kontrolle darüber, wie und wann Dinge gezeichnet werden, was nützlich ist, wenn Sie eine "Frame-Skip"-Technik implementieren wollen. Dies ist nützlich, wenn Sie eine "Frame-Skip"-Technik implementieren möchten. Beachten Sie, dass dies nicht nur verhindert, dass Instanzen auf den Bildschirm gezeichnet werden, sondern dass das Draw-Ereignis komplett unterdrückt wird, was bedeutet, dass Sie vorsichtig sein sollten, da jegliche Spiellogik, die in diesem Ereignis vorhanden ist, ebenfalls nicht ausgeführt wird.

Eine wichtige Sache, die man über diese Funktion wissen muss, ist, dass, wenn man sie gleich zu Beginn des Spiels aufruft, bevor der erste Frame gerendert wird (d.h.: das Create Event des ersten Objekts im ersten Raum des Spiels), dann wird das Spielfenster nicht gerendert. Dies könnte für Multiplayer-Projekte nützlich sein, die ein dediziertes server benötigen, das nichts rendern muss. Beachten Sie jedoch, dass es dadurch nicht headless wird, da GameMaker dies nicht unterstützt - Sie können es also nicht auf einem server ausführen, das keine grafische Oberfläche hat.

Syntax:

draw_enable_drawevent(enable);

ArgumentTyp Beschreibung
Enable Set to true oder false.

Rückgabe:

Beispiel:

frame_skip ++;
if (frame_skip mod 5) == 0
{
    draw_enable_drawevent(true);
}
else
{
    draw_enable_drawevent(false);
}

Die obige code prüft eine Variable, und wenn sie true zurückgibt, wird das Ereignis "Zeichnen" aktiviert, andernfalls wird das Ereignis "Zeichnen" deaktiviert.
