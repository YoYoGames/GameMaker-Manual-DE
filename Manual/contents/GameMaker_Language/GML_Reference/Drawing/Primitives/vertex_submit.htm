vertex_submit

Sie können diese Funktion verwenden, um den Inhalt eines Vertex buffer an die Grafikpipeline zur Verwendung mit einem Shader zu übermitteln. Sie geben den zu verwendenden buffer Index, den zu verwendenden primitiven Basistyp (siehe die Konstanten unten) und die zu verwendende Textur an. Der primitive Basistyp wird nur für die Zuweisung der Reihenfolge verwendet, in der die Scheitelpunkte, die Sie in buffer gespeichert haben, gezeichnet und verbunden werden, aber die tatsächlichen Daten, die für jeden der Scheitelpunkte verwendet werden, sind die, die Sie beim Erstellen des Scheitelpunktes buffer definiert haben.

HINWEIS: Diese Funktion kann nur im Ereignis "Zeichnen" verwendet werden.

Ein visuelles Beispiel für die verschiedenen verfügbaren Basisprimitive finden Sie in der nachstehenden Abbildung:

Konstante Beschreibung
pr_pointlist Ein Primitiv, das aus einer Liste von Punkten besteht.
pr_linelist Ein Primitiv, das aus einzelnen Zeilen einer Liste besteht.
pr_linestrip Ein Primitivum, das aus einem aufeinanderfolgenden Streifen von Linien besteht.
pr_trianglelist Ein Primitivum, das aus einzelnen Dreiecken in einer Liste besteht.
pr_trianglestrip Ein Primitivum, das aus einem aufeinanderfolgenden Streifen von Dreiecken besteht.

Syntax:

vertex_submit(buffer, primitive, texture);

Argument Typ Beschreibung
buffer Vertex Buffer ID The buffer to use.
primitive Primitiver Typ Konstante The primitive base type.
texture Textur-Zeiger The texture to use (or -1 for no texture).

Rückgabe:

N/A

Beispiel:

shader_set(shader_prim);
vertex_submit(buff, pr_trianglelist, sprite_get_texture(sprite_index));
shader_reset();

Die obige code wird den in der Variable "buff" indizierten Vertex buffer zum Zeichnen mit dem Shader "shader_prim" übermitteln, wobei eine Dreiecksliste als Basisprimitiv und die Textur des sprite für die Instanz, auf der die code läuft, verwendet wird.
