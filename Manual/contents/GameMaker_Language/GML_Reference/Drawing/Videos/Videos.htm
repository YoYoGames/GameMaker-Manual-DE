Video-Wiedergabe

Wie spielt man Videos ab?

Befolgen Sie diese Schritte, um Videos auf runtime zu laden und abzuspielen:

  • Fügen Sie eine Videodatei zu den eingeschlossenen Dateien( Verzeichnis/datafiles ) Ihres Projekts hinzu.
  • Laden Sie das Video mit video_open().
    • Alternativ können Sie auch ein Video von einer URL laden.
  • Zeichnen Sie sie in einem Draw-Ereignis mit video_draw().
    • Diese Funktion zeichnet das Video auf eine interne Oberfläche, die sie als Teil ihres Rückgabe-Arrays zurückgibt.
    • Sie müssen diese Oberfläche manuell mit draw_surface() zeichnen, damit das Video erscheint.
      • Das Zeichnen von YUV-Videos erfordert zusätzliche Schritte.
  • Wenn das Video beendet ist, entfernen Sie es aus dem Speicher, indem Sie video_close() aufrufen.
    • Verwenden Sie die Async Callbacks, um code auszuführen, wenn das Video endet.

Video Codecs & Lizenzierung

GameMaker wird nicht mit Codecs ausgeliefert, sondern verwendet die Codecs, die auf dem platform vorhanden sind, auf dem das Spiel läuft. Das bedeutet, dass Ihre Videos in einem Format kodiert sein müssen, das auf der/den vorgesehenen Zielplattform(n) dekodiert werden kann.

Für die Verwendung bestimmter Formate müssen Sie möglicherweise Lizenzgebühren entrichten, die von der Lizenz des jeweiligen Codecs und den Einnahmen Ihres Spiels abhängen. Codecs werden auch auf jedem platform unterschiedlich lizenziert, so dass bei Spielen, die auf mehreren platforms vertrieben werden, Vorsicht geboten ist.

Beachten Sie, dass Videoformate (wie z. B. .mp4, .avi, .mkv usw.) einfach Container sind, die Video- und Audioströme umhüllen, die beide separat kodiert werden können und möglicherweise separate Lizenzen erfordern.

Für die Verbreitung auf der Konsole platforms erkundigen Sie sich bei Ihrem Kundenbetreuer, welche Formate Sie verwenden dürfen, ohne zusätzliche Lizenzgebühren zahlen zu müssen.

Formate

Die Videoformate, die Sie in Ihr Spiel laden und abspielen können, hängen von Ihrem Ziel ab platform, und den Codecs, die von diesem platform unterstützt werden.

Funktionen

Sie können Videodateien unter runtime mit den folgenden Funktionen laden, abspielen und bearbeiten:

  • video_öffnen
  • video_schließen
  • Video_Zeichnen
  • video_set_volume
  • video_pause
  • video_resümee
  • video_aktivieren_schleife
  • video_seek_to

Die folgenden Funktionen werden verwendet, um Informationen über das aktuell geladene Video zu erhalten:

  • video_ist_schleppend
  • video_get_volume
  • video_get_duration
  • video_get_position
  • video_get_status
  • video_get_format

Asynchrone Rückrufe

Das Async Social-Ereignis wird in den unten angegebenen Phasen ausgelöst, wobei der Schlüssel "type" in der async_load-Map auf den angegebenen Wert gesetzt wird:

  • "video_start": Wenn die Wiedergabe des Videos beginnt, d. h. wenn der Status von video_status_preparing zu video_status_playing wechselt.
  • "video_end": Wenn das Video endet. Dies wird nicht ausgelöst, wenn das Video auf Schleife eingestellt ist.

Hier ein Beispiel für ein Async Social-Ereignis, das auf Video-Rückrufe reagiert:

var _type = async_load[? "type"];

if (_type == "video_start")
{
    obj_video.display_video = true;
}
else if (_type == "video_end")
{
    obj_video.display_video = false;
    video_close();
}

Die obige code liest den Typ aus der async_load-Map. Wenn das Video startet, wird eine Variable in einem Objekt auf true gesetzt, damit das Video angezeigt werden kann.

Wenn das Video endet, wird dieselbe Variable auf false gesetzt und dann video_close() aufgerufen, um das Video aus dem Speicher zu entfernen.
