oberfläche_verkleinern

Diese Funktion ändert die Größe einer Fläche auf die angegebenen Maße (in Pixeln). Die "surface_id" ist die einer Fläche, die Sie zuvor erstellt haben, oder die application_sufaceund die Funktion wird die Größe dieser Fläche ändern. Beachten Sie, dass der Inhalt der Fläche weder beschnitten noch gestreckt wird, sondern dass die aktuelle Fläche zerstört und mit demselben Handle (surface_id) mit den neuen Abmessungen neu erstellt wird. Das bedeutet, dass die Fläche gelöscht und neu gezeichnet werden muss (es sei denn, es handelt sich um die application_surface, in diesem Fall macht GameMaker dies automatisch).

HINWEIS: Wenn Sie die Größe der Anwendungsoberfläche ändern, werden diese Änderungen erst zu Beginn des nächsten Zeichnungsrahmens registriert, was bedeutet, dass der Aufruf der surface_get_width() oder surface_get_height() Funktionen im selben Ereignis oder Schritt die vorherigen Werte zurückgeben.

Syntax:

surface_resize(surface_id, w, h);

ArgumentTyp Beschreibung
surface_id The ID of the surface to change.
w The width of the new surface.
h The height of the new surface.

Rückgabe:

Beispiel:

if view_wport[0] != surface_get_width(application_surface) || view_hport[0] != surface_get_height(application_surface)
{
    surface_resize(application_surface, view_wport[0],view_hport[0]);
}

Die obige code prüft die Größe des Ansichtsports mit der der Oberfläche "aplication_surface", und wenn sie unterschiedlich ist, wird die Oberfläche neu dimensioniert.
