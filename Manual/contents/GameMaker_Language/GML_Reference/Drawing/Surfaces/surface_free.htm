Oberfläche_frei

Wenn Sie mit Flächen arbeiten, sollten Sie diese Funktion immer dann verwenden, wenn Sie die Flächen nicht mehr benötigen. Oberflächen belegen Speicherplatz und müssen daher entfernt werden, normalerweise am Ende eines Raums, aber das kann jederzeit geschehen, je nachdem, wofür Sie sie verwenden. Andernfalls kann es zu Speicherlecks kommen, die das Spiel verlangsamen und schließlich zum Absturz bringen.

HINWEIS: Bei der Arbeit mit Flächen besteht die Möglichkeit, dass sie jederzeit nicht mehr existieren, da sie im Texturspeicher abgelegt sind. Sie sollten IMMER überprüfen, ob eine Fläche existiert, indem Sie surface_exists() überprüfen, bevor Sie sie direkt referenzieren.

Syntax:

surface_free(surface_id);

ArgumentTyp Beschreibung
surface_id The ID of the surface to be freed.

Rückgabe:

Beispiel:

if keyboard_check_pressed(vk_escape)
{
    surface_free(surf);
    room_goto(rm_Menu);
}

Die obige code prüft, ob eine Taste gedrückt wurde, und wenn sie eine entdeckt, gibt sie den Speicher frei, der für die in der Variablen "surf" indizierte Oberfläche reserviert ist, und wechselt dann den Raum.
