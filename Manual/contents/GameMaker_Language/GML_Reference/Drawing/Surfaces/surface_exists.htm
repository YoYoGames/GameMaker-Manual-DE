oberfläche_existiert

Diese Funktion ist bei der Arbeit mit Oberflächen aufgrund ihrer flüchtigen Natur unerlässlich. Oberflächen werden immer im Texturspeicher gehalten, was bedeutet, dass sie von einem Moment auf den anderen zerstört werden können (z.B. wenn ein Bildschirmschoner auf Fenstern gestartet wird oder wenn sie auf einem Android Gerät minimiert werden), so dass Sie immer prüfen sollten, ob eine Oberfläche existiert, bevor Sie etwas mit ihr tun (dies schließt das Zeichnen auf dem Bildschirm ein). Das folgende Beispiel code zeigt eine typische Verwendung dieses Befehls im Draw-Ereignis einer Instanz, um zu prüfen, ob eine Oberfläche vorhanden ist, und sie neu zu erstellen, wenn sie entfernt wurde (beachten Sie, dass die Oberfläche ursprünglich im Create-Ereignis des Objekts erstellt wurde).

Syntax:

surface_exists(surface);

ArgumentTyp Beschreibung
surface_id The ID of the surface to check.

Rückgabe:

Beispiel:

if !surface_exists(surf)
{
    surf = surface_create(1024, 1024);
}
else
{
    if (view_current == 1)
    {
        draw_surface(surf, 0, 0);
    }
}

Die obige code prüft, ob eine in der Variablen "surf" indizierte Oberfläche existiert, und wenn nicht, wird sie neu erstellt. Ist dies der Fall, wird geprüft, welche Ansicht gerade gezeichnet wird, und wenn es Ansicht[1] ist, wird die Oberfläche gezeichnet.
