file_copy

Diese Funktion kopiert die angegebene Datei und gibt ihr den von Ihnen gewählten neuen Namen.

Wenn Sie eine Datei auf Android aus dem Bereich " Enthaltene Dateien" (der sich in Ihrer APK befindet) kopieren, wird sie zuerst in den Speicher kopiert. Seien Sie sich also bewusst, dass das Kopieren sehr großer Dateien aus der APK dazu führen kann, dass den Geräten der Speicher ausgeht. Auf dem HTML5-Ziel verwenden wir die HTML5 Local Storage API für das Speichern auf der Client-Seite, und die Funktionen unterstützen keine binären Dateioperationen, so dass wir davon ausgehen, dass Sie eine Textdatei kopieren.

Die Funktion schlägt fehl, wenn die Datei newname bereits existiert.

WARNUNG! Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker in einer Sandbox arbeitet! Weitere Informationen finden Sie im Abschnitt über Dateisystembeschränkungen.

Syntax:

file_copy(fname, newname);

ArgumentTyp Beschreibung
fname The name of the file to copy.
newname The name of the new, copied file.

Rückgabe:

N/A

Beispiel:

if file_exists("level1.txt")
{
    file_copy("level1.txt", "level2.txt");
}

Dabei wird nach einer Datei gesucht, und wenn sie existiert, wird sie kopiert.
