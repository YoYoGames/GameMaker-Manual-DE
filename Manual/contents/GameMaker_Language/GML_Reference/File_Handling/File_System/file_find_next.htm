file_find_next

Diese Funktion gibt den Namen der nächsten Datei zurück, die der zuvor angegebenen Maske und den Attributen (definiert durch file_find_first()). Wenn keine solche Datei existiert, wird die leere Zeichenkette zurückgegeben.

WARNUNG Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker in einer Sandbox arbeitet! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

HINWEIS Diese Funktion funktioniert bei allen C++-Zielen platforms (Windows, Mac, iOS, Android), ABER der Filter flags funktioniert nur bei Windows.

Syntax:

file_find_next();

Rückgabe:

Beispiel:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

code prüft, ob das angegebene Verzeichnis existiert, und wenn ja, geht es dorthin und gibt die erste gefundene "schreibgeschützte" .doc Datei zurück. Dann sucht es nach zwei weiteren Dateien und schließt den Dateifinder.

Sie können nach einer beliebigen Anzahl von Dateien suchen, indem Sie eine while Schleife:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

Das obige code erstellt ein leeres Array, um alle gefundenen Dateinamen zu speichern, und beginnt mit der Suche nach Dateien, die nur gelesen werden können .doc. Wenn der Dateiname keine leere Zeichenkette ist, wird er in das Array files verschoben, und dann wird versucht, die nächste Datei zu finden. loop wird so lange wiederholt, bis eine leere Zeichenkette gefunden wird (was bedeutet, dass es keine passenden Dateien mehr gibt). Dann wird loop beendet und der Dateifinder am Ende geschlossen.
