Ini-Dateien

INI-Dateien sind kleine, leichtgewichtige Dateien, die mit den meisten platforms kompatibel sind. Sie sind ideal, um kleine Informationen zu speichern, wie z.B. Interface-Einstellungen, lokale Highscores, Level-Daten usw... und sind somit eine großartige und einfache Möglichkeit, um Informationen aus Ihren Spielen zu speichern. Wenn Sie nach fortgeschritteneren Möglichkeiten suchen, um Informationen in Dateien zu speichern, sollten Sie sich einige der anderen Dateifunktionen wie Textdateien, Binärdateien und Puffer ansehen.

WICHTIG! Alle GameMaker-Projekte haben eine Datei namens "options.ini", Sie können also keine Datei mit diesem Namen benennen (Sie erhalten einen uid-Fehler, wenn Sie versuchen, Ihr Spielprojekt zu kompilieren).

Bitte beachten Sie, dass es für Spiele mit Lokalisierungen sehr wichtig ist, dass akzentuierte Buchstaben aus externen Dateien gelesen werden können. Das bedeutet, dass Sie die ini-Datei zunächst im UTF8-Format erstellen und sie dann in GameMaker als eingebundene Datei hinzufügen sollten, damit sie beim Ausführen des Spiels exportiert und anstelle der standardmäßigen ini-Datei im ANSI-Format verwendet wird, die von den ini-Funktionen erstellt wird, wenn zuvor keine angegeben wurde. Auf diese Weise können Sie die Datei korrekt lesen und schreiben, wobei alle Akzente und nicht-römischen Buchstaben erhalten bleiben.

Für ini-Dateien gibt es die folgenden Funktionen:

  • ini_open
  • ini_close
  • ini_write_real
  • ini_write_string
  • ini_read_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
