ini_open

Dies öffnet eine ini_file zum Lesen und Schreiben. Wenn die ini_file an dem Ort, den Sie prüfen, nicht existiert, kann GameMaker eine erstellen, aber nur, wenn Sie Daten in sie schreiben. Wenn Sie nur Informationen aus der ini-Datei gelesen haben, dann werden die Standardwerte für die Lesefunktion zurückgegeben, aber die ini-Datei wird nicht erstellt.

Bitte beachten Sie, dass Sie immer nur eine Ini-Datei geöffnet haben können und dass Sie ini_close() verwenden, sobald Sie mit dem Lesen/Schreiben der Ini-Datei fertig sind, da die Informationen erst dann auf der Festplatte gespeichert werden. Festplatte gespeichert werden (sie werden auch im Speicher gehalten, bis die Datei geschlossen wird).

WARNUNG! Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker in einer Sandbox läuft! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

Syntax:

ini_open(name);

ArgumentTyp Beschreibung
name The filename for the .ini file.

Rückgabe:

Beispiel:

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Dadurch wird die Datei "savedata.ini" geöffnet und der Wert "score" im Abschnitt "save1" mit dem Schlüssel "score" ausgelesen, dann wird die Datei wieder geschlossen. Sollte es keinen Wert unter "save1", "score" oder keine "savedata.ini" geben Datei vorhanden ist, wird score auf 0 gesetzt (der Standardwert). Beachten Sie, dass sich die Ini-Datei im Unterverzeichnis "Settings" befindet, dem Ordner, in dem sich die Ini-Datei der im Asset-Browser enthaltenen Dateien befindet.
