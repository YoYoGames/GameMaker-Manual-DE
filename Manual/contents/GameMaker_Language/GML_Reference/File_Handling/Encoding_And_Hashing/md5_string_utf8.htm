md5_string_utf8

In der Kryptografie ist MD5 (Message-Digest-Algorithmus 5) eine weit verbreitete kryptografische Hash-Funktion mit einem 128-Bit-Hash-Wert, die in einer Vielzahl von Sicherheitsanwendungen eingesetzt wird. Sie wird auch häufig verwendet, um die Integrität von Dateien und Zeichenketten zu überprüfen. Diese Funktion nimmt eine UTF-8-Eingabezeichenfolge (mit einer variablen Anzahl von Bytes pro Zeichen) und gibt den 32-stelligen hexadezimalen MD5-Hashwert zurück, der für diese Zeichenfolge eindeutig ist. Auf diese Weise können Sie einen sicheren Schlüssel erzeugen, der gespeichert und verwendet werden kann um die Integrität der an eine externe server gesendeten (oder von ihr empfangenen) Informationen zu überprüfen.

HINWEIS: Es gibt zwei Formate für die MD5-Kodierung, UTF-8 und Unicode. Beide Formate sind vorgesehen, um die Kommunikation mit verschiedenen server zu erleichtern, aber am häufigsten wird Unicode verwendet.

HINWEIS: MD5 ist nicht völlig sicher und kann geknackt werden. Siehe diese Seite für weitere Informationen.

Syntax:

md5_string_utf8(string)

ArgumentTyp Beschreibung
string The string to hash.

Rückgabe:

Beispiel:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Die obige code kodiert eine DS-Map in einen JSON-String. Anschließend wird ein MD5-Hash generiert und in einer Ini-Datei gespeichert, so dass dieser später zur Überprüfung der Integrität des JSON verwendet werden kann, falls die gleichen Informationen später von server empfangen werden. Das JSON wird dann gesendet.
