sha1_string_unicode

In der Kryptografie ist SHA-1 eine kryptografische Hash-Funktion, die von der Nationalen Sicherheitsbehörde der Vereinigten Staaten entwickelt wurde und in mehreren weit verbreiteten Anwendungen und Protokollen wie dem beliebten Git eingesetzt wird, wo sie zur Überprüfung von Dateiänderungen verwendet wird. Diese Funktion nimmt eine Unicode-Zeichenkette als Eingabe und gibt einen 160-Bit-Message-Digest im ASCII-Format zurück, zum Beispiel:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

Auf diese Weise können Sie einen sicheren Schlüssel generieren, der gespeichert und verwendet werden kann, um die Integrität der Informationen zu prüfen, die z. B. an eine externe server gesendet (oder von dieser empfangen) werden.

HINWEIS: Es gibt zwei Formate für die sha1-Kodierung, UTF-8 und Unicode. Beide sind vorgesehen, um die Kommunikation mit verschiedenen server zu erleichtern, aber am häufigsten wird Unicode verwendet.

HINWEIS: SHA-1 ist nicht völlig sicher und kann geknackt werden. Siehe diese Seite für weitere Informationen.

Syntax:

sha1_string_unicode(string)

ArgumentTyp Beschreibung
string The string to hash.

Rückgabe:

Beispiel:

var hash, str;
str = base64_encode(game_data);
hash = sha1_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

Die obige code kodiert eine Zeichenkette mit base64 und erzeugt dann einen sha1-Hash. Schließlich werden sowohl der Hash als auch die kodierte Zeichenkette an eine server gesendet.
