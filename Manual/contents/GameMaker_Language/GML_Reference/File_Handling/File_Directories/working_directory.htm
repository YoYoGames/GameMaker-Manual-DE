working_directory

Working_directory kann tatsächlich zwei verschiedene Werte zurückgeben, je nachdem, wofür Sie es verwenden.

Wenn Sie eine Datei auf die Festplatte schreiben, zeigt working_directory auf den Bereich des lokalen Speichers, der für Ihr Spiel auf dem Zielgerät reserviert ist (in Windows ist dies "%LOCALAPPDATA%/gamedir/", wobei "gamedir" das Verzeichnis mit dem Namen Ihres Spiels ist).

Beim Lesen von der Festplatte kann working_directory jedoch entweder der lokale Speicher (game_save_id) oder der Bereich sein, in dem die eingeschlossenen Dateien gespeichert sind.

HINWEIS Die Variable working_directory gibt den Pfad einschließlich des letzten Backslashs zurück.

Wenn Sie z. B. eine Standarddatei *.txt in Ihr Spiel eingebunden haben und diese mit working_directory in den Speicher einlesen, wird diese Datei aus dem Bereich entnommen, in dem die eingebundenen Dateien gespeichert sind. Wenn Sie diese Informationen dann mit working_directory in eine Textdatei schreiben, wird die Datei in den lokalen Speicher der Zielplattform geschrieben, von wo sie beim nächsten Mal gelesen wird, es sei denn, Sie weisen GameMaker an, die Datei zu löschen. In diesem Fall verweist working_directory erneut auf den Bereich, in dem die Included Files gespeichert sind.

Es sei darauf hingewiesen, dass es im Allgemeinen nicht notwendig ist, diese Option zu verwenden, sondern dass es ausreicht, den Dateinamen selbst (und etwaige zusätzliche Pfadinformationen) anzugeben. Zum Beispiel dies:

file_text_open_read(working_directory + "my_file.txt");

wäre besser so zu machen:

file_text_open_read("my_file.txt");

WARNUNG Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker in einer Sandbox arbeitet! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

Syntax:

working_directory

Rückgabe:

String

Beispiel:

ini_open(working_directory + "temp_ini.ini");

Dies öffnet eine Ini-Datei aus dem Arbeitsverzeichnis des Spiels (und erstellt sie, falls sie noch nicht vorhanden ist). Dies kann der lokale Speicher oder der Bereich sein, in dem sich die eingebundenen Dateien befinden, je nachdem, ob die gesuchte Datei an einem der beiden Orte existiert.
