Handhabung von Dateien

Irgendwann bei der Entwicklung Ihres Spiels wird es notwendig sein, Informationen extern zu speichern und diese Informationen abrufen zu können. Zum Beispiel möchten Sie vielleicht Anmeldedaten, Höchstpunktzahlen oder den Fortschritt des Spielers durch Ihre Levels speichern. Was auch immer der Grund ist, Sie müssen wissen, wie Sie die verschiedenen Dateifunktionen nutzen können, die Ihnen bei der Arbeit mit GameMaker zur Verfügung stehen.

Es ist sehr wichtig zu wissen, dass GameMaker standardmäßig eine Sandbox ist, was bedeutet, dass es eine Reihe von Einschränkungen und kleineren Komplikationen bei der Verwendung der verfügbaren Dateisystemfunktionen gibt. Es ist auch erwähnenswert, dass das Dateisystem je nach platform, auf das Sie zielen, etwas anders funktioniert. Weitere Informationen hierzu finden Sie im folgenden Abschnitt:

  • Das Dateisystem

Sie können das Sandboxing auf den Desktop-Zielen (Windows, macOS und Ubuntu (Linux)) in den Spieloptionen für das gewählte platform deaktivieren, so dass Sie dann von überall aus speichern und laden können, innerhalb der Grenzen der Betriebssystemberechtigungen.

Wenn Sie von Steam die Erlaubnis erhalten haben, ihre spezielle API mit Ihrem Spiel zu verwenden, werden Sie feststellen, dass Ihnen auch ein neuer Satz von Dateifunktionen zur Verfügung steht. Diese werden im Abschnitt über die Steam-API in diesem Handbuch erläutert.

Schließlich sollten Sie auch beachten, dass es einige deutliche Unterschiede zwischen bestimmten Zielen platforms gibt, die bei der Verwendung der Dateifunktionen zu berücksichtigen sind. Diese sind:

  • iOS: Wenn Sie unter iOS eine eingebundene Datei aus einem Ordner laden, d. h. mit einem Pfad wie z. B. "Backgrounds/background1.png", müssen Sie den Ordner NICHT als Teil des Pfads angeben, sondern es reicht aus, den Dateinamen anzugeben. Bei allen anderen platforms müssen Sie den/die Ordner als Teil des Pfades zur Datei angeben.
  • macOS und Ubuntu (Linux): Auf macOS und Ubuntu wird bei Dateinamen und Pfaden auf platforms zwischen Groß- und Kleinschreibung unterschieden. Um mögliche Probleme zu vermeiden, verwendet GameMaker standardmäßig für alle Namen/Pfade Kleinschreibung.
  • HTML5: Bei HTML5 funktioniert das Speichern und Laden in und aus dem lokalen Speicher wie auf den verschiedenen Funktionsseiten beschrieben. Wenn Sie jedoch eine Datei von server laden, dürfen Sie nur die Funktion buffer_load_async()verwenden, da das synchrone Laden in den meisten Browsern veraltet ist und in absehbarer Zeit überflüssig werden wird. Beachten Sie auch, dass Sie im lokalen Speicher des Browsers keine Verzeichnisse erstellen oder löschen können.
