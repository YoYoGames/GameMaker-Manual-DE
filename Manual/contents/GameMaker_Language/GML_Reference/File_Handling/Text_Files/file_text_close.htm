file_text_close

Sobald Sie die Arbeit mit einer bestimmten Datei beendet haben (egal ob Sie aus ihr lesen oder in sie schreiben), müssen Sie die Datei wieder schließen, da sonst die Gefahr besteht, dass Sie die darin enthaltenen Informationen verlieren. Auf diese Weise wird auch verhindert, dass Speicherlecks entstehen, und es wird sichergestellt, dass Sie die Dateibeschränkung nicht überschreiten, wenn Sie mehr als 32 Dateien geöffnet haben.

Syntax:

file_text_close(fileid);

ArgumentTyp Beschreibung
fileid The id of the file to close.

Rückgabe:

Beispiel:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
    file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

Die obige code öffnet eine Datei zum Schreiben und geht dann loops durch die bereits in die Datei geschriebenen Textzeilen, bis sie das Ende erreicht. An diesem Punkt wird eine Zeichenkette geschrieben und die Datei wieder geschlossen.
