Text-Dateien

Textdateien sind eine nützliche Methode, um große Datenmengen außerhalb des Spiels zu speichern. Im Gegensatz zu ini-Dateien können Sie bis zu 32 Textdateien gleichzeitig geöffnet haben und auch große Datenmengen in diese Dateien schreiben und lesen. Sie können jedoch selbst entscheiden, wie diese Daten strukturiert werden sollen, da es keine feste "Abschnitts-" oder "Schlüsselstruktur" wie bei *.ini gibt.

Bitte beachten Sie, dass es für Spiele mit Lokalisierungen sehr wichtig ist, dass akzentuierte Buchstaben aus externen Dateien gelesen werden können. Das bedeutet, dass Sie die txt-Datei zunächst im UTF8-Format erstellen und sie dann in GameMaker als eingebundene Datei hinzufügen sollten, damit sie beim Ausführen des Spiels exportiert und anstelle der standardmäßigen txt-Datei im ANSI-Format verwendet wird, die von den GameMaker-Dateifunktionen erstellt wird, wenn zuvor keine Datei bereitgestellt wurde. Auf diese Weise können Sie die Datei korrekt lesen und schreiben, wobei alle Akzente und nicht-römischen Buchstaben erhalten bleiben.

Es gibt die folgenden Funktionen, die sich mit Dateien befassen:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
