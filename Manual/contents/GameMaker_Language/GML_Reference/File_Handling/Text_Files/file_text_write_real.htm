file_text_write_real

Mit dieser Funktion können Sie eine Zahl in die zuvor geöffnete Textdatei schreiben. Da der zu schreibende Wert eine reelle Zahl sein kann, werden alle Dezimalstellen mit einem "."-Punkt als Trennzeichen geschrieben. Wenn die Datei bereits Informationen enthält, werden diese Informationen gelöscht und die Zeichenkette wird an den Anfang der Datei geschrieben, es sei denn, Sie haben die Datei mit der Funktion file_text_open_append(). Sie können dies auch vermeiden, indem Sie die Funktion file_text_readln() Funktion zusammen mit der file_text_eof() die Funktion loop verwenden, um den Inhalt der Datei bis zum Ende zu durchsuchen und dann mit dem Schreiben zu beginnen.

Es ist wichtig zu beachten, dass sehr große Zahlen, die mit dieser Funktion in eine Textdatei geschrieben werden, möglicherweise in ein vereinfachtes Standardformat wie "6.6624e+003" umgewandelt werden, das nicht korrekt in GameMaker zurückgelesen werden kann. Um solche Probleme zu vermeiden, sollten Sie den Wert stattdessen in einen String konvertieren und die Funktion file_text_write_string() verwenden.

Syntax:

file_text_write_real(fileid, val);

ArgumentTyp Beschreibung
fileid The id of the file to edit.
val The real value to write to the file.

Rückgabe:

Beispiel:

var file = file_text_open_write(working_directory + "hiscore.txt");
for (var i = 0; i < 10; ++i;)
{
    file_text_write_real(file, scr[i]);
    file_text_writeln(file);
    file_text_write_string(file, scr_name[i]);
    file_text_writeln(file);
}
file_text_close(file);

Die obige code öffnet eine Datei zum Schreiben und loops durchläuft dann zwei Arrays, wobei jeder Array-Wert in eine neue Zeile der Datei geschrieben wird. Die Datei wird dann geschlossen, wenn die loop beendet ist.
