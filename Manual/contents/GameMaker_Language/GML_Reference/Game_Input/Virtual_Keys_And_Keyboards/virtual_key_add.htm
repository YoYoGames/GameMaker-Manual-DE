virtual_key_add

Mit dieser Funktion können Sie "Berührungen" eines Bildschirmbereichs Tastaturereignissen zuordnen. Das bedeutet, dass, sobald Sie einen Bereich einer virtuellen Taste zugewiesen haben, alle Berührungen dieses Bereichs das Tastaturereignis auslösen, das der Taste entspricht, die Sie dem Bereich zugewiesen haben. Sie können jede von Ihnen definierte virtuelle Taste auch einer Variablen zuweisen, die dann in den weiteren Funktionen für virtuelle Tasten verwendet werden kann, um sie ein- und auszublenden oder zu löschen. Diese Tasten werden pro Raum zugewiesen und bei einem Raumwechsel vom GameMaker automatisch entfernt.

Die tatsächliche Position der virtuellen Taste basiert auf der Bildschirmposition und nicht auf der Raumposition, so dass die x/y-Werte absolut auf dem Bildschirm sind. Das bedeutet, dass Sie sich keine Gedanken über die Verwendung von Ansichten oder die relativen Raumkoordinaten machen müssen. Sie können die Tasten einfach im Create-Event eines Objekts definieren (Sie müssen eine virtuelle Taste nur einmal pro Raum definieren) und dann Ihre Taste sprites im Draw-GUI-Event zeichnen, wobei die Größe der GUI-Ebene mit der des Bildschirms übereinstimmen sollte.

Syntax

virtual_key_add(x, y, w, h, keycode);

Argument Typ Beschreibung
x Real The x coordinate (left side) of the virtual key on the screen
y Real The y coordinate (top side) of the virtual key on the screen
w Real The width of the virtual key
h Real The height of the virtual key
keycode Virtuelle Schlüsselkonstante (vk_*) Which keyboard key event should be triggered by touching this area

Rückgabe:

Virtual Key ID

Beispiel:

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

Die obige code erstellt eine virtuelle Taste im Format 64x64 Pixel, die auf dem Bildschirm bei (32, 32) positioniert ist und bei Berührung das Ereignis vk_left auslöst, und weist den Index dieser virtuellen Taste einer globalen Variablen zu.
