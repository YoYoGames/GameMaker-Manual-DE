keyboard_virtual_show

Diese Funktion kann verwendet werden, um die virtuelle Tastatur auf dem Gerät anzuzeigen, auf dem das Spiel läuft. Wenn Sie diese Funktion aufrufen, müssen Sie für jedes der ersten drei Argumente eine der folgenden Konstanten angeben:

  • Tastaturtyp: Mit dieser Konstante wird festgelegt, welcher Tastatursatz auf der virtuellen Tastatur verfügbar ist. Die verfügbaren Tastaturtypen sind:
Virtuelle Tastatur Typ Konstante
Tastatur Typ Beschreibung
kbv_type_default Der Standard-Tastaturtyp für das aktuelle System.
kbv_type_ascii Eine reine ASCII-Tastatur.
kbv_type_url Eine normale Tastatur, die für die Eingabe von URLs optimiert ist. Verfügt in der Regel über eine ".com"-Taste oder andere Domänen-Tasten sowie "/"- und "."-Tasten.
kbv_type_email Eine normale Tastatur, die für die Eingabe von E-Mails optimiert ist. Enthält normalerweise die Zeichen "@" und ".".
kbv_type_numbers Eine reine Zahlentastatur, die in der Regel als Ziffernblock dargestellt wird.
kbv_type_phone Eine Telefontastatur. Normalerweise nur Zahlen mit den Tasten "*" und "#".
kbv_type_phone_name Eine Tastatur, die für die Eingabe einer Telefonnummer und eines Namens optimiert ist. In der Regel ähnlich wie eine ASCII-Tastatur, jedoch mit einer begrenzten Auswahl an Sonderzeichen.

  • Rückgabetyp: Diese Konstante wird verwendet, um festzulegen, was auf der Rückgabe-/Aktionstaste der virtuellen Tastatur angezeigt wird. Die verfügbaren Rückgabearten sind:
Virtuelle Tastatur Rückgabetyp Konstante
Rückgabe Typ Beschreibung
kbv_returnkey_default Der Standardtitel der Return-Taste für das aktuelle System.
kbv_returnkey_go Setzt den Titel der Return-Taste auf "Go".
kbv_returnkey_google Setzt den Titel der Rückgabetaste auf "Google" oder in manchen Fällen auf ein allgemeines Suchsymbol.
kbv_returnkey_join Setzt den Titel der Return-Taste auf "Go".
kbv_returnkey_next Setzt den Titel der Return-Taste auf "Weiter".
kbv_returnkey_route Setzt den Titel der Return-Taste auf "Route".
kbv_returnkey_search Setzt den Titel der Eingabetaste auf "Suchen" oder in manchen Fällen auf ein allgemeines Suchsymbol.
kbv_returnkey_send Setzt den Titel der Return-Taste auf "Senden".
kbv_returnkey_yahoo Setzt den Titel der Rückgabetaste auf "Yahoo" oder in manchen Fällen auf ein allgemeines Suchsymbol.
kbv_returnkey_done Setzt den Titel der Rückgabetaste auf "Erledigt".
kbv_returnkey_continue Setzt den Titel der Return-Taste auf "Weiter".
kbv_returnkey_emergency Setzt den Titel der Return-Taste auf "Notruf".

  • Typ der automatischen Großschreibung: Mit dieser Konstante wird festgelegt, wie die über die virtuelle Tastatur eingegebenen Wörter automatisch großgeschrieben werden sollen. Die verfügbaren Typen der automatischen Großschreibung sind:
Virtuelle Tastatur Autokapitalisierung Typ Konstante
Autokapitalisierung Typ Beschreibung
kbv_autocapitalize_none Die automatische Großschreibung ist deaktiviert.
kbv_autocapitalize_words Wörter werden automatisch großgeschrieben.
kbv_autocapitalize_sentences Die Sätze werden automatisch großgeschrieben.
kbv_autocapitalize_characters Alle Zeichen werden großgeschrieben.

Das letzte argument ist die Aktivierung/Deaktivierung des Textvorhersagemodus, der auf true gesetzt wird, um ihn zuzulassen, und andernfalls auf false. Beachten Sie jedoch, dass nur weil er zugelassen ist, nicht bedeutet, dass er auch verwendet wird, da dies von den Einstellungen des Benutzers auf dem Gerät abhängt. Im Textvorhersagemodus erzeugt die virtuelle Tastatur keine normalen GameMaker-Tastendruckereignisse. Stattdessen werden nur das zuletzt gedrückte Zeichen und die Tastaturstring-Variablen aktualisiert. Dies liegt daran, dass nicht erkannt werden kann, ob eine Änderung im internen Textfeld, das zur Erkennung von Tastendrücken verwendet wird, von einer tatsächlichen Taste der virtuellen Tastatur oder von einem Textvorschlag stammt. In diesen Fällen sollten Sie die Eingabe von keyboard_string lesen und nicht irgendeine rohe Tasteneingabe.

Es ist auch wichtig zu beachten, dass der Benutzer je nach platform OS unterschiedliche Tastaturen mit unterschiedlichen Fähigkeiten erhält, mit den folgenden Vorbehalten für die Verwendung pro Ziel:

  • UWP: Die UWP-Implementierung unterstützt keine return_key_type, autocapitalization_type oder predictive_text_enabled. Die Übergabe dieser Parameter an eine Funktion führt nicht zu Fehlern, sie haben nur keine Auswirkungen auf die Tastatur.

  • Android: Auf Android wird der Tastaturtyp kbv_type_phone_name nicht unterstützt und zeigt stattdessen eine ASCII-Tastatur an, und der Rückgabetastentyp kann nur kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send oder kbv_returnkey_done sein. Wenn andere Rückgabetastentypen auf diesem platform verwendet werden, verwendet das System standardmäßig die unten aufgeführten Ersatzrückgabetasten:
Nicht verfügbare Return-Taste Ersatzschlüssel
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV/FireTV: Benutzerdefinierte Return-Tasten werden auf ASCII-Tastaturen nicht unterstützt - die Standard-Return-Taste wird immer angezeigt.

  • tvOS: Prädiktiver Text und automatische Großschreibung werden auf tvOS nicht unterstützt, und die Höhe der Tastatur, die von Systemereignissen und der Funktion keyboard_virtual_height() zurückgegeben wird, entspricht standardmäßig der Bildschirmhöhe, da sie sich über den gesamten Bildschirm erstreckt und die Funktionen zur Rektaldimensionierung der Tastatur auf tvOS deaktiviert sind. Beachten Sie auch, dass physische (Bluetooth-)Tastaturereignisse erst dann übertragen werden, wenn die virtuelle Tastatur geöffnet wurde.

Der Aufruf dieser Funktion erzeugt ein asynchrones Systemereignis, bei dem die async_load DS-Map mit den folgenden Schlüssel/Wertpaaren gefüllt wird:

  • "event_type" - dies ist der "Status der virtuellen Tastatur", wenn er durch Funktionen der virtuellen Tastatur ausgelöst wird.

  • "screen_height" - die Höhe der virtuellen Tastatur (in Pixel). Dieser Wert ist 0, wenn die Tastatur unsichtbar ist.

  • "keyboard_status" - der aktuelle Status der Tastatur, zurückgegeben als eine der folgenden Zeichenketten:
    • "versteckt"
    • "versteckt"
    • "zeigend"
    • "sichtbar"

Syntax

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

Argument Typ Beschreibung
keyboard_type Virtuelle Tastatur Typ Konstante Determines the keyset available on the virtual keyboard (see tables at the top).
return_key_type Virtuelle Tastatur Rückgabetyp Konstante Determines what is shown on the return/action key of the virtual keyboard (see tables at the top).
autocapitalization_type Virtuelle Tastatur Autokapitalisierung Typ Konstante Determines how/if the words typed via the virtual keyboard will be autocapitalized (see tables at the top).
predictive_text_enabled Boolesche Set to true/false zum Aktivieren/Deaktivieren der Texterkennung.

Rückgabe:

N/A

Beispiel:

if input == false
{
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
}

Die obige code wird die virtuelle Tastatur des Betriebssystems aufrufen, wenn die angegebene Variable nicht auf true gesetzt ist.
