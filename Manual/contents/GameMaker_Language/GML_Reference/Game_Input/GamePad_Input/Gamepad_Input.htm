Gamepad-Eingabe

DerGameMaker verfügt über eine Reihe spezieller Funktionen, mit denen Sie sowohl analoge als auch digitale Steuerelemente von mehreren angeschlossenen Gamepads erkennen können. Diese Funktionen funktionieren ähnlich wie die Geräteeingänge, d. h. Sie können bis zu vier verschiedene angeschlossene XInput-Gamepads (und bis zu 8 DirectInput-Gamepads) erkennen und die Eingaben von jedem einzelnen mit denselben Funktionen verarbeiten. Beachten Sie, dass ein asynchrones Systemereignis ausgelöst wird, wenn ein Gamepad an Ihr Gerät angeschlossen wird (oder wenn es entfernt wird), so dass Sie die Situation mit den entsprechenden Funktionen bearbeiten können.

Die Gamepad-"Steckplätze" sind von 0 aufwärts indiziert, und der tatsächliche Steckplatz, der einem Gamepad zugewiesen wird, hängt von einer Vielzahl von Faktoren ab, nicht zuletzt vom Betriebssystem, unter dem das Projekt läuft. Auf dem Windows-Ziel sind die Slots 0 bis einschließlich 3 nur für Xinput-Gamepads vorgesehen, d. h. Xbox360-Controller und kompatible Geräte. Sie können jedoch auch die Steckplätze 4 bis einschließlich 11 für DirectInput-Gamepads überprüfen, was bedeutet, dass Sie viele andere Controller-Modelle erkennen können, wenn sie über diese Steckplätze angeschlossen sind. Auf anderen platforms können Pads auf jedem Steckplatz erkannt werden, den das Betriebssystem ihnen zugewiesen hat, z. B. Steckplatz 3 oder Steckplatz 20 oder mehr. Android-Geräte speichern beispielsweise Bluetooth-Gamepads in einem Steckplatz und reservieren diesen Steckplatz für dieses Gamepad in der Zukunft, unabhängig davon, ob es angeschlossen ist oder nicht. Sie können also nicht davon ausgehen, dass ein einzelnes angeschlossenes Gamepad mit Steckplatz 0 verbunden ist, da dies wahrscheinlich nicht der Fall ist.

Es ist zu beachten, dass bei der Verwendung von DirectInput-Gamepads auf Windows oder generischen Gamepads auf anderen platforms die unten angegebenen Konstanten aufgrund der fragmentierten und nicht standardisierten Art und Weise, in der die API von den Controller-Herstellern implementiert wird, möglicherweise nicht genau den Tasten entsprechen, die Sie erwarten, wenn sie gedrückt werden. Aus diesem Grund ist es empfehlenswert, dass Sie eine Art Gamepad-Setup-Bildschirm in Ihren Spielen haben, in dem die Spieler die Gamepad-Tasten basierend auf den Eingaben eines angeschlossenen Geräts neu definieren können, um etwaige Probleme zu entschärfen (es gibt Gamepad-"Mapping"-Funktionen, die bei Windows Desktop, Ubuntu, macOS und Android helfen können, während Sie dies bei allen anderen Zielen mit code selbst tun müssen).

Die folgenden Gamepad-Funktionen sind ebenfalls vorhanden und werden verwendet, um die eingebauten Konstanten den direkten physischen Eingaben eines bestimmten Gamepads zuzuordnen. Diese Funktionen sind nur für den Windows Desktop, Ubuntu, macOS, und Android Ziel platforms und auf Windows, werden sie nur für direkte Eingabegeräte gültig sein. Der GameMaker enthält zwar Zuordnungen für eine Reihe verschiedener Gamepads, die auf der SDL Gamepad Controller DB basieren, aber aufgrund der riesigen Anzahl von Controllertypen und -marken ist es unmöglich, die GML Konstanten den richtigen Eingängen für jedes Fabrikat und Modell zuzuordnen, daher haben Sie mit diesen Funktionen die Möglichkeit, Ihre eigenen Zuordnungen zu erstellen.

  • gamepad_get_mapping
  • gamepad_test_mapping
  • gamepad_remove_mapping

Es ist zu beachten, dass Direct Input Gamepads im kooperativen Modus betrieben werden, was bedeutet, dass Ihr Spiel nur dann Zugriff auf sie hat, wenn es die Vordergrundanwendung ist, was wiederum dazu führt, dass Direct Input Controller "verloren" gehen, wenn das Spiel den Fokus verliert, und dann wieder "gefunden" werden, wenn es wieder den Fokus erhält (dies kann im Systemereignis erkannt und behandelt werden). Ebenso werden keine Eingaben von Gamepads erkannt, wenn das Spiel nicht im Fokus ist, und wir empfehlen, die Funktion os_is_paused() oder window_has_focus() zu verwenden, um dies zu erkennen und das Spiel zu pausieren oder etwas Ähnliches, da jede Taste, die zu dem Zeitpunkt gedrückt wird, wenn das Spiel den Fokus verliert, diesen Zustand beibehält, bis das Spiel den Fokus wiedererlangt.

Kompatibilität

Die folgende Liste zeigt die derzeitige Kompatibilität von platforms (bitte beachten Sie, dass sich diese Liste mit zukünftigen Updates ändern wird):

  • Windows wird in vollem Umfang unterstützt, wobei bis zu 12 Geräte gleichzeitig angeschlossen werden können (nummeriert von 0 bis 11, wobei 0 - 3 XInput-Geräte und 4 - 11 DirectInput-Geräte sind). Die Neuzuordnung von Controllerkonstanten ist ebenfalls möglich.
  • macOS wird mit bis zu 4 angeschlossenen Geräten gleichzeitig unterstützt, und diese Geräte können NUR vom Typ Playstation3 oder Xbox 360 sein. Bitte beachten Sie, dass die Option "Für Mac AppStore erstellen" in den Mac-Spieloptionen ausgeschaltet sein muss, damit die Pad-Unterstützung funktioniert. Die Neuzuordnung von Controller-Konstanten ist ebenfalls erlaubt.
  • Ubuntu unterstützt auch Gamepad-Eingabe, aber Sie müssen möglicherweise zusätzliche Bibliotheken aus dem Ubuntu Repository installieren. Sie können dies ganz einfach tun, indem Sie ein Kommandozeilenterminal öffnen und Folgendes eingeben (dadurch wird sowohl die GUI-Unterstützung für den Joystick/das Gamepad als auch die Joystick/Gamepad-API selbst installiert - die Neuzuordnung von Steuerungskonstanten ist nicht zulässig):

    sudo apt-get install jstest-gtk
    sudo apt-get install joystick.

  • HTML5-Spiele mit Gamepads werden von den meisten gängigen Browsern unterstützt, außer von Safari. Das Remapping von Controller-Konstanten ist nicht erlaubt.
  • Webbrowser erkennen Gamepads nur, wenn eine Taste gedrückt oder eine Achse bewegt wird, so dass sie möglicherweise nicht sofort zu Beginn des Spiels verfügbar sind. Dies gilt für den Opera GX und das HTML-Ziel platforms.
  • Die Gamepad-Unterstützung erstreckt sich auch auf iOS mit dem iCade-Gehäuse. Die linke Achse entspricht dem Stick-Controller (obwohl die Eingabe digital und nicht analog ist), die vier "Gesichtstasten" entsprechen den Tasten auf der Vorderseite des Gehäuses und die vier Schultertasten entsprechen denen auf der Rückseite des Gehäuses.
  • Android (und Amazon Fire) Export unterstützt NYKO-Controller und generische Bluetooth-Controller, aber nur, wenn sie aktiviert sind, was bedeutet, dass Sie die iCade/Bluetooth-Option im Abschnitt "Allgemein" der Android Spieloptionen ankreuzen müssen. Sie benötigen API-Level 12, um vollständig zu funktionieren, und es ist zu beachten, dass GameMaker alle Bluetooth-Geräte, mit denen Ihr Gerät gekoppelt ist, als verbunden registriert, unabhängig davon, ob es tatsächlich verbunden ist oder nicht. Daher sollte dies bei der Zuweisung und Überprüfung von "Slots" berücksichtigt werden. Beachten Sie, dass auch die Neuzuordnung von Controller-Konstanten erlaubt ist.
  • Wenn Sie auf der PS4 das Touchpad-Tracking verwenden möchten, müssen Sie die device_mouse_* Tasten verwenden. Die Neuzuordnung von Controller-Konstanten ist nicht erlaubt.
  • Auf Windows UWP-, XBoxOne- und Nintendo Switch-Zielen werden Gamepads vollständig unterstützt, aber die Neuzuordnung von Controller-Konstanten ist nicht zulässig.

Idealerweise sollten Sie auf allen Zielrechnern platforms eine Liste der verfügbaren Gamepad-"Steckplätze" auflisten und dann überprüfen, ob irgendwelche Geräte erkannt werden, etwa so:

var _maxpads = gamepad_get_device_count();

for (var i = 0; i < _maxpads; i++)
{
    if (gamepad_is_connected(i))
    {
        // do stuff with pad "i"
    }
}
