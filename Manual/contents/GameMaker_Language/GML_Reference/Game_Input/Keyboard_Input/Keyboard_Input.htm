Tastatur-Eingabe

Für den Umgang mit der Tastatur in GameMaker gibt es eine Vielzahl von Funktionen, die verschiedene Tastaturzustände wie gedrückt oder losgelassen erkennen können. Es gibt auch einige, die alle Tastendrücke als Zeichenkette speichern oder die Ihnen sagen können, was die zuletzt gedrückte Taste war, sowie andere, mit denen Sie den Tastaturstatus komplett löschen können.

HINWEIS Diese Funktionen sind nur für den Windows/Mac/Ubuntu-Desktop platforms gedacht. Es kann sein, dass einige der eingebauten Variablen und Konstanten auf anderen platforms nicht gültig sind und viele der Funktionen auf Mobiltelefonen nicht funktionieren.

Jedes Eingabezeichen einer Taste (oder mehrerer Tasten) wird durch seinen UTF8 code definiert, der ein numerischer Wert ist. Dieser Wert kann für jedes Zeichen mit der Funktion ord() Funktion abgerufen werden, aber GameMaker verfügt auch über eine Reihe von constants für die am häufigsten verwendeten Sondertasten der Tastatur und eine spezielle Funktion. Normalerweise verwendet man eine Kombination aus ord() und den keyboard_check*() Funktionen, etwa so:

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

Die obige Funktion prüft also die Taste "A", und wenn sie gedrückt wird, wird die horizontale Geschwindigkeit des Objekts auf -5 gesetzt. Beachten Sie, dass die Verwendung von ord() auf diese Weise nur dann korrekt funktioniert, wenn die Eingabezeichenfolge nur ein Zeichen lang ist und eine Zahl von 0 bis 9 oder ein großgeschriebenes römisches Zeichen von A bis Z ist. Die Funktion ord() gibt einen vollständigen UTF8-Wert zurück, aber die Funktionen von keyboard_check*() erkennen nur A - Z und 0 - 9. Was aber, wenn Sie die Pfeiltasten verwenden möchten? Oder wenn man eine Aktion mit der "Shift"-Taste ändern möchte? Nun, dafür hat GameMaker eine Reihe von vk_ Konstanten (vk_ steht für virtuelle Taste), die Sie anstelle von ord() verwenden können:

vk_anykey Tastencode, der bedeutet, dass eine beliebige Taste gedrückt wird
vk_left Tastencode für die linke Pfeiltaste
vk_right Tastencode für die rechte Pfeiltaste

vk_down Tastencode für die Abwärtspfeiltaste
vk_enter Eingabetaste

vk_control eine der Steuertasten

Die folgenden Konstanten können nur mit keyboard_check_direct() verwendet werden:

Virtuelle Schlüsselkonstante (vk_*)
Konstante Beschreibung
vk_lshift linke Umschalttaste
vk_lcontrol linke Steuerungstaste
vk_lalt linke Alt-Taste
vk_rshift rechte Umschalttaste
vk_rcontrol rechte Steuertaste
vk_ralt rechte Alt-Taste

Es folgt ein kleines Beispiel für die Verwendung der vk_ Konstanten:

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

Die obige code erkennt, ob die Tabulatortaste gedrückt wird und erstellt eine Instanz des Objekts "obj_Menu", wenn dies der Fall ist.

Wenn Sie nach einem Schlüsselzeichen suchen, das nicht 0 - 9, A - Z oder eine der VK-Konstanten ist, dann sollten Sie eine der keyboard_* -Variablen prüfen, wie zum Beispiel keyboard_lastchar:

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
}

Die folgenden Funktionen stehen für die Arbeit mit einer Tastatur zur Verfügung:

HINWEIS: Diese Funktionen funktionieren nicht, wenn Sie eine virtuelle Tastatur auf dem Bildschirm verwenden.

  • io_clear
  • keyboard_check
  • keyboard_check_pressed
  • keyboard_check_released
  • keyboard_check_direct
  • keyboard_clear
  • keyboard_set_map
  • keyboard_get_map
  • keyboard_unset_map
  • keyboard_set_numlock
  • keyboard_get_numlock

Es gibt auch einige Funktionen, mit denen man Tastendrücke simulieren kann:

  • keyboard_key_press
  • keyboard_key_release

Schließlich sind die folgenden Variablen in GameMaker eingebaut und können aufgerufen werden, um Informationen über den Tastaturstatus und die Eingabe zu erhalten:

HINWEIS: Wenn Sie die virtuelle Tastatur verwenden, wird nur die Variable keyboard_string Variable mit der Tastatureingabe aktualisiert werden.

  • keyboard_key
  • keyboard_lastkey
  • keyboard_lastchar
  • keyboard_string
