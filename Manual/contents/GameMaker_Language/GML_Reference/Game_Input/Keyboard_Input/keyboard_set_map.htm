keyboard_set_map

Manchmal möchte man bei der Entwicklung eines Spiels, dass eine Taste die gleiche Funktion hat wie eine andere. Zum Beispiel benutzen viele Leute die WASD-Tasten für die Bewegung, aber viele Leute benutzen auch die Pfeiltasten! Was kann man also tun? Nun, man könnte code das Bewegungssystem zweimal verwenden, aber das ist etwas kompliziert und zum Glück überflüssig, denn mit dieser Funktion können Sie eine Taste einer anderen "zuordnen", so dass jede Eingabe mit einer der beiden Tasten als dieselbe interpretiert wird. Dazu wählt man eine Taste, die man zuordnen möchte (Taste2 - dies ist die Taste, für die Sie code schreiben) und eine Taste, die Sie zuordnen wollen (Taste1). Danach werden Tastendrücke auf eine der beiden Tasten von GameMaker als von Taste2 kommend interpretiert. Du kannst diese Funktion auch verwenden, um ein System zu entwerfen Sie können diese Funktion auch verwenden, um ein System zu entwerfen, bei dem der Benutzer seine eigenen Tasten zum Spielen definieren kann, indem er einfach die Benutzereingabetaste der Taste zuordnet, die Sie im Spiel kodiert haben.

Die Funktion nimmt einen Keycode-Wert, wie er von der Funktion ord() zurückgegeben wird (nur Großbuchstaben von A-Z oder Zahlen von 0-9), oder eine der vk_* Konstanten, die auf der Hauptseite Tastatureingabe aufgeführt sind.

WICHTIG! Die Taste, die Sie zuordnen (Taste1), kann nicht mehr als ihr eigentliches Tastensymbol verwendet werden, sobald Sie diese Funktion verwenden. Wenn Sie z.B. den Pfeil nach oben auf "W" legen, können Sie das Drücken der Taste "W" nicht mehr als "W" erkennen. der Taste "W" nicht mehr als "W" erkennen, sondern wird immer als Pfeil nach oben betrachtet. Um dies rückgängig zu machen, ordnen Sie entweder die Taste sich selbst zu - also wären sowohl Taste1 als auch Taste2 ord("W") - oder verwenden Sie die Funktion keyboard_unset_map().

Syntax:

keyboard_set_map(key1, key2);

ArgumentTyp Beschreibung
key1 This is the key that key1 is to be mapped to
key2 This is the key that is to be mapped

Rückgabe:

Beispiel:

keyboard_set_map(ord("A"), vk_left);

Im obigen Beispiel code wird die Taste "A" der linken Pfeiltaste zugeordnet. Das bedeutet, dass der Spieler entweder die Taste "A" oder die linke Pfeiltaste benutzen kann und dass alle code, die für den linken Pfeil geschrieben wurden, auch auf die Taste "A" Taste reagieren.
