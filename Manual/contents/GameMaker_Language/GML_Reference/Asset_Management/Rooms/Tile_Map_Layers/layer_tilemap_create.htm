layer_tilemap_create

Mit dieser Funktion können Sie einem Layer eine Kachelsatz-Ressource zuweisen, die in Ihrem Projekt als Kachelkarte verwendet werden soll. Sie geben die Layer-ID (die Sie erhalten, wenn Sie den Layer mit layer_create()) oder den Ebenennamen (als String - dies wirkt sich auf die Leistung aus) und dann eine Anfangsposition (x, y), um die Kachelkarte dem Raum hinzuzufügen, die zu verwendende Kachelsatz-Ressource und dann die Breite und Höhe der Kachelkarte in Zellen (z. B.: eine Breite von 20 und eine Höhe von 10 erzeugt eine Kachelkarte mit 200 Zellen, die 20 Zellen breit und 10 Zellen hoch ist), wobei die Größe der Zellen selbst durch den gewählten Kachelsatz bestimmt wird. Es ist zu beachten, dass Sie keine Kacheln an negativen Positionen innerhalb der Kachelkarte platzieren können, d.h. alle Kacheln müssen innerhalb der Zellbereiche 0 bis Breite - 1, 0 bis Höhe - 1 platziert werden.

Syntax:

layer_tilemap_create(layer_id, x, y, tileset, width, height)

Argument Typ Beschreibung
layer_id String oder Ebenen-ID The unique ID value of the layer to target (or the layer name as a string)
x Real The x position of the tile map in the room
y Real The y position of the tile map in the room
tileset Kachelsatz Asset The Tile Set asset to be used
width Real The width tile map (in cells)
height Real The height of the tile map (in cells)

Rückgabe:

Tile Map Element ID

Beispiel:

global.back_layer = layer_create(10000);
global.back_tiles = layer_tilemap_create(global.back_layer, 0, 0, tmap_Trees, 16, 32);

Die obige code erstellt eine neue Ebene und fügt dann ein neues Kachelkartenelement hinzu, wobei die Position der Kachelkarte auf (0,0) sowie der zu verwendende Kachelsatz und die Breite und Höhe der Kachelkarte festgelegt werden.
