Kachelkartenebenen

Mit dem GameMaker-Raumeditor können Sie in jedem beliebigen Raum Kacheln mithilfe von Ebenen hinzufügen. Wie der Name schon sagt, ist eine Kachelkartenebene einfach eine Sammlung von Kacheln, die sich alle in der gleichen Tiefe innerhalb des Raums befinden, und durch das Stapeln von Ebenen können Sie einige Dinge über oder unter anderen zeichnen. Sie können auch bestimmte Aspekte von Ebenen über code steuern, Dinge hinzufügen oder entfernen oder bestimmte Eigenschaften der Ebene oder deren Inhalt zur Laufzeit ändern. Beachten Sie, dass Sie im Raumeditor auf eine Kachelkarte pro Ebene beschränkt sind (die Kachel-Ebene), aber wenn Sie mit ihnen in code arbeiten, können Sie einer einzelnen Ebene mehrere Kachelkarten zuweisen, wobei jede Karte ihre eigene eindeutige ID und ihre eigenen Eigenschaften hat, also:

  • Das Kachelset ist die Ressource, die Sie zum Erstellen von Kachelschichten und Kachelkarten verwenden
  • Die Kachel-Ebene ist das, was Sie im Raum-Editor erstellen, um Kachel-Sets hinzuzufügen
  • Als Kachelkarte bezeichnet man die Sammlung von Kacheln, die einer Ebene hinzugefügt werden, entweder im Raumeditor oder durch code, als einzelnes Element auf dieser Ebene

Neben den oben aufgeführten Funktionen zum Festlegen von Kachelzuordnungen gibt es auch eine Reihe von Funktionen, mit denen die Daten einzelner Kacheln bearbeitet werden können. Ein "Blob" von Kacheldaten besteht aus dem Kachelindex und ein paar zusätzlichen Bits, die anzeigen, ob die Kachel gedreht, gespiegelt oder gewendet wurde. Sie haben auch die Möglichkeit, den Kacheldaten Ihre eigene Bitmaske hinzuzufügen. Dies ist allerdings nur für fortgeschrittene Benutzer geeignet, da Sie damit Bits ausblenden, die für den Kachelindex reserviert sind - im Wesentlichen sind 19 Bits für Ihre Kachelindizes reserviert, aber wenn Sie nur 8 davon verwenden, bleiben 11 Bits übrig, die Sie zur Erstellung einer benutzerdefinierten Maske verwenden können. Diese Maske kann dann in code verwendet und überprüft werden, um benutzerdefinierte Kachelkollisionen oder ähnliches zu erstellen. Normalerweise braucht man jedoch keine eigenen Kacheldaten-Blobs zu erstellen, sondern verwendet stattdessen die Funktion tilemap_get() verwenden, um die Kacheldaten zu erhalten, sie dann mit den unten stehenden Funktionen zu bearbeiten und sie dann mit tilemap_set().
