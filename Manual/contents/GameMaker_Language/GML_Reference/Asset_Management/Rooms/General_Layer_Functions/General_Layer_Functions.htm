Allgemeine Funktionen der Ebene

Der GameMaker-Raumeditor ermöglicht es Ihnen, Ihrem Raum Ebenen mit verschiedenen Assets hinzuzufügen. Dies können Kachelkarten (unter Verwendung von Kachel-Sets), Hintergründe, sprites oder sequences und Instanzen sein. Beachten Sie jedoch, dass Sie im Gegensatz zum Raum-Editor nicht auf einen einzigen Asset-Typ pro Ebene beschränkt sind, sondern mehrere Asset-Typen zu derselben Ebene hinzufügen können - Sie können beispielsweise einen Hintergrund und eine Kachelkarte auf derselben Ebene haben. Beachten Sie, dass jedes Asset, das einer Ebene hinzugefügt wird, als ein Element dieser Ebene betrachtet wird.

Dieser Abschnitt beschreibt die allgemeinen Funktionen, die bei der Arbeit mit Raumebenen verwendet werden, sowie einige spezielle Funktionen für die Arbeit mit Instanz-Ebene Elementen:

  • layer_get_all_elements
  • layer_get_element_layer
  • layer_get_element_type
  • layer_get_x
  • layer_get_y
  • layer_get_hspeed
  • layer_get_vspeed
  • layer_create
  • layer_destroy
  • layer_x
  • layer_y

Wir haben auch ein paar zusätzliche Funktionen, die sich auf das Anvisieren von Ebenen in einem anderen Raum beziehen. Es kann sein, dass Sie etwas in einem Raum ändern wollen, der nicht der aktuelle Raum ist, oder vielleicht wollen Sie Räume prozedural generieren, oder eine Reihe von Dingen. Um dies zu ermöglichen, verwenden wir die folgenden Funktionen:

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

Hier setzen Sie den Zielraum mit der entsprechenden Funktion und rufen dann die Ebenenfunktionen wie gewohnt auf. Diese Ebenenfunktionen gelten nun nur für den Zielraum und nicht für den aktuellen Raum. Wenn Sie fertig sind, rufen Sie die Reset-Funktion auf, um den Bereich code wieder auf den aktuellen Raum zurückzusetzen.

WARNUNG! Während Sie einen anderen Raum anvisieren, können Sie alle normalen Ebenenfunktionen verwenden , außer dass Sie keine Instanzen mit instance_create_layer() oder instance_create_depth()erstellen, noch ist die Ebenenfunktion layer_add_instance() verfügbar sein.

Schließlich, und dies ist ausschließlich für die Verwendung mit Instanzen auf Ebenen (obwohl die Ebene selbst andere Elemente haben kann, werden diese von den folgenden Funktionen nicht beeinflusst), haben Sie eine Reihe von Funktionen, die verwendet werden können, um alle Instanzen zu deaktivieren oder zu reaktivieren:

  • instance_deactivate_layer
  • instance_activate_layer
