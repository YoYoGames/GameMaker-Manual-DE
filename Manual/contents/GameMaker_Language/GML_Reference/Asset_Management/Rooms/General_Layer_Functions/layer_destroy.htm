layer_destroy

Diese Funktion zerstört die angegebene Ebene. Sie geben die ID der Ebene an (die Sie erhalten, wenn Sie die Ebene mit layer_create()) oder den Ebenennamen (als Zeichenkette - dies hat Auswirkungen auf die Leistung) und die Ebene wird aus dem aktuellen Raum entfernt. Wenn es sich um eine Ebene handelt, die im Raumeditor erstellt wurde, wird die Ebene beim nächsten Verlassen des Raums und der Rückkehr mit dem ursprünglichen Inhalt neu erstellt; ist der Raum jedoch persistent, wird die Ebene entfernt, es sei denn, die Raumpersistenz wird wieder ausgeschaltet. Wenn du eine Ebene auf diese Weise zerstörst, wird auch ihr gesamter Inhalt entfernt, d.h. alle Referenz-IDs für Hintergründe oder Kachelkarten usw. sind nicht mehr gültig und alle Instanzen, die der Ebene zugeordnet sind, werden zerstört (wobei gleichzeitig ihr Destroy-Event ausgeführt wird, falls sie eines haben, sowie das Clean Up-Event).

Syntax:

layer_destroy(layer_id)

Argument Typ Beschreibung
layer_id String oder Ebenen-ID The unique ID value of the layer to destroy (or the layer name as a string)

Rückgabe:

N/A

Beispiel:

if !instance_exists(obj_Bullet_Parent)
{
    layer_destroy(global.Bullet_Layer);
}

Die obige code prüft, ob Instanzen des Objekts "obj_Bullet_Parent" existieren, und wenn nicht, wird die Ebene mit der in der globalen Variablen gespeicherten ID zerstört.
