asset_get_type

Mit dieser Funktion können Sie den Typ des Assets, auf das verwiesen wird, anhand seines Namens (einer Zeichenkette) ermitteln. Es wird eine der unten aufgeführten Konstanten zurückgegeben.

Syntax:

asset_get_type(name);

Argument Typ Beschreibung
name Zeichenfolge The name of the game asset to get the type of (a string).

Rückgabe:

Asset Type Constant

Konstante Beschreibung
asset_object Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_sound Der Vorname bezieht sich auf einen Klang.
asset_room Der Vorname bezieht sich auf einen room.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_path Der angegebene Name verweist auf einen Pfad.
asset_script Der Vorname bezieht sich auf einen script.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_timeline Der Vorname bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequence Der angegebene Name bezieht sich auf eine Sequenz.
asset_unknown Der angegebene Name bezieht sich auf einen Vermögenswert, der entweder nicht existiert oder nicht zu den oben genannten gehört.

Beispiel:

if asset_get_type("pth_Path_" + string(global.Game)) == asset_unknown
{
    show_debug_message("Path doesn't exist!!!");
}
else
{
    path_index = asset_get_index("pth_Path_" + string(global.Game));
}

Die obige code überprüft einen dynamisch erstellten Asset-Namen, um festzustellen, ob das Asset vom richtigen Typ ist. Ist dies nicht der Fall, wird eine Fehlermeldung angezeigt, andernfalls wird der Asset-Name verwendet, um das Asset der Instanz zuzuordnen.
