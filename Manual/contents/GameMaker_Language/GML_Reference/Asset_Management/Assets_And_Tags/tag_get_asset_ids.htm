tag_get_asset_ids

Mit dieser Funktion können Sie alle Assets eines bestimmten Typs ermitteln, denen die angegebenen Tags zugewiesen sind. Sie geben entweder ein einzelnes Tag (als String) oder ein Array an, wobei jedes Element im Array ein Tag (als String) ist, sowie den Typ des zu prüfenden Assets. Der Typ des Assets sollte eine der folgenden Konstanten sein:

Konstante Beschreibung
asset_object Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_sound Der Vorname bezieht sich auf einen Klang.
asset_room Der Vorname bezieht sich auf einen room.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_path Der angegebene Name verweist auf einen Pfad.
asset_script Der Vorname bezieht sich auf einen script.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_timeline Der Vorname bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequence Der angegebene Name bezieht sich auf eine Sequenz.

Die Funktion gibt ein Array zurück, wobei jedes Element in dem Array ein einzelner Asset-Indexwert ist. Wenn es keine Assets des Typs gibt, die den/die angegebenen Tag(s) haben, wird ein leeres Array zurückgegeben.

Syntax:

tag_get_asset_ids(tags, asset_type);

Argument Typ Beschreibung
tags String oder Array von Strings A single asset tag string or an array with various asset tags.
asset_type Anlagenart Konstante An asset type constant (listed above)

Rückgabe:

Array

Beispiel:

var _paths = tag_get_asset_ids("enemy", asset_path);
var _num = irandom(array_length(_paths) - 1);
path_start(_paths[_num], 1, path_action_reverse, false);

Die obige code verwendet das Tag "enemy", um alle Pfad-Assets mit diesem Tag zu finden, bevor sie eines zufällig auswählt und es der Instanz zuweist, die die code ausführt.
