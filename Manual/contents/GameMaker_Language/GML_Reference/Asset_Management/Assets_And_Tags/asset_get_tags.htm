asset_get_tags

Mit dieser Funktion können Sie alle einem Asset zugewiesenen Tags aus dem Asset-Browser abrufen. Sie geben entweder den Asset-Namen (als String) oder den Asset-Index an, und die Funktion gibt ein Array von Tags für dieses Asset zurück. Wenn keine Tags gefunden werden oder ein Fehler auftritt (d.h. der angegebene Namensstring existiert nicht), ist das zurückgegebene Array leer. Wenn Sie einen Asset-Indexwert angeben, müssen Sie auch den optionalen Asset-Typ argument (eine Konstante) angeben, da Assets verschiedener Typen denselben Index haben können, auch wenn sie nicht denselben Namen haben dürfen. Die verfügbaren Asset-Typen sind in der nachstehenden Tabelle aufgeführt:

Konstante Beschreibung
asset_object Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_sound Der Vorname bezieht sich auf einen Klang.
asset_room Der Vorname bezieht sich auf einen room.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_path Der angegebene Name verweist auf einen Pfad.
asset_script Der Vorname bezieht sich auf einen script.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_timeline Der Vorname bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequence Der angegebene Name bezieht sich auf eine Sequenz.

Syntax:

asset_get_tags(name_or_index, [asset_type]);

Argument Typ Beschreibung
name_or_index String oder Asset The name of the asset (a string) or its index.
[asset_type] Anlagenart Konstante OPTIONAL! The type of asset to get the tags for, only used when supplying an index for the first argument.

Rückgabe:

Array

Beispiel:

my_tags = asset_get_tags(object_get_name(object_index));

Die obige code ruft alle Tags ab, die dem Objekt zugewiesen sind, von dem aus die Instanz, auf der die code läuft, erstellt wurde.
