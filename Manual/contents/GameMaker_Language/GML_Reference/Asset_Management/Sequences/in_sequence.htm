in_reihenfolge

Dies ist eine eingebaute Variable, die Teil der Instanzvariablen ist, die für jede Objektinstanz in deinem Spiel erstellt wird. Wenn die Instanz von einem sequence gesteuert wird, gesteuert wird, gibt diese Variable den Wert true zurück, andernfalls gibt sie den Wert false zurück. Diese Variable ist schreibgeschützt und kann nicht geändert werden.

Beachten Sie, dass diese Variable zu false wird, nachdem das steuernde Sequence beendet wurde, auch wenn das Sequence Element noch existiert, und zu true wird, wenn das Sequence Element erneut abgespielt wird.

Sie können diese Variable in Ihren spieler- (oder CPU-) kontrollierten Objekten verwenden, um sicherzustellen, dass sie nur von code bewegt werden, wenn sie sich nicht in einer Sequence befinden. Dies kann sich als nützlich für Zwischensequenzen erweisen, wenn Sie dieselben Instanzen in einem Sequence (mit die Funktion sequence_instance_override_object() ), und Sie müssen sicherstellen, dass sie nicht durch ihre üblichen code bewegt werden können, während die Cutscene aktiv ist.

Syntax:

in_sequence

Rückgabe:

Beispiel:

if (!in_sequence)
{
    x += move_x;
    y += move_y;
}

Die obige code prüft die Variable in_sequence, und wenn sie false ist (was bedeutet, dass die Instanz nicht von einer sequence gesteuert wird), dann fügt sie move_x und move_y zur Position der Instanz hinzu. Position der Instanz hinzu und stellt sicher, dass sie sich nur bewegt, wenn sie sich nicht in einer Sequence befindet.
