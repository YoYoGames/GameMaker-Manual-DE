Die Spur Struktur

Die Track-Struktur kann entweder fürAsset-Tracks auf oberster Ebene oder fürParameter-Tracks auf untergeordneter Ebene verwendet werden, und das Verhalten des Tracks wird durch zwei Dinge definiert: seinen Namen und seinen Typ. Track structs werden mit der Funktion sequence_track_new() erstellt und können von sequence Assets über die Eigenschaft activeTracks der Sequence Instance Struct oder die Eigenschaft tracks der Sequence Object Struct abgerufen werden.

Die in der Trackstruktur verfügbaren Eigenschaften sind:

Sequenz Spur Struktur
Variabel Typ Beschreibung
type Sequenz Spurart Konstante Dies enthält eine Sequenzspurtyp Konstante die den Typ des Tracks beschreibt, um den es sich handelt. Dies kann eine der Konstanten sein, die im Abschnitt "Typ" unten angegeben sind.
name Zeichenfolge Wenn Sie eine "Top-Level"-Asset-Spur erstellen, kann der Name, den Sie hier vergeben, eine beliebige Zeichenfolge sein, die Sie zur Identifizierung der Spur benötigen. Für Parameterspuren müssen Sie jedoch bestimmte Zeichenfolgen angeben, um GameMaker mitzuteilen, welche Art von Parameterspur Sie erstellen. Dies wird weiter unten im Abschnitt "Name" erläutert.
tracks Array von Sequenz Spur Strukturs Diese Eigenschaft ermöglicht den Zugriff auf die Liste der Spuren, die Kinder dieser Spur sind. Wenn diese Eigenschaft erhalten wird, wird ein Array von Sequenz Spur Strukturs zurückgegeben, und beim Setzen dieser Eigenschaft sollte ein Array von Sequence Track Structs angegeben werden sollte.
visible Boolesche Dies zeigt an, ob diese Spur sichtbar ist (der Wert ist true) oder nicht (der Wert ist false). Sie können diesen Wert abrufen oder setzen und wenn eine Spur nicht sichtbar ist, wird auch keine ihrer untergeordneten Spuren gezeichnet.
keyframes Array von Sequenz Keyframe Strukturs Diese Eigenschaft ermöglicht den Zugriff auf die Liste der Keyframe-Strukturen für den Track. Beim Abrufen dieser Eigenschaft wird ein Array von Keyframe-Strukturen zurückgegeben, und beim Setzen dieser Eigenschaft sollte ein Array von Keyframe-Strukturen angegeben werden.
enabled Boolean This indicates whether this track is enabled. If the track is an asset track it isn't drawn when disabled, if it's a parameter track the track's values aren't applied.

Name

Die Eigenschaft name kann eine der folgenden Zeichenketten sein:

  • "Position" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, bedeutet dies, dass der Inhalt dieser Spur als Positionsdaten verwendet werden soll.
  • "Skalierung" - wenn dies auf einer Spur vom Typ seqtracktype_real gesetzt wird, bedeutet dies, dass der Inhalt dieser Spur als Skalierungsdaten verwendet werden soll.
  • "Rotation" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, bedeutet dies, dass der Inhalt dieser Spur als Rotationsdaten verwendet werden soll.
  • "blend_multiply" oder "image_blend" - wenn dies auf eine Spur vom Typ seqtracktype_colour gesetzt wird, bedeutet dies, dass der Inhalt dieser Spur als Farbmultiplikationsdaten verwendet werden soll. Die Keyframe-Daten für eine solche Spur sollten ein Array aus vier Werten im Format [A, R, G, B] sein. Beachten Sie, dass die Werte für jede Komponente zwischen 0 und 1 ausgedrückt werden, wobei 0 dem HEX-Wert #00 und 1 dem HEX-Wert #FF entspricht (0 - 255, wie in der Farbauswahl für Bildüberblendungsspuren im Sequences Editor angezeigt).
  • "image_speed" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, bedeutet dies, dass der Inhalt dieser Spur als Wert für die Bildgeschwindigkeit verwendet werden soll.
  • "image_index" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, bedeutet dies, dass der Inhalt dieser Spur als Bildindexwert verwendet werden soll.
  • "image_angle" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, bedeutet dies, dass der Inhalt dieser Spur als Bildwinkelwert verwendet werden soll.


    Parameternamen für Textspuren (nur bei Textspuren verwendet):
  • "frameSize" - wenn dies auf einer seqtracktype_real Spur gesetzt ist, zeigt dies an, dass der Inhalt dieser Spur als Rahmengröße für den Text verwendet werden soll.Jeder Keyframe für diese Spur sollte zwei Kanäle haben, 0 und 1, wobei 0 die X-Größe und 1 die Y-Größe des Frames speichert.
  • "characterSpacing" - wird auch bei seqtracktype_real Spuren verwendet und dient als Zeichenabstand für den Text. Jeder Keyframe für diese Spur sollte einen einzelnen Kanal haben, der den Abstandswert enthält.
  • "lineSpacing" - wird auch bei seqtracktype_real Spuren verwendet und dient als Zeilenabstand für den Text. Jeder Keyframe für diese Spur sollte einen einzelnen Kanal haben, der den Abstandswert enthält.
  • "paragraphSpacing" - wird auch bei seqtracktype_real Spuren verwendet und dient als Absatzabstand für den Text. Jeder Keyframe für diese Spur sollte einen einzelnen Kanal haben, der den Abstandswert enthält.
