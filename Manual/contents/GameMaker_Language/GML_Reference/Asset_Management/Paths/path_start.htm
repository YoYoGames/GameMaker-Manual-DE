path_start

Ein Pfad wird aus einer Reihe von definierten Punkten erstellt, die miteinander verbunden sind und dann zur Planung der Bewegungen einer Instanz verwendet werden. Sie können mit codeoder im Pfad-Editor erstellt werden und werden einer Instanz zur Verwendung im Spiel zugewiesen. Du würdest dann diese Funktion verwenden, um deiner Instanz mitzuteilen, welchem Pfad sie folgen soll, mit welcher Geschwindigkeit sie dem Pfad folgen soll (gemessen in Pixeln pro Schritt), wie sie sich verhalten soll, wenn sie das Ende des Pfades erreicht, und ob sie der absoluten oder relativen Pfadposition folgen soll. Der letzte Teil bedeutet, dass die Instanz den Pfad entweder genau so startet und verfolgt, wie du ihn im Pfad-Editor entworfen und platziert hast (absolut), oder sie startet und verfolgt den Pfad von der Position aus, an der die Instanz erstellt wurde (relativ).

Wie Sie im obigen Bild sehen können, folgen die beiden Instanzen demselben Pfad, der mit path_start() gestartet wurde, aber eine der Instanzen folgt dem absoluten Pfad, während die andere dem relativen Pfad folgt. Es sollte beachtet werden, dass die Instanz zum Anfang des Pfades "springt", wenn relativ auf false (absolut) eingestellt ist, egal wo im Raum sie sich befindet.

Die Konstanten, die das Verhalten der Instanz bei Erreichen des Endes des Pfades bestimmen, sind auf dieser Seite angegeben.

Syntax:

path_start(path, speed, endaction, absolute);

Argument Typ Beschreibung
path Pfad Anlage The path index to start.
speed Real The speed of which to follow the path in pixels per step, negative meaning going backwards.
endaction Trassenende Aktion Konstante What to do when the end of the path is reached (see the constants listed above).
absolute Boolesche Whether the calling instance should follow the absolute path as it is in the editor (true) or a relative path to its current position (false).

Rückgabe:

N/A

Beispiel:

path_start(path, 4, path_action_reverse, 0);

Dadurch folgt die aufrufende Instanz dem in der Variablen "path" indizierten Pfad mit einer Geschwindigkeit von 4 Pixeln pro Schritt. Sie folgt dem Pfad relativ zu ihrer eigenen Position, und wenn sie das Ende erreicht, kehrt sie die Richtung um und geht den Weg zurück, den sie gekommen ist.
