Shader-Konstanten

Neben den Shader-Funktionen und -Konstanten, die in den OpenGL ES Shading Language (GLSL ES) -Referenzseiten definiert sind, stehen Ihnen auch eine Reihe von Shader-Konstanten zur Verfügung, die es nur in GameMaker gibt.

Die folgenden Display-Matrix-Konstanten können als Array-Indizes verwendet werden, wenn die Shader-Array-Konstante gm_Matrices verwendet wird:

Konstante Beschreibung
MATRIX_VIEW Diese Array-Indexkonstante enthält den Index der aktuellen Ansichtsmatrix. Der zurückgegebene Index wird als Array-Wert verwendet, wenn die Konstante gm_Matrices innerhalb des Shaders code aufgerufen wird.
MATRIX_PROJECTION Diese Array-Indexkonstante enthält den Index für die aktuelle Projektionsmatrix. Der zurückgegebene Index wird beim Aufruf der Konstante gm_Matrices innerhalb des Shaders code als Array-Wert verwendet.
MATRIX_WORLD Diese Array-Indexkonstante enthält den Index der aktuellen Weltmatrix. Dies kann für Dinge wie Beleuchtung verwendet werden, wenn Sie Lichtinformationen im Weltraum haben. Der zurückgegebene Index wird als Array-Wert beim Aufruf der Konstante gm_Matrices innerhalb des Shaders code verwendet.
MATRIX_WORLD_VIEW Diese Array-Indexkonstante enthält den Index für das Ergebnis der Welt- und der Ansichtsmatrix, die miteinander multipliziert werden. Dies wird oft für Dinge wie Nebel verwendet. Der zurückgegebene Index wird als Array-Wert beim Aufruf der Konstante gm_Matrices innerhalb des Shaders code verwendet.
MATRIX_WORLD_VIEW_PROJECTION Diese Array-Indexkonstante enthält den Index für das Ergebnis der Welt-, Ansichts- und Projektionsmatrizen, die miteinander multipliziert werden. Dies ist die normale Transformationsmatrix, die für die Vertex-Positionen verwendet wird. Der zurückgegebene Index wird als Array-Wert verwendet, wenn die Konstante gm_Matrices innerhalb des Shaders code aufgerufen wird.
MATRIX_MAX Dies ist kein Array-Index, sondern gibt die Größe des Matrix-Arrays im Vertex-Shader zurück.
MAX_VS_LIGHTS Dies ist kein Array-Index, sondern gibt die Anzahl der Lichter im Vertex-Shader zurück.

Die folgende Konstante ist auch für die Beleuchtung verfügbar:

Konstante Beschreibung
MAX_VS_LIGHTS Die maximale Anzahl der im Shader verfügbaren Punkt- und Richtungslichter

Die folgenden vordefinierten Matrix-Uniformen und Konstanten können in Ihrem Shader verwendet werden, um auf GameMaker-spezifische Werte zuzugreifen:

Konstante Beschreibung
gm_Matrices[matrix] Diese Array-Konstante gibt eine Transformationsmatrix zurück und ist eine der verfügbaren vordefinierten Uniformen, die GameMaker für Sie zur Verwendung im Shader-Editor code erstellt. Der Array-Index wird aus einer der oben aufgeführten Konstanten ausgewählt.
gm_BaseTexture Dies ist eine 2D-Sampler-Konstante, die die Textur des aktuellen Objekts zurückgibt, wie sie von GameMaker eingestellt wurde. Es wäre also die aktuelle sprite, Oberfläche oder Textur, die normalerweise beim Zeichnen verwendet würde, ohne dass der Shader aufgerufen wird.
gm_LightingEnabled Dies kann verwendet werden, um die GameMaker-Beleuchtung zu erhalten oder einzustellen, wenn 3D verwendet wird.
gm_FogStart Damit lässt sich der Abstand ermitteln, ab dem die Polygone mit der Nebelfarbe überblendet werden.
gm_RcpFogRange Damit lässt sich die Entfernung ermitteln, bei der der Nebel maximal ist und nichts mehr zu sehen ist.
gm_PS_FogEnabled Dies gibt true oder false zurück, wenn die GPU Pixelnebel aktiviert hat oder nicht.
gm_FogColour Dies kann verwendet werden, um die von GameMaker verwendete Nebelfarbe zu erhalten.
gm_VS_FogEnabled Dies gibt true oder false zurück, wenn der Grafikprozessor Vertex-Nebel aktiviert hat oder nicht.
gm_AlphaTestEnabled Dies kann verwendet werden, um Alphatests im Shader durchzuführen. Siehe gpu_set_alphatestenable() für weitere Informationen zum Alphatest.
gm_AlphaRefValue Dies kann verwendet werden, um den aktuellen Alpha-Test-Referenzwert zu erhalten. Siehe gpu_set_alphatestref() für weitere Informationen über den Alphatest-Referenzwert.
