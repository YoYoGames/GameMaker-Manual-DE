audio_listener_velocity

Diese Funktion kann verwendet werden, um dem Hörer Doppler-Effekte zu verleihen und eine Audio-Bewegung zu simulieren, die auf der vector basiert, die aus den gegebenen relativen x-, y- und z-Positionen aufgelöst wird. Wenn sich der Hörer selbst nicht bewegt oder die Bewegung nicht konstant ist, müssen Sie diese Werte normalerweise nicht einstellen. Wenn Sie aber zum Beispiel ein Scrolling-Spiel machen, bei dem der Spieler eine konstante Bewegung von unten nach oben und die Feinde eine konstante Bewegung von oben nach unten haben, würden Sie die Hörer- und Emitter-Geschwindigkeiten (für Emitter würden Sie audio_emitter_velocity() verwenden) auf die entsprechenden vectors einstellen, um den richtigen Dopplereffekt zu simulieren, wenn sie sich an der Spieler-Instanz vorbeibewegen.

HINWEIS wenn Sie mehrere Hörer haben, sollten Sie die Funktion audio_listener_set_velocity().

Die folgende Abbildung zeigt, wie dies für das oben genannte Beispielspiel eingerichtet werden könnte:

Syntax:

audio_listener_velocity(vx, vy, vz);

Argument Typ Beschreibung
vx Real The x velocity component of the listener (default 0).
vy Real The y velocity component of the listener (default 0).
vz Real The z velocity component of the listener (default 0).

Rückgabe:

N/A

Beispiel:

if speed > 0
{
    audio_listener_velocity(abs(hspeed), abs(vspeed), 0);
}

Die obige code prüft, ob die Geschwindigkeit der Player-Instanz über 0 liegt, und verwendet dann die entsprechenden absoluten Komponenten hspeed und vspeed, um die Zuhörergeschwindigkeit einzustellen.
