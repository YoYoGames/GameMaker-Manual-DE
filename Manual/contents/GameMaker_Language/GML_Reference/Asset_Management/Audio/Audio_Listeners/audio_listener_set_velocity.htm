audio_listener_set_velocity

Diese Funktion kann verwendet werden, um dem angegebenen Hörer Doppler-Effekte zu geben und eine Audio-Bewegung zu simulieren, die auf der vector basiert, die aus den angegebenen relativen x-, y- und z-Positionen aufgelöst wird. Der Standard-Hörerindex ist 0, aber Sie können die Funktion audio_get_listener_info() verwenden, um die verschiedenen verfügbaren Indizes für das Ziel platform zu erhalten.

Wenn sich der Hörer nicht bewegt oder die Bewegung nicht konstant ist, müssen Sie diese Werte normalerweise nicht einstellen, aber wenn Sie zum Beispiel ein scrollendes Spiel machen, bei dem sich der Spieler konstant von unten nach oben und die Feinde konstant von oben nach unten bewegen, würden Sie die Hörer- und Emitter-Geschwindigkeiten (für Emitter würden Sie audio_emitter_velocity()) auf die entsprechende vectors setzen, um den richtigen Dopplereffekt zu simulieren, wenn sie sich an der Spielerinstanz vorbeibewegen.

Die folgende Abbildung zeigt, wie dies für das oben genannte Beispielspiel eingerichtet werden könnte:

Syntax:

audio_listener_set_velocity(index, x, y, z);

Argument Typ Beschreibung
index Real- oder Audio-Hörer-ID The listener to change the velocity of (default 0).
x Real The x velocity of the listener (default 0).
y Real The y velocity of the listener (default 0).
z Real The z velocity of the listener (default 0).

Rückgabe:

N/A

Beispiel:

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

Das obige code prüft die Anzahl der verfügbaren Hörer und geht sie dann durch loops. Wenn ihre x-Geschwindigkeit nicht 0 ist, setzt es ihre Geschwindigkeitswerte auf 0, 0, 0.
