audio_play_sound

Mit dieser Funktion können Sie jede Sound-Ressource in Ihrem Spiel abspielen. Sie geben den Sound-Index an und weisen ihm eine Priorität zu, die dann verwendet wird, um zu bestimmen, wie mit den Sounds verfahren wird, wenn die Anzahl der abgespielten Sounds das von der Funktion festgelegte Limit überschreitet audio_channel_num(). Klänge mit niedriger Priorität werden zugunsten von Klängen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann zwischen 0 und 1 oder 0 und 100 liegen, da GameMaker sie gleich priorisiert). Dabei gilt: Je höher die Zahl, desto höher die Priorität, d.h. ein Sound mit der Priorität 100 wird gegenüber einem Sound mit der Priorität 1 bevorzugt. Das dritte argument dient dazu, den Sound loop zu erzeugen. Wenn Sie es auf true setzen, wird der Sound loop, bis er gestoppt wird, und wenn Sie es auf false setzen, wird der Sound nur einmal abgespielt.

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Ton zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn dann mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Tons gleichzeitig abspielen lassen, nur eine Instanz dieses Tons mit den Audiofunktionen bearbeiten können.

Syntax:

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

Argument Typ Beschreibung
index Sound Asset The index of the sound to play.
priority Real Set the channel priority for the sound.
loop Boolesche Sets the sound to loop or not.
gain Real OPTIONAL Wert für den Gewinn.
offset Real OPTIONAL Die Zeit (in Sekunden), auf die der Startpunkt gesetzt werden soll. Werte, die länger als die Länge des angegebenen Tons sind, werden ignoriert.
pitch Real OPTIONAL Der Pitch-Multiplikator (Standardwert 1).
listener_mask Real OPTIONAL Die Bitmasken-Daten, die für den Ton gesetzt werden sollen. Auf dem HTML5-Ziel hat dies keine Auswirkungen, da das Ziel nicht mehr als einen Hörer unterstützt.

Rückgabe:

Sound Instance ID

Beispiel 1:

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

Die obige code prüft die globale Variable "health", und wenn sie kleiner oder gleich 0 ist, wird 1 aus der globalen Variable "lives" entfernt und ein Ton abgespielt.

Beispiel 2:

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

Die obige code prüft, ob sich die Instanz hinter der rechten Raumgrenze befindet und spielt einen Ton mit einer Verstärkung von 1,1 und einer Tonhöhe von 2 (das Doppelte der normalen Tonhöhe).
