audio_debug

Diese Funktion kann verwendet werden, um Debug-Informationen über das Audiosystem anzuzeigen. true schaltet sie ein und false aus switch. Wenn Sie debugging Audio auf diese Weise einschalten, zeigt das Overlay die folgenden Informationen an:

  • Handle - Das OpenAL-Quellhandle (einzigartig für jeden gespielten Sound)
  • Data Handle - Das OpenAL buffer Handle (zeigt den gleichen Wert für mehrere Sounds mit dem gleichen Source Handle)
  • Verarbeitete Puffer - wie viele buffers gespielt wurden
  • Puffer in der Warteschlange - wie viele buffers in die Warteschlange gestellt wurden (und noch darauf warten, gespielt zu werden)
  • Gain - Der Verstärkungswert für das Audio (0 - 65536)
  • Play Offset - die Sampleposition (aktuelle Abspielposition) im aktuellen Puffer
  • Bars in der Warteschlange Buffers - ein kleines Kästchendiagramm der bearbeiteten/eingereihten buffers mit dem Play Offset (aktuelle Position) als blaue Linie

Für synchronisierte Audiogruppen haben Sie außerdem die Funktion audio_sync_group_debug().

HINWEIS Diese Funktion funktioniert nicht mit dem HTML5-Ziel platform.

Syntax:

audio_debug(enable);

ArgumentTyp Beschreibung
enable Enable (set to true) oder deaktivieren (false) Sie Audio debugging.

Rückgabe:

Beispiel:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

code Die obige Seite switch schaltet das Audio-Debug-Overlay ein oder aus, je nachdem, ob das Spiel im Debug-Modus läuft oder nicht.
