audio_play_sound_on

Mit dieser Funktion können Sie eine beliebige Klangressource über einen Emitter abspielen, wobei Änderungen an der Verstärkung, der Position, der Tonhöhe oder der Anschlagsstärke des Emitters sich darauf auswirken, wie der Benutzer den abgespielten Klang wahrnimmt. Sie geben den zu verwendenden Emitter-Index und den Sound-Index des abzuspielenden Sounds an und bestimmen, ob der Sound auf loop abgespielt werden soll oder nicht. Als Nächstes können Sie eine Priorität zuweisen, die dann verwendet wird, um zu bestimmen, wie Klänge behandelt werden, wenn die Anzahl der abgespielten Klänge die von der Funktion gesetzte Grenze überschreitet audio_channel_num(). Geräusche mit niedrigerer Priorität werden zugunsten von Geräuschen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann zwischen 0 und 1 oder 0 und 100 liegen, da GameMaker sie gleich priorisiert). Dabei gilt: Je höher die Zahl, desto höher die Priorität, d.h. ein Sound mit der Priorität 100 wird einem Sound mit der Priorität 1 vorgezogen.

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Ton zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn dann mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Tons gleichzeitig abspielen lassen, nur eine Instanz dieses Tons mit den Audiofunktionen bearbeiten können.

Syntax:

audio_play_sound_on(emitter, sound, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argument Typ Beschreibung
emitter Audio-Sender-ID The index of the emitter to use.
sound Sound Asset The index of the sound to use.
loop Boolesche Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Wert für den Gewinn.
offset Real OPTIONAL Die Zeit (in Sekunden), auf die der Startpunkt gesetzt werden soll. Werte, die länger als die Länge des angegebenen Tons sind, werden ignoriert.
pitch Real OPTIONAL Der Pitch-Multiplikator (Standardwert 1).
listener_mask Real OPTIONAL Die Bitmasken-Daten, die für den Ton gesetzt werden sollen. Auf dem HTML5-Ziel hat dies keine Auswirkungen, da das Ziel nicht mehr als einen Hörer unterstützt.

Rückgabe:

Sound Instance ID

Beispiel 1:

if global.SFX
{
    audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

Die obige code prüft die globale Variable "SFX", und wenn sie true zurückgibt, wird der in der Variable "snd_Explode" indizierte Sound über den in der Variable "s_emit" indizierten Emitter mit niedriger Priorität und ohne Schleife abgespielt.

Beispiel 2:

if hit == true
{
    audio_play_sound_on(s_emit, snd_Hit, false, 1, 1.3);
}

Die obige code prüft, ob die Instanz, die die code ausführt, getroffen wurde. Falls true, wird der Sound "snd_Hit" über den Emitter "s_emit" mit einer etwas höheren Verstärkung von 1,3 abgespielt. Die hier eingestellte Verstärkung wird mit der Verstärkung des Emitters multipliziert, um die endgültige Verstärkung zu erhalten, mit der der Ton abgespielt wird.
