audio_erstellen_abspielen_warteschlange

Diese Funktion bereitet eine buffer Warteschlange für Audio vor. Sie übergeben das Datenformat (derzeit werden nur buffer_u8 oder buffer_s16 unterstützt), die Abtastrate (die zwischen 1000hz und 48000hz liegen kann) und die Kanäle, die das Audio benötigt, aus einer der unten aufgeführten Konstanten:

Konstante Beschreibung
audio_mono Mono (einkanaliger) Ton.
audio_stereo Stereo (Zweikanal) Audio.
audio_3D 3D (5.1) Audio.


Beachten Sie, dass Sie alle Audio-Warteschlangen, die mit dieser Funktion erstellt wurden, wieder freigeben sollten, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, indem Sie die Funktion audio_free_play_queue().

Diese Funktion löst ein asynchrones Audio-Wiedergabe-Ereignis aus, wenn die Warteschlange das Ende erreicht hat, und in diesem Ereignis wird eine spezielle DS-Map in der Variablen async_load mit den folgenden Schlüssel/Wert-Paaren erstellt:

  • "queue_id" - die Warteschlange, die die Wiedergabe der Daten beendet hat
  • "buffer_id" - die buffer, die nicht mehr abgespielt wird von
  • "queue_shutdown" - wird während der normalen Wiedergabe auf 0 gesetzt und auf 1, wenn das Ereignis empfangen wird, weil audio_free_play_queue() aufgerufen wurde. Wenn dieser Wert auf 1 gesetzt ist, sollen keine weiteren Daten in die Warteschlange gestellt werden.

Syntax:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Argument Typ Beschreibung
queueFormat Puffer Datentyp Konstante The format of the buffer data to use (buffer_u8 oder buffer_s16).
queueRate Real The sample rate of the data in the buffer queue.
queueChannels Audio-Kanal-Typ The channels to use from one of the constants listed above.

Rückgabe:

Audio Queue ID

Beispiel:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

Die obige code erstellt eine neue Audio-Warteschlange und weist ihre ID einer Variablen zu, um sie später zu verwenden.
