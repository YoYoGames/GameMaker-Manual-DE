audio_erzeugen_puffer_klang

Mit dieser Funktion können Sie einen neuen Sound aus dem Inhalt eines buffer erstellen. buffer wurde zuvor erstellt (siehe die Pufferfunktionen für Details dazu) und es wurden bereits Daten hinzugefügt oder geladen. Sie übergeben ihn dann an diese Funktion mit dem Datenformat (derzeit werden nur buffer_u8 oder buffer_s16 unterstützt), der Abtastrate (die zwischen 1000hz und 48000hz liegen kann) und einem Offset in buffer, von dem die Daten abgeholt werden sollen. Sie müssen auch die Anzahl der Samples in buffer und die Kanäle, die der Sound benötigt, angeben. Diese Kanäle werden durch eine der folgenden Konstanten definiert:

Audio-Kanal-Typ
Konstante Beschreibung
audio_mono Mono (einkanaliger) Ton.
audio_stereo Stereo (Zweikanal) Audio.
audio_3D 3D (5.1) Audio.

Beachten Sie, dass Sie, nachdem Sie einen Sound erstellt haben, den zugehörigen Zeigerindex freigeben sollten, wenn er nicht mehr benötigt wird. audio_free_buffer_sound(). Wenn Sie dies nicht tun und dann die Variable neu zuweisen oder den Raum wechseln usw., geht die Sound-ID verloren und Sie haben ein Speicherleck. Beachten Sie auch, dass Sie buffer nicht löschen können, wenn ein Sound daraus erstellt wurde und der Sound nicht vorher freigegeben wurde. Sie sollten also zuerst den Sound (oder die Sounds) freigeben und dann buffer löschen. Es ist auch erwähnenswert, dass das Hinzufügen von etwas zu buffer oder das Ändern der Größe von buffer, nachdem ein Sound daraus erstellt wurde, zu unerwarteten Ergebnissen führt und nicht empfohlen wird - sobald Sie begonnen haben, Sounds aus einem buffer zu erstellen, sollten Sie es danach nicht mehr manipulieren.

Syntax:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);

Argument Typ Beschreibung
bufferId Puffer-ID The ID of the buffer to use.
bufferFormat Puffer Datentyp Konstante The format of the data in the buffer (buffer_u8 oder buffer_s16).
bufferRate Real The sample rate of the data in the buffer.
bufferOffset Real The offset into the buffer to read the sample data from (in bytes).
bufferLength Real The length of the buffer (the number of bytes of audio data, excluding the header).
bufferChannels Audio-Kanal-Typ The channels to use from one of the constants listed above.

Rückgabe:

Sound Asset

Beispiel:

var rate = 44100;
var hertz = irandom_range(220, 880);
var samples = 44100;

var bufferId = buffer_create(rate, buffer_fast, 1);
buffer_seek(bufferId, buffer_seek_start, 0);

var num_to_write = rate / hertz;
var length = buffer_get_size(bufferId);
var val_to_write = 1;

for (var i = 0; i < (samples / num_to_write) + 1; i++)
{
    for (var j = 0; j < num_to_write; j++)
    {
        buffer_write(bufferId, buffer_u8, val_to_write * 255);
    }
    val_to_write = (1 - val_to_write);
}

soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, length, audio_stereo);

Im obigen Beispiel wird eine buffer erstellt und dann prozedural mit Daten gefüllt. Diese Daten werden dann verwendet, um einen neuen Sound zu erstellen, der in der Variablen "soundId" gespeichert wird.
