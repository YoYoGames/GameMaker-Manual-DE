audio_freie_wiedergabe_warteschlange

Diese Funktion wird verwendet, um den mit der angegebenen Audio-Warteschlange verbundenen Speicher freizugeben. Der Index der Warteschlange ist der Wert, der zurückgegeben wird, wenn Sie die Warteschlange mit der Funktion audio_create_play_queue()erstellt haben, und diese Funktion sollte aufgerufen werden, wenn die Warteschlange nicht mehr benötigt wird, um Speicherlecks zu vermeiden. Das Freigeben der Warteschlange stoppt jeden Ton, der gerade abgespielt wird, und Sie können die buffer, von der ein Ton gestreamt wird, erst löschen, wenn die Warteschlange, der er zugewiesen ist, freigegeben wurde. Diese Funktion löst ein asynchrones Ereignis für die Audiowiedergabe aus, und in diesem Ereignis wird eine spezielle DS-Map in der Variablen async_load mit den folgenden Schlüssel/Wert-Paaren erstellt:

  • "queue_id" - die Warteschlange, die die Wiedergabe der Daten beendet hat
  • "buffer_id" - die buffer, die nicht mehr abgespielt wird von
  • "queue_shutdown" - wird während der normalen Wiedergabe auf 0 gesetzt und auf 1, wenn das Ereignis empfangen wird, weil Sie die Warteschlange mit dieser Funktion freigegeben haben. Wenn dieser Wert auf 1 gesetzt ist, wollen Sie keine weiteren Daten in die Warteschlange stellen.

Syntax:

audio_free_play_queue(queueIndex);

Argument Typ Beschreibung
queueIndex Audio-Warteschlangen-ID The index of the queue to free.

Rückgabe:

N/A

Beispiel:

var q_id = async_load[? "queue_id"];
if (q_id == audio_queue)
{
    audio_free_play_queue(audio_queue);
    buffer_delete(audio_buffer);
}

Die obige code würde in der asynchronen Audio-Wiedergabe-Ereignis gehen und prüft, ob das Audio, das die Wiedergabe beendet hat, ist aus der "audio_queue" und wenn es ist, gibt es die Warteschlange und die buffer mit ihm verbunden.
