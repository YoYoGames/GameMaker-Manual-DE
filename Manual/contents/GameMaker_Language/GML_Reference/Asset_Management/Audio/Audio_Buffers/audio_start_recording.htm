audio_start_aufzeichnung

Diese Funktion startet die Audioaufnahme von der indizierten Rekorderquelle. Sie können die Anzahl der Rekorderquellen mit der Funktion audio_get_recorder_count()Sobald Sie mit der Aufnahme beginnen, werden die Audiodaten in einem temporären buffer gespeichert und lösen ein asynchrones Audioaufnahme-Ereignis aus. Dieses Ereignis wird bei jedem Aufnahmeschritt ausgelöst und erstellt die spezielle DS-Map in der Variablen async_load mit den folgenden Schlüssel/Wert-Paaren:

  • "buffer_id" - die ID des temporären buffer, das Sie zum Abrufen der Audiodaten verwenden können
  • "channel_index" - der Index des Aufzeichnungskanals (wie von der Funktion zurückgegeben), von dem diese Daten stammen
  • "data_len" - die Länge der empfangenen Daten (in Bytes)

Beachten Sie, dass nach dem Auslösen des asynchronen Ereignisses die Audiodaten im temporären buffer gelöscht werden. Kopieren Sie daher die Daten in ein benutzerdefiniertes buffer, das Sie zuvor erstellt haben.

HINWEIS Die meisten platforms unterstützen die Aufzeichnung von Audio in irgendeiner Form, aber das bedeutet nicht, dass alle Geräte dies zulassen, selbst wenn die platform dies tut, daher sollten Sie immer überprüfen, ob die audio_get_recorder_count() einen Wert größer als 0 zurückgibt, um zu prüfen, ob Aufnahmegeräte verfügbar sind, bevor Sie die übrigen Aufnahmefunktionen verwenden.

Syntax:

audio_start_recording(recorder_index);

Argument Typ Beschreibung
recorder_index Real The index of the recorder source to use.

Rückgabe:

Buffer ID

Beispiel:

audio_record = audio_start_recording(0);

Die obige code startet die Aufzeichnung von der Rekorderquelle 0 und speichert den Kanalindex der Aufzeichnung in der Variablen "audio_record" zur Verwendung im asynchronen Audioaufzeichnungsereignis.
