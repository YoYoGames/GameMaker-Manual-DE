audio_destroy_stream

Wenn Sie zuvor einen Audiostrom aus einer Datei mit der Funktion audio_create_stream() erstellt haben und diesen Ton nicht mehr benötigen, können Sie diese Funktion aufrufen, um ihn aus dem Speicher zu löschen. Jeder weitere Aufruf des Tonindexes, nachdem er zerstört wurde, führt zu einem Fehler. Es sollte beachtet werden, dass dies den Stream freigibt, aber auf dem Zielsystem platform wird dies möglicherweise nicht in einem Speichermanager angezeigt. Das liegt daran, dass GameMaker Speicherressourcen bündelt, um einen Overhead bei der Speicherzuweisung zu vermeiden. Der Speicher bleibt also so lange belegt, bis er für etwas anderes benötigt oder für einen neuen Stream wiederverwendet wird.

HINWEIS: Diese Funktion ist für das HTML5-Ziel platform nicht verfügbar.

Syntax:

audio_destroy_stream(filename);

Argument Typ Beschreibung
filename Zeichenfolge Path to the audio file that was streamed.

Rückgabe:

N/A

Beispiel:

audio_destroy_stream(snd);

Die obige code löscht den in der Variablen "snd" indizierten Ton aus dem Speicher.
