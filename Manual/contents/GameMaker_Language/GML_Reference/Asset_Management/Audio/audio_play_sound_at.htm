audio_play_sound_at

Mit dieser Funktion können Sie eine beliebige Klangressource an einer bestimmten Position innerhalb des Audioraums abspielen. Sie geben den Klangindex an und weisen ihm dann eine Position im 3D-Raum zu. Die Standardposition des Hörers ist (0, 0, 0). Das bedeutet, dass Sie die x-Position auf einen negativen Wert setzen sollten, wenn der Hörer nicht verschoben wurde und Sie möchten, dass der Ton z. B. von links kommt (weitere Informationen zur Einstellung der Hörerposition finden Sie unter audio_listener_position()).

Sie können auch eine Abklingdistanz einstellen (bei 0 ist der Ton stumm, Standardwert ist 100), die den Ton ausklingen lässt, je weiter er sich von der Zuhörerposition entfernt. Wie die Ausblendung selbst zu hören ist, hängt von der Ausblendungsreferenz (das ist der Abstand, unter dem die Lautstärke der Quelle normalerweise um die Hälfte abnimmt) und dem Ausblendungsfaktor (der den Ton nur über die Ausblendungsreferenz hinaus beeinflusst) ab. Der voreingestellte Faktor ist normalerweise 1, und die Wirkung der verschiedenen Abschwächungswerte hängt vom gewählten Modell ab (eine vollständige Anleitung zu den verschiedenen Abschwächungsmodellen und der Verwendung dieser Werte finden Sie in der Funktion audio_falloff_set_model()).

Die nächsten beiden arguments dienen dazu, den Sound auf loop einzustellen oder nicht und dem Sound eine Priorität zuzuweisen. Diese Priorität wird dann verwendet, um zu bestimmen, wie Klänge behandelt werden, wenn die Anzahl der abgespielten Klänge die von der Funktion audio_channel_num(). Geräusche mit niedrigerer Priorität werden zugunsten von Geräuschen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann zwischen 0 und 1 oder 0 und 100 liegen, da GameMaker sie gleich priorisiert). Dabei gilt: Je höher die Zahl, desto höher die Priorität, d.h. ein Sound mit der Priorität 100 wird einem Sound mit der Priorität 1 vorgezogen.

Diese Funktion gibt eine eindeutige Indexnummer für den abgespielten Ton zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Tons gleichzeitig abspielen lassen, nur eine Instanz dieses Tons mit den Audiofunktionen bearbeiten können.

Syntax:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority, [gain], [offset], [pitch], [listener_mask]);

Argument Typ Beschreibung
index Sound Asset The index of the sound to play.
x Real The x position.
y Real The y position.
z Real The z position.
falloff_ref Real The falloff reference relative to the listener (clamp).
falloff_max Real The maximum falloff distance relative to the listener.
falloff_factor Real The falloff factor (default 1).
loop Boolesche Flags the sound as looping or not.
priority Real Set the channel priority for the sound.
gain Real OPTIONAL Wert für den Gewinn.
offset Real OPTIONAL Die Zeit (in Sekunden), auf die der Startpunkt gesetzt werden soll. Werte, die länger als die Länge des angegebenen Tons sind, werden ignoriert.
pitch Real OPTIONAL Der Pitch-Multiplikator (Standardwert 1).
listener_mask Real OPTIONAL Die Bitmasken-Daten, die für den Ton gesetzt werden sollen. Auf dem HTML5-Ziel hat dies keine Auswirkungen, da das Ziel nicht mehr als einen Hörer unterstützt.

Rückgabe:

Sound Instance ID

Beispiel 1:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);
}

Die obige code prüft die globale Variable "SFX" und wenn sie true zurückgibt, dann wird der in der Variable "snd_Waterfall" indizierte Sound an seiner Raumposition geloopt, mit einer Falloff-Referenz von 100, einem Falloff-Abstand von 300, einem Falloff-Faktor von 1 und einer niedrigen Priorität.

Beispiel 2:

if global.SFX
{
    audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1, 1, 2);
}

Die obige code prüft die globale Variable "SFX", und wenn sie true zurückgibt, wird der in der Variablen "snd_Waterfall" indizierte Sound an seiner Raumposition mit den angegebenen Falloff-Einstellungen und ohne Änderung der Verstärkung in einer Schleife abgespielt. Der Sound wird sofort ab Sekunde 2 abgespielt.
