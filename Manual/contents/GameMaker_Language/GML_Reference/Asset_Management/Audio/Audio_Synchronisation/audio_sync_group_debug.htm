audio_sync_group_debug

iese Funktion kann verwendet werden, um Debug-Informationen über eine beliebige Sync-Gruppe anzuzeigen (der Gruppenindex ist der Wert, der bei der Erstellung der Gruppe mit der Funktion audio_create_sync_group()). Sie können die Funktion mit einem Wert von -1 aufrufen, um switch das Overlay auszuschalten.

Wenn Sie debugging die Audio-Sync-Gruppen aufrufen, wird das Overlay angezeigt:

  • Welche Sync-Gruppe wird debuggt
  • Wie viele Kanäle sind enthalten?
  • Wie oft die Synchronisierungsgruppe fehlgeschlagen ist (d. h. sie hat versucht, zu viel Arbeit zu erledigen)
  • Eine Liste der Kanäle
    • Rot bedeutet, dass dieser Kanal derzeit keine Arbeit verrichtet
    • Gelb bedeutet, dass sich der Kanal im Anfangszustand befindet (noch nicht dekodiert oder abgespielt)
    • Grün bedeutet, dass die Spur gerade gesucht/dekodiert wird
    • Lila bedeutet, dass der Kanal keine Daten mehr hat.
  • Ein Profiling-Balken, der anzeigt, welche Arbeit der Dekodier-Thread gerade verrichtet
    • blau bedeutet, dass eine Datei zum ersten Mal gelesen wird
    • rot bedeutet, dass der Thread eine Datei liest, um herauszufinden, von wo aus er lesen soll
    • grün bedeutet, dass der Thread gerade Daten dekodiert
    • lila bedeutet, dass der Thread schläft, bis er arbeiten muss

Beachten Sie, dass Sie für allgemeine debugging von Audio auch die Funktion audio_debug().

HINWEIS: Diese Funktion ist für das HTML5-Ziel platform nicht verfügbar.

Syntax:

audio_sync_group_debug(group_index);

Argument Typ Beschreibung
group_index Audio-Synchronisationsgruppen-ID The group index to debug, or -1 to switch off the debugging.

Rückgabe:

N/A

Beispiel:

if (debug_mode)
{
    audio_sync_group_debug(sg);
}
else
{
    audio_sync_group_debug(-1);
}

code Die obige switch schaltet das Audio-Sync-Group-Debug-Overlay ein oder aus, das die in der Variable "sg" indizierte Sync-Group anzeigt, je nachdem, ob das Spiel im Debug-Modus läuft oder nicht.
