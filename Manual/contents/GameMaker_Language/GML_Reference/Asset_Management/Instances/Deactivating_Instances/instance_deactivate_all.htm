instance_deactivate_all

Diese Funktion deaktiviert alle Instanzen, die sich in dem Moment im Raum befinden, in dem code ausgeführt wird. Dies kann die Instanz einschließen, die die code ausführt, wenn das "notme"-Flag auf false gesetzt ist, aber normalerweise möchtest du, dass diese Instanz aktiv ist, in diesem Fall sollte das "notme"-Flag auf true gesetzt werden. Beachte, dass die Deaktivierung nicht sofort erfolgt und eine Instanz, die auf diese Weise deaktiviert wurde, bis zum Ende des Ereignisses, in dem die Funktion aufgerufen wurde, nicht als inaktiv betrachtet wird.

HINWEIS: Wenn Sie eine Instanz beim Start des Raums deaktivieren (d.h. aus der Raumerstellung code oder aus einem Instanzerstellungsereignis einer Instanz innerhalb des Raums), werden alle Instanzen, die aus dem Raumeditor im Raum platziert werden , weiterhin ihr Erstellungsereignis ausführen, bevor sie deaktiviert werden.

WARNUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, verhindert NICHT, dass deren Einbauten innerhalb der Physiksimulation interagieren. Dafür solltest du ihre phy_active Variable auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Syntax:

instance_deactivate_all(notme);

Argument Typ Beschreibung
notme Boolesche Whether to keep the calling instance activated (true) or not (false).
collision_space Collision Space  The collision space where instances will be deactivated

Rückgabe:

N/A

Beispiel:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Die obige code deaktiviert alle Instanzen mit Ausnahme derjenigen, die die code ausführt, und aktiviert dann eine Region innerhalb des Raums.
