Deaktivieren von Instanzen

GameMaker bietet Ihnen die Möglichkeit, Instanzen "abzuschalten", so dass sie in keiner Weise mehr bearbeitet werden. Technisch gesehen existieren sie nicht mehr wirklich, außer als Zeiger innerhalb des Deaktivierungsprozesses selbst, was bedeutet, dass eine deaktivierte Instanz in keiner Weise manipuliert oder verändert werden kann, bis sie wieder aktiviert wird. Diese Funktionen sollten also mit großer Vorsicht verwendet werden, da sie bei unsachgemäßer Verwendung zu Problemen führen können, insbesondere bei persistenten Objekten, da ein persistentes Objekt, das deaktiviert wurde, nicht in den nächsten Raum verschoben werden kann, wenn es nicht zuerst wieder aktiviert wird (wodurch es effektiv aus dem Spiel gelöscht wird). Beachte auch, dass Aktivierung und Deaktivierung nicht sofort erfolgen und eine Instanz, deren Zustand auf diese Weise geändert wurde, bis zum Ende des Ereignisses, in dem die Funktion aufgerufen wurde, nicht als aktiv/inaktiv betrachtet wird.

Es ist normalerweise nicht notwendig, Instanzen bei jedem Spielschritt zu deaktivieren, da dies zu Leistungsproblemen führen kann. Stattdessen wird empfohlen, diese Funktionen nur alle paar Schritte in einem Alarm (z. B.) auszuführen oder wenn die Kameraperspektive ihre Position geändert hat, und es ist besonders wichtig, dass Sie diese Funktionen nicht im Draw-Ereignis verwenden, da dies zu schwerwiegenden Fehlern in Ihrem Spiel führen kann.

Eine deaktivierte Instanz existiert im Spiel nicht mehr, aber auf die Variablen einzelner Instanzen kann weiterhin zugegriffen werden. Sie können jedoch nicht with(object) oder with(instance) verwenden, sondern müssen direkt auf die Instanz über ihre eindeutige ID in der folgenden Form zugreifen:

val = inst.variable;

Im obigen Beispiel wäre "inst" eine Variable, die die ID der deaktivierten Instanz enthält, und dies ist die einzige Möglichkeit, auf eine deaktivierte Instanz zu verweisen, ohne dass sie zuvor aktiviert wurde.

HINWEIS Wenn Sie eine Instanz beim Raumstart deaktivieren (d.h. aus dem Raumerstellungs-Ereignis code oder aus einem Instanzerstellungs-Ereignis einer Instanz innerhalb des Raums), werden alle Instanzen, die aus dem Raum-Editor im Raum platziert werden, weiterhin ihr Erstellungs-Ereignis ausführen, bevor sie deaktiviert werden.

ACHTUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, stoppt NICHT die Interaktion der Geräte innerhalb der Physiksimulation. Daher sollten Sie die phy_active Variable auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Zum Deaktivieren von Instanzen gibt es die folgenden Funktionen:

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
