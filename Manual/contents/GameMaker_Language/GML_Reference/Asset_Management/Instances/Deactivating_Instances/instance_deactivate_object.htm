instance_deactivate_object

Mit dieser Funktion können Sie eine einzelne Instanz oder alle Instanzen eines bestimmten Objekts aus allen zuvor aktivierten Instanzen deaktivieren. Beachten Sie, dass Sie, wenn Sie eine Instanz oder ein Objekt deaktiviert haben, das als persistent gekennzeichnet ist, es müssen Sie es vor dem Raumwechsel erneut mit der Funktion instance_activate_object() reaktivieren, bevor Sie den Raum wechseln, da es sonst nicht übernommen und stattdessen verworfen wird. Beachte auch, dass die Deaktivierung nicht sofort erfolgt und eine Instanz, die auf diese Weise deaktiviert wurde, bis zum Ende des Ereignisses, in dem die Funktion aufgerufen wurde, nicht als inaktiv betrachtet wird.

HINWEIS: Wenn Sie eine Instanz beim Start des Raums deaktivieren (d.h. aus der Raumerstellung code oder aus einem Instanzerstellungsereignis einer Instanz innerhalb des Raums), werden alle Instanzen, die aus dem Raumeditor im Raum platziert werden , weiterhin ihr Erstellungsereignis ausführen, bevor sie deaktiviert werden.

WARNUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, verhindert NICHT, dass deren Einbauten innerhalb der Physiksimulation interagieren. Dafür solltest du ihre phy_active Variable auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Syntax:

instance_deactivate_object(obj);

ArgumentTyp Beschreibung
obj The object or instance to deactivate (the keyword all can also be used).

Rückgabe:

Beispiel:

instance_deactivate_object(obj_Enemy);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Die obige code deaktiviert alle Instanzen des Objekts "obj_Enemy" und aktiviert dann eine Region innerhalb des Raums.
