Instanzen

Instanzen werden aus Objekten erstellt und können, sobald sie erstellt sind, über GML code und GML Visual Actions geändert und manipuliert werden. Die wichtigsten Instanzfunktionen im Zusammenhang mit Instanzen, die verfügbar sind und am häufigsten verwendet werden, sind:

  • instance_create_layer
  • instance_create_depth
  • instance_destroy
  • instance_exists
  • instance_change
  • instance_copy
  • instance_find
  • instance_furthest
  • instance_nearest
  • instance_number
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

Beachten Sie, dass es zwei zusätzliche eingebaute Variablen gibt (es handelt sich um Variablen mit globalem Geltungsbereich ) und eine Funktion, die sich auf die Verwendung von Instanzen bezieht und für die Abfrage bestimmter Details nützlich sein kann:

  • instance_id_get
  • instance_id
  • instance_count

Es gibt eine Reihe von eingebauten Variablen, die in jeder Instanz eines Objekts verfügbar sind und die verwendet werden können, um verschiedene Aspekte zu steuern, wie die Instanz angezeigt wird, wie sie andere Assets verwendet und wie sie auf Kollisionen reagiert, usw., die alle hier aufgelistet sind:

  • Instanz-Variablen

Instanzen verfügen über eine Array-Variable, die zum Abrufen oder Festlegen von Alarmereigniswerten verwendet werden kann, aber es gibt auch zwei Funktionen, mit denen dasselbe erreicht werden kann:

  • alarm_get
  • alarm_set

Schließlich hast du auch die Möglichkeit, Instanzen zu deaktivieren, was bedeutet, dass sie keine ihrer Ereignisse mehr verarbeiten, obwohl sie im Spiel noch existieren. Die diesbezüglichen Funktionen sind auf der folgenden Seite zu finden:

  • Deaktivieren von Instanzen
