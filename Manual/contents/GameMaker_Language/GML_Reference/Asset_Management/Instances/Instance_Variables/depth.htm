depth

Wenn Sie ein Objekt erstellen, können Sie ihm eine anfängliche Tiefe zuweisen, die definiert, wie die Instanzen dieses Objekts im Raum gezeichnet werden, wenn das Spiel gespielt wird, und diese Variable kann verwendet werden, um diesen Tiefenwert zu erhalten und zu ändern, während das Spiel läuft. Normalerweise brauchst du diese Variable nicht zu verwenden, da du die Instanzen so einstellst, dass sie zu diskreten Layern hinzugefügt werden, die wiederum auf eine bestimmte Tiefe eingestellt sind, aber es kann sein, dass du die Tiefe einer Instanz mit diesem Wert ändern willst, in welchem Fall ein "temporärer Layer" speziell für die Instanz mit der angegebenen Tiefe erstellt wird. In diesem Fall wird eine "temporäre Ebene" speziell für die Instanz in der angegebenen Tiefe erstellt. Beachten Sie, dass diese temporäre Ebene aus dem Speicher entfernt wird, wenn sich keine Instanzen in der gleichen Tiefe befinden (im Gegensatz zu regulären Ebenen, die auch dann erhalten bleiben, wenn sich nichts auf ihnen befindet).

WICHTIG! Wenn Sie die Variable depth ändern und GameMaker die Ebenen verwaltet, gibt die eingebaute layer Variable -1 und nicht die Ebenen-ID zurück, da verwaltete Ebenen nicht über code manipuliert werden können.

Je niedriger der Tiefenwert einer Instanz ist, desto "näher an der Kamera" wird diese Instanz gezeichnet, während ein höherer Tiefenwert bedeutet, dass die Instanz "weiter weg von der Kamera" gezeichnet wird, d.h.: -1000 wird über -100 gezeichnet, das wiederum über 0 gezeichnet wird, und so weiter.

Beachten Sie, dass Instanzen eines Objekts, die dieselbe Tiefe haben, über- oder untereinander gezeichnet werden können, auch wenn sie im Raumeditor übereinander angeordnet zu sein scheinen. Wenn Sie garantieren wollen, dass etwas über oder unter alles andere gezeichnet wird, sollten Sie die Tiefe (oder Ebene) immer explizit festlegen. Beachte auch, dass es einen minimalen (-16000) und maximalen (+16000) Tiefenwert gibt, außerhalb dessen Instanzen nicht gezeichnet werden, obwohl sie immer noch existieren und Ereignisse verarbeiten.

HINWEIS Sie können die Tiefe einer Instanz nicht in ihrem Draw-Ereignis festlegen (alle anderen Ereignisse sind in Ordnung).

Beachten Sie, dass die Änderung der Tiefe einer Instanz die Anwendung von Filtern und Effekten auf die Instanz verändern kann, da bei einer Änderung der Tiefe unter die Tiefe einer Effekt-Ebene deren Effekt nicht mehr auf die Instanz angewendet wird.

Syntax:

depth

Rückgabe:

Real (single precision floating point value)

Beispiel:

if y != yprevious
{
    depth = -y;
}

Die obige code prüft, ob sich die y-Position geändert hat, und wenn ja, wird auch die Tiefe entsprechend angepasst.
