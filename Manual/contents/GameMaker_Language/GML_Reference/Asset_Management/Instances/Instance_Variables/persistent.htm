persistent

Diese Variable kann gelesen werden, um herauszufinden, ob die Instanz als persistent gekennzeichnet ist oder nicht, oder sie kann verwendet werden, um die Persistenz der Instanz auf true (persistent) oder false (nicht persistent) zu setzen. Eine persistente Instanz ist eine Instanz, die von Raum zu Raum "mitgenommen" wird, was bedeutet, dass sie (zum Beispiel) nur einmal zu Beginn des Spiels erstellt werden muss und in allen weiteren Räumen vorhanden sein wird. Bei der Persistenz ist Vorsicht geboten, da man leicht den Überblick über persistente Instanzen verlieren kann, was später bei der Entwicklung des Spiels zu Problemen führen kann.

Wenn Sie eine Instanz mit aktivierter Persistenz in einem Raum erstellen, wird diese Variable auf true gesetzt und ihr wird eine Ebene oder eine Tiefe zugewiesen (je nachdem, welche Funktion Sie zum Erstellen der Instanz des Objekts verwendet haben). Wenn der Raum gewechselt wird und der folgende Raum keine Ebene mit dem gleichen Namen oder der gleichen Tiefe wie die zugewiesene hat, wird eine neue Ebene für die Instanz erstellt, die über die Räume hinweg persistent ist. Wenn du bei der Erstellung der Instanz einen Namen für die Ebene vergeben hast, dann wird die neue Ebene auch den gleichen Namen wie die ursprüngliche Ebene haben, und wenn du der Instanz eine Tiefe zugewiesen hast, dann wird die neue Ebene "_layer_XXX" heißen, wobei "XXX" ein Hex-Wert ist, der verwendet wird, um der Ebene einen eindeutigen Namen zu geben. Diese Ebenen werden bei Raumende aus dem Raum entfernt. Ein weiterer Hinweis zu Ebenen und persistenten Instanzen ist, dass wenn du der Instanz bei der Erstellung eine benannte Ebene zugewiesen hast und es in den folgenden Räumen eine andere Ebene mit demselben Namen gibt, dann wird die persistente Instanz der Ebene mit demselben Namen zugewiesen , unabhängig von der Tiefe der Ebene. Wenn eine persistierte Instanz in einen Raum mit einer Ebene in der gleichen Tiefe wie bei der Erstellung der Instanz verschoben wird, wird sie nicht dieser Ebene zugewiesen, sondern es wird eine neue Ebene in der gleichen Tiefe erstellt (gemäß der zuvor erläuterten Benennungskonvention).

Bei einem persistenten Objekt werden weiterhin die Ereignisse Spielstart, Spielende, Raumstart und Raumende ausgelöst. Wenn Sie jedoch das Spiel neu starten (z. B. mit der Funktion game_restart() ), werden alle persistenten Objekte entfernt und existieren nur noch, wenn sie vom Spiel neu erstellt werden. Beachten Sie auch, dass, wenn Sie ein persistentes Objekt deaktivieren, es nicht mehr von einem Raum in einen anderen gelangen kann, es sei denn, es wird vor dem Auslösen des Raumende-Ereignisses wieder aktiviert.

Syntax:

persistent

Rückgabe:

Beispiel:

if !persistent
{
    persistent = true;
}

Die obige code prüft die Instanz, um zu sehen, ob die Persistenz als false gekennzeichnet ist, und wenn die Instanz NICHT persistent ist, wird das Persistenzkennzeichen auf true gesetzt.
