Ebene

Diese eingebaute Variable wird für jede Instanz in einem Raum erstellt und enthält den Layer-ID-Wert des Layers, dem die Instanz zugeordnet ist. Dieser Wert kann dann in anderen Funktionen verwendet werden wie layer_get_depth() verwendet werden oder er kann geändert werden, um die Instanz in eine andere Ebene zu verschieben, aber beachte, dass du einen Fehler erhältst, der dein Spiel zum Schließen zwingt, wenn die zugewiesene Ebene im aktuellen Raum nicht existiert. Wenn du eine Ebene zuweist, musst du die eindeutige Ebenen-ID angeben, die von der Funktion layer_get_id() zurückgegeben wird (wenn du benannte Raumebenen verwendest), oder wie sie von der Funktion layer_create() (wenn du deine eigenen Ebenen zur Laufzeit erstellst).

WICHTIG Wenn Sie die Instanz mit der Funktion instance_create_depth() Funktion erstellt oder die Variable depth Variable manuell geändert haben, wird die der Instanz zugewiesene Ebene zu einer "verwalteten" Ebene, d. h. einer Ebene, die von GameMaker automatisch gesteuert und verwaltet wird. In diesen Fällen gibt die Variable layer den Wert -1 zurück.

Syntax:

layer;

Rückgabe:

Layer ID

Beispiel:

layer = layer_create(-1000);

Die obige code erstellt eine neue Ebene mit einer Tiefe von -1000 und setzt dann die Variable layer auf die zurückgegebene Ebenen-ID, wodurch die Instanz von der Ebene, auf der sie sich gerade befindet, auf die neu erstellte Ebene verschoben wird.
