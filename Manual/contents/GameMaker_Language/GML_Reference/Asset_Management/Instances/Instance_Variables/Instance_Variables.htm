Instanz-Variablen

Wenn Sie ein neues Objekt erstellen, werden bestimmte Variablen bereits mit Standardwerten initialisiert. Diese Variablen nennen wir Instanzvariablen, da sie für jede Instanz, die Sie von dem Objekt in Ihren Spielräumen platzieren, erstellt werden, und sobald eine Instanz erstellt ist, sind die Werte dieser Variablen eindeutig für diese Instanz und nur für diese Instanz. Einige dieser Variablen werden Sie häufig verwenden, wie z. B. x und y, während andere weniger benötigt werden, je nachdem, was Sie tun möchten, aber im Allgemeinen sind sie sehr nützlich und wenn möglich sollten Sie sie verwenden, anstatt Ihre eigenen benutzerdefinierten Variablen zu erstellen.

Die meisten Instanzvariablen können nicht nur gelesen, sondern auch gesetzt werden, so dass du die Eigenschaften und das Verhalten einer Instanz ändern kannst, indem du einfach den Wert einer bestimmten Variable veränderst - du kannst zum Beispiel verhindern, dass eine Instanz gezeichnet wird, indem du einfach die eingebaute Variable visible auf false setzt. Im Folgenden findest du die verschiedenen Variablen, die für alle Instanzen aller Objekte in deinem Spiel initialisiert werden.

WICHTIG Wenn sich eine Instanz in einem sequence befindet, werden einige dieser Variablen - wie image_xscale / image_yscale / image_angle / x / y - überschrieben, wenn sequence jeden Schritt nach dem Start der Wiedergabe aktualisiert.

Diese Variablen befassen sich mit allgemeinen Instanzeigenschaften:

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

Diese Variablen beziehen sich auf die Position und Bewegung der Instanz:

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

Objekt-Eigenschaften

Die folgende Variable enthält die eindeutige Index-ID des Objekts, aus dem die Instanz erstellt wurde:

  • object_index

Diese Variablen beziehen sich alle auf die sprite, die der Instanz zugewiesen ist, und können verwendet werden, um zu ändern, was und wie gezeichnet wird:

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

Diese Variablen beziehen sich auf die Kollisionsmaske:

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Diese Variablen beziehen sich auf Pfade und darauf, wie die Instanz mit einem Pfad interagiert, falls er zugewiesen wurde:

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

Diese Variablen dienen dazu, eine Instanz so einzustellen, dass sie eine Zeitleiste verwendet:

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

Sequenzen

Die folgenden Variablen speichern Informationen über die Sequence, die die Instanz zu einem bestimmten Zeitpunkt steuern können:

  • in_sequence
  • sequence_instance

Für die Physikfunktionen des GameMaker gibt es eine Vielzahl von eingebauten Variablen, die der Übersichtlichkeit halber in dem Teil des Handbuchs zu finden sind, der alles rund um die Physiksimulation behandelt:

  • Physik
