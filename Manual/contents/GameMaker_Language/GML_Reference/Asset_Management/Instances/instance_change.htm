Instanz_Änderung

Sie können diese Funktion verwenden, um eine Instanz eines Objekts in eine andere Instanz eines anderen Objekts zu verwandeln und dabei zu entscheiden, ob die Ereignisse "Zerstören" und "Aufräumen" der ursprünglichen Instanz und das Ereignis "Erstellen" der neuen Instanz ausgeführt werden sollen. Auf diese Weise kannst du zum Beispiel eine Bombe in eine Explosion verwandeln - in diesem Fall wäre "perf" argument wahrscheinlich wahr, da du möchtest, dass die Bombe ihr Zerstörungs- und Aufräum-Ereignis sowie die Explosion ihr Erstellungs-Ereignis ausführt - oder du könntest deinen Spielercharakter in einen anderen verwandeln - in diesem Fall wäre "perf" argument wahrscheinlich falsch, da du nicht möchtest, dass die Instanzen ihre Erstellungs- und Zerstörungs-/Aufräum-Ereignisse ausführen.

Es ist erwähnenswert, dass das Ändern der Instanz bedeutet, dass Sie bis zum nächsten Schritt keine weiteren Aktionen mit dieser Instanz durchführen sollten, insbesondere wenn Sie versuchen, auf Variablen usw. zuzugreifen, da dies einen Fehler verursacht. Im Grunde genommen ändern Sie die Instanz, aber sie ist bis zum Ende des aktuellen Schritts nicht wirklich verfügbar, so dass ein direkter Zugriff auf die darin enthaltenen Variablen (z. B. durch den Aufruf von obj_Changed.x) nicht funktioniert.

WARNUNG! Wenn Sie eine physikaktivierte Instanz ändern, werden die physischen Eigenschaften nicht auf die neue Instanz übertragen, in die sie geändert wird. Daher sollten Sie code verwenden, um den physikalischen Zustand der aktuellen Instanz auf die neue Instanz zu übertragen, oder Sie sollten die physikalischen Eigenschaften der neuen Instanz in ihrem Erstellungsereignis oder im Objekteditor definieren. Aus diesem Grund wird empfohlen, dass du diese Funktion nicht mit physikfähigen Instanzen verwendest, sondern eine Kombination aus instance_destroy() und instance_create_layer()zu verwenden.

Syntax:

instance_change(obj, perf);

Argument Typ Beschreibung
obj Objekt Anlage The new object the calling object will change into.
perf Boolesche Whether to perform that new object's Create and Destroy events (true) or not (false).

Rückgabe:

N/A

Beispiel:

if keyboard_check(vk_enter)
{
    instance_change(obj_Player_Swimming, false);
    exit;
}

Die obige code wird die aufrufende Instanz in eine Instanz von "obj_Player_Swimming" ändern, ohne das ursprüngliche Instanzzerstörungsereignis noch das neue Instanzerstellungsereignis auszuführen, wenn der Benutzer die "Enter"-Taste drückt, und den code -Block verlassen, wenn er fertig ist.
