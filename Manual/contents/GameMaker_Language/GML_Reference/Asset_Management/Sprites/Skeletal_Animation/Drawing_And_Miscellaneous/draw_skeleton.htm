zeichnen_skelett

Diese Funktion ist nur für sprites geeignet, die mit einem Skelettanimationsprogramm wie Spine erstellt wurden. Sie können diese sprites zwar mit den normalen Zeichenfunktionen (wie draw_self()), aber es kann vorkommen, dass Sie ein einzelnes Bild zeichnen oder eine Änderung der Hauttextur zeigen möchten, ohne die Eigenschaften von sprite zu ändern. In diesen Fällen können Sie sprite mit dieser Funktion zeichnen, bei der Sie den Index sprite und dann den Namen der Animation angeben, von der Sie das Einzelbild abrufen wollen (eine Zeichenkette, wie sie in dem Programm festgelegt wurde, mit dem Sie sprite erstellt haben). Als Nächstes gibst du den Namen des zu verwendenden Skins an (ebenfalls wie bei der Erstellung von sprite festgelegt) und einen Frame-Wert. Mit dem Frame-Wert teilt man GameMaker mit, welcher Teil der Animation gezeigt werden soll, und er ist ein Wert zwischen 0 und dem image_number. Der Rest von arguments ist derselbe wie beim normalen Zeichnen von sprite und ändert die Skalierung, den Winkel, die Überblendungsfarbe und den Alphawert des endgültigen Animationsrahmens, der gezeichnet wird.

WARNUNG! Aufgrund der Art und Weise, wie Skelettanimationen zwischen verschiedenen "Schlüsselbildern" interpoliert werden, ist mit diesem Aufruf ein zusätzlicher Overhead verbunden. Daher wird empfohlen, das Zeichnen von skelettbasierten Animationen sprites mit dieser Methode zu vermeiden, wenn es nicht unbedingt notwendig ist.

Syntax:

draw_skeleton(sprite, animname, skinname, frame, x, y, xscale, yscale, rot, colour, alpha);

Argument Typ Beschreibung
sprite Sprite-Asset The index of the sprite to draw.
animname Zeichenfolge The name of the animation to get the frame from (a string).
skinname Zeichenfolge The name of the skin to use (a string).
frame Real The animation frame to draw (from 0 to image_number - 1).
x Real The x coordinate of where to draw the sprite.
y Real The y coordinate of where to draw the sprite.
xscale Real The horizontal scaling of the sprite, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the sprite, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the sprite. 0=normal, 90=turned 90 degrees counter-clockwise etc.
colour Farbe The colour with which to blend the sprite.
alpha Real The alpha of the sprite (from 0 to 1 where 0 is transparent and 1 opaque).

Rückgabe:

N/A

Beispiel:

draw_skeleton(spr_Jump, "jump", "skin1", 0, x, y, image_xscale, image_yscale, image_angle, c_white, 0.5);

Die obige code zeichnet das erste Bild der gegebenen Skelettanimation sprite mit dem Animations-Set "jump", dem als "skin1" definierten Skin und in der gleichen Skalierung und Position wie in den Instanz-Eigenschaften definiert, aber mit einem Alpha-Wert von 0,5.
