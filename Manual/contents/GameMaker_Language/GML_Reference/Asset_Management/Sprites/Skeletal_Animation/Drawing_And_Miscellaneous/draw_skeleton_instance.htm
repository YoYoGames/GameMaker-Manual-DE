zeichnen_skelett_instanz

Diese Funktion ist nur für Instanzen gedacht, denen sprites zugewiesen ist, die mit dem Skelettanimationsprogramm Spine erstellt wurden. Sie können diese sprites zwar mit den normalen Zeichenfunktionen (wie draw_self()), aber es kann vorkommen, dass du ein einzelnes Bild zeichnen oder eine Änderung der Hauttextur zeigen möchtest, ohne die sprite Eigenschaften für eine Instanz zu ändern. In diesen Fällen kannst du die sprite, die der Instanz zugewiesen wurde, mit dieser Funktion zeichnen, wobei du die eindeutige Instanz-ID und dann den Namen der Animation angibst, von der du das Bild erhalten möchtest (eine Zeichenkette, wie sie in dem Programm festgelegt wurde, mit dem du die sprite erstellst). Als Nächstes gibst du den Namen des zu verwendenden Skins an (wiederum wie bei der Erstellung von sprite festgelegt) und einen Frame-Wert. Der Frame-Wert teilt GameMaker mit, welcher Teil der Animation gezeigt werden soll, und ist ein Wert zwischen 0 und dem image_number. Der Rest von arguments ist derselbe wie beim normalen Zeichnen von sprite und ändert die Skalierung, den Winkel, die Überblendungsfarbe und den Alphawert des endgültigen Animationsrahmens, der gezeichnet wird.

WARNUNG! Aufgrund der Art und Weise, wie Skelettanimationen zwischen verschiedenen "Schlüsselbildern" interpoliert werden, ist mit diesem Aufruf ein zusätzlicher Overhead verbunden. Daher wird empfohlen, das Zeichnen von skelettbasierten Animationen sprites mit dieser Methode zu vermeiden, wenn es nicht unbedingt notwendig ist.

Syntax:

draw_skeleton_instance(instance, animname, skinname, frame, x, y, xscale, yscale, rot, colour, alpha);

Argument Typ Beschreibung
instance Instanz-ID The index of the instance to draw the sprite from.
animname Zeichenfolge The name of the animation to get the frame from (a string).
skinname Zeichenfolge The name of the skin to use (a string).
frame Real The animation frame to draw (from 0 to image_number - 1).
x Real The x coordinate of where to draw the sprite.
y Real The y coordinate of where to draw the sprite.
xscale Real The horizontal scaling of the sprite, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the sprite, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the sprite. 0=normal, 90=turned 90 degrees counter-clockwise etc.
colour Farbe The colour with which to blend the sprite.
alpha Real The alpha of the sprite (from 0 to 1 where 0 is transparent and 1 opaque).

Rückgabe:

N/A

Beispiel:

draw_skeleton_instance(obj_Player, "jump", "skin1", 0, x, y, 1, 1, 0, c_white, 1);

Die obige code zeichnet das erste Bild der Skelettanimation sprite, die der angegebenen Instanz zugewiesen ist, unter Verwendung des Animationssets "jump" und der als "skin1" definierten Haut.
