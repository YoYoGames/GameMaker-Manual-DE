skelett_schlitz_daten_instanz

Mit dieser Funktion können Sie eine (vorab erstellte) DS-Liste mit Daten für jeden der verfügbaren Anhangs-Slots für die Spine sprite, die mit der aktuellen Instanz verbunden ist, füllen (einschließlich aller Anhangsänderungen). Diese Daten werden in Form einer DS-Map zurückgegeben, die die folgenden Schlüssel/Wertpaare enthält:

  • "Name": Der Name des Steckplatzes.
  • "Knochen": Der Name des Knochens.
  • "Anhang": Der Name des Anhangs, wenn einer verwendet wird, oder "(none)", wenn kein Anhang zugewiesen ist.

Die Werte für jeden Schlüssel sind Zeichenketten und können dann in den anderen Skelettanhangsfunktionen für diese Arten von sprite verwendet werden. Beachten Sie, dass die erstellten DS Map nicht zerstört werden, so dass Sie loop durch die DS-Liste gehen und jede der erstellten Maps selbst zerstören müssen.

Syntax:

skeleton_slot_data_instance(list);

Argument Typ Beschreibung
list DS-Liste ID The ID of the DS list to populate with the DS maps.

Rückgabe:

N/A

Beispiel:

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data_instance(list);
for (var i = 0; i < ds_list_size(list); i++)
{
    var map = list[| i];
    if open
    {
        if map[? "attachment"] == "(none)"
        {
            open = false;
            slot_name = map[? "name"];
        }
    }

    ds_map_destroy(map);
}

ds_list_destroy(list);

Die obige code erstellt eine DS-Liste und füllt sie dann mit den Slot-Daten für die Instanz sprite. Diese Daten werden dann geparst, um die einzelnen DS-Maps mit den Slot-Daten zu extrahieren. Anschließend wird geprüft, ob ein leerer Slot vorhanden ist, und wenn ja, wird der Variable "slot_name" der leere Slotname zugewiesen, bevor die DS-Map zerstört wird. Schließlich wird die DS-Liste zerstört, da sie nicht mehr benötigt wird.
