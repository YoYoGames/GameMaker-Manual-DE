skeleton_attachment_create_colour

Wenn Sie Ihre Skelett-Animation sprite erstellen, können Sie Anhänge-Slots und Anhänge zuweisen, die in sie hineinpassen. Dabei handelt es sich einfach um Bilder (sprites), die von der Animation getrennt sind, sich aber zusammen mit dem Bone bewegen, an dem sie angehängt sind. Normalerweise würden Sie diese Anhänge in Ihrem Animationsprogramm (Spine) zuweisen, aber mit dieser Funktion können Sie Ihre eigenen zur Laufzeit erstellen, indem Sie ein sprite -Asset aus Ihrem Spiel verwenden. Die Funktion erfordert, dass Sie dem Anhang einen Namen (als String) geben und dann die sprite_index und image_index sowie den x- und y-Ursprung (der sich von dem auf sprite in den sprite -Eigenschaften definierten unterscheiden kann). Außerdem können Sie alle Transformationen festlegen, die auf das Bild angewendet werden sollen, wenn es angehängt wird, sowie die Farbe, die mit dem Bild gemischt werden soll, und seinen Alpha-Wert (Transparenz). Wenn der Anhang erfolgreich erstellt wurde, gibt die Funktion 1 zurück, und wenn nicht (Sie haben einen ungültigen sprite Index angegeben, oder die Basis sprite ist keine Spine sprite), dann gibt sie -1 zurück.

Es ist erwähnenswert, dass die Instanz sprite eine Mischfarbe und eine Alpha-Einstellung haben kann (image_blend und image_alpha) haben kann, ebenso wie der Attachment-Slot (siehe die Funktion skeleton_slot_colour_set()), so dass die endgültige Farbe und der Alphawert, die das zugewiesene Attachment sprite für den Slot haben wird, eine Mischung aus all diesen Werten sein wird.

Syntax:

skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha);

Argument Typ Beschreibung
name Zeichenfolge The name (as a string) of the attachment to create.
sprite Sprite-Asset The sprite_index to get the attachment image from.
ind Real The image_index to get the attachment image from.
xorigin Real The x origin for the image being used.
yorigin Real The y origin for the image being used.
xscale Real The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the image: 0 = normal, 90 = turned 90° counter-clockwise etc.
Colour Farbe The colour value to use (A constant, integer or hex value).
Alpha Real The alpha value to use (from 0 to 1).

Rückgabe:

Real (1 if successful, -1 if not)

Beispiel:

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Die obige code würde den aktuell zugewiesenen Namen der Anlage für den Steckplatz mit dem Namen "slot_leftHand" überprüfen, und wenn eine leere Zeichenkette zurückgegeben wird, wird eine neue sprite angelegt.
