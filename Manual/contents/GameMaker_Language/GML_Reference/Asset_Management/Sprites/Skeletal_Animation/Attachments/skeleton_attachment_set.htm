skelett_anhaengung_set

Einer Skelettanimation sprite können andere Bilder als Anhänge hinzugefügt werden, wobei diese Bilder zu einem benannten Slot hinzugefügt werden (der Name wird beim Erstellen des Anhangs-Slots in Ihrem Animationsprogramm angegeben) und sie werden zusammen mit der Animation der aktuellen sprite gezeichnet. Mit dieser Funktion können Sie ein Attachment auf einen bestimmten Slot setzen, wobei Sie die Namen (als Strings) des Slots und des Attachments angeben müssen. Diese Namen werden durch das verwendete Animationsprogramm definiert, oder (im Falle des Attachments) durch den Aufruf von skeleton_attachment_create().

Beachten Sie, dass Sie auch einen sprite_index als Anhang übergeben können, und dieser sprite wird verwendet, oder Sie können -1 verwenden, um den Anhang aus dem Slot zu entfernen. Wenn Sie einen sprite -Index als argument übergeben, wird ein Anhangs-Slot erstellt, der den Namen des sprite als Zeichenkette für den Namen des Slots verwendet (wenn Sie also z. B. spr_sword verwenden, wird ein Anhangs-Slot "spr_sword" erstellt), und der Slot verwendet den ersten Bildindex (0) des sprite sowie seinen x- und y-Ursprungsoffset mit einer Skalierung von (1,1) und einer Drehung von 0.

Syntax:

skeleton_attachment_set(slot, attachment);

Argument Typ Beschreibung
slot Zeichenfolge The slot name (a string) to get the attachment of.
attachment String oder Sprite Asset The name (as a string or a sprite_index) of the attachment image.

Rückgabe:

String

Beispiel:

if skeleton_attachment_get("slot_leftHand") == ""
{
    skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

Die obige code würde den aktuell zugewiesenen Namen der Anlage für den Steckplatz mit dem Namen "slot_leftHand" überprüfen, und wenn eine leere Zeichenkette zurückgegeben wird, wird eine neue sprite angelegt.
