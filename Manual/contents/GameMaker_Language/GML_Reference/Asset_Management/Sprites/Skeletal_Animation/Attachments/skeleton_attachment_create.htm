skelett_attachment_create

Wenn Sie Ihre Skelett-Animation sprite erstellen, können Sie Anhänge-Slots und Anhänge zuweisen, die in sie hineinpassen. Dabei handelt es sich einfach um Bilder, die nicht Teil der Animation sind, sich aber zusammen mit dem Bone bewegen, an dem sie angehängt sind. Normalerweise würden Sie diese Anhänge in Ihrem Animationsprogramm zuweisen (Spine), aber mit dieser Funktion können Sie Ihre eigenen zur Laufzeit erstellen, indem Sie ein sprite Asset aus Ihrem Spiel verwenden. Die Funktion erfordert, dass Sie dem Anhang einen Namen geben (als String) und dann den zu verwendenden sprite_index und image_index festlegen sowie den x- und y-Ursprung (der sich von dem durch sprite in den sprite Eigenschaften definierten unterscheiden kann), und Sie können auch alle Transformationen festlegen, die Sie auf das Bild anwenden möchten, wenn es angehängt wird. Wenn der Anhang erfolgreich erstellt wurde, gibt die Funktion 1 zurück, und wenn nicht (Sie haben einen ungültigen sprite Index angegeben, oder die Basis sprite ist keine Spine sprite), dann gibt sie -1 zurück.

Syntax:

skeleton_attachment_create(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot);

Argument Typ Beschreibung
name Zeichenfolge The name (as a string) of the attachment to create.
sprite Sprite-Asset The sprite_index to get the attachment image from.
ind Real The image_index to get the attachment image from.
xorigin Real The x origin for the image being used.
yorigin Real The y origin for the image being used.
xscale Real The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale Real The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot Real The rotation of the image. 0=normal, 90=turned 90 degrees counter-clockwise etc.

Rückgabe:

Real (1 if successful, -1 if not)

Beispiel:

skeleton_attachment_create("sword", spr_Weapons, 0, 0, 80, 1, 1, 90);
skeleton_attachment_create("knife", spr_Weapons, 1, 0, 45, 1, 1, 90);
skeleton_attachment_create("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Die obige code würde den aktuell zugewiesenen Namen der Anlage für den Steckplatz mit dem Namen "slot_leftHand" überprüfen, und wenn eine leere Zeichenkette zurückgegeben wird, wird eine neue sprite angelegt.
