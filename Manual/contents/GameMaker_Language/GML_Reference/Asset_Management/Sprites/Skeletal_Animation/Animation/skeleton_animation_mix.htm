skelett_animation_mix

Sie können switch Animationsgruppen einfach mit der skeleton_animation_set() Funktion verwenden, aber dies kann zu einem Überspringen oder Stottern führen, wenn eine Animation durch eine andere ersetzt wird. Um dies zu verhindern, können Sie den Mix-Wert zwischen zwei Animations-Sets einstellen und sprite wird zwischen ihnen interpolieren. Normalerweise solltest du dies im Erstellungsereignis der Instanz mit der Skelettanimation tun, da der Wert nur einmal gesetzt werden muss und GameMaker alle weiteren Änderungen an der sprite mit den Animationssets in dieser Instanz interpoliert. Beachten Sie, dass der Wert für die Dauer von 0 bis 1 reicht, wobei ein Wert von 0,5 eine "halb und halb"-Interpolation von einem Set zum anderen bedeuten würde.

HINWEIS Spine 4.0 ändert die Funktionsweise der Rotationsanimation, da es die Rotation nicht mehr auf dem kürzesten Weg interpoliert, sondern die absoluten Werte in Ihren Keyframes für die Interpolation verwendet. Dies kann zu unerwartetem Verhalten führen, wenn Sie Ihre Spine Animation von einer älteren Version aktualisieren, und es können Änderungen in Ihrem Spine Projekt erforderlich sein, damit sich Rotationen wie gewünscht verhalten.

Syntax:

skeleton_animation_mix(animfrom, animto, duration);

Argument Typ Beschreibung
animfrom Zeichenfolge The name (a string) of the first animation set to interpolate from.
animto Zeichenfolge The name (a string) of the second animation set to interpolate to.
duration Real The duration of the interpolation (from 0 to 1)

Rückgabe:

N/A

Beispiel:

skeleton_animation_set("walk");
skeleton_animation_mix("walk", "jump", 0.2);
skeleton_animation_mix("jump", "walk", 0.4);

Die obige code würde in das Create Event einer Instanz mit einer Skelettanimation sprite gehen und setzt die Animationsmischdauer für die Interpolation zwischen den beiden Animations-Sets "walk" und "jump".
