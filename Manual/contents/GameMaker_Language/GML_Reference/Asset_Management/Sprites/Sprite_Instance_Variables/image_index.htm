bild_index

Ein sprite besteht aus einem oder mehreren Unterbildern, die das sprite animiert erscheinen lassen können, wenn sie switch von einem zum anderen wechseln, oder zwischen denen in code umgeschaltet werden kann, um verschiedene "Zustände" zu erzeugen, ähnlich wie eine Schaltfläche in Fenstern. Wenn die sprite animiert ist, kannst du das aktuelle Bild der Animation abrufen, indem du die Variable image_index überprüfst, oder wenn du den Zustand eines statischen sprite ändern willst, kannst du ein neues Teilbild auswählen, indem du diese Variable auf das gewünschte Teilbild der sprite setzt. Bitte beachte, dass Änderungen in dieser Variable nur dann sichtbar sind, wenn die Instanz entweder kein Draw-Ereignis hat (und GameMaker daher standardmäßig die sprite zeichnet) oder mit einer der Zeichenfunktionen wie draw_self() oder draw_sprite_ext() gezeichnet werden (indem man den image_index in die entsprechende argument eingibt).

Beachten Sie, dass das direkte Setzen dieser Variable auf einen Frame keine Broadcast-Nachrichten auslöst, die möglicherweise auf diesem Frame vorhanden sind.

Beachten Sie auch, dass Sie bei der Verwendung der Skelettanimation sprites immer noch die Werte von image_index abrufen und einstellen können - siehe die Funktion skeleton_animation_get_duration() für Beispiele, wie man das macht.

Syntax:

image_index;

Rückgabe:

(single precision floating point value)

Beispiel:

if (image_speed > 0)
{
    if (image_index >= image_number - 1) instance_destroy();
}

Die obige code prüft, ob die sprite animiert ist, und wenn ja, dann prüft sie, ob der aktuelle image_index auf dem letzten Frame ist und zerstört in diesem Fall die Instanz.
