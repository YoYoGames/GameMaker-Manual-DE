sprite_set_cache_size

Bei der Bildüberblendung kann HTML5 nicht dynamisch vorgehen, wie es bei einer ausführbaren Datei möglich wäre. Daher muss GameMaker eine überblendete Kopie des Bildes vorübergehend speichern und dann laden. Diese Funktion legt fest, wie viele überblendete Kopien des angegebenen sprite zwischengespeichert werden können, bevor die alten überschrieben werden. Die Voreinstellung ist 4. Dies gilt für alle Unterbilder des sprite.

Syntax:

sprite_set_cache_size(ind, max);

ArgumentTyp Beschreibung
ind The index of the sprite to change the cache size of.
max The maximum number of cached copies of the sprite that can be stored.

Rückgabe

Beispiel:

sprite_set_cache_size(sprite0, 2);

Dies setzt die sprite cache aller Sub-Images von sprite0 auf 2 Kopien.
