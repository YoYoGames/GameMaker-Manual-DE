sprite_replace

Diese Funktion funktioniert fast genau so wie sprite_add()nur dass sie nicht den Index der zu importierenden sprite zurückgibt, sondern einen zuvor erstellten Index sprite überschreibt. Wenn Sie diese Funktion verwenden, sollten Sie einen sprite Index verwenden, der mit anderen Funktionen wie sprite_add() oder sprite_create_from_surface()oder sogar sprite_duplicate()erstellt und in einer Variablen gespeichert wurde, und nicht einen sprite Ressourcenbaum. Sie können ein sprite aus den Spiel-Assets mit dieser Funktion ersetzen, aber das bedeutet, dass Sie die Referenz-ID für das sprite, das Sie ersetzen, verlieren, unabhängig davon, ob Sie game_restart() aufrufen oder nicht, und wird daher nicht empfohlen. Unabhängig von der sprite, die ersetzt wird, erstellt diese Funktion eine neue Texturseite für das Sprite. Daher ist bei der Verwendung dieser Funktion Vorsicht geboten, da sie sich negativ auf die Leistung auswirken kann, indem sie die Anzahl der für das Rendern erforderlichen Texturwechsel erhöht.

Die zu ladende Bilddatei sollte immer das Format *.png haben, und alle Bilder, die in animierte sprites umgewandelt werden sollen, sollten ein "Streifen"-Format haben (siehe das Bild unten). Sie werden in die angegebene Anzahl von Unterbildern aufgeteilt, wobei die Regel sprite width = strip width / sub images gilt.

Wie Sie im obigen Bild sehen können, wurde sprite auf einem dunkelvioletten Hintergrund platziert, und dieser kann entfernt werden, indem Sie "removeback" argument auf true setzen. Dies funktioniert, indem Sie das untere linke Pixel des sprite auf die dortige Farbe überprüfen und diese dann als die zu entfernende Farbe verwenden. Wäre zum Beispiel im obigen Bild die Farbe des unteren linken Pixels grün, würden alle grünen Teile von sprite entfernt und der Rest des lila Hintergrunds ignoriert.

Wenn Sie die Option "Hintergrund entfernen" wählen, möchten Sie vielleicht auch, dass GameMaker die Ränder von sprite glättet, indem Sie die Option "Glatt" argument auf true setzen. Dadurch wird lediglich ein halbtransparenter Rand um die Kanten von sprite erzeugt, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für sprite angeben. Dies ist der Punkt, an dem sprite auf der Instanz, die es verwendet, "fixiert" wird, und wird immer relativ zur linken oberen Ecke (0,0) eines sprite -Unterbildes berechnet. Bei einem sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, liegt der Ursprung in der Mitte.

Standardmäßig werden die Begrenzungsrahmen für alle neuen sprites automatisch berechnet (der genaue Begrenzungsrahmen hängt von der Größe und der Transparenz der sprite ab), Sie können dies jedoch anpassen. sprite_collision_mask().

HINWEIS: Je nach gewähltem Ziel platform können Sie Dateien nur in begrenztem Umfang speichern und laden. Weitere Informationen finden Sie unter Dateiverwaltung.

HINWEIS: Wenn Sie diese Funktion in Ihrem HTML5-Zielspiel verwenden, um Ressourcen von einem externen server zu laden, sollten Sie sich darüber im Klaren sein, dass aufgrund des XSS-Schutzes in Browsern Versuche, Ressourcen von anderen Domains zu laden, blockiert werden können und möglicherweise leere Ergebnisse liefern.

Syntax:

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

Argument Typ Beschreibung
ind Sprite-Asset The index of the sprite to permanently replace.
fname Zeichenfolge The filename of the image to make the new sprite.
imgnumb Real The number of frames the sprite will be cut up into horizontally. Use 1 for one single image or *.gif.
removeback Boolesche Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Boolesche Indicates whether to smooth the edges.
xorig Real The x coordinate of the origin, relative to the sprite's top left corner.
yorig Real The y coordinate of the origin, relative to the sprite's top left corner.

Rückgabe

N/A

Beispiel:

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

Die obige code ersetzt das in "spr_banner" indizierte Bild-Asset durch ein anderes, das aus einer externen Quelle geladen wurde.
