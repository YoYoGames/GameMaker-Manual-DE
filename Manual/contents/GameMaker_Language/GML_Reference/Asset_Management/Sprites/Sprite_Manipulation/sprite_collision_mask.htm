sprite_collision_mask

Mit dieser Funktion können Sie die Eigenschaften der Kollisionsmaske festlegen, die eine sprite haben soll. Wenn Sie als Bounding-Box-Modus entweder Automatik (0) oder Vollbild (1) wählen, können die einzelnen Bounding-Box-Werte auf 0 gesetzt werden. Bei einer benutzerdefinierten Maske (2) müssen Sie diese Werte jedoch selbst festlegen. Die verschiedenen Bounding-Box-Werte beziehen sich immer auf die linke obere Ecke von sprite (unabhängig vom x- und y-Ursprung), die als Position (0, 0) gilt.

Beachten Sie jedoch, dass jede andere als rechteckige Maske bei der Auflösung von Kollisionen mehr Rechenleistung erfordert, was zu einem Leistungsabfall führt. Im Allgemeinen sollten Sie andere Maskentypen als rechteckige nur dann verwenden, wenn es unbedingt notwendig ist.

HINWEIS: Diese Funktion erlaubt nicht die Art der gedrehten Rechteck-Kollisionsmaske.

Die Art der Maske, die eingestellt werden kann, ist eine der folgenden Konstanten:

Bounding Box Art (Form) Konstante
Konstante Beschreibung
bboxkind_rectangular Ein rechteckiges (nicht rotierendes) Rechteck als Kollisionsmaske
bboxkind_ellipse Eine elliptische Form der Kollisionsmaske
bboxkind_diamond Eine rautenförmige Kollisionsmaske
bboxkind_precise Eine präzise Kollisionsmaske, wobei sich die Maske an die nicht-transparenten Pixel der sprite anpasst, basierend auf dem angegebenen Toleranzwert (siehe unten).


Mit der Toleranz schließlich wird festgelegt, wie genau die Maske ist (bei einer "Vollbild"-Maske hat dies keine Auswirkung), wobei eine Toleranz von 0 bedeutet, dass die Maske jedem einzelnen Pixel folgt, dessen Transparenz über 0 liegt, während andere Werte den Umfang der Kollisionsmaske je nach Transparenz der Pixel verschieben.

HINWEIS: Diese Funktion ist nur für Bitmaps nützlich sprites und funktioniert nicht mit SWF oder JSON (Spine) sprites.

HINWEIS: Diese Funktion funktioniert nur bei hinzugefügten sprites oder duplizierten sprites und nicht direkt bei vorgefertigten Ressourcen. Sie können sprites duplizieren, indem Sie die Funktion sprite_duplicate().

Syntax:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argument Typ Beschreibung
ind Sprite-Asset The index of the sprite to set the bounding box of.
sepmasks Boolesche Whether to create collision masks for each sub-image of the sprite (true), oder eine Maske für alle (false).
bboxmode Real What kind of bounding box to use. 0 = automatic, 1 = full image, 2 = user defined.
bbleft Real The maximum left position of the bounding box.
bbtop Real The maximum top position of the bounding box.
bbright Real The maximum right position of the bounding box.
bbbottom Real The maximum bottom position of the bounding box.
kind Bounding Box Art (Form) Konstante The kind of mask, a constant (see the table in the description).
tolerance Real Indicates the tolerance in the transparency value (0=no tolerance, 255=full tolerance).

Rückgabe

N/A

Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

Die obige code lädt ein sprite aus einer externen Quelle und speichert den neuen Index in der Variablen "spr". Die code stellt dann die neue sprite so ein, dass sie für jedes ihrer Teilbilder eine genaue Kollisionsmaske hat.
