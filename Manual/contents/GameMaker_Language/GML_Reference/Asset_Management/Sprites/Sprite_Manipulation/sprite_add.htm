sprite_add

Mit dieser Funktion können Sie ein Bild als sprite hinzufügen, indem Sie es aus einer externen Quelle laden, wobei die zu ladende Bilddatei immer entweder im Format *.png, *.gif, *.jpg/jpeg oder *.json vorliegen sollte (*json Dateien werden zum Laden von Skelettanimations-Sprites verwendet, die mit Spine erstellt wurden ). Die Funktion gibt den neuen Index sprite zurück, der dann in jedem weiteren Code verwendet werden muss, der sich auf sprite bezieht. Wenn Sie diese Funktion mit HTML5 verwenden oder ein Bild von einer URL abrufen, erzeugt diese Funktion auch ein asynchrones Ereignis Image Loaded. Auf dieser Seite finden Sie weitere Informationen.

Alle Bilder, die in animierte sprites umgewandelt werden sollen - mit Ausnahme von *.gif (siehe unten) - sollten ein "Streifen"-Format haben (siehe das Bild unten) und werden in die angegebene Anzahl von Unterbildern aufgeteilt, wobei die Regel Sprite-Breite = Streifenbreite / Unterbilder gilt.

Wie Sie im obigen Bild sehen können, wurde sprite auf einem dunkelvioletten Hintergrund platziert, und dieser kann entfernt werden, indem Sie "removeback" argument auf true setzen. Dies funktioniert, indem Sie das untere linke Pixel des sprite auf die dortige Farbe überprüfen und diese dann als die zu entfernende Farbe verwenden. Wäre zum Beispiel im obigen Bild die Farbe des unteren linken Pixels grün, würden alle grünen Teile von sprite entfernt und der Rest des lila Hintergrunds ignoriert.

Wenn Sie die Option "Hintergrund entfernen" wählen, möchten Sie vielleicht auch, dass GameMaker die Ränder von sprite glättet, indem Sie die Option "Glatt" argument auf true setzen. Dadurch wird lediglich ein halbtransparenter Rand um die Kanten von sprite erzeugt, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für sprite angeben. Dies ist der Punkt, an dem sprite auf der Instanz, die es verwendet, "fixiert" wird, und wird immer relativ zur linken oberen Ecke (0,0) eines sprite -Unterbildes berechnet. Bei einem sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, liegt der Ursprung in der Mitte.

Wenn Sie ein Bild im Format *.gif hinzufügen, müssen Sie nur den Dateinamen und die Bildnummer angeben (die für das erste Bild 1 sein sollte) und können alle anderen arguments auf 0 belassen, wobei nur das erste Bild des GIF geladen wird. Bei *.json sprites sollte die Bildnummer 1 sein, und die Einstellungen "removeback" und "smooth" werden ignoriert.

Standardmäßig werden die Begrenzungsrahmen für alle neuen sprites automatisch berechnet (der genaue Begrenzungsrahmen hängt von der Größe und der Transparenz der sprite ab), Sie können dies jedoch anpassen. sprite_collision_mask().

Es sollte beachtet werden, dass der Speicherbedarf beim Hinzufügen einer sprite auf diese Weise größer ist, als Sie vielleicht erwarten. Das liegt daran, dass GameMaker die sprite als Texturseite speichert und sie auch im GPU-Speicher abgelegt wird, so dass der Gesamtspeicher größer ist als bei einer Bilddatei der gleichen sprite zu erwarten wäre.

HINWEIS: Wenn Sie eine sprite in GameMaker laden, müssen Sie daran denken, sie wieder zu entfernen (mit sprite_delete()) zu entfernen, wenn sie nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

HINWEIS: Unter iOS müssen Sie, wenn Sie eine eingebundene Datei aus einem Ordner laden, d. h. mit einem Pfad wie z. B. "Backgrounds/background1.png", müssen Sie den Ordner NICHT als Teil des Pfades angeben, sondern es reicht aus, den Dateinamen anzugeben. Unter allen anderen platforms müssen Sie den/die Ordner als Teil des Pfades zur Datei angeben.

HINWEIS: Wenn Sie bei HTML5 ein sprite von einem sicheren server laden, müssen Sie möglicherweise den Cross-Origin-Typ festlegen und einen relativen Pfad statt eines absoluten Pfads verwenden. Siehe http_set_request_crossorigin() für weitere Einzelheiten.

Syntax:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argument Typ Beschreibung
fname Zeichenfolge The name (a string file path) of the file to add.
imgnum Real Use to indicate the number of sub-images (1 for a single image or for a *.gif).
removeback Boolesche Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Boolesche Indicates whether to smooth the edges if transparent.
xorig Real Indicate the x position of the origin in the sprite.
yorig Real Indicate the y position of the origin in the sprite.

Rückgabe:

Sprite Asset

Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

Die obige code lädt eine sprite in das Spiel und speichert ihren Index in der Variablen "spr".
