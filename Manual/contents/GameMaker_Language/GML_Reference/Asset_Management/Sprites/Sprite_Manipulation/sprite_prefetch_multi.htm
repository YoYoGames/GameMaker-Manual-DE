sprite_prefetch_multi

Diese Funktion kann verwendet werden, um eine Anzahl von Textur-Seiten, die die angegebene sprites enthalten, im Voraus zu holen (in den Textur-Speicher zu laden). Sie liefern ein Array mit den sprite Indizes (wie bei der Erstellung des sprite Assets definiert) und die Texturseiten, auf denen sie sich befinden, werden in den Speicher geladen. Beachten Sie, dass die Funktion -1 zurückgibt, wenn Prefetch für die gewählte Ressource nicht unterstützt wird oder das Ziel platform HTML5 ist, oder sie gibt 0 zurück, wenn alles korrekt funktioniert hat.

HINWEIS: Da die Textur auf den meisten Geräten in den Texturspeicher geladen wird, kommt es zu Leistungseinbußen, daher wird empfohlen, dass Sie sprite_prefetch_multi() zu Beginn eines Levels aufzurufen, um ein Abwürgen zu vermeiden.

Syntax:

sprite_prefetch_multi

Argument Typ Beschreibung
array Array von Sprite-Assets Array with the sprite indices to fetch

Rückgabe:

Real (-1 or 0)

Beispiel:

spr_a[0] = spr_Player_Aura1;
spr_a[1] = spr_Player_Aura2;
spr_a[2] = spr_Player_Aura3;
spr_a[3] = spr_Player_Aura4;
sprite_prefetch_multi(spr_a);

Die obige code erstellt ein Array, in dem jedes Element einen sprite Index enthält. Dieses Array wird dann verwendet, um diese sprite Texturen im Speicher zu platzieren.
