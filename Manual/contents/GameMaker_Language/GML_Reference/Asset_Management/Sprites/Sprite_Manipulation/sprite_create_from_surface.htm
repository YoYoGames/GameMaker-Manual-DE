sprite_create_from_surface

Mit dieser Funktion können Sie eine sprite aus einer zuvor initialisierten Oberfläche erstellen (der Wert der Oberflächenindex-ID wird zurückgegeben, wenn Sie die Oberfläche mit surface_create()). Die x- und y-Koordinaten, die Sie dann in die Funktion eingeben, sollten sich auf die (0,0) Position der Oberfläche (die obere linke Ecke) beziehen und nicht auf das Spielfenster (oder die Ansicht, falls Sie eine aktiv haben). Die Breite und Höhe arguments sind in Pixeln angegeben und definieren die Breite und Höhe des zu verwendenden Teils der Oberfläche.

Wenn Sie die Option "removeback" argument auf true setzen, wird eine Farbe aus sprite entfernt, indem das untere linke Pixel von sprite auf die dortige Farbe überprüft wird und diese dann als zu entfernende Farbe verwendet wird.

Wenn Sie die Option "Hintergrund entfernen" wählen, möchten Sie vielleicht auch, dass GameMaker die Ränder von sprite glättet, indem Sie die Option "Glatt" argument auf true setzen. Dadurch wird lediglich ein halbtransparenter Rand um die Kanten von sprite erzeugt, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für sprite angeben. Dies ist der Punkt, an dem sprite auf der Instanz, die es verwendet, "fixiert" wird, und wird immer relativ zur linken oberen Ecke (0,0) eines sprite -Unterbildes berechnet. Bei einem sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, liegt der Ursprung in der Mitte.

Standardmäßig werden bei allen neuen sprites die Begrenzungsrahmen automatisch berechnet (der genaue Begrenzungsrahmen hängt von der Größe und der Transparenz der sprite ab), Sie können dies jedoch anpassen. sprite_collision_mask().

HINWEIS Wenn Sie eine sprite in GameMaker mit dieser Methode erstellen, müssen Sie daran denken, sie wieder zu entfernen (mit sprite_delete()), wenn sie nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Argument Typ Beschreibung
index Oberfläche ID The index of the surface to create from.
x Real The x position to copy from.
y Real The y position to copy from.
w Real The width of the area to be copied (from the x position).
h Real The height of the area to be copied (from the y position).
removeback Boolesche Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Boolesche Indicates whether to smooth the edges.
xorig Real Indicates the x position of the origin in the sprite.
yorig Real Indicates the y position of the origin in the sprite.

Rückgabe

Sprite Asset

Beispiel:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

Die obige code erstellt eine Oberfläche und speichert ihren Index in der lokalen Variablen "surf". Dann wird diese Oberfläche angepeilt, gelöscht und mehrere sprites übereinander gezeichnet. Schließlich wird ein neues sprite aus dem auf der Oberfläche gezeichneten zusammengesetzten Bild erstellt und sein Index wird "spr_Custom" zugewiesen, bevor der von der Oberfläche verwendete Speicher freigegeben wird.
