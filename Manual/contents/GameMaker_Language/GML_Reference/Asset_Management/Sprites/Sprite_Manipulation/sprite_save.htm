sprite_save

Mit dieser Funktion können Sie ein beliebiges Teilbild von sprite unter dem angegebenen Dateinamen auf der Festplatte speichern. Das sprite muss unter runtime hinzugefügt worden sein ( sprites kann nicht über IDE gespeichert werden) und die Datei muss mit der Erweiterung *.png gespeichert werden.

HINWEIS: Je nach gewähltem Ziel platform können Sie Dateien nur in begrenztem Umfang speichern und laden. Weitere Informationen finden Sie unter Dateiverwaltung.

Syntax:

sprite_save(ind, subimg, fname);

Argument Typ Beschreibung
ind Sprite-Asset The index of the sprite to save.
subimg Real The sub-image of the sprite to save.
fname Zeichenfolge The filename for the saved sprite.

Rückgabe

N/A

Beispiel:

var surf, spr_custom;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);
sprite_save(spr_custom, 0, "Player_Custom_Sprite.png");
sprite_delete(spr_Custom);

Die obige code erstellt eine Oberfläche und speichert ihren Index in der lokalen Variablen "surf". Dann wird diese Fläche angepeilt, gelöscht und mehrere sprites übereinander gezeichnet. Aus dem auf der Oberfläche gezeichneten zusammengesetzten Bild wird ein neues sprite erstellt, dessen Index der lokalen Variablen "spr_Custom" zugewiesen wird, bevor der von der Oberfläche verwendete Speicher freigegeben wird. Schließlich wird das resultierende sprite in einer Datei gespeichert und dann ebenfalls aus dem Speicher entfernt.
