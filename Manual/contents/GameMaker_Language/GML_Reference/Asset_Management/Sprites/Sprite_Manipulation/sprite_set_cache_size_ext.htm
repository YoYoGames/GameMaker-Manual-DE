sprite_set_cache_size_ext

Bei der Bildüberblendung kann HTML5 nicht dynamisch vorgehen, wie es bei einer ausführbaren Datei möglich wäre. Daher muss GameMaker eine überblendete Kopie des Bildes vorübergehend speichern und laden. Diese Funktion legt fest, wie viele überblendete Kopien des angegebenen sprite zwischengespeichert werden können, bevor alte überschrieben werden. Die Voreinstellung ist 4. Sie wird auf ein einzelnes Teilbild des sprite angewendet.

Syntax:

sprite_set_cache_size_ext(ind, index, max);

Argument Typ Beschreibung
ind Sprite-Asset The index of the sprite to change the cache size of.
index Real The sub-image of ind to change the cache size of.
max Real The maximum number of cached copies of the sprite that can be stored.

Rückgabe

N/A

Beispiel:

sprite_set_cache_size_ext(sprite0, 0, 2);

Dadurch wird die sprite cache des ersten Teilbildes von sprite0 auf 2 Kopien gesetzt.
