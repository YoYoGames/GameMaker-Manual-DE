sprite_get_speed

Mit dieser Funktion kann die Geschwindigkeit sprite abgerufen werden, wie sie für die Ressource sprite im Sprite-Editor definiert ist. Der zurückgegebene Wert kann dann z.B. verwendet werden, um zu berechnen, wie viele Frames für verschiedene Sprites gezeichnet werden können image_speed. Beachten Sie, dass der Rückgabewert sehr unterschiedlich sein wird, je nachdem, welche Art von Geschwindigkeit die im Sprite Editor angewendet wurde, entweder Frames pro Sekunde oder Frames pro Spielframe. Die folgenden zwei Beispiele verdeutlichen dies:

  • Wenn Sie ein sprite haben, das 1 Bild pro Sekunde zeichnet, und die Bildgeschwindigkeit auf 0,5 einstellen, wird es mit 0,5 Bildern pro Sekunde zeichnen. Wenn die Bildrate Ihres Spiels 60 Bilder pro Sekunde beträgt, zeichnet sprite 1 Bild pro 120 Spiel-Frames.
  • Wenn Sie ein sprite haben, das 1 Bild pro Spielbild zeichnet, und die Bildgeschwindigkeit auf 0,5 einstellen, wird es 0,5 Bilder pro Spielbild zeichnen. Wenn Ihre Spielbildrate 60 Bilder pro Sekunde beträgt, zeichnet sprite 30 Bilder pro alle 60 Spiel-Frames.

Sie können herausfinden, welche Art von Animationsgeschwindigkeit bei der Definition von sprite verwendet wurde, indem Sie die Funktion sprite_get_speed_type()und Sie können die Animationsgeschwindigkeit und den Typ mit der Funktion sprite_set_speed().

Syntax:

sprite_get_speed(index)

ArgumentTyp Beschreibung
index The index of the sprite to get the speed of

Rückgabe:

Beispiel:

s_speed = sprite_get_speed(sprite_index);
s_type = sprite_get_speed_type(sprite_index);

Die obige code ruft die sprite Geschwindigkeit und den sprite Animationstyp ab und speichert sie in Instanzvariablen für die zukünftige Verwendung.
