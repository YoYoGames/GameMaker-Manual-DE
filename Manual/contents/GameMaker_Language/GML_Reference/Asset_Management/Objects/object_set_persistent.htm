object_set_persistent

Mit dieser Funktion können Sie die Persistenz eines bestimmten Objekts einstellen. Ein persistentes Objekt wird von Raum zu Raum "mitgenommen" und behält seine Eigenschaften zu jeder Zeit bei (weitere Informationen zur Persistenz finden Sie im Abschnitt über den Objekteditor). Das Setzen der Persistenz des Objekts bedeutet, dass alle Instanzen dieses Objekts, die nach dem Ändern der Persistenz angelegt werden, persistent angelegt werden, während Instanzen, die sich bereits im Raum befinden, möglicherweise nicht betroffen sind.

Wenn Sie eine Instanz mit aktivierter Persistenz in einem Raum erstellen, wird ihr eine Ebene oder eine Tiefe zugewiesen (je nachdem, welche Funktion Sie zum Erstellen der Instanz des Objekts verwendet haben). Wenn der Raum gewechselt wird und der folgende Raum keine Ebene mit demselben Namen oder derselben Tiefe wie die zugewiesene hat, wird eine neue Ebene für die Instanz erstellt, die über die Räume hinweg persistent ist. Wenn du bei der Erstellung der Instanz einen Namen für die Ebene vergeben hast, dann wird die neue Ebene auch den gleichen Namen wie die ursprüngliche Ebene haben, und wenn du der Instanz eine Tiefe zugewiesen hast, dann wird die neue Ebene "_layer_XXX" heißen, wobei "XXX" ein Hex-Wert ist, der verwendet wird, um der Ebene einen eindeutigen Namen zu geben. Diese Ebenen werden bei Raumende aus dem Raum entfernt. Ein weiterer Hinweis zu Ebenen und persistenten Instanzen ist, dass wenn du der Instanz bei der Erstellung eine benannte Ebene zugewiesen hast und es in den folgenden Räumen eine andere Ebene mit demselben Namen gibt, dann wird die persistente Instanz der Ebene mit demselben Namen zugewiesen , unabhängig von der Tiefe der Ebene. Wenn eine persistierte Instanz in einen Raum mit einer Ebene in der gleichen Tiefe wie bei der Erstellung der Instanz verschoben wird, wird sie nicht dieser Ebene zugewiesen, sondern es wird eine neue Ebene in der gleichen Tiefe erstellt (gemäß der zuvor erläuterten Benennungskonvention).

Bitte beachten Sie, dass es sich hierbei nicht um eine Instanzfunktion handelt! Sie können das Persistent-Flag einzelner Instanzen mit der persistent setzen und so zehn Instanzen als persistent kennzeichnen, obwohl das betreffende Objekt das Flag in IDE auf false gesetzt hat. Und selbst wenn du das Persistent-Flag des Objekts mit dieser Funktion auf true änderst, bleiben alle Instanzen, die sich derzeit im Raum befinden, so wie sie waren, und nur Instanzen, die nach dem Aufruf der Funktion erstellt werden, beginnen mit dem Persistent-Flag auf true.

Syntax:

object_set_persistent(index, pers);

Argument Typ Beschreibung
index Objekt Anlage The index of the object to change.
pers Boolesche Whether the object should be flagged as persistent (true) oder nicht (false).

Rückgabe:

N/A

Beispiel:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

Die obige code prüft, in welchem Raum sich die Instanz gerade befindet, und wenn sie sich im angegebenen "rm_final" befindet, setzt sie die Persistenz des Objekts "obj_Player" auf false und erstellt dann die Instanz.
