event_inherited

Diese Funktion ruft das aktuelle Ereignis des übergeordneten Objekts der Instanz auf. Normalerweise erbt eine Instanz, wenn sie ein übergeordnetes Objekt hat, automatisch alle Ereignisse des übergeordneten Objekts. Wenn aber (zum Beispiel) Ihr übergeordnetes Objekt ein Erzeugungsereignis hat und Sie Ihrem untergeordneten Objekt eines hinzufügen, werden alle Instanzen des untergeordneten Objekts das neue Erzeugungsereignis ausführen, das Sie hinzugefügt haben, und nicht das des übergeordneten Objekts. Wenn Sie sowohl das Ereignis des übergeordneten Objekts als auch das Ereignis des untergeordneten Objekts desselben Typs verwenden müssen, sollten Sie diese Funktion verwenden, da sie das Ereignis des übergeordneten Objekts ausführt, bevor sie mit dem Rest der code oder den Aktionen fortfährt, die das Ereignis des untergeordneten Objekts enthält.

Syntax:

event_inherited();

Rückgabe:

Beispiel:

event_inherited();
switch (sprite_index)
{
    case spr_Enemy_1: dmg += 2; break;
    case spr_Enemy_4: dmg -= 1; break;
    case spr_Enemy_10: dmg +=10; break;
}

Die obige code ruft das geerbte übergeordnete Ereignis auf (in dem wir die Variable "dmg" initialisieren) und ändert dann die Variable "dmg". Wenn für die Instanz, die dieses code ausführt, kein übergeordnetes Ereignis angegeben ist, würde ein Fehler "unbekannte Variable" auftreten, da dmg nicht definiert wurde.
