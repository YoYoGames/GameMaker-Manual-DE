event_perform_async

Diese Funktion wird verwendet, um eines der in GameMaker verfügbaren asynchronen Ereignisse auszuführen. Sie geben die Async-Ereigniskonstante (siehe Tabelle unten) und eine DS-Map an, die im aufgerufenen Async-Ereignis in der Variablen async_load zur Verfügung stehen wird.

Asynchroner Ereignistyp Konstante
Konstante Beschreibung
ev_async_web_image_load Ereignis "Bild geladen
ev_async_web HTTP-Ereignis
ev_async_dialog Dialog-Ereignis
ev_async_web_iap In-App-Kauf Ereignis
ev_async_web_cloud Cloud-Ereignis
ev_async_web_networking Vernetzungsveranstaltung
ev_async_web_steam Dampf-Ereignis
ev_async_social Gesellschaftliches Ereignis
ev_async_push_notification Ereignis Push-Benachrichtigung
ev_async_save_load Ereignis speichern/laden
ev_async_audio_recording Ereignis Audioaufnahme
ev_async_audio_wiedergabe Ereignis "Audiowiedergabe
ev_async_system_event System-Ereignis

Nicht-asynchrone Ereignisse können aufgerufen werden mit event_perform().

Beachten Sie, dass die in der zweiten argument angegebene DS-Map nicht manuell zerstört werden muss, da sie von der Funktion automatisch zerstört wird, nachdem das Async-Ereignis ausgeführt wurde.

Syntax:

event_perform_async(type, ds_map);

Argument Typ Beschreibung
type Asynchroner Ereignistyp Konstante The type of event to perform (see the table above).
ds_map DS-Karten-ID The DS map to use as async_load in dem aufgerufenen Ereignis.

Rückgabe:

N/A

Beispiel:

var _map = ds_map_create();

_map[? "id"] = "custom_async_event";
_map[? "result"] = true;
_map[? "data"] = { a: 13, b: 16 };

event_perform_async(ev_async_social, _map);

Die obige code erstellt eine DS-Map und füllt sie mit benutzerdefinierten Einträgen, die im Async-Ereignis gelesen werden sollen. Anschließend wird das Async Social-Ereignis mit der neu erstellten Karte ausgeführt, die als async_load für das aufgerufene Ereignis übergeben wird.
