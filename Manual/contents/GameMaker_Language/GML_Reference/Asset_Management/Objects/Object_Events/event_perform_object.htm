event_perform_object

Diese Funktion funktioniert genauso wie event_perform() mit dem Unterschied, dass Sie dieses Mal Ereignisse von einem anderen Objekt angeben können. Es gibt hier viele Optionen, die eine vollständige Simulation aller möglichen Ereignisse ermöglichen, aber beachten Sie, dass dies buchstäblich nur alle code im Ereignis ausführt und das Spiel nichts ändert, um es selbst manuell auszulösen, z.B. wenn Sie ein Tastaturdruck-Ereignis wählen, wird das Ereignis ausgelöst, aber die entsprechende Taste wird nicht als gedrückt erkannt. Oder wenn Sie ein Alarmereignis auslösen, wird der Alarmzähler nicht auf -1 gesetzt, sondern weiter heruntergezählt. Eine vollständige Liste der verfügbaren Konstanten, die diese Funktion benötigt, finden Sie auf der Seite für diese Funktion event_perform().

HINWEIS Aktionen in dem mit dieser Funktion aufgerufenen Ereignis werden auf die aktuelle Instanz und nicht auf Instanzen des angegebenen Objekts angewendet.

Syntax:

event_perform_object(obj, type, numb);

Argument Typ Beschreibung
obj Objekt Anlage The object that should have its event triggered.
type Ereignistyp Konstante The type of event to perform.
numb Real- oder Ereignisnummer Konstante The specific event, if one is necessary (otherwise, just use 0).

Rückgabe:

N/A

Beispiel:

event_perform_object(obj_Player, ev_keypress, ord("W"));

Dies würde das Ereignis ausführen, das mit Tastaturprüfung gedrückt > W-Taste vom Objekt "obj_Player" in der aktuellen Instanz verbunden ist.
