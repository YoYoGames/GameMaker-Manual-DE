Objekte

In GameMaker gibt es Objekte und Instanzen. Objekte sind im Wesentlichen die Basisvorlage für eine Instanz, und als solche sind sie nie direkt in einem Raum vorhanden... nur Instanzen des Objekts werden im Raum platziert. Dieses bedeutet, dass du, wenn du etwas für alle Instanzen, die du erstellst, ändern möchtest, dies durch Änderung des Objekts tun kannst.

WICHTIG! Wenn Sie etwas an einem Objekt ändern, werden keine aktuell im Raum vorhandenen Instanzen geändert, sondern nur die, die nach der Änderung erstellt werden.

Sie können auch auf Informationen über ein Basisobjekt zugreifen und diese verwenden, um Verhaltensweisen oder Änderungen im Spiel zu diktieren. Man kann zum Beispiel nach der ID eines übergeordneten Objekts suchen, und wenn eine Instanz mit diesem übergeordneten Objekt gefunden wird, kann man im folgenden code prüfen object_index überprüfen, um zu entscheiden, was getan werden soll.

GameMaker stellt Ihnen eine Reihe von Funktionen zur Verfügung, mit denen Sie verschiedene Details über ein Objekt erfahren können. Beachte, dass ein Objekt keine Instanz (eine Entität im Spiel) ist, sondern lediglich eine Ressource, aus der alle Instanzen erstellt werden. Das bedeutet dass die Rückgabewerte dieser Funktionen von den tatsächlichen Werten der Instanzen im Raum abweichen können, da eine einmal erstellte Instanz durch code und GML Visual-Aktionen verändert werden kann.

  • object_index
  • object_exists
  • object_get_name
  • object_get_mask
  • object_get_parent
  • object_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • object_get_physics
  • object_is_ancestor

Außerdem gibt es eine Reihe von Funktionen, mit denen Sie die Eigenschaften eines Objekts einstellen können. Es ist zu beachten, dass alle Instanzen dieses Objekts, die bereits im Raum vorhanden sind, von diesen Funktionen nicht betroffen sind, aber alle neuen Instanzen dieses Objekts Es wird daher empfohlen, die Eigenschaften eines Objekts niemals zu ändern, wenn Instanzen dieses Objekts im aktuellen Raum vorhanden sind.

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible

Sie können auch bestimmte Funktionen verwenden, um Ereignisse von code aus einer bestimmten Instanz oder einem bestimmten Objekt heraus zu erzeugen. Diese Funktionen können sehr nützlich sein, insbesondere im Umgang mit Parents und Children, und werden im folgenden Abschnitt beschrieben:

  • Erzeugen von Objektereignissen
