object_is_ancestor

Diese Funktion kann verwendet werden, um ein Objekt zu überprüfen, ob es ein Vorfahre eines anderen Objekts ist oder nicht. Daher müssen beide arguments Objektindizes und nicht die Instanz-IDs sein, und der erste ist immer das Objekt, von dem man herausfinden möchte, ob es ein Kind des zweiten argument ist, das immer das Objekt ist, das man als Vorfahre (Elternteil) überprüfen möchte.

Syntax:

object_is_ancestor(obj, par);

Argument Typ Beschreibung
obj Objekt Anlage The object that is being checked as the child.
par Objekt Anlage The object that is being checked as the ancestor (parent).

Rückgabe:

Boolean

Beispiel:

if object_is_ancestor(object_index, obj_Enemy)
{
    instance_destroy();
}

Die obige code prüft, ob die Instanz, die die code ausführt, ein Kind des Objekts "obj_Enemy" ist, und wenn ja, wird es zerstört.
