font_add_sprite_ext

Mit dieser Funktion können Sie einen "Sprite Strip" ( sprite selbst muss ein sprite -Asset aus dem Asset-Browser sein, oder ein sprite, das Sie mit sprite_add zu den Spielressourcen hinzugefügt haben) verwenden, um ein neues Font-Asset zu erstellen, bei dem jedes Unterbild ein einzelnes Symbol oder ein Buchstabe ist. Anders als bei der normalen font_add_sprite() die eine bestimmte Reihenfolge für die Unterbilder der sprite hat, ordnet diese Funktion die sprite Unterbilder auf der Grundlage der argument "string_map" der Funktion zu. Diese argument ist eine Zeichenkette, mit der du GameMaker mitteilen kannst, in welcher Reihenfolge die Unterbilder der sprite Schriftart angeordnet sind, und die beim Schreiben von Text automatisch zugeordnet wird. Wenn du also zum Beispiel eine String-Map von "AaBbCcDdEeFfGgHh" hast, muss dein sprite Font die Unterbilder in dieser Reihenfolge haben. Sie können als "Leerzeichen" jedes beliebige Zeichen definieren, z. B. eine einzelne Zeile in der gewünschten Größe, und alle Leerzeichen im Text werden in dieser Breite gerendert (aber das gewählte Bild wird nie gerendert), oder wenn Sie keine sprite für Leerzeichen angeben, wird stattdessen die Breite des breitesten Zeichens in der Schrift sprite verwendet.

Sie können auch festlegen, ob der Abstand der Schrift proportional (true) oder nicht proportional (false) sein soll. Bei einer proportionalen Schrift richtet sich der Abstand nach der tatsächlichen Breite der Buchstaben (der Buchstabe "i" nimmt also weniger Platz ein als der Buchstabe "w"), während bei einer nicht proportionalen Schrift der Abstand auf der Breite des Unterbildes basiert, wie bei einer einzeiligen Schrift. Schließlich können Sie den Abstand zwischen den einzelnen Buchstaben beim Schreiben festlegen. Dieser kann ein beliebiger ganzzahliger Wert sein, wobei 0 für keinen Abstand steht (die Buchstaben "berühren" sich, wenn sie proportional sind).

Die Funktion gibt einen Indexwert zurück, der in einer Variablen gespeichert werden sollte, da dieser in allen weiteren codes benötigt wird, die sich auf diese Schriftart beziehen, oder sie gibt -1 zurück, wenn die Funktion aus irgendeinem Grund fehlschlägt.

Es ist erwähnenswert, dass die Schriftausrichtungsfunktionen (wie draw_set_halign()), sollte der Ursprung der Schrift sprite auf die linke obere Ecke gesetzt werden. Wenn Sie andere Werte verwenden, müssen Sie beim Zeichnen von Text mit dieser Schriftart den Ursprungsversatz berücksichtigen.

HINWEIS Wenn Sie eine Schriftart in GameMaker laden, müssen Sie daran denken, sie wieder zu entfernen (mit font_delete()) zu entfernen, wenn Sie sie nicht mehr benötigen, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

font_add_sprite_ext(spr, string_map, prop, sep);

Argument Typ Beschreibung
spr Sprite-Asset The sprite to add a font based on.
string_map Zeichenfolge String from which sprite sub-image order is taken.
prop Boolesche Set as proportional font or not.
sep Real The space to leave between each letter.

Rückgabe:

Font Asset

Beispiel:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

Die obige code erstellt ein neues Font-Asset aus der in der Variablen "spr_CalcFont" indizierten sprite und speichert den Index des neuen Fonts in der Variablen "global.Font". Die Unterbilder der Schriftart sprite werden auf die angegebene Zeichenfolge abgebildet.
