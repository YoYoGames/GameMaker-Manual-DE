font_set_cache_size

Beim Überblenden und Einfärben von Bildern kann HTML5 nicht dynamisch vorgehen, wie es bei einer ausführbaren Datei möglich wäre. Daher muss GameMaker eine Kopie der überblendeten Bilder vorübergehend speichern und bei Bedarf laden. laden, wenn er sie braucht. Diese Funktion legt fest, wie viele Mischkopien der angegebenen Schriftart zwischengespeichert werden können, bevor die alten überschrieben werden. Der Standardwert ist 4.

HINWEIS Für die Schriftarten sprite sollten Sie die entsprechende Funktion für sprites verwenden, sprite_set_cache_size().

Syntax:

font_set_cache_size(ind, max);

ArgumentTyp Beschreibung
ind The index of the font to change the cache size of.
max The maximum number of cached copies of the font that can be stored.

Rückgabe:

Beispiel:

font_set_cache_size(fnt_MainMenu, 2);

Dadurch wird die Schrift cache der in der Variablen "fnt_MainMenu" indizierten Schrift auf 2 Kopien gesetzt.
