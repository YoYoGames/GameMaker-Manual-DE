schrift_textur_seite_grösse

Diese eingebaute Variable kann verwendet werden, um die Größe der Texturseite zu ermitteln oder festzulegen, wenn die Funktion font_add(). Wenn Sie eine Schriftart mit dieser Funktion hinzufügen, erstellt GameMaker eine Texturseite cache mit den erforderlichen Glyphen bis zu der durch diese Variable festgelegten Größe. Der Vorgang ist wie folgt:

  • Jedes Zeichen, das Sie verwenden, wird von der Schrift in einer Texturseite zwischengespeichert
  • Wenn die Texturseite voll ist (weil Sie große Zeichen oder viele verschiedene Zeichen verwenden), entfernt GameMaker Zeichen von der cache und ersetzt sie durch neue, die Sie benötigen.
  • GameMaker versucht, die Textur nicht zu groß zu machen, weil nicht alle Geräte mit großen Texturen umgehen können, und er versucht, die Textur nicht zu klein zu machen, weil das Entfernen und Wiedereinfügen von Zeichen in die Textur CPU-Zeit in Anspruch nimmt und das Spiel langsam läuft.

Wenn Sie glauben, dass Sie eine größere oder kleinere Texturseitengröße als die standardmäßige 1024px benötigen, die GameMaker verwendet, können Sie den Wert mit dieser eingebauten Variablen einstellen. Wenn Sie die Funktion font_add() nicht verwenden, hat diese Einstellung keine Auswirkungen auf die Leistung Ihres Spiels, da die in IDE hinzugefügten Schriftarten auf den regulären Texturseiten zwischengespeichert werden (was Sie über den Schriftarteneditor und die Spieloptionen für das Ziel platform steuern können).

Syntax:

font_texture_page_size;

Rückgabe:

Real (maximum width/height in pixels)

Beispiel:

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

Die obige code setzt die Schriftart cache texture page auf eine maximale Größe von 512px und fügt dann eine neue Schriftart zum Spiel hinzu.
