font_get_first (veraltet)

Wenn Sie in GameMaker eine Schriftart definieren, können Sie eine Reihe von Zeichen festlegen, die enthalten sein sollen. Das liegt daran, dass die Schriftart selbst nicht in Ihr Spiel aufgenommen wird (aus rechtlichen Gründen), sondern ein Bild der der Schriftart auf einer Texturseite enthalten ist, und das ist es, was Ihr Spiel verwenden wird (genau wie jedes andere Grafikobjekt). Das bedeutet, dass du die Anzahl der Zeichen, die du verwendest, auf ein Minimum beschränken und nur den Bereich der Zeichen angeben solltest, den die Ihr Spiel benötigt, um den Texturspeicher so optimal wie möglich zu nutzen. Mit dieser Funktion können Sie das Startzeichen (als ASCII-Wert) ermitteln, das verwendet wurde, als die Schriftart dem Spiel hinzugefügt wurde.

HINWEIS Aufgrund von Änderungen in der Handhabung von Schriftarten wird diese Funktion immer 32 zurückgeben, da GameMaker das Leerzeichen (32) in die Schriftart erzwingt und auch keine auch keine Zeichen mit codes kleiner als 32 zulässt. Neue Funktionen, die ähnliche Aufgaben erfüllen, werden in einem zukünftigen Update hinzugefügt.

Syntax:

font_get_first(ind);

ArgumentTyp Beschreibung
ind Index of the font to check.

Rückgabe:

Beispiel:

num = font_get_first(fnt_Main);

Die obige code speichert den ASCII-Wert des ersten Buchstabens des Schriftbereichs für die in "fnt_Main" indizierte Schrift.
