font_add_sprite

Mit dieser Funktion können Sie einen "Sprite Strip" verwenden, um ein neues Font-Asset zu erstellen, bei dem jedes Unterbild ein einzelnes Symbol oder einen Buchstaben darstellt. Der Index ist die UTF8-Zuordnungsnummer für das erste zu verwendende Zeichen, und die zu verwendenden Bereiche sind die gleichen wie bei einer normalen Schrift, beginnend mit "!", aber denken Sie daran, dass Sie nur das einschließen sollten, was Sie brauchen, und dass Sie die ord() Funktion verwenden, um den richtigen UTF8-Wert für den Anfangsbuchstaben zu erhalten. Die erstellte Schriftart sprite hat dann die Unterbilder, die auf eine einzelne Glyphe der Schriftart gesetzt sind, wobei die "erste" Glyphe den image_index 0 hat. Sie können als "Leerzeichen" ein beliebiges Zeichen definieren, z. B. eine einzelne Zeile in der Größe, die das Leerzeichen haben soll, und alle Leerzeichen im Text werden in dieser Breite gerendert (aber das gewählte Bild wird nie gerendert), oder wenn Sie keine sprite für Leerzeichen angeben, wird stattdessen die Breite des breitesten Zeichens in der Schriftart sprite verwendet.

Sie können auch festlegen, ob die Schrift proportional (true) oder nicht proportional (false) sein soll. Bei einer proportionalen Schrift richtet sich der Abstand nach der tatsächlichen Breite der Buchstaben (der Buchstabe "i" nimmt also weniger Platz ein als der Buchstabe "w"), während bei einer nicht proportionalen Schrift der Abstand auf der Breite des Teilbildes basiert, wie bei einer einzeiligen Schrift. Schließlich können Sie den Abstand zwischen den einzelnen Buchstaben beim Schreiben festlegen. Dieser kann ein beliebiger ganzzahliger Wert sein, wobei 0 für keinen Abstand steht (die Buchstaben "berühren" sich, wenn sie proportional sind).

Die Funktion gibt einen Indexwert zurück, der in einer Variablen gespeichert werden sollte, da dieser in allen weiteren codes benötigt wird, die sich auf diese Schriftart beziehen, oder sie gibt -1 zurück, wenn die Funktion aus irgendeinem Grund fehlschlägt.

Zu beachten ist auch, dass die Zeichentabelle keine Leerzeichen enthalten darf. Wenn Sie also beispielsweise '.' und '0', aber nicht '/' verwenden, müssen Sie entweder trotzdem '/' einfügen oder ein leeres Teilbild zwischen den beiden verwendeten Zeichen lassen.

sprite selbst kann ein sprite -Asset aus dem Asset-Browser sein oder eines, das aus einer Datei geladen wird, indem zuerst sprite_add verwendet wird, um den Index des geladenen sprite zu erhalten (der dann in der Funktion verwendet wird). Es ist wichtig zu beachten, dass die Funktionen zur Schriftausrichtung (wie draw_set_halign()) bei Verwendung einer Schriftart sprite erwartungsgemäß funktionieren, der Ursprung der Schriftart sprite auf die linke obere Ecke eingestellt sein muss. Wenn Sie andere Werte verwenden, müssen Sie beim Zeichnen von Text mit der Schriftart den Ursprungsversatz berücksichtigen.

HINWEIS Wenn Sie eine Schriftart in GameMaker laden, müssen Sie daran denken, sie wieder zu entfernen (mit font_delete()) zu entfernen, wenn Sie sie nicht mehr benötigen, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

font_add_sprite(spr, first, prop, sep);

Argument Typ Beschreibung
spr Sprite-Asset The sprite to add a font based on.
first Real The index of the first available character.
prop Boolesche Set as proportional font or not.
sep Real The space to leave between each letter.

Rückgabe:

Font Asset

Beispiel:

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

Die obige code erstellt ein neues Font-Asset aus einem sprite -Asset und speichert den Index des neuen Fonts in der Variablen "global.Font" zur späteren Verwendung.
