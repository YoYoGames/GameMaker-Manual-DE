steam_is_screenshot_requested

Diese Funktion fragt die Steam API ab, um zu sehen, ob die Taste für die Aufnahme eines Screenshots des Spiels gedrückt wurde. Die Funktion gibt nur für einen Schritt (Spieltick) true zurück, wenn die Taste gedrückt wurde, und gibt zu allen anderen Zeiten false zurück.

Bitte beachten Sie, dass diese Funktion, wenn die Screenshot-Taste gedrückt wird, nur einmal für jeden Schritt, in dem sie gedrückt wird, true zurückgibt und bei allen weiteren Aufrufen innerhalb desselben Schritts false zurückgibt. Wenn z. B. im aktuellen Frame ein Screenshot angefordert wird und Sie diese Funktion im Step-Ereignis aufrufen, um dies herauszufinden, erhalten Sie true; wenn Sie sie jedoch in Draw GUI erneut aufrufen, um zu prüfen, ob ein Screenshot angefordert wurde, erhalten Sie false, da die Funktion bereits im Step-Ereignis "verbraucht" wurde. Um den Rückgabewert der Funktion mehrfach innerhalb desselben Frames zu verwenden, empfiehlt es sich, ihn in einer Variablen zu speichern und diese auszulesen, anstatt die Funktion erneut aufzurufen.

HINWEIS: Diese Funktion macht keinen Screenshot für Sie. Sie signalisiert nur, dass die Taste gedrückt wurde und Sie müssen die Funktionen GameMaker Studio 2 screen_save() oder screen_save_part() verwenden, um eine lokale Kopie der hochzuladenden Datei zu speichern.

Syntax:

steam_is_screenshot_requested();

Rückgabe:

Boolean

Beispiel:

if steam_is_screenshot_requested()
{
    var file = "Catch_The_Haggis_" + string(global.scrn_num) + ".png");
    screen_save(file)
    steam_send_screenshot(file, window_get_width(), window_get_height());
    global.scrn_num += 1;
}

Die obige code wird die Steam API nach einer Screenshot-Anfrage abfragen und wenn dies der Fall ist, wird ein eindeutiger Name für die Bilddatei generiert, ein Screenshot gemacht und die Datei an die Steam Community-Seite für den Benutzer gesendet.
