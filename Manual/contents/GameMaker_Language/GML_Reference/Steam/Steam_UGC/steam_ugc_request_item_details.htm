steam_ugc_request_item_details

Diese Funktion kann verwendet werden, um Informationen über eine bestimmte Datei-ID abzurufen. Sie geben die Datei-ID und ein maximales Alter für die Überprüfung an (weitere Informationen finden Sie in den Steam docs), dann gibt die Funktion einen eindeutigen asynchronen ID-Wert zurück, der zur korrekten Überprüfung der Details im Steam-Async-Ereignis verwendet werden kann. Dieses Ereignis hat die async_load DS-Map, die mit den folgenden Schlüssel/Wert-Paaren aufgefüllt wird:

  • "id" - Die asynchrone ID, die von der Funktion zurückgegeben wird, die das Ereignis ausgelöst hat
  • "result" - Das Ergebnis der Operation (ein reeller Wert). Dies ist entweder die Konstante GML ugc_result_success oder eine andere reelle Zahl. Sie sollten also auf diese Konstante prüfen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn andernfalls etwas nicht korrekt funktioniert hat. Die übrigen möglichen Werte, die zurückgegeben werden, werden als Ergebnis des Steam "EResult"-Wertes angezeigt, und Sie sollten in der Datei steamclientpublic.h in den SDK-Headern alle 89 möglichen Werte finden.
  • "event_type" - Die Zeichenfolge"ugc_item_details".
  • "cached_data" - Ist true, wenn die zurückgegebenen Daten von der lokalen cache stammen, oder false, wenn sie vom Server stammen.
  • "published_file_id"" - enthält die eindeutige Kennung der veröffentlichten Datei für das Element
  • "file_type" - Der Typ der verwendeten Datei
  • "creator_app_id" - Die Steam ID des Artikelerstellers
  • "consumer_app_id" - Die Steam ID des Artikels consumer
  • "Titel" - Der Titel des Artikels
  • "description" - Die Beschreibung des Artikels
  • "steam_id_owner" - Die Steam ID des Eigentümers des Artikels
  • "time_created" - Die Zeit, zu der das Objekt erstmals erstellt wurde
  • "time_updated" - Das letzte Mal, dass der Artikel aktualisiert wurde
  • "time_added_to_user_list" - Der Zeitpunkt, zu dem der Artikel abonniert wurde
  • "visibility" - Die Sichtbarkeit des Objekts (siehe steam_ugc_set_item_visibility() für die zurückgegebenen Konstanten)
  • "banned" - Ob der Artikel verboten wurde(true) oder nicht(false)
  • "accepted_for_use" - Angabe, ob der Gegenstand zur Verwendung zugelassen wurde(true) oder nicht(false)
  • "tags_truncated" - Kurzversion der Tags als Array
  • "Tags" - Ein Array mit den Tags für den Artikel
  • "handle_file" - Der eindeutige Datei-Handle für das Element
  • "handle_preview_file" - Das eindeutige Handle für die Bildvorschau für den Artikel (kann als argument verwendet werden mit steam_ugc_download() um ein Vorschaubild herunterzuladen)
  • "Dateiname" - Der Name der Artikeldatei
  • "file_size" - Die Größe der Artikeldatei
  • "preview_file_size" - Die Größe des Vorschaubildes
  • "url" - Die vollständige URL für den Artikel
  • "votes_up" - Die Anzahl der erhaltenen Up-Votes
  • "votes_down" - Die Anzahl der erhaltenen Abwärtsstimmen
  • "Punktzahl" - Die Gesamtpunktzahl des Artikels
  • "account_id_owner" - Die Account-ID des Steam ID-Besitzers (diese kann in der Funktion steam_ugc_create_query_user_ex())

Syntax:

steam_ugc_request_item_details(published_file_id, max_age_seconds);

Argument Beschreibung
published_file_id Die eindeutige Datei-ID für die zu prüfende UGC.
max_age_seconds Das "Alter" der zu prüfenden Daten (empfohlen 30 - 60 Sekunden).

Rückgabe:

Async ID

Erweitertes Beispiel:

In diesem Beispiel senden wir eine Detailanfrage für ein Element und analysieren dann die resultierende async_load DS-Map, um einige Variablen zu setzen. Zunächst senden wir die Anfrage ab:

steam_details = steam_ugc_request_item_details(global.fileID, 60);

Die obige code fordert Details zum Element mit der in der globalen Variable gespeicherten Datei-ID an und löst ein Steam Async-Ereignis mit den zurückgegebenen Informationen aus. In diesem Ereignis können wir dann die Karte parsen und einige der Werte in Variablen speichern, die dann verwendet werden können, um die Informationen für den Benutzer anzuzeigen:

var map_id = async_load[? "id"];
var result = async_load[? "result"];
if (map_id == steam_details) && (result == ugc_result_success)
{
    mTitle = async_load[? "title"];
    mDesc = async_load[? "description"];
    mTags = async_load[? "tags"];
    m_hPreviewFile = async_load[? "handle_preview_file"];
    m_hOwnerSteamId = async_load[? "steam_id_owner"];
    mOwnerAccountId = async_load[? "account_id_owner"];
    mPubFileId = async_load[? "published_file_id"];
    mScore = async_load[? "score"];
}
