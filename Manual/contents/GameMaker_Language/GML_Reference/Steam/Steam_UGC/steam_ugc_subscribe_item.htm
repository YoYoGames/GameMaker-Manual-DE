steam_ugc_subscribe_item

Diese Funktion kann verwendet werden, um ein UGC-Element zu abonnieren. Sie geben die ID der veröffentlichten Datei an, und die Funktion gibt einen eindeutigen Async-ID-Wert zurück, der verwendet werden kann, um zu überprüfen, ob das abonnierte Element im Ereignis Steam Async heruntergeladen wurde. Dieses Ereignis enthält die async_load DS-Map, die mit den folgenden Schlüssel/Wert-Paaren gefüllt wird:

  • "id" - Die asynchrone ID, die von der Funktion zurückgegeben wird, die das Ereignis ausgelöst hat
  • "result" - Das Ergebnis der Operation (ein reeller Wert). Dies ist entweder die Konstante GML ugc_result_success oder eine andere reelle Zahl. Sie sollten also auf diese Konstante prüfen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn andernfalls irgendetwas nicht korrekt funktioniert hat. Die übrigen möglichen Werte, die zurückgegeben werden, werden als Ergebnis des Steam "EResult"-Wertes angezeigt, und Sie sollten in der Datei steamclientpublic.h in den SDK-Headern alle 89 möglichen Werte finden.
  • "event_type" - Die Zeichenfolge "ugc_subscribe_item".
  • "published_file_id" - Die eindeutige ID für das veröffentlichte Element.

Syntax:

steam_ugc_subscribe_item(published_file_id);

Argument Beschreibung
published_file_id Die eindeutige Datei-ID für die zu abonnierende UGC.

Rückgabe:

Async ID

Beispiel:

steam_sub = steam_ugc_subscribe_item(global.pubFileID);

Mit der obigen code wird das Element mit der in der globalen Variable "pubFileID" gespeicherten Datei-ID abonniert (und heruntergeladen).
