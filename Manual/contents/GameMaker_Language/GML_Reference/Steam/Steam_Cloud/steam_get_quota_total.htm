steam_get_quota_total

Wenn Sie die Steam Cloud zum Speichern und Synchronisieren von Dateien verwenden, müssen Sie ein Kontingent für den Speicherplatz einrichten, den Ihr Spiel benötigen wird. Dieses Kontingent wird für jedes Cloud-fähige Spiel pro Benutzer und pro Spiel festgelegt. Wenn das Kontingent für Spiel X beispielsweise 1 Megabyte beträgt, kann jedes Steam -Konto, dem Spiel X gehört, maximal 1 Megabyte an Daten zu diesem Spiel in der Cloud speichern. Alle anderen Cloud-fähigen Spiele, die der Benutzer besitzt (z. B. Spiel Y), sind von den von Spiel X gespeicherten Daten nicht betroffen. Das Standardkontingent für neue Steamworks-Spiele beträgt ein Gigabyte, aber Sie können dies über das Steamworks-Kontrollfeld für Ihr Spiel ändern.

HINWEIS: Sobald das Kontingent erschöpft ist , werden Dateien nicht mehr geschrieben. Wenn Sie es für möglich halten, dass das Kontingent für den Benutzer Ihres Spiels erschöpft sein könnte, sollten Sie code erstellen, um damit umzugehen, denn wenn Sie nichts tun, lassen Sie die Benutzer in einer Situation zurück, in der sie nicht in der Lage sind, Dinge zu beheben, und das wird zu einer schlechten Erfahrung mit Ihrem Spiel führen.

Syntax:

steam_get_quota_total();

Rückgabe:

Real

Beispiel:

if steam_is_cloud_enabled_for_app()
{
    quota = steam_get_quota_total();
}

Die obige code prüft, ob die steam Cloud für das Spiel aktiviert ist, und wenn ja, ermittelt sie die Größe des Speicherkontingents und speichert sie in einer Variablen.
