Zeitquelle Verfallsarten

Zeitquelle Verfallskonstante
Konstante Beschreibung Wert
time_source_expire_nearest Die Zeitquelle läuft in dem Rahmen ab, der ihrer Ablaufzeit am nächsten liegt. 0
time_source_expire_after Die Zeitquelle läuft mit dem ersten Frame nach ihrer Ablaufzeit ab. 1

Zeitquellen, die Sekunden als Einheit verwenden, laufen nicht immer in einem Spielframe ab. Sie können ablaufen, nachdem ein Spiel-Frame verarbeitet wurde, aber bevor ein neuer Frame mit der Verarbeitung beginnen kann.

Der Callback für eine Zeitquelle kann jedoch nur in einem Spielframe ausgeführt werden, so dass der Ablauftyp einer Zeitquelle steuert, wann ihr Callback ausgeführt wird, falls er in der Mitte von zwei Frames abläuft.

Bei der Verwendung von time_source_expire_nearest wird der Callback für eine Zeitquelle auf dem Frame ausgeführt, der der Ablaufzeit am nächsten ist, was der Frame davor oder der Frame danach sein kann.

Bei der Verwendung von time_source_expire_after wird der Callback für eine Zeitquelle immer auf dem Frame nach ihrer Ablaufzeit ausgeführt.
