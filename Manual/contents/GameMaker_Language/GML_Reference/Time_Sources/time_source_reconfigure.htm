zeit_quelle_rekonfigurieren

Diese Funktion wird verwendet, um die Haupteigenschaften einer Zeitquelle zu ändern, ohne eine völlig neue Zeitquelle erstellen zu müssen.

Sie geben eine vorhandene Zeitquelle an und legen dann die Eigenschaften fest, die auch in time_source_create()festgelegt sind, außer der übergeordneten. Ausführliche Informationen zu diesen Eigenschaften finden Sie auf dieser Seite.

Die angegebene Zeitquelle wird zurückgesetzt und deaktiviert und muss dann neu gestartet werden.

Syntax:

time_source_reconfigure(id, period, units, callback, [args, repetitions, expiry_type]);

Argument Typ Beschreibung
id Zeitquellen-ID The Time Source zu rekonfigurieren
period Real The period that the Time Source läuft für, in den angegebenen Einheiten
units Zeitquelle Einheit Konstante The units that the given period is in
callback Methode The method to call when the Time Source Läuft ab
args Array OPTIONAL Ein Array, das die arguments enthält, die an die Methode übergeben wird
repetitions Real OPTIONAL Die Anzahl der Wiederholungen der Zeitquelle, oder -1 für unbestimmte Wiederholungen
expiry_type Zeitquelle Verfallskonstante OPTIONAL ob die Zeitquelle in dem Frame, der ihrem Ablauf am nächsten liegt, oder im nächsten Frame abläuft

Rückgabe:

N/A

Beispiel:

function change_spawn_delay(new_delay)
{
    time_source_reconfigure(obj_game.spawn_time_source, new_delay, time_source_units_frames, obj_game.spawn_method, [], -1);
    time_source_start(obj_game.spawn_time_source);
}

Damit wird eine neue Funktion erstellt, die die Spawn-Verzögerung für Gegner im Spiel ändert.

Angenommen, das Spiel verwendet eine Zeitquelle namens obj_game.spawn_time_source, um Feinde zu spawnen, dann muss diese Zeitquelle aktualisiert werden, sobald sich die Spawn-Verzögerung ändert.

Diese Funktion tut genau das, indem sie die Zeitquelle mit der neuen Verzögerung rekonfiguriert und sie dann wieder startet.
