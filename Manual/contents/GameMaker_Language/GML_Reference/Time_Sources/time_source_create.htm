zeit_quelle_erstellen

Diese Funktion erstellt eine neue Zeitquelle. Lesen Sie Zeitquellen für einen Überblick.

Das Anlegen einer neuen Zeitquelle startet diese nicht automatisch; Sie müssen time_source_start() aufrufen, um sie zu aktivieren.

Eine neue Zeitquelle wird nur einmal ausgeführt, es sei denn, die Wiederholungen argument sind angegeben (siehe unten).

Sie müssen eine Zeitquelle zerstören, indem Sie time_source_destroy() zerstören, wenn Sie sie nicht mehr benötigen.

Argumente

Hier eine Aufschlüsselung der Argumente der Funktion:

Elternzeit Quelle

Dies ist die übergeordnete Zeitquelle, die die neue Zeitquelle steuert. Dies kann time_source_global, time_source_game oder eine bereits vorhandene benutzerdefinierte Zeitquelle sein.

Siehe: Eingebaute Zeitquellen

Zeitraum

Dies ist eine Zeitspanne, die in Sekunden oder Frames ausgedrückt werden kann, je nachdem, welche Einheit auf der nächsten Seite argument angegeben ist.

Wenn Sie Frames als Einheit verwenden, muss die Periode eine ganze Zahl sein. Nicht ganzzahlige Werte werden abgerundet, mit Ausnahme von Werten kleiner als 1, die auf 1 aufgerundet werden.

Einheit

Dies ist die Einheit, in der der Zeitraum ausgedrückt wird, und kann time_source_units_seconds oder time_source_units_frames sein.

Siehe: Zeitquelleneinheiten

Sie können einen Wert für die Schlagzahl pro Minute verwenden, indem Sie time_bpm_to_seconds() verwenden.

Rückruf & Args

Sie müssen einen "Callback" angeben method angeben, der aufgerufen wird, wenn die Zeitquelle abläuft.

Sie können optional ein Array angeben, das die arguments enthält, die an die Methode übergeben werden soll, wenn sie aufgerufen wird.

Das Array selbst wird nicht an die Methode übergeben, sondern jedes Element des Arrays wird als separates argument übergeben.

Wenn Ihre Funktion zum Beispiel arguments (x, y, object) erwartet, dann könnte Ihr Args-Array wie folgt aussehen: [30, 600, obj_player].

Wiederholungen

Sie können optional angeben, wie oft Ihre Zeitquelle wiederholt werden soll. Ein Wert von 1 bedeutet, dass sie nur einmal läuft, was der Standardwert ist.

Sie können die Gesamtzahl der Wiederholungen angeben oder -1, um die Wiederholungen auf unbestimmte Zeit zu beschränken.

Wenn Sie diesen Wert beispielsweise auf 3 setzen und Ihre Zeitquelle auf "4 Sekunden" eingestellt ist, dann ist die Zeitquelle insgesamt 12 Sekunden lang aktiv und ruft die Callback-Methode alle 4 Sekunden auf.

Art des Verfalls

Dies kann time_source_expire_nearest oder time_source_expire_after sein.

Siehe: Zeitquellen-Verfallstypen

Syntax:

time_source_create(parent, period, units, callback, [args, repetitions, expiry_type]);

Argument Typ Beschreibung
parent Zeitquellen-ID The parent Time Source die die neue Zeitquelle steuert
period Real The period that the Time Source läuft für, in den angegebenen Einheiten
units Zeitquelle Einheit Konstante The units that the given period is in
callback Methode oder Skript Funktion The method or script function to call when the Time Source Läuft ab
args Array OPTIONAL Ein Array, das die arguments enthält, die an die Methode übergeben wird
repetitions Real OPTIONAL Die Anzahl der Wiederholungen der Zeitquelle, oder -1 für unbestimmte Wiederholungen
expiry_type Zeitquelle Verfallskonstante OPTIONAL ob die Zeitquelle in dem Frame, der ihrem Ablauf am nächsten ist, oder im nächsten Frame abläuft

Rückgabe:

Time Source ID

Beispiel 1:

var _my_method = function()
{
    instance_destroy();
}

var _time_source = time_source_create(time_source_game, 300, time_source_units_frames, _my_method);

time_source_start(time_source);

In diesem Beispiel soll sich die Instanz 300 Frames später selbst zerstören.

Die code erstellt zunächst eine Methode, die einfach aufruft instance_destroy().

Anschließend wird eine Zeitquelle erstellt, die von der Zeitquelle des Spiels geerbt wird. Sie setzt ihre Periode auf 300 Frames.

Schließlich wird die Zeitquelle gestartet.

Beispiel 2:

var _my_method = function()
{
    show_debug_message("A second has passed!");
}

global.time_per_second = time_source_create(time_source_game, 1, time_source_units_seconds, _my_method, [], -1, time_source_expire_after);

time_source_start(global.time_per_second);

In diesem Beispiel erstellen wir eine globale Zeitquelle, die einmal pro Sekunde abläuft. Diese code würde an der Wurzel einer script.

Auf code wird zunächst eine Methode erstellt, die eine Meldung in das Ausgabeprotokoll ausgibt, die besagt : "Eine Sekunde ist vergangen".

Anschließend wird eine neue Zeitquelle erstellt, die von der Zeitquelle des Spiels geerbt wird. Sie setzt ihre Periode auf 1 Sekunde.

Die Methode wird an die Zeitquelle übergeben, damit sie jedes Mal aufgerufen werden kann, wenn sie abläuft. Für die arguments wird ein leeres Array angegeben.

Die Anzahl der Wiederholungen ist auf -1 gesetzt, so dass die Zeitquelle nie aufhört und sich immer wieder wiederholt. Der Ablauftyp ist so eingestellt, dass der Callback beim ersten Frame nach Ablauf der Zeitquelle ausgeführt wird.

Schließlich wird die Zeitquelle gestartet.
