gif_add_surface

Mit dieser Funktion können Sie ein Bild aus einer Oberfläche in eine GIF-Datei speichern. Sie geben die ID der GIF-Datei (wie sie von der Funktion gif_open()) und eine Oberflächen-ID (zurückgegeben von der Funktion surface_create()), sowie eine Verzögerungszeit zwischen den Bildern. Die Verzögerungszeit wird in 1/100 Sekunden berechnet, aber beachten Sie, dass nach der Erstellung des GIFs, wenn Sie es z. B. für Werbezwecke verwenden, verschiedene Browser die Frame-Verzögerung leicht unterschiedlich interpretieren können - siehe hier für weitere Details. Sie können (optional) X- und Y-Offset-Werte für die Fläche angeben, die zum GIF hinzugefügt werden soll. In diesem Fall beginnt der Teil, der zum GIF gezeichnet wird, an der Offset-Position und nicht an der Standardposition (0,0), so dass Sie nur einen Teil der Fläche zum Hinzufügen auswählen können. Sie können auch einen Quantisierungswert angeben, der eine ganze Zahl zwischen 0 und 3 ist, wobei 0 für volle Quantisierung und 3 für Nullquantisierung steht (niedrigere Werte führen zu einer Qualitätsverschlechterung, erzeugen aber ein kleineres GIF). Beachten Sie, dass es keinen großen Unterschied zwischen der Standardqualität von 2 und der höchsten von 3 gibt, und dass die Verwendung von 3 die Erstellung des GIFs erheblich verlangsamt, so dass Sie bei der Verwendung des Maximalwerts vorsichtig sein sollten.

Beachten Sie, dass es keine eingebauten Grenzen für die Anzahl der Frames oder die Größe dieser Frames in Ihrem GIF gibt, aber wenn Sie zu viel Speicher verwenden, wird die Funktion fehlschlagen und -1 zurückgeben, andernfalls wird sie 0 zurückgeben.

Syntax:

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

Argument Typ Beschreibung
gif_index GIF-ID The ID of gif to add the surface data to
surface Oberfläche ID The ID of the surface to use as the added frame
delay_time Real The delay time for the frame (in 100ths of a second)
[xoffset] Real OPTIONAL Der Versatz entlang der X-Achse der Oberfläche, der von
[yoffset] Real OPTIONAL Der Versatz entlang der Y-Achse der Oberfläche, der von
[quantization] Real OPTIONAL Der Quantisierungsgrad von 0 bis 3 (0 niedrigste Qualität, 3 höchste Qualität, Standard ist 2)

Rückgabe:

Real

Beispiel:

if save_gif == true
{
    if (count == 0)
    {
        gif_image = gif_open(room_width, room_height);
    }
    else if (count < 30)
    {
        gif_add_surface(gif_image, application_surface, 6/100);
    }
    else
    {
        gif_save(gif_image, "GameCapture.gif");
        count = 0;
        save_gif = false;
    }
    count++;
}

Mit der obigen code wird eine GIF-Bilddatei mit 30 Einzelbildern aus der Anwendungsoberfläche erstellt und anschließend gespeichert.
