screen_save_part

Mit dieser Funktion können Sie einen Screenshot eines Teils des Spiels so speichern, wie er gerade in der Spielanzeige erscheint. Die Auswahlkoordinaten basieren auf der absoluten Größe des Bildschirms (oder des Fensters, wenn es sich nicht um einen Vollbildschirm handelt), so dass (0,0) immer die linke obere Ecke des Bildschirms (oder des Fensters) ist. Das folgende Bild zeigt, wie diese Funktion funktioniert:

Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert. Es ist wichtig zu beachten, dass die Funktion für den Aufruf im Draw GUI End Event vorgesehen ist, was konsistente Ergebnisse für alle platforms liefern sollte. Die Verwendung der Funktion in einem anderen Ereignis funktioniert möglicherweise nicht wie erwartet und kann zu unterschiedlichen Ergebnissen auf verschiedenen Zielen und sogar Geräten führen.

HINWEIS Diese Funktion funktioniert nicht mit dem HTML5-Ziel.

Syntax:

screen_save_part(fname, x, y, w, h);

ArgumentTyp Beschreibung
fname The filename for the screenshot to save
x The x coordinate of the top left corner of the area of the screen to capture
y The y coordinate of the top left corner of the area of the screen to capture
w The width of the area of the screen to capture
h The height of the area of the screen to capture

Rückgabe:

Beispiel:

if keyboard_check_pressed(ord("S"))
{
    screen_save_part("Screens\Screen_" + string(num++) + ".png", 0, 0, 100, 100);
}

Die obige code macht einen Screenshot von einem Teil des Spielbildschirms, der 100 Pixel im Quadrat groß ist, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Ordner gespeichert und mit einer Variablen nummeriert, die dann hochgezählt wird.
