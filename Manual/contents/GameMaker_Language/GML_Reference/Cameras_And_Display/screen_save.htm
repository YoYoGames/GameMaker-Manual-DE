Bildschirm_sichern

Mit dieser Funktion können Sie einen Screenshot des Spiels speichern, wie er gerade im Spielfenster erscheint.

Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert, das sich unter Windows unter <drive>:/Users/YOURUSERNAME/AppData/Local/[Game Name]/ und unter Mac unter ~/Library/Application Support/[Game Name]/ befindet. Bei dem aufgenommenen Bild handelt es sich um das endgültige Rendering der Anwendungsoberfläche des Spiels oder (wenn die Anwendungsoberfläche deaktiviert wurde) um ein Bild des gesamten Bildschirms oder Fensters.

Es ist wichtig zu beachten, dass die Funktion so konzipiert ist, dass sie im Draw GUI End Event aufgerufen werden kann, was zu konsistenten Ergebnissen auf allen platforms führen sollte. Die Verwendung der Funktion in einem anderen Ereignis funktioniert möglicherweise nicht wie erwartet und kann zu unterschiedlichen Ergebnissen auf verschiedenen Zielen und sogar Geräten führen.

Wenn Sie mehr Kontrolle über das, was gespeichert wird, benötigen, sollten Sie die Funktion screen_save_part().

HINWEIS Diese Funktion funktioniert nicht mit dem HTML5-Ziel.

Syntax:

screen_save(fname);

Argument Typ Beschreibung
fname Zeichenfolge The filename for the screenshot to save

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(ord("S"))
{
    screen_save(working_directory + "Screens\Screen_" + string(num++) + ".png")
}

Die obige code macht einen Screenshot des Spiels, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Verzeichnis gespeichert und mit einer Variablen nummeriert, die dann hochgezählt wird.
