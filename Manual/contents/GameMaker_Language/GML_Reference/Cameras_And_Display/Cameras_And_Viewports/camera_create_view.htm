kamera_erstellen_ansicht

Diese Funktion erstellt eine neue Kamera und legt dann ihre Ansicht im Raum fest, bevor sie den eindeutigen ID-Wert zurückgibt, um die Kamera in zukünftigen Funktionsaufrufen zu identifizieren. Die Kamera wird mit den Werten erstellt, die Sie in dieser Funktion für Position, Größe und andere Details festlegen, die mit den Werten korrelieren, die Sie im Raum-Editor festgelegt haben (siehe hier). Beachten Sie, dass diese Funktion eine variable Anzahl von Argumenten annehmen kann, wobei nur die x/y-Position und Breite/Höhe arguments obligatorisch sind. Wenn arguments nicht angegeben wird, werden die Standardwerte arguments verwendet.

Beachten Sie, dass Kameras, wie alle dynamischen Ressourcen, bei ihrer Erstellung Speicherplatz belegen. Das bedeutet, dass Sie darauf achten müssen, Kameras nicht unnötig zu erstellen und nicht benötigte Kameras durch Aufruf der Funktion camera_destroy(). Das bedeutet auch, dass Kameras von Raum zu Raum bestehen bleiben, wenn sie einer global -Variablen zugewiesen werden (sie bleiben auch bestehen, wenn sie Instanz- oder lokalen Variablen zugewiesen werden, aber Sie können nicht mehr auf sie zugreifen, nachdem Sie den Raum gewechselt haben, daher das Speicherleck), so dass Sie zum Beispiel eine neue Kamera erstellen, sie einer globalen Variablen zuweisen und dann zu Beginn jedes Raums die camera_set_* -Funktionen verwenden können, um die Eigenschaften einzustellen, bevor Sie sie einem Viewport zuweisen.

Syntax:

camera_create_view(room_x, room_y, width, height, [angle], [object], [x_speed], [y_speed], [x_border], [y_border])

Argument Typ Beschreibung
room_x Real The x position (within the room) of the left side of the area the camera view captures.
room_y Real The y position (within the room) of the top of the area the camera view captures.
width Real The width of the area the view captures.
height Real The height of the area the view captures.
angle Real OPTIONAL Der Winkel, der für die Kameraansicht eingestellt werden soll (von 0° - 360°, Standard ist 0).
object Objekt Asset- oder Instanz-ID OPTIONAL Die Objekt-ID (oder Instanz-ID) für die zu verfolgende Kameraansicht (Standard ist -1, kein Objekt).
x_speed Real OPTIONAL Die maximale horizontale Geschwindigkeit, mit der sich die Ansicht bewegen kann (auf -1 gesetzt für "sofortiges" Folgen, Standard ist -1).
y_speed Real OPTIONAL Die maximale vertikale Geschwindigkeit, mit der sich die Ansicht bewegen kann (auf -1 gesetzt für "sofortiges" Folgen, Standard ist -1).
x_border Real OPTIONAL Der minimale horizontale Abstand zwischen dem Rand des Ansichtsbereichs und der Instanz, der er folgen soll (bevor die Ansicht beginnt, sich zu bewegen, und wenn sie so eingestellt ist, dass sie einer Instanz folgt, ist der Standardwert 0).
y_border Real OPTIONAL Der minimale vertikale Abstand zwischen dem Rand des Ansichtsbereichs und der Instanz, der er folgen soll (bevor die Ansicht beginnt, sich zu bewegen, und wenn sie so eingestellt ist, dass sie einer Instanz folgt, ist der Standardwert 0).

Rückgabe:

Camera ID

Beispiel:

camera_destroy(view_camera[0]);
view_camera[0] = camera_create_view(0, 0, 640, 480, 0, obj_Player, 5, 5, -1, -1);

Die obige code entfernt die aktuelle Kamera, die dem Ansichtsfenster[0] zugewiesen ist, und erstellt dann eine neue Kamera, definiert ihre Ansichtseigenschaften und weist diese dann dem Ansichtsfenster[0] zu.
