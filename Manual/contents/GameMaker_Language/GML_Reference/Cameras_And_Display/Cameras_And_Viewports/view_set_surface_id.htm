view_set_surface_id

Mit dieser Variablen können Sie den Inhalt eines Ansichtsfensters festlegen, das auf eine Oberfläche gezeichnet werden soll. Bei der Arbeit mit Oberflächen ist es oft erforderlich, den gesamten sichtbaren Bereich des Bildschirms auf die Oberfläche zu übertragen, so dass Sie ihn mit dieser Funktion einem Ansichtsfenster zuweisen. Dies bedeutet, dass alles, was in dem gewählten Ansichtsfenster angezeigt wird, nun auf die zugewiesene Oberfläche gezeichnet wird und der Inhalt des Ansichtsfensters nicht mehr angezeigt wird, was bedeutet, dass Sie entweder:

  • Aktivieren Sie eine neue Ansicht und zeichnen Sie die Oberfläche nur in dieser Ansicht (mit view_current um zu prüfen, welche Ansicht gezeichnet wird)
  • Zeichnen Sie die Oberfläche im Draw GUI- oder Post Draw-Ereignis einer Instanz, da diese Ereignisse unabhängig von Ansichten sind.

Wenn Sie diese Funktion verwenden, geben Sie den Index des Ansichtsports (von 0 bis 7) und einen Oberflächenindex (entweder den application_surface oder den eindeutigen Indexwert, der von der Funktion surface_create()zurückgegeben wird) oder, wenn eine Fläche bereits zugewiesen wurde und Sie diese entfernen wollen, den Wert -1. Weitere Beispiele für das Setzen des Ansichtsanschlusses auf eine Fläche finden Sie in der Variablen view_surface_id.

Syntax:

view_set_surface_id(view_port, surf)

Argument Typ Beschreibung
view_port Real The view port to target (0 - 7)
surf Oberfläche ID The surface to apply to the view

Rückgabe:

Real

Beispiel:

if view_get_surface_id(0) == -1
{
    view_set_surface_id(0, global.vSurf);
}

Die obige code prüft, ob dem View Port[0] eine Oberfläche zugewiesen wurde, und wenn nicht, wird eine zugewiesen.
