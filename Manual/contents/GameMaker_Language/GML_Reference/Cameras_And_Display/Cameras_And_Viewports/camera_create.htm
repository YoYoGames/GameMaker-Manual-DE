Kamera_erstellen

Mit dieser Funktion wird eine "leere" Kamera erstellt, d. h. eine Kamera ohne definierte Ansichts- oder Projektionseinstellungen. Sobald Sie die Kamera erstellt haben, können Sie die verschiedenen Matrixfunktionen verwenden, um die Ansichts- und Projektionsmatrizen einzustellen. Sie können diese Funktion auch verwenden, um die Kamera zu erstellen und dann die verschiedenen camera_set_* Funktionen zu verwenden, um die Ansicht einzurichten. Beachten Sie jedoch, dass Sie mindestens eine Position und eine Ansichtsgröße angeben müssen, da die resultierende Kamera sonst nicht gültig ist und seltsame Ergebnisse liefert, wenn sie auf einen Ansichtsanschluss angewendet wird (wenn Sie die Objektverfolgung, die Grenzen oder die Bewegungsgeschwindigkeit nicht einstellen, werden diese auf die Standardwerte für eine Ansichtskamera im Raumeditor gesetzt).

Beachten Sie, dass Kameras, wie alle dynamischen Ressourcen, bei ihrer Erstellung Speicherplatz belegen. Das bedeutet, dass Sie darauf achten müssen, Kameras nicht unnötig zu erstellen und nicht benötigte Kameras durch Aufruf der Funktion camera_destroy(). Das bedeutet auch, dass Kameras von Raum zu Raum bestehen bleiben, wenn sie einer global -Variablen zugewiesen werden (sie bleiben auch bestehen, wenn sie Instanz- oder lokalen Variablen zugewiesen werden, aber man kann nicht auf sie zugreifen, nachdem man den Raum gewechselt hat, daher das Speicherleck), so dass man zum Beispiel eine neue Kamera erstellen, sie einer globalen Variablen zuweisen und dann zu Beginn jedes Raums die erforderlichen Ansichts- und Projektionsmatrizen und den zu verwendenden Ansichtsanschluss festlegen kann.

Syntax:

camera_create();

Rückgabe:

Camera ID

Beispiel:

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

Die obige code erstellt eine neue Kamera und weist ihre ID view[0] zu. Die Ansichts- und Projektionsmatrizen werden dann erstellt und in lokalen Variablen gespeichert, bevor sie zum Einstellen der Kamera verwendet werden.
