Rollback-Fehler

Wenn Sie mit dem Rollback-System arbeiten, können bestimmte Fehler auftreten. Bei diesen Fehlern handelt es sich um Runner-Fehler, die angezeigt werden, während das Spiel läuft:

In der folgenden Tabelle sind die möglichen Fehler, die bei der Arbeit mit dem Rollback-System auftreten können, sowie eine Beschreibung aufgeführt. Sie enthält auch eine Lösung für jede spezifische Fehlermeldung.

Fehler Beschreibung
Die Ebene zum Erstellen von Spielern konnte nicht ermittelt werden. Sie sollten den Namen der Ebene an die Funktion rollback_define_player übergeben oder die Ebene Instanzen erstellen. Wenn das Player-Objekt mit der Funktion rollback_define_player definiert wurde, versucht das Rollback-System, das Player-Objekt auf der Ebene zu erstellen, deren Name optional als zweiter Parameter übergeben werden kann (oder auf der Ebene mit dem Namen "Instances", falls nicht definiert). Wenn die Ebene nicht über ihren Namen gefunden werden kann, wird dieser Fehler ausgelöst. Die Lösung besteht darin, entweder explizit den Namen der Ebene festzulegen, auf der die Player erstellt werden sollen, oder eine Ebene mit dem Namen "Instances" zu erstellen, wenn Sie den Namen der Ebene nicht explizit festlegen.
Spielerobjekte müssen verwaltet werden, wenn Multiplayer-Spiele laufen. Wenn das Player-Objekt mit der Funktion rollback_define_player definiert wurde und das Player-Objekt nicht als verwaltet gekennzeichnet ist, wird dieser Fehler ausgelöst. Die Lösung besteht darin, das Player-Objekt im Objekteditor als verwaltet zu kennzeichnen.
Die Benutzerinformationen für player_id N konnten nicht abgerufen werden. Wenn das Rollback-System nicht in der Lage war, Informationen von server über den Spieler zu erhalten, wird dieser Fehler ausgelöst. Überprüfen Sie die Entwicklerkonsole, um zu sehen, ob Anfragen blockiert wurden.
Konnte rollback_chat nicht aufrufen, bevor es vollständig mit dem Spiel verbunden war. Versuche, auf das Ereignis rollback_game_info zu warten. Die Chatfunktion ist erst verfügbar, wenn der Spieler dem Raum beigetreten ist. Lösung: Deaktivieren Sie den Chat, bis das Ereignis rollback_game_info eintrifft (siehe Rollback-Ereignisse).
rollback_use_manual_start muss vor rollback_create_game und rollback_join_game aufgerufen werden. Diese Fehler werden ausgelöst, wenn ein Funktionsaufruf gegen interne Zustandsmaschinenbeschränkungen verstößt. Die Fehlermeldung schlägt eine Lösung vor.
rollback_use_player_prefs muss vor rollback_create_game und rollback_join_game aufgerufen werden.
rollback_use_player_prefs muss vor rollback_set_player_prefs aufgerufen werden.
rollback_set_player_prefs muss vor rollback_start_game aufgerufen werden.
rollback_define_player muss vor rollback_create_game und rollback_join_game aufgerufen werden.
rollback_define_input muss vor rollback_create_game und rollback_join_game aufgerufen werden.
rollback_define_mock_input muss vor rollback_create_game und rollback_join_game aufgerufen werden.
Lesen Sie keine Spielerinformationen, bevor das Spiel begonnen hat. Aufruf von rollback_get_info().
Keine Spielereingaben lesen, bevor das Spiel begonnen hat. Aufruf von rollback_get_input().
rollback_define_input_frame_delay muss vor rollback_create_game und rollback_join_game aufgerufen werden
rollback_define_input_frame_delay muss vor rollback_create_game und rollback_join_game aufgerufen werden.
rollback_set_player_prefs konnte die Einstellungen nicht setzen Die server hat die Spielervoreinstellungen abgelehnt. Dies geschieht normalerweise in den folgenden Fällen: 1. Ein Benutzer versucht, die Spielervoreinstellungen zurückzusetzen, während der andere Benutzer das Spiel bereits gestartet hat. In diesem Fall sollte das Spiel die zuletzt erfolgreich gesetzten Benutzervorgaben verwenden, wir garantieren, dass jeder Spieler einige Benutzervorgaben setzen sollte, bevor das Spiel gestartet werden kann. 2. Die Größe der seriellen Spielerpräferenzen überschreitet die maximale Größe der Spielerpräferenzen (derzeit 800 Bytes).
rollback_create_game, Anzahl der Spieler auf N gesetzt, muss aber im Bereich 1 bis 4 liegen Die Funktion rollback_create_game wurde mit einer falschen Anzahl von Spielern aufgerufen. Die Lösung besteht darin, den ersten Parameter von rollback_create_game so zu ändern, dass er im Bereich von 1 bis 4 liegt.
rollback_create_game konnte die Standard-Spielereinstellungen nicht setzen. Siehe die Beschreibung für "rollback_set_player_prefs konnte die Einstellungen nicht setzen".
Multiplayer-Rollback wird nur im operagx-Ziel unterstützt. Die Funktion rollback_create_game wird aufgerufen und das Ziel ist nicht auf OperaGX gesetzt. Derzeit wird Multiplayer nur auf OperaGX platform unterstützt. Die Lösung ist, das Ziel auf OperaGX zu setzen.
Ein Spiel konnte nicht erstellt werden (unbekannter Sitzungstyp). Dies ist ein interner Fehler des Rollback-Systems. Melde ein Problem an das GameMaker-Team.
Es konnte keine player_id auf dieser Instanz gefunden werden, du musst sie an rollback_get_input() übergeben. Die Funktionrollback_get_input kann player id als Parameter akzeptieren. Wenn der Funktion keine Parameter übergeben wurden, wird sie das player_id-Feld eines Spielerobjekts verwenden, um Eingaben zu erhalten. Wenn der Parameter player_id nicht gesetzt ist, wird dieser Fehler ausgelöst. Wenn das Spielerobjekt mit rollback_define_player erstellt wurde, sollte die player_id gesetzt sein. Lösung: 1. Übergeben Sie den Parameter player_id explizit an die Funktion 2. Erstellen Sie den Spieler mit rollback_define_player.
Info player_id N ist außerhalb des Bereichs. rollback_get_info wird mit der falschen player_id aufgerufen (oder das Spielerobjekt hat einen falschen Wert im player_id Feld). Lösung: Der Parameter player_id muss im Bereich von 1 bis 4 liegen.
Instanz <id> von Objekt <id> (<name>), auf das in den Rollback-Daten verwiesen wird, konnte nicht gefunden werden. Interner Fehler des Rollback-Systems. Melden Sie ein Probleman das GameMaker-Team.
Keine Ahnung, wie man OBJECT_KIND=<kind> erstellt.
RefID <id> hat kein Objekt (fehlerhafte Eingabe?).
Ungültiger ReferenceKind <id> (missgestaltete Eingabe?).
Die Instanzzuordnung ist aktiviert, aber wir haben versucht, einen Index außerhalb des Bereichs zu deserialisieren. <id> liegt nicht in [0, <id>].
Ungültiger RValue bei der Deserialisierung.
Versuch, eine gelöschte Instanz wiederzubeleben!
RValue mit kind=<id> (<name>) kann nicht serialisiert werden.
Leere Eingangsdefinition. rollback_define_input wurde mit einer leeren Eingabe aufgerufen.
Die Eingabedefinition enthält die Konstante <Schlüsselcode> zweimal, was nicht zulässig ist. Eine Taste code kann nur einem einzigen Eingang zugeordnet werden. Prüfen Sie, ob ein Schlüssel code mehr als einmal in der Eingangsdefinition gesetzt ist.
Die Eingabedefinition mit der Bezeichnung <label> ist kein reeller Wert oder ein Feld von reellen Werten. Alle Werte, die Eingabewerte beschreiben, sollten den Typ Real.
Leere Mock-Input-Definition für player_id N. Siehe "Definition eines leeren Eingangs".
Mock-Definition enthält nicht-realen Wert. Siehe "Eingabedefinition mit Label <label> ist kein reeller Wert oder Array von Reals".
