spieler_id

Wenn Sie mit rollback_define_player() verwenden, um ein Spielerobjekt zu definieren, erstellt das System automatisch eine Instanz dieses Objekts für jeden Spieler. Jede Instanz dieses Objekts erhält die Variable player_id, in der die ID des Spielers gespeichert wird.

Diese ID beginnt bei 0, was zunächst der Gastgeber selbst ist. Alle Spieler ab 1 sind "Peers", die dem Spiel beigetreten sind.

Wenn Sie rollback_define_player() nicht verwenden, wird diese Variable für keine Objekte erstellt. In diesem Fall müssen die Player-Instanzen und ihre IDs manuell verwaltet werden, wie unter Definieren eines Player-Objekts erläutert.

Syntax:

player_id;

Rückgabe:

Real

Beispiel:

// Create event
if (player_id == 0)
{
    x = 300;
}
else if (player_id == 1)
{
    x = 500;
}

Die obige code ändert die X-Position der Player-Instanz auf der Grundlage ihrer ID.

Sie können diese ID auch als Array-Index verwenden, um zum Beispiel die Ergebnisse aller Spieler zu speichern:

// Step event
var _coin = instance_place(x, y, obj_coin);
if (_coin != noone)
{
    obj_game.scores[player_id]++;
    instance_destroy(_coin);
}

// Draw event
var _my_score = obj_game.scores[player_id];
draw_text(x, y, _my_score);

Das obige Step-Ereignis code erhöht den Punktestand des Spielers in einem Array in obj_game, wenn er mit einer Münze kollidiert. Diese Münzinstanz wird dann zerstört.

Das obige Draw-Ereignis code ruft den Punktestand des Spielers aus demselben Array ab und zeichnet seinen Wert.
