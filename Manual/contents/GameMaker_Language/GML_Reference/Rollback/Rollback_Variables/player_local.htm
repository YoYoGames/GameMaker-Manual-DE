spieler_lokal

Wenn Sie mit rollback_define_player() verwenden, um ein Spielerobjekt zu definieren, erstellt das System automatisch eine Instanz dieses Objekts für jeden Spieler. Jeder Instanz dieses Objekts wird die Variable player_local zugewiesen, die angibt, ob die Instanz des Spielers lokal ist oder nicht.

Wenn Sie rollback_define_player() nicht verwenden, wird diese Variable für kein Objekt erstellt.

Sie können diese Variable verwenden, um anzuzeigen, welcher Spieler lokal ist. Es wird nicht empfohlen, diese Variable in einer Weise zu verwenden, die Ihre Spiellogik beeinflusst, die für lokale und entfernte Spieler gleich sein sollte.

Syntax:

player_local;

Rückgabe:

Boolean

Beispiel:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Die obige code wird im Draw-Ereignis ausgeführt und prüft, ob die Player-Instanz lokal ist. In diesem Fall wird ein Pfeil sprite über der Instanz gezeichnet, um dem Spieler mitzuteilen, welche Instanz er gerade steuert.
