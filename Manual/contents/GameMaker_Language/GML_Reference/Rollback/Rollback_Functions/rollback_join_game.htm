rollback_join_game

Diese Funktion versucht, einem Spiel beizutreten. Sie müssen sich auf einer URL befinden, die die ID des virtuellen Raums enthält, dem Sie beitreten wollen.

Wenn einem Raum beigetreten wurde, wird true zurückgegeben, und wenn keinem Spiel beigetreten wurde, wird false zurückgegeben.

Wenn ein neuer Spieler einem Spiel beitritt, in dem Sie bereits vertreten sind, wird das Ereignis Rollback Event ausgelöst. Weitere Informationen finden Sie unter Rollback-Ereignisse.

Die Option dry_run argument ist optional und ermöglicht es, die Funktion auszuführen, ohne einem Spiel beizutreten, wenn sie auf true gesetzt ist. Damit wird geprüft, ob einem Spiel beigetreten werden kann, ohne dass man ihm tatsächlich beitritt.

Syntax:

rollback_join_game([dry_run]);

Argument Typ Beschreibung
dry_run Boolesche Disabled by default (false). Wenn die Funktion aktiviert ist(true), tritt sie dem Spiel nicht bei, sondern gibt einfach true zurück, wenn ein Spiel beigetreten werden kann, oder false, wenn nicht.

Auf diese Weise können Sie diese Funktion nur verwenden, um zu prüfen, ob der Spieler eingeladen wurde.

Rückgabe:

Boolean

Beispiel:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_create_game(2, false);
}

Die obige code definiert ein Spielerobjekt und versucht dann, einem Spiel beizutreten. Wenn er nicht beitreten konnte, wird stattdessen ein neues Spiel mit maximal 2 Spielern und deaktiviertem Sync-Test erstellt.

Ein ausführliches Beispiel finden Sie unter Erstellen eines Multiplayer-Spiels.
