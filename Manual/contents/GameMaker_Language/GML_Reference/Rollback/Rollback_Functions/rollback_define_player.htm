rollback_define_player

Diese Funktion definiert das Objekt, das für Spieler im Mehrspielermodus verwendet werden soll. Dies ist optional, und wenn es verwendet wird, erstellt das System automatisch Instanzen für Spieler, die sich verbinden, und zerstört Instanzen für Spieler, die die Verbindung abbrechen.

Standardmäßig werden dabei Spielerinstanzen in der Ebene "Instanzen" erstellt, die in Ihrem Spielraum vorhanden sein muss. Wenn Sie eine andere Ebene verwenden möchten, geben Sie deren Namen als zweiten argument an.

Informationen zum Erstellen eines Spiels mit oder ohne diese Funktion finden Sie unter Definieren eines Spielerobjekts. Falls verwendet, muss diese Funktion vor der join/create-Funktion ausgeführt werden.

Jede mit dieser Funktion erstellte Instanz erhält die auf dieser Seite aufgeführten Instanzvariablen.

Syntax:

rollback_define_player(object, [layer_name]);

Argument Typ Beschreibung
object Objekt Anlage The object that will be used for creating player instances.
layer_name Zeichenfolge OPTIONAL Der Name der Ebene, auf der die Player-Instanzen erstellt werden. Standardmäßig "Instances".

Rückgabe:

N/A

Beispiel:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

Die obige code definiert ein Spielerobjekt und versucht dann, einem Spiel beizutreten. Wenn er nicht beitreten konnte, beginnt er stattdessen ein neues Spiel mit maximal 2 Spielern und deaktiviertem Sync-Test.
