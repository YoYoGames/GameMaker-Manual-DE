rollback_leave_game

Diese Funktion verlässt die aktuelle Multiplayer-Sitzung und ermöglicht es dem Spieler, einem neuen Spiel beizutreten oder ein neues zu erstellen.

Wenn der Gastgeber eines Raumes diesen verlässt, geht der Raum in den Besitz eines anderen Spielers über, so dass das Spiel fortgesetzt werden kann. Das Spiel endet erst, wenn alle Spieler den Raum verlassen haben.

Vermeiden von Fehlern

Nachdem der Spieler eine Multiplayer-Sitzung verlassen hat, führt GameMaker weiterhin alle Spiele aus code, kehrt jedoch in den "Vorstart"-Zustand zurück, in dem Sie keine verwalteten Instanzen erstellen oder ändern können.

Das bedeutet, dass Sie Ihre Spielerinstanzen beim Verlassen eines Spiels manuell zerstören und jede andere Spiellogik mit rollback_game_running in einer Bedingung anhalten müssen, um Fehler zu vermeiden.

Siehe: Zustand vor Rollback-Start

Ausgelöstes Ereignis

Wenn ein Spieler ein Spiel verlässt, in dem Sie anwesend sind, wird das Ereignis Rollback Event ausgelöst. Siehe Rollback-Ereignisse für weitere Informationen.

Syntax:

rollback_leave_game();

Rückgabe:

N/A

Beispiel:

if (keyboard_check_pressed(vk_escape))
{
    rollback_leave_game();
}

Die obige Seite code veranlasst den Spieler, das Spiel zu verlassen, wenn die Escape-Taste gedrückt wird.
