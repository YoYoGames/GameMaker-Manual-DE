rollback_define_mock_input

Diese Funktion kann während des Sync-Tests verwendet werden, um dem angegebenen Player eine temporäre "Schein"-Eingabe zuzuweisen. Dadurch werden die für den Player definierten Steuerelemente und alle zufälligen Eingaben von Sync Test außer Kraft gesetzt.

Informationen zu seiner Verwendung finden Sie unter Mock Input.

Syntax:

rollback_define_mock_input(player_id, input_struct);

Argument Typ Beschreibung
player_id Real The ID of the player to assign mock input to.
input_struct Struktur A struct containing input names as variables, and their assigned inputs as values.

Rückgabe:

N/A

Beispiel:

rollback_define_input({
    fire: mb_left,
    interact: vk_space,
    left: ord("A"),
    right: ord("D")
});

rollback_define_mock_input(1, {
    fire: vk_control,
    interact: vk_shift,
    left: ord("J"),
    right: ord("L")
});

In der obigen code werden zunächst die Eingaben für die Spieler definiert. Dann wird eine Scheineingabe für Spieler 1 definiert, wobei dieselben Namen beibehalten, aber unterschiedliche Eingaben zugewiesen werden. Auf diese Weise können zwei Personen das Spiel lokal über die Tastatur spielen.
