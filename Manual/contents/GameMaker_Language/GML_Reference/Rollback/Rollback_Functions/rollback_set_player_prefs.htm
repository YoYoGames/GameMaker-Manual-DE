rollback_set_player_prefs

Nach dem Aktivieren der Einstellungen wird diese Funktion verwendet, um die Einstellungen für den lokalen Spieler zu setzen. Das bedeutet, dass sich die Funktion auf die lokale Eingabe und nicht auf die Rollback-Eingabe verlassen muss.

Die Präferenzen können von beliebigem Typ sein. Wenn Standardeinstellungen festgelegt sind, stellen Sie sicher, dass Sie denselben Datentyp und dieselbe Struktur verwenden.

Wenn ein Spieler seine Präferenzen setzt, erhält jeder im Spiel ein Rollback-Ereignis.

Wenn diese Funktion verwendet wird, muss sie vor der Funktion join/create ausgeführt werden.

Bitte lesen: Spieler-Einstellungen

Syntax:

rollback_set_player_prefs(preferences);

Argument Typ Beschreibung
preferences Jede The preferences to set for the local player.

Rückgabe:

N/A

Beispiel:

rollback_set_player_prefs({
Charakter: obj_customizer.character_selected,
color: obj_customizer.color_selected
});

Die obige code legt die Einstellungen für den Player fest und liest Werte aus einem "Customizer"-Objekt, das nicht verwaltet (lokal) ist.
