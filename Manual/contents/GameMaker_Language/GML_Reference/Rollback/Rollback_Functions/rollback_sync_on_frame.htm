rollback_sync_on_frame

Diese Funktion prüft, ob alle Spieler im aktuellen Frame synchronisiert sind, d. h. ihre Zustände beruhen nicht auf Vorhersagen, sondern auf den tatsächlich von ihnen empfangenen Daten. In einem solchen Fall gibt sie true zurück.

Wenn die Spieler im aktuellen Frame nicht synchronisiert sind, wird false zurückgegeben, und es wird versucht, alle Spieler zu synchronisieren.

Der Versuch, Player mit dieser Funktion zu synchronisieren, kann zu einem Einfrieren führen. Sie müssen sicherstellen, dass diese Funktion nicht ständig in einem Step-Ereignis aufgerufen wird.

Verwenden Sie sie stattdessen als Umschaltfunktion. Siehe das Beispiel unten auf der Seite.

Die Variable rollback_confirmed_frame wird automatisch aktualisiert, wenn alle Spieler synchronisiert sind.

Syntax:

rollback_sync_on_frame();

Rückgabe:

Boolean

Beispiel:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

Dieses code wird im Step-Ereignis eines Player-Objekts ausgeführt.

Es wird geprüft, ob es keinen siegreichen Spieler gibt und ob der aktuelle Spieler die für den Sieg erforderlichen Punkte hat.

Wenn beides zutrifft, bedeutet das, dass der Spieler gewinnen sollte. Zuvor wird jedoch geprüft, ob alle Spieler synchronisiert sind, um sicherzustellen, dass der Zustand korrekt ist.

Wenn alle Spieler synchronisiert sind, wird die Variable winning_player gesetzt. Dies führt dazu, dass die erste Bedingung falsch wird, wodurch sichergestellt wird, dass rollback_sync_on_frame() nicht erneut aufgerufen wird.

Das bedeutet, dass die Funktion als Umschaltfunktion verwendet wird, anstatt kontinuierlich bei jedem Bild zu laufen.
