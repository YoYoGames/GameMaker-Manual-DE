Debug_Ereignis

Diese Funktion erzeugt ein benutzerdefiniertes Debug-Ereignis, das in der Grafikansicht des Debuggers angezeigt wird, wenn ein Spiel im Debug-Modus läuft. Wenn Sie möchten, dass Meldungen angezeigt werden, wenn Sie sich nicht im Debug-Modus befinden, verwenden Sie show_debug_message().

Die Funktion nimmt auch zwei reservierte Zeichenketten an, um die Ausführung von debugging mit externen Anwendungen wie Visual Studio zu erleichtern. Diese Zeichenketten sind:

  • "OutputDebugOn" - Dies ermöglicht einen Aufruf von OutputDebugString für das Windows-Ziel, was bedeutet, dass alle Debug-Ausgaben - alles, was Sie im Ausgabefenster sehen - von Visual Studio oder von Drittanbieter-Anwendungen eingesehen werden können.
  • "OutputDebugOff" - Deaktiviert das oben beschriebene Verhalten.
  • "BreakOnError" - Diese Option ist nur für Windows YYC-Builds und bedeutet, dass Projekte nicht den üblichen code Fehlerbildschirm anzeigen, wenn runtime einen Fehler entdeckt, sondern einfach weitermachen und abstürzen. Dies ermöglicht es Ihnen, den Stacktrace innerhalb von Visual Studio zu sehen, wenn debugging.
  • "ResourceCounts" - Listet alle derzeit aktiven Ressourcen auf, wie z.B. Datenstrukturen, Zeitquellen, Oberflächen, etc.
  • "DumpMemory" - Gibt Auskunft über den aktuellen Speicherverbrauch.

Syntax:

debug_event(string)

Argument Typ Beschreibung
string Zeichenfolge The custom debug event string to use.

Rückgabe:

N/A

Beispiel:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

Die obige code prüft, ob eine Oberfläche existiert, und wenn nicht, wird ein Debug-Ereignis in der Grafikansicht des Debuggers ausgelöst (das Spiel muss im Debug-Modus gelaufen sein, damit dies sichtbar ist) und die Oberfläche wird neu erstellt.
