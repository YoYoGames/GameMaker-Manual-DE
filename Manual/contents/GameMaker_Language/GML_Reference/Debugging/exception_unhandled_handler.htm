exception_unhandled_handler

Diese runtime Funktion kann verwendet werden, um die Standard-Fehlermeldung des GameMaker zu "übernehmen", wenn eine runtime Ausnahme auftritt. Sie liefern eine Methode oder Funktion, die aufgerufen wird, wenn die Ausnahme auftritt, und ihre erste argument wird eine Struktur (bekannt als Exception-Struktur), die Informationen über die Ausnahme enthält. Diese Struktur hat das folgende Format:

Ausnahme Struktur
Variabel Typ Beschreibung
message Zeichenfolge Eine kurze Nachricht für die Ausnahme
longMessage Zeichenfolge Eine längere Nachricht für die Ausnahme
script Zeichenfolge Eine Zeichenfolge, die beschreibt, woher die Ausnahme kam
line Real Die Zeilennummer, von der die Ausnahme ausging
stacktrace Array von Zeichenfolges Ein Array von Strings, das den Aufrufstapel für den Frame darstellt, in dem die Ausnahme erzeugt wurde

Sie können auch undefined als argument angeben, wodurch der angepasste Exception-Handler (falls gesetzt) entfernt und die Standard-Fehlermeldung von GameMaker wiederhergestellt wird.

Diese Funktion gibt die zuvor eingestellte Exception-Handler-Methode zurück. Wenn sie jedoch nie geändert wurde oder zuvor mit undefined zurückgesetzt wurde, dann gibt sie einfach undefined zurück.

Die Übergabe einer Methode an diese Funktion bedeutet, dass dem Benutzer keine Fehlermeldung angezeigt wird und Sie die Möglichkeit haben, die Ausnahme selbst zu behandeln. Sie können diese Funktion jedoch nicht verwenden, um aus der Ausnahme auszubrechen und Ihr Spiel fortzusetzen, da das Spiel auch nach dem Aufruf des Ausnahmebehandlungsprogramms beendet wird. Sie können auch keine Funktionen aufrufen, die das Spiel länger als bis zum letzten Frame laufen lassen (z. B. game_restart()), da das Spiel nach dem Absturz geschlossen werden muss.

Beachten Sie, dass sich der Kontext für die Ausführung der Methode nicht in einem Ereignis befindet, so dass zu diesem Zeitpunkt nichts auf dem Bildschirm ausgegeben werden kann. Im Allgemeinen ist es am sichersten, an dieser Stelle eine Datei mit den Informationen zum Ausnahmekontext zu speichern, damit sie - zum Beispiel - beim nächsten Durchlauf des Spiels analysiert und verarbeitet werden können.

Beachten Sie, dass, wenn Sie die Anweisung return in der bereitgestellten Methode verwenden, der angegebene Wert in eine Ganzzahl umgewandelt und als Exit code von runner verwendet wird.

Syntax:

exception_unhandled_handler(user_handler);

Argument Typ Beschreibung
user_handler Methode The method (user-defined function) to use as the exception handler, or undefined um sie auf die Standardeinstellungen zurückzusetzen.

Rückgabe:

Method (or undefined)

Beispiel:

exception_unhandled_handler(function(ex)
{
    // Print some messages to the output log
    show_debug_message( "--------------------------------------------------------------");
    show_debug_message( "Unhandled exception " + string(ex) );
    show_debug_message( "--------------------------------------------------------------");

    // Write the exception struct to a file
    if file_exists("crash.txt") file_delete("crash.txt");
    var _f = file_text_open_write("crash.txt");
    file_text_write_string(_f, string(ex));
    file_text_close(_f);

    // Show the error message (for debug purposes only)
    show_message(ex.longMessage);

    return 0;
});

Die obige code erstellt eine Exception-Handler-Methode und verwendet diese, um eine Datei mit den Ausnahmedaten zu speichern; außerdem wird die lange Fehlermeldung zu Debug-Zwecken in einem neuen Popup-Fenster angezeigt.
