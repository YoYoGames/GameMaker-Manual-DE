Fehlersuche

Bei der Entwicklung eines Spiels ist es oft sehr wichtig, dass Sie bestimmte Details der Spielmechanik zu bestimmten Zeitpunkten kennen oder in der Lage sind, bestimmte Schlüsselwerte für einige Funktionen einzugeben, um verschiedene Methoden und Ideen für Ihr Spiel zu testen. Zu diesem Zweck stellt GameMaker eine Reihe von Debug-Funktionen zur Verfügung, die im Debug-Modus (oder auf einem Windows -Target) Meldungen anzeigen und nach Eingaben fragen, die aber beim Kompilieren für andere Targets modules ignoriert werden, da diese Funktionen das Spiel in ein geschlossenes loop versetzen, bis die Eingabe aufgelöst ist.

HINWEIS Obwohl sie auf bestimmten platforms funktionieren können, sollten sie nicht für andere Zwecke als debugging verwendet werden.

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

Abgesehen von diesen speziellen Funktionen, die Ihnen bei der Erstellung Ihres Spiels helfen, verfügt GameMaker auch über eine lange Liste von Fehlermeldungen, die entweder beim Schreiben Ihres Spiels code, beim Kompilieren Ihres Spiels oder beim Ausführen des Spiels selbst angezeigt werden können. Diese Meldungen weisen auf den jeweiligen Fehler hin und können verwendet werden, um ein Problem innerhalb eines bestimmten Objekts oder einer script Funktion zu lokalisieren, was bei der Arbeit an debugging sehr hilfreich ist. Es ist jedoch nicht immer offensichtlich, was genau die Fehlermeldung bedeutet, daher werden in den folgenden Abschnitten alle möglichen Meldungen aufgelistet und erklärt, was sie bedeuten, um Ihnen das Leben zu erleichtern:

  • Syntax-Fehler
  • Compiler-Fehler
  • Läufer-Fehler

Weitere Informationen zum Debuggen Ihres Spiels finden Sie auch im Abschnitt über den Debugger.
