debug_get_callstack

Diese Funktion generiert ein Array von Strings als "Callstack", in dem das aktuelle script zuerst aufgelistet ist und dann alle anderen scripts, die ausgeführt wurden, damit das aktuelle script ausgeführt werden konnte. Das genaue Format der Zeichenkette hängt von dem gewählten Ziel platform ab, aber meistens besteht es aus dem Skript-/Ereignisnamen, dann einem Doppelpunkt : und der Zeilennummer, ähnlich wie hier:

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

Die Funktion ermöglicht die Übergabe eines optionalen argument, der die maximale Tiefe des zurückgegebenen Aufrufstapels angibt. Dieser Wert ist die Anzahl der scripts, die in dem zurückgegebenen Array enthalten sind, beginnend mit dem aktuellen script. Wenn dieses Argument nicht angegeben wird, wird der vollständige Aufrufstapel zurückgegeben.

Beachten Sie, dass das zurückgegebene Array immer 0 an der letzten Position enthält, nachdem der Aufrufstapel scripts aufgelistet wurde.

Syntax:

debug_get_callstack([maxdepth])

Argument Typ Beschreibung
[maxdepth] Real OPTIONAL Die maximale Tiefe des zurückgegebenen Aufrufstapels

Rückgabe:

Array

Beispiel:

if debug_mode
{
    if (keyboard_check(vk_escape))
    {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length_id(_a); ++i;)
        {
            show_debug_message(_a[i]);
        }
    }
}

Die obige code prüft, ob der Debug-Modus aktiviert ist, und wenn ja, ob eine Taste gedrückt gehalten wird. In diesem Fall wird der aktuelle Aufrufstapel auf der Konsole ausgegeben, mit einer maximalen Tiefe von 4.
