fps

In GameMaker gibt es zwei Möglichkeiten, um die Geschwindigkeit zu bestimmen, mit der dein Spiel läuft. Die game speed (wie in den Spieloptionen angegeben) und die fps (Bilder pro Sekunde). Diese Werte werden oft verwechselt, aber im Grunde ist der eine die Anzahl der Spielschritte, die GameMaker in einer Sekunde ausführen sollte, während der andere (die fps) die Anzahl der CPU-Schritte ist, die GameMaker tatsächlich in einer Sekunde bis zu einem Maximalwert der Raumgeschwindigkeit selbst ausführt. Um die wahre fps zu erhalten, d.h. die tatsächliche Anzahl von CPU-Schritten pro Spielschritt, verwenden Sie die fps_real Variable.

Diese schreibgeschützte Variable gibt den aktuellen fps-Wert als Integer-Wert zurück. Bitte beachten Sie, dass die Funktion nur einmal pro Spielschritt aktualisiert wird und daher von einem Wert zum anderen zu "springen" scheint, aber das ist ganz normal.

Syntax:

fps

Rückgabe:

Beispiel:

if debug_mode
{
    draw_text(32, 32, "FPS = " + string(fps));
}

Die obige code prüft, ob sich das Spiel im Debug-Modus befindet, und wenn dies der Fall ist, werden die aktuellen fps auf dem Bildschirm angezeigt.
