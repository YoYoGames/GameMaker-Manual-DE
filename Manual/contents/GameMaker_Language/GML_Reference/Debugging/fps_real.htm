fps_real

In GameMaker gibt es zwei Möglichkeiten, um die Geschwindigkeit zu bestimmen, mit der dein Spiel läuft. Die game speed (wie in den Spieloptionen angegeben) und die fps (Bilder pro Sekunde). Diese Werte werden oft verwechselt, aber im Grunde ist einer die Anzahl der Spielschritte, die GameMaker in einer Sekunde ausführen soll (Spielgeschwindigkeit), während der andere die Anzahl der CPU-Schritte ist, die GameMaker tatsächlich in einer Sekunde ausführt (die tatsächliche fps), und dieser Wert ist in der Regel viel höher als die Spielgeschwindigkeit, sinkt aber, wenn Ihr Spiel komplexer wird und mehr Rechenleistung benötigt, um die eingestellte Raumgeschwindigkeit beizubehalten.

Diese schreibgeschützte Variable gibt den aktuellen fps-Wert als Integer-Wert zurück. Bitte beachten Sie, dass die Funktion nur einmal pro Spielschritt aktualisiert wird und daher von einem Wert zum anderen zu "springen" scheint, aber das ist ganz normal.

HINWEIS: Auf dem HTML5-Ziel gibt diese Variable stattdessen ein Vielfaches der Bildwiederholfrequenz des Monitors zurück, da GameMaker sich beim Timing und der Versendung auf den Browser verlassen muss.

Syntax:

fps_real

Rückgabe:

Beispiel:

if debug_mode
{
    draw_text(32, 32, "FPS = " + string(fps_real));
}

Die obige code prüft, ob sich das Spiel im Debug-Modus befindet, und wenn ja, wird die aktuelle reale Geschwindigkeit auf dem Bildschirm angezeigt.
