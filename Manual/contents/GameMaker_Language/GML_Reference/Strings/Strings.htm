Streicher

Irgendwann werden Sie bei der Entwicklung Ihres Spiels Text verwenden müssen. Der GameMaker verfügt über eine ganze Reihe von Funktionen, die es Ihnen ermöglichen, Strings auf vielfältige Weise zu manipulieren, einschließlich des Einfügens eines Strings in einen anderen, des Kopierens von Strings und der Möglichkeit, Strings nach den darin enthaltenen Ziffern oder Buchstaben zu analysieren. Im Allgemeinen kann eine Zeichenkette nur durch das Hinzufügen von Text innerhalb von doppelten Anführungszeichen " " erstellt werden. Einfache Anführungszeichen werden nicht akzeptiert, und Sie können die Zeichenkette auch nicht über mehrere Zeilen aufteilen und erwarten, dass GameMaker sie so wiedergibt, als wären die Zeilenumbrüche Zeilenumbrüche (es sei denn, es wird ein Stringliteral @ verwendet, wie weiter unten erläutert).

Es ist erwähnenswert, dass es bestimmte Konventionen gibt, die Sie bei der Erstellung von Zeichenketten verwenden können, die sich hauptsächlich auf die Verwendung von Escape-Zeichen beziehen. Dies sind Zeichen, denen ein "\" Symbol vorangestellt ist. Wenn Sie also zum Beispiel Anführungszeichen in eine Zeichenkette einfügen möchten, würden Sie etwa so vorgehen:

str = "Hello\"World\"";

GameMaker hat auch volle vier Byte breite Unicode-Zeichen-Unterstützung, so dass Sie dekodieren und kodieren Unicode-Zeichen in den oberen Grenzen des Standards (einschließlich - aber nicht beschränkt auf - Emoji). Um mit Unicode-Zeichen umzugehen, können Sie das "\" verwenden, das jedem Unicode-Literal vorangestellt wird - Hexadezimalziffern, denen ein "u" vorangestellt wird, zum Beispiel "\u00e2" für "á" - wobei die Ziffern die Nummer des Unicode-Zeichens sind. Wenn Sie auf diese Weise mit Unicode arbeiten, müssen Sie sich der Tatsache bewusst sein, dass GameMaker alle Ziffern nach dem "u" interpretiert, wenn Sie also z.B. "áa" schreiben wollen, sollten Sie verwenden:

"\u00e2\a"

oder

"\u00e2\u61"

oder

"\u00e2" + "a"

da die Verwendung von "\u00e2a" das Unicode-Zeichen "ส" ergeben würde (was im Grunde zu "\ue2a" würde).

GameMaker kann auch mit hexadezimalen Zeichen umgehen, die normalerweise als Hexadezimalziffern nach "0x" geschrieben werden, zum Beispiel "0xff", wobei die Ziffern die Nummer des zu verwendenden Zeichens sind. In GameMaker werden diese mit "\x" und dann dem Hexadezimalwert geschrieben. Diese und andere vordefinierte Escape-Zeichen sind in der folgenden Tabelle aufgelistet:

Konstante Beschreibung
\n Zeilenumbruch
\r Rückführung des Wagens
\b Rücktaste (0x08)
\f Formularvorschub (0x0c)
\t Horizontaler Reiter (0x09)
\v Vertikaler Reiter (0x0b)
\\ Backslash selbst (0x5c)
\a Warnung (0x07)
\u[Hex Digits] Unicode-Zeichen einfügen
\x[Hex Digits] Hex-Literalzeichen einfügen
\[Octal Digits] Oktales Unicode-Zeichen einfügen

HINWEIS: Strings unterstützen Formularvorschub, vertikale Tabulatoren usw., was aber nicht bedeutet, dass dies auch beim Rendern der Fall ist, und beim Zeichnen von Strings können diese Zeichen ignoriert werden.

Sie können auch wortwörtliche Zeichenfolgenliterale erstellen, indem Sie der gesamten Zeichenfolge das Zeichen @ voranstellen:

var test = @"
Line breaks
over multiple
lines
";

Die obige code rendert die Zeichenfolge über mehrere Zeilen, als ob ein Zeilenumbruch-Escape-Zeichen enthalten wäre. Ein wortgetreues String-Literal ähnelt den String-Literalen der Vorgängerversion GameMaker, aber auch sie verwenden doppelte oder einfache Anführungszeichen und müssen mit einem @ -Symbol versehen werden. Sie können in der Datei code über mehrere Zeilen umgebrochen werden und unterstützen KEINE Escape-Zeichen, d. h. @"Hello\World" versucht nicht, das W bei World zu escapen und wird wortgetreu gespeichert. Beachten Sie jedoch, dass Sie bei der Verwendung von String-Literalen wie diesem den String umbrechen müssen, wenn Sie Anführungszeichen als Teil des Strings einschließen möchten, d. h:

var test = @"Hello " + "\"" + @"World" + "\""

Zu beachten ist auch, dass das Unicode-Zeichen 9647 (▯) verwendet wird, um fehlende Glyphen zu ersetzen, die in der gewählten Schriftart vorhanden sind, wenn diese im Zeichenereignis gerendert wird. Wenn Ihre Schriftart also zum Beispiel das °-Symbol nicht enthält, wird beim Schreiben von 90° tatsächlich 90▯ erzeugt.
