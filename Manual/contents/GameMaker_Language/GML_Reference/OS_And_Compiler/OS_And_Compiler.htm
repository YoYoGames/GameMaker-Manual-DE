Betriebssystem und Compiler

Bei der Entwicklung von plattformübergreifenden Spielen ist es oft von entscheidender Bedeutung, die Details des Betriebssystems zu erfahren, das das Gerät, auf dem das Spiel läuft, verwendet. Dinge wie die Sprache, die Version oder der Netzwerkstatus können verwendet werden, um Ihr Spiel an das System anzupassen, auf dem es läuft, und die Erfahrung des Endbenutzers so gut wie möglich zu gestalten. GameMaker verfügt daher über eine Reihe spezieller Funktionen, die verwendet werden können, um die notwendigen Informationen zu erhalten.

Die folgenden Funktionen und Variablen stehen Ihnen zur Verfügung, um bestimmte Informationen über das Betriebssystem oder den Browser zu erhalten, auf dem Ihr Spiel läuft:

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Bestimmte Funktionen werden vorkompiliert, wenn Sie Ihr Spiel ausführen (d.h. sie werden verwendet, um zu definieren, wie das endgültige Spiel kompiliert wird), und einige werden auch verwendet, wenn das Spiel kompiliert wurde, um spezifische Details über die runtime Umgebung zu erhalten. Hier finden Sie die vollständige Liste dieser Funktionen:

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • GM_project_filename
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

Auf Windows ist die Auflösung des Windows Thread-Schedulers standardmäßig auf 1ms eingestellt, wenn Ihr Spiel läuft. Die folgenden Funktionen sind verfügbar, um die Auflösung des Windows Thread-Schedulers zur Laufzeit abzurufen und zu ändern:

  • scheduler_resolution_get
  • scheduler_resolution_set

Schließlich kann es bei der Arbeit mit Erweiterungen erforderlich sein, dass Sie externe Funktionen unter runtime definieren und aufrufen können:

HINWEIS: Dies sind Legacy-Funktionen, und Sie sollten alle Erweiterungsfunktionen im Erweiterungseditor selbst definieren.

HINWEIS: Diese Funktionen gelten nur für die Zielsysteme Windows und macOS platforms.

  • external_define
  • external_call
  • external_free
