gml_freigabe_mode

Wenn Ihr Spiel kompiliert wird, enthält es auch bestimmte interne Funktionen zur Fehlerprüfung während der Ausführung. Diese internen Überprüfungen decken viele Aspekte eines Spiels code ab, darunter Bereiche, Parameter und einige allgemeine Aspekte der internen GML compiler und sind die in vielen Fällen die endgültigen Fehlermeldungen hervorbringen. Diese Funktion kann jedoch verwendet werden, um switch diese internen Prüfungen auszuschalten (d.h. sie vollständig aus dem kompilierten Spiel zu entfernen), was einen Geschwindigkeitsschub für Ihr endgültiges Projekt bedeutet.

Bitte beachten Sie, dass dies bedeutet, dass das kompilierte Spiel abstürzen oder ein sehr unerwartetes Verhalten zeigen kann, wenn Sie in Ihrem Spiel code etwas tun, das nicht korrekt ist. Wenn zum Beispiel auf eine DS Grid an einer Koordinate außerhalb der definierten Größe zugegriffen wird, würden Sie im Entwicklermodus eine compiler Fenstermeldung erhalten, aber die vorhandenen Prüfungen würden es ermöglichen, dass das Spiel weiterläuft. Im Freigabemodus gibt es diese Überprüfungen jedoch nicht, und so könnte die code entweder einen NULL-Zeiger oder einen fehlerhaften Zeiger zurückgeben, der wiederum auf einen beliebigen Speicher verweist. Im besten Fall werden also die falschen Daten zurückgegeben, im schlimmsten Fall kann etwas im Speicher überschrieben werden - Daten, Grafiken oder sogar code selbst. Ein weiteres Beispiel wäre die Erstellung von Primitiven - wenn Sie die Daten für Farbe, Position, Textur usw. in einer anderen Reihenfolge als derjenigen liefern, die Sie bei der Erstellung des Vertex-Formats festgelegt haben, kann das Spiel abstürzen oder unerwartete Ergebnisse liefern, wenn der Release-Modus eingeschaltet ist.

Syntax:

gml_release_mode(flag);

ArgumentTyp Beschreibung
flag Either true um den Freigabemodus festzulegen oder false für die allgemeine Entwicklung.

Rückgabe:

Beispiel:

gml_release_mode(true);

Das obige Beispiel code wird das Spiel so einstellen, dass es beim Kompilieren den Release-Modus verwendet.
