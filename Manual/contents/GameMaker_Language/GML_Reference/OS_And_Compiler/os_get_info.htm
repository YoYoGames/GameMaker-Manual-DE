os_get_info

Diese Funktion gibt ein DS Map mit detaillierten Informationen über das Betriebssystem, auf dem das Spiel läuft. Die genauen Informationen, die zurückgegeben werden, hängen vom Betriebssystem und dem Gerät ab. Es wird daher empfohlen, diese Funktion auf allen platforms zu testen, die Sie ansprechen möchten, und die zurückgegebenen Werte mit einer Funktion wie show_debug_message() auszugeben. Beachten Sie, dass die DS-Map nicht automatisch aus dem Speicher gelöscht wird, und Sie sollten die ds_map_destroy() Funktion verwenden, wenn Sie sie nicht mehr benötigen.

Nachfolgend finden Sie einige Details zu den zusätzlichen Daten, die der DS-Karte speziell für die folgenden platforms zur Verfügung gestellt werden. Dabei ist zu beachten, dass die Karte auf allen Plattformen außer HTML5 den Schlüssel "is64bit " hat, der entweder true oder false lautet, je nachdem, ob die runner im 64-Bit-Modus läuft oder nicht.

Windows / UWP

Auf Windows Desktop und Windows UWP erhalten Sie einige zusätzliche Daten von DX11 mit den folgenden Tasten, die in der DS-Map verfügbar sind (nützlich für Erweiterungen):

  • udid: Eine eindeutige Kennung für die Maschine
  • video_d3d11_deviceZeiger auf das DX11-Gerät
  • video_d3d11_contextDX11: Zeiger auf den DX11-Kontext
  • video_d3d11_spawnchainZeiger auf die DX11-Swapchain
  • video_adapter_vendoridString: String mit der Vendor-ID des Adapters
  • video_adapter_deviceidString mit der Geräte-ID des Adapters
  • video_adapter_subsysidString: String mit der Subsystem-ID des Adapters
  • video_adapter_revisionString mit der Revisionsnummer des Adapters
  • video_adapter_descriptionString mit der Beschreibung des Videoadapters
  • video_adapter_dedicatedvideomemoryString mit der Größe des Videospeichers
  • video_adapter_dedicatedsystemmemoryString mit der Größe des Systemspeichers (wird vom DX11-Adapter verwendet)
  • video_adapter_sharedsystemmemoryString mit der Größe des gemeinsamen Systemspeichers

macOS

Die Funktion gibt einige begrenzte OpenGL-Informationen unter macOS zurück:

  • udid: Eine eindeutige Kennung für die Maschine
  • gl_vendor_string: GL_VENDOR von OpenGL, Beispiel - "Intel Inc."
  • gl_version_string: GL_VERSION von OpenGL, Beispiel - "2.1 INTEL-16.2.16"
  • gl_renderer_string: GL_RENDERER von OpenGL, Beispiel - "Intel(R) Iris(TM) Graphics 6100"

Android

Auf Android platform erhalten Sie einige zusätzliche Daten von verschiedenen APIs mit den folgenden Schlüsseln, die in der DS-Karte verfügbar sind:

  • android_tv: Dies gibt true zurück, wenn das Gerät als Android TV-Gerät erkannt wird, oder andernfalls false.
  • GL_VERSIONdie vom Treiber gemeldete Version von OpenGL
  • GL_VENDORDer Hersteller von OpenGL, wie vom Treiber gemeldet
  • GL_RENDERERRenderer von OpenGL, wie vom Treiber gemeldet
  • GL_EXTENSIONSdie vom Treiber gemeldeten verfügbaren Erweiterungen von OpenGL
  • GL_SHADING_LANGUAGE_VERSIONVersion von GLSL, die von diesem OpenGL-Treiber unterstützt wird
  • GL_MAX_TEXTURE_SIZE: die maximale Texturgröße, die von diesem OpenGL-Treiber unterstützt wird
  • SDK_INTSDK_INT: Wert von android.os.Build.VERSION.SDK_INT (siehe Hinweis unten)
  • RELEASEandroid.os.Build.VERSION.RELEASE: Wert von android.os.Build.VERSION.RELEASE
  • MODELandroid.os.Build.MODEL: Wert von android.os.Build.MODEL
  • DEVICEWert von android.os.Build.DEVICE
  • MANUFACTURERandroid.os.Build.MANUFACTURER: Wert von android.os.Build.MANUFACTURER
  • CPU_ABIandroid.os.Build.CPU_ABI: Wert von android.os.Build.CPU_ABI
  • CPU_ABI2android.os.Build.CPU_ABI2: Wert von android.os.Build.CPU_ABI2
  • BOOTLOADERWert von android.os.Build.BOOTLOADER
  • BOARDWert von android.os.Build.BOARD
  • VERSION: Wert von os.version aus der Methode Android System.getProperty.
  • REGIONWerte von user.region aus der Methode Android System.getProperty
  • VERSION_NAME: Wert des Versionsnamens dieses Pakets - siehe hier für weitere Informationen.
  • PHYSICAL_KEYBOARDWenn wir glauben, dass eine physische Tastatur vorhanden ist, wird der String "TRUE" verwendet. "FALSE"

HINWEIS: Informationen zu allen android.os.Build-Variablen finden Sie in der Android Developer-Dokumentation.

iOS & tvOS

Hier sind die Schlüssel, die auf iOS und tvOS zurückgegeben werden (detaillierte Informationen finden Sie in der Apple Developer Dokumentation):

  • name: Zeichenkette nach Angaben von [[UIDevice currentDevice] name]
  • systemName: Zeichenkette nach Angaben von [[UIDevice currentDevice] systemName]
  • systemVersion: Zeichenkette nach Angaben von [[UIDevice currentDevice] systemVersion]
  • model: Zeichenkette nach Angaben von [[UIDevice currentDevice] model]
  • localisedModel: Zeichenkette nach Angaben von [[UIDevice currentDevice] localizedModel]
  • uiIdiom: Zeichenkette nach Angaben von [[UIDevice currentDevice] userInterfaceIdiom]
  • platform: hw.machine string gesammelt aus sysctlbyname
  • hwModel: hw.model string gesammelt aus sysctlbyname
  • freeDiskSpace: Der freie Speicherplatz
  • totalDiskSpace: Der gesamte Festplattenplatz
  • cpuFrequency: Die Frequenz der CPU
  • busFrequency: Die Busfrequenz
  • cpuCount: Die CPU-Anzahl
  • totalMemory: Der Gesamtspeicher des Geräts
  • userMemory: Der Benutzerspeicher
  • maxSocketBufferSize: Die maximale Größe der Steckdose buffer
  • Zusätzliche Schlüssel mit OpenGL-Grafikinformationen

PlayStation 4

Informationen zu den hier erwähnten Funktionen finden Sie in der PS4-Dokumentation.

Die Funktion gibt die folgenden Tasten auf einer PlayStation 4-Konsole zurück:

  • display_safe_area_ratio: Ein Fließkommawert, wie er von sceSystemServiceGetDisplaySafeAreaInfo() zurückgegeben wird.
  • is_neo_mode: Dies ist 1, wenn das Spiel auf einer Neo (PS4 Pro) läuft und 0, wenn es auf einer normalen PS4 läuft.
  • enter_button_assign: Ein ganzzahliger Wert, wie er von sceSystemServiceParamGetInt(SCE_SYSTEM_SERVICE_PARAM_ID_ENTER_BUTTON_ASSIGN, &enterassign) zurückgegeben wird; ein Wert von 0 bedeutet, dass die Kreistaste als "Zuweisen" verwendet wird und 1 bedeutet, dass stattdessen die X-Taste verwendet wird.

PlayStation 5

Die Funktion liefert die folgenden Tasten auf einer PlayStation 5-Konsole:

  • display_safe_area_ratio: Wie bei PS4 (siehe oben).
  • display_resolution: "HD", "4K" oder "unbekannt"
  • display_dynamic_range: "HDR", "SDR" oder "unbekannt"
  • display_refresh_rate: "120Hz", "60Hz" oder "Unbekannt"

Xbox Eins

Die auf der Xbox One zurückgegebenen Informationen entsprechen größtenteils denen von Windows, die Tasten video_adapter_* und udid sind jedoch 0 (mit Ausnahme von video_adapter_description, das eine leere Zeichenfolge "" ist). Auch der Schlüssel video_d3d11_swapchain ist nicht vorhanden.

Unter platform finden Sie außerdem die folgenden Informationen:

  • is_xbox_one_x: Dies ist 1, wenn das Spiel auf einer Xbox One X läuft und 0, wenn es auf einer Basis Xbox One läuft.

Xbox Serie X/S

Die Informationen, die auf den Konsolen der Xbox Serie X/S zurückgegeben werden, sind größtenteils die gleichen wie auf der Xbox One, jedoch wurden die video_d3d11_* Tasten durch video_d3d12_* ersetzt.

Hier sind einige Tasten, die diese platform zusätzlich zu den unter Xbox One aufgeführten zurückgibt:

  • video_d3d12_cmdqueue: Zeigt auf die DX12-Befehlswarteschlange
  • video_d3d12_cmdlist: Zeigt auf die DX12-Befehlsliste
  • video_d3d12_currentrt: Zeigt auf das aktuelle DX12-Rendering-Ziel

Syntax:

os_get_info()

Rückgabe:

DS Map ID

Beispiel:

if (os_type == os_android)
{
    var _info = os_get_info();
    
    if (_info[? "android_tv"])
    {
        global.android_tv = true;
    }
}

Die obige code prüft, ob das aktuelle Betriebssystem Android ist, holt sich die OS-Informationen und liest die zurückgegebene Karte, um zu prüfen, ob das Spiel auf einem Android Fernseher läuft; in diesem Fall setzt sie die benutzerdefinierte Variable global.android_tv auf true.
