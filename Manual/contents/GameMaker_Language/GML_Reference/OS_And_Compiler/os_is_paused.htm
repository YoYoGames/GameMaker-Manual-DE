os_ist_pausiert

Wenn Sie für mobile Geräte entwickeln, müssen Sie eine Möglichkeit haben, zu überprüfen, ob das Gerät noch auf Ihr Spiel konzentriert ist oder nicht, vor allem, wenn Ihr Spiel auf einem iOS, Android oder Windows Phone Gerät läuft, kann es durch Telefonanrufe und andere Anwendungen unterbrochen werden. Mit dieser Funktion können Sie überprüfen, ob das Gerät angehalten ist oder nicht.

Es ist zu beachten, dass diese Funktion alle code, die im code -Block enthalten sind, auf Geräten ausführt, GameMaker aber unmittelbar danach angehalten wird, während das Gerät caches das Spiel als Hintergrundprozess startet. Wenn du also etwas wie das unten angegebene Beispiel code hast, wird die zu erstellende Instanz erst dann erscheinen oder eines ihrer Ereignisse ausführen , wenn das Spiel aus dem Cache-Status neu gestartet wurde. Das bedeutet, dass Sie es vorziehen könnten, dass die Funktion os_is_paused einen Alarm aufruft und dann im Alarm-Ereignis (das gesetzt wird, wenn das Gerät angehalten wird, aber nicht ausgeführt wird, bis es wieder freigegeben wird) ein Pausenmenü, einen Neustart der Musik usw. erstellt.

Wenn Sie diese Funktion auf Windows, Ubuntu (Linux), MacOS oder HTML5 platforms verwenden, wird die Funktion nur für einen Schritt auf true umgeschaltet, wenn das Spiel den Fokus verliert, und kehrt dann wieder zu false zurück, bis das Spiel wieder den Fokus erhält (und ihn wieder verliert). Dies liegt daran, dass diese platforms keine cache Anwendungen und Spiele enthalten und diese daher im Hintergrund weiterlaufen.

HINWEIS: Auf mobilen Geräten werden Töne und Musik angehalten, wenn das Betriebssystem pausiert wird. Sie werden erst dann wieder gestartet, wenn Sie sie mit der entsprechenden Funktion gezielt starten.

Syntax:

os_is_paused()

Rückgabe:

Beispiel:

if os_is_paused()
{
    if (!instance_exists(obj_PauseMenu))
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

Das obige code prüft, ob das Spiel pausiert wurde, und wenn ja, erstellt es eine Instanz des Objekts "obj_PauseMenu", falls noch keine existiert.
