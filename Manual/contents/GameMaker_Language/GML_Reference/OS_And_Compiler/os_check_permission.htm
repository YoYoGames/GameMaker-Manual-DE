os_check_permission

Mit dieser Funktion können Sie prüfen, ob der Benutzer dem Spiel eine bestimmte Berechtigung erteilt hat. Sie geben die zu prüfende Berechtigung als String im Format "android.permission.<permission>" an. Um zum Beispiel die Berechtigung RECORD_AUDIO zu prüfen, würden Sie also

os_check_permission("android.permission.RECORD_AUDIO");

Die Funktion gibt eine von drei Konstanten zurück - siehe unten - um dem Spiel mitzuteilen, wie der Status der Berechtigung ist. Weitere Informationen zu App-Berechtigungen finden Sie in der Android-Dokumentation.

WICHTIG! Diese Funktion ist nur für das Android-Ziel.

Syntax:

os_check_permission(permission)

Argument Typ Beschreibung
permission Zeichenfolge The permission to check (a string)

Rückgabe:

Permission State Constant

Erlaubnisstatus Konstante
Konstante Beschreibung
os_permission_granted Dies bedeutet, dass die Genehmigung erteilt wurde.
os_permission_denied Dies bedeutet, dass die Genehmigung nicht erteilt wurde.
os_permission_denied_dont_request Dies zeigt an, dass die Erlaubnis entweder durch die Telefoneinstellungen blockiert wurde oder dass der Benutzer die Anfrage zuvor abgelehnt und "Nicht mehr nachfragen" gewählt hat.

Beispiel:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

Das obige code prüft den Betriebssystemtyp, und wenn er Android ist, führt es eine Überprüfung der Berechtigungen durch, und wenn die Berechtigung "INTERNET" noch nicht erteilt wurde, fordert es sie an.
