netzwerk_senden_broadcast

Mit dieser Funktion können Sie die Daten von einem buffer lokal an einen Bereich von IP-Adressen senden (weitere Informationen zu buffers siehe Referenz - Puffer). Der Bereich ist auf den des Geräts beschränkt, auf dem server läuft. Wenn das Gerät also eine IP-Adresse von 92.168.11.130 hat, werden die Daten über den Bereich 92.168.11.* gesendet. Die Funktion gibt die Anzahl der gesendeten Datenbytes zurück oder eine Zahl kleiner als 0, wenn das Senden fehlgeschlagen ist.

Diese Funktion funktioniert nur, wenn sie mit UDP verwendet wird - Ihr server muss TCP sein und Ihr Client muss einen UDP-Client-Socket haben, der mit network_create_socket_ext() erstellt werden, um alle von server gesendeten Broadcasts zu empfangen.

Diese Funktion wird nicht funktionieren, wenn sie in einem Projekt verwendet wird, das auf dem HTML5-Ziel läuft.

Syntax:

network_send_broadcast(socket, port, bufferid, size);

Argument Typ Beschreibung
socket Netzwerk-Socket-ID The id of the socket to use.
port Real The port that the server will use.
bufferid Puffer-ID The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Rückgabe:

Real

Beispiel:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Die obige code schreibt den Namensstring des aktuellen server (gespeichert in "global.ServerName"), dann schreibt sie in einen binären buffer mit der ID "broadcast_buffer". Diese Daten werden dann lokal an einen Bereich von IPs (die Geräte-IP wird derzeit als Broadcast-Basisbereich angenommen) an Port 6511 gesendet.
