netzwerk_verbinden_raw

Mit dieser Funktion können Sie eine Verbindungsanfrage an server senden. Die Funktion nimmt die Socket-ID, über die die Verbindung hergestellt werden soll (siehe network_create_socket()) und erfordert die Angabe der IP-Adresse, zu der eine Verbindung aufgebaut werden soll (eine Zeichenkette), sowie des Ports, über den die Verbindung aufgebaut werden soll. Wenn die Verbindung fehlschlägt, wird ein Wert kleiner als 0 zurückgegeben. Der Unterschied zwischen dieser Funktion und network_connect() ist, dass diese Funktion eine Verbindung zu jeder beliebigen server herstellen kann und nichts mit den Rohdaten macht, was bedeutet, dass Sie die Protokolle selbst auf der Seite server implementieren müssen. Beachten Sie, dass die Funktion standardmäßig synchron ist, was bedeutet, dass Ihr Spiel zu "hängen" scheint, während die Verbindung hergestellt wird. Sie können einen Timeout-Wert für die Verbindung festlegen oder alternativ die Erstellung asynchron machen, indem Sie die Funktion network_set_config()verwenden, oder alternativ die Funktion network_connect_async_raw() stattdessen.

HINWEIS Sie können diese Funktion nicht für HTML5 verwenden. Verwenden Sie für WebSockets die Funktion Async.

Syntax:

network_connect_raw(socket, url, port);

Argument Typ Beschreibung
socket Netzwerk-Socket-ID The id of the socket to use.
url Zeichenfolge The URL or IP to connect to (a string).
port Real The port to connect to.

Rückgabe:

Real or Network Socket ID

Beispiel:

client = network_create_socket(network_socket_tcp);
network_connect_raw(client, "www.macsweeneygames.com", 6510);

Die obige code erstellt einen neuen TCP-Socket und versucht dann, über diesen eine "rohe" Verbindung zu der angegebenen URL an Port 6510 herzustellen.
