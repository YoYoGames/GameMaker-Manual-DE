netzwerk_senden_udp

Mit dieser Funktion können Sie Daten über das Netzwerk mittels UDP an eine server senden. Die Funktion nimmt die Socket-ID, über die eine Verbindung hergestellt werden soll, die URL, zu der eine Verbindung hergestellt werden soll, und den zu verwendenden Port entgegen. Dann müssen Sie die Puffer-ID angeben, die die zu sendenden Daten enthält (weitere Informationen über buffers finden Sie unter Referenz - Puffer) und schließlich die Größe (in Bytes) der Daten. UDP ist "verbindungslos", d.h. Sie stellen keine Verbindung her, sondern senden ein Paket direkt an eine IP-Adresse, und server erhält die eingehenden Daten von einer IP-Adresse und muss sie "wie sie sind" verarbeiten. Die Funktion gibt die Anzahl der gesendeten Bytes zurück, oder eine Zahl kleiner als 0, wenn das Senden fehlgeschlagen ist. Es ist erwähnenswert, dass die endgültige Größe der gesendeten Daten, die von dieser Funktion zurückgegeben wird, auch die GameMaker Header-Informationen enthält, was zusätzliche 12 Bytes sind.

Diese Funktion wird nicht funktionieren, wenn sie in einem Projekt verwendet wird, das auf dem HTML5-Ziel läuft, und HTML5-Projekte können auch kein UDP empfangen.

Syntax:

network_send_udp(socket, url, port, bufferid, size);

Argument Typ Beschreibung
socket Netzwerk-Socket-ID The id of the socket to use.
url Zeichenfolge The url or IP to connect to (a string).
port Real The port to connect to.
bufferid Puffer-ID The id of the buffer to get the data from.
size Real The size (in bytes) of the data.

Rückgabe:

Real

Beispiel:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Die obige code sendet ein UDP-Paket an die durch die URL definierte server auf dem Port 6510. Die Daten werden dem buffer entnommen, das in der Variablen "buff" indiziert ist.
