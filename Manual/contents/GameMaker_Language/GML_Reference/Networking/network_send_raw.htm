network_send_raw

Mit dieser Funktion können Sie ein "rohes" Datenpaket über das Netzwerk senden. Die Funktion benötigt die Socket-ID, über die eine Verbindung hergestellt werden soll, und dann die Buffer-ID, die die zu sendenden Daten enthält (für weitere Informationen über buffers siehe Referenz - Buffer) und schließlich die Größe (in Bytes) des Datenpakets. Die gesendeten Daten werden von GameMaker in keiner Weise formatiert und die empfangenden Geräte erhalten die Daten als Stream, was bedeutet, dass du sie selbst verarbeiten musst. Die Funktion gibt die Anzahl der gesendeten Bytes zurück, oder eine Zahl kleiner als 0, wenn das Senden fehlgeschlagen ist.

Optionen Argument

Das letzte argument ist optional und wird nur bei WebSockets verwendet. Es erlaubt Ihnen, zwischen dem Senden von Binär- oder Textdaten zu wählen. In diesem Argument kann eine der beiden Konstanten angegeben werden:

Netzwerk Sendetyp Konstante
Konstante Beschreibung
network_send_binary Senden einer binären Nachricht
network_send_text Senden einer Textnachricht

Die APIs für einige platforms akzeptieren nur Textnachrichten, wenn sie WebSockets verwenden (z. B. Twitch), daher kann in solchen Fällen die Konstante network_send_text verwendet werden. Wird diese argument nicht angegeben, werden standardmäßig binäre Daten gesendet.

Syntax:

network_send_raw(socket, bufferid, size, [options]);

Argument Typ Beschreibung
socket Netzwerk-Socket-ID The id of the socket to use.
bufferid Puffer-ID The id of the buffer to get the data from.
size Real The size (in bytes) of the data.
options Netzwerk Sendetyp Konstante OPTIONAL Wird für WebSockets verwendet, um zwischen Text- und Binärdaten zu wählen; wenn nichts angegeben wird, werden Binärdaten gesendet.

Rückgabe:

Real

Beispiel:

buff = buffer_load("player_save.dat");
network_send_raw(sock, buff, buffer_get_size(buff));

Die obigen Informationen laden eine zuvor gespeicherte buffer in den Speicher und geben die buffer id zurück, die in der Variablen "buff" gespeichert wird. Diese vollständige buffer wird dann als Rohdatenpaket über das Netzwerk unter Verwendung des Sockets, der durch die Variable "sock" identifiziert wird, gesendet.
