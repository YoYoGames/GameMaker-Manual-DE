Vernetzung

Die GameMaker-Netzwerkfunktionen basieren auf Sockets, einem Industriestandard auf allen platforms. Mit Sockets haben Sie die Möglichkeit, sowohl Clients als auch servers mit Hilfe von GML zu erstellen, was es Ihnen ermöglicht, sogar ein Einzelspielerspiel mit dem Client/Server-Modell zu erstellen (was es später ermöglicht, problemlos Multiplayer hinzuzufügen).

Im Grunde ist ein Socket ein Objekt, das senden, empfangen, eine Verbindung herstellen und Ports im Netz abhören kann. Diese Funktionen beziehen sich ausschließlich auf TCP/IP-Verbindungen, auf denen das Internet basiert (genauer gesagt auf IPv4). Im Wesentlichen können Sie damit einen Socket mit einer IP-Adresse (z. B. 192.168.1.10) mit einem anderen Socket mit einer anderen IP-Adresse (z. B. 192.168.1.11) verbinden. Sobald die beiden Sockets verbunden sind, können sie Daten hin- und herschicken, indem sie die network_send_* Funktionen (wie network_send_packet()) und des speziellen asynchronen Ereignisses, das von ihnen ausgelöst wird. Dieses asynchrone Ereignis erzeugt eine DS-Map, die die empfangenen Daten sowie weitere Details enthält. Die vollständigen Details finden Sie im Abschnitt über das asynchrone Netzwerkereignis.

Anstatt dass Programme jeden einzelnen Netzwerkverkehr, der auf einen Rechner eintrifft, lesen und verarbeiten müssen, werden bei IP-Adressen auch Ports verwendet, die von 0 bis 65535 nummeriert sind. Auf diese Weise erhält jedes Programm nur die Datenpakete, an denen es interessiert ist, und nicht alles, was von anderen Programmen gesendet wurde. Das spart auch beträchtliche CPU-Zeit, da es die Datenverarbeitung stark einschränkt, so dass man mit GameMaker auch auf Ports "hören" kann. Ein server erstellt also einen Socket und bindet ihn an einen bestimmten Port, den er abhören soll. Er wird dann über Verbindungen, Verbindungsabbrüche und Datenübertragungen von diesem Socket benachrichtigt.

Die Daten-"Pakete" selbst bestehen aus Binärdaten, die einem buffer entnommen werden. Dieses buffer kann mit den GameMaker Buffer-Funktionen erstellt und manipuliert werden.

Es ist erwähnenswert, dass Sie, wenn Sie ein Projekt für das HTML5-Ziel erstellen, nicht in der Lage sein werden, die server Erstellungsfunktionen aufgrund von Browser-Einschränkungen zu verwenden, und Sie müssen Ihre eigene server für dieses platform mit etwas wie Electron oder NodeJS schreiben. Beachten Sie auch, dass, wenn Ihr Spiel platform übergreifend ist und Sie eine Verbindung zu einer HTML5-Instanz des Spiels von einem server, das auf einem mobilen oder Desktop-Gerät läuft, vorhersehen, dann kann Ihr server nur mit der HTML5-Instanz über das Web Socket-Protokoll kommunizieren (dies wird mit der Konstante network_socket_ws oder network_socket_wss gemacht - siehe network_create_server() für weitere Informationen). Das bedeutet, dass Ihr server möglicherweise zwei Sockets auf verschiedenen Ports erstellen muss, einen für die Verbindung mit Nicht-HTML5-Zielen (Android, Windows, etc...) und einen für die Verbindung mit dem HTML5-Ziel.

  • netzwerk_erstellen_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • network_connect_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
