netzwerk_verbinden

Mit dieser Funktion können Sie eine Verbindungsanfrage an server senden. Die Funktion nimmt die Socket-ID, über die die Verbindung hergestellt werden soll (siehe network_create_socket()) und verlangt die Angabe der IP-Adresse, zu der die Verbindung aufgebaut werden soll (eine Zeichenkette), sowie des Ports, über den die Verbindung aufgebaut werden soll. Wenn die Verbindung fehlschlägt, wird ein Wert kleiner als 0 zurückgegeben. Die Verbindung verwendet ein spezielles Protokoll, das sicherstellt, dass nur GameMaker-Spiele miteinander verbunden werden. Wenn Sie jedoch eine Verbindung zu einem server herstellen müssen, das kein GameMaker-Spiel ist, können Sie network_connect_raw(). Beachten Sie, dass die Funktion standardmäßig synchron ist, was bedeutet, dass Ihr Spiel während des Verbindungsaufbaus zu "hängen" scheint. Sie können einen Timeout-Wert für die Verbindung festlegen oder alternativ die Erstellung asynchron machen, indem Sie die Funktion network_set_config() verwenden. Alternativ dazu können Sie auch die Funktion network_connect_async() verwenden.

HINWEIS Sie können diese Funktion nicht für HTML5 verwenden. Verwenden Sie für WebSockets die Funktion Async.

Syntax:

network_connect(socket, url, port);

Argument Typ Beschreibung
socket Netzwerk-Socket-ID The id of the socket to use.
url Zeichenfolge The URL or IP to connect to (a string).
port Real The port to connect to.

Rückgabe:

Real or Network Socket ID

Beispiel:

client = network_create_socket(network_socket_tcp);
network_connect(client, "192.134.0.1", 6510);

Die obige code erstellt einen neuen TCP-Socket und versucht dann, über diesen eine Verbindung mit der angegebenen IP-Adresse an Port 6510 herzustellen.
