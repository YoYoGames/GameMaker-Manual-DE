game_save_buffer

Dies ist eine Variante der Funktion game_save(). Bitte lesen Sie zuerst die entsprechende Seite, da sie wichtige Informationen zu ihrer Verwendung und auch zu dieser Funktion enthält.

Mit dieser Funktion können Sie den aktuellen Zustand des Spiels in einem zuvor erstellten "grow" buffer (siehe Puffer) speichern, der dann jederzeit wieder mit game_load_buffer() Funktion. Diese Funktion ist für die Verwendung in jeweils einem einzelnen Raum gedacht (d.h. es ist nicht ratsam, buffer in einem Raum zu speichern und dann zu versuchen, buffer aus einem anderen Raum zu laden) und sollte idealerweise nur für Checkpoints oder Level-Neustarts verwendet werden.

HINWEIS: Diese Funktion ist sehr begrenzt und für Anfänger gedacht, um ein Kontrollpunktsystem schnell zum Laufen zu bringen. Fortgeschrittene Benutzer ziehen es vielleicht vor, ihr eigenes System mit Hilfe der Dateifunktionen auf code zu speichern, da das Spiel keine dynamischen Ressourcen wie Datenstrukturen, Oberflächen, hinzugefügte sprites usw. speichern wird.

Syntax:

game_save_buffer(buffer);

ArgumentTyp Beschreibung
buffer The buffer id to save to.

Rückgabe:

Beispiel:

if keyboard_check_pressed(ord("S"))
{
    global.Checkpoint = true;
    game_save_buffer(save_buff);
}

Die obige code setzt eine globale Variable auf true und speichert dann den aktuellen Spielzustand in der Variable "save_buff" auf buffer, wenn die Taste "S" gedrückt wird.
