Spiel_sichern

Dies ist eine alte Funktion, die zum Speichern des aktuellen Spielzustands verwendet werden kann und nicht mehr empfohlen wird. Verwenden Sie stattdessen die Dateifunktionen, um ein benutzerdefiniertes Speichersystem zu erstellen, bei dem Sie nur bestimmte Spieldaten speichern und laden.

Diese Funktion speichert den aktuellen Zustand des Spiels, jedoch nicht die dynamischen Ressourcen, die zu diesem Zeitpunkt verwendet werden, wie z. B. Datenstrukturen, Oberflächen, Assets, die auf runtime hinzugefügt wurden, usw. Wenn eine solche Speicherdatei geladen wird, sind alle Spielaktualisierungen, die nach der Speicherung vorgenommen wurden, nicht sichtbar, da die Version des Spiels wiederhergestellt wird, die zur Erstellung der Speicherung verwendet wurde.

Beachten Sie, dass Speicherdateien, die mit dieser Funktion erstellt wurden, von aktualisierten Runtime-Versionen möglicherweise nicht unterstützt werden. Daher ist es unerlässlich, diese Funktion durch ein neues Speichersystem zu ersetzen, um die Kompatibilität mit zukünftigen Updates von GameMaker und aus den oben genannten Gründen auch mit Updates für Ihr eigenes Spiel zu gewährleisten.

Syntax:

game_save(filename);

ArgumentTyp Beschreibung
filename The name of the file to save the game to.

Rückgabe:

Beispiel:

if keyboard_check_pressed(ord("S"))
{
    global.Saved = true;
    game_save("Save.dat");
}

Die obige code setzt eine globale Variable auf true und speichert dann das Spiel in der Datei "Save.dat", wenn die Taste "S" gedrückt wird.
