Web und HTML5

GameMaker stellt verschiedene Funktionen für die Interaktion mit dem Internet oder mit server zur Verfügung, die auf allen verfügbaren Zielsystemen platforms genutzt werden können. Diese Funktionen sind:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

Abgesehen von den oben aufgeführten allgemeinen Funktionen verfügt das HTML5-Ziel über einige zusätzliche Webfunktionen, die speziell für die Spiele entwickelt wurden, die auf dieses Ziel ausgerichtet sind:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Neben diesen Funktionen für das HTML5-Ziel module gibt es auch eine spezielle JavaScript-Variable, die in den seltenen Fällen verwendet werden kann, wenn der Inhalt des html5game-Ordners an einem völlig anderen Ort (URL) gespeichert ist. Normalerweise ist dieser Pfad in der JavaScript-Datei fest codiert und relativ zu dem Ort, von dem aus das JavaScript aufgerufen wird, aber Sie können die Variable g_GameMakerHTML5Dir vor dem Aufruf des Spiels script setzen, und das Spiel wird von dem neu definierten Ort aus gestartet. Zum Beispiel:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Es ist auch erwähnenswert, dass Sie script Funktionen mit dem Präfix gmcallback_ kennzeichnen können, was bedeutet, dass sie nicht verschleiert werden, wenn Sie das Projekt für HTML5 kompilieren. Dies ist sehr nützlich, wenn Sie die oben aufgeführten clickable_* Funktionen verwenden (weitere Informationen finden Sie hier).
