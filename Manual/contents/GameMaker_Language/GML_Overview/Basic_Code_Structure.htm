Grundlegende Struktur Code

Code wird in Blöcken geschrieben, und ein typischer Codeblock besteht aus einer Reihe von Anweisungen, sogenanntenStatements, die dann von GameMaker interpretiert und verwendet werden, um etwas in Ihrem Spiel zu bewirken. Dieses "Etwas" kann so einfach sein wie die Addition von 2 und 2, um 4 zu erhalten, oder so komplex wie das Weglaufen eines Feindes, wenn seine Gesundheit unter einen bestimmten Wert fällt. Die tatsächliche Struktur des Programms kann stark variieren, je nachdem, welche Funktionen es verwendet, aber auf die Grundlagen heruntergebrochen sieht es einfach so aus:

<statement>;
<statement>;
...

Anweisungen sollten mit einem ';'-Symbol getrennt werden, um Fehler bei der Variablendeklaration zu vermeiden und Ihr code sauber und aufgeräumt zu halten. Sie können Anweisungen auch als Block "gruppieren", indem Sie die geschweiften Klammern {} verwenden, so dass sie zusammen ablaufen, wie in dem folgenden bedingten Beispiel:

if (<conditional>)
{
    <statement>;
    <statement>;
    ...
}

HINWEIS: GameMaker Language akzeptiert auch begin und end anstelle der geschweiften Klammern {}, obwohl dies in der Regel nicht die gebräuchlichste Art ist, dies zu tun:

if (<conditional>)
begin
    <statement>;
    <statement>;
    ...
end
