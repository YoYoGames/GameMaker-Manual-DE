Laufzeitfunktionen

Die allgemeine Definition einer Funktion ist etwa so:

Eine Funktion hat eine Eingabe und eine Ausgabe, und die Ausgabe ist mit der Eingabe verbunden.

In GameMaker bedeutet das zwei verschiedene Dinge, die aber beide auf die gleiche Weise funktionieren:

  • Laufzeitfunktionen - Eine Laufzeitfunktion ist eine Funktion, die als Teil der GameMaker-Sprache geliefert wird (GML). Sie sind in die Sprache integriert und können verwendet werden, um Dinge in Ihren Spielen auszuführen. Unter GML runtime stehen Ihnen zahlreiche Funktionen zur Verfügung, die alle in der Sprachreferenz des Handbuchs erläutert werden.
  • Skript-Funktionen und Methoden - Skript Funktionen und Methoden-Variablen sind Funktionen, die Sie selbst aus den Grundbausteinen der GameMaker Language (GML) erstellt haben und die als Teil ihrer runtime Funktionen code enthalten können.

In beiden Fällen hat eine Funktion die Form eines Funktionsnamens, gefolgt von den Eingabeargumenten zwischen eckigen Klammern () und getrennt durch Kommas (wenn die Funktion keine Eingaben hat arguments, werden nur eckige Klammern verwendet). Diese Seite konzentriert sich auf die Laufzeitfunktionen (d.h. die Funktionen, die in GameMaker eingebaut sind, um die GameMaker Language zu bilden), aber die unten aufgeführten allgemeinen Regeln gelten für alle Funktionstypen.

Hier ist ein Überblick über die Struktur einer Funktion:

<function>(<arg0>, <arg1> ,... <argN>);

Einige Funktionen können Werte zurückgeben und in Ausdrückenverwendet werden, während andere einfach nur Befehle ausführen, wie in den beiden folgenden runtime Funktionsbeispielen dargestellt:

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

HINWEIS Überall, wo im Handbuch "N/A" als Rückgabewert für eine Funktion angegeben ist, bedeutet dies, dass die Funktion keinen Wert als Teil ihrer Operation zurückgeben soll. Wenn Sie versuchen, einen Rückgabewert für diese Funktionen zu ermitteln, erhalten Sie den Wert undefined.

Manchmal kann der Rückgabewert einer Funktion ein Wert sein, den Sie in einer Zuweisung verwenden möchten, aber Sie sollten beachten, dass es unmöglich ist, eine Funktion direkt als linke Seite einer Zuweisung zu verwenden. Zum Beispiel würde die folgende code, um die Geschwindigkeit einer Instanz einzustellen, zu einem Fehler führen:

instance_nearest(x, y, obj).speed = 0;

Der Rückgabewert für den Ausdruck in diesem code Beispiel ist eine ganze Zahl (der eindeutige ID-Wert für die nächste Instanz) und muss daher in Klammern eingeschlossen werden, um auf diese Weise verwendet zu werden und die erforderliche Instanz korrekt zu adressieren (weitere Informationen finden Sie unter Adressierung von Variablen in anderen Instanzen ). Die obige code würde korrekt geschrieben werden als:

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

function test ()
{
    test();

    if (test)
    {
        test();
    }
}

Der Abschnitt " Sprachreferenz" des Handbuchs listet alle runtime Funktionen auf, die Ihnen zur Verfügung stehen, und gibt Beispiele für ihre Verwendung sowie weitere wichtige Informationen darüber, was sie zurückgeben oder welche Ereignisse sie auslösen können usw.. Weitere Informationen über benutzerdefinierte script Funktionen und Methoden finden Sie im Abschnitt über Skriptfunktionen und im Abschnitt über Methodenvariablen.
