Ausgang

Die Anweisung exit hat die folgende Syntax:

exit;

exit beendet einfach die Ausführung der aktuellen Skriptfunktion, method, oder event. Beachten Sie, dass es hier je nach Anwendungsbereich einen leichten Unterschied in der Verwendung gibt:

  • Wenn Sie exit in einer Funktion oder Methode script verwenden, wird die Funktion einfach verlassen und zu code zurückkehren, das die Funktion aufgerufen hat.
  • Wenn Sie exit in einem code -Block innerhalb des Ereignisses eines Objekts verwenden, wird das gesamte Ereignis beendet, auch wenn es verschiedene separate code -Blöcke gibt, nachdem exit aufgerufen wurde.
  • Wenn Sie exit in einem übergeordneten Ereignis verwenden und dieses Ereignis mit event_inherited() in einem untergeordneten Objekt aufrufen, wird nur das übergeordnete Ereignis beendet und das Ereignis des untergeordneten Objekts (das event_inherited() aufgerufen hat) wird fortgesetzt.

Bei Verwendung in einem Ereignis wird exit in der Regel verwendet, um zu verhindern, dass eine Instanz weitere code ausführt, wenn eine bestimmte Bedingung erfüllt ist (oder nicht). Die folgende Seite code zeigt ein Beispiel für die Verwendung, in diesem Fall innerhalb eines Kollisionsereignisses, obwohl es in jedem Ereignis verwendet werden kann.

if (!visible)
{
    exit;
}

other.hp -= attack;
other.coins -= 4;
coins += 4;

Die obige code prüft, ob die aktuelle Instanz nicht sichtbar ist, in diesem Fall verlässt sie den code Block, andernfalls geht sie weiter und führt den Rest des code aus.

HINWEIS: Sie beendet nicht die Ausführung des Spiels. Hierfür müssen Sie die Funktion game_end().
