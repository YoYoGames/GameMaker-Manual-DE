Schalter

In einer Reihe von Situationen möchten Sie Ihre Instanzen in Abhängigkeit von einem bestimmten Wert unterschiedliche Aktionen ausführen lassen. Sie können dies mit einer Reihe von aufeinanderfolgenden if / else Anweisungen tun, aber wenn die Zahl der möglichen Optionen zwei oder drei übersteigt, ist es in der Regel einfacher, die Anweisung switch zu verwenden.

Eine switch -Anweisung hat folgende Syntax:

switch (<expression>)
{
    case <constant1>:
        <code>
    break;

    case <constant2>:
        <code>
    break;
    
    // more cases (with breaks)

    default:
        <code>;
}

Dies funktioniert folgendermaßen:

  • Zuerst wird der Ausdruck ausgeführt.
  • Anschließend wird das Ergebnis mit den verschiedenen Konstanten nach jeder der case Anweisungen verglichen. Mit "Konstante" ist gemeint, dass der Wert in dem Fall kein variabler Ausdruck sein kann, sondern ein fester Wert eines beliebigen Datentyps, wie "fight" oder 3 oder das Schlüsselwort noone.
  • Die Ausführung beginnt bei der ersten case Anweisung mit dem passenden Wert, bis eine break Anweisung auftaucht.
  • Wenn keine case -Anweisung den passenden Wert enthält, wird die default -Anweisung ausgeführt. Es ist nicht erforderlich, eine default -Anweisung zu haben, und wenn keine angegeben ist, wird keine Aktion ausgeführt, wenn es keine passenden Fälle gibt. Die Anweisung default kann an beliebiger Stelle in einem switch -Block stehen, wird aber üblicherweise am Ende, nach allen Fällen, platziert.

HINWEIS Die Anweisung switch führt code innerhalb eines case so lange aus, bis ein break auftaucht. Wenn Sie am Ende Ihrer Fälle keine breaks verwenden, führt dies dazu, dass die switch zum nächsten case und sogar zu einem default Abschnitt "durchsickert", wenn keine breaks im Weg sind. Dies kann zu unbeabsichtigtem Verhalten führen, da die Ausführung eines case mehrere cases zur Folge haben kann. Stellen Sie daher sicher, dass Sie break verwenden, wo es angebracht ist.

Ein einfaches Beispiel für die Verwendung einer switch -Anweisung wäre etwa so:

switch (player_lives)
{
    case 3:
        draw_sprite(20, 20, spr_face_healthy);
    break;

    case 2:
        draw_sprite(20, 20, spr_face_hurt);
    break;

    case 1:
        draw_sprite(20, 20, spr_face_fatal);
    break;

    default:
        draw_sprite(20, 20, spr_face_fainted);
    break;
}

Beachten Sie, dass mehrere case Anweisungen verwendet werden können, um dieselbe Anweisung auszuführen, da die break nicht immer für jede einzelne case erforderlich ist. Wenn es für eine bestimmte case keine break Anweisung gibt, wird die Ausführung einfach mit der code für den nächsten Fall fortgesetzt, z.B.:

switch (keyboard_key)
{
    case vk_left:
    case ord("A"):
        x -= 4;
    break;

    case vk_right:
    case ord("D"):
        x += 4;
    break;

    case vk_up:
    case ord("W"):
        y -= 4;
    break;

    case vk_down:
    case ord("S"):
        y += 4;
    break;
}

Das obige code verwendet switch, um zu prüfen, ob ein Tastaturereignis vorliegt, und vergleicht dieses dann mit den einzelnen aufgelisteten case. Wenn einer der erforderlichen Werte erfüllt ist, wird die entsprechende code ausgeführt. Beachten Sie, dass switch mehrere cases überprüfen und code bis zum nächsten break ausführen kann, damit verschiedene Tasten verwendet werden können, um dasselbe Ergebnis zu erhalten.

Jedes case kann seine eigene code haben, so dass Sie eine Art "Vererbungssystem" einrichten können, bei dem ein Fall seine eigene code und dann auch die code für den nächsten Fall ausführt.
