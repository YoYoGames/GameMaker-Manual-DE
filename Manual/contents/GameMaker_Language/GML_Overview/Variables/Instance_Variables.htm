Instanz-Variablen

Eine Instanzvariable wird innerhalb einer Instanz eines Objekts erstellt und gilt als eindeutig für diese Instanz - d. h., viele Instanzen desselben Objekts können dieselbe Variable haben, aber jede Variable kann einen anderen Wert enthalten, da sie eindeutig für jede Instanz. Aber wie wird eine Instanzvariable erstellt? Nun, Sie erstellen neue Variablen, indem Sie ihnen einfach einen Wert zuweisen, wie in diesem kleinen Beispiel gezeigt (dies wird als Deklaration der Variable bezeichnet):

potions = 12;
life = 100;
name = "Jock MacSweeney";
strength = 5.5;
armour = -2;

Wie Sie sehen, müssen Sie nur den Namen und einen Wert angeben (der Wert kann ein beliebiger Datentyp sein), um die Variable zu setzen und sie innerhalb einer Instanz des Objekts, für das Sie kodieren, zu verwenden (beachten Sie, dass der Wert aus dem Rückgabewert einer Funktion oder dem Ergebnis einer Operation zwischen anderen Variablen usw. stammen kann...). Diese Variablen können dann auf verschiedene Weise innerhalb der Instanz verwendet und verändert werden, zum Beispiel könnte diese code in einem Kollisionsereignis sein und verwendet werden, um einen Betrag von der Variablen "life" abzuziehen:

life -= 5 + armour;

Wenn "life" auf 100 steht, hat es jetzt einen Wert von 97 (100 - (5 + -2) = 97). Nun, das ist ein einfaches Beispiel, und Sie könnten "armour" durch den tatsächlichen Wert von -2 ersetzen, aber was passiert, wenn dieser Wert an mehreren Stellen fest kodiert ist und Sie ihn dann ändern wollen? Sie müssten ALLE Ihre code durchgehen und jede -2 durch den neuen Wert ändern, was zeitaufwendig und sehr fehleranfällig ist! Wenn Sie jedoch eine Variable verwenden, brauchen Sie ihr nur einen neuen Wert zuzuweisen, und code verwendet von da an automatisch diesen neuen Wert, was die Sache sehr viel flexibler und einfacher zu beheben macht sollte es ein Problem geben. Auch wenn sich ein Wert nicht ändert, ist es viel einfacher, sich zu merken, was eine Variable mit dem Namen "life" bedeutet, als nur eine Zahl zu sehen.

GameMaker hat auch eine Sammlung von "eingebauten" Instanzvariablen, die Sie kennen sollten, da Sie vielleicht eine Ihrer eigenen Instanzvariablen mit demselben Namen benennen oder eine eigene globale Variable mit demselben Namen haben möchten und sich wundern, warum Sie Fehler erhalten. Sie sind jedoch leicht zu erkennen, da sie im code -Editor in einer anderen Farbe angezeigt werden und auch bei der automatischen Vervollständigung auftauchen und in der Leiste am unteren Rand des Code-Editors angezeigt werden.
