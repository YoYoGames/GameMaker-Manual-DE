async_load

Diese Variable ist global und wird verwendet, um eine DS-Map zu speichern, wenn sie in asynchronen Ereignissen verwendet wird, und -1 zu allen anderen Zeiten. Der tatsächliche Inhalt der DS-Map hängt von der Art des asynchronen Ereignis-Callbacks sowie von der Funktion ab, die zum Auslösen des Ereignisses verwendet wurde, daher finden Sie auf den einzelnen Seiten für diese Ereignisse vollständige Angaben zu allen möglichen DS-Map-Inhalten.

Syntax:

async_load

Rückgabe:

DS Map ID

Erweitertes Beispiel:

sprite =  sprite_add("site.com/path/image.png", 16, true, true, 0, 0);

Die obige code würde in einem Ereignis aufgerufen werden, um ein sprite von einer externen URL zu laden. Dies würde dann das asynchrone Ereignis "Bild geladen" auslösen, bei dem Sie die Karte async_load analysieren würden:

if ds_map_find_value(async_load, "id") == sprite
{
    if (ds_map_find_value(async_load, "status") >= 0)
    {
        sprite_index = sprite;
    }
}

Die obige code prüft zunächst die ID der Karte async_load und dann den Status des Callbacks. Wenn der Wert größer oder gleich 0 ist (was einen Erfolg signalisiert), wird das Ergebnis des Callbacks verwendet, um den sprite Index der Instanz auf das neu geladene Bild zu setzen.
