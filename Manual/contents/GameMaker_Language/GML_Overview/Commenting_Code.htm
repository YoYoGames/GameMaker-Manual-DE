Kommentierender Code

Bei der Zusammenarbeit mit anderen Personen, bei der Bearbeitung eines großen Projekts oder auch nur zu Ihrer eigenen Information und debugging ist das Hinterlassen von Kommentaren in Ihren Ereignissen code und scripts sehr wichtig. In GameMaker gibt es verschiedene Mechanismen, die Ihnen helfen, Notizen und Kommentare zu code Abschnitten und sogar ganzen Blöcken von code zu hinterlassen, die in der Ereignisliste des Objekts, das Sie gerade bearbeiten, zu sehen sind. Auf diese Weise können Sie Notizen für sich selbst und für Ihre Kollegen hinterlassen, oder um ein besonders kniffliges Stück code zu erklären, oder auch einfach nur, um sich daran zu erinnern, was etwas tut, oder was Sie in Zukunft tun wollen.

Um einen einfachen einzeiligen Kommentar zu hinterlassen, können Sie als erstes // vor dem Text verwenden. Zum Beispiel:

//initialize variables
sr = 10;
hp = 100;

Sie können auch mehrzeilige Kommentare hinterlassen, um Lob auszusprechen, einen kompletten Abschnitt von code für debugging auszulassen oder sogar Anmerkungen dazu zu machen, wie Sie code in Zukunft ändern wollen. Hierfür können Sie /* ... */ wie dieses:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

Abgesehen von Kommentaren können Sie auch spezielle #region -Tags verwenden, um einen Abschnitt von code für Code Folding zu markieren, und jeder Bereich kann einen Kommentar enthalten. Das funktioniert so: Sie fügen ein #region Tag - zusammen mit einem beliebigen Kommentartext - am oberen Ende eines Abschnitts von code ein und fügen dann ein entsprechendes #endregion Tag am unteren Ende des Abschnitts ein. Dann können Sie den code"falten", d.h. ausblenden, so dass Sie die Anzahl der im Editor gleichzeitig sichtbaren Zeilen von code verringern können:

Die Website code sieht in etwa so aus:

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

Abschließend sei noch erwähnt, dass Sie bei der Arbeit mit Ereignissen im Objekteditor einen speziellen Kommentar "description" am Anfang des Ereignisses verwenden können, um den allgemeinen Inhalt des Ereignisses anzugeben, der dann im Ereignisfenster des Objekteditors angezeigt wird. Dieser Kommentar sollte in der allerersten Zeile des Editors code stehen und ist wie folgt formatiert:

/// @description <This Is An Event Description>

Dies wird nun im Ereignisfenster angezeigt, z. B:

Beachten Sie, dass Sie in den Voreinstellungen des Objekteditors einen Standardkommentar festlegen können, der für alle Ereignisse erstellt wird.

Wenn Sie Skriptfunktionen schreiben und eine gewisse Kontrolle darüber haben möchten, wie sie von IDE behandelt werden (z. B. automatische Vervollständigung oder Anzeige der script Parameter), sollten Sie Ihre Kommentare in JSDoc-Notation schreiben, wie hier erklärt.
