Arbeitsbereiche

Nachdem Sie sich angemeldet und ein neues Projekt gestartet haben, zeigt Ihnen GameMaker den ersten Arbeitsbereich mit einigen grundlegenden Fenstern, die an die IDE angedockt sind. Im Allgemeinen ist die workspace einfach ein Bereich, in dem Sie die verschiedenen Assets für Ihr Spiel organisieren können, während Sie arbeiten:

Wie Sie sehen können, befindet sich die anfängliche workspace auf einer Registerkarte am oberen Rand des Bildschirms (und Sie können sie umbenennen, indem Sie auf die Registerkarte doppelklicken), aber Sie können weitere workspaces für das Projekt erstellen, indem Sie auf die an der Seite klicken, wodurch Sie mehrere mögliche workspaces für jedes einzelne Projekt erhalten. Wenn Sie z. B. an der Interaktion zwischen dem Spieler und mehreren feindlichen Objekten arbeiten, haben Sie den Spieler in einer eigenen workspace und die feindlichen Objekte in einer anderen, und vielleicht noch eine workspace, die nur die scripts zeigt, die beide verwenden.

Eine weitere wichtige Funktion von workspaces ist, dass Sie auf die Registerkarte klicken und sie - bei gedrückter Maustaste - aus dem Hauptfenster IDE in ein eigenes Fenster ziehen können, was die Organisation sehr erleichtert, wenn Sie beispielsweise mehrere Bildschirme verwenden. Sie können diese sekundären workspace Fenster auch wieder mit dem Hauptfenster zusammenführen, indem Sie die Registerkarte wieder auf das erste Fenster ziehen. Beachten Sie, dass Sie zwar scheinbar zwei Instanzen von IDE laufen lassen, aber beide für dasselbe Projekt sind. Sie können nicht ein Projekt in der einen und ein anderes in der anderen Instanz haben, es sei denn, Sie öffnen ausdrücklich zwei Instanzen von GameMaker.

Wenn Sie GameMaker zum ersten Mal starten, wird Ihr workspace bereits von einigen Fenstern bevölkert, die an IDE" angedockt" sind. Diese werden im Folgenden kurz erklärt:

  • Ausgabe - In der Abbildung oben sehen Sie auch das Ausgabefenster. Es gibt eine Reihe von Unterregisterkarten in diesem Fenster, die sich auf die Versionskontrolle, die Suche und die Fehlersuche beziehen, wobei die erste Registerkarte für die Konsolen-/Compilerausgabe vorgesehen ist, die Ihnen zeigt, was passiert, wenn Sie ein Spiel zum Testen kompilieren oder wenn Sie ein endgültiges Paket für die Verteilung erstellen. Auf dieser Registerkarte werden auch alle Debug-Meldungen angezeigt, die Sie von Ihrem Projekt an runtime senden möchten, und sie kann so konfiguriert werden, dass sie unterschiedliche Mengen an Informationen anzeigt. Wenn Sie das Ausgabefenster schließen und es wiederherstellen möchten, können Sie das Windows-Menü verwenden.
  • Assets - Auf der rechten Seite des Bildschirms finden Sie die Asset-Browser. Hier können Sie die verschiedenen Assets, die Ihr Spiel verwendet, erstellen und bearbeiten sowie Konfigurationen, Spieloptionen, Raumanordnung und andere Dinge erstellen und ändern. Assets werden erstellt, indem du mit der rechten Maustaste auf auf einen Asset-Ordner (oder irgendwo in der Hauptansicht des Asset-Browsers) klickst und Erstellen wählst, oder indem du auf das Plus-Symbol oben klickst und ein Asset aus dem FensterAsset erstellen auswählst (beachte, dass du mit dieser Methode mehrere Assets desselben Typs gleichzeitig erstellen kannst, indem du den Wert unten einstellst):

    Wenn Sie eine dieser Optionen wählen, wird eine neue, leere Ressource des angegebenen Typs für Sie erstellt, wobei das Namensfeld hervorgehoben ist, so dass Sie ihr sofort einen Namen geben können. Beachten Sie, dass der Asset-Browser standardmäßig an IDE angedockt ist. Sie können ihn jedoch in ein eigenes Fenster verschieben, indem Sie auf auf den Text "Asset-Browser" am oberen Rand klicken und ihn ziehen. Sie können ihn jederzeit wieder andocken, indem Sie ihn an die Seiten oder den Boden von IDE ziehen. Wenn Sie das Asset-Browser-Fenster schließen und es wiederherstellen möchten, können Sie das Windows-Menü verwenden. Weitere Informationen zu den verfügbaren Assets finden Sie im Abschnitt über Editoren.

Arbeitsbereich RMB-Menü

Auf workspace können Sie jederzeit das Rechtsklick-Menü aufrufen, indem Sie mit der rechten Maustaste auf klicken, wodurch sich das folgende Menü öffnet:

Wenn Sie auf Assets klicken, wird eine Liste aller Assets im Asset-Browser angezeigt, die Sie erstellen können, und wenn Sie eines davon auswählen, wird die Asset-Ressource für Sie erstellt und der Fokus von workspace auf das Editorfenster für diese Ressource gelegt. Wenn Sie auf die Option Fenster klicken, wird eine Liste aller derzeit geöffneten Fenster auf workspace angezeigt, und Sie können eines davon auswählen, um den Fokus von workspace auf dieses Fenster zu legen. Schließlich haben Sie die Option Gehe zu , die das folgende Fenster öffnet:

In diesem Fenster können Sie alle Elemente, Funktionen, Spieloptionen und Einstellungen in Ihrem Projekt nach dem von Ihnen eingegebenen Suchbegriff durchsuchen. Wenn Sie z. B. "Icon" als Suchbegriff eingeben, werden alle Elemente in Ihrem Projekt angezeigt, die "Icon" in ihrem Namen enthalten, z. B.: Wenn ein sprite spr_Icon_Play ist, wird es hier angezeigt, ebenso wie das Objekt " obj_Iconoclast", und es werden sogar die Spieloptionen für die verschiedenen Icons angezeigt, z. B. "Icon für hdpi". Wenn Sie auf eines der in der gefilterten Liste angezeigten Objekte klicken, wird es im aktuellen workspace geöffnet, oder der Fokus von workspace wird darauf gerichtet, wenn es bereits geöffnet ist.

Beachten Sie, dass Sie auch die Tastenkombination / + T verwenden können, um das Fenster Gehe zu zu öffnen, und dass die Anzahl der aufgelisteten Elemente (standardmäßig) auf maximal 16 eingestellt ist. Sie können diese Höchstzahl zusammen mit anderen Suchoptionen im Abschnitt "Gehe zu " in den allgemeinen Einstellungen ändern.

Weitere Informationen zu Workspace Navigation finden Sie im Abschnitt über IDE-Eingabe und Navigation.

Andocken

Wir haben bereits kurz erwähnt, dass Sie viele Fenster an das aktuelle Fenster workspace andocken können und dass Sie die bereits angedockten Fenster wieder lösen können. Wenn Sie z. B. oben auf auf die Registerkarte Asset-Browser klicken und sie nach links vom aktuellen workspace ziehen, werden Sie sehen, dass sie zu einem frei schwebenden Fenster wird.

Wenn Sie bestimmte Fenster an die Seiten von workspace (oder nach unten) ziehen, werden sie angedockt, d. h. sie werden im Wesentlichen Teil des IDE Fenster-Overlays und sind nicht mehr unabhängig. Ziehen Sie dazu das Fenster auf den Bereich, an den Sie andocken möchten, und es wird hervorgehoben, um anzuzeigen, dass Sie das Fenster dort andocken können:

Auf diese Weise können Sie eine personalisierte IDE Erfahrung schaffen, die zu Ihrem Arbeitsablauf und Ihrer Arbeitsweise passt. Es ist erwähnenswert, dass alle angedockten Fenster einzeln ein- und ausgeblendet werden können, indem Sie auf die Schaltfläche neben den angedockten Elementen klicken (in der Abbildung oben auf dieser Seite hervorgehoben), oder Sie können alle angedockten Elemente über die Schnellschaltfläche oben auf IDE ein- und ausblenden. Beachten Sie auch, dass IDE die angedockten Fenster zwischen den Sitzungen speichert.

HINWEIS: Einige Fenster verschwinden aus dem Dock, wenn Sie den Editor wechseln, z. B. im Raumeditor. Wenn Sie jedoch zu dem betreffenden Editor zurückkehren, werden sie wieder im Dock geöffnet.
