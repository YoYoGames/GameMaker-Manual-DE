Das Ausgabefenster

Wenn Sie ein Projekt zum ersten Mal in GameMaker öffnen, wird das Ausgabefenster am unteren Rand des Bildschirms angedockt. Dieses angedockte Fenster enthält verschiedene Registerkarten, die abhängig von bestimmten Umständen die verschiedenen Ausgabeinformationen für Ihr Projekt anzeigen. Sie können das Andockfenster schließen, indem Sie auf die Schaltfläche unten in der IDE klicken, und Sie können auf klicken und eine beliebige Registerkarte im angedockten Fenster in ein anderes Andockfenster ziehen, um ihre Position zu ändern, oder Sie können auf in der Inhaltsleiste des Andockfensters klicken und es in die workspace ziehen, um ein eigenständiges Fenster zu erstellen:

Sie können auch auf klicken und eine angedockte Ausgabe in eine andere ziehen, um ein geteiltes Ausgabefenster zu erstellen, wie im folgenden Beispiel gezeigt, in dem die beiden Fehlerausgaben auf derselben Registerkarte platziert wurden (Sie können auch langsam auf auf eine Registerkarte klicken, um ihren Namen zu ändern):

Sie können den Standardzustand von IDE jederzeit wiederherstellen, indem Sie im Fenster Layouts die Option Layout zurücksetzen wählen, und Sie können jede geschlossene Registerkarte über das Menü Fenster erneut öffnen. Sie können auch die Art und Weise ändern, wie der Inhalt der einzelnen Ausgabefenster angezeigt wird, indem Sie die Voreinstellungen für die Ausgabefenster bearbeiten.

Ausgabe

Im allgemeinen Ausgabefenster werden alle Ausgaben von compiler angezeigt, sowie alle Meldungen, die Sie Ihrem Spiel mit die Funktion show_debug_message() Funktion hinzugefügt haben. Die meisten der anfänglich angezeigten Informationen sind lediglich Debug-Informationen darüber, wie das Spiel aufgebaut wird und und können daher im Allgemeinen ignoriert werden. Wenn Sie jedoch ein Problem bei der Erstellung Ihres Projekts für ein bestimmtes Ziel platform haben, kann dies hilfreich sein, um die Ursache zu finden und Informationen für den Support bereitzustellen, falls Sie diesen kontaktieren. Beachten Sie, dass Sie die Menge der hier angezeigten Informationen im Abschnitt Kompilieren der Voreinstellungen ändern können.

Suchergebnisse

Sie können das Fenster Suchen und Ersetzen mit dem Tastaturkürzel / + + " F" öffnen oder das Menü Bearbeiten aufrufen. Sobald Sie Ihre Suchbegriffe eingegeben haben, werden die Ergebnisse in diesem Fenster im Format angezeigt:

[object] - [event] - [Line Number]: [search string]

Wenn der Suchbegriff in einer script gefunden wird, wird er einfach angezeigt:

[script] - [Line Number]: [search string]

Sie können dann auf auf einen der zurückgegebenen Einträge doppelklicken, um das betreffende Asset an der richtigen Stelle zur Bearbeitung zu öffnen.

Dieses Fenster zeigt auch die Ergebnisse der Feather-Funktion "Alle Referenzen suchen".

Quellenkontrolle

In diesem Fenster werden alle Ausgaben für Ihr SCM-Plugin angezeigt. Weitere Informationen über die Einrichtung der Versionskontrolle erfahren Sie hier.

Haltepunkte

Haltepunkte sind Stellen in Ihrem Spiel code oder GML Visual, an denen Sie GameMaker angewiesen haben, die Ausführung des Projekts anzuhalten, während Sie sich im Debug-Modus. Sie können einen Haltepunkt an einer beliebigen Stelle im Spiel loop mit der Taste " F9" hinzufügen, und wenn Sie dies tun, erscheint er in dieser Ausgabelasche. Sie können sie aktivieren und deaktivieren (ohne sie zu entfernen) Sie können sie aktivieren und deaktivieren (ohne sie zu entfernen), indem Sie auf das Kontrollkästchen auf der linken Seite klicken. Wenn Sie mit der rechten Maustaste auf einen Haltepunkt klicken, erhalten Sie ein kleines Menü, mit dem Sie den Haltepunkt entfernen oder das Code/GML Visual Fenster zu öffnen, in dem sich der Haltepunkt befindet.

Syntax-Fehler

Syntaxfehler sind die Fehler in Ihrer code, die der GameMaker-Syntax-Checker aufgespürt hat. Es gibt viele Gründe, warum Sie einen syntax Fehler bekommen können - wie z.B. ein ";" in Ihrem code, wenn ein ":" erwartet wird - und diese sollten behoben werden, bevor Sie versuchen, Ihr Spiel zu testen oder zu kompilieren. Jeder syntax Fehlereintrag im Ausgabefenster kann mit der linken Maustaste doppelt angeklickt werden, um ein Fenster an der markierten Stelle zu öffnen, damit der Fehler behoben werden kann.

Syntaxfehler werden aktualisiert, während Sie Ihre code schreiben (es gibt eine kurze Pause zwischen der Eingabe und dem Erscheinen von Fehlern in diesem Fenster, um zu verhindern, dass Fehler für unvollendete code gemeldet werden), und folgen dem Format:

[object] - [event] - [Line Number] - [Position In Line]: [error string]

Oder wenn der Fehler in einer script zu finden ist, wird es sein:

[script] - [Line Number] - [Position in line]: [error string]

Beachten Sie, dass syntax Fehler in der Regel verhindern, dass das Spiel kompiliert wird. Es gibt jedoch auch Fehler, die angezeigt werden, die das Kompilieren nicht verhindern, die aber dennoch behoben werden sollten. Die fraglichen Fehler sind:

  • für deklarierte Variablen, die nirgendwo verwendet werden
  • für irgendwo verwendete Variablen, die noch nicht deklariert wurden

In der Regel deutet beides auf einen Tippfehler in einem Variablennamen oder einen anderen leicht zu machenden Fehler hin - und wird daher zur Überarbeitung markiert -, aber es kann auch Absicht dahinter stecken, da Sie die Variable für die zukünftige Verwendung deklariert haben, aber noch nicht dazu gekommen sind, sie tatsächlich zu verwenden. Aus diesem Grund kann Ihr Spiel mit diesen Fehlern trotzdem kompiliert werden. Eine vollständige Liste der möglichen syntax Fehlermeldungen finden Sie hier.

Kompilier-Fehler

Ein Compiler Fehler tritt auf, wenn Ihr Spiel auf eine Art von Fehler stößt, den der syntax Checker nicht erkennen konnte, oder wenn ein Fehler damit zusammenhängt, wie Sie die Kompilieroptionen eingestellt haben. In diesem Fall meldet Ihr Spiel einen compiler Fehler und hört auf zu laufen. Alle compiler Fehler werden auch im Fenster Allgemeine Ausgabe angezeigt, aber sie werden auch hier separat aufgeführt (da sie im Rest der allgemeinen Ausgabe "verloren" gehen können).

Die Fehlermeldungen auf compiler haben alle das gleiche Format:

[object] - [event] - [Line Number]: [error string]

Wenn der Fehler in einer script gefunden wird, wird er einfach sein:

[script] - [Line Number]: [error string]

Sie können dann auf auf einen der compiler Fehlereinträge doppelklicken, um das betreffende Asset an der Stelle zu öffnen, an der der Fehler aufgetreten ist. Eine vollständige Liste der möglichen Fehlermeldungen finden Sie hier.

Federnachrichten

Das Fenster Feather Messages zeigt Ihnen alle von Feather gemeldeten Fehler, Warnungen und Vorschläge. Mit einem Doppelklick auf einen Eintrag in der Liste können Sie zu dem entsprechenden Fenster code navigieren.

Weitere Informationen finden Sie unter Feather-Nachrichten (Feather aktiviert).
