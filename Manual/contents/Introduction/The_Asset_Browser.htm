Der Asset-Browser

Der Kern Ihres Spiels wird aus Assets erstellt, die dem Asset-Browser hinzugefügt werden, der sich standardmäßig auf der rechten Seite der IDE befindet. Hier können Sie alles hinzufügen, was Ihr Spiel zum Laufen braucht, einschließlich eines Spielraums, Sprites, Objekte, Pfade und eine Reihe anderer Dinge. Ein grundlegendes Spiel in GameMaker benötigt einen Raum (neue Projekte werden immer mit einem bereits erstellten Raum-Asset erstellt) und normalerweise mindestens ein Objekt und ein sprite. Wahrscheinlich werden Sie aber noch viel mehr verwenden!

Im Asset-Browser können Sie die Assets für Ihr Spiel erstellen und organisieren sowie bestimmte andere Aspekte wie platform spezifische Einstellungen und Konfigurationen ändern. Im Folgenden finden Sie Erklärungen zu den einzelnen Abschnitten.

Suchleiste

In der Suchleiste können Sie einen Text eingeben und damit die Anzeige in der Hauptliste der Kühlstellen darunter filtern. Wenn Sie z. B. "Raum" eingeben, werden alle Kühlstellen angezeigt, die "Raum" in ihrem Namen enthalten. Um die Asset-Liste wieder auf die Standardansicht zurückzusetzen, löschen Sie einfach den Text in der Suchleiste.

Asset hinzufügen

Mit der Schaltfläche Asset hinzufügen können Sie ein neues Asset in die Assetliste aufnehmen. Wenn Sie auf diese Schaltfläche klicken, öffnet sich das folgende Fenster:

Sie können wählen, welche der beiden obigen Ansichten Sie verwenden möchten, indem Sie entweder auf das Listensymbol oder das Rastersymbol klicken. Sie können auch auf das Symbol Importieren klicken, um ein bereits vorhandenes Asset aus einem anderen Projekt zu importieren. Wenn Sie nichts importieren möchten, wählen Sie einfach das zu erstellende Asset aus und klicken unten auf die Schaltfläche Erstellen. Beachten Sie, dass Sie hier nicht nur ein einziges Asset hinzufügen können, da Sie unten im Fenster die Anzahl der zu erstellenden Assets festlegen können. Klicken Sie dann auf Erstellen, und die Anzahl der gewählten Assets wird dem Asset-Browser für Sie hinzugefügt. Alle auf diese Weise erstellten Assets werden am Ende der Asset-Liste außerhalb von Ordnern hinzugefügt und im aktuellen workspace zur Bearbeitung geöffnet.

Die Assets, die Sie hier hinzufügen können, sind weiter unten in diesem Abschnitt des Handbuchs aufgeführt.

Filter

Wenn Sie auf die Schaltfläche Asset-Filter klicken, wird das folgende Fenster geöffnet:

Hier können Sie wählen, wie die im Asset-Browser aufgeführten Assets gefiltert und angezeigt werden sollen. Sie können sie von A - Z oder von Z - A anordnen, zuerst die Gruppen und dann die Kühlstellen innerhalb der Gruppen oder alles zusammen anordnen. Wenn Sie den Filter " Benutzerdefinierte Reihenfolge " wählen, können Sie ein oder mehrere Assets auswählen und sie im Asset-Browser nach oben oder unten ziehen und nach Belieben positionieren (beachten Sie, dass die Option "Gruppen zuerst" deaktiviert wird).

Sie können auch einen oder mehrere Asset-Typen zum Filtern auswählen, und nur die Assets, die sich im Projekt dieser Typen befinden, werden in der Asset-Liste angezeigt. Wenn Sie "Beliebiger Typ" wählen, wird der Filter zurückgesetzt. Sie können auch nach den von Ihnen zugewiesenen Tags filtern (Tags werden weiter unten in diesem Abschnitt des Handbuchs behandelt). Wenn Sie einen anderen Filter als "Beliebiger Typ" gewählt haben, ändert die Filter-Schaltfläche ihre Farbe, um anzuzeigen, dass der Filter aktiv ist.

Schnellzugriff

Dieser Bereich des Asset-Browsers bietet Ihnen verschiedene Möglichkeiten, schnell auf wichtige oder häufig verwendete Assets zuzugreifen, und ist in die folgenden Abschnitte unterteilt:

  • Zuletzt verwendet: Hier finden Sie eine Liste der zuletzt verwendeten Assets. Standardmäßig werden hier maximal 10 Assets aufgelistet, aber Sie können dies in den Asset-Browser-Voreinstellungen ändern.
  • Bevorzugte Objekte: Wenn Sie der Asset-Liste einige Assets hinzugefügt haben, haben Sie vielleicht einige, auf die Sie häufig zugreifen. Um die Arbeit zu erleichtern, können Sie diese Assets als "Favoriten" kennzeichnen, wodurch sie zu dieser Liste hinzugefügt werden und Sie sie schnell finden können. Um ein Asset als Favorit zu kennzeichnen, klicken Sie mit der rechten Maustaste auf das Asset und wählen Sie im Pop-up-Menü die Option Favorit. Beachten Sie, dass Sie die rechte Maustaste erneut verwenden können, um die Markierung eines Assets aufzuheben , und wenn Sie die Schaltfläche auf dem Hauptordner Favoriten verwenden, können Sie Alles löschen wählen, um alles zu entfernen.
  • Raumreihenfolge: Hier können Sie die Reihenfolge ändern, in der Ihre Räume ausgeführt werden sollen. Für alle Projekte ist mindestens ein Raum erforderlich. Der Raum am Anfang der Liste ist der "Heimat"-Raum und wird als erster Raum betreten, wenn das Spiel beginnt (er ist mit einem "Heimat"-Symbol gekennzeichnet ). Um die Reihenfolge der Räume schnell zu ändern, klicken Sie einfach auf und ziehen Sie den Raum in der Liste nach oben oder unten, und lassen Sie dann die Maustaste los, um ihn an die neue Position zu setzen. Wenn Sie auf das "Home"-Symbol klicken, öffnet sich der Raummanager, und wenn Sie mit der rechten Maustaste auf einen Raum klicken, können Sie einen Unterraum erstellen. Weitere Informationen über die Anordnung von Räumen und Unterräumen finden Sie auf der Seite über den Raummanager.
  • Gespeicherte Filter: Wenn Sie die Filteroptionen verwenden, möchten Sie die getroffene Auswahl vielleicht für spätere Zwecke speichern. Richten Sie dazu den Filter ein, wie weiter oben auf dieser Seite beschrieben, und klicken Sie dann mit der rechten Maustaste auf diese Option und wählen Sie Filter speichern. Der Filter wird in dieser Liste gespeichert, und Sie können ihn bei Bedarf umbenennen. Um den Filter zu verwenden, doppelklicken Sie einfach auf ihn oder verwenden Sie die rechte Maustaste Menüoption Filter anwenden. Sie können gespeicherte Filter auch über das Menü RMB löschen und umbenennen .
  • Schlagwörter: Allen Assets im Asset-Browser können benutzerdefinierte Tags zugewiesen werden. Diese werden dann hier aufgelistet und können zum Filtern der in der Asset-Liste angezeigten Assets verwendet werden. Durch einen Doppelklick auf auf ein beliebiges Tag (oder über die rechte Maustaste Option Nach Tag filtern) können Sie dieses Tag auf den Asset-Listenfilter anwenden. Alle angewendeten Markierungen werden am unteren Rand des Asset-Browsers im Abschnitt Informationen angezeigt. Weitere Informationen zu Markierungen finden Sie weiter unten auf dieser Seite.
  • Spiel-Optionen: Hier können Sie schnell auf die verschiedenen Spieloptionen für Ihr Projekt zugreifen. Diese steuern verschiedene Aspekte des Projekts im Zusammenhang mit dem Einrichten und Kompilieren für die verschiedenen Ziele platforms (welche Spieloptionen verfügbar sind, hängt von Ihrem Lizenztyp ab). Weitere Informationen zu den einzelnen Optionen finden Sie im Abschnitt über die Spieloptionen.

Vermögensliste

In diesem Bereich des Asset-Browsers werden alle verschiedenen Assets aufgelistet, die Sie in Ihrem Spiel verwenden. Standardmäßig - wenn Sie ein neues Projekt erstellen - wird dieser Bereich mit zahlreichen Ordnergruppen gefüllt, die mit den verschiedenen Arten von Assets beschriftet sind, die hinzugefügt werden können, aber diese können umbenannt oder sogar entfernt werden, und Sie müssen nicht in jeder Gruppe Assets der gleichen Art speichern. So kann es z. B. sinnvoller sein, eine Gruppe von Assets für jede Ebene Ihres Spiels zu erstellen als für jede Art von Asset. Sie können also Ordnergruppen mit den Namen "Ebene 1", "Ebene 2" usw. erstellen und alle verschiedenen Assets für jede Ebene im selben Ordner speichern. Wie Sie die Dinge organisieren, bleibt wirklich Ihnen überlassen!

Um der Liste ein neues Asset hinzuzufügen, können Sie die Schaltfläche Asset hinzufügen am oberen Rand des Asset-Browsers verwenden oder das Menü der rechten Maustaste an beliebiger Stelle in der Asset-Liste öffnen und dann die zu erstellende Ressource auswählen:

Sobald das Asset erstellt wurde, wird es im workspace geöffnet und Sie können es in etwas umbenennen, das zu Ihrem Spiel passt. Beachten Sie jedoch, dass Asset-Namen nur alphanumerisch sein können, nicht mit einer Zahl beginnen und nur den Unterstrich "_" als zusätzliches Zeichen verwenden dürfen. Sie können auch das Menü der rechten Maustaste verwenden, um mit der Option " Gruppe erstellen " neue Ordnergruppen zu erstellen, die Assets enthalten, und diese können beliebig benannt werden. Wenn Sie ein Asset oder eine Ordnergruppe umbenennen möchten, können Sie einen langsamen Doppelklick auf ausführen oder die RMB Menüoption Umbenennen verwenden.

Eine wichtige Funktion des Asset-Browsers ist die Möglichkeit, Assets einzufärben, damit sie in der Liste besser zu erkennen sind. Klicken Sie dazu mit der rechten Maustaste auf den Asset-Listenknoten auf der linken Seite und wählen Sie dann eine Farbe aus der sich öffnenden Farbauswahlpalette. Wenn Sie dies für einen Ordner tun, werden auch alle in diesem Ordner enthaltenen Assets eingefärbt:

Wenn Sie die Farbauswahlpalette von einem solchen Knoten aus öffnen, können Sie mit der rechten Maustaste auf eine der Farbfelder klicken, um eine benutzerdefinierte Farbe zu erstellen.

Einzelne Kühlstellen können auch unabhängig von der Gruppenfarbe eingefärbt werden, indem Sie die RMB auf der Kühlstellennotiz oder die Palette unten im Menü RMB verwenden (obwohl dies in den Voreinstellungen des Asset-Browsers deaktiviert werden kann). Beachten Sie, dass Sie oder / zusammen mit der linken Maustaste verwenden können, um mehrere Kühlstellen auszuwählen und die Farbe allen auf einmal zuzuweisen:

Wie Sie die Dinge einfärben, bleibt Ihnen überlassen - es kann nach Art der Anlagegüter, nach Verwendung oder nach jedem anderen Kriterium erfolgen!

Abschließend sei noch darauf hingewiesen, dass Sie Ordner und Assets auswählen und dann auf klicken können, um sie aus einer anderen Ordnergruppe in der Asset-Liste heraus- oder in eine andere hineinzuziehen. Sie können auch einzelne oder alle Ordnergruppen löschen, wenn sie nicht benötigt werden.

Informationen

Die Informationsleiste am unteren Rand des Asset-Browsers zeigt Ihnen an, wie viele Assets sich insgesamt in der Asset-Liste befinden und wie viele Assets gerade ausgewählt sind. In diesem Bereich werden auch alle Tags angezeigt, die zum Filtern der Asset-Liste verwendet wurden (siehe unten für weitere Informationen zu Tags), und Sie können darauf klicken, um den Tag-Filter bei Bedarf zu entfernen.

Zoom-Steuerelemente

Mit den Zoom-Reglern können Sie den Inhalt des Asset-Browsers vergrößern oder verkleinern, und zwar mit einem Minimum von 50% bis zu einem Maximum von 250% des aktuellen DPI-Wertes des GameMaker IDE. Wenn Sie diese Einstellung auf den Standardwert von 100% zurücksetzen möchten, doppelklicken Sie einfach auf auf den Prozentwert links neben dem Schieberegler. Beachten Sie, dass in den Einstellungen des Asset-Browsers eine Reihe von Voreinstellungen zur Verfügung stehen, um die Funktionsweise zu steuern.

Tags

Wie bereits in verschiedenen Abschnitten erwähnt, bietet der Asset-Browser die Möglichkeit, Tags zu einzelnen Assets und zu Gruppenordnern hinzuzufügen. Tags sind ein sehr leistungsfähiges Werkzeug, nicht nur zum Organisieren und Filtern der im Asset-Browser aufgelisteten Assets, sondern auch beim Programmieren Ihres Spiels im Allgemeinen, da Sie zugewiesene Tags in code verwenden können.

Um eine Markierung zu erstellen, müssen Sie das Menü RMB auf einem Kühlgut verwenden und dann die Option Tags bearbeiten auswählen. Daraufhin wird der Tag-Editor angezeigt, in den Sie das gewünschte Tag eingeben können. Drücken Sie dann entweder oder geben Sie ein Komma ein, um es anzuwenden:

Sie können einem einzelnen Objekt mehrere Markierungen zuweisen. Sie können auch oder / zusammen mit der linken Maustaste verwenden, um mehrere Assets auszuwählen und ihnen allen gleichzeitig eine Markierung zuzuweisen. Nach dem Hinzufügen sind alle Markierungen im Menü Schnellzugriff und in den Filteroptionen (wie in den obigen Abschnitten erläutert) sichtbar und können so verwendet werden, um die Asset-Liste schnell zu filtern und nur die Elemente mit der/den gewählten Markierung(en) anzuzeigen.

Wenn Sie einer Ordnergruppe eine Markierung zuweisen, wird diese nicht automatisch den darin enthaltenen Assets zugewiesen.

Es ist wichtig zu beachten, dass Assets, die erstellt werden, während Tags Teil des aktiven Filters sind , automatisch mit diesen Tags versehen werden. Wenn Sie also beispielsweise alle Assets unter dem Tag "Ebene 1" anzeigen, wird allen neuen Assets, die bei aktivem Filter erstellt werden, bei der Erstellung ebenfalls das Tag "Ebene 1" hinzugefügt.

Schließlich können Tags auch in Ihrem Projekt code verwendet werden, um ein beliebiges Asset zu identifizieren und dann darauf zu reagieren, was sie zu einem sehr leistungsfähigen Werkzeug macht, da Sie Verhaltensweisen auf der Grundlage eines Tags oder einer Gruppe von Tags ausführen können code. Stellen Sie sich zum Beispiel vor, Sie haben ein Projekt mit einem "Die On Touch" Tag. Bei der Programmierung des Projekts können Sie dieses Tag überprüfen und die passende code Funktionalität auswählen. Später können die Künstler und Leveldesigner neue Assets jeder Art erstellen und sie mit dem Tag "Die On Touch" kennzeichnen und das Asset dann bei der Gestaltung eines Levels verwenden. Jedes code, das so programmiert wurde, dass es dieses Tag verwendet, wird diese neuen Assets automatisch einbeziehen.

Weitere Informationen zur Verwendung von Tags im Projekt code finden Sie im Abschnitt über Assets und Tags.

RMB-Menü

Wenn Sie mit dem Asset-Browser arbeiten, können Sie mit der rechten Maustaste auf eine beliebige Ressource oder einen Ordner klicken, um ein Menü mit Optionen zu öffnen:

Die genauen Optionen, die Ihnen zur Verfügung stehen, hängen von dem Kontext ab, auf den Sie geklickt haben. Im Folgenden zeigen wir den Inhalt, wenn das Menü auf einem Asset oder einer Ordnergruppe in der Asset-Liste geöffnet wird (in anderen Kontexten werden einige dieser Optionen angezeigt, aber nicht alle):

  • Erstellen - Erstellen Sie ein neues Asset. Wenn Sie dies auswählen, wird eine Liste der verfügbaren Asset-Typen angezeigt, die erstellt werden können, und Sie können einen auswählen. Wenn Sie dieses Menü für ein bestimmtes Asset öffnen, gibt es eine zusätzliche Option zum Erstellen einer Ressource desselben Typs, die direkt darunter aufgeführt ist, so dass Sie nicht nach dem Asset-Typ in der Liste suchen müssen.
  • Bearbeiten (Alle) - Öffnet den Editor für die Ressourceneigenschaften des/der ausgewählten Assets oder der Ordnergruppe (beachten Sie, dass Sie auch auf auf ein einzelnes Asset doppelklicken können, um es zur Bearbeitung zu öffnen).
  • Tags bearbeiten - Öffnet den Tag-Editor, in dem Sie Tags zu den ausgewählten Assets oder Ordnergruppen hinzufügen oder entfernen können.
  • Umbenennen - Benennen Sie das Asset oder die Ordnergruppe um.
  • Duplizieren - Erstellt ein Duplikat des oder der ausgewählten Assets.
  • Favorit - Markieren Sie ein oder mehrere Assets als Favorit und fügen Sie sie dem Schnellzugriffsbereich oben im Asset-Browser hinzu.
  • Vorhandenes hinzufügen - Fügt vorhandene Assets aus einem anderen GameMaker-Projektordner hinzu. Bitte beachten Sie, dass Assets, die mit einer Version vor 2.3.0 erstellt wurden, konvertiert werden müssen, da sich das Projektdateiformat mit dem Update auf 2.3.0 geändert hat. Das bedeutet, dass es möglicherweise besser ist, das gesamte Projekt zu importieren, alles zu konvertieren und dann an einem neuen Ort zu speichern, wenn die Assets häufig in andere Projekte importiert werden.
  • Vorhandenes aus meiner Bibliothek hinzufügen - Fügt der Asset-Liste Assets aus einem Element in Ihrer Marketplace Bibliothek hinzu. Bitte beachten Sie, dass Marketplace Assets, die mit einer Version vor 2.3.0 erstellt wurden, konvertiert werden müssen, da das Projektdateiformat mit dem Update 2.3.0 geändert wurde.
  • Löschen - Löscht das/die markierte(n) Asset(s) aus dem Projekt. Dies kann nicht rückgängig gemacht werden!
  • Gruppe erstellen - Fügen Sie der Asset-Liste eine neue Ordnergruppe hinzu.
  • Kinder erweitern - Erweitert die aktuell ausgewählte Ordnergruppe mit allen darin enthaltenen Unterordnern.
  • Alle aufklappen - Alle Ordnergruppen in der Asset-Liste aufklappen.
  • Alle verpflichten - Verpflichten Sie alle Ordnergruppen in der Asset-Liste.
  • In neuem Asset-Browser öffnen - Diese Option ist nur für Ordnergruppen verfügbar und öffnet die Gruppe in einem neuen Asset-Browser-Fenster, das nur den Inhalt der übergeordneten Gruppe enthält, für die die Option ausgewählt wurde.
  • Im Explorer öffnen - Dies öffnet den Datei-Explorer an dem Ort, an dem die Datei für die Ressource in Ihrem Projektordner gespeichert wurde.
  • Farbfelder - Hier können Sie eine neue Farbe für das zu verwendende Asset auswählen. Bei Verwendung für eine Ordnergruppe erhalten alle Assets innerhalb des Ordners die gewählte Farbe, bei Verwendung für ein oder mehrere Assets überschreibt sie die für die Ordnergruppe eingestellte Farbe (es sei denn, Sie haben diese Option in den Voreinstellungen des Asset-Browsers außer Kraft gesetzt).

Asset-Typen

Nachdem Sie nun wissen, wie der Asset-Browser funktioniert, sollten Sie die verschiedenen Asset-Typen kennenlernen, die Sie ihm hinzufügen können. Im Folgenden geben wir Ihnen einen Überblick über die verschiedenen Asset-Typen, die Sie in Ihren Projekten verwenden können:

Animation Kurven

Eine Animationskurve ist eine Ressource, die eine oder mehrere Kurven enthält, die darstellen, wie sich ein Wert im Laufe der Zeit ändert, und kann lineare Interpolation oder Katmull-Rom-Interpolation verwenden, um zwischen den Werten in der Kurve zu wechseln. Die Werte, die Sie einstellen, können zwischen -1 und 1 auf der vertikalen Achse liegen, und die Dauer entlang der horizontalen Achse ist immer von 0 bis 1 normalisiert, was es einfach macht, verschiedene Zeitbereiche mit Multiplikatoren in Ihrem Spiel code anzuvisieren. Animationskurven bestehen aus "Kanälen", und jeder Kanal kann seine eigenen Kurveneinstellungen haben, so dass Sie z. B. eine räumliche Position mit zwei Kanälen für die x/y-Position oder einen Farbverlauf mit 4 Kanälen zur Darstellung eines Farbformats beschreiben können. Animationskurven-Assets können bei der Arbeit mit sequences verwendet werden und sind auch über code zugänglich, was sie zu einem leistungsstarken Werkzeug bei der Erstellung Ihrer Spiele macht.

Weitere Informationen über das Hinzufügen von Animationskurven finden Sie im Abschnitt über Animationskurven.

Erweiterungen

Erweiterungen sind zusätzliche Dateien und code, die Sie hinzufügen können, um die GameMaker-Funktionen zu erweitern und Ihrem Spiel Assets aus dem Marketplace hinzuzufügen. Das Format einer Erweiterung variiert je nach dem endgültigen Ziel platform, obwohl sie im Allgemeinen in C++, C# oder JavaScript geschrieben sind.

Weitere Informationen zum Hinzufügen von Erweiterungen finden Sie im Abschnitt über den Erweiterungseditor.

Schriftarten

Wenn ihr in eurem Spiel Text zeichnen wollt, wird dieser Text standardmäßig in der Schriftart Arial 12 Punkt gezeichnet, aber um interessantere oder einzigartig aussehende Texte zu erstellen, werdet ihr wahrscheinlich andere Schriftarten verwenden wollen. Hier kommt der Font Editor zum Einsatz. Hier kannst du eine Schriftart auswählen, die du auf deinem Computer hast, und sie in GameMaker importieren, um sie in deinem Spiel zu verwenden, wobei du verschiedene Attribute wie Größe, Gewicht oder Stil festlegen kannst. Sobald sie importiert und als Ressource hinzugefügt wurde, kannst du sie mit den entsprechenden code oder Aktionen zum Zeichnen einstellen.

HINWEIS: Wenn Sie Ihrem Spiel Schriftarten hinzufügen, vergewissern Sie sich, dass Sie die erforderliche Lizenz haben, um die Schriftarten weiterzugeben, es sei denn, sie sind gemeinfrei oder urheberrechtsfrei.

Das Menü RMB für Schriftarten enthält die folgenden Optionen (sowie die weiter oben auf dieser Seite aufgeführten):

  • Texturgruppe zuweisen - Wenn Sie diese Option wählen, wird eine Liste der verfügbaren Texturgruppen angezeigt, aus der Sie die Schrift auswählen oder bei Bedarf eine neue erstellen können. Wenn im Asset-Browser mehrere Schriften ausgewählt sind - oder Sie haben einen Gruppenordner mit Schriften ausgewählt -, werden alle Schriften der gewählten Texturgruppe hinzugefügt. Weitere Informationen zu Texturgruppen finden Sie hier.
  • Schriftart neu generieren - Mit dieser Option können Sie GameMaker zwingen, die Schriftart cache neu zu generieren. Dies ist in der Regel nur erforderlich, wenn die Schriftart beschädigt ist oder sich auf dem System geändert hat.

Weitere Informationen über das Hinzufügen von Schriftarten finden Sie im Abschnitt über den Schriftarten-Editor.

Anmerkungen

Notizen sind einfach code Editor-Fenster, in die Sie alles schreiben können. Sie sind dafür gedacht, code Schnipsel, Spielinformationen, Aufgabenlisten, Teamkommunikation usw. zu speichern.

Weitere Informationen zum Hinzufügen von Notizen finden Sie im Abschnitt über den Notiz-Editor.

Objekte

Objekte sind eine besondere Ressource, die wir verwenden, um Aspekte eines Spiels zu steuern und bestimmte Dinge zu tun. Meistens sind sie mit Sprites verknüpft, so dass man sie im Spielraum sieht, aber manchmal werden sie als Controller "hinter den Kulissen" verwendet, um Dinge in Bezug auf den Benutzer oder für das Timing usw. zu tun. Man kann ihnen Verhaltensweisen zuweisen und sie können auf bestimmte Ereignisse sowie auf einander reagieren, und die meisten Dinge, die man in einem Spiel sieht, basieren auf Objekten und ihren Interaktionen. Das meiste, was du im Spiel siehst, basiert auf Objekten und ihren Interaktionen. Beachte, dass wir "basiert auf" sagen, weil du nicht wirklich Objekte direkt in den Spielraum platzierst, sondern Instanzen dieser Objekte, die im Grunde genommen Kopien (oder Klone, wenn du willst) der Objektressource sind. Das ist sehr wichtig zu wissen, denn Instanzen und Objekte sind nicht dasselbe und haben jeweils ihre eigenen Funktionen, die sie beeinflussen können.

Alle Objekte haben eine Reihe von Eigenschaften, die Sie im Objekteditor angeben können, wie z.B. sprite, ob es Physik verwendet oder nicht, oder ob es ein "Kind"-Objekt eines anderen ist. Sie haben auch eine Reihe von eingebauten Variablen, die dann in Aktionen, scripts und code verwendet werden können. Diese eingebauten Variablen werden verwendet, um die Position der Instanz zu definieren, wenn sie im Spielraum platziert wird, die Animationsgeschwindigkeit, die Bewegungsrichtung und eine ganze Reihe anderer Dinge. Um zu steuern, wie sich das Objekt im Laufe der Zeit verhält, wird jedes Spielbild in eine Reihe von Ereignissen aufgeteilt, so dass Sie code oder Aktionen in ein bestimmtes Ereignis einfügen können, das nur dann abläuft, wenn das Ereignis ausgelöst wird, und Ereignisse können durch Dinge wie das Drücken der Maustaste oder Kollisionen mit anderen Instanzen ausgelöst werden.

Objekte sind im wahrsten Sinne des Wortes die Bausteine Ihres Spiels und bilden zusammen mit den Räumen den Kern eines jeden Projekts, das Sie erstellen.

Das Menü RMB für Objekte enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten Optionen):

  • Alle Ereignisskripte öffnen - Wenn Sie diese Option wählen, werden alle zugewiesenen Ereignisse für das Objekt in einem neuen workspace als script Registerkarten geöffnet.

Weitere Informationen zum Hinzufügen von Objekten finden Sie im Abschnitt über den Objekteditor.

Pfade

Bei der Erstellung Ihrer Spiele kann es vorkommen, dass Sie eine Instanz eines Ihrer Objekte benötigen, um einem Pfad durch ein Level zu folgen. Das kann man tun, indem man einfach ein Array von Positionen im Raum erstellt und die Instanz dann zwischen ihnen hin- und herbewegt, aber das kann mühsam sein und ist schwierig zu ändern, zu testen und für verschiedene Dinge anzupassen. In diesem Fall sollten Sie eine Pfadressource erstellen.

Die Grundidee der Pfad-Ressource ist recht einfach - man definiert einen Pfad, indem man ihn im Pfad-Editor zeichnet, dann kann man eine Aktion (oder code) in einem Ereignis eines Objekts platzieren, um das Objekt anzuweisen, diesem bestimmten Pfad im Spielraum zu folgen. Sie können die Geschwindigkeit festlegen, mit der das Objekt dem Pfad folgt, sowie eine Reihe anderer Aktionen, die sich auf die Position und Ausrichtung des Pfades im Raum beziehen.

Weitere Informationen zum Hinzufügen von Pfaden finden Sie im Abschnitt über den Pfad-Editor.

Zimmer

Alle Spiele, die du in GameMaker entwickelst, brauchen mindestens einen Raum, um zu laufen (es können aber auch viele, viele mehr sein), und ein Raum ist im Grunde ein Raum, in dem du Instanzen der Objekte platzierst, die dein Spiel ausmachen und in dem sich die Handlung des Spiels abspielt. Der Raum-Editor ist jedoch eine der mächtigsten Ressourcen, die dir zur Verfügung stehen, da er dir nicht nur erlaubt, Instanzen von Objekten hinzuzufügen, sondern auch den "Blick" in den Spielraum einzurichten, die Hintergründe und Kacheln hinzuzufügen, die gezeichnet werden, um die Spielwelt zu erstellen, zusätzliche Spezialeffekte zu erstellen, die unabhängig vom Objekt/Instanz-System sind, und auch code unabhängig von einem Objekt auszuführen. Räume können auch so eingestellt werden, dass sie Eigenschaften von anderen Räumen erben , d. h. Sie können z. B. einen einzelnen Raum mit einer Reihe von Kacheln erstellen und dann einen weiteren Raum erstellen, der diese Kacheln erbt, so dass Sie sie nicht noch einmal erstellen müssen.

Das Menü RMB für Zimmer enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten Optionen):

  • Untergeordneten Raum erstellen - Hiermit wird ein neuer Raum als untergeordneter Raum des ausgewählten Raums erstellt. Untergeordnete Räume sind neue Raum-Assets, die die Eigenschaften, Ebenen und Ebeneninhalte des übergeordneten Raums erben und im Raum-Manager mit dem übergeordneten Raum verknüpft angezeigt werden.

Weitere Informationen über das Hinzufügen von Räumen finden Sie im Abschnitt über den Raumeditor.

Skripte

Eine script Ressource ist einfach ein Container für eine oder mehrere Funktionen, die Sie geschrieben haben. Nehmen wir an, Sie möchten, dass alle Ihre feindlichen Objekte eine Reihe verschiedener Instanzen erzeugen, wenn sie sterben - wie etwas Gold, eine Explosion und etwas Blut sowie einen Sound abspielen. Nun könnten Sie alle erforderlichen code in jedes Objekt einfügen, das sie benötigt, aber im Allgemeinen ist es besser, eine einzelne Funktion in script resource zu erstellen und diese dann aufzurufen. Das bedeutet, dass Sie nur eine Zeile code in die Objekte einfügen müssen, um diese neue geskriptete Funktion aufzurufen, und wenn Sie etwas ändern möchten, brauchen Sie es nur einmal in script zu ändern, und alle Objekte führen automatisch die neue code aus, anstatt jedes Objekt durchzugehen und dieselbe Sache an mehreren Stellen zu ändern.

Das Menü RMB für scripts enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten Optionen):

  • In GML Visual / GML konvertieren - Damit wird das ausgewählte script in GML Visual oder GML konvertiert. Wenn script GML ist, wird es in eine Aktion " Code ausführen" eingefügt, während GML Visual direkt in GML code übersetzt wird (wie im Abschnitt GML Visual in Code ändern gezeigt).

Weitere Informationen zum Hinzufügen von scripts finden Sie im Abschnitt über den Skript-Editor.

Sequenzen

Ein sequence -Asset ist im Wesentlichen eine Sammlung von anderen Ressourcen, die eine dynamische Animation im Laufe der Zeit durchführen. Sie können sprites, Instanzen, Sounds und andere Dinge enthalten, und jedes dieser Elemente kann so eingestellt werden, dass es sich bewegt, die Farbe ändert oder im Laufe der Zeit innerhalb der sequence selbst startet/stoppt, indem die Elemente bestimmten Spuren innerhalb der sequence zugewiesen werden. Auf diese Spuren können dann verschiedene Attribute angewendet werden, die statisch sein oder sich im Laufe der Zeit ändern können. Die gesamte Bearbeitung einer sequence erfolgt im Sequenzeditor, der aus folgenden Elementen besteht:

  • die Leinwand - hier platzieren Sie die Elemente, aus denen die Sequenz besteht
  • die Spurtafel - hier platzieren Sie Assets, um Asset-Spuren zu erstellen, und erstellen Parameterspuren, um die Assets im Laufe der Zeit zu manipulieren
  • Dope Sheet - hier können Sie die Keyframes der einzelnen Spuren hinzufügen/entfernen/bearbeiten und steuern, wie sie sich im Laufe der Zeit verhalten.

Beachten Sie, dass Sie sequences innerhalb von sequences haben können, um komplexe Animationen und Effekte zu erstellen, und sequences kann auch über code aufgerufen und geändert werden, um Ihnen so viel Kontrolle wie möglich darüber zu geben, wie sie in Ihrem Spiel angezeigt und verwendet werden.

Weitere Informationen zum Hinzufügen von sequences finden Sie im Abschnitt über den Sequences Editor.

Shader

Shader sind ein sehr leistungsfähiges Werkzeug, mit dem die Grafiken, die Ihr Spiel auf den Bildschirm rendert, manipuliert werden können. Sie ermöglichen unglaublich schnelle Effekte, die z. B. vom Hinzufügen eines subtilen Farbtons zu einem sprite bis hin zu Verzerrungseffekten auf dem gesamten Bildschirm reichen können. Im Grunde handelt es sich um ein zweiteiliges Programm, das direkt auf der Grafikkarte selbst läuft, was es sehr schnell macht, da der Grafikprozessor die ganze Arbeit erledigt und CPU-Zyklen für Ihr Spiel freigibt code. Der vollständige Shader besteht aus einem Vertex-Shader-Programm und einem Fragment-Shader-Programm (auch bekannt als Pixel-Shader). Diese beiden kleinen Programme arbeiten zusammen, um das, was die Grafikkarte auf den Bildschirm rendert, zu manipulieren. Dadurch können Sie die Position, die Farbe und die Alphawerte, die tatsächlich auf dem Bildschirm angezeigt werden, in Echtzeit beeinflussen buffer.

GameMaker unterstützt die folgenden Shader-Sprachen:

Shader-Sprache Ziel-Plattform
GLSL ES Alle Zielplattformen
GLSL Mac und Ubuntu (Linux)
HLSL11 Windows, UWP, XboxOne
PSSL Playstation 4


Das Menü RMB für Shader enthält die folgenden Optionen (sowie die weiter oben auf dieser Seite aufgeführten):

  • Shader-Typ - Hier können Sie den Typ der Shader-Sprache einstellen, die für die Shader-Ressource verwendet wird.

Weitere Informationen zum Hinzufügen von Shadern finden Sie im Abschnitt über den Shader-Editor.

Klingt

Sound ist ein wichtiger Bestandteil eines jeden Spiels, sowohl um dem Spiel mehr Tiefe zu verleihen und Feedback in Form von Soundeffekten zu geben, als auch um Atmosphäre in Form von Musik zu erzeugen. GameMaker akzeptiert Dateien im Format WAV, MP3 und OGG.

Im Allgemeinen werden WAV Dateien für kurze Soundeffekte verwendet, da sie, obwohl sie in der Regel größer sind, sofort abgespielt werden, da sie nicht dekodiert werden müssen. MP3 und OGG Dateien sollten für Hintergrundmusik oder Effekte verwendet werden, die über einen längeren Zeitraum abgespielt werden oder eine relativ große Dateigröße haben. Diese Dateien sind viel kleiner als eine WAV-Datei, haben aber einen CPU-Overhead, da sie vor dem Abspielen dekodiert werden müssen.

Das Menü RMB für Sounds enthält die folgenden Optionen (sowie die weiter oben auf dieser Seite aufgeführten):

  • Im Soundmixer öffnen - Damit werden alle ausgewählten Sounds im Soundmixer geöffnet.
  • Audiogruppe zuweisen - Wenn Sie diese Option wählen, wird eine Liste der verfügbaren Audiogruppen angezeigt, aus der Sie eine auswählen können, um den Sound hinzuzufügen oder eine neue Gruppe zu erstellen, falls erforderlich. Wenn im Asset-Browser mehrere Sounds ausgewählt sind - oder Sie einen Gruppenordner mit Sounds ausgewählt haben -, werden alle Sounds der gewählten Audiogruppe hinzugefügt. Weitere Informationen zu Audiogruppen finden Sie hier.

Weitere Informationen zum Hinzufügen von Sounds finden Sie im Abschnitt über den Sound-Editor.

Das verwendete Bild muss nicht unbedingt ein einzelnes statisches Objekt darstellen. Es kann auch ein "Strip"-Bild sein, d.h. mehrere Bilder in einer einzigen Datei, die dann zu einem einzigen animierten sprite verwendet werden können. Tilesets

Kachelsätze werden von den sprite -Ressourcen übernommen, aber als separate Ressource klassifiziert, da GameMaker sie bei der Erstellung Ihres Spiels anders behandelt. Im Grunde ist ein Kachelset ein einzelnes Bild, das GameMaker in einzelne Abschnitte unterteilt, basierend auf den Werten, die du für die verschiedenen Einstellungen angibst. Du kannst diese dann im Raumeditor (oder prozedural durch code) verwenden, um eine Kachelkarte in deinem Raum zu erzeugen. Sie eignen sich hervorragend für die Gestaltung von statischen Objekten im Raum, wie z. B. Terrain, Wände, Kulissen usw., da sie nicht den gleichen Overhead wie Objekte haben.

Oben sehen Sie ein Beispiel für sprite, das als Kachelsatz zum Erstellen von Wänden verwendet werden würde. Wie Sie sehen können, kann es in 64x64-Pixel-Stücke "aufgeteilt" werden, die dann im Raumeditor platziert werden können.

HINWEIS Kacheln sind immer quadratisch, wenn du also etwas anderes brauchst, solltest du eine Instanz mit einer sprite oder einen Asset-Layer im Raumeditor verwenden.

Das Menü RMB für Tilesets enthält die folgenden Optionen (sowie die weiter unten auf dieser Seite aufgeführten):

  • Texturgruppe zuweisen - Wenn Sie diese Option wählen, wird eine Liste der verfügbaren Texturgruppen angezeigt, aus der Sie eine auswählen können, um das Tileset hinzuzufügen oder gegebenenfalls eine neue Gruppe zu erstellen. Wenn im Asset-Browser mehrere Tilesets ausgewählt sind - oder Sie einen Gruppenordner mit Tilesets ausgewählt haben -, werden alle Tilesets der gewählten Texturgruppe hinzugefügt. Weitere Informationen zu Texturgruppen finden Sie hier.

Weitere Informationen über das Hinzufügen von Kachelsätzen finden Sie im Abschnitt über den Kachelsatz-Editor.

Zeitleisten

In den meisten Spielen möchte man, dass bestimmte Dinge zu bestimmten Zeitpunkten passieren. Man kann versuchen, dies mit Hilfe der Alarm-Ereignisse in einer Instanz zu erreichen, aber wenn die Dinge zu kompliziert werden, funktioniert das nicht mehr, zumal man auf nur zwölf Alarme beschränkt ist. Aus diesem Grund gibt es die Ressource Zeitleiste. In einer Zeitleiste gibst du an, welche Aktionen zu einem bestimmten Zeitpunkt in der Spielzeit stattfinden müssen, und du kannst alle Aktionen verwenden, die für ein Objekt in seinen verschiedenen Ereignissen sowie code verfügbar sind. Sobald du eine Zeitleiste erstellt hast, kannst du sie einem Objekt zuweisen, und die Instanz dieses Objekts wird dann die Aktionen und code zu den angegebenen Zeitpunkten ausführen, wenn sie in einem Raum platziert oder erstellt wird.

Weitere Informationen zum Hinzufügen von Zeitleisten finden Sie im Abschnitt über den Zeitleisten-Editor.
