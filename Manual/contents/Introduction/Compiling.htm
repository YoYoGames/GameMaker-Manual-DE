Kompilieren

Das Kompilieren Ihres Spiels kann zweierlei bedeuten: Kompilieren zum Testen oder Kompilieren, um eine endgültige ausführbares Paket für ein bestimmtes Ziel platform zu erstellen. Auf dieser Seite werden beide Möglichkeiten im Detail erklärt.

Kompilieren für Tests

Sie können Ihr Spiel zum Testen kompilieren, indem Sie einfach auf die Schaltfläche "Play" oben auf der Website IDE klicken. Sie können das Spiel auch im Debug-Modus ausführen, indem Sie auf die Schaltfläche Debug klicken. Dadurch wird das Spiel gestartet, aber auch das Debug-Fenster geöffnet, in dem Sie die Leistung Ihres Spiels überwachen und eventuelle Probleme erkennen können (weitere Informationen finden Sie im Abschnitt über Debugging ).

Ziel-Einstellungen

Standardmäßig läuft und debuggt GameMaker mit der eingebauten virtuellen Maschine (VM), die mehr oder weniger dem verwendeten Desktop-Betriebssystem entspricht. GameMaker ist jedoch eine platform übergreifende Engine und Sie können ausführbare Pakete Ihrer Projekte auf einer Reihe verschiedener Zielsysteme platforms testen, debuggen und kompilieren (die genaue platforms hängt von den Details Ihrer Lizenz ab). Um das aktuelle Ziel platform zu ändern, können Sie auf auf die Schaltfläche Ziele klicken, um das Fenster Ziele zu öffnen, das in etwa so aussieht (die genauen Details hängen von Ihrem Lizenztyp ab):

Oben, neben der Schaltfläche "Ziele", finden Sie die aktuellen Einstellungen, die Ihnen die platform und die derzeit verwendeten spezifischen Einstellungen anzeigen. Der Rest des Fensters enthält die Details und Optionen für alle verfügbaren Ziele, die Sie stattdessen auswählen können. Jeder Abschnitt dieses Fensters wird im Folgenden erläutert:

Ausgabe

Jedes Ziel platform kann ein oder mehrere Ausgabeformate haben, die wichtigsten davon sind:

  • VM: Das VM-Ziel (Virtual Machine) verwendet einen generischen Runner für jedes platform und interpretiert dann das code für Ihr Spiel. Im Allgemeinen wird diese Option aufgrund der schnelleren Erstellungszeiten zum Testen verwendet, aber sie bietet nicht denselben Leistungsschub wie die YYC-Option (falls verfügbar). Sie können diese Option verwenden, um ausführbare Pakete für kleinere Spiele zu kompilieren oder für Spiele, bei denen die Leistung kein Problem darstellen wird.
  • YYC: Der YYC(YoYo Compiler) generiert C++ code aus Ihrem GML code, bevor er das C++ compiler des Ziels verwendet, um es in native code für das Ziel platform zu kompilieren. Er entfernt nicht benötigte Funktionen und führt eine Reihe anderer Optimierungsverfahren durch, um ein kleineres und leistungsfähigeres ausführbares Paket zu erstellen.

    Dies kann die Leistung Ihres Spiels um mindestens das Zwei- oder Dreifache steigern, insbesondere bei logiklastigen Spielen, und ist ideal für größere oder CPU-intensive Projekte. Die Kompilierzeiten können länger sein und Sie sollten immer die compiler cache löschen, bevor Sie ein endgültiges vollständiges Asset-Paket für ein Ziel platform erstellen. Beachten Sie, dass das YYC-Ziel möglicherweise die Installation zusätzlicher Tools für das gewählte platform erfordert, da es sonst nicht funktioniert - weitere Informationen dazu finden Sie im YoYo Games Help Center sowie auf den einzelnen Zielvorgaben-Seiten in diesem Handbuch.
  • JavaScript: Das JavaScript-Ziel ist nur für bestimmte Ziele verfügbar, wie das HTML5-Ziel, und legt fest, dass das Spiel in reinem JavaScript kompiliert wird. Dies verwendet ECMAScript 2015 (oder ES6) für die JS-Ausgabe.

For 2024.800+ beta versions of GameMaker, some platforms may also include the "GMRT" and "GMRT VM" outputs as part of the GMRT (GameMaker Runtime) open beta.

Erstellen eines endgültigen ausführbaren Pakets

Übersicht

Sie können einfach auf die Schaltfläche Ausführbare Datei erstellen im IDE klicken, um die Erstellung von compiler zu starten, oder Sie wählen Ausführbare Datei erstellen aus dem Menü Erstellen. In beiden Fällen wird der Erstellungsprozess gestartet, der vom ausgewählten Ziel platform abhängt.

Auf dem Opera GX-Ziel öffnet sich ein spezielles Fenster, in dem Sie Ihr Spiel kompilieren und auf GX.games hochladen können; auf allen anderen Zielen öffnet sich ein Dateiexplorer-Fenster, in dem Sie den endgültigen Namen für die ausführbare Datei Ihres Spiels eingeben können, bevor Sie auf Speichern klicken, um den Kompilierungs- und Erstellungsprozess zu starten. Sobald Sie dies getan haben, werden die notwendigen Dateien generiert, so dass Sie das Spiel wie gewünscht verteilen können.

Für bestimmte platforms müssen Sie zum Kompilieren Ihres Spielprojekts die richtigen Build-Tools (siehe hier) und auch die entsprechenden Plattformvoreinstellungen ausgefüllt haben.

Je nach Lizenz haben Sie die Möglichkeit, Ihr Spiel über die Kommandozeile zu erstellen. So können Sie eine kontinuierliche Integration für Ihr Projekt einrichten und den Erstellungs- und Testprozess optimieren. Auf dieser Seite finden Sie weitere Informationen zur Erstellung über die Befehlszeile.

HINWEIS Bevor Sie einen endgültigen Build Ihres Projekts für die Veröffentlichung erstellen, sollten Sie immer den Asset Compiler Cache löschen (mit dem "Besen"-Symbol oben auf IDE). GameMaker cache viele Ihrer Spieldateien, um die Kompilierungsgeschwindigkeit so gering wie möglich zu halten, und diese können manchmal beschädigt werden. Daher ist es sicherer, diesen cache zu löschen, bevor Sie einen Release-Build erstellen.

Die maximale Größe des endgültigen Spielpakets beträgt 4 GB, außer bei 32-Bit Windows, wo sie 2 GB beträgt. Siehe Tipps zur Verringerung der Größe des fertigen Spiels.

Zielformate

Jede Zieloption speichert in einem platform spezifischen Format, das unten aufgeführt ist:

  • Opera GX - Wenn Sie für das "Opera GX"-Ziel kompilieren, sehen Sie das "Opera GX Packaging"-Fenster, das Sie zunächst auffordert, sich mit Ihrem Opera-Konto anzumelden:. Beachten Sie, dass Sie sich hier nicht anmelden müssen, wenn Sie sich bereits mit Ihrem Opera-Konto bei GameMaker angemeldet haben.

    Nachdem Sie sich angemeldet haben, beginnt das Programm mit der Kompilierung Ihres Spiels und lädt es auf GX.games hoch. Du kannst dann auf "Spiel auf Opera bearbeiten" klicken, um es auf der GX.games DevCloud zu bearbeiten und zu veröffentlichen:
    Bitte lesen Sie diese Seite, um mehr über den Upload-Prozess von Opera GX zu erfahren, und diese Seite, um die verschiedenen Exporte, wie YYC und Mobile, einzurichten.
  • Windows - Beim Kompilieren für das allgemeine Betriebssystem Windows werden Sie zunächst aufgefordert, zwischen der Erstellung eines Installers und eines Zip-Pakets zu wählen, wobei der Installer eine einzelne ausführbare Datei ist, die Ihr Spiel installiert, und die Zip-Datei eine einzelne komprimierte Datei im Format *.zip ist, in der alle Ihre Spieldateien gespeichert sind (die Dateien müssen extrahiert werden, damit das Spiel läuft). Wenn Sie das Kontrollkästchen Verpackungsoption merken aktivieren, merkt sich GameMaker diese Wahl für alle zukünftigen Kompilierungen (dies kann in den Windows-Einstellungen zurückgesetzt oder geändert werden). Weitere Informationen finden Sie im YoYo Games Help Center. Bitte beachten Sie, dass das Windows Ziel 32bit oder 64bit ausführbare Dateien kompilieren kann, je nachdem was Sie in den Allgemeinen Spieloptionen ausgewählt haben. Wir empfehlen dringend, dass alle neuen Projekte die x64 Runtime verwenden, um 64bit Executables zu erstellen, da die 32bit runtime in Zukunft veraltet sein und entfernt werden wird.
  • Ubuntu (Linux) - Beim Kompilieren für Ubuntu werden Sie gefragt, ob Sie eine .AppImage- oder eine .zip-Datei erstellen möchten, die entweder eine x64- oder eine ARM -Datei enthält (AppImage unterstützt jedoch nicht ARM). Weitere Informationen finden Sie im YoYo Games Help Center.
    .AppImage wird für die Weitergabe als Paket über jeden Verteiler außer Steam verwendet, während .zip nur für das Hochladen auf Steam verwendet wird (da es speziell Steam Runtime verwendet). Aus diesem Grund kann die erstellte .zip-Datei nicht wie eine normale Zip-Datei geöffnet werden.
  • HTML5 - Wenn Sie sich für HTML5 entschieden haben, wird eine index.html Datei (dies ist der Standardname, aber Sie können auch einen eigenen Namen in den Platform Einstellungen vergeben) zusammen mit einem Ordner, der Ihre Spieldateien enthält, erstellt und an dem angegebenen Ort gespeichert. Damit Ihr Spiel funktioniert, müssen Sie diese beiden Dateien auf einen Server hochladen. Die Seite index.html kann auch so angepasst werden, dass Ihr Spiel mit einer anderen Hintergrundfarbe oder an einer anderen Position usw. angezeigt wird. Dafür sind jedoch HTML-Kenntnisse erforderlich, und Sie können auch Ihre eigene Indexdatei angeben, wenn Sie das Paket erstellen (siehe HTML5-Spieloptionen). Weitere Informationen finden Sie im YoYo Games Help Center.
  • Android - Für Android -Geräte können Sie in dem Fenster, das zum Speichern des Spiels angezeigt wird, wählen, ob Sie eine *.apk oder eine *.aab (Android App Bundle) Datei erstellen möchten:

    Welchen Dateityp Sie wählen, hängt davon ab, welchen Store Sie ansteuern möchten, wobei die *.aab Datei für Google Play erforderlich ist, während die *.apk Datei für andere Stores verwendet werden kann. Weitere Informationen finden Sie im YoYo Games Help Center.
  • iOS - Beim Kompilieren nach iOS wird eine Datei xarchive erstellt, die dann in Xtools verwendet wird, um das endgültige Paket iOS zu erstellen. Beachten Sie, dass Sie zum Kompilieren für iOS einen Apple Mac-Computer mit OSX oder höher sowie die entsprechenden Zertifikate und Berechtigungen benötigen. Weitere Informationen finden Sie im YoYo Games Help Center.
  • macOS - Beim Kompilieren für das Ziel macOS werden Sie zunächst gefragt, ob Sie eine DMG- oder eine Zip-Datei erstellen möchten:
    Mit "Als DMG verpacken" wird ein .dmg-Installationspaket erstellt, das zur Installation des Spiels auf einem Mac-Computer verwendet werden kann. Beachten Sie, dass während des Erstellungsprozesses das Installationsprogramm auf dem Mac erscheint, das nach Abschluss des Erstellungsprozesses wieder verschwindet.

    Mit "Package as Zip" wird entweder eine *.app -Datei oder eine *.pkg -Datei erstellt, je nachdem, ob Sie sie später in den Mac App Store hochladen möchten oder nicht. Wie bei iOS benötigen Sie einen Apple Mac-Computer, auf dem OSX oder höher läuft, sowie die entsprechenden Zertifikate und Berechtigungen. Weitere Informationen finden Sie im YoYo Games Help Center.
  • Windows UWP - Für Windows UWP erstellt GameMaker ein *.appxbundle Paket, das dann in den Windows Store hochgeladen oder anderweitig verteilt werden kann. Wenn Sie auf die Schaltfläche Ausführbare Datei erstellen klicken, werden Sie aufgefordert, GameMaker mitzuteilen, welche Art von Paket Sie erstellen möchten (für ARM, x64 und/oder x86-Architekturen), und Sie sollten die Ziele auswählen, für die das Spiel unterstützt wird (Sie müssen mindestens eines auswählen, können aber alle drei oder eine beliebige Kombination auswählen):

    Wenn Sie das Kästchen Package For Store Upload (Paket für den Store-Upload) ankreuzen, wird als endgültiges Paket eine *.appxupload -Datei erstellt, die laut Microsoft für die Einreichung von Apps im Store verwendet werden sollte (wie in diesem Artikel hier erläutert, werden alle ausgewählten Architekturen einbezogen, genau wie bei *.appxbundle). Weitere Informationen zum Einrichten und Kompilieren für UWP platform finden Sie im YoYo Games Help Center.

Sobald Sie Ihr ausführbares Asset-Paket erstellt haben, können Sie die Datei an andere Personen weitergeben oder sie auf Ihrer Website zum Herunterladen bereitstellen. Sie können diese Dateien auch auf verschiedene Hosting-Dienste für den individuellen Vertrieb oder sogar auf Online-Shops (wie Google Play, Apple App Store oder Microsoft Store) für den allgemeinen Vertrieb und Verkauf hochladen.

Beachten Sie, dass es Ihnen freisteht, die mit GameMaker erstellten Spiele auf jede beliebige Weise zu vertreiben, einschließlich des Verkaufs. Dies setzt natürlich voraus, dass die sprites, Bilder und Sounds, die Sie für die Erstellung des Spiels verwendet haben, ebenfalls vertrieben oder verkauft werden können und dass Sie die Rechte an allen Assets besitzen, und es setzt auch voraus, dass das Spiel den YoYo Games EULA für GameMaker entspricht.
