Erstellung eines Marketplace Assets

Wenn Sie die Option " Paket erstellen" aus dem IDE Marktplatz-Menü auswählen, gelangen Sie zu folgendem Arbeitsbereich: Hier müssen Sie zunächst die Publisher-Identität auswählen, unter der Sie das Asset-Paket veröffentlichen möchten (die Sie über Ihr Publisher-Dashboard eingerichtet haben sollten), und dann einen Anzeigenamen und eine Versionsnummer angeben. Dann geben Sie einen Anzeigenamen und eine Versionsnummer ein. Auf der Grundlage des Anzeigenamens und des verwendeten Publisher-Kontos wird dann eine Paket-ID für Sie erstellt, die Sie jedoch nach Belieben ändern können, solange sie das umgekehrte URL-Format beibehält. Anschließend müssen Sie die Art des zu erstellenden Asset-Pakets auswählen:

  • Asset: Dies ist eine Sammlung von Assets (sprites, scripts, Objekte, etc...), die Sie auf dem Marktplatz verkaufen möchten
  • Demo: Dies ist eine Demo zur Demonstration eines Konzepts oder einer Spielidee
  • Tutorial: Das hochgeladene Objekt ist ein Tutorial, das mit GameMaker IDE ausgeführt werden kann (siehe diese Seite für weitere Informationen).

Danach können Sie im rechten Bereich auswählen, welche Assets aus dem Projekt Sie dem Asset-Paket hinzufügen möchten. Sie können alle oder nur einige Ihrer Assets hinzufügen oder auch solche entfernen, die Sie versehentlich hinzugefügt haben. Es wird vorausgesetzt, dass Sie die hochzuladenden Assets zuvor in GameMaker erstellt haben und dass das aktuelle Projekt alle Assets enthält, die Sie hochladen möchten.

MERKEN SIE SICH DAS! Um Anlagenpakete erstellen oder verwenden zu können, müssen Sie zuvor ein Benutzerprofilzertifikat erstellt haben. Sie können dies über die Marketplace Einstellungen tun, die hier erklärt werden.

Wenn Sie mit der Auswahl der Assets zufrieden sind und Ihr Paket auf Marketplace hochladen möchten, klicken Sie einfach unten auf die Schaltfläche " OK ". Das Asset-Paket wird erstellt und auf Marketplace hochgeladen. Nach Abschluss des Uploads wird eine neue workspace mit einer Browseransicht der Asset-Seite geöffnet, die Sie bearbeiten können. Weitere Informationen zur Einrichtung dieser Seite finden Sie auf der folgenden Seite:

  • Auflistung einer Anlage

Wenn Sie ein Marketplace Asset aktualisieren müssen, gehen Sie mehr oder weniger genauso vor wie bei der Erstellung des ursprünglichen Pakets, nur dass Sie jetzt das zu aktualisierende Paket aus einer Dropdown-Liste der verfügbaren Pakete auswählen müssen:

Bei der Aktualisierung können Sie die Paket-ID nicht ändern, und Sie sollten sich bewusst sein, dass die angezeigte Versionsnummer die aktuelle Versionsnummer ist und Sie diese gegebenenfalls erhöhen sollten, da die Aktualisierung sonst fehlschlägt (da Sie nicht zwei Asset-Pakete mit derselben Versionsnummer haben können). Es gibt auch ein Feld, in dem Sie einen Text für das Änderungsprotokoll für das Asset hinzufügen können. Dieser wird zusammen mit dem Asset hochgeladen und auf der Seite Marketplace angezeigt.

WICHTIG! Assets, die mit GameMaker Version 2.3+ erstellt wurden, stehen nur denjenigen Benutzern zur Verfügung, die Version 2.3 und höher verwenden. Wenn Sie ein Asset aus einer früheren Version aktualisieren, ist dieses Asset für Benutzer, die nicht aktualisiert haben, weiterhin verfügbar und wird mit der aktualisierten Version 2.3+ koexistieren.

Hochladen durch die Webseite

Sie können Assets über die eigentliche Webseite Marketplace und nicht über IDE hochladen, was notwendig sein kann, wenn der Upload über IDE aus irgendeinem Grund fehlgeschlagen ist. Zunächst erstellen Sie das Paket und versuchen, es wie oben gezeigt über IDE hochzuladen. Wenn der Upload fehlgeschlagen ist, gehen Sie zum Ausgabefenster und suchen im Protokoll nach einer Zeile, die anzeigt, wo das erstellte Paket vor dem Uploadversuch gespeichert wurde. Zum Beispiel:

Unter Windows:

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

Unter macOS:

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Sie müssen zu diesem Dateispeicherort navigieren und die Datei *.YYMP an einen sicheren Ort kopieren, da dies die Datei ist, die Sie auf Marketplace hochladen werden.

Zu diesem Zeitpunkt müssen Sie Marketplace in Ihrem Webbrowser öffnen und zum Asset-Listing für das Asset navigieren. Gehen Sie dann zum Abschnitt Versionen und klicken Sie auf die Schaltfläche Version hochladen, wie in der Abbildung unten gezeigt:

Dadurch gelangen Sie auf die Seite zum Hochladen der neuen Version, wo Sie bei Bedarf Informationen über die hochzuladende Version hinzufügen und die Datei bereitstellen können, indem Sie auf die Schaltfläche Datei auswählen klicken und die zuvor erstellte Datei *.YYMP auswählen:

Wenn der Upload abgeschlossen ist und Sie mit den Details zufrieden sind, können Sie auf die Schaltfläche Speichern klicken, um das Asset im Verzeichnis zu speichern. Beachten Sie, dass sich das Asset einige Minuten in der Bearbeitungsphase befindet, bevor es veröffentlicht werden kann.
