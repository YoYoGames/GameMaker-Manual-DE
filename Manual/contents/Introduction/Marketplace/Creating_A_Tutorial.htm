Erstellen einer Marketplace Anleitung

Die Erstellung eines Tutorials ähnelt der Erstellung eines regulären Marketplace -Asset-Pakets, nur dass Sie jetzt bestimmte Dateien hinzufügen und eine erste Tutorial-Seite (und optional eine Video-Seite) einfügen müssen. Das Tutorial workspace sieht in etwa so aus:Wie oben bei den allgemeinen Marketplace Assets beschrieben, müssen Sie einen Herausgeber auswählen und dann Ihrem Tutorial einen Namen und eine eindeutige ID (umgekehrtes URL-Format) geben und dann in der Dropdown-Liste "Typ" Tutorial auswählen. Daraufhin erhalten Sie zusätzliche Optionen zum Hinzufügen einer Basis-HTML-Datei, mit der das Tutorial beginnt, sowie einer (optionalen) Videodatei und einer URL. Die Videodatei sollte die von Marketplace gelieferte Vorlage verwenden - wie auch der Rest des Tutorials -, die Sie hier erhalten können. Wenn der Benutzer das Tutorial lädt, wird das Video in einen separaten Frame innerhalb von IDE geladen. Beachten Sie, dass die Seiten, die Sie für die Startseite und die Videoseite angeben, relative Pfade innerhalb der enthaltenen Dateien sind, und Sie können einen Explorer öffnen, um etwas anderes als die Standardpfade zu wählen, mit denen GameMaker die Felder auffüllt.

Für die eigentliche Erstellung Ihres Tutorials müssen Sie Ihrem Tutorial-Projekt eingeschlossene Dateien hinzufügen, und diese müssen auf eine ganz bestimmte Weise hinzugefügt werden, damit IDE richtig erkennt, welche Dateien für das Tutorial verwendet werden sollen. Wir haben also zwei Hauptordner:

  • Tutorial-Inhalt: Alle Dateien Ihres Tutorials HTML, die CSS für die Gestaltung und die dafür benötigten Bilder (nicht die für die Ressourcen benötigten Bilder, sondern die, die im Tutorial angezeigt werden sollen) sowie die Video-Vorlage, falls erforderlich, gehören hierher. Sie können auch Unterordner anlegen, wenn Sie das Lernprogramm z. B. in Kapitel aufteilen möchten, aber Sie sollten mindestens eine Basisdatei HTML haben, die als Startseite für das Lernprogramm verwendet wird.
  • TutorialResources: Dieser Ordner enthält zwei weitere Unterordner für Bild- und Tonmaterial . Wenn Ihr Lernprogramm erfordert, dass der Benutzer ein sprite hinzufügt (zum Beispiel) und Sie es in das Lernprogramm aufnehmen möchten, dann würden Sie die Bildressourcen in den Ordner "Images" legen. Diese Ordner werden in das Lehrgangsprojekt importiert, und wenn der Benutzer den Editor sprite öffnet und ein Bild hinzufügen möchte, wird der Standardpfad für das Explorer-Fenster auf diesen Speicherort verweisen, was es dem Benutzer erleichtert, die erforderlichen Elemente zu finden.

Damit Ihr Lernprogramm innerhalb von IDE funktioniert , müssen Sie diese Ordnerstruktur sowie eine HTML -Startseite mit einer CSS -Datei für das Styling haben. Es gibt eine Reihe spezieller Links, die in Ihren Tutorial-Projekten verwendet werden können, um zu Teilen eines Videos zu springen oder das Handbuch zu einem bestimmten Abschnitt oder einer Funktion zu öffnen. Diese sind alle in den offiziellen Tutorial-Vorlagen von Marketplace enthalten.

WICHTIG! Wir empfehlen Ihnen dringend, die Tutorial-Vorlagen aus dem Marketplace herunterzuladen, bevor Sie versuchen, Ihre eigenen Tutorials zu erstellen. Sie enthalten Beispiele für alle wichtigen Teile des HTML sowie CSS und Bilder zur Erstellung von Schaltflächen und Menüs innerhalb des Tutorials, so dass Sie sich um den Inhalt kümmern können und nicht darum, wie er dargestellt wird.

Sobald Sie die enthaltenen Dateien eingerichtet und die erforderlichen Informationen eingegeben haben, können Sie auf "OK" klicken und das Lernprogramm wird auf Marketplace hochgeladen. Wenn Sie fertig sind, öffnet sich ein workspace auf der Asset-Liste, damit Sie es bearbeiten und Symbole, Text, Screenshots usw. hinzufügen können. Sobald das Tutorial-Asset veröffentlicht wurde, können die Benutzer es im Abschnitt Tutorials auf der Startseite finden, wo es heruntergeladen und installiert werden kann.
