Fehlersuche

Bei der Programmierung Ihres Spiels, ob mit GML Code oder GML Visual, können sehr leicht Fehler gemacht werden - die Verwendung der falschen Variablen, die Übergabe der falschen arguments oder die Verwendung der falschen Funktionen sind nur einige der häufigsten Fehler, die wir alle machen - und diese Fehler werden nicht immer von der Syntaxprüfung erkannt, die im Skript/Aktionseditor enthalten ist. Das bedeutet, dass diese Fehler erst sichtbar werden, wenn Sie Ihr Spiel tatsächlich ausführen oder kompilieren, und selbst dann kann es manchmal schwierig sein, genau zu wissen, was falsch läuft. Es kann daher von entscheidender Bedeutung sein, zu wissen, wie sich Ihr Spiel auf dem gewählten Gerät verhält, und zu sehen, welche Instanzen sich im Raum befinden, welche globalen Variablen erstellt wurden und eine ganze Reihe anderer Details. Das Überprüfen dieser Dinge wird als "Debugging Ihres Codes" bezeichnet, und alle Fehler oder Probleme, die Sie finden, werden "Bugs" genannt.

Sie können viele dieser Dinge selbst überprüfen, indem Sie so genannten "Debug-Code" in Ihre Spiele einbauen, der die gewünschten Informationen auf dem Bildschirm anzeigt, oder Sie können dafür sorgen, dass bei einem Fehler Meldungen angezeigt werden, aber das ist oft eine Menge Arbeit, und wenn Sie sich nicht sicher sind, wo das Problem überhaupt liegt, kann es eine zeitraubende Angelegenheit von Versuch und Irrtum sein, Ihren Fehler zu finden. Dafür hat GameMaker einen speziellen Testmodus, den Debug-Modus, der das Debug-Modul startet, und es gibt auch ein umfassendes Fehlerberichtssystem, das beim Kompilieren Meldungen anzeigt, die Ihnen helfen, Probleme mit Ihrem Spiel aufzuspüren.

In den folgenden Abschnitten werden die verschiedenen Möglichkeiten zur Fehlerbehebung in Ihrem Spiel beschrieben:

Fehlerberichterstattung

Wenn während der Kompilierung und Ausführung eines Projekts ein Fehler auftritt, wird dies mit einer Meldung im Ausgabefenster für Kompilierungsfehler gemeldet, ähnlich wie in der folgenden Abbildung:

Die meisten compiler Fehlermeldungen folgen dem gleichen Schema, mit einer Meldung, die das Objekt angibt, in dem der Fehler aufgetreten ist, dann das Ereignis zusammen mit der Zeilennummer und schließlich eine kurze Beschreibung des Fehlers selbst. Diese Art von Informationen, die von der Fehlermeldung bereitgestellt werden, ist sehr wichtig, da sie die syntax und sprachbezogene Fehler erkennt und meldet, die Ihr fertiges Spiel zum Absturz bringen werden, aber sie meldet keine subtileren Fehler und gibt auch kein Feedback zur Leistung. Für diese Art von Problemen sollten Sie den Debug-Modus verwenden (siehe unten).

Abgesehen von diesen Kompilierfehlern können auch syntax Fehler auftreten. Diese werden erkannt, wenn Sie Ihre GML Visual Actions oder code hinzufügen, und werden im Ausgabefenster für Syntaxfehler angezeigt, ähnlich wie in der folgenden Abbildung dargestellt:

Wie bei den Fehlern von compiler verweist das Fenster syntax auf das Objekt, das Ereignis und die spezifischen Zeilen, die das Problem verursachen, und wird oft mit einer kurzen Erklärung des Fehlers angezeigt.

Debug-Modus

Wenn Sie die Dinge genauer überprüfen möchten, können Sie das Spiel im Debug-Modus ausführen, indem Sie auf die Schaltfläche Debug Run klicken. In diesem Fall wird das Debug-Modul zusammen mit dem Spiel geöffnet. Unter module kannst du die Funktionsweise deines Spiels überwachen, von der globalen Leistung bis hin zu den Werten einzelner Variablen in einzelnen Instanzen. Weitere Informationen über das Debug-Modul Module, seine Möglichkeiten und seine Verwendung finden Sie auf der folgenden Seite:

  • Der Debugger

Beachten Sie, dass Sie bei Verwendung von Debug Module auch Haltepunkte zu Ihrem code oder GML Visual hinzufügen können. Ein Haltepunkt ist ein Punkt im Spiel loop, an dem Sie möchten, dass das Debugging module pausiert und Sie zeilenweise durch das eigentliche Spiel code gehen können. Sie können Haltepunkte mit der Taste "F9" hinzufügen und sie werden im Ausgabefenster angezeigt, wo Sie sie nach Bedarf aktivieren/deaktivieren können.

Zusätzliche Tools

Als zusätzliches Werkzeug für debugging kompilierte Spiele gibt es bestimmte Kommandozeilenparameter, die verwendet werden können, wenn Sie das fertige Spielpaket auf Windows ausführen. Informationen darüber, wie dies funktioniert, finden Sie hier.

Sie können auch ein Debug-Overlay generieren, wenn Sie Ihr Spiel testen, das zusätzliche Debug-Informationen über die Leistung Ihres Spiels während der Ausführung anzeigt, und Sie können auch Debug-Meldungen an die Kompilierkonsole ausgeben, wo Sie Ihren eigenen Text eingeben können, um das Verhalten Ihrer Spiele zu debuggen.
