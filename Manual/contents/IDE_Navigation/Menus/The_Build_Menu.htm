Das Menü Bauen

Im Menü "Build" können Sie wählen, ob Sie Ihr Projekt zum Testen, für debugging oder als endgültige ausführbare Datei erstellen möchten. Die verfügbaren Optionen sind:

  • Ausführen - Führen Sie Ihr aktuelles Projekt auf dem gewählten Ziel platform zum Testen aus. Für die verschiedenen Kompilieroptionen, die beim Kompilieren zur Verfügung stehen, lesen Sie bitte den Abschnitt über Kompilieren. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche Ausführen in der IDE.
  • Debuggen - Führen Sie Ihr aktuelles Projekt zum Testen mit dem Debugger aus, damit Sie detaillierte Leistungsinformationen sehen und eventuelle Fehler oder Probleme beheben können. Weitere Informationen zu debugging finden Sie im Abschnitt Debugging. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche Debuggen in der IDE.
  • Bereinigen - GameMaker speichert Dateien im Cache, um die Kompilierungszeit beim Testen zu beschleunigen. Manchmal kann jedoch ein "veralteter" cache zu seltsamen und unerwarteten Fehlern in Ihrem Spiel führen. In diesem Fall sollten Sie als Erstes die cache von hier aus bereinigen und erneut testen. Es ist auch eine gute Idee, die cache zu bereinigen, bevor Sie eine endgültige ausführbare Datei für das Ziel platform erstellen. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche " Cache bereinigen " in der IDE.
  • Executable erstellen - Damit wird das aktuelle Projekt kompiliert und ein ausführbares Paket (oder Pakete) für das Ziel platform erstellt. Zunächst wird der Datei-Explorer geöffnet, damit Sie den Speicherort für das Projekt auswählen können, und dann wird das Spiel kompiliert. Weitere Informationen zum Einrichten der Kompilierziele finden Sie im Abschnitt über das Kompilieren. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche " Ausführbare Datei erstellen " in der IDE.
